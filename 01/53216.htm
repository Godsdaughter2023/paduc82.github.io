d_blade say:

AFAIK you cannot create extended partitions if you use GPT partition table (common on EFI systems). GPT only allows primary partitions (up to 128 I think). Thus, you cannot create logical partitions either, because they can be only created within extended partitions.

The solution with extended partitions is for systems like BIOS who cannot have more than 4 primary partitions and therefore needed that workaround since Windows is commonly taking 2-4 primary partitions for itself (one for recovery, one for MBR, one for the OS itself and one for user data).

But that does not matter to Linux though. Linux can run on any partition while Windows needs to be on a primary partition.

d_blade say:

As mentioned before, it does not matter to Linux whether its installed on an extended or on a primary partition. It only matters for Windows (if you were to install Windows on that partition). So what really matters is if you have MBR or GPT (which decides if you can create only 4 or an infinite number of primary partitions. If you can only create 4, you would need extended partitions as the other solutions suggest.)

I am almost certain that you have GPT, but well, we can check that.

On Windows:

Use the command line (press Windows button and type "cmd" and then press enter)

A command line window should open now.

Type now "diskpart" which may either open a new window or will continue in the command line window.

Type "list disk" there.  This will show you a table of your harddisk and whether they are GPT or not. It may look like this:

Disk ###  Status        Size    Free    Dyn Gpt
--------  ------------- ------- ------- --- ---
Disk 0    Online        200 GB   20 GB   *
Disk 1    Online        200 GB   20 GB       *

In this case, Disk 1 is GPT.

On Linux:

In terminal, type:  "parted - l" or "sudo parted -l"

This will show you a list of your hard disks and some information. Look for the line "Partition Table". It will probably say either "msdos" (=MBR) or "gpt".

AFKAIK, the installation of Manjaro on primary partitions should be no different than installing it on an extended partition. So there should be no issues there.

One noteworthy thing: Linux wants ideally 3 partitions for installation. When I tried Linux for the first time, I had some trouble understanding this because Windows makes it a lot easier for the average user.

Typical for Linux are a partition for root - also called "/" - for system relevant data or programs, a partition called "/home" (or similar) for your personal data (videos, documents, games, etc.) and a third partition for "swap" which is significantly smaller. The first two partitions are common, while a swap partition is kinda optional. Swap partitions are used to save or move data from your RAM to your Swap partition. This was common at a time, where computers had less than 1GB ram and swap partitions avoided freezing screens. Nowadays you dont really need them, because most PCs come with 4GB or more. If you are still worried, you can also create a swap-file (basically a file which act as a swap partition). In fact, if you are really lazy, you can also use one single partition for all system and personal data (but this is not recommended). (As for me, I only use 2 partitions, one for root and one for home).

If you are wondering how much GB you need, I suggest 40-50 GB for root and the rest for your personal home data. Optionally a swap partition as big as your RAM.
