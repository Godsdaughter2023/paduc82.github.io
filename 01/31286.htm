I suspect that you have a file corruption issue that's prevent ing Normal Mode boots.
Try this....
Boot to Safe Mode with Command Prompt
at the prompt type
EVENTVWR
hit the Enter Key
This should bring up the Event Viewer.
Look in each of the available Logs - one MAY fail with a 'corrupt file' error. If so, then we need to delete or rename the appropriate file. Make a note of which one fails.
If there's no failure, then you can skip the rest of this post.
 
Change the prompt location to the required one, using the following command at the prompt
 
CD   C:\WINDOWS\system32\config
 
Just to check you're at the right place, do a DIR command as follows
 
DIR   *.EVT
 
 - it should show three files
 
AppEvent.evt
SecEvent.evt
SysEvent.evt
 
It's fairly obvious which file relates to which log corruption from the names - once you know which log to rename, use the following command  to rename it (we'll rename it, because it may have useful information in it, for later forensics, if you feel inclined <g>)
 
REN  xxxEvent.evt   xxxEvent.evt.OLD
(replace the xxx with the appropriate three-letter sequence for the log)
 
hit the Enter key - it may ask for confirmation, accept it.
 
Then reboot and see if that's helped
 
Good Luck

Hi Rhuari,

I understand that you want to move programs from SSD to your SATA Hard Disk Drive (HDD). Ideally, one needs to install the programs afresh on the other drive so as to ensure smooth program performance. However, there is a workaround where you can copy the system image to the other drive.

So, you can create a System Image backup and Refresh your pc with that System Image disc. This process will use the System Image from the disc instead of default System Image saved on Windows Recovery partition.

By default, desktop apps are removed when you refresh a Windows 8-based computer, unless you create a custom image. After you create a custom system image, the image is used as the refresh image. This means that any existing image or OEM restore image is not used when you refresh your computer.

You may refer to the article below to create a System Image to refresh your system.

How to create a system Image to refresh your Windows 8 PC

(Article holds good for Windows 8.1)

Note: Though Creating System Image processes normally does not cause data loss, but, it is highly recommended to back up all your important data on any external drive before trying any of the suggestions above to create system image or to Refresh your PC.
