Hi,

I don't know what the OEM partition is all about... It is only 40mb and not good for much... It is located in a place where a GPT File Table is usually located when doing a Clean Install of Win 10 to a blank HDD...

They are not usually visible in Disk Management though... Only with 3rd party tools.. I have never seen a 40mb one... Usually 16mb or 128mb...

The first 500mb Recovery partition is located where a Clean Install of Win 10 will put it, but the 500mb seems a little too big... They are usually something like 350mb to 450mb...

You have a 350mb Recovery Partition that my guess would be was the 1st with the 500mb's added later...

Regardless... It is only safe to delete the 510mb and 350mb partitions and resize your OS partition to fill in the space... Deleting the 1st 500mb partition would not gain much, and it could be a disaster trying to use the space if you did delete it...

The only real problem with deleting the Recovery partition that is being used is that you would need a Recovery CD\Flash, or Installation Media to do some repairs...

I have noticed that the latest Insider Preview 17063 did not create one of these as usual... I don't know if Microsoft has fixed this problem, or if it is a bug...

I also wish I knew a way to determine which Recovery was the newest for sure... I do know this though... Whenever I do an upgrade the new Recovery Partition is always located right after the OS partition, and the space is actually taken from the OS partition even though there is a DATA Partition after the OS partition that could have been used...

I always delete the Recovery Partition that was created by the upgrade and Resize the OS Partition back to it's original size... Mostly so that backup images will fit in the OS partition without needing resized themselves...

I have suffered no significant issues with this other than what was mentioned...

If you want to get rid of yours the best thing to do would be to download a ISO Image of a bootable Disk Management program and burn it to a CD or Flash Drive...

Parted Magic_ pmagic_2013_08_01 and pmagic_2015_09_29 will work...

The newer versions are no longer free...

Partition Wizard_ pwfree91 and earlier will work... The free Version 10 is not bootable anymore...

Partition Wizard may be the easier of the two to use... More like using Windows..

I found the cause of the problem, at least in my case.  The Microsoft Virtual WiFI MiniPort Adapter uses the same MAC address as my hardware wireless adapter.  I use the Shrew Soft VPN client (64-bit support for Windows 7) and found, after a couple of hours of testing, that if I turned off the Shrew DNS proxy daemon service, my wireless adapter went back to using DNS.  After some searching on the Internet about the miniport adapter, I could see that there are a lot of symptoms associated with this adapter.  In my case, disabling the miniport adapter returned my machine to normal operation.  The funny thing is I run ipconfig a lot in my job, and I just don't remember seeing that adapter in my list before this week.
