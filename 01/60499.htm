suat.cini say:

I had this same exact problem with Windows 7, Windows 8. Windows 10 is no exception. But this problem is not Windows related according to my investigation. Not only AMD cards but Nvidia cards are also affected.

The new-generation gfx cards have several operating states, such as low-power state, high-power state. etc..

While you are doing something on your computer, gfx card switches from one power state to another. That is when you have the TDR issue.

I have an ATI card. The following is a screenshot from ATI power states.

The above file is a power state profile file in 

%userprofile%\appdata\Local\ATI\ACE  folder. I played with this file to change power states to avoid TDR issue up until I had a gBIOS file from my card manufacturer. After that I have no TDR issue. The gBIOS update changed the memory clock to 1100 MHZ constant and the core clock to 300 MHz low and 860 MHz high.

Want_0 value is the low-power state for GPU core of my gfx card. It is 100 MHz. ( Note after gBIOS update, my card does not use this profile file. )

Want_1 value is mid-power state for GPU core. It is 600 MHz.

Want_2 value is the hi-power state of the GPU core. It is 860 MHz.

Same is true for Memory clocks. They are all 1100 MHz now. They were different before I updated my gBIOS.

So the core and memory clocks change from one state to another constantly depending on your usage. Even when you are playing a game, there are times of low activity and high activity moments causing the gfx card to switch power states. This state change sometimes causes TDR.

I hope this clarifies the TDR issue.
