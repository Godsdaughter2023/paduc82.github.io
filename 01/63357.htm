a. user say:

Hello Mark,

Thank you for posting in Microsoft Community.

I understand that the live images are not opening in Windows PC. We will help you.

Extracting the Living Image Video
Copy the .jpg file to your PC using a file explorer. Change the extension of the .jpg to .zip. Now open it using extractor. You will see that it is indeed an archive zipped file when you do this.

You will see 2 initial entries. A file called "[Content_Types].xml" and a folder called "formats". Open the "formats" folder.

Inside the formats folder is another folder called "living", so let's open that folder as well.

Inside the living folder are 2 files.
1. living.jpg
2. living.mp4

The living.jpg is "end picture" that the living image is supposed to show AFTER playing the video portion. It is the same resolution as the video itself.
The living.mp4 is the living image video.

Extract both files, and you can use the much smaller living.jpg, rename it to the original .jpg file name, and save a lot of space in your pictures folder. You can rename the living.mp4 to the original filename but with the .mp4 extension and put it in a living images folder of your choosing. You can play the .mp4 file on your PC or phone either way.

Hope it helps. Keep us posted to help you better.

jeff d - windows camera team say:

Here's most likely what you're seeing:

The Photos app has a setting called "Show my cloud-only content from OneDrive." When this setting is On, the app will show *thumbnails* from OneDrive rather than the full JPG file.

In order to see living images play back on your PC, you need to SYNC your OneDrive camera roll folder into a PC folder that is INCLUDED IN YOUR PICTURES LIBRARY. This gets the full file onto your PC in a location the Photos app can see it, which will allow the app to retrieve the living image video that's embedded in the file.

This has nothing to do with the apps being universal or not.

The issue is how the Photos app is integrated with OneDrive--by design it pulls down thumbnails only and not the living image portion. This is because there currently isn't a way to query whether the underlying image has an associated living image.
