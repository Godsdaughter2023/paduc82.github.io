Crazy Cursor with GPS plugged in - Microsoft Community

hunterwanderer say:

Crazy Cursor with GPS plugged in


I am running Windows 7 and when I restart my computer with the GPS plugged in the cursor goes crazy .

I have loaded the latest garmin driver and driver for the new serial port to usb converter. ( I have just bought as the previous one was not compatible with windows 7 ). However the problem still persists.

I did not have this problem with XP or vista. 

otldevouterlink say:

This behavior is cause by the GPS being mis-identified as a modem or serial trackball or serial mouse. The mis-identification is done in the module serenum.sys, a plug-and-play device enumerator that is invoked by default by most serial drivers, including the Windows default serial drivers. Among other things, this module detects incoming data to try to identify various legacy serial devices using identification techniques that were devised in 1995 for pre-existing modems and pointer devices. See: http://technet.microsoft.com/en-us/library/cc767877.aspx for instance, or http://msdn.microsoft.com/en-us/library/windows/hardware/ff547451(v=vs.85).aspx.

 

The most discrete way to resolve this is to alter the .inf file for your serial driver to specify that serenum.sys should not be invoked during driver initialization for your particular device. Unfortunately, as far as I know, there is no standard way to do this. The option may or may not be available for your particular driver.

 

There are workarounds for disabling serenum.sys in earlier versions of Windows, but Windows 7 is so good as "repairing" itself, that it defeats many of these old techniques.
