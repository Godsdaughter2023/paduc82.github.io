Terminal PC WIndows 7 32 bit Professional

Remote PC WIndows XP Pro

bob.wiley say:

I could not figure out how to get the LPT1 and LPT2 ports to automatically show up on the RDP connection, but I did find a workaround.

Scenario:
New Windows 7 Pro machine in satellite office that must connect to XP Pro Machine in main office to run a program, the pogram is configured to print a certain part to an LPT port which can not be altered as it would mess up the software config in the main office.

What I did:

I noticed that even though the LPT ports weren't passing through the RDP session, TS ports were passing through as selectable ports in the printer properties box as TS01, TS02, TS03 ... etc., it appears that the TS## corresponds to the position the printer holds in the originating PC Printers & Faxes list (Win7 in this case).

Win7 PRO (satellite office)
Printers are installed to LPT ports, LPT1:Epson LQ-2550 (actually printing to OKI ML-390, I couldn't find a Win7 driver), LPT2:Brother Laser

WinXP Pro (main office, connecting to with RDP from satellite office)
Manually installed Epson LQ-2550 to port TS01 and Brother to TS02
Both printers printed documents originating from the XP machine to the 7 machine ... BUT, the software application is programmed to print certain documents to LPT1 port ... so I shared the Epson LQ-2550 on the XP machine and gave it a share name of "oki" and created a NET USE batch file on the desktop of the XP machine, when I log in from the Win7 machine I run the batch file.

      NET USE LPT1: \\127.0.0.1\oki /persistent:yes

All is well!

The following solved my "Checking on Updates" after Windows 10 Upgrade:

With this problem "command" was not accessible. Forced power down just repeated the problem after a few finishing-installation prompts. So, disconnect the network cable and:

1.  Install Windows 7 Repair/ Recovery Disc that was created when Windows 7 image was created just prior to the       attempted Windows 10 upgrade.

2.  Force power off by holding power on/off switch until the PC finally gives up.

3.  Power on the PC and press F12 to change the boot order to CD/DVD.

4.  Boot the CD.

5.  Choose the Windows 10 operating system after the machine warns you and reboots again.

6.  Choose "Repair".

7.  My new Windows 10 booted correctly afterwards.
