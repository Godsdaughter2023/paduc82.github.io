myronh say:

"...duplicate Program Files Folders (with and without x86 designation) have been created." That is one folder each (Program Files Folder x86 and Program Files Folders)? That would be correct.

"Does this mean that these duplicate Windows apps are needed for 32 bit and 64 bit software to be able to work with that type of software respectively?" In general, that's what it means. However, some of the programs just provide the opportunity to use the x64 version (such as IE 64). Some of the applications, such as IE and mail, are separate applications and can be deleted. If you want to uninstall one of the IE versions, remove the 64 bit version because flash doesn't come in 64 bit yet and therefore you can't get the full functionality. However, that will eventually change.

Many of the applications you mentioned are part of windows and don't uninstall separately, therefore you are stuck with both...since you now have a 64 bit OS. As more and more 64 bit applications come available, this will be a definate advantage. But if you are stuck with limited resources, particularly disk space, you might have been better staying with the 32 bit.

"It would have been nicer just to have one set of these Windows applications that works with both 32 bit and 64 bit software." In theory that would be great...unfortunately the architecture of the two are completely different. 64 bit can work with 32 bit, but not the other way around.
