a. user say:

Both excellent questions.

1) If possible you should make a backup image of your computer immediately after it finishes setting itself up, after that very first time you push the power button.

A backup image is an exact copy of your hard drive, in effect, an exact copy of your computer. There are severall advantages to doing this: a) should you need to return your computer to the way it was when you first brought it home, it's much faster to restore a backup image than it is to reinstall everything all over again; 2) a backup image is actually a single file, albeit a big one, and you can (you should) store that file outside your computer, so if your computer ever becomes unusable for whatever reason, you still have the image; and 3) you can remove the manufacturer's recovery partition and use its disk space for something else.

You do this with image backup software. In fact, image backup software makes it real easy to make an image backup every day, which is something you should definitely do. For more on that topic, see the next question:

2) Windows has a built-in image backup utility called System Image Backup. However, it was deprecated, meaning Microsoft abandoned it, more than a year ago as part of the Fall Creators Update v.1709. In fact, Microsoft explicitly recommends that people use a third-party image backup program. I use Macrium Reflect, but there are other image backup programs that you may prefer.

Backup and Restore (Windows 7) is the same backup program that was included in Windows 7. It was also included in Windows 10 in case you used that to make a backup in Windows 7 and now you want to restore that backup in Windows 10. It wasn't modified to be used for making new backups in Windows 10 and it isn't supported for that purpose. So like all other unsupported software, sometimes it works and sometimes it doesn't.

File History can be used to backup individual files and folders - you can't use it for an image backup - but it's awkward to use for copying all your personal data files and folders because it wasn't really built for that.

An image backup program like Macrium Reflect and others like it can also backup individual files, but you'll be happier with a real backup program, which has convenience features that run rings around File History.

Windows 2008 R2 Boot Problem - Microsoft Community

hammercr say:

Windows 2008 R2 Boot Problem

I have restored a BackupExec backup (both data and system state) from a server running Windows 2008 R2 to new hardware. New hardware contains PERC H710P RAID controller and Windows installation media does not contain a driver for it. So, my restore of system state wipes out my freshly installed system with PERC driver and system refuses to boot.

I have not been able to find a way to inject PERC drivers into the Windows 2008 R2 (which is the same as Windows 7-64) boot environment. I have tried building a modified Windows ISO with PERC drivers hoping its recovery option would be smart enough to add the new driver to boot environment. Not only it did not seem to be able to change boot sequence, it did not even load the PERC driver and I still had to load from a flash drive.

Does anyone have any ideas on how to add driver support during boot to a system that is already installed?

Thanks,

HammerCR

hammercr say:

This was solved yesterday. Here is an outline of the steps:

  1. Download the latest driver off of support.dell.com and extract the files onto a USB key.
  2. Insert the USB Key into the Server.
  3. Boot to the Windows OS Media and use "Shift+F10" to launch a command line window.
    1. It might be a good idea to use two windows, to launch diskpart in to get the Volume Letters
    2. diskpart.exe and then run "list volume" to view the volumes
  4. Run the following command:
    dism /image:C:\ /add-driver /driver:D:\percsas3.inf
    Note: C:\ is assumed to be the Windows System Drive Partition and D:\ is the USB Key.
  5. Once the driver has been loaded, exit out of the Windows Installer and reboot the system. The system might boot into the recovery utility, but choose to boot into Windows 2008 R2. System will reboot and you will be presented with a log-in screen.

Hopefully this will be useful to someone.
