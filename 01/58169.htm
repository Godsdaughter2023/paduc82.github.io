Assuming your BIOS set to EFI mode, and your Harddisk is on GPT.

Then the steps:

  1. Boot into USB
  2. Run Command prompt from Windows installation(You don't need to click Repair this computer - just press Shift-F10). Then run diskpart
  3. From diskpart>, run list volume to see all volumes you have. There will be one FAT32 volume which is 100MB,may have the name EFI Partition(no name in my case)
  4. Run select volume x , where x is the volume number of your 100MB one.
  5. Run assign letter=<Driver letter>: (recommend using O N or M for <Drive letter>, avoiding first 10 letters in Alphabet), ex: assign letter=O:
  6. Run exit to quit diskpart, get you back to commandprompt
  7. On EFI system, bcd store file should be located in O:\EFI\Microsoft\Boot\BCD (note this BCD is a file without extension) the tutorial is on MBR system, explaining your steps failed. Run cd /d O:\EFI\Microsoft\Boot, now your working dir should be O:\EFI\Microsoft\Boot>
  8. Run attrib BCD -h -r -s  (Not need to worry, I didn't backup bcd with export command but surely the next step will backup your bcd for real)
  9. Run ren BCD oldBCD.bak
  10. Run bootrec /rebuildbcd, this time, the process should give you at least 1 installation. Just YES or ALL to save the process.
  11. Now, close Command prompt as well as Installation windows, it will restart your system. DO NOT HARD RESET by Power button or reset button, it will kick you back in Automatic repair loop again as your volumes suffer "sudden stop" which requires chkdsk.
  12. You should be back to login screen now
