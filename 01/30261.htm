From the details you have provided, it indicates there is some problem with the partition you were using. Probably, the partition which is missing would have bad sectors. As suggested in error logs I would suggest you to perform Disk checking which will attempt to scan and fix any bad sectors in the partition space if any.

Please see the steps below to check a drive for errors

1.     Open Computer by clicking the Start button, and then clicking Computer.

2.     Right-click the drive that you want to check, and then click Properties.

3.     Click the Tools tab, and then, under Error-checking, click Check now. If you're prompted for an administrator password or confirmation, type the password or provide confirmation.

4.     To automatically repair problems with files and folders that the scan detects, select Automatically fix file system errors. Otherwise, the disk check will report problems but not fix them.

5.     To perform a thorough check, select Scan for and attempt recovery of bad sectors. This scan attempts to find and repair physical errors on the drive itself, and it can take much longer to complete.

6.     To check for both file errors and physical errors, select both Automatically fix file system errors and Scan for and attempt recovery of bad sectors.

7.     Click Start.

Depending on the size of your drive, this might take several minutes. For best results, don't use your computer for any other tasks while it is checking for errors.

For more information please see http://windows.microsoft.com/en-US/windows7/Check-a-drive-for-errors

cmdrkeene say:

File History can't be browsed using the file system.  You should never try to browse these files using the normal file browser.  Use the File History interface only or you will corrupt your backups.  Even just clicking the "administrator continue" option to access the files can change the ownership and break your backups.

It can't be browsed in the file system, it can only be browsed using the File History interface, which reads the manifest file to determine which versions to display as you navigate forward and back in the timeline interface:

I didn't actually modify all these files on Sunday at 8:52 PM.  In fact, most haven't been modified in many months.  So although I see it on all the days in the timeline, there's really not this many extra copies of it.  It's intelligently using the hard drive by only storing extras when there's something new.  So if I went back in time to October 12th (the real time I last modified this PDF file), I could restore it to what it looked like on that day.  Or I can even open this file here and then navigate among the versions directly, rather than just browsing the timeline.

---

If (For one example) you just want to make an extra copy of your Documents on a thumb drive every day, you have a few options. I like File History, but maybe it's overkill for your needs.  But it has other benefits -- such as caching the backup until you plugin your drive next time so it can write all the backups.   You could use a batch file, set with Windows Task Scheduler to fire every day, but it would fail if the flash drive wasn't connected at that time (something File History would take care of for you).

You could write the batch file to make a folder with the current date, then copy the whole document folder into it, every day.  When the flash drive ran out of space you'd have to manually take care of it.  This would also cause a lot of duplication as I said above, so another option is to use ROBOCOPY command and only copy the files that have changed, overwriting the previous backup.

Another tool that does this job is one that has a strong following of enthusiasts, called Microsoft SyncToy.  Read up on that you may like it better than any of the above.

a. user say:

Hi Stephanie,

The concern usually occurs due to driver incompatibility as you have mentioned. You can run the program in compatibility mode to be able to use it. Simply follow these steps:

  1. Go to Start, then on the Cortana search bar, type in Control Panel. Hit Enter.
  2. A new window should appear. On the top-right corner of the window. Look for Search control panel bar.
  3. Type in Troubleshooting, then click on Troubleshooting on the results pane.
  4. Under Programs, click on Run programs made for previous versions of Windows.
  5. A new dialog box should appear. Follow the on-screen instructions.

For more options regarding this concern, please check this article out: Make older programs compatible with this version of Windows. In addition, you can run the Windows app troubleshooter to automatically fix incompatibilities with applications.

If you have any more concerns, don't hesitate to come back to us.
