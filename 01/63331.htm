Surface Pro 3 Cooling Issues - Microsoft Community

ashleysolutions say:

Surface Pro 3 Cooling Issues

          I have a Surface Pro 3 with a Core i7 processor and a 256 GB solid state drive.  I know many people have had issues with the Surface Pro 3 reaching unbearably hot temperatures.  I have had what I'm assuming are the same--if not very similar--issues where the right-hand side of the device (specifically the middle of the upper-right side of the display and the portion of the back directly behind it) become too hot to touch for more than a couple of seconds without being burned, and during which the CPU temperatures remain consistently around 80 degrees on the Celsius scale and can spike to just above 90 degrees.

          At first, the issue seems randomly intermittent at best, and eternally persistent at worse, but after experimenting with many different settings for the past few months, I believe I have discovered the source of the issue.  Whether this is accurate for the majority of people is yet to be determined, but I am 99% certain that it's the cause of the problem for me.

          The issue has been around since I first got my device about 15 months ago.  I only noticed it at first when I was doing work that should stress the CPU, so I payed little heed, and simply dismissed it as normal performance.  After a couple of months, I began to notice it getting hot when updating and doing somewhat heavy workloads like downloading large files or running multiple programs in Java at once.  As I began to notice the issue more and more, I started experimenting.  My first hypothesis was that using the surface pen would cause it to start overheating, and once it got hot, I found that it was necessary to shut down and wait for the device to cool off completely before it would regain its normal operating function.  While I have found that using the pen will cause the Surface to heat up more quickly than it probably should in some cases, it is not the source of the overheating.

          As of the time of writing, I believe that the issue stems from the Surface's "Battery saver" mode.  What appears to be the issue is that when using Battery saver, the device attempts to save as much power as possible by reducing the processes it's doing, dimming the screen, and being reluctant to run the fan unless it's absolutely necessary.  The problem with that is it's too aggressive with throttling the fan and it won't engage at a high enough speed until it's already too late for the fan to cool the device down again and then it needs to run the fan at its maximum speed just to keep the processor barely in the realms of safe temperatures.  Because of this, the battery life is reduced significantly because while it can save power in the short term, it has to work very much overtime once the processor heats up.  I've noticed that the battery actually lasts significantly longer (by two to three hours in some cases) when you leave it out of Battery saver mode the whole time you're using your device compared to if you have Battery saver on the whole time.  I have had the device dead from fully charged in an hour and twenty minutes one time while using Battery saver, while it normally lasts 4-5 hours without it.

          So assuming that this issue is not confined only to me, I would say that Microsoft has some serious optimization issues to look at in order to resolve the overheating epidemic that plagues their Surface Pro 3s.  If anyone else could verify this assessment, I would much appreciate it, and I would hope Microsoft would be able to use this information to help resolve the overheating issues people have been reporting.

csadoian say: You are probably running Windows XP.  For some time now there has been a real problem with computers running XP and Windows Update.  I work as an independent IT consultant and for the past few months on Microsoft's Patch Tuesday (2nd Tuesday of the month) I started getting people calling me complaining of their computers slowing to a crawl.  After I investigated the problem it turns out the SVCHOST process you mentioned is part of Window Update and is running at 99%, causing the slowdown.  If you bring up Task Manager and kill the task your computer will return to it's normal speed.  However, when you kill the task other Windows services will go down as well (such as audio).

I have searched the internet for a solution and, to the best of my knowledge, there is only one thing you can do.  Turn OFF Automatic Updates and DISABLE the Windows Automatic Update service, then reboot your system.  At that point your system will run normally, but you will not receive Windows Updates automatically, nor can you get them manually by using Internet Explorer.  However, it is a small price to pay to be able to use your computer again.

It's interesting to note this problem does not affect all computers running Windows XP, just some.  I have seen many XP computer's SVCHOST only go to 50% on Patch Tuesday.  But a great many will go to 99% and become unusable.  There are many so-called solutions out there for SVCHOST 99% (some dating back many years) but I have not found a solid solution for this problem that works every time.  I have built an XP system with a fresh install of Windows SP3, installed IE8 and the latest version of Windows Update and the machine will still go to 99% CPU usage when trying to do Windows updates.

I cannot believe Microsoft has not seen this problem.  I'm sure they have a few XP computers up there in Redmond that are doing the same thing.  Why they haven't offered a solution (or at least a workaround) is frustrating.

My theory is that Microsoft has moved much of their available resources (bandwidth, update servers, etc) to service their most recent products (Windows Vista, Windows 7, Windows 8, etc) which means the resources available to XP machines is getting pretty thin.  Unfortunately there is still a butt load of XP machines out there in the world, and they all call home on Patch Tuesday and want their updates.  Windows Update should be smart enough to let go of the CPU if it can't get an answer from the update servers, but for some reason on many machines it does not, hence the SVCHOST 99% CPU usage.

Hey Microsoft, are you listening?  Or is this a way to frustrate those remaining XP users into upgrading to Windows 8?
