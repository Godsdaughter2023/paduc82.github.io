I too have never come across anything like this and I can't tell you what's causing it. 

Is your everyday user account an "administrator" account or a "Standard user" account?  Change the IP address as you want, then log off your "administrator" account and log in as a user who is a "Standard user" and see if the problem recurs.  Because Standard users are not permitted access to a network connection's properties, software running in the Standard user account should not be able to make the change.

You might also want to start the computer in a "clean boot" configuration and see if you can determine what process is causing the change.  This method is tedious -- basically, you turn off everything but essentials and restart; if the problem goes away, re-enable half of the processes you turned off and restart again.  Rinse and repeat until you determine the culprit.  https://support.microsoft.com/en-us/help/929135/how-to-perform-a-clean-boot-in-windows

See the following articles for some additional ideas.  For best results, you should have an "administrator" account that you use "for emergency use only" so that you can exclude your everyday account from the ability to change things even if your everyday account is an "administrator" account.
