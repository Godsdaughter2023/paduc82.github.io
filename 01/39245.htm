entegy say:

Showing file count and size is actually not as easy as simply adding things up. There's a lot going on in the background, such as restore points and shadow copies of files. Even more hard to understand concepts such as hard links throw a bigger wrench into the mess.

Here's one modern example: Apple's new filesystem, APFS, doesn't copy files at all! If you copy a file on the same disk, it just makes a second name to that file. Changes to the new "copy" are saved independently of the underlying file. Can't just add up the file sizes/count then too!

3. The drive letter may be changed. e.g. E: maybe become F:, so I use disk management to change new E: to H: and change F: (old E:) to E:.
