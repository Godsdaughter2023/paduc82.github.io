Error 0X8007041D when setting up VPN connection on Windows 8 - Microsoft Community

vtcrusade say:

Error 0X8007041D when setting up VPN connection on Windows 8

So I had created a VPN connection to our corporate network on my new Windows 8.1 machine, but when I attempted to use the connection Windows would respond with a 0X8007041D error. Did the standard search for the error and found nothing related to VPN, just a bunch of people getting this error when connecting to Windows update.

After messing around with it for a day I found the solution and wanted to post it in case anyone else faces the same problem.

Our company uses PPTP to transverse the firewall, and apparently if you aren't real specific in telling Windows 8.1 that, you get some problems.

You'll need to edit your VPN connection settings (Your connection will be under the CHANGE ADAPTER SETTINGS in the Network and Sharing section of Control Panel) Click on PROPERTIES of your VPN connection. Then click on the SECURITY tab, then under the TYPE OF VPN drop box, change that to PPTP.

Next on the same security tab under the Data Encryption drop down, change it to Require Encryption and then under Authentication, click on the ALLOW THESE PROTOCOLS button. Check ON Microsoft CHAP version 2 (leave everything else unchecked). Click OK and then try using your connection again, you should connect just fine now.

To start Windows in safe mode or get to other startup settings:

  1. Select the Start Windows logo Start button button, then choose Settings Gear-shaped Settings icon.

  2. Select Update & security Circular arrows Sync icon > Recovery.

  3. Under Advanced startup select Restart now.

  4. After your PC restarts to the Choose an option screen, select Troubleshoot > Advanced options > Startup Settings > Restart.

  5. After your PC restarts, select a startup setting by pressing the corresponding number.
