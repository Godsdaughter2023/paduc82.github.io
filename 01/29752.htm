Unsigned USB drivers via Windows Update kills all USB Ports in Windows - Microsoft Community

komair say:

Unsigned USB drivers via Windows Update kills all USB Ports in Windows 8

Hi everyone,

I'm having the same issues as many with the USB ports. I have been using Windows 8 Pro x64 for a long time but after the latest batch of Windows Update (which installed multiple updates including KB2756872) all my USB ports were dead.

Both Keyboard and Mouse work fine in BIOS and also when in Automatic Recovery, etc.

I tried to system restore but looks like the last restore point was created after the damage had been done. So even after recovery the update was still installed. And for some reason none of my previous restore points were there anymore, even ones I had manually created. (There was no button at the bottom to show additional restore points!)

Luckily I still have a PS2 keyboard laying around. However, after uninstalling the updates my USB ports still did not work. 

Upon further investigation, I discovered a yellow exclamation mark next to the USB OpenHCD controller which stated "USB driver is not digitally signed" even though the driver is made by Microsoft.

So I tested it out... I disabled Driver Signing Enforcement and restarted the comp, and voila! Everything worked as normal. 

Looks like the Windows 8 Driver Signing Enforcement is preventing the USB drivers from loading at start up because they are not signed, thus killing the entire USB bus. 

If you're still struggling with this try restarting your computer with Driver Signing Enforcement Disabled and see if it works. You will need a PS2 keyboard/mouse or working touchscreen to navigate. 

Instructions:
http://tech.sixcolumns.com/2012/12/how-to-disable-and-bypass-driver-signature-enforcement-in-windows-8/

Could someone from Microsoft please flag this and have the appropriate team sign the OpenHCD driver and include it in distribution?

Thank you
