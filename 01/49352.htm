I have finally fixed this!!!!

Make sure you log out of ALL accounts before doing this!!

Okay so,  log onto your administrator account. Open the RUN command by opening the start menu, then typing "Run". Press enter. You'll see a dialog with a place to type... Type in: "NETPLWIZ" (you can type all lowercase, I typed in uppercase so you know which keys to press) and press enter...

If it still doesn't work, then you may have to set up a new admin account and do the same procedure to make sure it's set to only the Administrators group...

I asked previously ... about exporting mail to MS Outlook - I'd like to export direct from the store/database rather than the WLM application ... 

There is no difference. The program interface is simply a view of the records in the database index. The individual message files are only called up when a message is viewed in the reading pane or a message window. 

Can you not simply use Export messages to export direct to Outlook?

Each message is stored as an .eml file. You would otherwise have to use a third-party utility to convert these to a format that Outlook can digest. 

One way of moving messages to Outlook is to use an Outlook.com account as an intermediary.

  1. Create a new account at signup.live.com  
  2. Add this account in Windows Live Mail (Ctrl-Shift-T)
  3. Create a new folder under the account.
  4. Copy messages from a folder under Storage folders to the new folder, giving the program plenty of time to complete the operation.
    This folder and its contents will eventually find its way to the account at mail.live.com
  5. Add the Outlook.com account as an EAS account to Outlook 2013
    The new folder and its contents will eventually find its way to the account in Outlook.
  6. Repeat (3) and (4) for each folder.

For older versions of Outlook, you'd have to use the Outlook Connector instead of EAS.
