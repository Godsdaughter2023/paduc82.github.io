Is it OK to have spaces or capital letters in file names? - Microsoft Community

ecowhale.com say:

Is it OK to have spaces or capital letters in file names?

Should file names consist of all characters void of spaces?

Also, is it OK to capitalize letters in file names?

I'm asking to avoid known program or other conflicts and optimize indexing efficiency.

For example:

File name "Microsoft Answers.jpg" vs. "microsoft-answers.jpg" vs. "microsoft_answers.jpg"

ecowhale.com say: Lower Case

It is a very good idea to use all lower-case letters for all file names. There are two main reasons for this.

(1) If you use your files on the internet, you'll find that case-sensitivity matters. Whereas Windows will ignore the case, internet servers don't. We won't go into a detailed explanation here, but trust us when we say you'll have a massive grief reduction if all your internet files are consistently lower-case.

(2) Windows is notorious for changing the case of your file names without your permission. If your files are all lower-case, at least you'll recognise when this has happened and know what the case should be.

Using a consistent case reduces confusion and errors to such a significant degree that it really is worth it. Start now and you'll soon get used to it.

Spaces and Non-Alphanumeric Characters

For similar reasons to the lower-case argument, we strongly recommend that you use only letters, numbers, hyphens and underscores on your file names. These are all safe to use on the internet. Also, do not have spaces in the file name - this will confuse some operating systems and cause errors in web pages.

Batch Renaming

Often you will want to rename large numbers of files at once. For example, if you take 20 photos of your cat Fluffy, it would make sense to name the photos fluffy01.jpg , fluffy02.jpg , etc. By far the easiest way to do this is called "batch renaming", where one command renames all the files in sequence. For details, see our batch renaming tutorial.

Search doesn't work for numbers in contents of a file - Microsoft Community

vividsix say:

Search doesn't work for numbers in contents of a file

Windows 7 Search doesn't seem to work for partial numeric strings.

I put 123456789 inside a text file.

Search will only work if I search for the exact numeric string i.e. contents:123456789

If I try to search for 34567 (contents:34567) it will not find the file.

Any ideas?
the larch say:

[edit: I found a way without using Command Prompt.  See later reply.]

 

Windows 7 Search can only search successfully if the entered string appears in the text separately, with the exception that the string can be at the start of a string that appears in the text.  This behaviour is true for text in general, not just numbers.  Other than downloading some other search utility (there are many free ones), you can use Command Prompt, navigate to where you want to search (if you haven't done that before, you'll have to learn some stuff there), and enter:

findstr /s /m 34567 * > c:\results.txt

 

The /s is to enable searching subdirectories (subfolders).  The /m is to suppress showing all the lines of text that contained a match, and just show the filenames (and subdirectory paths).  The * is to indicate that you want to search all files in the current directory.  You could put *.txt to restrict searching to text files, for instance.  The > c:\results.txt part is to take the search results, and instead of displaying them on the screen, save them to a new text file named results.txt to be made in the C drive.
