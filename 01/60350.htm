rgharper911 say:

You almost certainly will not.  Each chk file is a fragment of a file that existed before the drive crashed.  In the days of fairly simple files, text files and such, you could often piece together the pieces of chk that made up one file.  Nowadays files are unreadable, each blob looks like every other blob, and there's no way to reassemble them.

This utility makes it easy to see which versions are loaded :

Set VIEW in DriverView to HIDE Microsoft drivers - then update those without Dump in their
names.

DriverView - Free - utility displays the list of all device drivers currently loaded on your system.
For each driver in the list, additional useful information is displayed: load address of the driver,
description, version, product name, company that created the driver, and more.
http://www.nirsoft.net/utils/driverview.html

For Drivers check System Maker as fallbacks and Device Maker's which are the most current.
Control Panel - Device Manager - Display Adapter - write down the make and complete model

of your video adapter - double click - Driver's tab - write down the version info. Now click
UPdate Driver (this may not do anything as MS is far behind certifying drivers) - then Right
Click - Uninstall - REBOOT this will refresh the driver stack.

Repeat that for Network - Network Card (NIC), Wifi, Sound, Mouse and Keyboard if 3rd party
with their own software and drivers and any other major device drivers you have.

Now go to System Maker's site (Dell, HP, Toshiba as examples) (as rollback) and then Device
Maker's site (Realtek, Intel, Nvidia, ATI as examples) and get their latest versions. (Look for

each installation.

shows up. This is because some drivers rollback before the latest is installed (sound drivers
particularly do this) so install a driver - reboot - check to be sure it is installed and repeat as
needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check
manually by model.

Manually look at manufacturer's sites for drivers - and Device Maker's sites.

marks-h say:

For  anyone else having the same problems loading FSX and getting the 1628 Failed to complete installation message, this is what I did which worked for me. I hope it works for others.

1. Load the FSX CD in the CD drive, open File Explorer and locate the DVDCheck.exe file on the CD.

2. Select and 'Right click' the file and select 'Properties' then select the 'Compatibility'  tab.

3. In the Compatibility section choose the version of Windows that FSX was able/used to run. (In my case it used to run fine using Windows 7 Pro 64bit). Check the tick box and then press 'OK'

4. Double click the file 'DVDCheck.exe' file and the installation should start....


5. You may get a message to either 'Repair' or Remove' as below. - Select 'Repair'.

6. The installation should start and eventually finish.


NB. When you run the FSX program after the installation, you may get an message saying... 

FSX Scenery.CFG File error
Local Scenery Directory
(Addon\Scenery) in Scenery Area 115 not found

As below....

If you do, then select the SCENERY.CFG file in the C:\ProgramData\Microsoft\FSX directory.

Take a copy of the file and rename it as SCENERY.CFG.old or something.

Then open the SCENERY.CFG file in Notepad and scroll down until you find the entry:

[Area.115]
Title=Addon Scenery
Local=Addon Scenery
Layer=115
Active=TRUE
Required=FALSE

Delete all this section (highlighted in red) within the SCENERY.CFG file and Save the SCENERY.CFG file. The FSX program should now run without the message appearing.


Hope this helps.
