davem121 say:
Hi Samuel

It depends on the method you used . . .

Did you just upgrade to the Pro version or did your perform a full clean install of Windows 10 Pro?

Open Windows File Explorer (folder icon on your Taskbar)

On the left, click 'This PC'

On the right, double click your hard drive

Do you see a Windows.Old folder, if so open that and navigate down into the users folder

Do you see your files there?

If so, immediately copy those files out into your user account folders (Documents, Pictures . . etc) because Windows will automatically delete that WIndows.Old folder in a few days, and all your data will be lost . .

robert aldwinckle on forums say:

When I enter into Device Manager and expand the keyboards category, I found that the driver is named "HID keyboard device" and is duplicated.

I think you need to drill down deeper on that tack.   E.g. getting into Driver File Details I find for Surface Touch Cover Filter Device 3 drivers

kbdclass.sys  kbdhid.sys  and  SurfaceTouchCover.sys

The first two are referenced for  HID Keyboard Device  (which is as far as you got).

So, do you have SurfaceTouchCover.sys  anywhere?

A way to answer that question would be to use this PowerShell script

PS>cd \
PS>LS -Re -Fi SurfaceTouchCover* -Fo -EA:SilentlyContinue | sort LastWriteTime | ft LastWriteTime, Length, FullName -AutoSize -Wrap

I actually started that first at cd \Windows but then after backing up to the root discovered that I had apparently downloaded something that contained the driver.   So, to have a look at that I would filter by directory only from that list and by the actual driver filename.   Although it is possible to do that with another execution of the above pipeline with an appropriate extra stage each time.  I would redo it either with a pipe to clip or just capture the output of the above and filter it manually.   I did the latter and then did

PS>$FL = Get-Clipboard
PS>$FL | sls "\.sys"

Note that a . is a pattern character and needs to be escaped if you want to match only that character.

Turns out I had either downloaded or reinstalled something earlier this year that looks identical to something created in 2013.  If instead a reinstall was being done it would have been originally done in 2016.

2013-08-08 16:23:43  37992 C:\Windows\System32\DriverStore\FileRepository\surfacetouchcover.inf_amd64_876425d265d2887b\SurfaceTouchCover.sys

I'm not sure what all that implies.  E.g. I don't know if I actually installed what I had downloaded or where I got it.  Here's what it got called when unzipped:  SurfacePro2_Win10_160501_2
