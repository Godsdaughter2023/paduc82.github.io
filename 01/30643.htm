Resolution for issue: MMC.exe blocked for your protection windows - Microsoft Community

hdjam say:

Resolution for issue: MMC.exe blocked for your protection windows 10

RESULTS MAY VARY

Workaround (if in a hurry)...

  1. Open Command Prompt as an administrator.
  2. Type mmc and press Enter.

TL;DR resolution (detailed version below)...

  1. On system with working MMC... Navigate to C:\Windows\System32\CatRoot
  2. Copy all .cat files to the plagued workstation in the same folder.
  3. Do NOT overwrite files.
  4. Reboot.
  5. If you are not in, repeat steps 1 and 2 and DO overwrite the files.
  6. Reboot.

Hello Everyone,

I have seen a lot of posts online for this same issue (image below). I plagued myself with the task of getting beyond this frustrating issue. By reading online, I tried many things, but I was not willing to tamper my security settings. MMC is a VERY powerful tool that I appreciate. I copied files from a non-preview build to a preview build and the fix still worked for me. I foresee this being a very useful fix for a lot of frustrated people. Please enjoy the information below.

What didn't work for me, but might for you...

  • Running chkdsk command
  • Running sfc /scannow command
  • Opening MMC in admin console/PowerShell
  • Using DISM to restore files SFC could not fix. This returned an error for me.
  • Turning off SmartScreen (not recommended)
  • Disabling UAC (not recommended)

What I didn't try...

  • All registry fixes I read about. Generally I was seeing security risks as the only way I saw others attempt to get around the issue was to stop the popup altogether. A very risky choice...

What I did to fix the problem (Detailed)...

1) Open MMC on another workstation with the same Windows installation type (Windows 10, find another Windows 10... Windows 8, try another with Windows 8, etc). If it opens, continue these steps.

2) Open the folder path: C:\Windows\System32\CatRoot\.

A) There are 2 folders in here for me on both machines. Both are a pre-determined set of numbers...

B) One was empty, the other had a handful of .cat files.

4) Create a new folder on the desktop. The name Catroot Files will do.

5) Copy all the .cat files from the CatRoot folder to the new Catroot Files folder on the desktop.

6) Optional: After the copy process is completed...Right click the folder, highlight Send to and select Compressed (zipped) folder.

7) Copy the files to the plagued workstation. (Network share, flash drive/USB, etc)

8) Extract the zipped folder as needed.

9) Select all the .cat files in the folder. Copy the files. (Ctrl + A followed by Ctrl + C)

10) Open the folder path C:\Windows\System32\CatRoot\.

11) Open the folder that is named closely (or matches) from the working workstation. There should be some .cat files in the folder.

A) If there are no files in both folders... Try matching the numbers of the folder that contains the files on the working machine.

12) Paste the files to the new location. If there are files, it is likely a prompt will appear... "Want to overwrite files?" Select no/do not overwrite.

13) Once the transfer process completes, restart the computer.

14) If the issue persists, redo step 12 AND overwrite the files.

15) That's it!

Hopefully this helps a lot of people resolve this problem. Hopefully I posted this in the proper topic as well! (First post ever, if not in a loooong time)
