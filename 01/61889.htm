Best practice of hard drive partition on Windowes Server 2008 and - Microsoft Community

hasan jamal siddiqui say:

Best practice of hard drive partition on Windowes Server 2008 and Windowes 2012

What is the best practice of hard drive partition on Windowes Server 2008 and Windowes 2012
keith langmead say:

Might be worth asking over at http://social.technet.microsoft.com/Forums/en-US/winservergen/threads as you may get more responses.

 

That said I'd say it very much depends on what you're intending to do with the server, and of course how much space you have available.

 

For a general server I'd probably go with two volumes, the C:\ drive for the system files, and a second volume for your data, for instance E:\. I'd recommend at LEAST 30GB for the C:\, preferably 40-50GB, since updates, patches etc will burn through that over time, and it's far easier to start big than try expanding later.

 

If you're running Terminal Services then you'll likely need a much bigger C:\ since much of the user profile data gets stored on there so you can run out of space quite easily.

 

Like I said, it all depends on what you're doing with it, what space and disks you have available etc, there's not really a one size fits all answer.

Hi,

 

Internet connection Sharing is a technique used to connect a computer to other computers in a network.

 

May be not all the services that run Internet Connection Sharing(ICS) are enabled. I'd suggest you to start from the beginning and make sure the below dependency services are enabled.

a. Click Start, Run, type services.msc, and click OK.

b. To enable a service, find it in the right pane and double-click its entry.

c. Be sure its Startup Type is set to Automatic and click OK.

All the following services must be thus configured in order to use ICS:

Application Layer Gateway Service
Network Connections
Network Location Awareness (NLA)
Plug And Play
Remote Access Auto Connection Manager
Remote Access Connection Manager
Remote Procedure Call (RPC)
Telephony

After verifying and/or configuring Automatic startup for each of these services, close the Services window and restart the computer before attempting again to enable ICS.
