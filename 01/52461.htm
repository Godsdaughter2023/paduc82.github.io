¡firedog say:

I rather thought that Windows Essentials was not supported and wouldn't install on Vista. Still, there's not much difference between the versions of WLMail, so this is not having any effect on your current problem.

I don't know where you got those settings from, but you should be using port 465 with SSL and SMTP authentication.

Right-click on the account name in the folder pane and select Properties.

  • On the Servers tab, ensure that My server requires authentication is selected.
  • On the Advanced tab, change the outgoing mail port to 465 and select This server requires a secure connection (SSL).

You can usually find the settings to use on the email service provider's website.

¡firedog say:

I'm not surprised you're confused.

You have access to two different Verizon webmail sites. Do you see the same inbox at both sites, or are they different?

  • If they're different, we're talking about two separate accounts and you've mixed them up when configuring WLMail.
  • If they're the same, then there's some Verizon-specific jiggery-pokery going on that I can't penetrate.

When that's been established, the existing account has to be reconfigured to use its proper credentials (username and password), and another set up if necessary to access the other mailbox if there is one.

 

The Email address on the General tab of the account's properties sheet is normally the one that shows in the From: line of messages you send. Unless there's a different Reply address, it's the one that will be used when someone replies to a message from you.

On the Servers tab, you have an Email username. For some servers, this is a unique identifier otherwise known as Screen name. In many cases, it's the same as the part of the email address before the '@' symbol, but needn't necessarily be. While it's usual these days for the Email username to be the same as the full email address, Verizon seem to insist on what they call 'your Verizon Online username'.

It's this combination of a username and associated password that gives you access to your mailbox so your inbound messages can be downloaded.

In most cases, the same credentials will be used when sending messages. This doesn't have to be the case - sending and receiving use different servers, so it's quite possible to use one set of credentials for receiving messages and a completely different set for sending. In this case, the credentials would be entered on the dialogue that opens when you click the Settings button on the Servers tab.

        Is there a way of reloading my task bar without having to restart my computer? Thx.

        I use W7U.

        ronnie vernon say:

        Hi

        You can restart the UI without rebooting.

        1. Press CTRL+SHIFT+ESC to start the Task Manager.
        2. Select the Processes Tab.
        3. Right click the Explorer.exe process and select End Process. The Taskbar will disappear.
        4. In Task Manager, select File/New Task (Run...). Type Explorer.exe and press OK.

        The Taskbar will return and should be displayed properly.

        I would recommend checking for and installing the newest drivers for the installed graphics card.

        Hope this helps.
