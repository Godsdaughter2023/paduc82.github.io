When you try to install updates on a Windows-based computer by using the Windows Update or Microsoft Update Web site, you may receive the following error message: 0x8007000B

 

This problem may occur if the Windows file system transaction log is corrupted. The Windows file system uses the transaction log to recover system transactions when a file error occurs.

Follow these methods:

 

Method 1: To fix this problem, follow these steps:

  1. Press Windows key + X.
  2. Click Command prompt (Admin).Collapse this imageExpand this image If you are prompted for an administrator password or for confirmation, type the password, or click Allow.
  3. At a command prompt, type the following command, and then press ENTER:

fsutil resource setautoreset true c:\

Note These steps assume that Windows is installed in the default location, on drive C. If this is not the case, adjust the drive letter of the folder path to match your configuration.

Restart the computer.

net stop wuauserv

2. If it can't be stopped, restart the PC.

Net stop bits

Net stop WUAUSERV

4. Go to c:\windows and rename SoftwareDistribution folder to SoftwareDistribution.old

Net start bits

6. Go to Windows Update and check for updates.

*If an error occur, try to check the updates again, it should work.

6. Download all important update and check from the Windows Store if you can now download Windows 8.1
