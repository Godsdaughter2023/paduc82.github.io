    al jarvi say:

    First find out what version of Windows 7 is running on the computer you want to remotely access/control. Go to the Start button and type winver in the search window. Windows will report the version in a popup on your desktop.

    ***********************************

    Presuming the computer is running Windows 7 Ultimate or Pro...

    The basic steps are...

    • Enable Remote Desktop [RDC] on the Windows 7 Ultimate RDC host PC.
    • If the RDC login ID on the Windows 7 Ultimate RDC host PC is not an administrator then add the user to the Remote Desktop Users Group. Remember RDC login user IDs are local to the Windows 7 Ultimate RDC host PC not the RDC client PC.
    • Make sure there is an exception in the Windows Firewall, or any third party firewall, on the Windows 7 Ultimate RDC host PC.
    • If the Windows 7 Ultimate RDC host PC is behind a router make sure TCP Port 3389 is forwarded to the LAN IP (static is preferred) of the Windows 7 Ultimate RDC host PC.

    You can test port forwarding of TCP Port 3389 through your router by going to the http://www.canyouseeme.org test site. For help with setting up port forwarding through any router go to the PortForward.com site for help. Ignore references to UDP Port 3389.

    http://portforward.com/english/applications/port_forwarding/Remote_Desktop/Remote_Desktopindex.htm

    To test call the Windows 7 Ultimate RDC host PC from another PC or laptop on your LAN using its LAN IP (static is preferred).

    http://theillustratednetwork.mvps.org/RemoteDesktop/TroubleshootingDiagrams/Basic.html

    To call from a remote location use either the public IP of any router the Windows 7 Ultimate RDC host PC is behind or a fully qualified domain name [FQDN] from DynDNS (which I presume you have setup already).

    Its highly recommended you use Network Level Authentication [NLA] if you remotely connect from a Win 7, Vista or XP SP3/SP2 client.

    http://windows.microsoft.com/en-US/windows7/What-types-of-Remote-Desktop-connections-should-I-allow

    http://theillustratednetwork.mvps.org/RemoteDesktop/RDP6ConfigRecommendations.html#SP3

    http://theillustratednetwork.mvps.org/ScreenShots/RDC-NLAandServerAuth/Host-UseNLA.jpg

    http://theillustratednetwork.mvps.org/ScreenShots/RDC-NLAandServerAuth/ClienttoServer-UseServerAuth.jpg

    Make sure you use the latest client if your connecting from a Vista or XP SP3/SP2 client machine. Windows 7 includes the newest RDC client software.

    http://support.microsoft.com/kb/969084

    General help written for Vista but close enough to Win 7...

    http://theillustratednetwork.mvps.org/RemoteDesktop/RDP6ConfigRecommendations.html
