mike hall... always say:
"R.Ortheris" wrote in message news:d3996d6b-0bec-4261-a77d-44ecf83d6987...

The process to do this has hardly changed so do as you would have done long ago.
 
If you do move the page file, ensure that it is moved to a fixed secondphysical drive, NOT a logical drive on the same physical drive as the OS works from.
 
By leaving the page file within the OS partition, the heads do not have to travel far to get to the page file.
 
If you move the pagefile to another partition, the heads now have to go out of their way to access it and everything takes that much longer. 
 
A pagefile being accessed by a different set of heads on another physical drive will increase efficiency, but the time taken to change the pagefile parameters would probably amount to a year's worth of inefficiency caused by leaving the page file within C. Unless you are running processes  where access to the pagefile is critical, C is the best place for it..
