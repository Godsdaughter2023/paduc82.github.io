tomwasabomb say:

I ran into this problem (when you choose sleep from the menu, it appears to sleep for about a second, then returns to the login screen.)  After some investigation, here is what I've found:

  1. When the computer doesn't sleep, it's because something (a device driver) is waking it up.
  2. To find out what's waking it up, you can run a special tool from an administrative command line.
    1. To get this command line, click the WindowsKey-X-A sequence.
    2. Use the powercfg utility to find out what is waking it up.  The command line options are documented here.  I ran "powercfg -devicequery wake_armed" to get a list of all the devices that have the ability to wake up the computer.
    3. You can use the 'powercfg -devicedisablewake "device name" to disable the ones listed in the wake_armed command (one at a time).  For example, this might look like: powercfg -devicedisablewake "Microsoft USB IntelliMouse Explorer"
  3. If you run this for all your devices, the insomniac behavior should stop, but that means when you move the mouse or click the keyboard, your computer will not automatically wake up.  So you can do a binary search to find the one that's causing the problem.
  4. You might also try "powercfg -lastwake" to see if that gives you a hint about which device is the problem, then only disabling that one.
  5. You can also use the Windows Event Viewer to find out which device was waking up your computer.  (Windows->run and type 'eventvwr').  Click Windows in the left pane, then system and filter current log.  Drop down the event source combo box and pick "Power Troubleshooter".  The event log fragment you see will identify the evil device.
  6. Another approach you could take is to remove all USB devices from your computer and see if it sleeps.  Start putting them back one by one until the problem recurs.

Anyway, amazingly, it all makes sense (only after going through this).  The problem you're encountering might be different from mine, and these ideas may not solve the problem for you, but it does seem like the powercfg utility is the key.  You may find your answer in some of the other command line options of that utility.

saktinathmukherjee say: The problem has been solved as per the following steps.
1) Open "Mail" . Go to Settings-> Manage Accounts-> Select Yahoo. Yahoo Account Settings screen pops up-> Delete Account.
2) Log in to your Yahoo A/c. Go to Account Info page. Click Account Security. At this point
you will be asked to re-enter your Yahoo User ID & Password.
3) Turn ON "Two-step verification". At this stage you will be asked to verify your Mobile No. to receive a Verifying Code, which will be sent through SMS. Enter the Verifying Code to enable Two-Step Verification.
4) Turn ON "Allow apps that use less secure sign in" .
5) Click "Manage App Password". From the Drop Down list of Apps, Select Other Apps. In the next box identify it as "Mail for Windows 10".
6) Click "Generate". A password will be generated. Copy it to Clip board by long pressing.
7) Open "Mail". Go to Settings-> Manage Accounts-> Add Accounts. Type Yahoo Mail.
8) Yahoo Account Settings page opens. In the password box copy from Clip board the password generated in Step 6. Save.
