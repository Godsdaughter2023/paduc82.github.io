lemp say:
If that's really all of the information that is available, you'll have to resort to trial and error.

Use either the built-in System Configuration Utility (Start > Run > msconfig > OK) or (better) Autoruns.

In msconfig, click on the "Startup" tab.  In Autoruns, click on the "Logon" tab.

In either case, disable half of the entries and reboot.  If the error does not appear, the problem program is in the half you disabled (by UNchecking).  Conversely, if the error does appear, then the problem is in the half that you did not disable.

Keep narrowing things down by halves until you find the culprit.  Then uninstall it (and reinstall it, if you still need it).

Note that one or more of the programs in the startup list will be related to your antivirus.  Either disconnect your computer from the Internet while you are doing this troubleshooting or leave the a/v program enabled until you find the culprit (or conclude that it's the a/v program itself that's causing the error).

P.S.:  A quick Google search for Error code 1/-3/10900 suggests that the problem may be related to Nero 7, so if you have that and it's configured to start when Windows starts, I'd disable that and see if the problem goes away.  To uninstall Nero, see http://www.nero.com/enu/tools-utilities.html

lemp say:

Although you haven't identified the specific models of your printers or answered whether the IP addresses assigned to the printer ports properly correspond to the IP addresses actually configured on the printers, what I suggest at this point is attempting to get the printing subsystem as close as possible to its state when Windows was installed. 

Microsoft used to have an automated Fixit routine that did this (see https://support.microsoft.com/en-us/help/2793718/you-experience-print-spooler-error-messages-after-installing-or-upgrad and more detailed explanation here: https://blogs.technet.microsoft.com/askperf/2012/02/24/microsoft-fixit-for-printing/).

However, in its zeal to move everyone to Windows 10, Microsoft "retired" the Fixit routine (as you will see if you click the links).  Fortunately, the Microsoft team that developed the routine preserved it as a Powershell script that you can download and run .

Although you can try the "Light" mode first, I suspect that given what you've done to this point, you'll need the "Full" mode.  As explained in the TechNet article linked above, "the Full mode will restore the print spooler to defaults and will require a complete reinstall of your print drivers and printers."

Directions:

  1. Disconnect the printer(s) from the computer if connected by USB
  2. Click Start, type Programs and Features into the Search box, and press Enter.
  3. Uninstall any software relating to your printers and restart the computer
  4. Go to https://github.com/jdickson289/Print-Reset-Tool/ and download print-reset.ps1 and make a note of where you save the file
  5. Click Start, type PowerShell into the Search box, right-click on the link to Windows PowerShell, select "Run as administrator" and respond as needed to the UAC prompt 
  6. Type the following command into the PowerShell window and press Enter
    Set-ExecutionPolicy Unrestricted
  7. In response to the "security risks" question, type Y and press Enter
  8. Type the following command into the PowerShell window and press Enter.  In typing the command, use the full path of the location where you saved the print-reset.ps1 file.  In the example shown below, print-reset.ps1 was saved in C:\Temp
           C:\Temp\print-reset.ps1 -full

    As I noted earlier, you first can try running the script in "Light" mode --            C:\Temp\print-reset.ps1 -light
    but I doubt that it will help
  9. Type R to run the script
  10. After the script completes, restart the computer
  11. Download the appropriate driver package(s) from HP -- https://support.hp.com/us-en
  12. Run the downloaded file; you should do this from an account with computer administrator privileges
  13. Use the "Add a printer" wizard (in Devices and Printers) to "Add a network, wireless, or Bluetooth printer." 
  14. The search should find your printers by IP address.  Select one and complete the wizard.  When you are asked, take the recommended action to "Use the driver that is currently installed."
  15. Go to Devices and Printers and confirm (a) that the printer is installed and set as the Default Printer and (b) right-click on the icon, select Printer properties, and click "Print Test Page"
  16. Repeat from step 13 for additional printers

This is what the PowerShell window should look like as you proceed through steps 6 through 9 (I didn't enter R because I didn't want to reset my own printing system).  Don't forget to use the actual location where you saved the print-reset.ps1 file rather than C:\Temp as shown below.

Note that in step 8 if you omit the -full switch and run the script with no switches, you'll get a complete description of what the script does.  See the screenshot below.
