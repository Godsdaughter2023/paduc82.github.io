One thing to be aware of ... merely uninstalling a printer and then reinstalling often does NOT result in the installation of a new driver.  If there was a problem with the previous installation, it often is propagated into subsequent attempts to install.  The way to get around this is described in Article One ... particularly Step 4.  If you suspect driver corruption, you need to remove the old drivers using the File > Server Properties before installing a new driver. 

Windows installs hardware devices using INF files.  These are located in %windir%\inf.  In reality, the previous statement is incomplete.  The first time a device is installed, Windows uses the INF file to create a corresponding PNF file located in the same directory.  The PNF file is a "precompiled" version of the INF file and Windows setup actually uses the PNF file because it is "more efficient" to do so.

If you download a new driver and only replace the INF file and do not delete the old driver files, when you "update" the driver, Windows setup looks at the old PNF file which points to the old files and uses them.

frederik long say:

So I now have two identical backups of my date files, one in the system image backup and the other in file history backup, is that correct?  If so what genius created this space wasting backup system?

These two backups are not in the least identical.

  • Image Backup. This backup file contains a complete image of your system drive. You can use it to restore Windows in case your hard disk fails. You create a new image maybe once a year.
  • File History. This backup contains your personal data files. You can use it to restore any of your files in case you lose or damage them. This backup is updated continuously.

It seems that the genius who designed this system did have his thinking cap where it belongs, on his head. You could, of course, argue that the image backup should not contain user data. Indeed, but this shakes the whole foundation of storing user data on the system partition. I keep it elsewhere.
