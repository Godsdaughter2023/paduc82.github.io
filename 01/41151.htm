vladeb say:

I posted a thread about this yesterday, but somehow it is unavailable for public view, got send to experts for review or something.

Anyway I have the same issue and while there may be many reasons, in my case I was able to narrow it down to this servicing stack:

C:\windows\winsxs\amd64_microsoft-windows-servicingstack_31bf3856ad364e35_10.0.16299.212_none_17098c7c31fd065f

It was installed by KB4058258. What is does it executes a servicing job that deletes quite a few drivers, including acpi.inf and disk.inf. Which makes system unbootable. The only way is to restore to a point when you still had acpi.inf and disk.inf in c:\windows\inf. If you have a system that's unbootable you can actually boot into recovery environment and check using command line if you have those files. If you don't you're done, there is nothing you can do other than a system restore.

The scheduled task that executes this servicing job is started when computer is idle, so if you have a restore point, even with this KB installed, but from a moment before this servicing stack was executed you can just disable the Scheduled Task  \Microsoft\Windows\Servicing\StartComponentCleanup, this will prevent the drivers corruption.

So this is what I have right now, KB4058258 installed (don't have a restore point without it...) and this task disabled.
