Browse to key HKEY_Local Machine > Software > Microsoft > Windows NT > CurrentVersion
Change EditionID to Core (double click EditionID, change value, click OK). In your case it should at the moment show Pro

Close the Registry Editor
Insert the Windows 10 Home installation media, do an in-place upgrade by launching the setup from desktop (not booting with Windows 10 Home install media)
When asked, enter your valid Windows 10 Home product key and select what to keep (settings, personal files and apps, only personal files or nothing)
Windows 10 Home in-place upgrade will be done now.

Windows Connection Manager Service - Microsoft Community

mattwilling say:

Windows Connection Manager Service

I've been running Windows 10 on my development system at work (Dell XPS 8700, 32GB RAM, Realtek network hardware, Realtek audio hardware, ATI graphics card) for five to six weeks. Last week, I noticed that on a regular basis it slowly seemed to lose network connectivity. The network icon in the notification area would show the yellow triangle, and then the red error indicator, at which point, I would have no connection to the network or the Internet. At the same time, the speaker icon would also get a red error indicator; running the audio troubleshooter would fix this for a period, and then it would go again. Running the network troubleshooter was fruitless: no fault was reported. Only resetting the machine would allow me access. Often, the network icon on the lock screen would already be showing the error indicator before I'd even logged in, though the icon in the notification area didn't seem to keep sync.

Strangely, a Hyper-V virtual machine running on the system did have connectivity to both our local network and the Internet. I wondered if Hyper-V (or its virtual network connection) could be at fault, so I uninstalled it temporarily. The problem continued to occur.

I checked for new drivers on a number of occasions, but could only find a new video driver, which I duly installed. I tried deleting and reinstalling the network device from Device Manager, but to no avail.

I investigated the System event log and discovered that a number of Windows services were constantly stopping unexpectedly and being restarted. These were: the Security Center service, the Windows Connection Manager service, the TCP/IP NetBIOS Helper (lmhosts), the Event Log service, the DHCP Client and the Windows Audio service. This seemed interesting, but I didn't know why such a broad range of processes should be unexpectedly stopping on a regular basis.

Today, I discovered a post that suggested that this behaviour might be due to the fact that all these services normally run under the same process ID, so that if one of them fails, it takes them all down. The page I read suggested using the SC Config <Servicename> Type= own to run each service under its own process ID (you need to do this from an administrator command prompt). I tried this and it fixed the problem to some extent.

The audio service no longer stops, and I have had full connectivity all day, so the immediate issue is solved. However, the network icon, when left-clicked, shows no networks at all, and the event log shows that the Windows Connection Manager service is still stopping unexpectedly. This seems like the culprit that was dragging the others down. Viewing the Operational log in Applications and Services/Microsoft/Windows/Wcmsrv showed nothing untoward as far as I could tell, so I have left it there for the moment.

I am posting this for two reasons: hopefully anyone else in the same boat will be able to resolve the issue and keep their network connection (and audio) up and running; also, perhaps someone has an inkling as to why this service is crashing and can offer some useful tips as to how to prevent it, or can use this information to diagnose any internal fault that might be present.

I am running Windows 10 at home on a different system and haven't encountered this issue and a work colleague who is running it on a Dell laptop which also has Realtek audio and network hardware has reported no issues on his machine.

files and they can be problematic in all versions of
Movie Maker.

Best bet IMHO would be to copy the VOB files to
a folder on your local hard drive and convert them
to the .WMV format before importing into Movi Maker.

There are many converters available on the net...
some free, some retail, some better than others...
the following freeware  is just one example that
works well for me.

(FWIW...it's always a good idea to create a System
Restore point before installing software or updates)

read *all* the install screens...there may be some
items you want to uncheck***)

If a green Install button appears when you launch
Freemake...you can ignore it and it should disappear
when you import files.

To download the full version instead of using the
web installer...see the following:

Freemake Apps - Offline version
http://www.freemake.com/offline/offline_all/
(If you want to install Freemake freeware
to your PC without the Internet connection,
download these files)

and drag/drop your video files on the main screen.

Or...Left click the Video button and Browse to the
folder your video files are saved in.

Select (highlight) the files you wish to import into
Freemake and left click the "Open" button.

The "WMV Output Parameters" screen will open. 

Choose a "Save To" folder so you can find the
converted files.

smlunatick say: You can not set up a dual boot system with an existing Windows 7 drive as the secondary drive.  You will need to re-install that second drive's Windows 7 in order to allow it to pass the  "boot" process.

Also, if you continue to attempt to set up this dual-boot process, you can not use any OEM software to re-set up the second hard drive.You are "breaking" the OEM license agreement when using the second drive in a different PC.

Also, Windows 7 will need to be re-installed twice:  once on "your" PC and the second time  when you return it to the original PC.

One suggestion:  since you have two PCs and they both seem to use the same video card interface, you can buy a "third" graphics card.  It does not have to be the same level has your "main" graphics cards.

When Setting up Dual boot on a clean system you can install the 1st O/S with only drive 0 connected. However in order for the boot menu to work properly you need to have both drives connected when you install the second O/S.

 

When you installed the second O/S it was the only drive but using the boot menu option to force the system to boot from the second drive causes problems because the drive numbers with both disks attached will be wrong (not the drive letters)

 

You need to reinstall your second O/S with the 1st drive attached and use the Windows Boot Menu to select the O/S instead of the F12 Boot Menu. You can still protect your own Windows installation with passwords and hiding the disk in the other O/S

 

Apart from hiding the drive by removing the drive letter there is a registry mod that will lock the drive out.
