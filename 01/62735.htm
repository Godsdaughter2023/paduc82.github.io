Use only the Media Creation Tool which prepares the media to boot for both UEFI and Legacy BIOS. Rufus will only make one or the other and isn't as reliable. http://windows.microsoft.com/en-us/windows-10/m...

I notice there appears to be an OS on DISK0 installed to Legacy as shown by the presence of the System Reserved Active partition, while the DISK4 OS is installed to UEFI. This means the system has both EFI and Legacy, requiring the CSM or Legacy BIOS setting to be enabled in addition to the UEFI.

Since nothing was said about a Dual Boot is it possible there is an old System drive now used for storage that did not have its System Reserved partition wiped or deleted, as this can very well cause the problems you describe. An old System drive should actually be wiped to get all code out of the boot sector, with at least Diskpart Clean Command. Otherwise it will pause to query the Active flag which signals a System boot partition in Legacy BIOS, complicated even more by the fact your NVME install is to UEFI.

 Besides removing the system reserved partition or wiping the Windows 7 storage drive to get it cleanest,  I would go into UEFI setup and change To UEFI only.

If DISK4 is the install you did to the NVME why does it still have the 500GB factory OEM partition at the front of it? Was the drive not wiped before it was installed?

greg carmack - windows mvp 2010-2020 say:

Vascilios, If you are not still booting Windows 7 on the machine then the System Reserved partition should be removed, especially since it holds an active flag that will cause the UEFI's firmware Compatibility Service Module to pause and query it during the boot process.

Besides this there is bootable code left on an OS drive that can interfere if the drive isn't properly wiped before reusing it for data storage

So absolute best would be to move the data off to wipe the drive with Diskpart Clean Command and reformat:
http://www.sevenforums.com/tutorials/52129-disk...
https://www.pcworld.com/article/248980/how-to-p...

Until you're able to do that you should at least delete the System Reserved partition in Disk Management, or from a Partition manager like Easeus. If necessary remove the Active flag first: http://www.sevenforums.com/tutorials/197157-par...
If it balks use these same steps to remove an OEM partition: http://www.mydigitallife.info/2008/02/29/delete...

If this doesn't solve the boot problems then there is still the issue with the NVME not having it's 500mb Factory boot partition at least deleted  so it doesn't conflict with the new EFI system partition actually doing the booting.

If now or later you want to get it cleanest wipe the drive with Clean Command to get all code in the boot sector and partition table. this will resolve best the problem with two boot partitions on the drive which could very well conflict - by the presence of the 500mb partition, or the code it wrote in the boot sector of the drive which was not cleaned first.

How can I connect Windows XP to multiple networks, Wi-Fi for internet, and another for file sharing?

On windows XP I need to connect to two different networks. One wifi connection for internet and other one for file sharing.

 Different gateways, different static address, same subnet mask. But it gives problem when run simultaneously and needs

one network to be disabled. Is there any way to make them run simultaneously?

original title: Tweo different networks simultaneously on windows xp

wunders say: Go to where you have set up the static IP addresses for your network adapters.  Leave the "Default Gateway" blank for your local file sharing adapter and insert a valid Default Gateway for your wifi adapter to the internet.  This will, of course, limit your local file sharing adapter to a single subnet -- which should be OK for home or small office use.

This should work so long as the two physical subnets have two different subnet numbers. Generally this means that the first three numbers of the adapter's IP address cannot the same on both adapters --  if they are, then you need to make one of the subnets different. (i.e. one is 192.168.1.xxx  and the other would be 192.168.2.xxx).  This is typically accomplished in a Router's configuration page.
