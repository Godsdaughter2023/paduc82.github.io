    Windows 8.0, Windows 8.1 or Windows 10 came preinstalled on my computer, how do I find the product key?

    In Windows 8.0, Windows 8.1

    Press Windows key + X

    Click Command Prompt (admin)

    In Windows 10:

    Press Windows key + X

    Click Windows PowerShell (admin)

    Why can't I copy a file from a USB drive even when giving Admin permission

    I am restoring files from an older failed computer.  I have them now on a USB drive.  My Windows 10 machine will not allow me to make the copy from the USB drive, even though I've granted Administrator permission.  

    I've now logged in *as* the Administrator.  Same denial of permission. 

    I tried "takeown" - No joy.  It tells me I can't take ownership of files on insecure drives (this is a USB drive).

    I tried "icacls" - No Joy.  In fact, this one is worse, because using my specific command:

    icacls *.* /grant:r

         or

    icacls * /grant:r

    The system tells me:  "invalid parameter /grant:r"

    Even though, clearly and plainly and irrevocably in the help it says:

    ICACLS name [/grant[:r] sid:perm[...]]

    So, what part of "/grant:r" is invalid? 

    So, never giving up, I tried:

    icacls *.* /grant Administrator:r

    This works on directories, but *not* files on the USB.  It still tells me access is denied, even though "takeown" clearly and irrevocably states that "File ownership cannot be applied on insecure file systems; there is no support for ACLs."

    Therefore, since there are no ACLs for the system, I should not have to grant specific permissions, and access *cannot* be denied, because there is no ACL!  Which is it?  Why is it?  

    SOLUTIONS:   

    Solution #1:  If this happens and you have an older Windows computer, it appears you can share the USB drive on the OLD NON-WINDOWS 10 (emphasis important) computer.  It will be important that both computers are on your internet connection (at home or in the office).  Then, on the Windows 10 computer, go to Windows Explorer, select Network, and locate the other computer.  You'll have to log in to the other computer.  Then you can copy from the USB drive attached to the other computer with no permission problems.   I did it this way just to see if I could copy from the USB through the share using Windows 10; and I did!   Unfortunately, if you don't happen to have another non-Windows 10 computer lying around (maybe borrow a friend's that isn't yet a Windows 10 machine), or a Web server (see Solution #2 below), you might just be out of luck.

    Solution #2:  Here's the ultimate gotcha on why the above problem is so stupid.  I setup a web server on another non-Windows machine and put the files there.  I then was able to connect to that web server and download the files *with no issues and no special permissions needed*.   Now, someone from the security team at Microsoft explain to me why it is less secure to copy a file from a USB I own than to be able to download a file from a web server on an arbitrary computer on the network!!!   Also someone from the security group at Microsoft explain to me why I can't run the new Microsoft Edge browser from the Administrator account, but I *can* run Firefox from the same account!  Does Microsoft OS not trust its own Edge, but is OK with FireFox?

    The solutions I posted above should be embarrassing to the Windows 10 team.  The hoops one must jump through to simply copy a file from a USB drive.  The interesting thing is, most casual non-tech users will not have a solution or the technical skills to do what I did or even try the things I did.  They'll have to take their old computer to someone who will charge them lots of money to simply copy a file from a USB that wasn't formatted from Windows-10, or not created from another Windows-10 computer?!  I have no clue why Windows 10 could not copy the files.  There *is* no sane reason why.
