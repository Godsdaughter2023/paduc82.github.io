Hi Alex,

Let us perform the method below and check if it helps:

 

Disable Numlock from Registries

 

Numlock cannot be disabled with shift key as we do in case of Caps lock but we can disable it temporarily whenever the computer restarts or just permanently disable numlock. And to do that refer the steps below:

 

Note: Important this section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, click the following article number to view the article in the Microsoft Knowledge Base:

322756 (http://support.microsoft.com/kb/322756/ )

How to back up and restore the registry in Windows

 

1. Close all windows and programs that are running on your computer.

 

2. Press the Windows key and R key simultaneously to open the Run dialog box.

 

3. Type regedit, then press Enter.

 

4. Navigate  to HKEY_CURRENT_USER/Control Panel/Keyboard.

 

5. Double click on IntitialKeyboardIndicators which on the right side of registry window.

 

6. Enter 0 (the number zero) in the Value Data field to disable the numlock after logging to the computer.

 

7. Enter 1 to disable the numlock entirely.

 

8. Click OK then close the registry Window.

 

9. Restart your computer. To re-enable numlock, follow the same process, but change the Value Data to 2.

john_barnett say:

Hello M_K_M,

 

You are not going to get a 'free' copy. When the operating system is installed by the PC manufacturer the manufacturer either supplies original CD's or else the image is placed on a hidden partition on the hard drive. Normally the PC's User Guide provides details of how to create recovery discs so that you can copy the image over to a set of CDs. The User guide will also give instructions on how to return the PC back to factory conditions by using the image on the hidden partition.

I would contact HP and ask what method was used, i.e., hidden partition or CDs, to restore your PC back to factory conditions. They may have recovery disc which they can supply to you (there may be a charge for this). Alternatively they may be able to supply you with an XP disc. Unfortunately there will be a charge for the XP disc which may be £50 or more.

If you try to buy a copy of XP from the internet be careful as there are numerous pirated copies around. Microsoft has also stopped supporting XP now as the operating system is quite old. It maybe worth investing in a more modern operating system such as Windows 7, assuming your PC is compatible with Windows 7. Also when buying the operating system a 'retail' copy is better than an OEM. OEM copies cannot be installed on another computer, they are tied to the first computer that they are installed upon. A retail copy you can move to another computer when that PC dies. Retail copies are more expensive but are worth the extra money.

When choosing to Install the dfrg.inf file, it might appear that "nothing happens", but something happened (it is apparently just not exciting enough to report).

Running the XP Disk Degragmenter you might see messages similar to these:

Windows cannot connect to the Disk Defragmenter engine
MMC could not create the snap-in
Navigation to the webpage was cancelled

Do you now or have you ever had a third party disk defragmentation tool (like Diskeeper, Auslogics, etc.) installed and then uninstalled it?  Uninstalling them will sometimes not put back the original XP defragmenter settings so you have to put them back manually by reregisteing a couple of DLL files.

It is fine to register the DLLs even if they are already registered.

To do that, click Start, Run and in the box enter each of the following lines one at a time and then click the OK button to execute each of the commands one at a time:

regsvr32  dfrgsnap.dll
regsvr32  dfrgui.dll

When the process completes for each DLL you register, you should see a message similar to the following:

DllRegisterServer in <DLL Name>... Succeeded.

After reregistering both of the DLLs, reboot and test your XP Disk Defragmenter.
