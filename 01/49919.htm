lemp say:
I'm really sorry about the situation you're in.  It's rubbing it in a little, but this shows why it's good practice to make regular backups to some external media (external hard drive, DVDs, online backup).

One of the things that first occurred to me was that you had a corrupt profile.  When this happens for whatever reason, Windows XP creates a new profile (using the template that is stored as "Default Profile") and gives it a name similar to the one you were using.  For example, if your original profile was named "Owner", the newly-created profile might be named "Owner.001".  You might not see the .001 extension if you have "Hide extensions for known file types" turned on in Tools > Folder Options (this is the default), but I would have thought that you would have seen two folders with the same name in C:\Documents and Settings.

If you determine that you do have a corrupt profile, it's usually easy to retrieve the data and put it into the correct locations of the new profile (which starts out with empty "My Document" and "My Pictures" folders).  See
http://support.microsoft.com/kb/811151
http://support.microsoft.com/kb/318011

Here's a kind of last-ditch effort.  Download, install, and run JDiskReport.  This little Java program will scan your disk (or whatever disk or directory you specify) and show you both graphically and textually what files are taking up how much space on your drive.  If your drive space allocation is showing 60-70% used and you believe that this is about where you were before, JDiskReport should be able to find the files.  Once you have located them, we can go about getting them back where they belong.

                        net start msiserver
