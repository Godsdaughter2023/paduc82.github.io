I believe the one you need to get your desired input method is "Canadian French", not Canadian Multilingual Standard, unless you mixed up your picture links. 

In any case, be aware that there are only two types of keyboards we will likely run into here in Canada.  Both are QWERTY (that is the order of the keys starting with Q and going right along the keyboard from the Q)

The first is the 101/104 key ANSI keyboard (typically a US keyboard, and likely what your gaming keyboard is)

The second is the 102/105 key ISO keyboard which is what the multilingual standard is fully compatible with.

HOWEVER: Windows still allows you to use the Multilingual and French keyboard layout on the US keyboard.  The only difference is that the Ù is not provided.  You have to use the accent and then type the U.  Everything else should be available, except there are slight differences in placement.  The main being that the À with the accent is above the enter key rather than to the left of the enter key.  This is because the row above the enter key has one more key on the american keyboard.  On the same row as the enter key it has one less key on the American keyboard. 

But notice the numbers I stated in the paragraph above? Those numbers refer to the total number of keys on a keyboard with a number pad.  101 is the minimum for the American ANSI keyboard.  It could have up to 104 depending on whether it has some additional function keys around the spacebar.  In this case, the spacebar would be slightly shorter to accommodate the extra keys.  Now compare the number on the ANSI and the ISO keyboard.  The ISO always has one more key.  That extra key is on the row with the shift keys.

If you have an ISO keyboard, then the left shift key should be shorter than the right shift key to make room for this extra key that the Ù is on.  If you have this type of keyboard with the short left shift key (ISO), then you should not notice any strangeness from the picture you posted.  If however you have an American 101 key keyboard, which I suspect you do, then you will get confused because the keys wont be in familiar places, and that U key is missing.  At least windows allows you to use the multilingual or French, no matter which keyboard you have!!  And the changes are minor.  However, just enough of a difference to create confusion.

Multiple touchscreen monitors not supported by Windows 7 (and - Microsoft Community

ericst01 say:

Multiple touchscreen monitors not supported by Windows 7 (and possibly Windows 8.x?) when in Nvidia Surround mode

I'm posting this as a service to others... would have saved me many hours of frustration, and some money, had this been made clear elsewhere.

Scenario:  3 Dell P2714T touchscreen monitors, connected to a GTX 680, linked as a single virtual display (5760 x 1080 pixels) using Nvidia Surround.  Windows 7 Pro SP1 x64.  All relevant drivers updated and OS updates completed.

Triple-wide display works beautifully, as expected.

Touchscreen (I.e., using touch to execute mouse actions) is not functional, because Windows is unable to correctly identify the correct horizontal touch location in the above scenario.

Something in the Windows OS is unable to sense that there are 3 touch-input devices running adjacent to one another, the moment Nvidia Surround mode is active.  The only area of the 3 touchscreens that retains horizontal touch fidelity is the dead centerline of the middle monitor.  But as one touches either to the right or to the left of center of the middle monitor, Windows "detects" a touch event progressively more laterally than one has actually touched.  It seems to be some kind of mis-scaling problem.  Windows touch can't reconcile a 1920 pixel wide individual monitor screen with a 5760 pixel wide virtual display.  So, for instance, touching 1 cm to the right of the centerline of the middle monitor will register a touch event a couple of centimeters to the right of centerline.  Touching the extreme right border of the middle monitor will register a touch event at the right border of the rightmost monitor.  2X scaling error I'm guestimating? Mirror image situation with touch events mis-scaling when touching to the left of midline of middle monitor.

Would respectfully request that Microsoft re-create the "Scenario" described above.  They can use any 3 touchscreen monitors, Dell brand probably isn't essential.  They could use any Nvidia Surround-capable card.  They could use Windows 7 or 8.x.  Should be able to easily reproduce the problem. 

Hopefully one day Microsoft will come up with an OS fix.  This is not a problem for the monitor manufacturers, or for Nvidia.  It is a problem with improper touchscreen scaling in ultra-wide virtual desktop multi-monitor environments.  I would even go so far as to venture to guess that the solution might be surprisingly simple, by adjusting a scaling factor.

One final note:  Others have noted the proper use of the "Tablet PC Settings" applet in the Windows Control Panel.  That is correct; you can use this applet to get Windows to correctly sense individual touchscreens in a multiple touchscreen monitor environment.  The problem is that you cannot be using Nvidia Surround mode.  (I suspect that the same limitation may apply to AMD Eyefinity, but can't prove that yet.)  If you want your triple-touchscreen-monitor to function as a single virtual display of 4-5 thousand pixels across, Table PC Settings applet will flake out and lose awareness of the 3 touchscreens.  Turn off Nvidia Surround (and simply run as a primary touchscreen monitor in the middle at say 1920 x 1080, with 2 additional 1920 x 1080 touchscreen monitors flanking it on the right and on the left, using simple Windows "Extend Desktop" mode), and Table PC Settings applet will permit you to preserve horizontal touch fidelity across all 3 displays.  The problem is that many of us need to run Nvidia Surround in certain applications to present a single ultra-wide virtual display to the software (simulation, engineering tool, game, etc.).  Microsoft has only solved half of the problem thus far.

Hope this helps.

a. user say:

Drivers need to be installed in a particular order, and motherboard drivers come first. In addition, drivers come from hardware manufacturers - not Microsoft - and that's where you should go to get them. You should always use Windows 10 drivers for your hardware.

You're on the right track. You should know specifically which drivers you need to install and in what order. You should obtain your drivers from the hardware manufacturers and have the installers available on a USB flash drive before you begin installing Windows 10.

Drivers delivered through Windows Update contain a very basic level of support for people who don't know how or are fearful of installing software. You'll get the best performance from your computer if you get your drivers from the hardware manufacturers.

andre da costa say:

If you have access to the native Windows 10 drivers for components such as the motherboard and video card, certainly install those.

Yes, critical drivers such as your motherboard should be installed first, this normally is the chipset driver which you can download in advance from the manufacturers website.

Windows 10 will normally include basic class drivers for the display adapter you can install after.
