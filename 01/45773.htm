a. user say:

If "Autorun for Windows" means the program Autoruns from Sysinternals it can look intimidating if you are looking at the Everything tab because it displays a lot of information but everything it shows does not necessarily mean everyone of the items you see is starting with Windows.

To narrow things down look at only the Logon tab, and then everything under this registry key:

HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

In this example there is only one startup program listed - msseces.exe which is Microsoft Security Essentials:

That will usually match the enabled items you see is the System Configuration Utility msconfig.exe Startup tab.

Note that disabling items in Autoruns or msconfig does not uninstall anything - it just keeps it from starting so you can disable all startup items, some startup items, a few at a time and restart your system after each adjustment to check the results.

If you want to share a list of your startup items to see if there are some things you might be able to disable here is one way to do that:

Download and install the free version of CCleaner from here (pay attention to the installation checkboxes):

http://www.piriform.com/ccleaner

Choose to Customize the installation and uncheck all the options except to put a shortcut on the desktop.

CCleaner will let you manage most Startup items and also do some other cleanup oriented operations.  Some people will recommend you not to use the registry functions in CCleaner so avoid those features for now.  I have no issue with it myself, but it is sometimes a contentious issue I prefer to avoid whenever possible!

You can uninstall CCleaner later if you don't like it.

You can disable startup items, see how things look and if you want - totally delete unneeded startup items, or just leave them disabled while you continue troubleshooting.  

Note that disabling things does not uninstall anything and if there is a problem, you can just enable the item again.  If you are sure you don't want it at all, then you can delete it.

After launching CCleaner and seeing what startup items you can disable/delete if you still are not sure about your startup items, do this:

Launch CCleaner and choose Tools, Startup and in the bottom right, click the "Save to text file" button to save the Startup information to a text file on your desktop with a name you can remember - something like startup.txt

Save the startup information text file to your desktop (or someplace you can find it) open the text file with a text editor (notepad or WordPad), select all the text (Ctrl-A), copy all the highlighted text to the Windows clipboard (Ctrl-C) and then paste (Ctrl-V) the contents of the startup list back here in your next reply and perhaps somebody can tell you other things to consider.

diagnostic says that your computer archetecture is AMD64.

That's standard.  It is also represented by X64.

FWIW my guess is that your main symptom is an example of the difference between having an update installed versus having it applied.  I saw an example of that with a Surface firmware update.  I kept getting it each month and wondered why.  At the time I was using AU's "download automatically and prompt" method of installing updates, so that I could have a better idea of what was going on with them.  Surface Support suggested running AU with it set to fully automatic.  Doing that revealed that a lot more updates than that one had been held "pending restart" even though numerous restarts had been performed which should have been sufficient for them.  E.g. that's the difference between being installed and actually applied. 

To see if this is actually the case for you, you could use the KB article's File Information list and check for version and other file information being consistent with that update being applied or superseded.  However, if there is some other operation that needs to be done by an "apply" step (such as my firmware update) you might still not be able to detect whether it has actually been done.  Then switching to full automatic updates might be the only way to find out if you were seeing the same kind of symptom that I had.

 

Good luck

Robert Aldwinckle
