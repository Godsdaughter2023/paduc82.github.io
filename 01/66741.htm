mark aggar say:

If you use Devconx64.exe (or devcon.exe for 32 bit systems ) from the Windows Driver Kit to enable or disable a device, it will create the necessary hardware profiles and toggle the device between enabled and disabled as you dock and undock.

http://social.technet.microsoft.com/wiki/contents/articles/182.how-to-obtain-the-current-version-of-device-console-utility-devcon-exe.aspx

Once you have extracted devconx64.exe from the WDK, go to the command line prompt (elevated) and run it with the findall * option to list all of the device instanceIDs.

e.g. devconx64 findall * | more

(hint: devconx64 help findall) 

Find your device's instanceID and then disable it in the mode you want it disabled (e.g. if docked and you want it disabled when docked, run the following command when docked)

devconx64 disable 'instanceID'.

The device should now be disabled (check with device manager)

If you undock, the device should re-enable, and vice versa.

It works for me on 8.1 and even on Windows 10 Technical Preview.

mark aggar say: To answer the original posters actual question, you could also use devconx64 in conjunction with the task scheduler to enable or disable devices at logon.
