wunders say:

The error that the certificate cannot be verified up to a trusted authority indicates just that. You listed part of the path up to the certified authority.  The problem usually is that the "trusted authority" usually is one of the root certificates that is installed on your computer.  When versions of Windows newer than XP run into this problem, they simply go out and update the needed certificate.  Windows XP has problems updating its root certificates so after a period of time one company will buy out another and re-issue a new certificate that won't be in your trusted list.  So what you need to do is go out and download a new Root Certificate list from Microsoft and install it on your machine.

 

KB931125 (above) points to the latest update package.  At the moment, the package is located here:

  < http://www.microsoft.com/downloads/details.aspx?FamilyId=6525f124-54fc-4990-bc58-d47d4d02fca9 >

 

Windows requires a validation for this download, so if your machine won't validate, then find another machine that will; go to that link, validate with that machine and download the update package, then transfer it to your machine and double-click on it to install.

 

If you don't update your Root Certificates, you will just run into similar problems downstream and more often as time goes by.

 

HTH,

  JW

Try resetting the software distribution folder:
Before carrying out this procedure, disconnect from the Internet. That means turning off Wi-Fi and disconnecting from a wired connection. If you don't, Windows 10 will indicate some files are in use and cannot be modified or the command will be non-responsive. Also, you should restart your computer after disconnecting from the Internet. The easiest way to do it is to enable Airplane mode from the Action Center - Windows key + X.


1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

2. Stop the BITS, Cryptographic, MSI Installer and the Windows Update Services. To do this, type the following commands at a command prompt. Press the “ENTER” key after you type each command.

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Typethe following commands in the Command Prompt for this. Press the ENTER key after you type each command.

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

5. Type Exit in the Command Prompt to close it and then restart the computer.

cmdrkeene say:

The built-in administrator account is for emergency use only (disaster recover if your actual user account gets broken).  The built-in administrator account is limited: it can't connect to Microsoft Accounts or business accounts, it can't run apps, and it should not be used for "regular daily computing needs".  It's disabled by default and you should only use it if you have an emergency need to use it.
