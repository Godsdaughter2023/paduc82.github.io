https://www.computerworld.com/article/3112358/m...

It is just a file logging error messages. If you have having no problems, you can safely delete it.

This seems to be a problem that comes and goes in Windows. A few other people have reported the same thing recently. In theory, as the log hits a limt, something like 20-30 MB (not GB) it is supposed to be rolled over into a CBSPersist*****.LOG. Then those files are supposed to be compressed into .CAB files. I checked my system and couldn't identify any CAB files that look like they are part of the CBS system. Part of the process seems to have been broken.

Note: these log files can also “overflow” into the C:\Windows\Temp folder as “Cab_xxxx_X” files

One suggested fix is:

* In start menu find services.msc

* Right click on it and select “Run as Admin”

* In the displayed list, find the Windows Modules Installer service, “TrustedInstaller”

* Right click on it and select “Stop” (Do NOT close the services dialog yet)

* Set the startup type to “Manual”

* In File Explorer, go to C:\Windows\Logs\CBS\

* Delete or move or Rename any ALL of the CBS.LOG or Cbspersist_XX.log files

* In file Explorer, go to C:\Windows\Temp

* Delete all CAB_xxxx_x files

* Go back to the Services dialog

* Right click on the TrustedInstaller service again, Select “Start”

* Close the services dialog.

* Restart the computer.

This should stop the broken process that was generating all of these files to fill you drive.

What I really meant was whether you were able to print from your Windows 7 computer right after you first installed the printer.

The reason I ask is because the symptoms you describe are actually fairly common, but the printer normally is able to print wirelessly at the time it is installed even if it stopsprinting some time thereafter.

By default, wireless printers are configured to obtain an IP address automatically from your router.  The problem is that the Windows printing system very rarely is capable of determining if a printer's IP address has changed after installation.  This results in print jobs continuing to be sent to the old IP address and because there is no longer a printer at that address, the print jobs just disappear.

Diagnose as follows.

A.  Determine the IP address that is configured in Windows

  1. Go to Devices and Printers, right-click on the icon for your printer, and select "Printer properties"
  2. On the "Ports" tab, look at the line that is checked and highlighted for the IP address; you probably will have to click and drag on the faint vertical line in the header row between "Port" and "Description" in order to see the entire IP address.
  3. Cancel out of Printer properties at this point

B.  Determine the IP address that is configured on the printer

  1. Do this using the printer's front panel controls
  2. For the HP OfficeJet Pro 8740:

C.  Extra Step

Please also start the HP P&S Doctor.  In Step 1, where you're asked to choose a product, you should see an IP address under the name of your printer.  Please report this IP address.  If the IP address from Step A is not the same as the IP address from Step B, I'm curious to see which one is reported by the P&S Doctor.

If the IP address reported on the printer's network configuration page is not the same as the IP address shown in the "Printer properties" dialog on your computer, follow the directions below.  Otherwise, Reply here for more help.  Include the IP address information from both sources in your Reply.

  1. Go to Devices and Printers, right-click on the printer's icon, and "Remove device"
  2. Click the link at the top of the Devices and Printers window to "Add a printer"
  3. Click "Add a network, wireless, or Bluetooth printer," wait until your wireless printer is detected, and complete the wizard.

Although the above procedure should get your printer working, it may break the scanner function.  If you do these steps, please let me know if you can both print and scan.

If the above procedure enables printing, the proper way to permanently deal with this situation is to give the printer a "static" or fixed IP address.  See page 105 of your User Guide (http://h10032.www1.hp.com/ctg/Manual/c05055346). 

If you do set a static IP address it must be in the same subnet (that is, the first 3 octets of the IP address must be the same as your current IP address).  For example, if the printer's IP address is 192.168.1.101 any static IP address must start with 192.168.1.  The last 3 digits have to be greater than 0, less than 255, and outside the range used by your router to set IP addresses. 

If you want to do this but can't figure out how, tell me the name and specific model of your router (look at the label on the back or bottom) and I'll try to look up the information for you.
