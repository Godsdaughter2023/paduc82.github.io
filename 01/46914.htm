As Leonard mentioned, there isn't an option to capture just a part of your HDV footage. I'm not sure if it is a technical limitation of HDV and the way it is encoded, but it has been this way since Movie Maker started allowing capture of HDV footage.  This is in contrast to DV-AVI footage, where Movie Maker does let you capture just a portion of the video.

1) Same as Leonard (capture the whole tape, then trim to the section you need).

2) Look for other HDV capture utilities (perhaps something that came with your HV40) to get just a segment of the tape. Then edit in Movie Maker as usual.

3) Use a component video or HDMI capture device to play back a portion of the tape and re-encode just that section. The actual solution to your problem is this - While the camera is in HDV/DV playback mode with a tape that has been recorded in HDV, the camera is misidentified by windows and will not give you the option to manually control the camera from the desktop. Therefore, before connecting the camera, go into settings while in playback mode (on the camera) and select DV LOCKED mode from the menu. You will then be presented with full control over the camcorder via the desktop. However, this is not a solution, just a work-around...

Canon HV40

I too am having the same issues with Windows 10 not recognising the Canon HV40 camcorder.

The camera has two playback modes: HDV/DV and DV LOCKED modes.

The issue I believe is this: Whilst connected to the IEEE1394 Firewire port...

SD LOCKED mode, computer uses "msdv.inf" file and is listed in device manager as an "imaging device".

HDV/DV mode, computer uses "mstape.inf" file and is listed as a "sound, video and games controller".

msdv.inf specifically lists the camcorder by name but has no configuration for the camera while in HDV mode, this code is missing. Therefore, mstape is used instead. which incorrectly places the device in sound video and games controllers, thus making it inaccessible for programs such as Windows Movie Maker. And because of this miss-identification, no other video editing software recognises the camera in HDV mode either.

I have attempted to copy and paste specific references to the Canon Camcorder from msdv.inf to mstape.inf without success, as apparently these are system protected files, even when using notepad in "open as admin" role. So, I am at a loss. If Microsoft could provide an updated msdv.inf file which will identify the camcorder whilst it is set to HDV playback mode, that would completely fix the problem. There is no hardware problem or firmware problem with the camera or firewire configurations.
