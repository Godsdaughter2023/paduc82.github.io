balaji kundalam say:

Recycle Bin is a mechanism in Windows to temporarily store deleted files which can be restored later or permanently deleted. Once a file is being moved to Recycle Bin, the file gets stored in $Recycle.bin folder present at the root of each hard drive partition. Inside $Recycle.bin, Recycle Bin is the folder which is the current OS's Recycle Bin folder. Other folders, like S-1-5-21-1648051874-3491260351-376072663-1004, are created by other operating systems (of course Windows, but different user account and different physical computer). In such cases, the user information protects the folder, making current user not permitted to access the folder.

The folder name, Recycle Bin, inside $Recycle.bin with the dustbin icon is governed by desktop.ini (configuration file) present inside the folder. Deleting the file will show Recycle Bin as a folder with name like S-1-5-21-1648051874-3491260351-376072663-1004 with a folder icon. 

To delete all the folders, you have three options:

1. Format the hard drive.

2. Open Command Prompt via installation media and delete the folders.

3. Acquire ownership of the folders and clear/restore the contents.

You can refer the following link for acquiring ownership of files/folders:

https://technet.microsoft.com/en-us/library/Cc753659.aspx

A Code 10 error is generated in Device Manager in one of the following situations:

• Device Manager can't start the device.
• One of the drivers that the device needs does not start.
• Device Manager has insufficient information to recognize the error that is bubbled up by the device driver.
