frederik long say:

Since the 210 GByte unallocated partition is separated from your System partition (drive C:) by the active 300 MByte partition, you cannot extend drive C:. You would first need to move the 300 MByte partition to the far end. 3rd party partition managers can do this but the native disk manager can't.

Here is a laborious but safe way to achieve your aim without using 3rd party tools:

  1. Create an image of your installation. Store it on an external disk.
  2. Burn a Windows System Repair CD.
  3. Boot the machine with this CD.
  4. Check if it can see the image you took in Step 1.
  5. Boot up normally.
  6. Delete the 300 MByte partition.
  7. Extend drive C:. Make sure to leave 310 MBytes free at the ar end.
  8. Create a primary & active NTFS partition at the far end.
  9. Boot the machine with the repair CD.
  10. Select the option to repair the boot environment. It should restore the boot environment.

If the above process fails then you can restore the image you created in Step 1.
