The Dreaded "Hardware Not Connected to Transport Drivers" - Microsoft Community

christopher00000 say:

The Dreaded "Hardware Not Connected to Transport Drivers" Problem

For well over 15 years, I (and a lot of other people) have suffered from the dreaded "Hardware not connected to transport drivers" problem -- also known as "IPv4/IPv6 Connectivity:  Not Connected."

Evidently, when a Microsoft OS is first installed, some function in that installation process recognizes and binds/hooks the network adapters to the operating system and it's transport drivers.  Basically, it connects the clients/services/protocols to the physical adapter, so that the adapter can communicate successfully with the OS.

Once that installation process is over, however...  God forbid that you ever accidentally delete a client or service or protocol -- because it disappears.  Forever...

And if you want to swap out one physical adapter for another -- the system can recognize the new adapter just fine -- but tell you that because it's not connected to transport drivers/IPv4/IPv6 protocols, it doesn't exist.

Which is my problem today.  I recently swapped out an Intel Centrino 1000N Wifi card for an Intel ac7260 on a Windows 7 system.  The Device Manager tells me the card is installed and working.  Network Connections tells me it's not connected, and is unusable.

I have scoured the Internet for days like the Flying Dutchman, looking for a solution.  I have spent hours on the phone with Intel support.  I have tried -- usually 3 or 4 times each -- every solution known/discussed/recommended.  As well as adaptations of my own.

Nothing has worked.

All that's left is reinstalling the OS from scratch.  Which, after having had my computer for several years, is not an option.  With all due respect:  starting over and spending 60-80 hours reinstalling software (some of which is no longer available) and tweaking the system -- is not a really a solution.  It's just a way to make people... go away.  It's, "It's not our problem."

Clearly, something occurs in OS installation that integrates network adapters into the system.  And it's a software issue -- because if you lose your client/service/protocol connections -- a reinstall of a backup of the active partition (by backup software) will reliably bring them back.

If there is a dark, closely held secret of how to do this, or an obscure tool ... I would dearly love to know about it.  Because all the fixes commonly found on the internet -- ALL of the fixes, honest and truly -- simply don't work.

And if there isn't -- it's time Microsoft created such a tool.  (With due respect, the Network Adapter Troubleshooter does not come close to fixing this issue).  Search the internet -- you will see citations for this problem going back, and back, and back.  It's a real problem -- for a lot of people -- with no definitive solution -- for a very long time now.

I mean:  really.

For shame...

It looks like the components of your app have something to do with the error message that you are getting. Here are 2 methods on how to disable Game DVR or Game Bar in Windows 10:

Method 1: Disable Game Bar & Game DVR

1. Navigate your mouse cursor to Start Button, click it to expand the menu. From the expanded menu, chose the ‘All Apps’ entry. It is located at the very end of the menu. Click on All Apps and scroll until you find the Xbox entry. Upon finding it, click the button. If prompted, connect to the Internet.

2. Next, when the Xbox screen appears look for the three horizontal lines in the upper left corner of the Xbox screen – the hamburger menu, and click on it. Now, scroll down to locate the Settings option.

4. A slider denoting the ON position of Record game clips and take screenshots using Game DVR should be visible to you.  Slide it to the Off position for disabling the recording element of Game DVR.

Method 2: How to turn off Xbox DVR through the Registry Editor

1. Run regedit to open the Registry Editor and then navigate to the following registry key:

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\GameDVR.

2. Right-click AppCaptureEnabled and set its value to 0. A value of 1 enables it, whereas 0 disables it.

Next navigate to the following key:

HKEY_CURRENT_USER\System\GameConfigStore

3. Right-click GameDVR_Enabled  and set its value to 0. A value of 1 enables it, whereas 0 disables it.
