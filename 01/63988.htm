You can try the steps below:

3) Reinstall IIS

http://support.microsoft.com/?id=930911

2) While installing IIS, when we check ON the features of IIS, this automatically

    There are two parts to the GPO:

    1. Computer Configuration > Preferences > Windows Settings > Files

    Source file: \\server\hiddenshare$\logo.jpg

    Destination file: %windir%\system32\oobe\info\backgrounds\backgrounddefault.jpg

    Action: Replace

    Remove this item when no longer applied: Yes

    2. Computer Configuration > Policies > Administrative Templates > Control Panel > Personalization

    Force a specific default lock screen image: Enabled

    - Path to lock screen image: %windir%\system32\oobe\info\backgrounds\backgrounddefault.jpg

    Prevent changing lock screen image: Enabled

    On Windows 7 and 8.1 clients, the first part of the GPO is applied successfully - the logo.jpg is copied to the local folder as backgrounddefault.jpg.

    However, the issue comes with the second part of the GPO. Windows 7 clients are happy, and apply the image to their lock screen. Windows 8.1 clients get a solid pale blue screen, with the clock at the bottom left.

    After a lot of testing with different settings in the lock screen image path setting, we found that Windows 8.1 will apply the custom image if you use the UNC path, or an explicit local path (i.e. C:\Windows\system32...), but it will not apply the image if the %windir% variable is used.

    So, is this a known change to the way that Windows 8.1 behaves with this setting, or something else? Given that the first part of the GPO successfully copies the file, we know both OS are happy using the variable, so I can't see why it wouldn't work for applying it to the lock screen.
