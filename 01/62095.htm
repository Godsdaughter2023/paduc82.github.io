I've got .NET Framework 1.1 / 1.1 Hotfix (KB929729) / 3.5 SP1 installed, but not 2.0 (as I have an impression from somewhere that 3.5 included 2.0 runtime or backward compatible with 2.0, hence i uninstalled it). eddiecjc say: Finally, I was able to update to Vista SP2 successfully.

The problem was because of that I had a ramdisk with size 2.75G. I guess it was not big enough for the update cache or something. Because I have a home computer with 4.75G of ramdisk, and it updated to SP2 without any problem.

So I had environmental variables of the user's TEMP/TMP pointed to the ramdisk R:\my_username, and system TEMP/TMP pointed to R:\System\TEMP.
I changed the user's TEMP/TMP point back to the original setting: %USERPROFILE%\AppData\Local\Temp, but left the system ones still as R:\System (because I forgot). Then do the SP2 upgrade after a reboot. Waala, SP2 finally updated successfully!

“Error during access: unknown user or wrong password”
