chriso747 say:

FSX has always been prone to OOMs even though users have had large amounts of RAM installed (as yours: 16GB).   This is because RAM (physical memory) is different to Virtual Address Space (VAS);  this is not the same as Virtual Memory for which you can alter the size allocated.   Because FSX is a 32 bit application, the maximum amount of VAS that can be allocated is 4 GB, for a 64 bit OS as you have.  If you had a 32 bit OS, then the VAS available is down to 2 GB.   So if the 4 GB limit is nearing its limit or exceeded, then the result is an OOM. There's much more to learn about this.  

A good, simple article has been written describing the causes and possible solutions for OOMs resulting from unavailable VAS.   It was written primarily for Win 7 by PMDG, but applies to Win10 also.   You can read about it here:

http://support.precisionmanuals.com/kb/a108/vas-management-stopping-out-of-memory-oom-errors.aspx

What you have done so far seems quite appropriate.   However, you've not mentioned any add on aircraft, scenery add ons (especially photoreal scenery), and other programs running within FSX that you might have.   All these are memory and VAS grabbers.   This article may help you.

I fly the B748 on long-haul routes.   My aircraft are default (though with enhancements and aerodynamic corrections) as is also my scenery.   I still used to get OOMs (first freezing, with the sound going, then no further prospect of recovery except a hard boot).   I initially used to save my flight during its various stages, so that in the event of a freeze/crash, I could reload FSX and get back into it - not a good way of doing things.   Now, during the climb profile, I pause the flight and go into Settings and reduce all Scenery Tab Settings to zero, Weather to Simple Clouds and FPS to 30 or less.  At high altitude, I don't need any of these settings that put a demand on CPU and Graphics.   For me, I've not had an OOM since. Then during descent profile, I readjust those settings so that the scenery comes back to normal.

You mentioned that FSX was running well under Win7 with the same settings but not in Win10.   My feeling is that Win10 is more demanding in its resource use, which of course, does not auger well when trying to run a beast like FSX.  So after going through that article, download Process Explorer to check on VAS usage and you could also try reducing your settings or not use add on stuff until you can find where your problems lie.

Hope you can get positive results.
