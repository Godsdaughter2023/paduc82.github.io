Browsing folders that contain large .tiff files may cause Windows - Microsoft Community

bobbymi say:

Browsing folders that contain large .tiff files may cause Windows Explorer to lag, hang or crash on a 64-bit Windows 7-based computer

When using Windows Explorer to browse folders on a 64-bit Windows 7-based computer, Windows Explorer may appear to lag, crash or hang.This issue can occur if the folder contains large .tiff files of 2GB or larger.In some cases this has been known to happen with large amounts of average sized .tiff files.Try one or more of the steps below, these methods have helped to resolve the issue.

Turn off the Preview Pane in Windows Explorer

1.      Double-click Computer

2.      Click the Hide the preview pane icon beside theGet Help icon in the upper right hand corner of the window

 

Set the Optimize this folder option to Documents

1.      In Windows Explorer right-click the folder that contains the large .tif files

2.      Click the Customize tab

3.      In the Optimize this folder for drop-down select Documents

4.      Click OK

 

Open the large .tif files in Photoshop and re-save the images using LZW compression, or one of the other lossless compression modes that TIFF supports.

 

Bobby

ipconfig and nslookup identified the issue with the address match between my local network and the office network.  I know there wasn't an issue with the remote system (a Windows 7 Professional system), as I used RDP to hit another server in the corp office and then RDP'd from that server to the target system successfully.

Anyone know why it does this?

 

Thanks.

 

This is really an RDP question, not really specific to Win 7.  The client system I was using was XP SP3-based.

al jarvi say:

You would want to have different subnets at either end of the VPN tunnel to preclude these type of issues, ie. your both using 192.168.1.X for example in your case, so the Remote Desktop [RDC] client will have a hard time resolving the address between your LAN and the work LAN. You might consider changing your LAN to something else to preclude issues like this, ie. 172.20.1.X or maybe 192.168.20.X for example...


Now your work network admins could change their network but in reality that probably won't happen...:-)

al jarvi say: Its not a Remote Desktop issue. Its a VPN issue and how you and the work folks have your networks setup...
