The amdxhc.sys is the AMD USB 3.0 driver and the usbfilter.sys is the AMD USB Filter driver.

The amdxhc.sys is dated Jan 3, 2012 and the usbfilter.sys is dated Nov 16, 2011.

Both of these drivers should be updated to the most recent version to ensure compatibility with the motherboard and with each other.

HP has an updated AMD USB 3.0 driver available but I do not see an updated AMD USB Filter driver available for your computer.

I might suggest to try to obtain the motherboard chipset package from AMD to ensure the most recent and compatible versions of both drivers.

You could try the "AMD Chipset Drivers" from the following link which should update both drivers.

http://support.amd.com/us/gpudownload/windows/Pages/raid_windows.aspx#2

I do recommend to create a System Restore Point prior to installing new motherboard drivers.

lemp say:

Thanks for the clarifications.  Some of the issues are easy to answer.

As you probably know, even when you are logged in as an "administrator" user in Windows 7, you are running as a "standard user" unless you explicitly "elevate" your rights.  You usually do this by right-clicking a program, selecting "Run as administrator" and responding to the UAC prompt.

Apparently some, but not all, of the operations in the powercfg command require elevation.  The syntax document (https://docs.microsoft.com/en-us/windows-hardware/design/device-experiences/powercfg-command-line-options) specifically calls out some of these (see, e.g., /systemsleepdiagnostics -- "This command requires administrator privileges and must be executed from an elevated command prompt") but doesn't mention anything about /requests.  That operation does, however, need to be executed from an Elevated Command Prompt.

With respect to the "wrong user" issue, use the WHOAMI command in your Command Prompt window.  Does it return cdmassey or Vc?  I expect that it will return cdmassey (assuming that's the account you're logged into).

The default "prompt" that shows in the Command Prompt window is the drive and path where you are when you invoke the Command Prompt.  

How did you open the Command Prompt?  Did you use Windows Explorer and navigate to C:\Windows\System32, find cmd.exe and double-click it?  Or did you use a shortcut?

If you used a shortcut to open the Command Prompt, right-click the shortcut and select "Properties."  What is there?  The default for "Start in" is %HOMEDRIVE%%HOMEPATH%

Now for the hard part -- why isn't your computer going to sleep?

I expect that there is some application that is active in the background.  I think you're on the right track with powercfg /Requests  Perhaps one of the more elaborate reports described in that syntax document might also help.

If you can't figure it out, reply with the results of /Requests and anything else you use and I'll try to help -- but no guarantees.
