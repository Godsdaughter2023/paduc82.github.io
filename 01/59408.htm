Am I correct in assuming that both Firefox and Chrome were installed & working OK before 28-29 June 2011?

Of the updates you installed on 28/29 June, the only possible troublemaker IMHO would be Optional, non-security update KB2552343 (unless Defender or Avira AntiVir or STUPIDAntiSpyware <wink> detected & "nailed" something during that timeframe).

Unfortunately, KB2552343 cannot be uninstalled so System Restore would usually be the last-resort to correct the behavior (assuming KB2552343 actually caused the behavior and I'm nowhere near convinced that it did...yet).

But there's a Catch-22: Using an available Restore Point which predates the new behavior could "hork" the install of any other updates you've installed, including the updates released on 12 July (i.e. KB2532531, KB2555917 and KB2507938) assuming they have been installed by now so I would NOT encourage or recommend that you use System Restore (yet).

Since Firefox & Chrome still work (if only in a "significantly diminished" manner), other troubleshooting possibilities could include:

2. Seeing if the behavior persists in a "clean boot" environment; cf. http://support.microsoft.com/kb/929135

3. Uninstalling both Firefox & Chrome (followed by a reboot) and then doing clean installs of each. [1]

4. Running the Network troubleshooter and/or working your way through the network connections tutorial and/or posting in the Win7 Networking forum (or possibly the Win7 Performance forum) for additional assistance.
