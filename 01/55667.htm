WMI (Windows Management Instrumentation) is a tool you can use for this. You would need to create a policy to launch the following scripts at boot time on each workstation. These are bare bones scripts only. You need someone with skills in VBScript (or PowerShell) to flesh them out and include the IP address and the user's name. The mailing part is handled by blat.exe which you can download from the Internet. Alternatively you could use CDO within the script itself.


'Monitor the insertion of a flash disk

Set oWMIService = GetObject("winmgmts:\\.\root\cimv2")
Set oWshShell = CreateObject("wscript.shell")

Set cEvents = oWMIService.ExecNotificationQuery _
    ("Select * From __InstanceOperationEvent Within 10 Where " _
    & "TargetInstance isa 'Win32_LogicalDisk'")

Do While True
    Set oEvent = cEvents.NextEvent
    If oEvent.TargetInstance.DriveType = 2 Then
        If oEvent.Path_.Class = "__InstanceCreationEvent" Then
            oWshShell.Run "c:\Tools\blat.exe .. .. .."
        End If
    End If
Loop

'============================

'Monitor the arrival of a large downloaded file

Set oShell = CreateObject("Wscript.Shell")
Set oFSO = CreateObject("scripting.filesystemobject")
Set oWMIService = GetObject("winmgmts:\\.\root\cimv2")

Set cMonitoredEvents = oWMIService.ExecNotificationQuery _
    ("SELECT * FROM __InstanceCreationEvent WITHIN 10 WHERE " _
        & "Targetinstance ISA 'CIM_DirectoryContainsFile' and " _
        & "TargetInstance.GroupComponent= " _
        & "'Win32_Directory.Name=""d:\\\\Downloads""'")

Do
    Set oLatestEvent = cMonitoredEvents.NextEvent
    sNewFile = oLatestEvent.TargetInstance.PartComponent
    sFileName = Split(sNewFile, "=")(1)
    sFileName = Replace(sFileName, "\\", "\")
    Set oFile = oFSO.GetFile(Replace(sFileName, Chr(34), ""))
    If oFile.Size > 1000000 Then oWshShell.Run "c:\Tools\blat.exe .. .. .."
Loop

[quote user="DerDevily"]

Wenn ich  das mit der Heimkonsole mache und den account permament lösche sind die Downloads dann auch weg ?

[/quote]


Downloads sind an den Kauf-Account gebunden, und der Kauf Account kann der Konsole eine Lizenz geben, diese Downloads ebenfalls zu nutzen. Sollte aber der Konsole mal etwas passieren (Blitzschlag, Hardwaredefekt, kleine Kinder und Orangensaft), und die Accounts wären gelöscht, gibt es keine Accounts mehr, die einer neuen Ersatz-Konsole die Lizenzerlaubnis erneut geben könnten.

Da an den Accounts Rechnungsinformationen über den Kauf dran hängen, die je nach Land und Gesetzeslage mehrere Jahre oder Jahrzehnte aufbewahrt werden müssen, kann Dein Account also auch nicht so einfach gelöscht werden.

Und da nur der Account einer Konsole das Recht geben kann, Lizenzen zu nutzen, möchtest Du auch nicht wirklich den Account löschen.

Sollte aber an dem Account kein Kauf dranhängen, dann lösche einfach den Account von der Konsole und vergiß einfach dass er existiert. Brauchst Du die E-Mail von dem Account, dann tausche die Mail gemäß der obigen Anleitung auf einer 360 von einem Freund oder mit Hilfe des Supports.

Falls das nicht Deine Wunsch-Lösung ist, wäre es sinnig, wenn Du uns einfach erklärst, warum Du Dir damals 3 Accounts angelegt hast und was Du bisher damit gemacht hast.
