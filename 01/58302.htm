Following are the steps for configuring Incoming VPN Connection in Windows 7

  1. First go to Control Panel and open Network and Sharing Center.
  2. Click on Change adapter settings.
  3. Press Alt+F and select New Incoming connection
  4. Put a check on who you’d like to give access to this computer or you can configure a new account by clicking on Add someone,after that click on Next.
  5. Put a check mark on Through the Internet. Click on Next.
  6. Select the protocols you want to enable for this connection. Click on Allow access.
  7. Make a note of the Computer name as this will be used by the client to connect to this computer and after that Click on Close.

Following are the steps for configuring Outgoing VPN Connection in Windows 7

  1. First go to Control Panel and open Network and Sharing Center.
  2. Click on Set up a new connection or network.
  3. Select Connect to a workplace then click on Next.
  4. Select Use my internet connection (VPN).
  5. Type the IP address or computer name where you wish to connect and your connection name under the Destination name field.
  6. Enter a User name and password that has been granted access to the host computer.
  7. For getting connected through created VPN connection, you can either right-click on the Network icon in the system tray and select Network and Sharing Center or go to Control Panel and open Network and Sharing Center, and click on Connect to a network.
  8. Now connect VPN Connection windows will appears.

the real kw15 say:

You can read elsewhere what it should do and how it precompiles stuff for the .Net framework that can alternatively be compiled on the fly if you find a way to stop it. But you'll probably be here because like me your computer was rendered useless by the Microsoft Common Language Runtime Native Compiler taking up all memory and thrashing your hard disk. Underneath this are two tasks that I believe start on idle viz. ngen.exe and ngentask.exe.

I searched high and low to find them, looking in Task Scheduler, and trying to kill the offending task that showed up in Task Manager. I couldn't find them.

They can be killed though. After you boot up and before the computer goes idle open a Windows Powershell as Admin (right click on the start button to do that) and have ready to go the following command:

taskkill /F /im ngen.exe

Have a go at taskkill /F /im ngentask.exe too. It'll make you feel better.

If you manage this, you can use your computer again as the disk activity quickly goes down.

I believe, as it seems to working for me today, that you can indeed prevent these tasks from starting. I did find where they live after a while - the location is

C:/Windows/Microsoft.NET/Framework/vX.XXXX

with X.XXXX a .NET Framework version number.

Armed with this information I fired up Task Scheduler, and using the leftmost pane (headed Task Scheduler (local) navigated through Task Scheduler Library > Microsoft > Windows > .NET Framework and lo! the offending tasks were revealed in 32 and 64 bit versions, triggering on Idle. Right click to disable.

The grim implication is that this will need to be done again whenever .NET Framework moves to a new version. 

But I hope this helps.

corkysanchez say:

Deleting a key in the registry is what finally worked for me. The problem was that ngen would try to keep reallocating memory over and over for the same corrupt DLL in the registry. It would eventually use up all the available memory on the computer.

Deleting the key "\HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\.NETFramework\v2.0.50727\NGenService\Roots" worked for me. Stopping the scheduled task or killing ngen.exe is just a temporary measure and doesn't address the problem.

How Microsoft could let a problem like this go on for so long is beyond me. Today was the first day a search resulted in an answer for me.

More information on the problem can be found here https://aloiskraus.wordpress.com/2017/12/13/the-case-of-ngen-exe-needing-50-gb-of-memory/
