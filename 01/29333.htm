a. user say:

I'm sorry to hear System Restore isn't working - I think that might have fixed you up if it is a software problem.  It does sound like a hardware problem, but let's rule out software issues before you spend money getting the hardware diagnosed and, if necessary, repaired (unless your system is still under warranty with Dell in which case I would start a trouble ticket immediately - they may be able to help you on the phone and if not they will walk you through the process of getting a technician to check out the system).

Perhaps something has happened to your keyboard drivers.  Go to the Dell Technical Support site and input your Product Key.  That should bring up information about your specific laptop.  Then check for keyboard drivers applicable to your system and to your version of Vista and download and install them.  Once you have the drivers, you can install them through Device Manager as follows: http://www.vistax64.com/tutorials/193584-device-manager-install-driver.html (or there may be a simple executable file downloaded which you just run - I'm not sure how this driver will be downloaded).  Once done, reboot and see if your keyboard works again.

If that doesn't work, try a Startup Repair by booting to the genuine Windows Vista Installation Disk (or one you can borrow from ANYONE) or from a Recovery Disk.  Here's the procedure: http://www.bleepingcomputer.com/tutorials/tutorial148.html.  To boot to the CD you may need to change the BIOS to make the CD-drive first in the boot sequence.  To do that, wait for the screen that tells you the F key to push to access the boot menu or boot setup.  Push it quickly. Make the changes, save your work, and exit.  Put the CD in the drive and reboot.  When prompted, push any key to boot from the CD.  When you reboot, does the keyboard now work.

If you don't have either disk, you can make a bootable Recovery Disk using http://neosmart.net/blog/2008/windows-vista-recovery-disc-download/ along with burning software like: http://www.snapfiles.com/get/active-isoburner.html and, of course, a blank CD.

If not, try a clean boot to see if some other program is interfering with the keyboard.  Here's the procedure: http://support.microsoft.com/kb/929135.  If the keyboard functions from a clean boot, then it's just a matter of finding the culprit causing the problem.  Follow the procedures in the article.  Once done, be sure to return the system to normal operating status.

If clean boot doesn't work, try booting into Safe mode (repeatedly clicking F8 during the boot process until the menu appears) and see if you can use the keyboard there.  This may be very helpful information in narrowing down the cause and solving the problem (if it isn't a hardware problem).

If that doesn't help, let's check some of your system files and hard disk for corruption.

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.  Try to post any corrupted files here so we can see if they can be repaired with good copies from the installation disk (unless there are too many).

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It may want to schedule itself to start at the next startup.  Agree and then reboot so the program can run. It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a potential cause.  Reboot when done and see if the keyboard now functions.

I hope this helps.

Good luck!

Windows XP will not use Windows Update (remember, it's been End of Life since April 2014)!

If you're using Windows Update from within Windows 7, that will only get the Windows 7 driver, not the XP driver.


Follow these steps (from the thread I linked to earlier) and let me know where things break down.

  1. Start XP Mode
  2. Download the XP driver from the link I gave you or use this direct link:  http://ftp.hp.com/pub/softlib/software13/printers/OJ8710/OJ8710_Basicx86_1881.exe
    Download it using IE from within XP mode and save the file to the XP Mode Desktop
  3. Make sure the printer is turned on and connected to the Windows 7 computer (the host system)
  4. In XP Mode, click USB in the menu bar.  You should see the name you assigned to the 8710 when you installed it in Windows 7.  The menu will probably show that the device is "Shared" -- like this:

  5. Click on that line and click "Yes" in the window that warns you that attaching a device will disable in the host system:
  6. Run the file that you downloaded from HP (OJ8710_Basicx86_1881.exe).  This will install the printer in XP Mode but it will be installed in such a way that you will only be able to use the printer when it is "attached" to the virtual machine -- which will make it unavailable in Windows 7.
  7. Release the printer from the virtual machine by clicking USB in the XP Mode toolbar and selecting Release-device name.
  8. Restart the virtual machine.   From the virtual machine window, click Action and then click Restart.
  9. Back in XP Mode, click Start, and then click Printers and Faxes.
  10. Under Printer Tasks, click Add a printer. The Add Printer Wizard opens.
  11. On the Local or Network Printer page, select Local printer attached to this computer and clear Automatically detect and install my Plug and Play printer.
  12. On the Select a Printer Port page, click the drop-down control and select one of the TS number ports and click Next.
  13. You should now be able to find the driver for your OfficeJet 8710 in the list.  Select it
  14. Select "Keep existing driver" and complete the Wizard.
  15. The printer that is installed will have the same name as the printer installed in step 6 with "(copy 1)" appended to the name. 
  16. After you confirm that you can print from the "(copy 1)" printer with the printer "shared" between XP Mode and Windows 7 (that is, you can print from both XP mode and Windows 7), I suggest that you open Printers and Faxes, right-click and "Delete" the instance of the printer installed at step 6 and then right-click and "Rename" the remaining printer to delete "(copy 1)" from its name.

smith animal say:

Hi Everyone,

Sorry for the later reply, I did not realize that there were responses actually being posted on this thread. An update on my issue personally; I realized the Xbox One reformatted my external hard drive on it's own without any noticeable prompts. What this did was format the drive to be compatible with games rather than movies or music. After being reformatted I could not get other devices to even recognize its existence such as my laptop or desktop computers. After doing a bit of research I was able to re-partition and reformat the hard drive back to it's original form and reload my movies / music. This process was a lot more complicated than the usual simple reformat. After I plugged it back into the Xbox One I actually got the prompt to reformat this time around and declined. Media player from that point on actually recognized the data on my external and everything worked properly.

Make sure Xbox One did not reformat your external to recognize games only. Note I am also using a Seagate 1TB external hard drive.
