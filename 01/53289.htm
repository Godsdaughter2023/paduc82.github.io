lemp say:

If you're just concerned with data files (Word docs, Excel files, music files, photos, etc.), then you can just copy them to the external drive, connect the drive to the new computer, and copy them from the external drive to the new computer.

Data files don't care if the operating system is 32-bit or 64-bit (or if it is Windows 7 or another version of Windows, for that matter).

Always use copy rather than move.  Once you've determined that the files are in their desired new location and not corrupted, you can go back and delete them from the prior location.  "Move" copies and then automatically deletes ... which can be a problem if for any reason the copy doesn't work properly.

You're correct that Windows Easy Transfer will NOT move programs.  You will have to install the programs from a CD/DVD or a downloaded installer file.  Before you remove something like Microsoft Office from a system on which it is working, make sure that you have the product key so that you can install it on the new computer.  If you don't have the product key on something like the CD/DVD jewel case, use a free utility such as Magical Jellybean Keyfinder (on the original computer) to find the product key.  https://www.magicaljellybean.com/keyfinder/

Windows Easy Transfer didn't work going from Windows 7 to Windows 10, so when Microsoft first released Windows 10, it arranged for a discounted version of Laplink's "PCMover" to be available to help with the transfer to Windows 10.  I don't recall exactly what the deal was then, but now the only discounted version is "PCMover Express" which, like all other such applications, does not move programs.  Laplink has, however, fancier versions of PCMover that purport to be able to move applications.  The "Home" version currently is $39.95 and the "Pro" version is $59.95, although Microsoft offers a discount on the latter to $49.95

http://web.laplink.com/feature_overview/

http://pcmover-10.laplink.com/

Disclaimer:  I have not used any version of Laplink's PCMover.  I'm only repeating the claims made by both Microsoft and Laplink about the ability to move programs.  I do know, however, that Laplink is a reputable company.

dutch32167 say:

In my experience as a copier technician, the WSD control is not the best way to set up a network printer or copier.  It is not actually a port, it is just listed as such.  What I have found to be the best most reliable method is to give the printer or copier a static IP address, then set up a TCP/IP port to use for the device.

To add a new port for an printer or copier that is already installed... (In Windows 7)

Start>Devices and Printers

Right click on the printer or copier and select "Printer Properties"  (not Properties at the bottom)

Click on the Ports tab, and click "Add Port"

Select Standard TCP/IP Port and then click "New Port"  (The Add Standard TCP/IP Printer Port Wizard opens)

Ensure that the copier or printer is turned on and has a static IP address...  click Next

Type the IP address of the copier or printer into the Printer name or IP Address box.  (Port name is automatically filled in)  Click next and the Wizard should give you a completion.

If not, you may have to browse a list of manufacturers to select the network card that your copier or printer uses, or you can select custom and choose between Protocols (raw or LPR) and to enable SNMP for your alerts.  (note that you may not necessarily get those alerts with the custom setup, if that is the case, I recommend uninstalling and reinstalling the copier or printer manually, and selecting the drivers yourself by browsing to the drivers on the disk that came with it or ones that you have downloaded from the manufacturer's website.

To add a new copier or printer in Windows 7...

Start>Devices and Printers>Add a printer>Add network, wireless or Bluetooth printer

(Windows will start searching for a copier or printer)  If you click on the device when it shows up and add it automatically, windows may very well install it with the WSD controller and automatically pick the driver.)

To manually install the copier or printer, select "The printer that I want isn't listed."

You now want to select "Add a printer using a TCP/IP address or hostname" and click Next

In the Device Type drop down box, select TCP/IP device and then type the IP Address of the printer or copier into the Hostname or IP address box.  (The port name box is automatically filled in)  At this point, you can have windows automatically install the driver or you can leave that box unchecked and manually select a driver from the list or browse to a folder with the correct driver.  If your printer or copier is printing gibberish, it's probably because of an incorrect driver.

The X1 Internal HDD is SATA II 5400rpm drive.

USB 3.0 after latency and overhead has about 3.2Gbps.

SATAII has 3.0Gbps of bandwidth.  

SATAIII has a 6.0Gbs of bandwidth.  

When connected to a USB 3.0 /or directly connected to a SATA II cable (like X1 internal drive)   in real world test a 5400rpm spindle drive read/write rate is around 60 - 80 MB/s and a 7200rpm is around 90 MB/s to 100 MB/s.  There is only a slight difference when it comes to load time.

To take full advantage of the USB 3.0 or SATA II cable and see a load times difference you have to go with SSD.  It will be doing read/write speeds  at 300MB/s.

Issue Summary:

We are aware of a potential issue you  may encounter with recent Office Security updates.   When installing these updates, the Windows Update control panel returns the error 646 or 0x80070646.

 

Technical Details:

The error 0x80070646 or 646 is an MSI installer error that will only occur when the updates are installed via the Windows Update control Panel.  Previous workaround to resolve the immediate problem requires the installation of the update from a standalone package from the Microsoft Download Center.  We do not recommend this workaround.

   

If Windows Installer logging is enabled, the following line should be noted as a symptom:

 

Error 1606. Could not access network location %APPDATA%\

 

 

 

Cause:

The problem is potentially a result of an inconsistent/incorrect environmental variable (%AppData%) as shown below:

 

 

Solution:

1.       Open the Registry editor.  To do so, Click Start, type regedit in the Start Search box, and then click regedit.exe in the Programs list.In the User Account Control dialog box, click Continue.

 

2.       Navigate to the following registry keysHKEY_USERS\.DEFAULT\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders

 

3.       The correct and default value forAppData should be %USERPROFILE%\AppData\Roaming. To change the value, double click on Appdata, then change the string in the value data field.

 

 

4.      Close the Registry editor, and try again Windows Update again. (A reboot shouldnot be required.)

 

5.       If the problem persists, repeat the steps 2 through 4 with the following registry keys:

·        HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders\

·        HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Explorer\User Shell Folders\
