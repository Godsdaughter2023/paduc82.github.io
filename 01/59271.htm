Hi FG,

I'm Bulldog, and the moderators of this community forum have designated me a "Launch Expert," which only proves how little you have to know to be considered an 'expert' around here. But I know how to help you. This really works.

First you need to know about DNS - the Domain Name System. On the internet, websites are identified by numbers, so when your web browser tries to connect with a website, it looks for the website by its number, not its name.

That's fine for a web browser, but it would be very inconvenient, if not virtually impossible, for us humans to remember the number for every website we want to visit. For humans it's much easier to remember names, like Amazon, Netflix, YouTube, etc.

When you ask for the Amazon website, for example, your browser gives that request to a 'domain name server' that tells your browser the corresponding number for the Amazon website, and then your browser goes to that number on the internet. All this happens in the blink of an eye so you don't notice it.

Your ISP operates a domain name server that processes all your web page requests ... but you don't have to use it. You can use a different DNS service, and this is how you block websites.

I happen to use a DNS service called OpenDNS, but there are plenty of other DNS services just like it. What makes OpenDNS unique - did I mention that it's free? - is that it offers filtering. When your web browser asks for a website that is filtered by OpenDNS, the service simply refuses to provide the numerical address for that website so you'll never see it.

You can use OpenDNS' own filters, which you can customize to your liking, and/or you can specify specific websites that you don't want to see, and OpenDNS will filter them. There's only one caveat: The filtering applies to every computer that connects to the internet through your router, so if a website is filtered for one person, it's filtered for everyone.

Troubleshooting Windows Backup Failure Access is Denied 80070005 - Microsoft Community

bigmog say:

Troubleshooting Windows Backup Failure Access is Denied 80070005

I ran into an issue attempting to using Windows backup to backup my data to a NTFS hard drive connected to my router. Every time that I ran the backup, it failed with error: Access is denied. 80070005

80070005


I checked the permissions on the destination folder and everything looked correct. I was not using authentication. "Everyone" has full access.


I found many other posts from others with this type of issue but never a good answer. When in doubt? Use Process Monitor!


Using Process Monitor from sysinternals.com , I started a capture and re-ran the backup. It failed again but this time I had a detailed log of events.


I created a some new filters to sort out which events were important:


Process Name: svchost.exe

Path includes 192.168.1.1 (the IP address of my router where the hard drive is connected)

Result: Access Denied.


Here's what I found:



The backup was failing because it wasn't able to create a folder at the path specified. I checked the permissions on that folder and sure enough, there was no write access granted:



Windows wouldn't let me change the permissions. Instead I used putty to ssh to my router and saw there too that write access was missing from the folder I was trying to backup to.


chmod 777

solved the problem nicely. I re-ran the backup and it completed without error.


The lesson here is unix permissions and windows permissions are different and don't work well together. If you have a unix based network device with a NTFS drive mounted, watch out.


Also, Process Monitor saves the day again.


I understand this topic is a bit technical but I hope it helps some others having this problem.
