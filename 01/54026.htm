Various User Accounts in Windows XP Professional

In Windows XP Professional ,  what  is the Logic behind enabling "Power Users" Account Holders (apart from , Of Course , Administrators) to run Legacy Applications whereas people with plain User Accounts & Guest Accounts cannot run the same ?

What are the typical Workplace Scenarios where :

wunders say: "Administrators", "Power Users", and "Users" are nothing more than groups to which individual users can become members.  By default, each of these groups is given certain privileges.  "Users" have the fewest privileges, "Power Users" have more privilege, and "Administrators" have the highest privileges.   "Guest" is simply a specialized user with less capability than any other user.  There is also the built-in "Administrator" user that can't be deleted and by default is a member of the "Administrators" group.

A person who just needs to use pre-installed programs like Microsoft Office and Internet Explorer would probably only need "User" privilege.  Power Users can alter certain system settings that Users can't, and Administrators can change file access privileges and add/delete other users on the computer as well as install applications.

A set of privileges is pre-defined in Windows but can be changed by an administrator.  Read the following article for a description about which users have which rights by default:

"User Rights"

64-bit:
C:\Windows\regedit.exe

32-bit:
C:\Windows\SysWOW64\regedit.exe

The same thing with the registry changes under 64-bit machine
