Eine 100%ige Garantie kann es dafür nur geben, wenn Du die Filme selber ins MKV Format komprimiert hast (und dann hättest Du die Videos auch in andere, für Dashboard und Media Center kompatible Formate konvertieren können).

MKVs die Du aus dem Internet ... ähm... "bekommst"... sind wie eine Packung Pralinen. Du weißt nie, was Du bekommst. MKV ist eben "nur" ein Container, der wie AVI eine fast beliebige Kombination aus Video und (auch mehreren) Audiospuren, sowie diverser Untertitel enthalten kann. Je mehr der Container also enthält, und je OpenSource fanatisch derjenige war, der die Audiospuren als OggVorbis ins MKV gepackt hat, desto geringer sind die Chancen, dass es auf der Xbox trotz Windows 7 und MKV Kompatiblem Divx Codec läuft.

Insofern: Die Chance mit Windows 7 Deine MKVs auch auf der Xbox zu sehen, sind zumindest höher als unter Vista/XP. Aber eine 100%ige Garantie kannst Du Dir nur selber geben, indem Du selber die Videos erstellst.

[quote user="Champ160979"]

Möchte nicht um sonst alles formatieren;-)

Dann hast Du noch nicht "Arminators nicht wirklich patentierte Methode zur Windows Installation ohne Formatieren" probiert. Wenn Deine Festplatte genug Platz hat, einfach mit BartPE oder dem Rettungsmodus von Vista/7 (in diversen c't Zeitschriften beschrieben) auf Deiner Festplatte einen Ordner "_backup" anlegen (der Unterstrich nur deshalb, damit der Ordner immer an erster Stelle im Dateifenster steht). Dann verschiebst Du den Inhalt Deines Systemlaufwerks in diesen Ordner.

Jetzt von der Windows 7 CD booten, und OHNE FORMATIERUNG Windows 7 installieren (beachte, dass es eine "reine" Windows 7 DVD sein muss, KEINE RettungsCD eines Computerherstellers, der einfach ein Festplattenimage drüberbügelt!).

Sobald Windows 7 läuft, und Dein Benutzer eingerichtet ist, verschiebst Du Deine "alten" Benutzerdaten aus dem "_backup\Users\(DEIN VISTA BENUTZERNAME)" in Deinen aktuellen "Windows 7" Benutzerordner und hast innerhalb weniger Stunden ein neues System wie vorher. Habe irgend wo gelesen, dass die xbox nur Dateien bis 4GB unterstützt. Ist das richtig?

Jein. Wie MasterScorp schon andeutet, sind die 4GB eine Beschränkung des FAT16/32 Dateisystems. Und da die meisten Leute ein Windows System haben, sind sie auf FAT beschränkt, was maximal 4GB große Dateien verwalten kann. Streamst Du die Videos übers Netzwerk von Deinem Windows PC mit NTFS Dateisystem, kannst Du größere Dateien benutzen.

the c. say:

Hello flashgordonjxn,


Who ever does the install of Windows, Setup requests that you enter your name, and optionally, the organization name. This registration information is displayed under Registered to in the General tab of System when you click Start, click Control Panel, click Performance and Maintenance, and then click System.

To Change it see instructions below:

  1. Click Start, and then click Run.
  2. In the Open box, type regedit, and then click OK.
  3. Locate the following registry key:
    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion
  4. To change the company name, do the following:

    In the right pane, double-click RegisteredOrganization. Under Value data, type the name that you want, and then click OK.
  5. To change the name of the registered owner, do the following:

    In the right pane, double-click RegisteredOwner. Under Value data, type the name that you want, and then click OK.
  6. Click Exit on the File menu to quit Registry Editor.

You have your choice; install either the 32bit or 64bit version, but NOT both at the same time.

One License is for ONE Installation.

And if you install either the 32bit or 64bit, you would then need another License to install Windows in a Virtual Machine on the same machine.

The License is for the operating system, not the computer/laptop.

irfan h say:

Hello SngBrdB,

If any of the software is currently using this file, it will be immediately recreated on your C drive.

If you are using any Third Party Defragment program, I suggest that you uninstall the program and then try to delete the journal.

Note: Deleting the change journal impacts the File Replication service (FRS) and the Indexing Service, because it would require these services to perform a complete (and time-consuming) scan of the volume. This in turn negatively impacts FRS SYSVOL replication and replication between DFS link alternates while the volume is being rescanned.

Check out this article that will give you more information on this:

http://technet.microsoft.com/en-us/library/cc788042(WS.10).aspx

Thanks,
