Turning off notifications - Microsoft Community

keith patton say:

Turning off notifications

Alphajax uses three types of push notifications...

Tile notifications - Used to update your tiles (primary tile and game tiles). You can turn these off via help and options in the game, these are sent by the AlphaJax servers.

In Game notifications - Used to update your games whilst your are app is running. You can turn these off via help and options in the game, these are sent by the AlphaJax servers.

Toast notifications - Used to notify of new game requests, Your Turn, You Won, You Lost notifications. These are controlled via your Xbox "Show Game notifications" settings. To change these settings on your phone go to Games Hub > ... > settings > Show game notifications (switch to Off). Changing Xbox settings may take a while to fully propogate so may not have an instant effect. 

Please note that unfortunately there is no way to control Xbox toast notifications on an app by app basis. So you cannot control toast notifications for an individual app such as AlphaJax at this time. This is not something the application developers have control over at this time.

Please note users do not receive random game requests after 14 days of inactivity on Alphajax.

a. user say:

Pretty much every 32 bit Windows call is available in 64 bit libraries as well.

Don't assume Kernel32.dll is 32 bit.  Yes, it's confusing.

The Kernel32.dll found in C:\Windows\System32 is really 64 bit code on an x64 machine.
The Kernel32.dll found in C:\Windows\SysWow64 is really 32 bit code on that same machine.

I know nothing of the LabVIEW software you're using.  I use Visual Studio to build applications, and you can build virtually the same Windows code into either a 32 bit or 64 bit executable, and virtually all of the system libraries have the same names for 32 and 64 bit builds.

Unable to create System Image After Upgrade to Windows 10 - Microsoft Community

jsiepmann say:

Unable to create System Image After Upgrade to Windows 10

After upgrading to Windows 10 from Windows 8.1, I was no longer able to do image backups after going to Backup & Restore and clicking on "Create a system image" or doing it as a part of my regular file backup by checking the box that asks if you want to include it in your regular backups. The problem turned out to be a permission problem and if you go to  Control Panel> Administrative Tools>Event Viewer you will probably see an error with Event ID: 513 which looks like this:

Cryptographic Services failed while processing the OnIdentity() call in the System Writer Object.

Details:

AddLegacyDriverFiles: Unable to back up image of binary Microsoft Link-Layer Discovery Protocol.

System Error:

Access is denied.

The following is a fix for this problem:

Step 1. Download the Accesschk program from http://download.sysinternals.com/files/AccessChk.zip . Create a folder C:\Temp and then download and unzip the accesschk.exe program and put it in the C:\Temp folder.

Step 2. Right click on the Windows icon in the task bar and click on Command Prompt (Admin) and then at the prompt type:


C:\temp\accesscheck -c mslldp

You will need the fix if your output is:

mslldp
  RW NT AUTHORITY\SYSTEM
  RW BUILTIN\Administrators
  RW S-1-5-32-549
  R  NT SERVICE\NlaSvc


Step 3. Type in the following command at the prompt and press Enter.

SC sdshow MSLLDP

The output should look like this:


D:(D;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BG)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SY)(A;;CCDCLCSWRPDTLOCRSDRCWDWO;;;BA)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SO)(A;;LCRPWP;;;S-1
-5-80-3141615172-2057878085-1754447212-2405740020-3916490453)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)

Step 4. If your output looks like that then copy the following into notepad (this gets rid of any formatting characters) as a single line (the "sc sdset …..WD" needs to be one continuous line) and then copy it from there to the command prompt and press Enter.


sc sdset MSLLDP D:(D;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;BG)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SY)(A;;CCDCLCSWRPDTLOCRSDRCWDWO;;;BA)(A;;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;SO)(A;;LCRPWP;;;S-1-5-80-3141615172-2057878085-1754447212-2405740020-3916490453)(A;;CCLCSWLOCRRC;;;SU)S:(AU;FA;CCDCLCSWRPWPDTLOCRSDRCWDWO;;;WD)


Step 5. To make sure it worked type in the following command at the prompt to and press Enter.


C:\temp\accesschk.exe -c mslldp

Your output should now look like this with the last line now added on: 

mslldp
  RW NT AUTHORITY\SYSTEM
  RW BUILTIN\Administrators
  RW S-1-5-32-549
  R  NT SERVICE\NlaSvc
  R  NT AUTHORITY\SERVICE

Step 6:  YOUR DONE! This should correct the error messages and allow you to do backups including images.
