cmdrkeene say:

The built-in "administrator account" (the one that actually has the name of 'administrator') cannot be linked to a Microsoft account.  So you won't see the unlink option when logged into that account.

The only resolution for you at this point would be to crease a new local-account.

pixelfields say:

You say the build-in Administrator account can't be linked to a Microsoft account - so why can't I change my password?

Using a cmd prompt as admin, if I try to change the password with net user administrator [new-password] command I get this error: 

"The system is not authoritative for the specified account and therefore cannot complete the operation. Please retry the operation using the provider associated with this account. If this is an online provider please use the provider's online site."

i.e. this account is linked as a Microsoft account and can't be changed locally

All I want is to remove (or at least change) my local Windows password without affecting my Microsoft account.

cmdrkeene say:

The built-in administrator account is not accessible by default.  Even if you override that and login, you should not be able to join it to an online account.  The built-in administrator also blocks access to the store and most universal apps.

If you somehow figured out how to override that and force it to link anyway, all I can really say there is congratulations on that accomplishment.  I can't imagine how to undo it, it shouldn't have even been possible in the first place. The best I can suggest is to create a new local administrator account and delete the old misconfigured one.

For any account that is connected to an online account (Microsoft Account, or a domain/workplace/school account), it is not possible to have a separate local account.  That is true regardless of whether or not the local account is a member of the localadministrators group or not.  The alternative is to use a secondary login method locally, such a finger, face, picture, or PIN number.

irfan h say:

Hey Pdyches,

Thank you for visiting the Microsoft Answers community forum.

You don’t need to reinstall windows. This is an issue related to File associations. Something must have corrupted your file associations.  

First, let's try the system file checker to try and fix any corrupted system files that may have caused this issue:

To run the System File Checker tool, follow these steps:
Click Start, and then type cmd in the Start Search box.
Right-click cmd in the Programs list, and then click Run as administrator.
If you are prompted for an administrator password or confirmation, type your password or click Continue
At the command prompt, type the following line, and then press ENTER:
sfc /scannow
When the scan is complete, test to see whether the issue that you are experiencing is resolved.

If it does not fix, please try this:
To fix this you will need to change the file association for the exe files on the computer. Here is a link to the article that will give you the information on this.
This will automatically fix the .exe file associations:
