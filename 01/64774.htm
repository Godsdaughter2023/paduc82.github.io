How are you connecting to the internet?

In most cases it would be through a router possibly provided by the ISP. This router issues your computer its IP settings including the DNS server address which is probably the router itself. MS were very clear about it ... WLM 2012 will cease to 'work',       

They stated that very clearly, yes, but what many users didn't realize from this disinformation campaign was that it was only one segment of the program that wouldn't work as before - the connection to Microsoft servers. I made an attempt to demystify the misleading information coming from Microsoft in this article:

... gmail says the client is not secure        

No. Google has a concept of 'less secure apps', which includes any application that accesses a Google account without using their preferred method of authentication, OAuth. You can see just how secure this is here, for example: OAuth 2.0 Hack Exposes 1 Billion Mobile Apps to Account Hijacking. There is no problem with the 'security' of Windows Live Mail.

... Outlook just didn't work,        

When an Outlook.com account has been migrated to the Outlook Mail platform, it will have to be set up again in Windows Live Mail. Until this has been done, it won't work, as you discovered. This is all documented in detail in this article: Windows Live Mail and Outlook Mail

Windows Live Mail hasn't been 'supported' since 2010. There has been no product team to support it. Stating that the program 'will reach end of support** on 10 January 2017' is meaningless, and serves only to sow doubt in the minds of the program's satisfied users.

* excluding the last update, that is, whose only purpose was to remove all references to SkyDrive in the program code and replace them with references to OneDrive.

sumit (independent advisor) say:
Activation depends on what type of license you have of Windows 7/8/10.

If Windows 7/8/10 came preinstalled with the computer, that is called an OEM license. For an OEM license, the license dies as soon as Motherboard dies. So essentially it is non-transferable. If that is the case you need to get a new license.

However, this is not same for retail licenses i.e. you purchased the license through Microsoft store. That does carry transfer rights. If that is the case, please see the below steps:

Run the activation troubleshooter on the new device using the below steps:
Go to Settings>Update and security>Activation and run the troubleshooter.
Select I recently changed hardware on this device.
Please note that Activation troubleshooter requires you to have digital license linked to Microsoft Account.

Alternatively, if that does not help, you would have to call Microsoft Support.
Contact Microsoft using the phone numbers below or use the chat support link(weekdays only)
https://support.microsoft.com/en-in/help/405170...

https://partner.support.services.microsoft.com/...


Please let us know if you need any assistance.

a. user say:

Can you give me some examples of the .exe files it's giving you this error about - I assume it's more than one given how you phrased your question.  This generally is caused by some type of corruption in the system files (and no, don't expect to hear from Microsoft anytime soon - LOL).

Do you know when this problem began?  Try a System Restore to a point in time BEFORE the problem began.  Here's the procedure: http://www.howtogeek.com/howto/windows-vista/using-windows-vista-system-restore/.  Be sure to check the box to show more than 5 days of restore points.  If the first attempt fails, then try an earlier point or two.  NOTE: You will have to re-install any software and updates you installed between now and the restore point, but you can use Windows Update for the updates.

Hopefully one of these procedures will resolve your problem (probably the System Restore or the Startup Repair).  If not, post back and we'll try something else.  Post back in any case so we'll know how it turned out.
