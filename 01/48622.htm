lemp say:

That means that your computer is doing nothing ("idle") three-quarters of the time.  That shouldn't be causing the fan to run loudly (which is what I assume you mean by "The noise is so loud that the computer seems be in a Race Mode").

Please click the Start Orb, type msinfo32.exe into the Search box, and press Enter.

When the System Information window populates, click Edit > Select All and then Click Edit > Copy

Then right-click in a Reply window here in the forum and Paste the data from System Info into your reply.

Svchost is, as the name suggest, a process that "hosts" one or more servicing routines that don't themselves rise to the level of a "process."  Usually, when svchost is taking a big chunk of the CPU's time, the culprit is either Windows Update or an antimalware program doing a system scan.

Right-click on the instance of svchost.exe that is using the large amount of CPU (because it's normal to have several instance of svchost.exe running at the same time, make sure you right-click on the correct one) and select "Go to service(s)."

You'll be switched to the "Services" tab of Task Manager with the services hosted by that instance of svchost.exe highlighted.  What are they?

canadian tech say:

In my recent experience, this is most likely happening because Windows Update is operating in the background.  To fix this:

  1. Go to Windows Update
  2. Change the setting to Never
  3. Restart your computer
  4. After the restart, click the start globe, type service into the text box that pops up
  5. Click on Services in the list
  6. In the Services window, find  Windows Update, Right-click on it and choose Stop.

I advise you to leave the WU setting at Never and they you take the responsibility for manually doing Windows Update when you choose to.

OK, I am going to suggest to remove the driver completely.

Create a System Restore Point first.

Then go to the following registry key (or the renamed registry key):

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\DbgMsg

Right-click on DbgMsg (or the renamed folder) and select "Delete" and then delete it.

Then go to the folder on the computer:

C:\Windows\System32\Drivers

In that folder locate the DbgMsg.sys.
