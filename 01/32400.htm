I noticed a Microsoft Bridge connection that I could not recall ever creating, and Google searches suggested was not necessary for my network. I disabled it. Then on the same screen, I went to my Wi-Fi connection, right clicked on it and on the context menu found that it was using the Bridged Connection. I disabled its use of the Bridged Connection. So I checked most of them - the ones I was pretty certain I needed - and then clicked OK. Lastly, I right clicked on the Wi-Fi connection again and disabled it, then re-enabled it to make sure the configuration was refreshed. When I attempted to connect to my home wireless network again, it worked!

Like I said, I have done a lot of tinkering over the past 24 hours when I first discovered the broken wireless behavior. And I was seriously contemplating resetting my device, which I did not want to do. So this was a big relief. Why the Bridge network was even there, I don't know. But the fix, in the end, was relatively easy. Hopefully this works for others.

no.compromise say:

Your bugchecks suggest you are experiencing memory corruption.  This can be caused by faulty hardware such as RAM, or a bug in a driver.

Some general guidance for this type of scenario is to consider testing memory with Windows Memory Diagnostic or memtest86. Note that memory that passes tests is not necessarily good memory - it just hasn't failed a test.  Consider systematically eliminating RAM from the system - run with a couple of modules for a while, and see how things go. Then try the other modules.

To see if a driver may be involved, consider enabling Driver Verifer.  Click start->type verifier.exe, and press enter.  Enable standard settings for all non-Microsoft drivers (it helps to sort by Provider and CTRL+Click to select multiple drivers).  Then reboot and wait for the next bugcheck, which will hopefully have more details, with verifier enabled.  Upload the crash dump to SkyDrive and provide the link.

In the event you are not able to boot successfully after enabling verifier, boot into safe mode, run verifier.exe again, and choose to Delete existing settings. Then reboot.

Another thing to consider is making sure that you have the latest BIOS available, and that you're using the latest drivers available.

Try eliminating some RAM modules from the system, not just moving them around.

Also consider running SFC /SCANNOW.

If you boot into safe mode, do you still experience bugchecks? What about if you do a clean boot, or device clean boot?
