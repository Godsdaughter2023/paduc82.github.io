andre da costa say:
Windows 7 System Builder licenses are to be used by certified System Builders who are reselling licenses. Technically, you are out of compliance if you are the one who is using the license. There is nothing preventing you from using it, just that you are not adhering to the distribution procedures.

OEM versions of Windows 7 are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system 

OEM licenses are to be installed by professional system manufacturers only. Under Microsoft's OEM License Agreement, they are not to be sold to end-users under any circumstance, and are to be preinstalled on a computer using the OEM Preinstallation Kit (OPK) before shipment to the customer, and must include at the very least the manufacturer's support contact information. They are, therefore, designed for installation only on a single computer and are not transferable, even if the original computer is no longer in use. This is not usually an issue for users who purchase new computer systems, because most pre-assembled systems ship with a preinstalled operating system. There are few circumstances where Microsoft will allow the transfer of an OEM license from one non-functioning system to another, but the OEM System Builder License Agreement (SBLA), as well as the OEM End User License Agreement (EULA) do not contain any allowance for this, so it is entirely up to Microsoft's discretion, depending on the situation

thundercloud9 say:

After a week and a half of troubleshooting this problem, I finally resorted to an unconventional method in order to resolve the issue.  As it turns out, there are free programs online that you can download in order to install Windows updates.  Some of the programs still use the Windows Update installer, while others use their own stand-alone installer.  In order to get past this issue, I had to download a stand-alone installer program like Autopatcher.  This is available from www.autopatcher.com or www.autopatcher.net.

The installation is pretty much self-explanatory.  When the program is run, it first downloads a rather large file that contains a compilation of updates.  Next, the program will display all the updates available for your computer.  Be forewarned that in order to get the updates to install, you will have to kill the Windows Update service.  Once you do this, Autopatcher will be able to install about 2 to 4 updates at a time before the svchost.exe CPU usage problem rears its ugly head.  Every time this happens, you will have to kill the Windows Update service to get the updates to resume.  This will result in several updates (those that were installing each time you killed the Windows Update service) not being installed.  However, these can be installed on subsequent passes with Autopatcher. 

You will likely need to make several passes with Autopatcher in order to install most of the updates.  Your situation may be different, but I eventually got to the point where Windows Update was actually able to find and download updates itself without the svchost.exe CPU usage issue getting in the way.  If you are persistent and patient, after many computer restarts in order to finish installing updates, hopefully you will be offered the upgrade to Windows 10. 

Let me tell you, once I was offered this upgrade, I took it without hesitation!  The upgrade to Windows 10 was a very smooth and relatively painless process--at least for me.  After using Windows 10 for a couple of weeks, I have become a big fan of the operating system.  In fact, it is likely that I will end up purchasing a new 2-in-1 PC with Windows 10 once computers with the new Intel Skylake processors are released. 

But I have to admit, this entire process has really soured my opinion of Microsoft.  There is something terribly wrong with the operating system (both Windows Vista and Windows 7 in my experience) when you can't even install updates due to a problem with Windows Update itself.  As for why this is occurring, I haven't the slightest idea.

Unfortunately, I have a feeling there are many Windows 7 users that will just resort to buying a new computer because their old computer is being crippled by the very service that is meant to keep it secure.  And to that end, the conspiracy theorist in me wonders if that's not at least an underlying goal of all this.  Intentional sabotage to boost the bottom line...I would certainly hope not.

Hello,

 

Thank you for posting on Microsoft Community. We regret the inconvenience caused and appreciate the time and effort spent in troubleshooting the issue on your own.

 

If you are not able to login into your account in safe mode, it means that the user profile you are trying to login is corrupted. Enable built-in administrator by following the steps below, create a new user profile, transfer all the data from the corrupted user to the newly created user profile and check if the issue is resolved.

 

Step 1: Enable built-in administrator account.

 

  1. Open elevated command prompt by entering Safe mode with Command prompt. To go to safe mode, use the power button on the sign-in screen and press Shift + Restart. This will take you to the recovery boot menu. Click Troubleshoot> Advanced options> Startup settings. You then see a list where you can choose Safe Mode with Command prompt.

  2. Type the following command: net user administrator /active:yes

  3. Login into the built in administrator user profile and create a new user profile with administrator privileges.

  4. Delete the corrupted user account from the built in administrator.

  5. Then lock your screen by pressing Windows key + L, and sign-in at the Administrator account.

  6. net user administrator /active:no

 

Step 2: To create a new user account follow the steps below:

 

  1. Open Elevated command prompt by right clicking on the Start icon and select Command prompt (Admin).

  2. Type net user NewAccount password /add and click Enter.

  3. Type net localgroup Administrators NewAccount /add and click Enter.

  4. Log off from the system and login to the new user account.

 

Refer to the link below to know more on “The user Profile Service failed the logon” (Applies to Windows 10 also):

https://support.microsoft.com/en-us/kb/947215
