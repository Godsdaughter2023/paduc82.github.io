Hi

What is the name of the 'particular executable'? You need to check with the developer of that program to make sure it works on the 64bit version of Windows 7. Most 32bit programs will work on the 64bit version of Windows, but some do not.

If this is a 32bit program, Windows 7 will automatically detect this, place the files in the proper folders, and registry the files. However, if you are performing this manually, you need to understand the differences between the 32bit and 64bit versions.

The folder/file structure is a bit different in the 64bit version. Windows 64bit includes an emulator that allows 32bit programs to run just as if they were installed on a 32bit version of Windows.

In this version you will see a C:\Program Files folder where all 64bit programs are installed. There is also a C:\Program Files (x86) folder where all of the 32bit programs are installed.

There is also a C:\Windows\System32 folder that contains all of the 64bit system files. (This name was retained in the 64bit version because so many 64bit programs were hard coded to look for this folder name) 

You will also see a C:\Windows\SysWOW64 folder which contains all of the 32bit system files that the installed 32bit programs use. (Just for reference, the WOW in that folder name means 'Windows On Windows)

These Active X Controls are usually installed by third party programs that use them. If you downloaded this file from another location, are you sure that it's the correct version number for that program?

The correct location in the 64bit version of Windows 7 to place the comctrl32.ocx file is the C:\Windows\SysWOW64 folder.

This also applies to the command prompt when you run the regsvr32 to register this file. You must use the 32bit version of cmd.exe that is also located in the C:\Windows\SysWOW64 folder. You must also run the cmd.exe elevated (right click cmd.exe and select the Run As Administrator option).

a_user_here say:

Where are you looking at how much memory you have?

Perhaps the Task Manager?

The Task Manager shows you the usable memory, not the memory installed.

If you right click on the Start button and select System, it will show both the installed and usable.

Now with that out of the way, I will go into why your are not going to 100% of the memory as "usable".

And as you suspect it mostly has to do with your graphics card.

The address space of a 32-bit operating system is 4 GB .  So even if you put in more memory it will not see it.

So you would think that if you put 4 GB in a machine you would see 4 GB usable, but that is not the case.

The main reason is because of either the RAM is being "shared" or because the address space is being used for memory mapped I/O.  If you are using a graphics chip built into your system it is going to be getting its memory from the system RAM.  The more dedicated to the video the less for your system.  But even with a dedicated video card you can lose "address space".  To improve transfer speeds from the video card to CPU some video cards use memory mapped I/O.  This basically means that the CPU can access it as if it was regular RAM.  This again takes up some of the address space that would be used for your system RAM.

There are also a few other devices in the system that use memory mapped I/O, but they tend to take a very small amount of memory space so you hardly notice them.

So in summary.  There is most likely nothing wrong with your system, and you might get more usable system memory by reducing how much memory your video card is using.

Going to a 64-bit operating system you may gain more usable memory and you may not depending on your video card and motherboard and such.

More information about it here: http://blogs.msdn.com/b/hiltonl/archive/2007/04/13/the-3gb-not-4gb-ram-problem.aspx

If you want to switch to 64-bit beware of the fact that it has to be a clean install, so you will have to backup your data files, do the clean install, install your needed programs, and then restore your data files.

ronnie vernon say:

Hi Chris

Your idea is a good one, but you are implementing it the wrong way.

By installing a SSD drive that is too small to hold all of the applications, you force yourself to try and install all of the applications on the old, slow platter type, hard drive.

Here is an analogy.

You are preparing to run a marathon. You spend months working out and getting prepared (installing the SSD drive).

When it comes time to run the marathon, you buy a pair of steel running shoes that each weigh 25 pounds (installing the applications on the slow hard drive).

The result is that you finish the race in last place, because you have nullified all of your preparation.


You need to obtain an SSD drive large enough to install the OS and all of the applications.

Use the old hard drive to store all of your personal data and backups that you rarely access.
