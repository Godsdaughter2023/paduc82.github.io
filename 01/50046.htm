Windows 8 on P5N-D: Disk IO retry errors (event ID 153) - Microsoft Community

paul ellsworth say:

Windows 8 on P5N-D: Disk IO retry errors (event ID 153)

I'm posting this because I fixed it and want to make sure it's somewhere online for others to find.

The details are ... I was getting event ID 153 errors (Disk IO failure, retries...).  I never got these using Windows 7.  After upgrading to Windows 8, I began getting them every 15-30 minutes... sometimes 2 at a time, sometimes more than that (10 or so).  Occasionally, Windows 8 would randomly freeze and sometimes crash, as well, which I assume might have been related.

I have had an OCZ Vertex 3 SSD as the install drive for a few months and a P5N-D motherboard (for a long time :) ).

After looking up the compatibility of the nforce chipset (750i I believe) and looking for drivers specifically for Windows 8 ... I found that they did not exist and there were no plans to upgrade the driver.  There are Windows 7 x64 drivers.

This led me eventually to a forum post that said, I forget in what context - not sure if it was Windows 8 specifically - to replace the nVidia driver for the Storage Controllers with the one available from Microsoft.  This worked!  I haven't had the errors since.that time, nor has Windows 8 crashed or frozen.

To do this, open device manager (in the start screen, search for Device Manager in Settings).  Find Storage Controllers and expand that.  There will likely be a Microsoft device and two nVidia SATA controller devices.  On both of the nVidia ones ... right click, Update (or change? I forget the exact text) Driver... choose to manually select the driver, and then choose to select the driver from the list.  The list should come up and there should be an nVidia one and a Microsoft one (PCI IDE Bus or something like that).  Choose the Microsoft one.  Do this for both nVidia controllers and then restart.

Accessing Windows Mail in 2 separate user accounts - Microsoft Community

shonab1 say:

Accessing Windows Mail in 2 separate user accounts

With Windows Mail, if 2 users on the same computer share the same email address can they not both view their email in their own separate user accounts or does Windows only set it up in the one user account?

Also, can Windows Mail be accessed from other computers or does it have to be the one it was set up on?

burrwalnut say:

If your ISP allows mail to be left on their server, it's a simple procedure to receive mail on more than one computer or on more than one user ID on the same computer. Open Windows Mail then:

1. On all users or all computers:
Tools > Accounts > Mail [mail account(s)] > Properties > Advanced Tab > and put a check in ‘Leave a copy of messages on server‘.

2. On one user or one computer:
Tools > Accounts > Mail [mail account(s)] > Properties > Advanced Tab > and put a check in ‘Remove from server after XX days’ (choose as appropriate).

3. Optional on one user or one computer (if you delete mail when closing the program in Tools > Options > Maintenance Tab):
Go to Tools > Accounts > Mail [mail account(s)] > Properties > Advanced Tab and put a check in ‘Remove from server when deleted from Deleted Items‘.

4. Send an email to yourself to check that it's working on all users/computers.

5. Unfortunately, Sent Items cannot be synchronised with a POP account but you can circumvent this by including your email address in the Bcc (blind carbon copy) box when creating important mail that you want stored on the other user account or other computer. Then, as the mail is received, simply drag it from the Inbox to Sent Items.
