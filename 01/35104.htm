Win 10 Pro with IE11 will not play Midi and Mp3 WEB site files

I have WEB sites with midi and mp3 music files that will not open when pages with music files are visited with updated WIN 10 Pro and IE11.  Asf files will play.  All sound devices are updated.  Previous versions of Windows and IE would play midi files.  I replaced midi files with mp3 files without success.  What is the solution?
larry stl say:

As I previously stated, I have the latest Windows Media Player with the required codecs.  And as previously stated, midi and mp3 play in Windows Media Player offline in my computer - just online is the problem.

I just now should have said "was the problem" as I solved it, probably as once posted on here but without the "complicated procedure" which in my post immediately before this I was told the fix was in a post too long ago to recover.

Bottom line is expecting Windows to do what it says is never a good thing.  Here is how I now have midi and mp3 music streaming from my WEB pages:

  1. Opened "Control Panel"
  2. Selected "Default Programs"
  3. Selected "Set your default programs" 
  4. Selected "Windows Media Player"
  5. Selected "Set this program as default" Use the selected program to open all file types and protocols it can open by default"   (NOTE:  this mp3 was not included along with a few other Windows Media Player "Descriptions"
  6. Selected .mp3 "Name", "Change program", and selected "Windows Media Player"

Finally, I made a minor change in HTML code in my WEB pages to include "file type" in respective pages: 

   <object data="Four_Freshmen_-_Graduation_Day.wav.asf" type="audio/video/x-ms-asf" width="80" height="42">
   <param name="autostart" value="true">
   <param name="loop" value="true">
   <embed src="Four_Freshmen_-_Graduation_Day.wav.asf" type="audio/video/x-ms-asf" autostart="true" loop="true" width="80" height="42">
   </embed>
   </object>

<object data="america.mid" type="audio/midi" width="80" height="42">

<param name="autostart" value="true">

<param name="loop" value="true">

<embed src="america.mid" type="audio/midi" autostart="true" loop="true" width="80" height="42">

</embed>

</object>

   <object data="happy.mp3" type="audio/audio/mpeg3" width="80" height="42">
   <param name="autostart" value="true">
   <param name="loop" value="true">
   <embed src="happy.mp3" type="audio/audio/mpeg3" autostart="true" loop="true" width="80" height="42">
   </embed>
   </object>

Maybe this will help others :)
