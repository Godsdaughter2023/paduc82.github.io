It amazes me that people all over the web that are suppose to be techy and all, do not know the basics of the Windows Operating Systems.

Let me explain one simple fact. Windows Operating Systems are backward compatible with devices and software. Such as Programs, Printers/Scanners and about every other device or software program there is.

So I will explain to you how to install your software or devices that you had on any previous Windows OS versions to the newest Windows OS version you now have.

First, I will also explain about your System hardware on your PC's/Laptops. When you purchase a PC/Laptop, it comes with the current Windows OS or you purchase one and install. The Manufacturers create drivers and software for that PC/Laptop for all the hardware in your PC/Laptop for that current Windows Version.

Now when Microsoft comes out with a newer Operating System, there are very few manufacturers that will create new drivers/software for the new operating system for all the hardware inside your PC/Laptop.

So this is what happens, Microsoft develops new generic drivers for all the hardware in your PC/Laptop. If they did not do this, then Windows would not install and would crash or possibly ruin your system. This is how it is able to install and run correctly.

Now if the Manufacturers do develop drivers/software for a newer OS version, then Microsoft may have them on the install media or use Windows Update while installing the OS or after the installation to install the correct drivers. Or they will install their generic drivers and then you can go to the OEM's website and download and install their OEM drivers/software for the current OS.

Now, as for all other devices and software/programs, since Windows XP, you can install devices and software/programs by using the "COMPATIBILITY MODE". Windows is backward compatible. I have programs, printers and devices all the way back from XP that install and run properly on Vista, 7, 8 and now 10. I have did this for 15 years on thousands of PC's/Laptops with about a 98% success rate. This is what Compatibility Mode is for.

So this is how to do it. This is for first time installs, or installs that fail after doing an upgrade to a newer Operating System or after a new service pack or creators updates.

The first thing to do is get your CD's or download the software/drivers for the specific task. Then note which OS that the drivers/software was developed for.

Note: Most all programs and devices are 32 bit installations (x86). But some devices such as printers/scanners will have software and drivers for both 32 bit and 64 bit Operating Systems. So make sure you check first and use or get the correct installer package. If the OEM only offers one package, it will most likely work as it is probably 32 bit and might install on 64 bit OS. Or the CD or installer file has both versions in it. You can install most all software and drivers for 32 bit on a 64 bit OS. But you can not install 64 bit software on a 32 bit OS.

I can explain why you're having trouble with your old programs. It's a little technical, but not hard to understand. The same explanation applies to old hardware.

Software developers and hardware manufacturers decide which versions of Windows they will support. The word 'support' in this context means:

  • the developer/manufacturer will test their product to see if it works properly;
  • if the product doesn't work properly as is, they will modify it to keep it working properly;
  • they will update their documentation to include whatever modifications that were made; and
  • they will train their customer service representatives in the modifications so that the reps, in turn, will be able to help the developer's/manufacturer's customers

All that taken together is what we mean when we say that a software developer or hardware manufacturer 'supports' any given version of Windows. If all that effort sounds expensive, it can be. There's a cost to supporting successive versions of Windows.

Clearly, only the software developer or hardware manufacturer can provide this support. Microsoft can't do it, because:

  1. Microsoft can't possibly be aware of the tens of thousands (probably more) of software applications and hardware devices that are in use all around the world - but let's say, for the sake of argument, that they could.
  2. Microsoft doesn't have detailed knowledge of how these programs and devices work, because Microsoft didn't create them - but let's say, for the sake of argument, that Microsoft was somehow able to lay their hands on this information.
  3. This knowledge belongs to the developer or manufacturer; it's their intellectual property and Microsoft has no legal right to modify it.

You can be sure that Microsoft wishes that every software program and hardware device in the world would work perfectly with Windows 10, because it's in their interest to have it that way. To that end, Microsoft makes available to developers and manufacturers detailed information about upcoming changes to Windows.

Nevertheless, Microsoft can't force a developer or a manufacturer to keep on supporting their products as Windows changes. That's a decision only the developer or manufacturer can make, and they usually make their decision based on - you guessed it - money. Developers and manufacturers aren't going to incur the costs of continued support once a product isn't bringing in enough money (or any money.) That's why older programs and hardware eventually lose support.

Even though Microsoft can't force developers and manufacturers to keep on supporting a product, Microsoft does have a neat trick up its sleeve: compatibility mode.

Even without support, a software program (and a hardware device) might keep on functioning perfectly well for quite a long time - even for years - as long as the software code for the program (or the driver, in the case of hardware) isn't affected by changes to Windows. I've got some very old hardware in my computer that lost support years ago, yet it works very well in Windows 10. (Shhh ... don't tell anybody.)

Knowing this, Microsoft makes it possible to 'persuade' a program (in your case) that it's actually being installed in an older version of Windows. That's necessary because one of the first things an installer does is to check whether it's being installed in a supported version of Windows. That's known as the compatibility check. Installing in compatibility mode makes the installer 'think' it's being installed in an older version of Windows.

Compatibility mode can get your program installed in Windows 10, but that's no guarantee that it will actually work properly, or at all, in Windows 10. If the program's developer isn't supporting the program in Windows 10, it may not work. And the longer a program has been unsupported, the less likely it is to work in a new operating system. It might work perfectly well, or it might work 'sort of' well, or it might not work at all.
