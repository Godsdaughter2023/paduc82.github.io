SUCCESS!!

If was the cluster size. After changing to a 4K cluster, the upgrade happened without error.

I could find no free way of changing the cluster size. Maybe some backup programs would allow restores on a properly formatted drive? I used the (paid) program Acronis Disk Director. Minitools Partition Wizard says it can do it with its paid version as well. A couple of words of warning -- Make sure you have a backup (Disk Director trashed my C: drive on the first time I used it). Ideally a second drive with a copy of your C: drive on it. In fact the whole process is easier if you clone your C: drive onto a second drive and then do the restructuring by having your old C: drive as the the drive that you don't boot from. Second warning from Disk Director -- you need to have enough free space on your C: drive as 4K clusters waste more space than 512 byte clusters.

From this point on, Windows should automatically install the build updates as they come out.

The bad news for you is that with a cluster size of 512 still, you are probably going to have the same error every time that you try to upgrade Windows 10, assuming that Microsoft uses the same process.

Two things to note here:

1. SCDEmu.sys was mentioned as a culprit, this is the PowerISO Virtual Drive driver. I would recommend temporarily removing this program to be sure it's not the issue. Optional visualization programs are: ImgBurn, Magic ISO, etc. DO NOT install Daemon Tools as an alternative.

2. In the dump that mentions NETIO.SYS as the culprit (NETIO.SYS is the Network I/O Subsystem). Usually in cases in which NETIO.sys is the culprit, it's due to a 3rd party firewall or an antivirus firewall tampering with the NETBIOS ports. Interestingly enough, in the same dump it mentions that it failed to load the sumbols for epfwwfp.sys which is the ESET Personal Firewall Driver. If you're still having issues after removing PowerISO, I would recommend temporarily disabling ESET for awhile to see if the issue subsides. If it does not, removal may be necessary (if you do, use this tool - do not use the traditional method of add / remove programs):

http://kb.eset.com/esetkb/index?page=content&id=SOLN2788

The following drivers need to be updated if available:

VMNET.SYS  - Tue Aug 02 16:14:54 2005 (VMware Virtual Network Driver). Update if available here. If an update is not available, remove the software using this driver. 2005 is simply too old for W7 to work with.

vmnetadapter.sys - Tue Aug 02 16:15:02 2005 (VMware virtual network adapter driver). Same as above.

vstor2.sys - Tue May 01 15:27:18 2007 (VMware Virtual Storage Volume Driver). Same as above.
