d.i.v. say:

"Delete confirmation dialogue box" only for "read-only" files in Windows 8 / Windows 8.1 (64-bit, home edition)

Is it possible to activate a "Delete confirmation dialogue box" exclusively for "read-only" files and folders within Windows 8 / Windows 8.1 (64-bit, home edition)?

I dimly recalled that there was such a thing in old versions of Windows. 

With "read-only" files and folders, I am referring to files and folders with the "read-only" attribute set.  This attribute was always respected in the old days of DOS, and is still generally respected by applications running under Windows such as Word, Acrobat, Paint, etc.  (For example, resaving/overwriting a "read-only" file generally is not allowed at all from such applications, and results in a message to that effect, requesting the user to try again, cancel, or use a different name.) 

However, in the Windows File Explorer in Windows 8 / Windows 8.1, there is no prohibition on deleting "read-only" files, and indeed there is not even any warning or confirmation dialogue box.  

Admittedly, there are occasions where it is intended to delete such files, which is why I prefer to activate a deletion confirmation dialogue box (or 'dialog box'), rather than a blanket prohibition. 

Setting the "read-only" attribute and activating a deletion confirmation dialogue box would have the following advantages:

  • Very easy and quick to change through the context menu in File Explorer (or even through the command prompt). 
  • Anyone can change permissions (unless prevented by NTFS permissions). 
  • Ability to delete is retained on the file/folder, but the user is prompted by a warning/reminder to ensure that these 'flagged' items are really intended to be deleted. 
  • There is convenient visibility of the setting(s) when browsing a directory, by selecting to view the "Attibutes" column in File Explorer. 

With only one apparent disadvantage: 

  • Copying of the setting for copied/transferred files and folders is not guaranteed, and inconsistent on different platforms.
