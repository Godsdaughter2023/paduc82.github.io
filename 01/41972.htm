chimel say:

The file associations are not between a file and a program, but between a file type (the file name extension like .jpg) and a program.
Never select the checkbox (or rather, always unselect it) "Always use the selected program to open this kind of file" if you want to open only this particular file with a specific program.
Even if you did select the checkbox, there was no need to restore your computer to reset the association, just "open with" the file again, select the previously associated program, and select the checkbox. Then "open with" the file with the other application unselecting the checkbox if it's just a one time task.

If you edit this file frequently, you can create a shortcut to it, specifying both the program to use and the file to open:

- From the Windows|Programs|Accessories menu, drag and drop the Paint icon to your Desktop
- Right-click "Properties" on the Desktop icon to change the following properties:
  - In the "Shortcut" tab, change the "Target" value to "%windir%\system32\mspaint.exe pic.jpg"
    (assuming your file is called pic.jpg and also located on the Desktop)
  - In the "General" tab, change the icon's name (first field) to something like "Edit pic.jpg with Paint"

thomas ferguson say:

When setup is invoked, part of the early process is a check of the BIOS including its date. If the date is after a certain date, the BIOS is checked against a list of 'bad' BIOSes; if the date is before a certain date, it is checked against a list of known 'good' BIOSes. 'Good' means that it complies with the ACPI standard, 'bad', that it does not.

Yours failed the test. Thus the message to contact the computer manufacturer for an update.

Enter the BIOS setup on system boot (usually done by pressing Delete, F2, F8, or F10 during statrup - check the manual) and see if there is a setting to enable ACPI. If so, do that and complete the boot process. Try installing again.

If no joy, it would be worth a call to Microsoft support. Installation issue calls are free. Do check the computer maker's site for BIOS update availability.

Post back if further help is wanted.

Tom Ferguson

vijay b say: Hi Edward,

1. Did you make any changes to the computer prior to the issue?
2. Did you login to computer into Microsoft account?

If you are unable to locate the Mail app in Windows 8/Windows 8.1, the chances are that you might have accidentally unpinned its icon from the Start screen. If this is the case, you can hover mouse to the bottom right corner of the window while you are on the Start screen, click the Search option, and type Mail in the available field. Once the Mail icon appears in the list, right-click the icon and click ‘Pin to Start’ to get the Mail icon back in the Start screen.

The above discussed process is very straightforward and simple if the Mail app is just unpinned from the Start screen. However the situation might be different if the Mail app is completely uninstalled from the Windows 8/Windows 8.1 computer altogether. If this is the case, you must reinstall the app by going to the Store.

In order to reinstall the Mail app from the Store, the computer must be connected to an active Internet connection.

Here is how you can reinstall the Mail app if it has been uninstalled from the Windows 8/Windows 8.1 computer:
a.  Log on to the Windows 8/Windows 8.1 computer with the administrator account.
b.  On the Start screen, click the Store tile.
c.  On the opened Store interface, in the search bar in the right, type Mail.
d.  From the displayed list of related apps that are available in the Store, click Mail, Calendar and People. On the opened Store interface, in the search bar in the right, type Mail.
e.  From the displayed list of related apps that are available in the Store, click Mail, Calendar and People.
f.  On the Mail, Calendar, and People window, click the Install button from the left, and follow on-screen instructions to reinstall the Mail app on Windows 8/Windows 8.1.
