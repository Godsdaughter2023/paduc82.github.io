Windows Disk Management will NOT shrink any partition past the first immovable file it encounters. If that file is right at the end of the partition then it will show 0 available to shrink.
Try either of these two freeware utilities instead:

GParted

http://gparted.sourceforge.net/

Minitool Partition Wizard

http://www.partitionwizard.com/free-partition-manager.html

Vince

 

As Gordon said, you can only decrease the partition size down to the first unmovable/locked system file.

a. user say:

That depends...Often F2 Opens Setup (BIOS) and F12 opens Boot Menu. You want to open BIOS with F2 first and search for drives. This will vary between older BIOS and newer style setups.

Some makers like Dell place setup in a tree format and you scroll down the menu with the up/down arrows and press Enter to expand a group but legacy BIOS like Award/Phoenix use tabs and you scroll through those with the forward and back arrow keys to change the tabs amn then up/down to move to the section you want, then enter to view Escape or F10...or similar variations.

What you want to find is a listing of the drives and see what show up. If nothing shows up then you may not have a drive connected properly or not correctly set for Master or Slave etc... NEVER TRY TO CHANGE DRIVE JUMPERS OR ADJUST/ATTACH DRIVE AND POWER CABLES WHILE THE COMPUTER IS ON!!! Power down and restart to repeat checking. Once you have all your drive found and reported correctly, save the settings (ESC key or F10 usually) and when you exit BIOS it will seek them like it's supposed to.

Now, if it doesn't and you are actually getting video (Check the monitor cable and power connections, you would be surprised) then use F12 and change the boot order. If you changed the boot order to boot from a DVD or removable drive (like a USB thumb drive or other storage) you should make sure that it is booting from the Windows hard drive FIRST.

All of these factors contribute to a computer failing to start, along with simple POWER SUPPLY FAILURE and accidentally pulling connections or leaving them unattached when working inside the computer.

Sometimes it's a lot less complicated that it seem, especially if you can hear the drives and fans running yet you have no video.
