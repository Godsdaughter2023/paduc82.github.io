mike hall... always say:

The manufacturer recovery partition should be left exactly as it was when the computer was started for the very first time.

 

Your problem arises when using a backup utility.

Backup utilities will never use space on the C drive, and always look for space on the NEXT available drive or partition. In your case, the next available drive was the recovery partition.

 

You should NEVER back up to a partition on the primary boot drive. This drive is worked harder than any other part of your computer, and the chances of failure are higher as a result.

 

Get an external hard drive if you want to backup anything, and ensure that it is disconnected when not required..

 

What to do about the recovery partition..

 

 Depending upon what backup utility you have used, go to the D drive and delete any and all of the backup files that have been made there.

 

Windows Backup uses the file extension BKF, so you would delete all files with the extension BKF.

Solved, Posted - Can't unblock font because Win7 keeps reblocking it!

Problem:  Windows 7 insist on keeping fonts blocked AFTER you've used Control Panel --> Fonts and clicked Property (of font) and clicked Unblock and OK.

The problem in my case was because Win7 insisted that because it thought the font came from another computer, that it should remain blocked. 

Solution: Do NOT use Control Panel's Font area to unblock fonts. 
1) Go to Local Disk (C:)
2) Open Windows Folder
3) Open Fonts Folder
4) Right click on Font and Choose Property
5) Click Unblock And then OK. (I did NOT click Apply and it worked). 
Unblocking blocked fonts this way, should work when Windows 7 insist that reblocking the font. 

(Edit: I changed my question and wrote the solution and a better way of stating the question just in case someone else has the same issue.)
