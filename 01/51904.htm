My suggestion, if you're faced with the same problems, is to create a new user account via elevated CMD (i.e. CMD as Administrator) * See instructions for launching an elevated cmd prompt below, if needed 

To create a new user with elevated CMD prompt:

1. Make sure you're logged on under an admin account.

2. To add a new user account on the local computer:

Example: Adds a new user account with the loginid Bill and with password jd8ufi1k9

 

3. To add new user to the Administrator group:

net localgroup administrators Bill /add

At this point you should be able to access a "working" instance of Windows 10 from which you can transfer files, continue to troubleshoot why your other account isn't loading properly or whatever. To find the files from your original user account, launch File Explorer and navigate to the original user account folder by scrolling down the left-hand navigation window and clicking This PC -> Local Disk -> Users and selecting the original user account. From there you can move files to the new user account, etc. as you wish. Do note that any documents/files/procedures you may have written that explicitly include path names referring to the original account will need to be adjusted.

I'm no Windows expert so there may be a better solution or way to go about it. That said, I was super frustrated with the way Windows 10 was misbehaving and this worked for me. I hope it works for you, too.

* There will be some people who read this and have no idea how to access CMD as Administrator. Here are a couple of ways to do it:

  1. If your right-click works on the Start menu, you should see "Command Prompt (Admin)" as the last option in the top section of the menu that appears
  2. Ctrl + Alt + Del and access Task Manager. Go File -> Run new task. In the "Create new task" dialog that comes up, type cmd and check the box that says "Create this task with administrative privileges" and click Ok
