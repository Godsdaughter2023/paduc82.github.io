The first question is whether you have a 64-bit CPU or a 32-bit one.
If it's 32-bit, you can not install a 64-bit version of Windows at
all.
 
If by "all the other information," you mean data files, yes, it's
possible to do something like what you say. But if you also mean
programs, no you can not do this. To run a program it needs to have
been installed under the copy of Windows you have booted to.
 
And even regarding data, in my view it's simpler to just make an
external backup of your data, then clean install 64-bit Windows and
restore the backup to it.
 
And if your reply might be that you don't have any procedure in place
for making regular external backups of the data important to you, let
me state that you are playing with fire. It's not a question of
*whether* you will lose all your data, it's only a question of *when*
it will happen.

shenan stanley say:
Clean install.  That's it.
 
Back up your stuff (maybe make an image) and then cleanly install the new architecture of OS.  Once installed, update it, install the AV software and install all your other applications then restore your stuff.
 
That's the way you do it.  You are changing architecture - clean install or no change in architecture.
 
If you don't have backups now - that's just unwise (at best.)  Fix that.
