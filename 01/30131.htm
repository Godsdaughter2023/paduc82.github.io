Its an issue with the EA server seeing the same IP address and not allowing a multiple console connection. A way to fix it is to go into the Network settings in My XBOX on both consoles. Have one pull up its automatic connection settings and look at the numbers. Leave the first console's settings alone. On the second console, manually input the DNS Server numbers. For the first DNS Server number copy it exactly. On the secondary DNS Server number, change the last value, I added only 1 to it. There are three or four number sequences in it between 0 and 255 (xxx.xxx.xxx), so if the automatically connected console says 210.96.5.16, manually input the second console to 210.96.5.17. worldexplorer say:

Not all message stored in messagesv12 database table (Skype App for Win 10)

Hello

I'm using the skype app on my smartphone (Android 7.0) and the skype for desktop app on a Windows 10 PC.

Now I also have the Skype App on my Windows 10 PC installed and would like to save the chat history in C:\Users\username\AppData\Local\Packages\Microsoft.SkypeApp_kzf8qxf38zg5c\LocalState. In this folder is a sqlite database file which contains all the message in the messagesv12 table.

My problem now is that some messages are missing. So somehow not all message are synched.

Why is that and how can it be solved? I'm using the newest version of all skype applications.

techfreak_ say:
There is no official way to extract messages/history from the Skype App db file. It is not a main.db file and functions differently. It may not contain all the information you want it to contain. If you want to attempt to use that method to extract messages, you will need to browse through your ~2 years worth of history so that some data can be cached. Whatever you can visually see in the client, in that session, may be present in the db file. You can either try manually scrolling through the history of all your individual contacts or a singular contact in one session. After that session, make a copy of your db file and extract/locate the data you need.  The only official method of copying history in the newer clients, is to select them and copy them to another application/document.

worldexplorer say: Thanks a lot for your answer. Where can I find the main.db?

techfreak_ say:
No current Skype version utilizes a main.db file. The closest you get to that is the s4l*.db file in the Skype for Windows 10 App. The other versions utilize non-SQL .ldb and .log files.  Skype 7.x had a main.db file in your Skype name folder in the %appdata%\Skype folder. I have no idea if/when they will release an official export tool for Skype 8.x history in Skype 8.x.
