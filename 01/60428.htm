watkincm say:

I had this same problem for several months - until today (Yeah!) 

The CPU and motherboard temperatures got so high that the system frequently shut itself down and would not restart unless I removed the battery, let it cool down and then reinserted the battery.

I bought a cooling fan which helped a little but the temperatures were still WAY too high.

Task Manager showed my CPU was running constantly at above 97%.  More than half of that was a set of processes called "Service Host: Local system (Network restricted) (12)".  Within minutes of starting my system, these processes were at 55-60% and just stayed there.

Several recommended fixes failed:

  1. SFC /scannow turned up nothing (Admin command prompt).
  2. Dism /Online /Cleanup-Image /RestoreHealth (Admin command prompt).
  3. Disable SuperFetch (Computer Management - Services).
  4. Removing specific USB devices (most notably audio) to lessen power loads.
  5. ...and more... nothing worked.

In the end, the fix for my machine was very simple.  Turn off Fast Startup.  (Power options / Choose what the power buttons do / uncheck "Turn on Fast Startup")

Apparently, Fast Startup reloads the machine from the hibernation file.  In effect, it reloads a super fast image precisely as it was when last shut down - including those horribly CPU gobbling processes that were the cause of overheating.

My system (a 2009 Lenovo T500) takes almost 4 minutes to boot now but the overheating has stopped, the Service Host processes are at normal levels and the CPUs are running pretty optimally (11% right this moment). 

I am planning to restore Fast Startup because a 20 second boot up is really nice but also plan to monitor my system processes closely in case these problems begin to return.

Hope this helps someone
