[quote user="Gimmyaname"]As for why you cant have both plugged in is because the hdmi port was an after thought on MS part.[/quote]

Not surprisingly, this is wrong. The AV cables block the HDMI port because the console should not have two video outputs active at the same time. The manufacturer can't be blamed, because these devices are sometimes sitting in a warehouse for months after they have already been assembled and imaged. Probably shipped from China to some other location far from where it was originally made.

Please try the steps below and check.

  • While logged-in as Administrator, right-click the drive's root directory and select Properties.
  • Click the Security tab, then the Advanced button.
  • In the Advanced Security Settings dialog for the drive, click the Owner tab, then the Edit button.
  • Click the Administrators group, check Replace owner on subcontainers and objects and click OK. If asked about replacing existing permissions with ones giving you Full Control, click Yes.
  • Back in the Security tab and on the original Properties sheet again, click the Edit button, and in the Security dialog click Remove for all entries for all accounts listed under "Group or user names", except for the well-known ones of System, Administrators, Users and Authenticated Users.

  • Click Advanced again to go back to the Advanced Security Settings dialog box and click the Change Permissions button. Check Replace permission entries on all child objects and click OK and OK.
  • Click OK.

3. Type slmgr.vbs –ipk xxxxx-xxxxx-xxxxx-xxxxx-xxxxx (x should be replaced by your 25 digit product key)

4. Hit enter

5. Type slmgr.vbs –ato (This will change the product key)

6. Hit enter

  1. Detach the Clipboard from the keyboard.
  2. Wet a cotton swab with rubbing alcohol. (Use 95% isopropyl alcohol or higher if possible.) Clean the silver pins on both faces of the three connectors located on the top middle surface of the keyboard hinge. Rub the cotton swab back and forth vigorously to clean the pins. Use a fresh swab surface to clean each of the connector faces.
  3. After you're done, make sure the pins are dry and that there are no cotton fibers on the connector. Reattach the Clipboard to the keyboard.

The toolkit runs a series of tests to diagnose your Surface device. The diagnosis and repair time averages 15 minutes or less, but can take an hour or longer. This will also download the latest updates for your Surface. Click here to learn how to run the toolkit.

  4. Click on Power, then click on the Run the troubleshooter button.

Download and install the firmware from this page. Choose the appropriate version for your device. To find out which version of Windows your device is running, press Windows logo key  + R, type winver, then select OK.

Thanks for your reply Patricia.

I didn't explain it very well in my original email, it's a value for the Open With key - value {09799AFB-AD67-11d1-ABCD-00C04FC30936} at the key HKEY_CLASSES_ROOT\*\shellex\ContextMenuHandlers\Open With. The web reports in many places a common bug across various Windows versions as per here: http://www.howtogeek.com/261076/how-to-fix-the-missing-open-with-option-on-the-windows-10-right-click-context-menu/ but that wasn't my problem.

Anyway, I've just fixed my empty Open with lists, it's all working now. What I did was download Nirsoft OpenWithView from: http://www.nirsoft.net/utils/open_with_view.html, it's a small utility that displays the list of all available applications in the 'Open With' dialog-box of Windows. And as I'm really familiar with what I have installed, I could see an invalid entry to an .exe that doesn't actually exist on my system. (The entry was created erroneously by a 17th Sep update to Tracker PDF-Xchange Editor) (And I've disabled all of the Tracker PDF-Xchange Editor Explorer context menus until they fix the bug, I did that with Nirsoft ShellExView, Nirsoft ShellMenuView and Nirsoft ShellMenuNew). The Open with menu worked fine again for all file types AFTER REBOOTING.

So in conclusion if anybody else has an Open with list that is empty, then it might be a good idea for them to suspect an invalid context menu entry in the registry and to use Nirsoft OpenWithView to try and find it. If it's not obvious try enabling/disabling half the list then the other half, etc., but remember to sign out/restart between changes. Hope this helps somebody. Thanks.
