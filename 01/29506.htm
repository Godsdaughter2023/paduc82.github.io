andre da costa say:

Windows 10 will be free for the first year to all Windows 7, Windows 8.1 and Phone 8.1 users. This means, as long you upgrade within the first 12 months (July 29th 2015 to July 29th 2016), you will not have to pay for Windows 10 at all.

Wow, thats cool, but does that mean, I will have to pay for Windows 10 after the first year?

No. Its a promotion to get as many users to upgrade their existing Windows 7 PCs and Windows 8.1 devices to Windows 10. Once you upgrade within the first year for free, its yours forever at no additional cost. As Microsoft Program Manager for Windows Terry Myerson described it:

This is more than a one-time upgrade: once a Windows device is upgraded to Windows 10, we will continue to keep it current for the supported lifetime of the device –at no additional charge.

Source:

I heard Microsoft representatives mention they are moving to a servicing model for Windows 10, does that mean Windows 10 is going to require a subscription?

No, Windows 10 remains a perpetual license, which means, you can upgrade to Windows 10 and use it forever without it expiring or going into any reduced functional mode.

Are you saying there will be no more versions of Windows after Windows 10?

Microsoft is moving Windows to a servicing model with Windows 10. The days of major revisions or what are called big bang releases are no more. As Microsoft Program Manager for Windows Terry Myerson described it:

This is more than a one-time upgrade: once a Windows device is upgraded to Windows 10, we will continue to keep it current for the supported lifetime of the device – at no additional charge. With Windows 10, the experience will evolve and get even better over time. We’ll deliver new features when they’re ready, not waiting for the next major release. We think of Windows as a Service – in fact, one could reasonably think of Windows in the next couple of years as one of the largest Internet services on the planet.

And just like any Internet service, the idea of asking "What version are you on?" will cease to make sense – which is great news for our Windows developers.

Source: http://blogs.windows.com/bloggingwindows/2015/01/21/the-next-generation-of-windows-windows-10/

Why is Windows 10 free, is there some catch?

No, its not a catch. We are computing in different times. Windows 10 is free so app developers can standardize on one platform and reduce the fragmented market of different versions and editions of Windows that have been an issue in the past. When everybody is on the same platform, it means better quality apps, a healthier ecosystem and more advances in hardware and software innovation.

Initializing page file on ramdisk periodically (40% of the time +/-) fails at Windows boot

Greetings --

For the past year or so I've set my Windows 7 32-bit, 8GB RAM system to initialize the page file automatically on a ramdisk (in order to reduce physical disk reads/writes and to speed up page file access).

Here's how I've set it up: I've installed Dataram Ramdisk and configured it to to use memory above 4 GB (which, as everyone here knows, 32-bit Windows can't otherwise use) as the ramdisk.  In setting it up I manually formatted the 4 GB unallocated ramdisk space as an NTFS volume and gave it the letter R: (what else!?). Then I had the software save an image file of the formatted ramdisk to my hard disk F:. Further configuration of the software has it initialize the ramdisk at boot and immediately copy the image file to the ramdisk, thereby creating a 4 GB NTFS formatted volume R:. (As an aside, I also use the ramdisk for the browser cache and the print spooler). The theory here is flawless. But for it to work in actual practice, the ramdisk has to be initialized and the image file copied to the ramdisk (making it a formatted drive R:) before Windows initializes the page file.

This works about 60% of the time, but for the other 40% Windows gives an error message to the effect that Windows has created the page file in a temporary location (it's always the C: drive, where the page file typically is located by default.). The only reasonable explanation I can think why it fails is that during roughly 40% of my computer's boots Window wants to initialize the page file before the ramdisk is ready to receive it. What I don't understand is why that happens.

This gets me to my main question: Is it possible to control the Windows boot sequence in such a way that the ramdisk is initialized and the image file copied to it on a consistent basis before Windows wants to initialize the page file? Operationally I guess this comes down to whether it's possible, during each boot sequence, to delay page file initialization and/or to advance ramdisk initialization earlier in the sequence. If so, how does one do that? If not, can anyone explain why the the order of the respective initializations usually works, but fails a fair amount as well?

Any insights on this matter would be appreciated.

RAS43

Create a small 20MB paging file on the boot volume (the partition where the Windows files are installed) that way the session manager will be able to start the Windows session and create the additional paging files without complaining.  You can have more than one paging on more than one disk, Windows will favor the paging file on the least busy disk.
