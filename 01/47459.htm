How to re-enable RDP confirmation on disconnect - Microsoft Community

brian d. hart say:

How to re-enable RDP confirmation on disconnect

I downloaded some Windows update last week that changed my RDP disconnect confirmation so that it gave me the option to "not ask this question next time". I mindlessly checked that box, as I generally do with all "do not ask again" popups, but of course now, I find myself disconnecting (instead of logging off) my RDP sessions accidentally--but without the failsafe message, and then I cannot remember which server or station I just left myself logged onto remotely (I manage several dozen of these).

I assume there is a registry entry to re-enable the disconnect confirmation. Can someone provide that reg location?

brian d. hart say:

I found it, surprisingly easily, once I really thought about the fact that this would be a per-user setting and looked in the HKCU hive in the registry instead of HKLM:

 

[HKEY_CURRENT_USER\Software\Microsoft\Terminal Server Client]
"ShowShutdownDialog"=dword:00000001

 

The value was set to zero when I checked the "Don't display this message again" box during RDP disconnect. I just changed it back to 1 to re-enable the dialog.
