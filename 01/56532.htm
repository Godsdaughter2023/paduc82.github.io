melananighthawk say:

"Mousekeys" is an Ease of Access setting to use the numberpad of any keyboard to control the cursor.

There *is* no mouse to give you a make/model for. There is only any standard keyboard, and built-in Accessibility settings in Windows.

http://windows.microsoft.com/en-au/windows7/use-mouse-keys-to-move-the-mouse-pointer

https://en.wikipedia.org/wiki/Mouse_keys

For some reason, MS Windows sets what should be the middle mouse button setting to "click both left+right". I don't understand why it should be set to a useless setting => as far as I know, there are no situations where clicking both left and right mouse button keys at the same time is actually useful.

I'd like to be able to use the middle-click instead of both-click, as both-click has no meaning/use.

melananighthawk say:

To put this in perspective:

  • There is a 31 year old standard for what the disability/accessibility option for Mousekeys (controlling the mouse cursor with only a keyboard) should provide.
  • Microsoft provides 3 modes. Two of the three modes Microsoft provide meet the standard. The third mode Microsoft provide (where a "click" sends both left and right click at the same time, instead of sending the "modifier button"/middle mouse button the standard sets out) is meaningless in modern Windows.
  • This means that of the 3 modes Microsoft provide, one is completely broken - it represents nothing usable or useful in the operating system itself.
  • Thus, my conclusion is that Microsoft are NOT meeting the minimum standards for accessibility in their current implementation of Mousekeys.

I can understand that third mode being "both-click" coming from a history where mice didn't have a third button originally - and the operating system had an option to emulate the middle mouse button by clicking both the left and right buttons at the same time.

However, it appears that Microsoft have removed the option to do middle button emulation - without updating the accessibility setting that depended on it being there. This breaks/makes meaningless the third mode of the Mousekeys accessibility feature!

This isn't a fault with a stand-alone mouse. There is no mouse hardware involved. These are basic accessibility/disability features built into Windows itself that are partly unusable!

Seems to be an issue with corruption of the page file which could be caused by a third party driver:

BugCheck 1A, {3f, b46, 8504451b, b26aa79b}

Probably caused by : Unknown_Image ( PAGE_HASH_ERRORS_INPAGE )

Followup:     MachineOwner
---------

 *** Memory manager detected 1 instance(s) of corrupted pagefilepage(s) while performing in-page operations.

ADDITIONAL_DEBUG_TEXT:  Memory Manager detected corruption of a pagefile page while performing an in-page operation.
The data read from storage does not match the original data written.
This indicates the data was corrupted by the storage stack, or device hardware.

Looking at the driver stack of some of the dumps shows involvement of the fltmgr.sys and fileinfo.sys so I might suspect the FAMv4.sys which appears to be related to "File Access Manager" software so I would recommend to uninstall that software.

I would also suggest to uninstall any NTI software that is installed as the UBHelper.SYS (NTI diver) is an older driver that may not be compatible with Windows 10.

Also. if the problem persists uninstall, at least temporarily, Frontier Security and use Windows built-in antivirus (Windows Defender) and firewall as third party security filter drivers can be problematic especially in a relatively new operating system.
