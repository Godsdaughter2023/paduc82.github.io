Hi Spiros, there is a known bug in the 1803 version of Windows 10. Microsoft are aware of this issue and hopefully are working on a patch . . .

For the moment, please try this workaround to set your preferred input language:

Right click your Start Button, choose Powershell (admin)

Run this command to get a list of installed languages on your PC

Get-WinUserLanguageList

Note down the LanguageTag of your preferred language

If say, your preferred language is US English, run this command and hit Enter:

$1 = New-WinUserLanguageList en-US

Then run this command to set the order of installed input languages

Set-WinUserLanguageList $1

Close Powershell and restart your PC

tryx3 say:

Expected action:

Would this scheduled task run under admin account? even through I login using user account and defragment is running at the background under admin account.  Would it be corrected interpretation? -

- Yes, this is correct - you are logged in as a standard user but defrag is making use of the Admin account.

Action Tab

Would it be correct to select following file?

No - I do not know why you keep referring to C:\defrag.bat when just entering this in Task scheduler

          Program/script     %windir%\system32\defrag.exe

          Add arguments    C:

does the job.  In addition, Windows treats batch files named after Windows utilities very strangely so you should never use the same name as a Windows utility.  In this case, I think Windows will run defrag.exe instead of your batch file so any arguments written in the batch file will be ignored.  Just set up Task scheduler as I have done and forget about using a batch file.

You can tell the task is running by looking at Task manager, Services, defragsvc.  If the task is running then this service will also be running [and the service will keep saying it is running for 5-10 minutes after the task completes as well].

You can check the results by opening Disk defragmenter [C:\Windows\System32\dfrgui.exe] and it will show you the last time any defrag activity took place as well as the last reported state of fragmentation [i.e. the state of fragmentation when the task last ended]. 

[Note that opening Disk defragmenter also sets the Task manager, Services, defragsvc to say it is Running so this can be confusing - the service is not the thing taking any action and just opening Disk defragmenter does not initiate any defrag action].
