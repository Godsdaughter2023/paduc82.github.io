what drivers does windows 10 update automatically - Microsoft Community

catbill say:

what drivers does windows 10 update automatically

How can we find out which drivers Windows automatically updates?

I just happened to check the drivers for an audio device and found that they needed to be updated.  I was under the assumption that Windows was automatically checking for updates.  If it is, how do we know which drivers it is checking for and which drivers we need to check for manually?

Thank you.


a. user say:

That's a great question. In the months leading up to the release of a new operating system, Microsoft asks hardware manufacturers to create device drivers it can include with Windows. The drivers that hardware manufacturers supply typically provide a basic level of functionality. If you want newer versions of drivers, you need to get them from the hardware manufacturers directly. Windows doesn't automatically check for updated drivers.

It does happen from time to time that a hardware manufacturer will provide updated drivers to Microsoft to be added to Windows and provided to users through Windows Update. This is unusual, however, and typically means that the original driver has a particular problem and the hardware manufacturer wants the fix to receive the widest possible audience.

An aside: People like to think they always need to have the newest drivers, probably because that's what they hear from Level 1 tech support that doesn't know any better. The fact is, you only need to update a driver if the newer version addresses a specific problem that your computer is experiencing.

Every driver update has Release Notes or some other form of change log that describes exactly what issues the update addresses. If none of them apply to you, you're not going to benefit by updating. I know that sounds contra-intuitive. Just because something is newer doesn't mean it's better.

---

 

Manual workaround for anyone hitting this is:

1) Remove the back of the laptop, locate the SSD drive (a rectangular board against the edge covered by a diagonal black bracket with 2 screws), and remove it carefully

 

2) Turn on and you'll get the Samsung prompt which includes pressing F4 for Recovery mode

 

3) Do so, and immediately push back in the SSD drive - be quick and be careful

 

4) Choose to restore and wait for it to boot into the restored Windows (8.0)

 

5) Immediately run the "SW Update" software and install the BIOS/Firmware update - this, btw, is the cause of the problem: a buggy Samsung BIOS, not that they've admitted it (yet) and reboot when prompted.

 

6) Turn it off and put the laptop back together

 

You can now probably repeat the 8.1 upgrade download and install process - I wanted a clean install so:

 

7) Obtain your ISO of 8.1 and associated key

 

8) Use Rufus to write a GPIT/FAT32 USB stick (google "Rufus USB" and choose the "GPIT for UEFI" option)

 

9) Use Windows 8.0 to boot into the UEFI (firmware) and ensure USB is set before the HDD/SDD 

 

10) Insert the USB stick and boot

 

11) I then chose to format the C: drive (do not touch the other partitions)

 

12) And finally 8.1 installed cleanly

 

13) Then on first login download the "SW Update" software off Samsung's website, install and run it

 

14) Ensure you choose manually all the drivers, ignoring the other rubbish

 

And you'll now have a 100% clean and working 8.1....
