I am thinking that by the term "downloading" you are actually referring to "installing" software. Downloading is just the act of transferring or copying files from one location to another, generally from a remote server. Installing is the act of placing software on a system to run routines and involves altering or adding system files, registry entries, and other changes.

Whether software installs per user or to all users is part of how the installation routine is written. Some will offer one or the other, but many do not. ¡firedog say:

That's just what Storage folders are for. If you move a message from an IMAP account folder to a user-created folder under Storage folders, it will be removed from the server at the next sync (or probably immediately, depending on the server). The message will after that only be available on the computer where it's stored. You can reverse the process to move a message back to the server.

Note that you can only move messages from an account to Storage folders. You can't move folders.

When performing larger-scale file operations in Windows Live Mail, it's a good idea to (a) work offline meanwhile, to avoid interference from scheduled sync operations, and (b) take it gently. A lot of things happen for each message that is moved - the move is recorded in a log file, the message file itself will be scanned by your AV and it will be re-indexed by Windows to update the search index and so on. When I move messages from one folder to another, I copy them in batches of not more than 100 messages at a time. This creates a new file for each message, leaving the old one intact. When I'm sure that all the messages have made it to their new folder, I Shift-Delete the old ones.
