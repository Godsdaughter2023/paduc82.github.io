wunders say:

"You cannot log on after you remove the computer from the domain"
  < http://support.microsoft.com/kb/317049 >

Unfortunately the above article usually only explains the problem and does not help much.  You need to log in and perform a system restore to recover from this but since you've broken the trust with the domain, you must use a local login.  The best candidate is the built-in administrator account.  From the login screen, hit Alt-Ctrl-Del twice and you should be able to type in "administrator" as the user and leave the password blank (assumes that nobody has assigned a password to this account in the past).  If this works, either perform a System Restore to a point in time before you removed it from the domain or just create new local users if you don't want it on the domain.

Failing this, there is a way to manually replace registry files with backups but it's messy.

HTH,
  JW

wunders say:

In a situation like this, and assuming you can't get your machine re-connected to the domain, your only option is to boot from a bootable CD/DVD and restore the registry manually from a point in time before you removed the computer from the domain.  Hopefully the cached logins will be restored and you will be able to log in.  My preference is to boot from a Live Linux CD such as Knoppix or Ubuntu .  If you choose Ubuntu, take care not to install it but select the "Try me" option when booting.  You can create other Windows-based boot disks such as Bart PE or  Ultimate Boot CD for Windows (UBCD4Win).  UBCD4Win has a utility that will do all the steps automatically:

     http://www.technibble.com/how-to-fix-windows-registry-hive/ >    (courtesy Daavee)

Warning: the Windows-based boot disks require a Windows XP install CD as a part of the build process (Microsoft Copyrights)
Otherwise, once you build the bootable CD, boot from the CD/DVD and follow the instructions in the following KB article:

"How to recover from a corrupted registry that prevents Windows XP from starting"
  < http://support.microsoft.com/kb/307545 >

Since you booted from a bootable CD, you can ignore the warnings about OEM systems and proceed beginning at Part 2, Step 6.  Move, copy, and rename the files as directed using commands native to the OS CD that you booted from (usually Right-Click -> Copy, or Right-Click Paste, or Right-Click -> Rename).  Pick a Restore Point (RP) directory dated before your mishap.  You won't have to reboot between Part 2 and Part 3, but at the end of Part 3, you must shutdown, remove the CD, and reboot your machine.  If you successfully log in at this point, don't forget to do Part 4 (System Restore).

HTH & Good Luck,
  JW

PS: After booting from the CD, you might take the opportunity to back up some of your files to a USB drive.
