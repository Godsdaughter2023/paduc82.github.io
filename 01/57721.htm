a. user say:

It is too bad that we know next to nothing about your system, how it got into this situation, what might have happened since the last time the system worked properly or what "used a windowsxp recovery console CD" means (used it for what).

I have some general ideas that might help with SESSION type failure messages and since you can get into the XP Recovery Console this may help:

Usually, a SESSION type message (of which there are several) comes after some antivirus/antimalware tool or some overzealous cleaning has deleted critical XP system files.

There are at least five possibilities (that I know of) for these missing files that result in the SESSION3 or SESSION5 type message and there is no harm in just replacing all 5 files while you are there...  
one or some may be missing, so you will just overwrite with the backup copies that should already be on your system, assuming XP is installed on your C drive:

C:\Windows\System32\smss.exe  <-------- most likely, so replace this one first
C:\Windows\System32\ntdll.dll
C:\Windows\System32\ntoskrnl.exe
C:\Windows\System32\winlogon.exe
C:\Windows\System32\drivers\ftdisk.sys

After booting into the XP Recovery Console, you should be in the C:\WINDOWS folder.  If you are not in the C:\WINDOWS folder you have some other problem that you need to fix first.

We know that XP keeps backup copies of critical system files in this folder:

c:\windows\system32\dllcache

Depending on how you installed Service Pack 3, there may also be backup copies in this folder:

c:\windows\ServicePackFiles\i386

You need to copy the first 4 files one at a time from one of those two folders into this folder where they belong:

c:\windows\system32

When copying files, you may be replacing existing files with the copies, so respond in the affirmative if asked to overwrite existing files.  If the copy succeeds,
you should see a message that the file was copied successfully.

In the Recovery Console, from the C:\WINDOWS prompt, enter the following commands:

copy c:\windows\system32\dllcache\smss.exe    c:\windows\system32

Reply in the affirmative to any "are you sure" messages so 1 file(s) is copied

If the file is not found and if you installed SP3 from a download enter this command:

copy c:\windows\ServicePackFiles\i386\smss.exe    c:\windows\system32

Just repeat this process for the next 4 files even if they are not missing.

The ftdisk.sys file needs to be copied into this folder:

c:\windows\system32\drivers

So you would enter:

copy c:\windows\system32\dllcache\ftdisk.sys   c:\windows\system32\drivers

Remove the CD and enter 'exit' in the Recovery Console and your system will reboot.
