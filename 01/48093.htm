¡firedog say:

Look for Mail.MSMessageStore file name and delete it (right click, delete).

That file is the index for the message database, recording which messages are in which folder and the status of each one. Deleting it will either cause the program to take the backup copy of the index into use or to create a complete new one if the backup copy is not available. It's difficult to say which happened in your case.

Replacing the index file will not affect the underlying messages, so they're most likely still there somewhere. Press Ctrl-Y to see the new folder tree, then look for your missing messages. You may find them buried several levels deep in a folder called Recovered items under Storage folders. If this folder exists, it won't be empty unless you emptied it.

All of my remarks about keyboard shortcuts, the folder pane and commands on the ribbon were referring to the Windows Live Mail program interface. You have been looking at the Windows file system using Explorer.

The store folder (the one called Windows Live Mail) is hidden by default, because it's not a good idea to mess about with its files and folders. Why are you trying to optimize Windows Recovery Environment tools?

 

This partition allocate space for the Windows RE tools image (Winre.wim, at least 250 MB), plus enough free space so that the partition can be captured by backup utilities.

 

For further information, refer to the section “Windows RE Tools partition” from the article:

Configure UEFI/GPT-Based Hard Drive Partitions

http://technet.microsoft.com/en-us/library/hh824839.aspx

 

Since it is the Windows Recovery Partition, it is better not to optimize or defrag the drive. When your computer will not boot and need to reinstall or repair windows, you need to access the partition using this tools from Windows Recovery partition.
