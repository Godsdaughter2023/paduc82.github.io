Let’s reinstall the USB Controllers.

Use only the Mouse in the procedure below:

First, please disconnect all USB devices and then perform the following steps:

1. Click "Start", type “Device Management” (without quotation marks) in the “Search” bar and press "Enter". Click “Continue” if necessary.
2. In Device Manager, double click to expand "Universal Serial Bus controllers", right click on the Host Controller, click "Uninstall" and click "OK".
3. Repeat the step 2 to uninstall all items under "Universal Serial Bus controllers".

Then, restart your computer and Windows 7 will reinstall all USB controllers automatically.

Use only the Keyboard in the procedure below:

First, please disconnect all USB devices. Then, go through the following steps:

1. Press "Ctrl + Esc" to open the “Start” menu.
2. Type “devmgmt.msc” in the Search bar and hit “Enter". Click “Continue” if necessary.
3. In Device Manger, press Tab and use "Up" and "Down" to move the highlight to "Universal Serial Bus Controllers".
4. Press "Right" and expand it.
5. Press "Down" to move to the first item and press "Delete" to remove it.
6. Repeat the step 5 and uninstall all items under "Universal Serial Bus Controllers".

Then, restart your computer and Windows 7 will reinstall all controllers automatically.

If it does not work, I suggest you also perform the following steps.

Method 2:

Delete the USB "Upper Filter" & "Lower Filter" Entry

Before we go any further, please first backup the Registry.

Important This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs.

For more information about how to back up and restore the registry in Windows:

http://support.microsoft.com/kb/322756

1. Click “Start”, type “REGEDIT” (without quotation marks) in the “Search” bar and press Enter.

Note: If UAC (User Account Control) pops up, please accept it.

2. Right click “Computer” (the root node) in the left pane, click “Export” under the “File” menu, choose “All” under “Export range”, and select “Desktop” in the “Save” in box and type backup in “File Name”. Click “Save”.

Note: The backup file is on the Desktop and named backup.reg. We can simply restore the registry by double-clicking the backup.reg file.

3. Click “Start”, type “REGEDIT” (without quotation marks) in the “Search” bar and press Enter again.
4. Locate the “UpperFilters” value under the following key in the registry:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{36FC9E60-C465-11CF-8056-444553540000}

5. On the “Edit” menu, click “Delete”, and then click “OK”.
6. Locate the “LowerFilters” value under the same key in the registry:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Class\{36FC9E60-C465-11CF-8056-444553540000}

7. On the “Edit” menu, click “Delete”, and then click “OK”.
8. Quit Registry Editor and restart the computer.

Hope this helps!

questions_ say:

A fresh install of Widows can become corrupt as the computer environments are different.

Software drivers can become corrupt as the computer environments are different.

When there are crashes the antivirus software drivers and the graphics drivers can appear in the crash even though they did not cause the crash.

When troubleshooting the drivers are uninstalled or replaced to create a stable computer environment.

With a stable computer environment the software is reinstalled to access the computer environment.

If there are no conflicts (AV software and other software can conflict with one another) then the re-installation of the software does not cause BSOD.  If there are no BSOD then the software can be used.  If there are BSOD then alternative software is used.

So during the troubleshooting period the removal of the Sophos software permits a better evaluation of the Intel GPU drivers.  If the computer is unstable with BSOD then there could be another misbehaving driver that was not yet identified.  If the computer is stable then the Intel GPU drive is ok and the Sophos software can be reinstalled.

Drivers can be stress tested with Windows driver verifier. 

This tool is frequently used when problematic drivers are not found.

Once problematic drivers are found they are fixed and the computer is then observed for problems.

On my computers I use it for preventative maintenance.

There is a pro and con to using this tool.

Data would need to be protected from any unexpected problems.

A NFO copy of msinfo32 is easier to use.

After you complete some of the steps in the prior post you can rerun the administrative command prompt which should collect in this format.
