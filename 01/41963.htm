Hi Neil. I'm Greg, an installation specialist, 9 year Windows MVP and Independent Advisor, here to help you.

I'm not sure why the drive map was left off the Disk mgmt screenshot so we cannot see the relationship of the drives to each other.

For example, I'm assuming Elements is your External but unless it's a bootable drive like a flash stick, it should never be marked Active since that is how a bootable partition is pointed to in an MBR partition scheme, and your Windows is installed in the more modern UEFI or GPT format. So that's wrong and possibly problematic, needs to have the drive set to Inactive if it's not itself intentionally bootable: http://partitionwizard.com/video-help/setactive...

The 10gb Recovery partition would be the factory Recovery partition due to it's size. All of the others are either OEM or Windows repair tools partitions which in previous Windows editions were on C, but now can be recreated fresh with each version Update. So there's no way to know which is the current bootable Repair partition, unless you want to delete them all and rely on bootable Recovery or Installation media to do Repairs if Windows ever won't start.

The 128mb Microsoft Reserved partition is reserved for future system use such as Bitlocker. It's not worth messing with.

What you have building up there with all the old Recovery partitions is a Disk Junkyard which is the reason most enthusiasts won't even run Factory installs but prefer to wipe the drive and Clean Install to get a much more lean install with fewer issues. What we have found is that in Windows 10 everything depends on the quality of the underlying install, the same reason enthusiasts have never run Factory or Upgrade installs because they know there would be endless issues.

For example there have been zero return problems by those who follow the gold standard Clean Install here: http://answers.microsoft.com/en-us/windows/wiki.... It is better and faster than any previous version of Windows, and a better install than any amount of money could buy. So when you get time I'd study over that to see if it's something I can help you do.

There is also an automated Fresh Start that reinstalls Windows while shedding factory bloatware that can cause issues and affect performance, saves your files, but doesn't clear the drive to get it cleanest: https://www.howtogeek.com/265054/how-to-easily-...

Otherwise I'd leave it as it is until you can do the Clean Install.

azeez n say: Hi deano,

Follow these steps and check if the issue persists.

Step 1:

These error messages occur if you aren’t the administrator of the computer or if you haven’t logged on as an administrator. Log on to the administrator account and check for the issue.

If this computer is on a network or on a Domain then the administrator might have imposed restrictions. Contact the network administrator to fix this problem.

It could also be an infection on your computer. Try running an online scan and check if that helps.

Safety Scan for Windows Vista and Windows 7

Malicious Software Removal Tool | Protect Your Computer



Step 2:

you may also need to change the HTM/HTML association in the registry.

Registry disclaimer

   a. Start, click Run, type "Regedit" in the Open box, and then click OK.
   b. Browse to HKEY_CURRENT_USER\Software\Classes\.html
   c. Right click the value for the .html key and select Modify...
   d. Change the value from "InternetExplorerHTML" to "htmlfile"

Repeat these steps for htm and .shtml keys if they exist. You may also want to check the xhtml and xhtm keys.
