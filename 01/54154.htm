You need to run chkdsk on whatever volume (C, D) where XP is installed - that is where the trouble is.

I had another thought - after you run chkdsk /r, reboot to F8 Advanced Boot Options and choose Safe Mode and see if things look better.

The thing you see sometimes with Norton (and this 000218 affliction), is your file system gets corrupted when Norton loads at startup. 

You run chkdsk/r and then that will allow you to boot one more time - long enough to uninstall Norton using their special uninstaller tools (but you will not boot again until you run chkdsk /r again).   This Norton thing started showing up some weeks ago on systems with old versions installed that got some automatic update.

When you boot in Safe Mode, Norton should not load, so that may get you going long enough to make some progress.

I know you may have uninstalled Norton, and that may not be the problem here, but I think that is what I would do.  You will have to run chkdsk /r many times more probably until you figure it out.
