7.       If the Startup status lists Stopped, click Start and then click OK.

lemp say:

There seem to be some idiosyncracies when attempting to start a PowerPoint file in presentation mode using Task Scheduler.

First, using a *.ppsx appears to open in edit mode when started from a scheduled task rather than in presentation mode (which works when starting from the Run box or Command Prompt).

As far as quotation marks are concerned, you need these if the file name -- including the full pathname if used -- has any spaces.  You can, of course, use the quotation marks even if you don't really need to do so.

Suppose you have a file named MyTest.pptx and it's saved in your Documents folder.  Also suppose your user account name is NERD.  The full path to that file (unless you've changed the normal defaults) is C:\Users\Nerd\Documents\MyTest.pptx.


Now suppose you have PowerPoint 2010.  The full pathname to the executable file is C:\Program Files (x86)\Microsoft Office\Office14\POWERPNT.EXE

For PowerPoint 2013, the file would be in Office15 and for PP 2016, in Office16.

Note that the path to Powerpnt.exe has spaces, so you'll need to use quotes.

  • Open Task Scheduler and click "Create Basic Task" in the right (Actions) pane.
  • Give the task a name and optionally add a short description; click Next
  • Select the trigger; click Next
  • Select the start time; click Next
  • Select "Start a program" as the Action; click Next
  • Type the following in the program/script box and click Next (note quotes because of the spaces in the name)
    "C:\Program Files (x86)\Microsoft Office\Office14\POWERPNT.EXE"
  • Type the following in the "Add arguments" box and click next (quotes here are optional, because there are no spaces in the name)
    /s "C:\Users\Nerd\Documents\MyTest.pptx"
  • Click Finish
  • Back at the main Task Scheduler window, select "Task Scheduler Library" in the left pane and scroll down to the name of the task you just created.  Right-click on it and select Run

The slideshow should open in presentation mode BUT ... it will be behind the open Task Scheduler window and the PowerPoint icon in the Taskbar will flash until you click it to bring the PowerPoint window to the front.  This is undesirable.

Do the following:

Open Notepad (click Start, type Notepad in the Search box, press Enter) and type the following:

@echo off
start "" "C:\Program Files (x86)\Microsoft Office\Office14\POWERPNT.EXE" /s "C:\Users\Nerd\Documents\MyTest.pptx"
exit

Click File > Save as

Select "Save as type" to be "All files (*)"

Save the file with a .bat extension (e.g., runppsx.bat)

Notes: 

  1. You need the two double quotes after the word start
  2. Change Office14 as appropriate for your version of Office
  3. Use your own user account name and file name
  4. Remember where you save the batch file (e.g., Documents\runppsx.bat)

Now create a scheduled task using the batch file:

  • Open Task Scheduler and click "Create Basic Task" in the right (Actions) pane.
  • Give the task a name and optionally add a short description; click Next
  • Select the trigger; click Next
  • Select the start time; click Next
  • Select "Start a program" as the Action; click Next
  • Type the following in the program/script box and click Next (optional quotes if no spaces in the name)
    "C:\Users\Nerd\Documents\runppsx.bat"
  • Click Finish
  • Back at the main Task Scheduler window, select "Task Scheduler Library" in the left pane and scroll down to the name of the task you just created.  Right-click on it and select Run

how can I fix "exception processing message C0000013 - Microsoft Community

yorickirwin say:

how can I fix "exception processing message C0000013

Running XP service pack 3 but frequently get following popping up "Exception Processing Message C0000013 Parameters 75b6bf7c 4 75b6bf7c 75b6bfc" Any possible solutions pls?

a. user say:

If you are seeing message like this:

No Disk, Exception Processing Message c0000013 Parameters 75b6bf7c 475b6bf7c 75b6bf7c

Until you can determine what the "real" problem is with your hardware, you can disable the pop up message by making a registry adjustment.

Before making registry changes, backup your registry with this popular free and easy to use tool:

http://www.snapfiles.com/get/erunt.html

Click Start, Run and in the box enter:

regedit

Click OK to open the registry editor and navigate to the following key:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Windows

On the right hand pane, check the value of the ErrorMode parameter.  

If the ErrorMode parameter is set to 0, double click ErrorMode and change the value to 2

(2 = disabled).

Click OK to save the changes, then click File, Exit to close the registry editor.

Reboot your system and check things now.

Setting the ErrorMode parameter to 2 stops the pop up message but will still log any hard errors to the Event Viewer log.

Read more about it here:

http://support.microsoft.com/kb/124873
