Some hotkeys on the keyboard don't work after OOBE. - Microsoft Community

danlin1 say:

Some hotkeys on the keyboard don't work after OOBE.

I found this issue just after OOBE, some hotkeys on the keyboard don't work, e.g. Airplane mode on/off hotkey. Half an hour later, I checked again, the hotkey could work. So is this an expected behavior from Microsoft's point of view? Why?

duoduobear say:

Yes. This is an expected result. In some devices, hotkeys need an UWP App support. UWP App would be installed during OOBE and some time after OOBE. If UWP App cannot complete installation, hotkeys cannot work. The UWP APP installation time depends on device configuration. Sometimes, it needs several minutes or more than half an hour.

Symptom:

Hotkey cannot work just after OOBE (out of box experience) for a new device or create a new account then first time log in with new account.

Workaround:

1. Wait a moment until UWP App would be installed completely.

Or

2. Manually launch the hotkey App at once. Windows will do UWP App installation at high priority if end user launch this App. So this App will be launched soon and hotkeys could work fine.  

So this is neither a Windows bug nor a device issue. Just wait a moment, this will be OK.

If you do not have a Microsoft Vista DVD, make a Bootable Repair Disk to do a Startup Repair:

 

 Download the ISO file at the supplied link, and make a Bootable Startup Repair Disk from it.

 

Go into your Bios/Setup, or Boot Menu, at startup, and change the Boot Order to make the DVD/CD drive 1st in the Boot Order, then reboot with the disk in the Drive.

 

At startup/power on you should see at the bottom of the screen either F2 or DELETE, to go into Bios/Setup, or F12 for the Boot Menu.

 

When you have changed that, insert that Bootable Disk you have made in the Drive, and reboot.
