I'm experiencing the same issue after an upgrade.  On a hunch I opened the "C:\Windows\ImmersiveControlPanel\Application.manifest" file in a text editor and compared the file with a system where it works to the system where it does not. What I found is that on the system where it was not working, the file was empty. 

I would try copying the file from the machine that is working but the folder and files are locked with special Windows permissions that it doesn't appear can be overridden.  The contents of the file should be: 

<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<assembly
    xmlns="urn:schemas-microsoft-com:asm.v1"
    xmlns:asmv3="urn:schemas-microsoft-com:asm.v3"
    manifestVersion="1.0">
  <dependency>
    <dependentAssembly>
      <assemblyIdentity
        type="win32"
        name="Microsoft.Windows.Common-Controls"
        version="6.0.0.0"
        processorArchitecture="*"
        publicKeyToken="6595b64144ccf1df"
      />
    </dependentAssembly>
  </dependency>
</assembly>

If this is not what you're seeing in the file when you open it in a text editor, then you have your culprit.   As to how to fix it, I don't know see below:

Microsoft has provided a Official Solution (http://answers.microsoft.com/en-us/windows/forum/windows_10-other_settings/settings-doesnt-launch-or-launches-the-store/ec439819-7ee4-4b4d-abdd-35d82e04c55f), which you should attempt first.   If that does not work for you, as it did not work for me; then you will need to replace the bad file with a good one. 

Because this is a core system file you should not attempt to fix this manually unless you know what you are doing.  To fix the file the following steps need to be take:

1. Configure your Windows 10 system to show the boot loader when loading.

2. Drop to a safe mode command prompt

3. Take ownership of the file

4. Change the ACLs to give yourself Modify access

5. Replace the files contents with those shown above

6. Change the ACLs back to remove Modify access

7. Reboot

Why would the TV connect but not the external monitor?  

HDMI support in W10 FCU is flakey.   Sometimes it works, sometimes it doesn't.   But there are workarounds to try when it doesn't:  Win+Ctrl+Shift+B  (more than once may be necessary with psychic pauses between retries).  You can also get a connection by using the Project dialog (Win-P) to change the display configuration.   Note that if you want to try using Second Screen Only  (Win-P, End, Enter) you must be prepared to respond to a Keep this? prompt whether you can see it or not.  Press Tab, Enter for that.  Best to have Narrator running then though (Win+Ctrl+Enter) so you can still be aware of what you may not be seeing yet.

Another thing that I think may be affected is HDMI audio.  For example, when I have that set as my default audio device while the HDMI video is not working I won't be able to hear Narrator (but can get visual indications that it is working with either Extend or Duplicate).   VGA is more reliable than HDMI video even when I leave the associated HDMI audio port (incongruously) active.  (I actually use a HDMI-VGA converter with the standard Surface HDMI adapter so I suppose I should not be critical about the Endpoint audio still appearing then.)

A new consideration that I have become aware of is HDCP (content protection).   Who knows maybe your TV makes that less of an issue that the external monitor would have no possibility of dealing with.  Maybe this is why VGA works better than HDMI?  Etc.
