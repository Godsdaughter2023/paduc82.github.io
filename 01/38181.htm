carolina calling say:

It is very likely that there is a copy kernel32.dll file in %SystemRoot%\system32\dllcache.

(%SystemRoot% is usually set to C:\WINDOWS.)

One may restore this file by booting into the Microsoft Windows XP™ Recovery Console (MWRC)

and copying the file from the DLLcache.

.

Boot to the Windows XP installation media and, at the first prompt, type: R

After much clacking, etc. MWRC will present a numbered list of Windows installations and a prompt.

e.g.:

 

1:  C:\WINDOWS

...

Which Windows installation would you like to log into

(To cancel, press ENTER)?

 

At this point select the appropriate Windows installation (usually 1 (C:\WINDOWS)).

There will be other gooble-dee-gook if yours is an OEM system.  MWRC will then

prompt for the Administrator's account password.  Type the password (or simply

ENTER if there isn't one).

 

At the prompt, type: DIR system32\dllcache\kernel32.dll

to see if it's there.  If so, type: COPY system32\dllcache\kernel32.dll system32

Type EXIT to reboot.

 

If not, ...

 

On both my Windows XP x86 (Intel) Home and Professional editions, this file shows up as

version 5.1.2600.5781, is dated 21 March 2009 and was install by the update associated

with KB959426 (MS09-15).  Please note that the version number will vary depending on

your system architechure, etc.  See KB959426.  Also, visit the TechNet article ->here<-.

Download the x86 (Intel) update ->here<- or the x64 (AMD) update ->here<-.  Once the

update is downloaded, run it with the -x switch (via CMD) and save the files in a directory

of your choosing (it must exist).

 

The 5.1.2600.5781 version is in the SF3QFE (Quick Fix Engineering) branch of the x86 download.

 

Copy the appropriate file to a jump drive, boot MWRC with the jump drive plugged in,

use the MAP command to find the jump drive's letter and copy kernel32.dll into  \WINDOWS\system32.

 

Good luck!
