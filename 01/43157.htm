To ensure minidumps are enabled:
* Go to Start, in the Search Box type: sysdm.cpl, press Enter.
* Under the Advanced tab, click on the Startup and Recovery Settings... button.
* Ensure that Automatically restart is unchecked.
* Under the Write Debugging Information header select Small memory dump (256 kB) in the dropdown box (the 256kb varies).
* Ensure that the Small Dump Directory is listed as %systemroot%\Minidump.
* OK your way out.
* Reboot if changes have been made.

a. user say:

Since you are seeing a REGISTRY_ERROR and if the system was working okay prior to this incident, what do you think happened to the system?

It might help to tell us some things about your system such as:

What is your system make and model?

What is your XP Version and Service Pack?

Describe your current antivirus and anti malware situation:  Microsoft Security Essentials (MSE), McAfee, Symantec, Norton, Spybot, AVG, Avira!,  avast!. Panda, Trend Micro, CA, Defender, ZoneAlarm, PC Tools, Comodo, etc.

If your system will not boot, you will not be able to retrieve any crash dump files (unless you boot on something else) and I don't think there will be any crash dump file with that error, but I am not sure about that.

The important thing is to describe what happened prior to this incident.

You tried to reinstall XP?  How did you try to reinstall XP? 

Does that mean you have a genuine bootable XP installation CD that is the same Service Pack as your installed Service Pack?

If you think you may have corrupted your registry somehow by tinkering with your system or downloaded and installed something prior to this incident, you might be able to choose to boot in Safe Mode and perform a System Restore to a date prior to the incident.  Note that you cannot undo a System Restore performed in Safe Mode.

If your reinstall attempt has not totally hosed up your system, you also might be able to restore just the registry by booting your system on a Hiren's Boot CD and using the Registry Restore Wizard.

From the Hiren's HBCD Menu, choose Programs, Registry, Registry Restore Wizard and it will show you all your current System Restore Points (each RP has a copy of the registry for that RP). and go back to a day when you think your system was working okay. 

Since the listed RPs are sorted by NUMBER and not by date, be careful to look at the dates of the RPs offered and not necessarily the number of the RP - so don't accidentally chose RP9 when you really need to choose RP90 - so be sure to pay attention to the dates.

Unless you are 100% sure what you are doing, never choose "Original Installation Settings".

Note that the Registry Restore Wizard is not equal to an XP System Restore.  Only the XP registry is restored.

You may start with the most recent Restore Point prior to the incident and if that doesn't work, go back to the next most recent one, etc. and keep your fingers crossed.

Here's how to get started:

Make yourself a Hiren's Boot CD which you can download from here:

http://www.hirensbootcd.org/download/

The ZIP file is large, so the download will probably take a little while to complete bit it is worth it.  Then unzip the download to extract the Hirens.BootCD.ISO file that will be used to create your new bootable CD.

Creating a bootable CD from a .ISO file is not the same as just copying the .ISO file to a blank CD.  You have to use software that understands how to burn a .ISO file to a CD to create a bootable CD.

In the Hiren's ZIP file are the BurnToCD.cmd file that you can double click to launch it, but I have never used it and prefer to use ImgBurn.

If you need a free and easy CD burning software package, here is a popular free program:

http://www.imgburn.com/

When installing ImgBurn, DO NOT install anything from UniBlue or the Ask toolbar (never install the ASK toolbar for anything).

Here are some instructions for ImgBurn:

http://forum.imgburn.com/index.php?showtopic=61

It would be a good idea to test your new bootable CD on a computer that is working.

You may need to adjust the computer BIOS settings to use the CD ROM drive as the first boot device instead of the hard disk. 

These adjustments are  made before Windows tries to load.  If you miss it, you will have to reboot the system again.

When booting on the Hiren's CD you will see a menu of options.  Choose the Mini XP option.  Then it will appear that Windows is being loaded and you will be presented with a desktop that has the look and feel of the Windows Explorer interface you are already used to using.

Remember:  You did not boot on your hard disk - you booted into the Hiren's desktop.
