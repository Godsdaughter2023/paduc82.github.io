UPDATE (Solved It) :

After nearly a month of frustration I solved this issue.

Open Windows PowerShell as Administrator

  • Type :Power Shell" in the app search bar next to Start
  • Right Click on Windows PowerShell and choose "Run as Administrator"

Paste the following command : 

Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

Hit enter and wait for a while till it finishes it.

Your problem should be solved after it is done running the command.

Note : make sure you have pasted/typed the command properly and its in a single line and there is nothing extra before you hit enter.

linney say:

PXE messages always makes me check the boot order in the Bios to make sure that it is correct.  For Windows to load it needs to access the Hard Drive, on your machine, this is not happening.  The hard Drive must be set ahead of any PXE (Network Boot).  The Invalid Boot Disk message is because you have a Floppy inserted that is not a bootable Floppy, so remove the Floppy from the drive.  Concentrate on the Boot order in the Bios.

To enter the Bios (see your computer screen at power on, your manual, or manufacturer's web site) you probably have to press something like the DEL Key, or one of the F Keys at boot up time, but the actual Key varies from model to model.  Many computers refer to this as "Entering Setup", Setup is just another term for the Bios.

Setting your Bios to any safe default settings may also fix this for you.

hairyfool say:

It is difficult to be sure but sounds like both routers are working with the same LAN side subnet. This would mean that the routing function on Router 2 would get totally confused because its WAN and LAN are in the same range.

If it is that tangled I would suggest that you strip everything out, set up Router 1 to connect to the ISP and plug a computer straight into it to make sure everything works and confirm its address range.

Then set up Router 2 but in the 1st instance do not connect its WAN port to the LAN of Router 1. Connect a Computer to its LAN, if it is the same one you will need to reboot or do ipconfig /release and ipconfig/renew to make it get a new address from Router 2 and check its range. If they are the same then you will need to log in to the Router 2 configuration page and change its IP Address and DHCP range to a different subnet. Only when you know that the networks are different should you plug Router 2 WAN to Router 1 LAN.

I have had similar problems when I made the mistake of connecting two LANs on different subnets together, it produced some very odd results until I realised. Trying to avoid sounding pompous but a tidy cabling setup if far easier to diagnose that a rats nest of cables. The last under desk network I sorted out ended up with one spare modem lead, two unused power supplies and 3 spare LAN leads. Also an ADSL splitter that was plugged in to an ADSL splitter with the router in the second one. Still not quite sure how it ever worked.

hairyfool say:

Just for clarification can you confirm the configuration.

Router 1, WAN port connected to the internet receiving its WAN IP settings from the ISP (IP Address, DG, DNS). It is running a DHCP server issuing IP Address range 192.168.X.Z with itself as DNS and DG (does it have WiFi?).

Router 2, WAN port (DSL) connected to a LAN port of Router 1 so receives its WAN IP settings from Router 1. It is also a DHCP server issuing IP Address range 192.168.Y.Z (X must be different to Y) again with itself as DNS and DG (WiFi?)

There should be no connection between the LAN side of both routers and the WiFi SSIDs must be different.

If by daisy chaining you mean you have connected the LAN sides together to give you more LAN ports then you need to disable the router WAN port (if possible) and DHCP on the second router, you cannot have 2 DHCP servers on the same network. DNS relay on the second router only works through the WAN interface.
