How to really take control of automatic updates in Windows 10 - Microsoft Community

srfreeman say:

How to really take control of automatic updates in Windows 10

It seems that there are a number of users of this forum that are having troubles with the automatic updates in Windows 10. Apparently the number of users this is important to is small enough that it has not interested any commercial developer and to date, it does not seem there are any of the users that take it seriously enough to do anything about it.

Humorously, many posts I have read, blame Microsoft for somehow leaving users out in the cold. Yet Microsoft has provided the tools for any user to take complete control of automatic updates.

Let us start a list of the tools Microsoft has made available...

1. The Windows Update Agent (WUA): This is arguably the most powerful tool in that it can access all facets of automatic updates in Windows 10 and it is scriptable or programmable, if you will. It is available to administrative users of all desktop and server editions of Windows 10. For guidance on this tool and how to use it, refer to: https://msdn.microsoft.com/en-us/library/windows/desktop/aa387099(v=vs.85).aspx

2. Windows Server Update Services (WSUS): This tool enables information technology administrators to deploy the latest Microsoft product updates to computers that are running the Windows operating system. By using WSUS, administrators can fully manage the distribution of updates that are released through Microsoft Update to computers in their network. For guidance on this tool refer to: https://technet.microsoft.com/en-us/windowsserver/bb332157.aspx

3. Windows Update for Business (WUFB): This collection of tools enables information technology administrators to keep the Windows 10 devices in their organization always up to date with the latest security defenses and Windows features by directly connecting these systems to Windows Update service. You can use Group Policy or MDM solutions such as Intune to configure the Windows Update for Business settings. Using Group Policy or MDM solutions such as Intune, you can control how and when Windows 10 devices are updated. In addition, by using Intune, organizations can manage devices that are not joined to a domain at all or are joined to Microsoft Azure Active Directory (Azure AD) alongside your on-premises domain-joined machines. For guidance on this collection of tools refer to: https://technet.microsoft.com/en-us/itpro/windows/manage/waas-manage-updates-wufb

4. Windows Task Scheduler (WTS): This tool enables you to automatically perform routine tasks on a chosen computer. The Task Scheduler does this by monitoring whatever criteria you choose to initiate the tasks (referred to as triggers) and then executing the tasks when the criteria is met. It is important to the discussion of automatic updates because in Windows 10 its library contains the tasks used by Microsoft to automate updates. This tool has been around for quite some time, with very little change in Windows 10. To make use of this concerning automatic updates, the user should become very well versed in the entire tool (there is a great deal of documentation available) and concepts concerning its use - this tool also controls very critical parts of your Windows installation. For guidance on this tool, start with referring to: https://technet.microsoft.com/en-us/library/cc721871(v=ws.11).aspx and https://msdn.microsoft.com/en-us/library/windows/desktop/aa383614(v=vs.85).aspx 

For what is new in Windows 10 for this tool, refer to: https://msdn.microsoft.com/en-us/library/windows/desktop/aa384138%28v=vs.85%29.aspx

These four tools should be enough to start the discussion. Let us know if you have others.
