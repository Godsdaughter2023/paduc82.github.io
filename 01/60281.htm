I understand that you are unable to boot to desktop due to error “secure boot violation”. Let me help you with the issue.

I would like to let you know that this issue may occur when Secure Boot has been enabled in your computer's BIOS. Secure Boot protects the integrity of the operating system and prevents unauthorized firmware, operating systems or UEFI drivers from interfering with the boot process.
 
Further, you may also want to know that Secure Boot must first be disabled before installing new hardware. For more information on disabling Secure Boot in your computer's BIOS, reference the manual that came with your motherboard or contact your motherboard manufacturer. If you are in an enterprise environment, contact your system administrator.
 

Note: Modifying BIOS/ complementary metal oxide semiconductor (CMOS) settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of BIOS/CMOS settings can be solved. Modifications of the settings are at your own risk.

Same problem here...but also note, you will also get the "BOOTMGR is missing" message from rebooting/trouble shooting.

Yes it's part of the same problem,  "Secure Boot" in the BIOS as explained before.

I also have a custom built ASUS motherboard with a BIOS that has a Secure Boot option.  (note; interesting that I also have the same exact build on two PC's, but only one is having this problem...Windows 7, Windows 10 is fine.  Something to think about)

To de-activate Secure Boot;  You must delete the KP key settings ONLY under the Secure Boot Tab. (save setup to a flash drive first, by inserting a flash drive before boot up.  Save key settings for a later date if needed.)

Then Make sure your Main Boot-Up Drive with the Boot Manager file is selected as the 1st drive in the BIOS and Save settings (F10).  Reboot and all should be well.
