a. user say:

Earlier versions of Windows had a built-in method of transferring files from an old to a new computer, but such a resource is no longer needed with the technology we have today.

Since your files are all backed up on a portable hard drive, restoring them to Windows 10 is a simple matter:

1- If you used a program to backup the files to the portable hard drive, use the same program to restore them to the new computer.

2- If you simply copied your files to the portable hard drive, copy them to the new computer.

One problem that you may encounter - and it has nothing to do with Windows 10 - is that you may have to take ownership of the folders. Right now, the folders belong to your old user account on your old computer. You new user account has to become the new owner of the folders.

If you run into that problem, just do a web search on: take ownership of folders in Windows. If you come across instructions for Windows 7 or 8, they'll work fine in Windows 10.

hairyfool say:

There is no such thing as IP Address Hijacking particularly if you have DHCP enabled, anybody who joins the network gets an IP Address automatically. If this is what you have been told I would be extremely sceptical about anything else you have been informed of. There is little point in turning off DHCP as once connected finding the address range is relatively simple. IP Addresses do not come into play until the computer is actually connected to the network, knowing what IP Address to use does not get you onto a Wi-Fi network

The security solution is the Wi-Fi access key. WPA-2 is a 128 bit encryption algorithm which requires time and specialist tools to break so make sure this is the system being used. Change the key at regular intervals, you could also change the SSID as this would stop automatic connection. Again don't be tempted to hide the SSID, somebody who can break WPA will have no trouble whatsoever finding the SSID and just makes your legitimate connection more unreliable. Until the Wi-Fi access is authenticated a computer is not on the network. On some systems you can enable MAC Address filtering which limits connection to those systems that you have provided the MAC Address of although again its use I limited against determined hackers.

If you don't know where DMP files are located, here's how to get to them:

 1.    Navigate to the %systemroot%\Minidump folder.

2. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Performance > Settings > Advanced > Ensure there's a check-mark for 'Automatically manage paging file size for all drives'.

3. Windows key + Pause key. This should bring up System. Click Advanced System Settings on the left > Advanced > Startup and Recovery > Settings > System Failure > ensure there is a check mark next to 'Write an event to the system log'.

Ensure Small Memory Dump is selected and ensure the path is %systemroot%\Minidump.

4. Double check that the WERS is ENABLED:

Start > Search > type services.msc > Under the name tab, find Windows Error Reporting Service > If the status of the service is not Started then right click it and select Start. Right click on battery icon in system tray > Power options > Click Change Plan Settings > Click Restore default settings for this plan

davem121 say:
Hi Andrew, it is not easy to remove a file association in Windows 10,

Open Windows File Explorer

On the view Ribbon (tab) check the box marked 'File Name Extensions'

Close File Explorer

Right click your Desktop and choose New - Text File

Change the file extension on that new file from .txt to .exe

Confirm the change

Right click a .jet file, choose open with - Choose another App

Check the box marked 'Always use this app to open .jet files' - IMPORTANT!

Scroll to the bottom and click 'More Apps'

Again scroll to the bottom and choose 'Find another app on this PC'

Navigate to the dummy file exe on your Desktop and click OK

The .jet file will not open

Delete the dummy exe file on your Desktop and restart your PC

The file association will then be broken!
