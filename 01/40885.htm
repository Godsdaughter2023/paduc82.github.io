a. user say:

Hi Barebuns,

I have simulated your issue on our end. 

First, to explain the Date Created issue on Windows Explorer, it shows the creation date of the file itself, not the video on the file. So every time you copy it to a new location, it creates the new file and that is why it shows the current date.

Second, if the metadata (the properties) of the video doesn't have the original creation date of the video, Photo Gallery uses the Date Modified data of the photo as the Date Taken and saves it on the Photo Gallery program. If this is the case, the Date Taken will not be reflected on the file/Internet Explorer unless it is explicitly changed on the Date Taken information on Photo Gallery.

If in case the video has a Date Taken metadata (you can confirm it if there is an entry in Date Taken on Windows Explorer), then it will be the one that Photo Gallery will use as the Date Taken information.

Please take note that this also applies to all files including pictures that can be opened on Photo Gallery.

Please let us know if you have more questions.
