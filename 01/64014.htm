steveb_cwb say:

Ok, I've fixed mine so here's what I did. No guarantees of course. (Edit: If you look at later posts, this doesn't seemed to have worked for a lot of users.)

How I diagnosed the issue.

  1. Right clicked on the Start button and selected Event Viewer.
  2. Opened Windows Logs > Application.
  3. There I noticed a lot of ESENT errors along the lines of Unable to read the header of logfile C:\Users\...\AppData\Local\TileDataLayer\Database\EDB00021.log
  4. Searched on Reddit found this: https://www.reddit.com/r/Windows10/comments/36rupz/errors_since_updating_to_10122/
  5. So that's what I tried.

What I did.

  1. Logged out and logged back in as another user.
  2. Typed PowerShell into the search.
  3. Right clicked on Windows PowerShell Desktop App and selected Run as administrator.
  4. When the shell appeared typed cd \Users\MyUserName\Local\AppData (replace MyUserName with your account name. If you don't know its exact format you can type the first letter and hit tab to got through the options.)
  5. Type dir and you should see TileDatalayer.
  6. Now type del TileDataLayer (Edit: you would probably be better typing ren TileDataLayer TileDataLayer_BAD so that you can put your original data back if this all goes pear shaped.)
  7. Log out and then log back in under your normal name. It is important to log back in with your fingers crossed or it might not work. Also I was wearing red shorts and a black T-shirt at the time. Don't know if that was important.

Good luck.
