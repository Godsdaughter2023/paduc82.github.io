Cortana is your digital agent and helps you get things done on your Windows 10 device.

Restart your device after installing the update.

If the issue persists, let's change the ownership of your C: Drive.

  1. Open This PC and locate the drive where Windows 10 is installed. By default, it should be C.
  2. Right-click the drive and choose Properties.
  3. Go to Security tab and click the Advanced button.
  4. Find Owner section and click Change.
  5. Select User or Group window will appear. Click the Advanced button.
  6. Click Find Now button. List of users and user groups will appear. Select the Administrators group and click OK.
  7. In Select User or Group window click OK.
  8. In Advanced Security Settings window owner should be changed to Administrators and Administrators group should be added to Permission entries list. Check Replace owner on sub containers and objects.
  9. Click Apply and OK. Wait for the process to complete.

In addition, you can add Full Control permissions to the drive for ALL APPLICATION PACKAGES group.

  1. Open your disk partition Properties and go to Security tab.
  2. Click the Edit button.
  3. Permissions for Local Disk window will open. Click Add button.
  4. Select User or Group window will appear. Click the Advanced button, and then click Find Now.
  5. Locate ALL APPLICATION PACKAGES on the list, select it, and click OK.
  6. Click OK again.
  7. ALL APPLICATION PACKAGES will now be added to the list. Click it and select Full Control. Click Apply and OK to save the changes.
  8. If the issue still persists, open disk partition properties, go to Security tab and add Full control to the Users group.

steve cochran say:

Bruce asked me to jump in here, since I'm more familiar with Winmail and the database and I wrote WMUtil.  The reason I wrote it is because there is no easy way to manually compact the database, and WMUtil just wraps the MS utility to make it easier for the users.

Here are some suggestions:

If you have the WinMail gadget running, get rid of it.

For MSE, go to the settings and disable scanning of the eml file type.  That will prevent the eml files from being scanned and deleted, which results in a database mismatch that corrupts the database (WMUtil fixes that).  If you are incautious enough to open an infecting attachment, MSE will still act in the background to block it.

Your issue stems from one of two possible ones, from what I can discern.  The database in Winmail could have been corrupted, so you can run WMUtil (and let it finish.  If you are worried, then make a backup of things, if you haven't already (see www.oehelp.com/backup.aspx#wm)).  Then see if the problem mail accounts work. 

If you still get the error, then the other issue could apply, which is that there may be a problem message on the server that Winmail can't handle and that is generating the error.  For that situation you would have to identify the problem message and get rid of it.  Since you already have 2 copies of your 20,000 messages you have in the POP3 inbox, I would suggest that you create a new mail folder on the server and move the messages from the server's inbox to it.   Then add another copy of the POP3 account in Winmail and see if it will download the newer messages.  That might rid you of the issue then.

See if any of that helps.
