techie007l say:

    In Windows 8x, all programs run as a limited user even on a administrative user account due to UAC, even if UAC is set to "disabled".  This could be causing the problem that you're experiencing with Virtual Audio Cables.

    First of all, try starting Virtual Audio Cables by right-clicking its desktop icon and clicking Run as Administrator, and see if it works as expected now.  If it has no desktop icon, you may need to locate the main executable for Virtual Audio Cables in Program Files or Program Files (x86) as there is no Start Menu in Windows 8x—unless you've installed a Start Menu replacement like Classic Shell, in which case you should be able to right-click the Start Menu program entry for Virtual Audio Cables and click Run as Administrator.

    If that works, a more permanent solution would be to right-click the Virtual Audio Cables icon (as described above), click Properties, go to the Compatibility tab, and tick Run this program as an Administrator, which should make it always elevate without you needing to choose Run as Administrator each time.

    If the above doesn't work for you, try disabling UAC by following the steps below.  Please note that "disabling" UAC via the Control Panel does not actually disable it, but only suppresses the elevation dialog.  Programs still run as a regular user until they request elevation, something that not all programs know to do.  Please note that UAC provides an important security layer to your computer and should not be disabled unless nessecary.  Disabling UAC in Windows 8x will also disable all Metro apps.  You can undo the change any time by changing the EnableLUA value back to 1.

  1. Start Registry Editor.
  2. Navigate to HKEY_LOCAL_MACHINE -> SOFTWARE -> Microsoft -> Windows -> CurrentVersion -> Policies -> System
  3. In the right-hand pane (which should have over a dozen values), look for a value named EnableLUA.  If you find it, skip to step 6.
  4. Right-click in an empty area in the right-hand pane, and click New -> DWORD (32-bit Value).
  5. Type EnableLUA as the name for the new value.
  6. Make sure its value is 0.  If not, double-click it and change it to 0.
  7. Close Registry Editor and reboot your computer.  Now all programs launched on an administrative account will run with administrative privileges.

sujatamoktan say:

Hi Mukund,

Thank you for posting your query in Microsoft Community.

The kernel power event ID 41 error occurs when the computer is shut down, or it restarts unexpectedly. When a computer that is running Windows starts, a check is performed to determine whether the computer was shut down cleanly. If the computer was not shut down cleanly, a Kernel Power Event 41 message is generated. An event 41 is used to report that something unexpected happened that prevented Windows from shutting down correctly. There may be insufficient information to explicitly define what happened. To determine what may have happened and to identify a potential resolution, it is important to know what the computer was doing at the time just before the event occurred.

I would suggest you to set the computer in a clean boot state and check if it helps. Setting the system in a clean boot state will help to identify if any third party applications/drivers or startup items are causing the issue.

Refer the article and check if it helps.

https://support.microsoft.com/en-us/kb/929135

Note: After clean boot troubleshooting steps, follow the section "How to reset the computer to start as usual after troubleshooting with clean boot" in the provided link to return your computer to Normal startup mode.
