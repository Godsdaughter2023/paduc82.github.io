a_user_here say:

There is nothing wrong with your memory.

The reason you can't get to 4GB is because the memory addresses are being reserved for your hardware.

Some hardware is what they call "memory mapped".  As in it looks the the CPU just like a memory location.

The biggest device that is going to do this, memory address wise, is your display hardware.

If you have display hardware built into your CPU or motherboard, one way to get some of that space back is to reduce how much memory the display hardware uses in the BIOS settings.

There is other memory address reserved for other hardware, but they are not that big.

Mostly I try to limit the display memory as much as possible because I don't play any graphics intense games, and so the minimum memory is always more than enough.

I have an AMD APU (CPU + GPU) on my machine.  The Gigabyte BIOS lets me get it down to 256MB so I have 3.7 GB of usable memory.  I have an ASUS motherboard that lets me get it down to 32MB.  It varies with what the motherboard supports.

a_user_here say:

People have been feeding you malarkey.

AMD had a 64-bit processor before Intel had one.

In fact the very first Windows to support 64-bit was for AMD only.

Your processor is a 64-bit processor.

If you select: Start -> Settings -> System -> About, it should say: x64-based processor

The main difficulty with changing from 32-bit to 64-bit is that you can't do it as an upgrade.

You need to backup your personal files, do the clean install removing everything, and then after Windows 10 64-bit is installed, restore your data files.

Also I should mention that with 4 GB you will not see much of an increase in memory, maybe none.  The memory that overlaps the video memory can't be relocated.  Also because a 64-bit address programs will use a bit more memory.  It isn't double because not every instruction has an address in it, but you can figure that it will be about 1.25 to 1.5 more in memory.  So unless you are going to go to at least 6GB, or better yet 8GB it probably isn't worth doing.  Also if you have some very old programs, like created before Windows XP, they might be 16-bit programs and will not run on a 64-bit operating system.  A 32-bit operating system can run 16 and 32 bit programs.  A 64 bit operating system can run 32 and 64 bit programs.  Also all your drivers will now have to be 64-bit for a 64 bit operating system.

The licensing for 32-bit/64-bit is the same.

Here is the details on the procedure.

http://answers.microsoft.com/en-us/windows/wiki/windows_10-win_upgrade/how-to-switch-from-32-bit-versions-of-windows-to/40d90c4e-8845-4bcb-bb2f-aeab312caf74

a_user_here say:

Also the main reason to go to 64-bit is to get access to more memory.

As in you have a program like a picture/video editing program that uses a lot of memory.

Or you want to run a lot of programs at the same time.

Or maybe you want to run virtual machines.

What 64-bit won't do is speed a program up, unless it was starved for memory.  And that is not normal.

Normally programs use a fraction of the memory they are allowed to use.

stteffann say: I am doing this for just one program that needs 4gb of ram to work, that is all... I know its not very useful to switch just for 0,75gb...

a_user_here say:

I see, well besides switching to 64-bit you will have to put in more memory.  The operating system itself is going to need as least about 1 GB itself, maybe more.  And you will probably want to be able to run a few more programs at the same time, like email or your web browser.  So with 4GB total, it would never have enough memory to give your program 4GB.  What would happen is that Windows would have to page out parts of the program data to disk, and then as you switch to doing other things read it back in.  This would make your machine really, really slow.  With one program needing 4GB I probably wouldn't go with less than 8GB total in the system.

Please also note that on an old system how much memory you can put into it, might actually be the limiting factor instead of if the processor is 64-bit or not.

a_user_here say:

Actually the more I think of it, it depends on how the stated that the program needed 4GB.  They could have either said the program needs 4GB, or that the system has to have 4GB in it.  If they are saying that it should be a total of 4GB minimum in the system then in reality being a bit short on that memory is probably not the problem at all.  Most likely the real problem is that they wrote the program as a 64-bit program, and as such you need a 64-bit operating system to run that program.  A 32-bit operating system can run 16 and 32 bit programs.  A 64-bit operating system can run 32 and 64 bit programs.

But frankly if they state that 4GB is the minimum I would go for at least a little bit more than that.

richardeiler say:

Hi,

Well, there are pretty much three options for connecting to the internet.

1. A wired (via ethernet cable) directly or indirectly (via network) to a providers modem.

2. Wi-Fi, either an onboard (internal) Wi-Fi card or a USB (external) Wi-Fi dongle can be used on either a protected account (with password) or an open Wi-Fi connection (provided that the device is within range, signal strong enough).

3. Via Smart Phone.

3a: If the phone is equipped to be (provide) a Hot Spot and the connecting device is equipped to recieve Wi-Fi.

3b: If the phone is equipped to be "tethered" for providing internet access (via USB cable) and the phone plan includes (allows) this option this also works well. Of course, speeds will/can ultimately vary depending on many factors involved, with each method/provider.

Let us know how it goes and good luck going forward.
