It's actually surprisingly easy!

You make an image backup of the HDD and then restore the backup to the SSD. You end up with everything exactly the same as it was, just on a SSD instead of a HDD. It's like moving a dollar from your left pants pocket to your right - same dollar, different pocket. And there are no worries about licensing or activation. I did this myself, and I'm no kind of computer genius.

Most retail SSDs come with the image backup software you'll need as well as detailed instructions on how to use it. If yours did not you can probably find the same on the SSD manufacturer's website. The manufacturer's technical support will help you make and restore the image.

Making and restoring an image is the one sure way to swap hard drives and get it right the first time. Not only that, making an image and restoring the image  - once you learn how - is over and done with very quickly.

If your SSD did not come with software and instructions, and the manufacturer doesn't supply them, I really think it's worthwhile to learn how to use a third-party image backup application. One good example of this is the free version of the popular Macrium Reflect. I won't lie: It will take some time to learn how to use this software. But when you see how easily and quickly you can switch from a HDD to SSD, and do it right the first time, I believe you'll agree that it was worth the effort.

waynraynwl say:

I was finally able to solve this problem permanently.

The answer is not Group Policy Editor, but Registry Editor.

Based on a comment from Deepak Handa found here:

http://answers.microsoft.com/en-us/ie/forum/ie11-iewindows8_1/lan-connection-settings-keep-changing-back-to/76a0f5d2-167f-41fa-bf40-1461b8c01642?page=2

Which is:

You can also manually set it through registry editing if your an advanced user
Windows Registry Editor Version 5.00
[HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Internet Settings]
"MigrateProxy"=dword:00000001
"ProxyEnable"=dword:00000001
"ProxyHttp1.1"=dword:00000000
"ProxyServer"="http://server:port"

1 for enable and 0 for disable.

... I went into the Regedit Internet Settings and found that the "ProxyEnable" DWORD had been changed to "0" (disabled).

First I set it to 1, which activated it. I still found though that the Proxy Server in Internet Explorer was greyed out. So I then used the Proxy Server Reset Auto-Fix found here:

https://support.microsoft.com/kb/2289942?wa=wsignin1.0

... And between the two of them, my Proxy Server seems to be functioning properly. Meaning it is no longer greyed out, and I can put in whatever proxy address I want and it sticks.

I hope this helps someone.
