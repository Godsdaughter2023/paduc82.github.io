Most of the time (in recent memory) when there is a problem after installing Microsoft updates, the problem is because the computer was somehow afflicted before/when the updates were installed and the updates merely bring to the surface an existing problem that was not obvious before. 

Security updates sometimes patch holes in XP and when the security holes are patched (and XP is "tighter"), the affliction can no longer do whatever is has been doing, things that used to be allowed to happen are no longer allowed to happen as a result of the tighter security, the system now has some new problem (especially when it no longer boots), and it appears that installing the updates broke the system when in fact, it is just a coincidence.

Another thing that can happen is that the Microsoft updates will change or update some critical XP system files.  Depending on what your antivirus or antimalware software is, those programs may interpret the new or updated files as being a threat or somehow infected and quarantine (effectively delete) the files it thinks are suspicious. 

The next time you reboot your system, you will have some kind of a problem or you won't boot at all if the files are missing.  Once again, the problem is not with the updates, it is some other condition on your system with your antivirus or antimalware programs you are using (they could not be aware of what the new files from Microsoft look like).
