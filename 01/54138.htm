john_barnett say: Hi Auvache,

Firstly it isn't wise to add your email address to forum posts. By advertising your email address you are an easy target for spammers. Luckily a forum Moderator removed your email details.

It looks as if you are being asked to disable applications that startup with Windows to determine if another application is interfering with the sound of your game. Sadly I'm no gamer so I can't offer specific advice on your particular problem. What I can tell you, however, is that disabling startup applications, while easy for an expert, isn't advisable for someone who is a complete novice. You would be using the System Configuration application in diagnostic mode and, frankly, for the novice, it is knowing what to disable and what not. There are plenty of people who 'will' give advice but that advice is no good unless it is backed up with straightforward instructions on how to go about doing the job in the first place and also what might happen if you do something wrong. The process of clean boot/start can also be time consuming, especially as you have to disable each application individually and reboot then re-enable before moving on to the next application. To be honest i'm more worried about you causing other problems by you tinkering with the system configuration.

It might be a good idea to repost a question in the forum you received the information of clean boot from and ask what in particular they suggest should be disabled and what shouldn't. If they can advise what is best left alone, Failing that you need someone to look at who knows what they are doing.

To launch the system configuration click the Start button and type: 'msconfig' (without the quotes)      (You must be administrator to open the configuration utility)

Once the confiuration utility has opened, click the Startup tab and familiarise yourself with what applications start when your computer boots. Don't tough anything until you know what to disable or if you do only disable one item at a time, reboot and then check to see if the problem still persists. If the problem doesn't go away then re-enable that application and move on to the next one.

Incidentally it is a good idea to create a backup image of your hard drive, preferably to a second hard drive or removable drive, before making any alterations. You should also create a system restore point to to be on the safe side. Paragon have a free backup application:  http://www.paragon-software.com/home/db-express/download.html if you don't already have backup software.

As I have said I would much prefer that you have someone who knows what they are looking for look at your problem.

Hi,

If you receive Windows Update error 80200001 while downloading or checking for updates, you might need to change or restart the Background Intelligent Transfer Service (BITS). This service allows Windows Update to download updates to your computer.

Restart the Windows Update services and check.

  1. Press Windows Key + R, type services.msc and then press Enter.

  2. If you are prompted for an administrator password or for a confirmation, type the password or click Continue.

  3. Locate Background Intelligent Transfer Service and check the Status.

  4. If the Status is Stopped, right-click Background Intelligent Transfer Services, and then click Start in the menu.

  5. Locate Windows Update and check the Status.

  6. If the Status is Stopped, right-click Windows Update, and then click Start in the menu.

  7. Close Services.

Hope it helps. If you have any other questions, please do not hesitate to reply. We will be glad to assist you.

scottymoore say:

Did this from both PS3 and Network Enabled Sony Bravia Television - both will display my Windows 7 Media Center Music, Videos, Photos, etc..  Here is what worked for me.

On the router (a Dlink DIR-655) I manually reserved the PS3 and Television ip addresses with a HOST name.  Those were PS3 and BRAVIA52.

Rebooted the router and then went to the windows machine and tried to ping PS3 and BRAVIA52 by their host names, which didn't work (something in my router setup I am sure) but rather than waste time, I simply pinged by ip address to verify it could contact them and manually edited the hosts file to include entries for the PS3 <actual ip address> and BRAVIA52 <actual ip address>.  Just google windows host file editing if you don't know how.  Pinged them again by their host names and this time they responded.

Still, that didn't work so I turned off the "Windows Firewall" from the control panel (but only for the local network).  Viola, all media now appeared in both the PS3 and BRAVIA52 that was located on my Windows 7 PC.  Looking back, you may simply try modifying the windows firewall settings first, this is usually the issue with these types of things, but I would also recommend reserving the ip address and host name on your router since it seems that my sony products wouldn't advertise their host names.

Hope that helps.

The good news is you can switch Windows out of S Mode for free to Windows 10 Home . . .
