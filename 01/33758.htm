cmdrkeene say:

This type of cleaning will slow the computers down more because it means every user will have to wait while the cache is re-built every day.  I do not recommend it.

Remember: extra files on the hard drive will never slow down a computer.  For speed and lag concerns, it only matters how many things are being processed by the CPU in memory.  Unless those files are actually being used, they won't slow down the computer.  And if you clear them, a program that needs that cache will run over 500% slower the next time you use the program.

If you are having slowness issues, it's best to press Ctrl-Shift-Escape and determine what process(es) are actually causing the slowness (which processes are using CPU and disk interactions).

See: http://cmdrkeene.com/fix-sluggish-slow-pc

cmdrkeene say:

Clearing cache files will only make the computer slower, use more bandwidth to re-download and re-populate the cache.

It will cause your computer to be slower and your bandwidth to be used more, but if you want, you could probably just set your browser to empty temp files when you exit.  Every time you visit a page it will have to redownload image/script/code assets again, but if you have fast connection and don't care about the lag, that's fine.

As for using a cleanup program: it will clean everything it has access to clean. So in typical scenario, the task would need to run as an admin account so it has access to every profile.
