ggvic say:

So I was able to finally conclude after MUCH research AND testing, that Surface Pro 3's Marvell AVASTAR Wireless-AC Network Controller does NOT support hosted network wireless hotspot functionality. At least, that's been my conclusion using Windows 10.

As another user (saberman2) pointed out in this thread: 

SPro3 does not yet support Virtual WiFi. And this is official. The actual statement is "SPro3 WILL NOT support Virtual WiFi".  It is has been depreciated and is not in their specifications for the WiFi hardware or driver.

Once I connected my TP-LINK Wireless Usb Adapter, which does show Hosted network supported: Yes (command: netsh wlan show drivers), voila! problem solved! At that point, I've had zero issues/errors starting the hostednetwork (command: netsh wlan start hostednetwork).

One additional thing I did have to do (which no post I could find online mentioned) is that in order for my hostednetwork to consistently provide internet connectivity to my mobile device every time I connected, I had to make sure that my mobile device had the following static configuration:

IP Address: 192.168.137.2

Subnet Mask: 255.255.255.0

Router: 192.168.137.1

DNS: 192.168.137.1

I hope this input can help reduce the annoyance and frustrations I experienced, for someone else who encounters similar issues. Sigh.
