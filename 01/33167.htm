Hi, if you or anyone else has this problem i searched and searched and couldn't find a solution anywhere most experts just ignore. I've sussed the problem out it has nothing to do with the temp folders it's the windows\Installer folder that causes this problem and the installer folder uses the temp folder. It's when you accidently delete or move the installer folder, move without putting a proper shortcut to where you've moved it to. The main error in this is 1632 which is "Temp folder is on a drive that is full or is inaccessible".

I was trying to install silverlight but it doesn't matter what you are trying to install it's apps that use the msi extension, the msi files are held in the installer folder so a dodgy installer folder will stop the install and give the error 1632.

Go to windows\Installer but make sure your hidden system folders are unhidden or you won't see it anyway. If there's no installer folder then just create a new folder and call it Installer then your problem will be solved. If there's a upturned tick on the installer folder and it's inaccessible this means you've moved it and created a shortcut, right click and click properties and look where the shortcut target is and go there if it's not there then it has been deleted so you just delete the shortcut and create new Installer folder.

If you don't know how just google "unhide folders in windows *whichever windows version here" and the solution will be there at the top. Make sure you click the "show hidden folders" and untick the "hide protected operating system files".
