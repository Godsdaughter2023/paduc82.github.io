MPO an JPS files should be supported by MS - Microsoft Community

j.clement say:

MPO an JPS files should be supported by MS

MPO is recognized by many picture editors, but is not recognized by explorer.  They are basically jpg format files, but with 2 images for L & R stereoscopic viewing.  It is possible to hack explorer by extracting the definitions for jpg and putting them back as definitions for MPO.  This has been done, and I have a suitably hacked registry file.  The problem with doing this is that while you can view the metadata, you should not change it.  Any changes will result in the R image being destroyed.  I also have another hack that allows the thumbnails to be displayed, without the metadata being available.

JPS is even easier.  By extracting and personalizing the registry info for JPS, you have access to all the metadata.  JPS is basically just a JPG with 2 pictures stored side by side with the L view on the L side.

Apparently MS does have some stereoscopic adaptations.  A 3D movie in side by side format (SBS) will play, but in 2D.  The player knows that it has to expand on 2X horizontally compressed frame (anamorphic) for display.

The problem with the hacks is that the thumbnail displayed for JPS shows both frames.  I would be nice if it could just show one side, which would make the thumbnail easier to see.  The MPO thumbnail is correct, but either the ability to change metadata should be blocked, or explorer should understand the format and protect the second image along with first so as to prevent its destruction.  Actually MPO could be used for more than 2 images, so any changes should take into account this possibility.

MS could support JPS in explorer immediately!  MPO requires some more intelligence.  If I could get the necessary incantations, I could fix my MPO hack so it would allow metadata display but not change.

As to the labeling of topics, 3D is most confusing because it can mean perspective drawing to simulate 3D or true stereoscopic display.

- HPET (High Precession Event Timer) is one of those functions that can either improve, or stuff up performance,

depending on the software in question. Apparently cpu heavy software is most likely to be affected.

- With some machines you have to disable HPET in BIOS (location varies between BIOS). Sometimes in both BIOS, and

Windows (with the command you used).

Apparently some newer BIOS don't have an option to disable/enable HPET.

- If you run -  bcdedit /deletevalue useplatformclock  -  and get an error about element not found, then HPET

is supposed to be disabled in the OS.
