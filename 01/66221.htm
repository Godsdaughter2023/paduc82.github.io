Photos is an app which is little bit farther from a matured app. It doesn't offer much to configure.

However, you can try following and check after each step. Let me know the results. (Note - None of the steps below will delete any data / picture. All your data / pictures will be preserved)

1. Run Windows Store Apps Troubleshooter
Type troubleshoot in Windows Start Search box > Click Troubleshoot > Scroll down to bottom and click Windows Store Apps > Click Run the Troubleshooter and follow on-screen directions.

2. Reset Photos App
Windows+X > Settings > Apps > Apps & Features > Locate Photos by Scrolling Down > Click Advanced Options > Click Reset
Go to Microsoft Store > Click 3 horizontal dots on top right > Downloads and Updates > Check if there is any update for this app > If yes, update it.

3. If Reset doesn't work, then you should uninstall using Powershell and reinstall the Photos App
Windows+X > Click Windows Powershell (Admin) > Copy the below command and right click in Powershell to paste it

get-appxpackage Microsoft.Windows.Photos | remove-appxpackage

Now, you need to reinstall it
Use below Powershell command to install

Get-AppxPackage -allusers Microsoft.Windows.Photos | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

4. Reset the Database of Photos App
Copy following and paste in Windows Start Search box
%LocalAppData%\Packages\
And now locate Microsoft.Windows.Photos_........folder
Make a copy of Microsoft.Windows.Photos_........folder (for the sake of backup) and then Delete everything from inside the folder

robert aldwinckle on forums say:

Supposedly a "magic bullet" we have is pressing Win+Ctrl+Shift+B but doing that only switched my Second Screen Only back to my Surface and Win-P, CursorUp, Enter did not work.  E.g. that would switch from PC Screen Only to Second Screen Only.  So far I think the most reliable workaround I have found instead, is to use Ctrl-Alt-Delete for Switch User.  Somehow that seems to refresh the HDMI.  (Various other things have worked, among them resorting to taking my session away with Remote Desktop Connection and then taking it back on the screen which then magically became available.)  Meanwhile I suggest you turn on Narrator so you can hear what you cannot see.  For example, after the Win-P switch from PC Screen Only to Second Screen Only Narrator could see the Keep--Yes or No (No is the default and the timer is too short to make a decision when all you have is Narrator and your keyboard.)  But I'm not sure it would have been a good idea to indicate Keep Yes while I still couldn't see anything.  It is so flakey that we really do need some way to manually force a refresh of the video system.  I'm thinking that something based on devcon might be needed but haven't yet done anything on that tack.  In case you are interested poster Lz. used it for a related problem with W8.1

BTW while testing for another incident today involving Virtual Display I found to my surprise that Win-k switched my session to the new monitor, so that may lead to yet another workaround, similar to the Remote Desktop Connection one.  I don't know what the procedure would be for getting it back but the good news is that in both cases as soon as the session was taken away the HDMI screen came back to life.  YMMV.
