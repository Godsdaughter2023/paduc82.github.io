The logon process in Safe Mode is the same as in Normal Mode. If you want a password in Safe Mode then you must also set it in Normal Mode.

To protect the BIOS, go into the BIOS setup screen and set both a User password and an Administrator password. Note this:

  • On desktops, BIOS password protection is weak. Removing the motherboard battery for a few minutes is sufficient to clear the password.
  • People who use BIOS passwords tend to forget them after a few years. They then ask for assistance in this forum.
  • Some laptop BIOS password mechanisms are so strong that the laptop needs to be returned to the factory for a password reset.

yockeyd say: This was posted awhile ago, but as I found it searching around, others may as well. This is the answer.

Pegasus completely covered the BIOS question, so I will only add that on some desktops, its possible to reset a BIOS password with a jumper setting on the motherboard. I should also add that its impossible to secure a machine completely without securing the hardware- if you really want to make a system impenetrable, you must lock down the case, not just the software. Also note that no matter what you do, its possible to undo it somehow.

As for safe mode, there are three main options for restricting it. First, you can edit registry files to disable it completely. WARNING: This means if you need safe mode, you can't access it. Period.

This method is described here, here, and described best here. Its worth noting the third link also shows how to disable just the safe mode with networking; if you are trying to keep users from circumventing security software (such as parental controls), this would let you do so and remain capable of using minimal safe mode (from there, its also possible to restore network safe mode). This does require any user who might re-enable safe mode be restricted from editing the registry.

The third link above also mentions a program which will disable safe mode. I have not used it, so I don't know if its good or not.

The second method is to use a third-party program to restrict safe mode. I only know of one program to do this, NoSafeMode. I don't know if it works or not, as I've never used it.

Pegasus also mentions password protecting safe mode. This is the third method. This doesn't mean there is a password for safe mode, but the user accounts have a password. Make sure all users require a password, including the default administrator password (or disable it). Look into the "net user" commands for how to verify passwords are set. Also, make sure only users who need admin access actually have it; others can be given different levels of access.

There are at least three ways to disable the default administrator account. A search will bring them up quickly enough, so unless someone asks I won't post them here.

Intel I217-V network adaptor Windows 10 looses random connection, - Microsoft Community

hansmaasnl say:

Intel I217-V network adaptor Windows 10 looses random connection, SOLVED

Hello all, i want to share my SOLVED PC problem ;

Problem : when streaming video, for example YouTube movies for say 30min to several hours, randomly the connection from Pc to internet gets lost,

when pinging, constant time out occurs .

Only option to do : reset Intel LAN adpater or restart W10, it runs fine for some hours, then same problem.

After long testing and read a lot of forums, i solved the problem. (i am a technician myself..)

First system summery ;

 

Mainboard ASUS Z170 Pro Gaming with CPU Intel i7

Memory 16Gb HyperX Predator DDR4 at 3200Mhz, XML

Integrated Intel i219-V onboard Lan controller with latest Intel driver 22.10.3.0

2x 250Gb SSD

2x 2 TB HDD

OS Windows 10 64bit, last build upgrade and up to date.

used Browser : Google Chrome, latest version

 

CAUSE ;

 

The Google Chrome Browser !!!!! (with other browsers NO problem, IE or Edge, this were the ones i had installed)

Why ?

Google Chrome Browser uses at the background an experimental protocol, the QUIC protocol.

This causes random that the connection of the Intel i219-V crashes !

When disable this protocol in Chrome, find it at this page in Chrome, "chrome://flags/" (default its on), then all the problems are gone !!!!!

 

I am happy again 

and nothing is wrong with the hardware !

 

Maybe Intel can make a new driver to solve this problem with the rediculous Chrome QUIC protocol

 

Note : I also have an extra Realtek LAN controller in this Pc, this one did not have problems with Chrome QUIC

 

Hope i could help you and a lot of other people........
