a. user say:

I don't believe this is possible. It is possible to install/boot from a bootable external HDD with some minor revisions http://www.mydigitallife.info/2009/07/15/how-to-install-windows-7-or-vista-from-bootable-portable-usb-flash-memory-key-or-hard-disk-drive/ (assuming your BIOS supports this) - say if your CD/DVD failed, to do recovery or re-installation, but that's not the same as running Vista from the External HDD.  I believe the major problem is the drivers which don't get loaded until Windows starts and Windows couldn't start without the USB drivers already loaded if it were on a USB External Drive (and the other drivers as well) and there may also be some issues with the boot files.  There may be other issues as well - I think XP and Vista are INTENTIONALLLY DESIGNED not to work in this way (For piracy reasons among others)..

I've read a post by Carey Frisch, MVP, who says flatly that it can't be done.  Here's an article on the subject which also flatly states that it can't be done with XP or Vista http://computer-drives-storage.suite101.com/article.cfm/can_you_boot_from_external_hard_disk_drive.

I've read some articles that say it is possible with XP using the proper BIOS and a whole bunch of changes to the Registry (changes which weren't identified) but I frantly doubt it.  Too many other articles argue against it.  Furthermore, it would be MUCH slower (no matter how fast the hard drive) because of the connection method - you wouldn't want to use it this way on a daily basis.

A lot of external hard drives when taken apart are compaatible with and will fit inside the computer case and work as internal drives - perhaps you can try that as an alternative (assuming you don't want to buy another internal HDD at this time).

To test your hard drive, check the manufacturer and then get the diagnostic utility from: http://www.techsupportforum.com/hardware-support/hard-drive-support/302602-hard-drive-diagnostic-utilities.html.  If it fails the test, replace it.  If it passes the test, then there's probably nothing wrong with it.  You can also run chkdsk /f /r from the elevated command prompt and see what happens - it you gets lots of bad sectors and a lot of corruption, then it's time to replace the drive before it fails totally..

I hope this helps (though it probably isn't the answer you were hoping for).

Good luck!
