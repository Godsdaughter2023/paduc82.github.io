¡firedog say:
That indicates that there's something really wrong with your installation. There shouldn't normally be more than a handful of these files in the store folder. Each time a transaction takes place in the program - a message is received or sent or read or moved, for example - the transaction is recorded in a file called edb.log. Whenever the edb.log file grows beyond 2MB, its name is changed by adding a serial number and a new edb.log file is taken into use. In quiet moments, these log files are replayed and the details of the transactions written to the main message database file (Mail.MSMessageStore). As the transactions recorded are written to the database, the log file is deleted. When Windows Live Mail is closed down, the program finishes replaying current log files and updates the check file.

Since this problem has clearly been going on for some considerable time, it's not certain that it can be remedied. I would have expected to see very poor program performance - have you not noticed? Why are you still using a version of the program released in 2010?

Repair will involve using database management tools in an administrator command window. Are you prepared to give that a go? It will be a complex and possibly frustrating exercise, but with any luck we can get those orphan log files replayed and your database back to a more normal situation.
