    thomas lbaron say:

    All, 

    I have resolved the problem in a quite interesting way. I did some more Googling and found a "hack" on YouTube, that seemed to resolve my issue. 

    Here is a short description that worked:

    Problem: I cannot delete or take ownership over a folder, that was created with a User that no longer exists. User was not deleted per se, but was removed during a roll-back to a previous Restore point. Files and Folders remained and one specific folder could not be deleted nor could ownership be changed to Administrator. 

    Resolution:

    1. In Root folder of C:-drive create a folder named "Windows.old"

    2. Move the folder, which could not be deleted, from Users to the new "Windows.old". Despite lack of deletion permission, it is possible to move the folder.

    3. Run the Desktop application "Disk Cleanup" on C:

    4. After 10 seconds you get the option overview to delete various forms of files and folders. Among these it shows "Previous Windows installation". Check that box and execute the Disk Cleanup. Ensure other boxes are unchecked (unless of course, you want to perform cleanup on other files and folders).

    5. Voila - Windows deletes the folder for you without any issues. 

    In my world this may be borderline to a security breach in Windows (that you can actually delete other users' files by performing the steps above), but I do not complain, as it resolved my issue.
