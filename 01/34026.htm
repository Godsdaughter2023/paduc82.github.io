Hi,

 

Basically the difference is the price and the support....

 

Prior to Jan. 31, you could buy and Upgrade for $39.99, and the OEM System Builder was $139.99...

 

Now, the upgrade is considerably more ($199?) than the OEM System Builder, but comes with Microsoft Support...

 

If Win 8 was like Win 7 and Vista, it would cost nearly $300 for a full non OEM version, that again, comes with Microsoft Support...

 

I might suggest that you try something like Acronis TrueImage and make a backup image of your installation of Win 8 while it is fresh... Then in 15 minutes, you can reload it and have a fresh installation should the need arise...

 

You might even want to make one of Win 7 prior to upgrading to Win 8... just in case...

 

Acronis compresses the images, so they take up considerably less then the actual OS drive\partition.

B.  Disabling the warning can lead to more serious problems.  You say that you're aware of the dangers of an overfilled hard drive.  One such danger is that when the disk space gets too low, System Restore will shut off on all drives, not just on the drive with low space (I believe that statement is correct, but I can't find the reference at the moment).

C.  Everybody and his brother points to that same registry edit.  That advice first appeared for Windows XP and I wonder if it is still valid for Windows 7.  As shown in the screenshot at the link provided by (PB & Jam), in Windows 7, there is no key named

  HKEY_CURRENT_USER\Software\Microsoft\ Windows\CurrentVersion\Policies\Explorer

So if you wanted to follow the directions in say, http://support.microsoft.com/kb/285107, (which "applies to" XP Pro and XP Home), you would have to add the steps indicated in bold below:

  1. Click Start, click Run, type regedit, and then click OK.
  2. Locate and then click the following key in the registry:
    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies
  3. On the Edit menu, point to New, and then click Key
  4. Name the new Key Explorer
  5. Click on the new key, that is, click on
    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\Explorer
  6. On the Edit menu, point to New, and then click DWORD Value.
  7. Type NoLowDiskSpaceChecks, and then press ENTER.
  8. On the Edit menu, click Modify.
  9. Type 1, and then click OK.

Note that in Windows XP, the key named

    HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\Explorer

DOES exist, as shown below:

Experiment with the registry at your own risk.  I strongly suggest that if you choose to do so that you first create a good backup by creating a System Restore point and/or using the free utility ERUNT.

a. user say:

Hello,

 

You may be able to selectively recover the deleted folders using the Previous Versions component. You will need to know the name of the folders that were deleted. Here is the step by step procedure.

 

1. You will first need to select the Show hidden files option.

    a. Open Windows Explorer and click Organize / Folder and search options. Click the View Tab.

    b. Place a check mark in the Show Hidden files, folders and drives option. Click Apply/OK.

    c. Close and restart Windows Explorer.

 

2. Open the WLM Store folder.

    a. Navigate to the following folder.

    C:\Users\your name\ AppData\ Local\ Microsoft\ Windows Live Mail

    (Where 'your name' is the name of the account that you use to logon to Windows 7)

    b. In this folder you will see all of the WLM storage folders. Each email account will have its own group of folders.

    c. Any general storage folders that you created will under the main Storage Folder item.

 

3. Make a copy of the Windows Live Mail folders previous version.

    a. Right click the Windows Live Mail folder and select Properties / Previous Versions Tab.

    b. After the list populates, select a version of the folder with a date and time prior to when the folders were deleted.

    c. Click the Copy Button. In the Copy dialog box, select the Desktop as the location to place the copied folder.

 

4. Move the previous versions of the deleted folders to the WLM store folder.

    a. Go to the Desktop and open the copied WLM folder. Locate the folders that were deleted.

    b. Right click the folder and select Copy. Go to the current WLM storage folders and paste the folder into the same location where it was deleted from.

 

After you have retrieved all of the deleted folders, you can delete the copied WLM previous version folder from the Desktop.
