Hi Jaime,

 

Thank you for writing to Microsoft Community Forums.

 

Usually this issue occurs if the Windows Time service is corrupted or the CMOS battery is drained. Though you have mentioned that you have tried restarting the Windows Time service, please ensure that the startup type of the service is set to automatic.  I would suggest to try the methods given below and let us know how it goes.

 

Method 1: Re-register Windows Time DLL file.

  • Click on Start and type CMD.

  • Right click on Command Prompt and select Run as administrator.

  • Type Regsvr32 W32time.dll and hit the Enter key.

  • Sync the system clock with Time.Windows.com, reboot the computer and check the results.

 

Method 2: Reregister Windows time process.

 

  • Click on Start and type CMD.

  • Right click on Command Prompt and select Run as administrator.

  • Type and run the following commands one by one.

  • Net Stop w32time

  • W32tm /unregister

  • W32tm /register

  • Net Start w32time

  • W32tm /resync

  • Reboot the computer and see if the time syncs with the Internet time.

     

    Method 3: Replace CMOS battery.

    Check if the correct time is set in BIOS and change it if BIOS doesn’t show the correct time. Refer to the system manual to know how to access BIOS settings.

     

Note: Modifying BIOS/ complementary metal oxide semiconductor (CMOS) settings incorrectly can cause serious problems that may prevent your computer from booting properly. Microsoft cannot guarantee that any problems resulting from the configuring of BIOS/CMOS settings can be solved. Modifications of the settings are at your own risk. 

 

If none of the above methods works, the issue could be with the CMOS battery which is responsible for maintaining the system time and date. In that case, I would suggest to consider replacing the CMOS battery.

slance310 say:

Hi,

I have read a number of reports of the Microsoft Installation flash drives being defective over the last couple of years...

There s a trick for creating and using the installation flash drive...

For Legacy Bios the flash drive needs to be "Initialized" as "MBR" and Formatted "NTFS"...

For UEFI Bios the flash drive needs to be "Initialized" as "GPT" and Formatted "FAT32"...

A free tool called RUFUS will create either as you choose, from a downloaded Microsoft ISO...

You can do this instead of creating a flash drive with the Create Media Tool by selecting different options...

The Create Media Tool can sometimes create the wrong type of drive, especially if it is uses on a different machine...

You may also need to use "Boot Options" at the POST screen to select a "UEFI USB Drive" as opposed to simply "USB Drive"... Sometimes the machine recognizes the flash drive correctly, and sometimes not... This is due to the Bios itself, and not Windows...

Things are not quite the same as the floppy disk and CD installation days... The newer UEFI Bios stuff really threw a wrench into that...

BTW... One way to tell if your machine is using UEFI Bios is the presence of a small FAT32 "EFI" partition on your OS HDD\SSD...
