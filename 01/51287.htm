sean lambert say:

To answer your batch file question:

if NOT "%errorlevel%" == "0" ping localhost >nul &goto :CompatCheckRunning

So it checks whether the current value of the errorlevel variable 0.  This is usually set by programs when they exit, and while the meaning is program specific, 0 typically means whatever was just above this was successful.  In this case, the last program was findstr, which was searching the output of some other command for the word "Ready".  If it found "Ready" anywhere, %errorlevel% should be 0.

The next part is "ping localhost >nul" which means send a network ping to yourself, and ignore any responses.  By default it will ping 4 times, which takes about 3 seconds.  Since we're ignoring the results of the ping, this is probably just an obscure way to say "wait 3 seconds".

Finally "&goto :CompatCheckRunning" which means "and then go back to the step labeled CompatCheckRunning".

I think maybe it wasn't "stuck" on this command, it was hitting this command repeatedly every few seconds.  It looks like there was some kind of scheduled task running in the background checking the compatibility of your system, and this batch file was checking on its progress every few seconds to see if it should continue on with whatever comes after this.

It's possible that the word "Ready" was never going to appear, since it looks like your system ist auf Deutsch...  =]

I'm glad you got it worked out!
