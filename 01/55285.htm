tryx3 say:

If your aim is to prevent the Windows 10 update then you are going about it the hard way.

Instead of your current approach you can apply this simple fix.  It achieves everything that the more complicated fixes do.

- - - - - - - - - - - - - - - - - -

Open Notepad, paste in these lines

Windows Registry Editor Version 5.00

[HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\GWX]

"DisableGwx"=dword:00000001

then save the file as DisableGWX.reg

and then run it by double-clicking or by right-clicking then selecting Merge.  [Either way, you'll get confirmation dialogs that you have to accept]

[This fix was originally posted in this forum by the admirable Frederich Long]

- - - - - - - - - - - - - - - - - -

You will never be troubled by Windows 10 again - its icon will disappear, it will not be downloaded or installed.  There will be no side-effects.  You will still be able to use Windows update as normal and you will not need to take any special action either before or after running Windows update, you just let it run as you always did before Windows 10 existed.

In particular, this solution avoids:-

  • Having to change any of your Windows update settings.
  • Inhibiting normal operation of Windows update [some of the other ways of avoiding the Windows 10 upgrade involve suppressing Windows update itself]. 
  • Having to review or remove any updates you have already installed.
  • Having to review the list of new Windows updates to hide any of them [I understand that an alternative solution involving GWX Control Panel can involve some sort of weeding out of particular updates depending on its operating mode].

I ran this solution in August 2015 and it has operated reliably ever since.  I have just repeated a Windows update check [1 April 2016] and this solution continues to work - 30 or 40 updates were listed, downloaded & installed because I had recently restored an old system image but Windows 10 was not mentioned anywhere.  I accepted every update listed including all the optional ones.

If you ever change your mind and you want to get the Windows 10 upgrade then you can just use RegEdit to delete the Registry key [HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\GWX] created by the DisableGWX.reg file.

lemp say:

Yes to show all users

Was System Idle Process showing around 97% or so above MsMpEng?  If so, it would seem that by the time the system "froze" whatever had caused the problem had stopped.  Unless you can visually spot the culprit before the freeze, you'll have to investigate this problem a different way.

Is there a way to trap the last program that ran (prior to lock up)?
Not that I know of.  The tool that comes to mind as the most likely to help in this situation is Process Monitor.  Unfortunately, this is not a simple tool to learn how to use.  If you want to try, download Process Monitor here --> https://technet.microsoft.com/en-us/sysinternals/processmonitor.aspx?f=255&MSPPError=-2147217396

Then watch the two half-hour videos linked on that page under "Learn more."  You can also check out some of the links under "Additional resources."

It's probably simpler, however, to approach this problem by systematic trial and error troubleshooting, even though it's a tedious, time-consuming process. 

You've already discovered that the problem does not occur when you start the system in a "clean boot" configuration.  This means that the problem is being caused by a program or service that runs during normal boot but not under clean boot.

In a clean boot configuration all of the non-Microsoft services are disabled and all of the programs that start when Windows starts are disabled.  If I were you, I would leave all of the startup programs disabled and re-enable all of the non-Microsoft services.  Reboot in this configuration.  If the lockup still happens, you know that the problem is not a service.  Conversely, if the lockup does not happen, you know that the problem is one of the non-Microsoft services.

If the problem is not a service, re-enable half of the startup programs and reboot.  If the lockup still happens, the problem is one of the programs you re-enabled.  Disable half of those and reboot again.

I think you can see where this is going.  Eventually, you should be able to zero in on one startup program or one non-Microsoft service that is causing the problem.  At that point, you can decide if you need to fix the misbehaving process or simply uninstall it or otherwise prevent it from ever running again.

Why doesn't the even viewer show any symptoms?
"Lock up" is not a precise term.  I assumed that what happened is that some software has entered a tight loop that consumed most of the CPU cycles and prevented anything else from happening.  I suppose there are other ways that the system can become non-responsive.  Because whatever is causing the problem is presumably an error that is not being properly handled by the software (unless it's malware), the software never gets to the point of issuing an error event and thus nothing appears in the Event Viewer.
