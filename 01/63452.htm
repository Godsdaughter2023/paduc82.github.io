considerate_guy say:

The project file (.wlmp) is just a snapshot of your editing state. The references to the source files on your computer in the form of folder and file names.

It's kind of like a recipe that says, "get the eggs out of the second shelf of the refrigerator" and add them to two cups of flour from third shelf of the pantry." Now if you decide to move those ingredients, or even worse, you remove those ingredients completely, your baker will be unable to follow the recipe.

In order for Movie Maker to restore your editing session back to the original state, it must still be able to find all your videos, photos and music where they were before. If you had them on the desktop and you move them or delete them, then Movie Maker's project file won't be able to load them back in.

So the key, if you still plan to make further edits, is to not touch your source files. Don't move them, rename them or delete them.

The fix, after 2 days of troubleshooting relates to the CNG Key Isolation service found at sevices.msc

At some time or another, this service is somehow set to Manual instead of being set to Automatic. If you set the service to Automatic, all problems are gone. I can do a video if anyone wants to see it? Hard to explain in text.

The problem I have is well... why? The description is "The CNG key isolation service is hosted in the LSA process. The service provides key process isolation to private keys and associated cryptographic operations as required by the Common Criteria. The service stores and uses long-lived keys in a secure process complying with Common Criteria requirements".

Yeah that makes perfect sense as to why the computer becomes unresponsive for a whole minute and I can't search for files.

The CNG Key Isolation service runs as LocalSystem in a shared process. It shares an executable file with other services. If the CNG Key Isolation fails to load or initialize, the error is recorded into the Event Log. Windows  startup should proceed, but a message box is displayed informing you that the Key Iso service has failed to start.

 

Dependencies
