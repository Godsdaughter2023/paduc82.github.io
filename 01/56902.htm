r_k_k say:

System Update Readiness tool takes a long time to run.


MS Update provided me the Readiness Tool only after my Update of SP2 failed and after I did a computer Restart.  I tried running the Tool for the first time and finally existed after 2 hours of running on my system because as the excerpt below stated, the scan typically takes supposedly less than 15 minutes to run.  Some scans might take *significantly* longer on some computers; 2 hours was 8-times significantly longer but not long enough and my scan seemed ‘hung’ at 80%  along the green status bar when I decided to cancel-out of the scan.

 “After you download the System Update Readiness Tool, it runs a onetime scan for inconsistencies that might prevent future servicing operations. This scan typically takes less than 15 minutes to run. However, the tool might take significantly longer on some computers. The Windows Update progress bar is not updated during the scan, and progress seems to stop at 60% complete for some time. This behavior is expected. The scan is still running and you should not cancel the update.”

 The Readiness Tool available through MS Update only runs once.  If the tool fails, or if you cancel out of the scan too soon, it is not updated again for availability through MS Update.  However, I decided to download the Standalone Readiness Tool—not available through the regular MS Update—to run on my system if I later decided to install SP2 after the problems have been resolved—as noted abundantly within this forum.  Here are the steps I followed to successfully run the Tool scan.

1)  D/L Standalone Readiness Tool package (KB947821) and saved to my own file folder called Tool Box.

2)  Under Norton Internet Security, I disabled AnitVirus AutoProtect for 5 hours and Turned Silent Mode ON for 6 hours.

3)  I started running KB947821, and after a few initial screens, I got the Installation Initialization Done!

4)  The scan started and the Installation status bar in the dialog box displayed, although it *never* turned green nor did it give any measure of the complete process.

5)  After 1 hour’ time, I decided to turn on Task Manager (TM) to see if the Tool scan was working—i.e. not hung up—and I noted up to 36% CPU use, heavy Memory use, and though the TM Resource Monitor (RM), that files were being scanned by Checksum and Expand.exe.  I recommend that if you are concerned if any scan or program is “doing something” and that it is not hung-up, confirm that activity through TM and RM.

6)  After 2 hours and 55 minutes of running the scan with no other use of my computer during that period, I finally got the message within the Dialog Box stating that ‘Installation Complete’.

7)  For confirmation through MS Update, I ‘checked for updates’ and the ‘Update for Windows (KB947821)’ appeared as Successful (before when I cancelled after 2-hours of scanning, I got a Failed message in my View Update History under MS Update).

Therefore when Microsoft states that something will take 15 minutes or less and that some computers may take significantly longer, realize that, in my case, that significantly longer was 12 times longer (12 times 15 minutes = 180 minutes approx. or 3 hours).
