Hi FW, this is a known issue in the 1803 Update,

It is the recovery drive, you are not meant to see it, it is not meant to have a drive letter, remove the drive letter and everything will return to normal:

Click your Start Button, type cmd, then right click Command Prompt and choose 'Run as Administrator'

Run this command and hit Enter

diskpart

Run this command and hit Enter

select volume D

Run this command and hit Enter

remove letter=D

Close Command Prompt then restart your PC

That's It!

ethan b. say:
Hey FWShea,

This is a recovery partition brought into view in 1803, you don't have to increase the size of this. You cannot remove, but only you must remove its mapping of its respective drive letter, which is D. Once you do this, it brings it back to its intended visibility, being hidden and out of sight. Once you do this, you won't be pestered with low disk space notifications anymore. You will need to remove the drive letter and mount point in DiskPart in order to do this.

Hit Windows key + R together to go into Run. Type in the following: diskpart

Enter the following in the new DiskPart window:

select volume D
remove letter=D

Does not work after a reboot? Try entering this instead in an administrator session of Command Prompt: mountvol D: /D

paul melia say:

After 3 insider preview updates I was left with windows.old, windows.old(1) and windows.old(2) folders in the root of C:\ and nothing I'd tried worked (CCleaner, Disk Cleanup, rd at the cmd prompt, and so on).

I've just worked out a solution, however; firstly, the problem is probably caused by the 2 trailing periods to the name of the folder (LocalState..).

To delete it you need to get ownership of that folder and remove the trailing periods from its name which you can do using an elevated command prompt and the old 8.3 file and folder naming convention, so in my case to get ownership of the folder:

takeown /F c:\Windows.old\users\Paul~1.MEL\AppData\Local\Packages\Micros~1.COR\LOCALS~2 /r /d y

and then to rename the folder to remove the trailing periods:

ren c:\Windows.old\users\Paul~1.MEL\AppData\Local\Packages\Micros~1.COR\LOCALS~2 local

You should now be able to right click and delete the windows.old folder in Explorer, giving affirmative response to any prompts.

Hope this helps.
