storm kraken say:

Problem solved: they say that the hardest bug to debug is the one that doesn't exist (i.e. the "bug" that is actually a feature).  Same thing is true for this problem I guess.

There was something I mentioned on the Dell forum that I should have mentioned here.  I didn't think it would be important, but it turns out it was.  My laptop keyboard has a little moon over the [F1] key, so naturally I assumed [Fn-F1] to be the sleep key, like on every other system where I've seen that symbol.  However, I had discovered a few months ago that on my system, pressing [Fn-Esc] caused the computer to hibernate (and the sleep key had been configured to hibernate at that point in time).  Since I was pretty sure that [Fn-F1] was the sleep key, I assumed that [Fn-Esc] was a key combination that Dell had set to always cause hibernation. 

Today, I pressed [Fn-Esc], and when my computer slept instead of hibernating, I discovered that I actually had it backwards:  [Fn-F1] is the permanent hibernate key, and [Fn-Esc] is the sleep key.  Why Dell did this I don't know, especially since as far as I know, none of my documentation indicates the existence of a sleep key or that the sleep key was [Fn-Esc].  But anyway, my problem is solved: changing the action of the "sleep button" in Control Panel changes [Fn-Esc] to whatever I choose.

Thank you for your patient help in troubleshooting this, even though the problem was due to an invalid assumption on my part and not any software issue.

I think we can fix this by disabling the problematic shell extension.  Shell extensions provide additional choices on right-click menus.  For example, the McAfee program might add a "scan this file with McAfee" option to the right-click menu of files.  If you experience a crash or delay when you right-click a file, it means that one of these extensions did not load properly.
