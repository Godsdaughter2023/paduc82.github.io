    At least there's usually multiple ways of doing things in Windows!  Try this:

  1. Press [Ctrl]+[Shift]+[Escape] to start Task Manager.
  2. Click File -> Run new task.
  3. Type "cmd" into the Run dialog, tick Create this task with administrative privileges and click [OK].
  4. A command prompt window should open.
  5. Type/copy the following text and press [Enter]: net user Admin123 MyPassword /add
  6. Type/copy the following text and press [Enter]: net localgroup Administrators Admin123 /add
  7. Log off and login to the new Admin123 account.  If it works correctly, you can migrate your data to the new user account.  You are welcome to replace "Admin123" with your username of choice (just enclose it in quotes if it has a space, and be sure to use the same name in both commands).  The login password will be "MyPassword".  You can easily change it later.

    The steps I gave in both of my posts above create local user accounts.  I intentionally didn't use Microsoft accounts because they tend just connect with your existing corrupted account after seeing the same email address, fixing nothing.  But, some of the extra features with apps and tiles are only available with a Microsoft account, so if you're the kind of user that likes tiles and apps, you'll want to upgrade to a Microsoft account.  Assuming you have successfully moved all your files to your new user account, you can delete your old, corrupted account.  Then, you have two options:

  • Connect your new account to your Microsoft account (Settings -> Accounts -> Your account -> Connect to a Microsoft account).  After doing this, your apps and tiles should work, but you will probably have to manually add them to your new Start Menu to make it look more like your laptop.
  • From your new account, create yet another user account, using Settings -> Accounts -> Family & other users -> Add someone else to this PC, and then following the prompts to either create a new Microsoft account or enter the credentials for your existing Microsoft account.  This should result in a default set of Start Menu tiles.

¡firedog say:

Windows Live Mail has not supported the use of stationery since the introduction of WLMail 2011 two years ago. To get your black background, you'll have to use workarounds.

One simple, but not 100% satisfactory, way of doing this is to create a signature that uses white text on a black background. Apply the signature to a message, then insert your images inside it. The black will stop at the last line of text, so you will see white space below that.

Such a signature might look like this:

<div style="background-color: #000000; font-family: 'Segoe UI Semibold', 'sans serif'; font-size: 13px; color: #ffffff; padding: 20px;">
Hi!<br>Love<br>Grandpa<br><br><br><br>
</div>

Copy that into a new Notepad document, edit it to suit, then save it as black-sig.htm (with file type All files selected in the Save dialogue). Then in WLMail, press Ctrl-Shift-O for Options and open the Signatures tab. Click New, select File and browser to the .htm file you just saved. Select other options on the tab as appropriate. Then in a new mail message (Ctrl-N), select Insert > Other > Signature. Start typing after Hi! to make sure you get white text, and use Shift-Enter for new lines. If you use just Enter, you may find that the text you type is black. Use Single photo on the Insert tab to place your pictures. Here's the result:






Another workaround is to create a template with a black background. This will give an all- black window, but again, you may have trouble with persuading the text to be white. Post back if you'd like those instructions .
