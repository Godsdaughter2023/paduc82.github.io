darian tab say:

We recommend following these steps to enable the built-in administrator in command prompt at boot.

  1. Boot from your Windows 10 installation USB or recovery drive, and open a command prompt at boot (Shift+F10).
  2. In the command prompt, type regedit, and press Enter.
  3. In the left pane of Registry Editor, click on the HKEY_LOCAL_MACHINE key.
  4. Click on File (menu bar), and on Load Hive.
  5. Open the drive (ex: D ) that you have Windows 10 installed on, and browse to the location below. Note: The drive letter (ex: C) will not always be the same at boot as it is from within Windows 10. D:\Windows\System32\config
  6. Select the SAM file, and click on Open.
  7. In the Load Hive dialog, type REM_SAM, and click on OK.
  8. In the left pane of Registry Editor, navigate to and open this key: HKEY_LOCAL_MACHINE\REM_SAM\SAM\Domains\Account\Users\000001F4
  9. In the right pane of the 000001F4 key, double click on the F binary value to modify it.
  10. In the first column of line 0038, change 11 to 10, click/tap on OK, and go to step 13 below. Note: You would do this by clicking to the left of 11 to place the cursor there, press the Delete key, then type 10.
  11. Close Registry Editor and the command prompt.
  12. Click on Continue to startup back in Windows 10.
  13. Close Registry Editor and the command prompt.
  14. Click on Continue to startup back in Windows 10.

When re-installing Windows, keep in mind that using a RAID array is counter-productive in a home environment:
  • It provides no protection against 99% of all problems that home users encounter (e.g. user error, virus infection, file system corruption, file corruption, bad drivers, registry problems, loss of data due to lack of external backup etc.). RAID is NOT some advanced type of data backup solution!
  • It adds a level of complexity that can cause big problems unless you're an expert on RAID arrays and drivers. What would you do, for example, if one or the other disk failed?

The only advantage of a RAID array is its ability to survive a disk crash without downtime. This can be important in a commercial environment but is of no value in a home.

There are a total of 5 locations in the Registry for that CLSID on a Windows 7 64bit system.

 

HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Classes\CLSID\{ECD4FC4D-521C-11D0-B792-00A0C90312E1}

 

HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Wow6432Node\CLSID\{ECD4FC4D-521C-11D0-B792-00A0C90312E1}

 

HKEY_LOCAL_MACHINE\SOFTWARE\Classes\CLSID\{ECD4FC4D-521C-11D0-B792-00A0C90312E1}

 

HKEY_CLASSES_ROOT\Wow6432Node\CLSID\{ECD4FC4D-521C-11D0-B792-00A0C90312E1}

 

HKEY_CLASSES_ROOT\CLSID\{ECD4FC4D-521C-11D0-B792-00A0C90312E1}

 

That CLSID is associated with the C:\Windows\system32\explorerframe.dll file and C:\Windows\SysWOW64\explorerframe.dll.

 

The ExplorerFrame.dll file provides the UI for most of the Buttons, Menus, and other UI components in Windows.

vijay a. verma say:
You can try to disable temporarily. I am assuming you have tried all other options.

Unlike majority of the default Windows apps, Cortana can not be disabled through Powershell. Microsoft wants people to use Cortana.

If you want to completely turn off Cortana, then you will need to resort to a registry fix (Note - It is recommended that you create a backup of your registry before modifying so that if you commit some mistake, you will be able to restore - https://support.microsoft.com/en-us/help/322756... )

Windows Key+R > Type regedit and Enter
Go to the key - HKEY_LOCAL_MACHINE > SOFTWARE > Policies > Microsoft > Windows
See if you have a directory named - Windows Search
If not, then right click on Windows folder > New Key > Give the name Windows Search
Click Windows Search > In the right-hand side pane, right-click in the empty area and choose New > DWORD (32-bit) Value. Type AllowCortana > Enter
Double-click AllowCortana, and type in 0 under Value Data.
Reboot your computer

Source - https://answers.microsoft.com/en-us/windows/for...
