tryx3 say:

I have done what you are trying to do [copied/moved password reset disks from one drive to another]. 


You only use the wizard once for each user account.  So, to tidy up your drives, delete all but the latest psw file now.


Once you have made the psw file, you can copy it wherever you want and it will work when you need it as long as it is in the root folder of an external drive and it is still called userkey.psw.  As you already know, the only Password reset disk that will work is the latest one made for that user account [on the same PC & the same OS].  Within those constraints, during storage you can hide it, rename it, give it a different file extension, zip it, make multiple copies of it - anything, as long as you don't tamper with its contents or create a new one and as long as you are able to put it back in the root folder with the correct name when it comes to needing to use it.


You can also continue to use the external drive for other purposes - having the psw file on it does not create any limitations.

tryx3 say:

1  For how to use the Password reset disk see the guidance at

Recovery method 2 - Password reset disk          [PDF version]


Precautionary measure 2 - Make & test a Password Reset Disk for every account          [PDF version]


2  In addition and this is very important

Create two additional password-protected admin accounts and keep their passwords written down somewhere secure [behind the boiler or in the attic].  This precaution guards against the user profile corruption that can & does occur with any user account.  getting stuck without a working admin account inhibits all recovery operations.  See the guidance at

Precautionary measure 1 - Create two additional admin accounts          [PDF version]


3  As a direct, although not as useful, response to your last post - Windows really dislikes accounts being renamed as it was not designed to cope with this.  It is impossible to predict how it will respond to Password reset disks before or after the renaming.  Personally, I would make a new Password reset disk for each renamed account and then use each one as a test. I think you should never try to rename the additional admin accounts I have recommended at para 2 above so that you can be confident that there is nothing to interfere with system recovery in the event of problems.


4  As you are dealing with multiple user accounts, you might also wish to consider how to keep all the Password reset disks.  There are probably other solutions but here's mine.  I have a USB that lives behind the boiler.  In it is a folder PSWs that has subfolders Admin1, Admin2, User1 & so on to match all the user account names I have made.  In each folder is the Password reset disk for that user account.  In the root of this USB [to ensure it is ready for immediate use], I have another copy of the Password reset disk for Admin1.  This scheme avoids having to keep multiple USBs with the different Password reset disks on them.  If the problem affects Admin1 then I am ready to go; if the problem affects another account then I can log on with Admin1 and copy the appropriate Password reset disk into the USB's root folder.  It's always copy rather than move so I don't accidentally overwrite & lose one of them.
