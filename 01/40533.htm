Hi,

 

In Windows 8, right-clicking on a program in the Start screen won’t display Run as administrator option. While Run as administrator option is available when you right-click on a program shortcut on the desktop, the same option doesn’t appear when you right-click on program shortcut in the Start screen. So, how to run a program with administrator rights in Windows 8? Here is how:

 

Step 1: Switch to the Modern Start screen and start typing the program’s name to see your program under Applications list.

 

Step 2: Right-click on the program name to see Advanced option.

 

Step 3: Click on Advanced and then select Run as administrator option.

 

You can also use the good old hotkey (Ctrl + Shift + Enter) method to run a program with admin rights. That is, for example, type CMD in Start screen and then press Ctrl + Shift + Enter keys to open elevated Command Prompt.

luizeduardo5 say:

After a LOT of trying and digging I figured out what the problem is and the answer is... it`s not a problem, but a windows feature and it`s present on any PC, Laptop or tablet that runs it at all times, even if you`re using a mouse.

So what is it? Well, it`s a snapping "feature" windows has that snaps whatever you`re trying to hold and drag to the cursor`s original location. Once you live the "deadzone" area, whatever you`re trying to drag will move to your where your cursor currently is at. I figured this out after opening Zbrush`s infos and noticing that it was in fact recognizing every touch and action I was doing with my surface pen and that not only the "lag" would happen when I was trying to sculpt, but also anywhere else on the screen, including interacting with the UI.

If you wanna get to this conclusion yourself too, try opening your desktop on your surface and then try holding and dragging a folder around. You`ll notice right away that the same thing that happens on zbrush also happens on your desktop. Your folder will stay in place even though the pointer under your tablet`s pen wont and then it`ll jump to the pointer`s location once it moves out of the "deadzone".

Anyways, I figured this could shine some light on this problem since now we know it`s not software specific, but a windows thing. Personally i imagine that`s one of the things that companies like Wacom fix when they release drivers for their own tablets, but since Zbrush is not as globaly used as software like photoshop and pixologic isn`t as big as Adobe, no driver has been done to get around it.


Sadly I don`t know how to fix it yet though, but I`ll post it hear if I find out.

luizeduardo5 say: I wanted to post an update on this issue.

I`ve been making some progress on debugging this. It seems like this isn`t actually a bug, but a windows feature. It`s a height and width setting for dragging that exists to prevent you from accidentally moving stuff and making copies of files. So far I`ve been able to find out how to freely change those settings, but it only seems to change anything when I`m using a mouse, so I imagine the drag settings for the Pen are kept separated from it.

If you wanna try it yourself, execute regedit by writting "regedit" on your windows search bar, then open HKEY_CURRENT_USER > Control Panel > Desktop and change the DragHeight and DragWidth to whatever value you want (the default should be 4).

I`ve faved this page so i can keep updating you if I find anything else. :)
