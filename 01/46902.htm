I found the source of the problem... at least on my machine... It has nothing to do with the build... I tested every build starting with 14393 to 17134.48...

To help make things a bit more clear, a bit of history of my machine...

The OS is on a Crucial 512GB SSD that is divided into two partitions. The OS is on a 128GB (C:\) partition, and there is a 384GB (W:\) Data partition..

When I created the image I only imaged the C:\ 128GB partition...

Despite this the image recorded the existence of the 384GB W:\ partition. It did not save the data though... just the fact that it exists and needs 384GB of space.. 

Earlier this year I purchased a used SSD identical to my main one along with a equivalent 256GB of the same model specifically for stuff like this... I got tied of trashing my main drive doing experiments so to speak...

I originally tried to reload the image to the 256GB SSD since the 128GB OS partition would fit... It turned out that this was a bad move and generated all of the errors about the Active partition being too small, or a Active Data partition needing turned off...

I eventually tried the identical spare 512GB SSD... Everything immediately clicked and I had a working image on the spare... WITH The 128GB OS partition AND the EMPTY 384GB Data partition...

I knew that a image created in this fashion HAD to be loaded to a drive of the same size or larger, but forgot about the issue with size of the unused data partition..

In other words, the drive needs to be the same size or larger for the image to load REGARDLESS of the number of partitions or amount of data..

Hello Pan,

The webcam mic's will not work with Cortana. The program is set up to go through either a Kinect or headphone mic unfortunately. Maybe in the future they will have this feature available.

Step 1: Uninstall the Graphic/Display drivers
