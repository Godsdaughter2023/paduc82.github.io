Version           : 17.6568.16901.0
PackageFullName   : microsoft.windowscommunicationsapps_17.6568.16901.0_x64__8wekyb3d8bbwe
InstallLocation   : C:\Program Files\WindowsApps\microsoft.windowscommunicationsapps_17.6568.16901.0_x64__8wekyb3d8bbwe

Well, looks like everyone is on the latest version and that it did not solve the problem the way that I had hoped for.

So now it gets even more complicated.

As you can see the version number returns in the PackageFullName as well as in the program folder name for microsoft.windowscommunicationsapps which is the package name for Mail and Calendar. This one here should be the one that is used. Any older versions of the programs may also be there in C:\Program Files\WindowsApps but not necessarily.

There will be several registry entries in the registry that reference the 'PackageFullName' microsoft.windowscommunicationsapps_17.6568.16901.0_x64__8wekyb3d8bbwe

The idea is to look through the registry for similar keys that have a different version number in them and to delete those keys if there is not a Program folder in C:\Program Files\WindowsApps (%ProgramFiles%\WindowsApps) to match the version in that key name.

Unfortunately that is where it may get complicated as that folder is hidden and will not allow you entrance.

I don't know if this works as I have taken ownership of that folder once before so therefor can not test it myself:

Copy/Paste %ProgramFiles% into File Explorer's folder path box and  *windowscommunicationsapps*x* in its search box.

This should give you the existing program folders for Mail&Calendar in the search result. It does for me.

You can now search the registry (regedit) by clicking HKEY_CURRENT_USER and start searching  (F3 key or ^F) from there for any 'Key' (uncheck 'Values' and 'Data') that starts with microsoft.windowscommunicationsapps_17

Any key in the registry that does not match with any the x64 or x32 versions in the WindowsApps folder search result should be renamed or deleted.

The idea is that somehow Windows uses one of these older keys to launch the App and then finds that the belonging Mail and Calendar programs for that Key are no longer on the system.

You can read more about it here

andre da costa say:
Open Start > Settings > Update & security > Activation
Click the link ‘Go to the store to upgrade or enter a product key’
Click the $99 button to make your purchase (the price might vary by region or depending on the edition you are upgrading from or upgrading to).
Follow the on screen instructions to make your purchase.
Within 5 to 10 minutes, your system will be upgraded to Windows 10 Pro, your personal files, apps and settings preserved. 
You won’t get a product key, you get a digital license, which is attached to your Microsoft Account used to make the purchase.
