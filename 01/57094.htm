    Can anyone help to explain why and how to fix the problem. Thanks so much
    hal hostetler - mvp-outlook say:
    The problem comes from the fact that your ISP (like just about ALL ISPs) will not allow you to use his sending server unless you're directly connected to his network.  This is called "Port 25 blocking" and it is a very widely used anti-spam measure.  The solution is to either use webmail (if your ISP offers it) when away from home, or to get an account with a national service like Live Mail or GMail (which both use a sending port other than 25 and require an authenticated/SSL connection) and use the sending server from this account.  I have exactly the same issue with my old dialup ISP, I can't send because I'm connected to Cox instead of them.  I fixed this by getting a GMail account, using the dialup ISP's POP server to receive, and using GMail's secure sending server to send email with...works good, lasts long time.

    malkeleah say: When you say "guest account" do you mean the actual built-in Guest account you see in Control Panel>User Accounts? If yes, then you don't want to use this account anyway.

    Do not use the Guest account you see in the User Accounts applet in Control Panel. The Guest account is a special system account, not one meant for when you are feeling hospitable. It is disabled by default in Windows XP, Vista, Linux, Unix, and OS X for a reason.

    From TechNet:

    "The Guest account is intended for users who require temporary access to the system. However, if this account is enabled, a security risk may exist because an unauthorized user could gain anonymous access to the system through this account."

    http://technet.microsoft.com/en-us/library/bb418978(TechNet.10).aspx

    So if you enabled Guest, disable it now and create a Standard/Limited (Vista, XP Pro/XP Home) for your visitors. Call the new user account something like "Visitor". The Visitor account will be able to get to the Internet. You may also want to take a look at the suggested setup of users in Vista below:


    You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista is a multi-user operating system with built-in system accounts such as Administrator, Default, All Users, and Guest. These accounts should be left alone as they are part of the operating system structure.

    You particularly don't want only one user account with administrative privileges on Vista because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it will require third-party tools and working outside the operating system.

    The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

    steven. s say:

    Hello Sallysweet2,

     

    Welcome to the Microsoft Answers Community!

     

    There are two locations of Temporary Internet Files. One is a Junction point (that points to the other location) and the other is the true location.

     

    The Junction point is located at “%userprofile%\AppData\Local\Temporary Internet Files” (without the quotes) and is very much like a shortcut to the true location of the Temporary Internet Files which is “%userprofile%\AppData\Local\Microsoft\Windows\Temporary Internet Files”. (without the quotes)

     

    A junction point is inaccessible and only serves to redirect programs that are searching for the Temporary Internet Files in the old location. Check the location under Microsoft\Windows and you should be able to access without any problems.
