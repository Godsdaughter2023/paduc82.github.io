a. user say:

Teen Geek's solution will, of course, work but you'll then be forced to use a 3rd party program for Zip files from now on instead of the functionality within Vista (which is easier to use).  Also, WinRar is shareware whereas 7-Zip is freeware so you may want to choose the latter.  You may want to fix Vista's zip functionality instead (or in addition to):

Chances are good that some system files in Vista have become corrupt. 


Then let's check some of your system files:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a cause.

If that doesn't work, then please post any repeating error messages from the Event Viewer concerning zip files (Start / Control Panel / Administrative Tools / Event Viewer).  Here's how to use Event Viewer: http://www.bleepingcomputer.com/forums/topic40108.html.

If that doesn't work, we'll need to do a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html along with the upgrade from an earlier version of Windows section of the following: http://support.microsoft.com/kb/918884.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side. If the other procedures don't work, THIS one is almost certain to work.  You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system cane with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.  

Hopefully one of these procedures will resolve your problem.  If not, post back and we'll try something else.  Post back in any case so we'll know how it turned out.

Good luck!

soda jack say:

Thanks to Astrograd of TeamXbox

First, some clarifications in commonly confused areas:

MS’s cloud agenda is NOT comparable to either OnLive nor Gaikai. Those services have rather few servers and render the entire game server-side and stream a video feed of the output to your tv, phone, tablet, etc. This is ‘traditional cloud gaming’ so to speak. It’s viewed as an end point for modern games consoles sometime down the road (several yrs from now), but at the present the prospect of streaming HD video to tens of millions of end users in a buttery smooth fashion requires so much user bandwidth as to be intractable.

Ok, so what ARE MS’s cloud ambitions? They do want to eventually do the aforementioned ‘traditional cloud gaming’ approach, but they seem to have found a middleground to work with in the interim. They want to divide game code into two areas: latency sensitive tasks and latency insensitive tasks. The former are mainly things like graphics rendering, enemy AI while in combat, and physics computations initiated immediately by the player. The latter includes global AI simulation, global physics and animation simulations, wildlife and ecosystem simulation, game economics, weather, day/night cycles, dynamically baked environmental GI lighting, large scale fluid simulations, and crowd AI. These lists aren’t exhaustive.

How does this help game graphics? Well, it can help improve tech graphics and frame rendering in two ways that I know of:

1) By handling environmental, dynamic/static GI lighting the cloud relieves the console’s GPU from having to handle all that computation/processing. This frees it up to concentrate on the latency sensitive aspects of graphics rendering.

2) By offloading even latency insensitive tasks that would otherwise need to be done locally, you free up resources for the processors to be doing advanced rendering operations.

How will this help X1 compare more favorably to PS4? PS4 has better raw GPU specs on paper, even though all the details of OS reservations and clock speeds for these consoles aren’t known yet. Nevertheless the tech comparisons when accounting for the cloud on X1 become less than cut and dry. Much of PS4’s GPU processing advantage comes from its 6 extra CU’s, or compute units. These are going to be leveraged by devs to help PS4’s GPU do complicated calculations that need to be heavily parallelized for fast results. These will mostly be things like physics and game AI most likely. But if X1 can have many of these calculations done in the cloud and streamed to the console for local rendering it’s conceivable that these types of GPGPU tasks on PS4 will have their competitive advantage largely negated (though probably not entirely).

Ok, so what will the end results (potentially) be? What the user sees could end up being a visually more appealing scene playing out on screen in X1 games. How is this possible without overcoming the PS4’s slight processing advantages mentioned previously, even by including the cloud? The answer is that PS4’s advantages in flops, a time dependent metric, don’t mean a whole lot when the computations being done remotely in the cloud aren’t particularly time dependent. They are latency insensitive tasks in the first place. So direct comparison on CU tasks isn’t all that helpful in most cases. Likewise, X1’s eSRAM can handle the typical calculations done in the CU’s that are latency sensitive, like AA and AF. Those are done for free on the eSRAM without impacting GPU rendering.

The real reason these computations will be helpful to visuals beyond just environmental lighting is because at this point in rendering history we are at a point where the shaders next gen are realistic enough and the IQ will be good enough and the texture and poly count fidelity, etc will be good enough that throwing more power at them yields highly diminished returns. What WILL net a big visual payoff are large improvements in the fidelity of physics and animation being displayed on screen. THAT is what determines to the human brain what looks convincing and believable and realistic. That’s why racers have seemed photorealistic for yrs now (100% of their anims are physics based, no awkward humanoids to attempt to animate realistically, thus they avoid the Uncanny Valley).

So how does the cloud help X1 here? Extremely high fidelity, dynamic physics calculations can be done in the cloud that aren’t possible on these consoles. Sure, devs could try pre-rendering them into scripted animation sequences that play out on demand in-game, but they can’t be dynamic that way.

Here is what I mean. This is a conceptual example, not an actual cloud computed scene (afaik at least).

Say you are playing Quantum Break and you look to your left and see this:

"Right Click > View Image" for full version

"Right Click > View Image" for full version

Now, perhaps you could influence the boat’s path somehow, or the cars/ppl/etc on the bridge, or otherwise influence the initial conditions of that scene. The cloud could then take those initial conditions and compute that collision dynamically, sending you the resulting anim sequence to be animated using local assets and rendered in-game by your GPU.

So long as the animation isn’t in need of being immediately influenced by the player’s action (meaning player hits a button, within a few frames you get a reaction), it could be computed in the cloud and the instructions sequencing the movement of the game assets is then streamed to the console for display. You may assume this severely limits the applicability of the cloud for stuff. After all, if you can’t interact with it immediately on a frame by fame basis what’s the point? Well, you’d be surprised how many things actually look cool but aren’t needed immediately. For instance, take this bit:

"Right Click > View Image" for full version

Once the player jumps into the air (or down from a height as the case may be), there is a window of many frames before he lands. That’s plenty of time to send the initial conditions to the cloud, have it calculate the resulting impact of the jump on the ground, compute the damage model iteratively, and send the data back to the console to be animated and rendered locally. Same can be said for any physics-based sequence where there are several frames between the initiation of the action and the playback of the animated results. Cars flying across a scene in mid air, time delayed explosions, etc. We could even see partial animation hybrids where you initiate an immediate animation sequence and as it’s playing back the cloud computes the latter 70% of the sequencing and streams that to the console, resulting in local computing of the initial 30% and cloud computing instructing the last 30%.

There will surely be other applications as well. How about hair/cloth on npc’s? Typically those aren’t animated to high fidelity in-game since it’s way too intensive to have your physics engine calculating the dynamic movement of hair/cloth locally for an entire city of ppl or even a moderately large play space full of npc’s. That can be done in the cloud. I contend that these animation-related areas are where the real gains could be made in game visuals. Movement is much more important to the brain’s sense of reality than shaders that we already have largely perfected in games. Now it’s just up to devs to find ways to exploit this tech and make the most of it.

Ongoing questions:

1) Will multiplatform devs use the cloud to meaningfully improve X1 games, either relative to their PC/PS4 counterparts or even to surpass them? Or just exclusives?

2) How will devs handle the transition in-game between items handled in the cloud outside a player’s sphere of influence and those within that interactive region?

3) How much data does this stuff need to send each second? What’s the bandwidth we are looking at? How does it stack up to my ISP’s datacap?

4) How soon and how thoroughly will devs be exploiting the cloud?

There are answers to these questions that may or may not limit the efficacy of leveraging the cloud. We shall see.
