a. user say:

I have seen this before and but did not take good notes about it, can't reproduce it but have an idea...

Depending on your user name, take a look in your Temp folder here:

C:\Documents and Settings\Jose\Local Settings\Temp

If you do not have your system set to display hidden files and folders, you will not be able to navigate to the Local Settings folder using Explorer, but you can click Start, Run and in the box enter:

C:\Documents and Settings\Jose\Local Settings\Temp   (replace Jose with your login name)

If you see a folder that starts with a name of WER something or another (like WER6d4a.dir00) delete that folder and all like it.  These WER folders are Windows Error Reporting folders and should be temporary and should be removed after you make your choice to send or not send the error information. It is a temporary folder anyway, so it doesn't matter - delete it/them.

Those WER...  folders are where XP keeps that information it wants to send to Microsoft and no matter if you choose to send the information or not, those WER... folder(s) should be deleted automatically.  If you still have some WER... folders hanging around, that could be confusing XP making it think it still needs to send an error report.

After deleting any lingering folder, reboot and see how things look.

Even doing a Disk Cleanup will sometimes not get rid of these temporary folders - if you put a check mark in the "Temporary files" check box, so you can try that too, go back to the folder and it should be reasonably clean and no WER... folders should be left.  If you still see some, just delete them.

Since it is unlikely that you will ever get a meaningful or helpful response from MS about your error even if you choose to send it, you can also completely disable this error reporting mechanism.  Some folks feel sending information about your system to MS is an invasion of their privacy even though the message says that no personal information will ever be sent to Microsoft (Right...  I'm so sure...).

Completely disabling the mechanism is an option, but sometimes it might be good to know that your system had a problem so you can investigate its, so XP lets you disable the automatic sending option, but can still notify you.

Hi

There was actually a good reason for implementing this change in Windows Explorer which was first implemented in Windows Vista.

When you are using Windows Explorer and you click to open a folder, the system displays the files in alphanumeric order.

If Auto-Arrange is turned Off, then the system must enumerate the position of every file in the entire folder and this can cause lagging (slow opening), especially in folders with a large number of files.

Step 3: Open a Command Window with Administrator privilege. Create a mount point.

Step 4: Mount the WIM - assuming .ISO is mounted at D:

Step 6: When done, unmount the image and delete the folder.

rmdir C:\WIM

Step 7: Restart PC.
