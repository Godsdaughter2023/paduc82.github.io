techfreak_ say:

1.  The Skype for Windows 10 App is a universal App that runs on several platforms using the same interface.  Both Skype for Windows Desktop and the App have access to Bots.  It really is just preference, at this time.  The App currently does not have all the features of the Desktop client.  Some are probably intentional, while other features have yet to be implemented.  One big positive about the App is its ability to run in the background even while your computer is sleeping, if you have a lower power or newer computer that supports Windows Modern Standby.  This way the application consumes less resources, while still running, waiting for IMs and Calls.

2.  At this time, you can use one or the other or keep them both.  I still default to the Desktop version but I stay logged out of the App version.  They are designed not to run at the same time.  One will generally close the other out.  You can choose to uninstall whichever one you do not use.  Be aware that the App is packaged in major update builds of Windows 10.  

You can sign into your desktop account in the Skype App, you are not locked into using the account you boot Windows 10 into.  Windows 10 Apps will generally auto sign you into your account once you sign into Windows, this is why the App would be signed into that account.  To avoid this action by ANY Windows App you can simply sign into your Windows 10 locally without using a Microsoft account.  This is what I do on all my Windows 10 machines.  This gives me the option to then choose what things I want to be signed into.  You can sign out of the Skype App by simply clicking on your profile picture in the lower left pane, then selecting the option to sign out.

3.  You can either put in a request for a refund or send the credit to your other account.  Sending is limited to certain whole dollar amounts (2, 5, 10).  Due to your amount, you should have no problems transferring it all.

mark bu say:

Hello GrantRobertson, 

To better determine the cause of this issue we will need to know the type of account being used. 

If your account is an IMAP/POP3 account you may need to confirm that your account is setup to the requirements of your email provider.  Your provider should be able to provide you with the correct settings for being able to send email. 

The account settings for IMAP/POP accounts can be accessed from Mail by going to Settings -> Manage Settings -> <account> -> Change mailbox sync settings -> Advanced mailbox settings.  From the sync settings page, the outgoing (SMTP) email server settings should match what your email provider requires for sending emails.   This should be in the format <servername>:<port>:1 (For example a GMAIL account setup for IMAP would be setup as imap.gmail.com:993:1).

Additonaly check the options for whether or not your email provider requires Outgoing server requires authentication, and Use the same user name and password for sending email.   If the later is not true, uncheck the use the same user name box and then enter the outgoing server user name.

Let's see if enabling the hidden administrator will do the trick. Follow these steps:

  1. Launch Command Prompt (Admin).
  2. Copy and paste: Net user administrator /active:yes then press Enter.
  3. Once its done, restart your device and sign in as Administrator.
  4. Check your user profile and change its type to Administrator if necessary.
  5. If the issue persist, create a test user account using the hidden administrator.

If the steps above will not work, check these other ways to enable hidden administrator:

  1. Press Windows key + R and type lusrmgr.msc then press Enter.
  2. Click Users > Double-click Administrator > Uncheck Account is disabled box under General tab.
  3. Click Apply and OK; Or
  4. Type secpol.msc in the Run command box instead then press Enter.
  5. Navigate to Local Security Policy > Local Policies > Security Settings.
  6. Under Policy entries, look for Accounts: Administrator account status.
  7. Double-click and select Enabled.
  8. Click Apply and OK.
  9. Restart your device.
