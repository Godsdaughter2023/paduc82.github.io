Will you please make IME popup font size a user scalable - Microsoft Community

najevi say:

Will you please make IME popup font size a user scalable parameter?

... Currently, that font is too small and it does not scale with the desktop global scale factor.

IME = Input Method Emulator

First of all I wish to acknowledge the following related question (pertaining to Chinese IME) from a little over two years ago. 

How to change candidate font size of Chinese input (Microsoft Quick)

It has gone unresolved in two years.

That is not necessarily damning because, most probably, the user community for various language IME product is a minority among all windows users. After all they do say that patience is a virtue.

 

The purpose of this discussion is to turn what was a "How to ... " question into a rather more specific feature request with illustrated examples of what currently exists and what could exist if the right talent were applied to the task.

In that older thread a MS representative suggested altering the font scale via control panel  --> appearance and personalization ... viz.

Sachin V Rao replied on 
Microsoft

I would suggest you to try changing the font size by following the below steps.
1. Press windows + X key and select control panel.
2. Select appearance and personalization.
3. Select display and you can check with changing the fonts.

Note: This can be changed for certain things like taskbar and menu fonts only.



In today's Windows 10 (now the Creator's update, v1703, OS  Build 15063.608) that scale is tweaked via the

Settings --> Display  -->  Scale and layout --> Change the size of text, apps, and other items

and has list selectable presets of 100% (Recommended), 125%, 150% and 175%

If you have ever tweaked this parameter you may, like I was, be repulsed by the wasted screen real estate on your contemporary computer display because across the entire desktop all: windows, taskbars, icons, gadgets and fonts scale. It must be a godsend for the sight impaired but it seriously compromises an otherwise high resolution display essentially scaling across the entire desktop all: windows, taskbars, icons, gadgets and fonts scale. It must be a godsend for the sight impaired but it seriously compromises an otherwise high resolution display essentially scaling a 1920x1080 display back to the equivalent of a 1024x768 display!

Whether you hate or like what the global scale factor does to fonts etc., the fact is that the IME popup dialogs that display candidate lists and dictionary entries do not scale with this global parameter.  See the contrasting screenshot images below if you don't believe me. The first is for 100% and the second is for 125%.

100% scale factor

125% scale factor

As you can see the input font is changing in size but the IME popup font sizes have not scaled with this parameter.

And quite frankly I would not want it to! The whole idea of making such a global tweak to fix an IME font size matter is clumsy at best.

A far better solution to the problem described by 

SA
SamsonTai2015 asked on 

along with 9 other users who clicked "I have the same question"

is for the IME advanced options to allow the user to specify a scale factor for the popup windows generated by the IME itself.

Yes, there are several tabs of advanced options for (the Japanese) IME, viz.

... but none of the parameters on those tabs address popup window font size for either the candidate list or the dictionary entries. Incidentally, I am calling them popups, you may call them dialog windows or UI windows however, the images above make it manifestly clear that one window lists candidate kanji for a given selection while a subsequent window is labeled "Standard Integrated Dictionary".

Please also note that the dictionary entry popup gratuitously uses two different font sizes in a kind of Primary:Secondary pair style; or Heading1:Heading2 if I am to compare it against the font "Format" drop down list provided at this web site. That may well be appropriate for a document but not so appropriate for the IME user interface.

Be that as it may, the fundamental problem is there is no method available to scale the fonts used by either popup in the IME user interface.

What I propose/suggest/request is:

  1. A pair of parameters that set these Primary and Secondary font sizes respectively.
  2. The IME user should have access to those parameters via one of the tabs on the IME Advanced options.
  3. If a user wants both font sizes to be the same then that should be permitted.
  4. If scalability via the global parameter under Settings --> Display is important then the two parameters I am proposing should be expressed as a percentage rather than a point (1/72 of an inch) size. If expressed as a percentage then I would suggest the list: 80%, 90%, 100%(recommended), 125%, 150%.

Why allow larger fonts? Well take a look at this screenshot of a plugin developed for chrome.

The designer of that plugin clearly understands that users wish to be able to read the content of the popup. Obscuring the popup information via a small font size does no good service to the IME user.

Thank you for considering this request. I hope that you find sufficient detail in this post to pass along to the developer/maintainer of Microsoft's various IME language products.

If you are a user of the Japanese IME or any other language IME and you can relate to what I have described but have additional (or even conflicting)  requirements then pleas epiggy back on this discussion and, who knows, we may attract attetion from the right person within Microsoft to see some action taken to further develop an otherwise very useful tool.
