The differences between the top security suits are minute.

Each one might miss something when tested, but in general they provide similar protection.

As long as a person uses a single computer connected to the Internet it does not matter which one is used.


The problem of using 3rd party security suits stems from their difficulties to handle variety of Network situations.

If you spend time on Computer Networking Forums you will amazed of the number of users that have to Uninstall and clean their computers in order to have a functional LAN.

The First level of protection with Broadband Internet connection is using a Cable/DSL Router even if the Network is comprised of one computer. The Router's NAT Firewall & SPI providing very god inbound protection, the Router’s WAN also does the ISP authentication, thus keeps the connection always alive.

Use Windows Vista/Win 7 Advanced software Firewall, and install Microsoft Security Essentials as AV and Antispyware.

http://www.microsoft.com/security_essentials/

It all working smooth, on both Cable Internet and DSL, maintaining very good Internet/Network security, without adding any cost to the system.

I configured many computers om peer-to-peer networks with security configured as described above, many of them are Giga LANs, and they all working very well, and secure achieving LAN transfers up to 70MB/sec. (B=Byte).

The only time that I deviate from this routine is when it is essential to actively control any application that tries to connect on its own to the Internet, and get Visual feedback during Internet/Network activities.

helpster say:

Hello,

So, Home Basic and Home Premium do NOT have the same options as Business, Ultimate and Enterprise.

Basic and Premium have 'Remote Assistance' and the other three SKU's have full Remote Desktop. This is likely the issue that you are running into.

And the experience is different as well. You can however, be on a Home Basic/Premium computer and connect using 'Remote Desktop' to a computer running Business, Ultimate and Enterprise. Just not the other way around. You would need to use Remote Assistance.

Remote assistance allows a remote user to connect to your system via an 'Invitation'. You can look in Help and Support for Remote Assistance or Desktop FAQ's for good explanations.

Remote Desktop is designed so you can log into your computer remotely to work. This does not require an invitaion. You do need to add users to a list for permissions (Basically be an Admiistrator or in the Remote User group.) 

So there are really just a couple things that you need to have in place to connect with Remote Desktop to another computer...easily anyway.

1) You are on the SAME network. Your compter gets its IP address from the same server(DHCP) or in the same 'manually assigned' IP address range.

2) You are in the same Workgroup or Domain. If not, this gets more complicated.

3) You need to go into System Properties and Enable Remote Assistance(Basic and Premium) and/or Remote Desktop (Business, Ultimate, Enterprise)

An Alternative to remote Assistance is Windows Live Messager. It has a nice little feature that lets you do much the same thing. It does require that someone on the remote computer send an invite and accept the connection...much as remote Assistance...kind of defeating the purpose...but pretty neat little free app for fixing someone computer remotely.

I hope this helps

misterfizzle say:

Hey Colonel,

I'm not the biggest computer smarty pants but I'll do my best to answer.

The MTU of EThernet is 1500 and PPPoE is 1492. MTU values are really important in the performance of online games to minimize packet loss. Since Xbox LIVE requires a broadband connection, our MTU allows for most home networks with a certain amount of congestion and latency across the line.

This is my understanding of the situation. If you are tech savvy and my explanation makes you want to tear your hair out, I apologize and would love a post back with some smarter insight of this issue.
