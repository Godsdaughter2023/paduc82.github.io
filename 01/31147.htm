k.c.s. say: Clyde,

I am assuming that if you use Windows Explorer to look in your Downloads folder that you see
patchpackagage.msp
sitting in the root Downloads folder there and not inside some other folder within Downloads.

To confirm this from the Command prompt type:

cd C:\Users\%USERNAME%\Downloads        <push enter>

If the prompt doesn't change to C:\Users\%USERNAME%\Downloads> then try it again using the quotation marks around the path like this:

cd "C:\Users\%USERNAME%\Downloads"        <push enter>

- Your command prompt should then change to:

C:\Users\%USERNAME%\Downloads>

- Once you know you are there, then make sure that your patch file is in that folder by typing:

dir patch*   <push enter>

- Confirm that you see the file name you are looking for "patchpackagage.msp" in the text that pops up in the window.
- You said the file name was called "patchpackagage.msp" but did you mean "patchpackage.msp"  (notice the difference in spelling of the word package?)
- Regardless, just look at the name of the file that gets listed and ensure you are typing in its full name and spelling it correctly as you see it on the screen when you are trying to run the update.

- If the patch file is in fact in that directory then from the command prompt you should still be at of:

C:\Users\%USERNAME%\Downloads>

Type in:

msiexec /update patchpackagage.msp   <Push Enter>

or

msiexec /update patchpackaage.msp   <Push Enter>

And see if it works.

----

If that doesn't work then you would want to try to type this in at the command prompt:

msiexec <Push Enter>

and make sure that a Window pops-up and says "Windows Installer" and you can click OK to close the window.


- If that works, try to download your patch file again into C:\Users\%USERNAME%\Downloads>and retry.

How to restore RAID after system reinstall? - Microsoft Community

jim-l say:

How to restore RAID after system reinstall?

My system consists an SSD (where Windows is installed) and two HDDs in a RAID-0 setup (using the built-in Disk Management). If I reinstall Windows on the SSD, will the RAID-0 fail?

hairyfool say:

Rebuild the Windows on the SSD as far as you can (updates, service packs, AV etc.) with the RAID set disconnected.


Once built connect the two drives and double check in the BIOS that both drives are detected before booting windows. It should detect the Header and add the pair.


Then look to get rid of the array, the speed benefits are marginal and the risks associated with it are significant, especially if using standard drives.

jim-l say:

@HairyFool,


Thanks for the reply. Why would you say the performance benefit is marginal?

hairyfool say:

Apologies if you already know this but...


The benefit of a striped array set is that the disk controller can read and write blocks of data to two separate devices at the same time so at that core level they should be twice as fast.


However the controller will get that data a single stream so now the bottleneck is going to be controller I/O. Then for a write the controller has to divide that stream up into separate blocks so that alternate blocks can be allocated to each device although block mode transfer can be marginally faster. Similarly for a read both devices will supply data but the controller has to get them in the right order to reconstruct the data file, complete the checksums and pass the requested file to the operating system.


Even with those overheads you will get a speed increase with a hardware controller in terms of pure data transfer. As far as the operating system is concerned the array is one device, it sends a file to be written and gets a confirmation of completion. It requests a file, it is supplied, job done. But that isn't what YOU do. You open programs, you select files and so on. In practice a hardware RAID-0 array can increase the overall performance of a system in use between 5 and 10% depending on typical use, in practice any statistical increase below 3% is relatively unnoticed. The variation can arise from the type of controller, stripe size used etc.


Software RAID depends on the operating system to carry out the block segregation and file reconstruction so will always have less benefit.


Then there is the fact that normal desktop disk drives are optimised for complete file reads and writes, this is why Western Digital and now Seagate produce specific drives for NAS devices, they can cope with block mode access better. Finally all disks have a specific mean time between failure, if you have two disks in a simple striped array the expected life has been halved and that MTBF is based on normal operation, a striped array is not the intended use.


So for normal operation expect an increase of less than 5% (slightly more for boot times and large file access) versus reduced reliability and the very concern you expressed in the 1st place. A mirrored pair will be readable as a single drive whatever happens, lose one of these and it dead.
