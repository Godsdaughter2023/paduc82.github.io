lemp say:

See the suggestions here: http://knowledge.seagate.com/articles/en_US/FAQ/205071en

You can turn explorer.exe (which among other things is responsible for your desktop display) off and turn it back on again using Task Manager.  Select explorer.exe on the "Processes" tab and click "End Process," but do not close the Task Manager window.  Now click File > New task (Run ...), type explorer.exe in the "Open" box, and press Enter.  Shutting down explorer.exe will NOT terminate your running programs.

Another one of the listed suggestions that might help is turning off "indexing."  I don't think it's necessary to disable the Indexing Service entirely; if you uncheck the box to allow indexing -- in the drive's Properties|General tab -- that should be sufficient.

Logging off your user account is annoying (everything you have open is shut down), but it's a bit less time consuming than shutting down the entire system.  Of course, if the process that has a file open on the external drive is running under some other account, logging off your user account won't affect that.

Instead of using workarounds such as those suggested above, see the comments about using Process Explorer (https://docs.microsoft.com/en-us/sysinternals/downloads/process-explorer) or Resource Monitor (built into Windows) or openfiles.exe (built into Windows) here: 

https://serverfault.com/questions/1966/how-do-you-find-what-process-is-holding-a-file-open-in-windows

In order to use openfiles.exe you first need to turn on "maintain objects list."  Do this from an Elevated Command Prompt (right-click cmd.exe and "Run as administrator) by running this command:

    openfiles /Local on

There likely will be a long list of open files, but you can skim down it quickly just looking for the drive letter of your external drive.  Because the list is almost certainly going to be longer than will fit in the normal 300 lines, pipe the output of the openfiles /Query command to "more" as shown below to pause the output (just press a key when you see "more"). 

See the second screenshot below.

     openfiles /Query | more        

For more information about localfiles, type the following commands in a Command Prompt window:

    openfiles /?

    openfiles /Query /?

    openfiles /Local /?
