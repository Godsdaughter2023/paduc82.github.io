kosh vorlon say:

Hi Harry,

 

Oddly enough, you can't actually do it in msconfig unless it's already there and simply not checked as opposed to not being there at all (though you'd think they'd include that option).

 

The simplest way is to add the program (or a shortcut to the program) in the All Programs Menu Startup folder.   Here's the procedure: http://www.ehow.com/how_5098465_add-startup-programs.html.  Remember, to do this you either need an existing shortcut (perhaps on the desktop or elsewhere in the All Programs menu) or you need to find the source program file (probably somewhere in C:\Program Files depending on what you want to add) and either copy it and add it to the Startup folder or right click on it and create a desktop shortcut to copy to the startup folder.  Once you reboot, that program will start during the initial startup process and you will now be able to control it using msconfig.exe if you desire.

 

Another option is to do it using the registry editor.  If you decide to try this, first create a System Restore restore point AND backup the registry before you begin.  Here's the procedure: http://www.ehow.com/how_7304438_add-program-msconfig-regedit.html.  I recommend using this ONLY if the previous procedure doesn't work for some reason as that procedure is simpler and safer.

 

If you want a simpler way to manage and control startup programs, try the free WinPatrol http://www.winpatrol.com/download.html which monitors changes to the startup programs if you set it to run in real-time so you know when new programs try to add themselves to your startup menu and, for purposes of this question, includes an Add button that opens a link to a Windows Explorer window where you can browse for the program you want to add, click open to select it, and then add it to the startup directly.  If you prefer it not run all the time, you can simply uncheck the box that includes this program and it will only run when you choose to run it (but when started, will function as otherwise indicated or configured).  It will not appear in the All Programs startup menu (well, WinPatrol will, but not the programs it adds to startup), but it will appear in WinPatrol (which can also manage what you want to start or not start with checkboxes just like in msconfig) and of course in msconfig - but only after a reboot completes the adding process.

 

I hope this helps.  Personally, I use WinPatrol myself and allow it to run in real-time.  Since I do a lot of installations, this has helped me often prevent the addition of a startup item I don't really need and reduces how often I actually need to check what's going on there.

 

If this answers your question without needing to repost to another forum, please mark it as an answer.  If not, then of course you should repost as suggested above.

 

Good luck!

lemp say:
The short answer is that those are various mechanisms that Windows uses in connection with networking.  Most Windows home computers have all 4 of these items installed and enabled (checked).  Your instinct to disable "file and printer sharing" for security is probably overkill if your cable modem includes a router with built-in firewall (as many, if not most do) but you are not alone in thinking that way.  See ("Tips") at http://compnetworking.about.com/od/windowsfilesharing/ht/enable_disable.htm

Most of the cable modems that I know of connect to the user's computer by Ethernet cable (rather than USB).  Assuming that's the case for you, you need to keep at least some of those items checked.

The "Client for Microsoft Networks" is the basic component of Windows networking.  It allows a computer to access resources on a Microsoft network, and is installed by default. If your computer is connected to your cable modem by Ethernet, you need to leave this item checked.

"File and printer sharing for Microsoft Networks" is another component of Windows networking. It allows other computers on a network to access resources on your computer by using a Microsoft network.  If you have only one computer connected to your cable modem, you can uncheck this.*  You only need this if you want to be able to share files or printers among several computers (for example, if you had a computer in one room with a printer attached to it by USB and another computer in a different room, you could set up "file and printer sharing" to enable the second computer to use the printer.

The QoS in "QoS Packet Scheduler" means Quality of Service.  It doesn't hurt to keep it enabled, but if you only have one computer, you can disable it.  See http://www.dslreports.com/faq/3688/

Internet Protocol (TCP/IP) is the core "language" by which computers communicate over many networks, including the Internet.  You must keep this enabled.

* Also, Windows Firewall can be configured to block "file and printer sharing."  See http://ecross.mvps.org/howto/firewall.htm

Solution
  1. Click Start | Control Panel | Administrative Tools | Services.
  2. Right-click System Restore Service and click Properties.
  3. From the Startup Type drop-down, select Disabled.
  4. Click OK.
  5. Close the Services and Administrative Tools windows.

  7. Enable System Restore and click Apply.
