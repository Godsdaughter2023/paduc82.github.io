rgee21 say:

OK, I fixed this but have no idea if the fix is a good one!

Child with issues had a load of setting in the registry Polices section; child without didn't - so I deleted them from the account with problems.

Steps:

Find SID of account using the following command:

wmic useraccount get name,sid

Make sure the problem user is logged on, so their registry is loaded (but you can still do all this as your main user).

Open regedit.exe

Find this key:

HKEY_USERS\(SID)\SOFTWARE\Policies\Microsoft\Windows\Safer\CodeIdentifiers

EXPORT IT SOMEWHERE SAFE

Delete all sub-levels and any entries except:

DefaultLevel

This worked for me, it may have blown away some other security, and registry editing is risky... if it seems to have caused issues, hopefully you could re-import it from your exported copy.
