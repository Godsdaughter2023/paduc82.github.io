¡firedog say:

I've no idea what the DecryptData entries in the log mean, unless this happens after restoring a backup after a major OS or hardware change. The encryption used by WLMail is based on the user profile, so a new user profile will mean a new encryption key.

Your ever-deepening folder hierarchy results from repeated importation of folders. When you import - from whatever source - the imported messages will be placed in a folder called Imported folders under Storage folders. An imported inbox from a mail account would thus appear as
Storage folders\Imported folders\Domain (user)\Inbox

 

If you then export all folders and subsequently import again, this folder will be called

Storage folders\Imported folders\Storage folders\Imported folders\Domain (user)\Inbox

 

and so on, but not ad infinitum, because you'll eventually run into a Windows maximum file path length restriction. Moral: after importing, move folders (e.g. the Domain (user) folder in the above example) to the top level under Storage folders, then delete the Imported folders folder once it's empty. If you'd rather have the messages in their 'proper' place - e.g. under Domain (user) in the folder pane instead of under Storage folders, then select them and use right-click > Move to folder to relocate them.

bowski477 say:

Try preforming the update offline to see if that fixes the issue.

Perform an offline system update

To perform this update, you will need:

  • A Windows-based PC with an Internet connection and a USB port.
  • A USB flash drive with a minimum 4 GB of space formatted as NTFS.

Most USB flash drives come formatted as FAT32 and will have to be reformatted to NTFS. Note that formatting a USB flash drive for this procedure will erase all files on it. Back up or transfer any files on your flash drive before you format the drive. For information about how to format a USB flash drive to NTFS using a PC, see How to format a flash drive to NTFS on Windows.

  1. Plug your USB flash drive into a USB port on your computer.
  2. Open the Offline System Update file OSU1.
  3. Click Save to save the console update .zip file to your computer.
  4. Unzip the file. For all Windows OS versions, unzip the file by right-clicking on the file and selecting Extract all from the pop-up menu.
  5. Copy the $SystemUpdate file from the .zip file to your flash drive.
    Note The files should be copied to the root directory, and there shouldn't be any other files on the flash drive.
  6. Unplug the USB flash drive from your computer.

¡firedog say:

This is a symptom of a confused message database index. The message list is taken from the index file, which stores some of the headers for each message (e.g. From, To, Subject and Date) together with status flags and location data. When you select a message in the list, the program consults the index and goes looking for the message to display its body. If it can't find the message where it's supposed to be, it may display the body of a different message or none at all.

The database may recover simply by closing down the program and waiting a few minutes before re-launching it. The program does its housekeeping on shut-down, so it keeps on working after the program interface has disappeared. If you disturb this process, e.g. by turning off your computer, the database index will get confused. It will also get confused if any of its files and folders are modified (e.g. moved, renamed or deleted) in Explorer rather than within the program.

 

If a couple of close-down - wait - re-launch cycles doesn't improve the situation, you may be able to re-index an affected folder like this:

  1. Right-click on the folder containing the folder you want to re-index and select New folder. Give it a suitable name.
  2. Right-click on your desktop and select New > Folder.
  3. Click Work offline.
  4. Arrange your windows so you can see both the message list and the new folder on the desktop.
  5. Select the messages in the folder you want to re-index, then drag them out of the program window and drop them on to the new folder. Wait a few moments for the process to finish. If there are a lot of messages, drag them in batches of say 50 at a time.
  6. Open the desktop folder and check that all of the messages have the extension .eml. If there is any that has ._eml, right-click on it, select Rename and change the extension to .eml.
  7. Select the messages in the desktop folder and drag them into the folder you created at (1).
  8. Wait a few minutes for the new messages to be indexed, then check whether they are working properly.
