We have been seeing black screen issues of several types after upgrading to Windows 10. The main causes observed so far are driver issues, output to unexpected port, and possibly also issues with the Lock Screen app.

Please use these steps:

If you see the black screen:

1. While at the black screen, try using CTRL+SHIFT+ESC to start Task Manager.

2. If it becomes visible, try finding LockAppHost.exe on the Details tab and ending the task.

3. If this resolves the black screen, use corruption resolution steps (dism, then sfc), and run the all apps PowerShell command. Try this -

  • Press Windows Key + R on your keyboard.
  • Key in PowerShell and hit Enter.
  • Right click on the PowerShell icon on the taskbar and select Run as Administrator.
  • Now paste the following command in the Administrator: Windows PowerShell window and press Enter key:
  • Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}
  • Wait for PowerShell to execute and complete the command. Ignore the few errors (in red color) that may pop up.
  • When it finishes, try hitting Start and hopefully it’ll start working. If not, this solution won’t apply to you, unfortunately.

4. If this does not work, use File, Run new task in Task Manager to start an Administrator CMD window. Use this to create a new administrator user with the following commands (changing “<password>” to something of the customer’s choosing):
net user test <password> /add
net local group Administrators test /add

5. This will give you a test user to try, which will confirm if the issue is user or system-specific.

Also Check the PC or display adapter manufacturer’s site for an updated driver.

Let us know if it helps. For further assistance feel free to post your queries in Microsoft forums. We will be glad to assist you.

sbus say:

ntfs.sys is always listed as the crashing program. That's the windows filesystem - you need to check your disk drive.

Go to Start-->All Programs-->Accessories and right-click on Command Prompt. Select "Run as Administrator" from the popup context menu you get when you right-clicked on Command Prompt. You'll get a black-background DOS box popup with a prompt ending in a '>'.

Type the command chkdsk /R and hit theEnter key. The '/R' option tells chkdsk to not only fix errors it finds (as versus just testing) but also tells it to scan your disk for bad sectors. Since you are testing your boot drive, it cannot run chekdsk while it's mounted; so it will come up with a question asking if you'd like to schedule chkdsk to run on your next system restart. Tell it youdo by typing Y in response to the question and hit the Enter key.

Then restart your system. When it reboots, before it loads windows it will test and repair your hard drive. Depending on drive size this could take from 3 minutes to several hours. But when it's done it will either have fixed filesystem errors and bad sectors on the drive and your problem will go away, or it will tell you that there's something wrong with the hard drive it cannot fix, and you'll need to get your system looked at by a professional.

 

Good luck!
