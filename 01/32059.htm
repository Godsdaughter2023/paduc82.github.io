Hello,

To relocate your public folders, the best option is to run your computer as Administrator. Follow the steps below:

  1. Close all windows > Press Windows key + X > Command Prompt (Admin).
  2. Click the Start button > Press and hold Ctrl + Shift then right-click on an empty area in the taskbar > Choose Exit Explorer on the pop-up menu.
  3. Go to the admin Command Prompt window > Type explorer.exe > Enter.
  4. Now you can open up the original public folders and should have full access to the location tab, including restore, move, and find target buttons.

Note: Once you've done moving your public folders, log off and then log back in to return explorer to normal state. This is an important security step.

Public folders are owned by the System [which outranks the permissions of Admin accounts].

You can either take ownership of the folders themselves or the Registry key in which their locations can be set manually

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer\Shell Folders

I have not heard of anybody doing this to the Public folders but there does not seem to be any reason to doubt that it can be done.

tryx3 say:

mmp,

Bulldog has raised a useful point. 

1  On your new folders [it involves less work to do this before adding any subfolders] -

  • Use each folder's right-click, Properties, Security tab, Advanced to assign to only Authenticated users the right to Modify, Write as well as Read & execute, List folder contents, Read. 
  • If you check the equivalent Users entry you should not allow them more than Read & execute, List folder contents, Read.
  • Resist any temptation to add in any level of access to "Everyone" because that was really only intended to allow use of the Guest account [which is rightly disabled by default].  Set up an individual Standard user account for every user even if you regard them as Guests.

2  The distinction between Authenticated users & Users has become blurred since Windows 7 [or possibly Vista].  So my above comment is veering on the cautious side so that I do not suffer from my own ignorance.

  • An Authenticated user is one who has to sign into the computer or who has to sign in to one of the computers in your network that you have granted sharing permissions to. 
  • A User is no longer defined anywhere that I can find but I pretend there is a difference when I set up security permissions just in case there is.
  • A Guest is somebody who is not signed in.  This type of user account seems to be hardly used nowadays and I have never been quite sure what the point of it was given that Standard user accounts could be created for anybody you actually wanted on your computer.
