The free upgrade offer will not expire. Once you already took advantage of it during the free offer period, its free for the entire life of the device. That means, your computer will get future revisions of Windows 10 and Updates, automatically for free.

2. I want to move to another computer / the OS crashes, and I need to reinstall Windows 10.

The ability to transfer will depend on what type of license you upgraded from, learn more:

How to Transfer your Windows 10 License to a New Computer

3. I would need to reinstall Windows 8.1 and buy the upgrade to Windows 10.

If you are reinstalling on the same computer, the answer is no. You can reinstall Windows 10 directly on the same computer.

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

There is no need to know or get a new key, but if you have issues with Digital Licensing or the Activation Troubleshooter, you can utilize an existing Windows 7 or Windows 8/8.1 product key you already ownto reactivate if necessary. or use the reset function in Windows 10.

    imaniceguy say:

    Hey, GJB. Seems like what the students are likely running into is how the University gateway/firewall is setup for handling unsolicited, inbound connectivity for P2P communications, and possibly consoles failing to acquire a Teredo address. Party Chat and many multiplayer titles use a P2P mesh topology for network communications, which requires open inbound communications to the Xbox One console. Teredo is used for a few things here, such as detecting the public IP/port of the console if it's behind a NAT (sounds like it doesn't apply here), detecting what type of inbound/outbound connectivity is allowed to the console behind whatever network gear the console is traversing to reach the outside world, acting as an endpoint for bubble packets to hit, and securing network communications between the console and its peers.

    So first thing first, if any of the students are getting an error that their console can't obtain a Teredo address, that will block Party Chat and many multiplayer games. UDP 3544 outbound needs to be open, and the console will default to a source port of UDP 3074 when targeting the Teredo server pool. If the console is getting a Teredo address, then next thing would be to see what Teredo is detecting as far as what inbound connectivity is allowed. If you go under Settings, and Network. The Open/Moderate/Strict NAT detection will be present here. If you go to Test Multiplayer Connection on this page, it will re-detect the NAT type and run a few more tests. After the Multiplayer Test is run, and before selecting "Continue", press and release both triggers and both bumpers on the controller at the same time. A more advanced test will run, and a more descriptive Teredo NAT type will show (IP Restricted, Port Restricted, Sequential Port Symmetric, etc.) What Teredo detects here will give us a good idea on next steps.
