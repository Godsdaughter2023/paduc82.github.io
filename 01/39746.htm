The problem is wrapped up with a desire to have my Windows PCs work somewhat the same as the UNIX systems I've used at work for the past three decades.  Specifically, I want the windows focus to work as it does in X Windows:  keyboard focus follows the mouse so that you can type in a window without having to click in it and without having it pop to the top of the window stack.

There is a capability built-in to Windows 7 to almost do this:  If you select Control Panel > Ease of Use > Ease of Access Center > Make the mouse easier to use and you check "Activate a window by hovering over it with the mouse" you'll get the keyboard focus going to that window, but it also raises it to the top of the stack.

Back in the day (Windows XP) there was the neat utility called "TweakUI" that allowed you set "X-mouse" mode.  But under the hood it was just doing registry edits, so when Windows Vista/7 came along and TweakUI was no longer available, people published regedit instructions or registry files (.reg files) that did what TweakUI had done.

To make this change and have it work correctly, you have to change two registry keys:  one (UserPreferencesMask) to change the window focus to follow the mouse and a second change (ActiveWndTrkTimeout) to set how long to wait until the window focus changes.  Unfortunately, either one of two things happened:

  • Micron changed the second key from "ActiveWndTrackTimeout" to "ActiveWndTrkTimeout" somewhere along the line, or
  • People who published the edits originally got the key wrong and called it "ActiveWndTrackTimeout" when it should have been "ActiveWndTrkTimeout".

Setting "ActiveWndTrackTimeout" does nothing and that's the cause of the problem.  I had used some old .reg files when rebuilding my personal PC and that's what caused the problem.  When I set "ActiveWndTrkTimeout" to a reasonable value (150ms, hexadecimal 96) things worked great.  I verified this by setting, unsetting, deleting, and adding back that key and got the behavior I expected.

For more information, see here for a forum post that clued me in to the issue, and see here for a discussion of registry edits to implement X-mouse behavior.

What a pain!  Oh well it's fixed now and I doubt I'll forget it next time!

Next problem to solve:  On my newly-minted Windows 10 system, why does clicking on the Cortana icon in the taskbar do nothing?

The solution:

If you've implemented X-mouse (that is, if you've set bit 0 in UserPreferencesMask) you also need to create and set the ActiveWndTrkTimeout key to something other than 0.  I found a value somewhere between 150ms and 250ms to work well.

You can do this with regedit.  You can also save these setting in a file ending in ".reg" and double-click the file to make the edits.

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Control Panel\Desktop]
"ActiveWndTrkTimeout"=dword:00000096

WARNING:  editing the registry can cause all sorts of problems.  Be careful and make sure you know what you're doing!
