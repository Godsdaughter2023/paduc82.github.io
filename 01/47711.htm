airmcfly say:

Hello Barb,

thanks for your reply.

For the community, I'd like to post the solution I've found.

*** SOLVED / SOLUTION ***

Problem abstract:

- After system boot and user logon, some icons in the task bar and StartIsBack start menu disappear for 10-15 seconds and start menu becomes unresponsive. This applies at least to Windows 8.1 Professional x64 / 64-Bit.

- This might happen while StartIsBack and Acronis True Image 2014 are both installed. A hardware wise identical machine without Acronis True Image 2014 does not show these symptoms.

- The problem seems also be connected to Windows (file) Explorer (explorer.exe).

- The machine investigated is definitely virus free.

- The initially reported coincidence with attaching type cover or not seems to have been just that: pure coincidence.

- Onset of this problem was completely spontanous, while machine was in idle and restarted some time later: No updates, no software installed, no internet surfing.

Observation:

- Windows Defender (antimalware service executable, MsMpEng.exe) pushes CPU load to very high levels at startup/system boot.

- The problem does not occur, when Windows Defender realtime protection is switched off.

- The problem still occurs when StartIsBack and Acronis True Image processes are excluded from Windows Defender or their start up is delayed.

Solution:

a) Use a different anti virus protection --- or ---

b) Delay Windows Defender real time protection startup.

To delay Windows Defender startup, do the following:

- Boot in safe mode

- Run regedit.exe as administrator

- Switch on delayed start for Windows Defender service:

Edit or add the following key in Windows registry:

Path:

Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\WinDefend

Settingname: DelayedAutoStart

Type: REG_DWORD

Value: 0x00000001 (1)

(Hexadecimal and decimal 1)

By default, startup delay of services is 120 seconds.

This value can be set by editing the following registry key, 30 seconds works perfectly for me. Maybe rather 45 seconds. Please consider, this setting is global on your machine.

Edit or add the following key in Windows registry:

Path:

Computer\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control

Settingname: AutoStartDelay (be careful, slightly different from above)

Type: REG_DWORD

Value: 0x0000001e (30) (chose "Basis: decimal" to enter delay in seconds directly into the input field)

(Hexadecimal 1e and decimal 30 for 30 seconds of startup delay)

- Restart your system normally.

For sure, this solution is not the most elegant one, but it works and maybe Microsoft will fix Windows Defender some time.

Thanks and cheers,

McFly
