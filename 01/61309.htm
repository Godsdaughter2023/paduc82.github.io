Driver Verifier is included in Windows 8, 7, Windows Server 2008 R2, Windows Vista, Windows Server 2008, Windows 2000, Windows XP, and Windows Server 2003 to promote stability and reliability; you can use this tool to troubleshoot driver issues.    Can anybody tell me WHY I get all these strange markings ( marked in RED) for hyphens, quotation marks, etc., on SOME parts of an e-mail I’m trying to print out, but NOT (marked in BLUE) in others ?   


The blue ones are normal ASCII characters (0x22 the quotation mark, 0x27 is the apostrophe). The red ones are "smart", "curly" or (properly) "typographer's" quotation marks:



The curly variety are in the 'high' ASCII range, at 0x91-94, different left and right. Because the range 0x80-9F isn't used in Unicode, somewhere along the line the equivalent Unicode characters have been substituted (0x2018-2019 and 0x201C-0x201D). Your printer can't cope with these and so prints instead the characters corresponding to the bytes used in the message transmission to represent them. You'll note that the single right quotation mark is transmitted as â€™ and appears between I and m in I'm and at the end of text enclosed in quotes, like sickâ€™ instead of sick’.

The difference in the two blocks of text presumably results from their having been composed in different editors, like Word and Windows Mail.

If the text is displayed properly in Windows Mail, the problem is with your printer.
