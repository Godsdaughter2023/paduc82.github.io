It's of the PAGE_FAULT_IN_NONPAGED_AREA (50) bug check.

This indicates that invalid system memory has been referenced.

Bug check 0x50 usually occurs after the installation of faulty hardware or in the event of failure of installed hardware (usually related to defective RAM, be it main memory, L2 RAM cache, or video RAM).

Another common cause is the installation of a faulty system service.

Antivirus software can also trigger this error, as can a corrupted NTFS volume.

If you want to see your product key:

It will be in one of three places:

1. Embedded in your PC’s UEFI firmware:
Many newer PCs that come with Windows 8 or 10 use a new method. The key for the version of Windows the PC comes with is stored in the computer’s UEFI firmware or BIOS. You don’t even need to know it–assuming you’re installing the same edition of Windows the PC came with, it should automatically activate and work without you needing to enter a key.

To know your embedded license key.
1. In Start menu, type “powershell”,
Type
(Get-WmiObject -query 'select * from SoftwareLicensingService').OA3xOriginalProductKey

2. Stored in software on your PC:
When you (or your PC manufacturer) installs Windows, Windows stores its product key in the registry. You can extract this product key, and–often–enter it when reinstalling Windows on your PC.

You’ll need to grab it from your operating system before you begin reinstalling Windows or it could be deleted if you format your hard drive.

I am referring a program to show key:
https://discuss.howtogeek.com/t/see-system-prod...

2. Printed on a sticker:
Some PCs use a technology called “System Locked Pr-installation,” or SLP. If your PC uses this, the product key on your PC–the one stored in the registry, and the one key-viewer applications display–will be different from the actual key your PC needs. The actual key is on a certificate of authenticity (COA) sticker on your PC or its power supply.

bossdweeb say:

Sounds like input lag. Input lag is not generally caused directly by graphics or mouse drivers. Input lag is the delay between

your monitor receiving the signal and the image being displayed. This can affect the mouse response time if there is a large lag.

Monitors have a fairly large range of input lag from one model to the next. IPS panels tend to have higher input lag than TN panels.

- Does the model DELL monitor you are using also have HDR ? If so, this could also increase input lag.

- If you are not using a high end Nvidia gpu this could also contribute to high input lag @4K if you are doing

graphics intensive tasks. Particularly if using the Wacom simultaneously with the 4K monitor.


"How can I disable more services than msconfig has?"

I see this has been a problem with Wacom for a long time. Have you enquired at Wacom forums regarding known

issues with Win10 tablet service ?

- It may not show more Services, but I generally go directly to - services.msc - via the Run Command to get full screen,

more options and full descriptions (Which has never had anything regarding the Tablet Service on my Win10, btw ).

- Are you unable to disable Tablet Mode via Settings > System > Tablet Mode ?

You cannot use Tablet Mode when running more than one monitor, but I don't know if this would affect using this setting.

- Have you tried using the Registry key to switch it off ?

HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\ImmersiveShell
TabletMode DWORD
0 = Off
1 = On
