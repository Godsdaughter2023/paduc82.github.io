Besides replying a year and a half late :-), those suggestions from Debleena S are not excellent ideas.  They are whatever the opposite of excellent ideas are, but at least there are links to KB articles and simply more of their Microsoft engaged Support Engineer wild goose chases.

If the watchdog.sys file was missing, you would see this error:

STOP: c000021a {Fatal System Error}
The Session Manager Initialization system process terminated unexpectedly with a status of 0xc0000034 (0x00000000 0x00000000).
The system has been shut down.

If the watchdog.sys file was corrupted, you would see this error

STOP: c000021a {Fatal System Error}
The Session Manager Initialization system process terminated unexpectedly with a status of 0xc000026c (0x00000000 0x00000000).
The system has been shut down.

Therefore if either of those conditions are true, the system will not boot into any kind of mode and you will not be able to apply those "Methods".  Those are just very weak guesses.

Here is what Microsoft TechNet has to say about watchdog.sys:

In Microsoft Windows XP SP1 and later operating systems, GDI (Graphics Driver Interface) uses a watchdog timer to monitor the time that threads spend executing in the display driver. The watchdog defines a time threshold. If a thread spends more time in a display driver than the threshold specifies, the watchdog tries to recover by switching to VGA graphics mode. If the attempt fails, the watchdog generates bug check 0xEA, THREAD_STUCK_IN_DEVICE_DRIVER.

Here is what I have to say:

Something is wrong with the video subsystem (drivers).  When it gets hung for "too long" (whatever too long means), the watchdog.sys is going to try to do something about it - that is the watchdog part of it.

You could adjust the threshold time for "too long", and that may fix the symptom of the problem, but it will not fix the real problem.

What you need to do is look at the video drivers and get them up to date.
