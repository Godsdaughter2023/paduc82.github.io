Customize Partitions During Windows 7 Install

1. Insert the install disk, restart and boot from the DVD - if prompted to “press any key...” press a key on your keyboard.

2. Click Install Now

3. At the setup screen, click Custom (Advanced)

4. Click Drive Options

5. Select the partition(s) you want to format/delete

6. Click Format or Delete - this will remove EVERYTHING on that partition

7. Create a new partition (if needed) and select a partition to install Windows on.

8. Continue with install.


More on Installing and reinstalling Windows 7:

http://windows.microsoft.com/en-US/windows7/Installing-and-reinstalling-Windows-7

trekdozer say: When you format the hard drive during the install process a boot partition will be created automatically - this is the preferred way to make a boot partition.

Having your page file (swap) on a different partition does not speed up performance. Putting the page file on separate physical hard drive can improve performance, but that is done after the installation is finished.



Move Page File

1. Open control panel and choose System And Security.

2. Click System, then click Advanced System Settings.

3. Click the Advanced tab.

4. In the performance section click on Settings.

5. Click Advanced and choose Change in the Virtual Memory section.

6. Uncheck Automatically Manage Page File size

7. Remove the page file on the C: drive and select a different drive for the page file.

wolfy762 say:

Essentially I needed to use the “metered connection” setting as a screen to prevent auto install of drivers/bloatware/malware (or any other unwanted update) and once the updates where trapped use wushowhide.diagcab to hide them.

1. Clean install of windows 10 with USB 3.0 16 drive/or restored from disk image

2. Did not set up wireless at first boot options

3. Changed >System Properties>Hardware>Device Installation Settings: No, let me Choose what to do/Never install driver software from Windows Update but checked the get device info

4. Changed >Control Panel\Network and Internet\Network and Sharing Center\Advanced sharing settings>Private profile>turn on network discovery UNCHECKED Turn on automatic setup of network connected devices.

5. Changed: Settings >Update & security>Windows Update>Advanced Options>Choose how updates are installed to: Notify to schedule restart

6. Allowed Wireless to connect

7. Changed: Settings >WI-FI>Advanced Options Set as Metered connection to ON

8. Joined the homegroup

9. Downloaded wushowhide.diagcab

10. Selected: Settings >Update & security>Windows Update> Check for Updates

11. Warnings for drivers and bloat come up and complains about the Metered connection

12. Ran wushowhide.diagcab >Next>Hide Updates: selected the updates/software/bloatware/malware I didn’t want.

13. Logout/Log back in or restart (Windows Update would not give me a retry option until I did)

14. Selected: Settings >Update & security>Windows Update> Retry

15. Selected: Settings >Update & security>Windows Update> Details (to verify the whole list)

-Update gave me a list minus the **** I hid

16. Selected: Settings >Update & security>Windows Update> Download (Good updates/security definitions etc)

 

Specifically for the Printers on the Network that are shared, I was having a problem with two sets of the same printer wanting to install. My solution was use the WSD (ONLY-had IPV4,IPV6,WSD as options) LAN search in the printer install software. Apparently the WSD is an automatic port created when the private network profile has “Turn on automatic setup of network connected devices” checked. I was able to stop this WSD printer version from showing up by unchecking it or using a public non sharing profile.

Don’t let anyone tell you that 3 fixes anything by itself, it seems to only have an effect on the printers and devices area but makes the devices visible for windows update to go install whatever it thinks is good.

Hope this helps someone
