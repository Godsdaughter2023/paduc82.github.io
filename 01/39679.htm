so-frustrated say:

SOLVED!

Naturally, this was only possible once I posted my question, just how these things work. :-)  I will describe the solution and hopefully it will be helpful to others.

Apparently, the trick here was to use 64-BIT drivers for the printer!  In my example above, I used a 32-bit driver (and Windows 10 did not complain), but that just was not good enough.

I went back to the Microsoft Update Catalog and downloaded an Amd64 driver, right-clicked and installed.  For the HP LaserJet 6L driver, got a bunch of "Cannot find the file specified" errors.  Elsewhere on the internets, I found out you can look in C:\Windows\INF\setupapi.dev.log to see what was falling over.  Turned out that I needed to create an 'Amd64' directory right underneath where PRNHP001.INF was located, and then copy all files from PRNHP001.INF's directory to it.  (Most likely, not all files needed copying in this manner; also, one could probably have hacked PRNHP001.INF itself to have it look in its own directory for files).

Now, I right-clicked 'Install' and it took a few seconds, then Windows told me the installation was successful (it said the same thing for the 32-bit driver...), except now I got the familiar print queue window entitled 'HP LaserJet 6L on AKI'!  Tried printing a test page and voila!

In a Windows 10 or a Windows 8-based portable computer, follow this procedure:

1. Connect an external monitor.

2. Change the screen settings to only the second screen (using the Windows logo key + P keyboard shortcut).

3. Change the screen settings to Duplicate (using the Windows logo key + P new keyboard shortcut).

4. Close the lid of the portable computer.

5. Unplug the external monitor.

6. Open the equipment cover.

7. Plug in the external monitor again.

In this scenario, the screen settings can revert to Only the second screen instead of remaining in duplicate mode.

Alternative solution:

To avoid this problem, use the Windows logo key + keyboard shortcut P to change the screen settings again to duplicate mode.

a. user say:

Hi,

Thank you for Contacting Microsoft Community.

In order to help you better, could you please confirm a couple of things like:

  1. Are you getting any specific error message?
  2. How did you take backup of files into D drive?

Method 1: To Take Ownership of All Files with the same File Extension in a Folder or Drive using TAKEOWN Command

In the elevated command prompt, type the command you want below, press Enter, when finished, close the elevated command prompt.

NOTE: Substitute full path of folder or drive with the full path of the folder or drive letter, and substitute file extension with the actual file extention that you want to take ownership of within quotes.

(To grant currently logged on user ownership of)

 takeown /F "full path of folder or drive\*.file extension"

(To grant administrators group ownership of)

 takeown /F "full path of folder or drive\*.file extension" /A

For example:

takeown /F "C:\Windows\*.txt" /A

Method 2: To Take Ownership of a Folder or Drive using TAKEOWN Command

NOTE: This command will take ownership of the folder or drive, and all files and subfolders in the folder or drive.

In the elevated command prompt, type the command you want below, press Enter, and when finished, close the elevated command prompt.

NOTE: Substitute full path of folder or drive with the full path of the folder or drive letter that you want to take ownership of within quotes.

(To grant currently logged on user ownership of)

 takeown /F "full path of folder or drive" /R /D Y

(To grant administrators group ownership of)

 takeown /F "full path of folder or drive" /A /R /D Y

For example:

takeown /F "F:" /A /R /D Y

takeown /F "F:\Folder" /A /R /D Y

For your reference follow the below link and check if it helps:

http://support.microsoft.com/en-us/kb/2623670/en-us
