Thank you for posting in Microsoft Community Forums, as the print spooler is not working as expected, please be assured that we will do our best to assist you with this issue.

 

Try the following methods and check:

 

Method 1: Let’s run the printer troubleshooter and check if that helps.

a) Move the mouse to the bottom left corner of the screen and right-click on the‘Start’ icon and select ‘Control Panel’.

b) In the search box, type ‘Troubleshooter’, and then click ‘Troubleshooting’.

c) Click on ‘Hardware and Sound’ and select ‘Printer’and click on ‘next’ to run the troubleshooter.

 

Method 2: Run a system file checker scan and check if that helps. Refer the following steps to run SFC scan.

a) Move the mouse pointer to bottom left of the screen where you see preview of start menu ,Right-click to bring up a list of Windows settings.

b) Click on ‘Command Prompt (Admin) to bring up elevated command prompt. Note: Click ‘Yes’ if it prompts for permission.

c) In the command prompt type in the following command: ‘sfc /scannow’.

 

Method 3:

Check if the printer spooler service is started. To do so:

a. Press ‘Win key + q’ on the keyboard.

b. Type Services in the search box.

c. Select view local services which appears at the left side of the screen.

d. Search for Print spooler service and double click on it.

e. Check if that service is started and the StartUp type is set to Automatic.

 

Method 4:

a. Open Windows Explorer which is next to the Internet explorer icon on the taksbar.

b. Click on the view tab, and click on options.

c. Click on the view tab in the new window which is open.

d. Scroll down and verify that “Show hidden files and folders” is checked and uncheck the option  “Hide protected operating system files”

e. Also verify “Hide extensions for known file types” is unchecked.

f. Now navigate to c:\windows\system32\spool\printers

g. Normally this folder should be empty but there is a problem when files with “spl” and “shd” extensions are there.

h. Delete the files and restart the computer. This may resolve the issue.

 

Hope this information helps. If you have further questions feel free to reply and we would be glad to assist.

If after running your chkdsk /r you still have the problem, you can generate a new desktop.htt file.

Never actually seen it, but have some notes that don't work right so I did some experimenting.

Each user has a Desktop.htt file in their profile and it looks like for some reason you are logging into XP with the Administrator account which most folks will say is not a good idea, but...

The Desktop.htt is just a text file that contains information about what your desktop should look like so you need to replace yours.

First adjust your Windows Explorer so you can see hidden files and folders:

Using Windows Explorer, navigate to the C:\ folder, Click Tools, Folder Options, View tab.
      
Make a note of these three settings (so you can change them back) and then change them so you can see everything:
      
         Enable:  Show hidden files and folders
         Disable:  Hide extensions for known file types
         Disable:  Hide protected operating system files  (respond in the affirmative to the warning)
      
Now you should see all kinds of good stuff.  This may be annoying later, so you might undo those adjustments when you are finished troubleshooting.

Navigate to the folder containing the Desktop.htt folder for the afflicted user.

In my case:

C:\Documents and Settings\Jose\Application Data\Microsoft\Internet Explorer

In your case, that looks like it should be:

C:\Documents and Settings\Administrator\Application Data\Microsoft\Internet Explorer

Locate and rename the potentially afflicted Desktop.htt file to something you can remember (like Desktop.xxx) so now the file is "missing".  Then minimize the open Explorer window.

To generate a new Desktop.htt file, you have to change something on your desktop, so get back to your desktop, right click, Properties, Desktop make a note of what your current background image is and the Position and Color options because you are going to change it.

Change the background image to something else and apply (this will generate a new Desktop.htt file for you), then change the background image back to what you want it to be using the settings you noted in the beginning.

Maximize the Explorer window for your user name so you can see inside that folder again and press F5 to refresh and you should have a new Desktop.htt file.

Now that you have a new Desktop.htt file, close all windows, reboot and see how things look.

If all is well, you can delete the suspicious Desktop.xxx file and undo the Explorer adjustments made earlier.

Here's how the Print screen button key on a keyboard should work.

 

The Print Screen (or PrtScrn or variant thereof) key does not immediately print the screen as the name might suggest – it’s a bit misleading. Instead it copies the current screen image to your clipboard. You can then paste this image into a suitable photo-editing (or similar) type program so that you can edit it or print it out from there.

 

An alternative way to the above is pressing Alt + Print Screen on your keyboard - this does exactly the same as the above, but the image copied to the clipboard is only that of the currently active window.
