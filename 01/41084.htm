brachole_800 say:

Simultenous Network Connections

I have a Lenovo Laptop, running XP Pro. At home, I use a CAT5 cable to connect to my router and Internet. I also have a wireless card, which I connect to another device (wirelessly), but doesn't acess the internet. This other device is only for playing music through my Stereo, and doesn't need to access the internet.

 

So the idea here is that I have 2 connections:

 

1 to access the internet, using the CAT5 cable

1 to connect to my stereo, using my wireless card, which doesn't acess the internet.

 

However, when both connections are connected, for some reasons, the Wireless takes over and I cannot access the internet anymore. In order to access the interner, my wireless connection cannot be on.

 

Isn't there a way to have both connection simultaneously, and still have acess to the internet. Can I set an order where my NIC comes first, and not my Wireless card?

 

Thank you!

lemp say:
Yes, there are "metrics" that you can set in the network routing table that are used to determine which route (i.e., adapter) to use to get to a particular destination, but this isn't applicable in your case (because the routes don't end up in the same place -- one goes to your router and the other to your music device).

  • Open Network Connections (Start > Run > ncpa.cpl > OK)
  • Right-click on the icon for your wireless adapter and select Properties
  • On the General tab, in "This connection uses the following items," click Internet Protocol (TCP/IP), and then click Properties.
  • On the General tab, click "Use the following IP address."  Enter the IP address and subnet mask for your music device and leave the box for "Default Gateway" blank.

wunders say: The first issue that can get you into this trouble is if both the wired connection and the wireless connection are on the same subnet.  Bring up a command prompt window (Start -> Run -> "cmd") and enter the command:
    ipconfig  /all
and examine the IP addresses of both connections.  If the first three numbers of the two adapter IP addresses are the same, then you are likely on the same subnet for both cards and this can cause problems that you are having.   If this is your case, reconfigure one of the networks to be on a different subnet.

If this isn't your problem, then you need to examine the routing table.  Print out the routing table by entering the following command in a command prompt window:
    route  print

"TCP/IP Routing Basics for Windows NT"  (pretty much applies to XP as well)
  < http://support.microsoft.com/kb/140859 >

Reading from bottom-up, the first line that starts "0.0.0.0" is likely the gateway that will be used for your internet connection.  You can change the card/interface that is used for internet connection by adjusting the "Metric".  The card with the lower Metric number will have the higher priority in being the gateway interface.  Instructions for changing the metric are here:

"An explanation of the Automatic Metric feature for Internet Protocol routes"
  < http://support.microsoft.com/kb/299540 >
