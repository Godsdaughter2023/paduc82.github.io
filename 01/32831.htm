Windows RT is not the same as Windows 7 or 8 because it is running on a completely different processor architecture. Even if you could get around the fact that Windows RT will only allow programs to be loaded from the Store any program intended to work on the x86 processor system used in standard PCs will not work on the NVidia Tegra ARM processor used in the Surface RT tablet (and in many others).

 

Hypothetically a program that used only Windows high level API calls recognised by both Windows RT and Windows 8 can run on both transparently.

hairyfool say:

I know exactly what a portable app is. It requires no installed supplementary drivers or libraries to run on a system, if any are required they are either integrated into the executable and extracted during runtime or are included in the same location as the executable and are referenced from that location.

 

No portable application does not depend in some way on the running operating system in order to display current status, error messages or to generate logs for its operation. If it cannot accept input during its operation or generate some form of output it has no function. It is the fundamental description of a program to have one or the other or both.

 

When you double click on an executable it is Windows (if that is the operating system) that has given you the ability to open its location, see the executable in order to select it and issue the command to run it. It is Windows that then loads that program into memory in order for it to run. If required it is Windows that displays the dialogue box to provide any input and display any results or error messages generated by the program. If the result is a log file it is Windows that provides access to the disk drive to create the file and if the result is a error status in is Windows that stores the error status so it can be interpreted by another program. If the output is to modify the status of the computer in some way it is Windows that gives access to that device or component. The Hardware Abstraction Layer allows access to the hardware itself if needed by a program and even that is part of Windows.

 

So I DO know what a portable program is, more importantly I also know what Windows is.
