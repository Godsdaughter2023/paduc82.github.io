Permanent Solution for Error 0x80070643 for KB3122947 - Microsoft Community

sumit (volunteer moderator) say:

Permanent Solution for Error 0x80070643 for KB3122947

As stated above, numerous users are encountering this error while Installing this update.More details on what it fixes:

http://support.microsoft.com/KB3122947

This update restores the user's setting preferences if the settings were not retained through the upgrade process.

After a close Inspection, I found that this update gets downloaded, but due to some problem It does not get Installed.

Reason:

It gets failed when you have removed Windows.old folder after upgrading to build 10586,because there is no path from which the update can extract the retained settings.

Simply saying, that is more of a Microsoft problem which would be solved by Monday, as the Tech support says.

SOLUTION

We can use DISM command to Install the downloaded file:

Open an Elevated command Prompt,Press Windows key+X and select Command Prompt(Admin))

  • For 64 bit systems, you need to Enter the following command:

dism /online /add-package /packagepath:C:\Windows\SoftwareDistribution\Download\c4a1b8896ce9fbfea96c1ee6890d52a5\windows10.0-kb3122947-x64.cab

  • For 32 bit systems,you need to Enter the following command:

dism /online /add-package /packagepath:C:\Windows\SoftwareDistribution\Download\b0a5da1b24245bc4237166e09bae92da\windows10.0-kb3122947-x86.cab

If you encounter any problems, reply back and I would be happy to assist.

Do you know what speed you should have? That is 0.009 or so megabits which is what ISP use. Next you want to figure out if it is just certain sites to confirm it is a internet speed issue and not something else.

You can use speedtest.net or do an internet search for IIS smooth streaming 1080p experience. The latter is a Microsoft demo of its adaptive streaming so it displays a more realistic speedtest. You will see the speed in the top right and in a chart in the lower left.

You can also change the network tab form kb to

mb by right clicking it.

After you get a speed reading let me know what it is. I would then turn off the router and modem and let them sit for 30 seconds then reconnect them. Then open a command prompt by hitting windows key + X and click command prompt in the power user menu. In the window type ipconfig/release hit enter then ipconfig/renew hit enter

Then ping your router IP which is the same IP you use to access your router from a browser, usually 192.168.1.1 but not always. If you don't know it you can type ipconfig/all and look for the wireless adapter or Ethernet depending on which connection you are using. If your router up is 192.168.1.1 then the three pings you will run are as follows.

ping 192.168.1.1 -n 50

ping 8.8.8.8 -n 20

ping Microsoft.com -n 20

Typically you run these is three different command prompt windows at the same time but I am guessing we will see where the issue lies. If you want to run them in 3 command windows use ping 8.8.8.8 t- same as above but you are using the t flag instead of the n followed by the number of times that you want to ping. The t flag will run until you hit ctrl c or close the window.

If the router ping is is high, more than a few ms it is a issue with your router to computer. If that ping is good but the 8.8.8.8 ping is high your internet service is slow. If the domain name ping is high only then the DNS has an issue. Let me know the results.
