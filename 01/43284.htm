techhead287 say:

I actually think that this file is not meant to be uninstalled.

It resides in the C:\WINDOWS\system32 folder and has been there ever since I bought the computer. Just to be sure it wasn't malware, I installed a fresh copy of Windows 7 in a VM and found the same file.

What's really strange about it is that it doesn't show up in MSCONFIG, REGEDIT, or any other places startup files would reside. All the other threads I've read have someone saying, "i found this process called ctfmon.exe i tried to disable it with msconfig but it just comes back".

I have a suspicion that the AVGUI.exe program, which is AVG's tray icon and stuff, is starting CTFMON.exe. When I've just logged on, Task Manager shows AVGUI.exe showing quite a bit of CPU usage, and when CTFMON.exe executes, AVGUI.exe's usage goes to 0.

It seems that yesterday's update replaced the AVGUI.exe program.

  1. AVG will start its update process.
  2. AVGMFAPX.exe, a process with the description, 'AVG Installer Application', will execute under the 'SYSTEM' user account.
  3. AVG will begin downloading updates.
  4. Once the updates are downloaded, they will be analysed to make sure they're not fake or are newer than the current definitions.
  5. The updates will be prepared. 'AVG Installer Application' will start using some CPU here.
  6. The updates will be installed. This process can take up to 5 minutes in an update like this.
  7. AVG will finish the update process. Multiple AVGMFAPX.exe processes will constantly be executing and dying.
  8. AVGUI.exe will be terminated and replaced with a process called AVGUIRUX.exe, which has the description, 'AVG UI Starter'. 
  9. AVGMFAPX.exe and a few other AVG processes will be using alot of CPU and performing alot of hard drive activity during this time, which can last up to 5 minutes. You may see your AVG desktop icon flash as AVGMFAPX.ex

    So there you go, my complete description of a major AVG update. I may be wrong, this just seems to be what happens.

    I'm guessing the only fix here would be to submit a bug report or something to AVG so they can release a fix update.

    Anyone with updates please post back.
