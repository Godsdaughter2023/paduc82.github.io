lemp say:

If the Shell Hardware Detection service was disabled but now automatic, restart the computer and see if it behaves any better.  If not, go back and check Shell Hardware Detection service to see if something is turning it off.

That may do it.  If not ...

Follow these directions to make sure that AutoPlay is turned on (I think you've already done this).  Use Option One here: https://www.sevenforums.com/tutorials/174838-autoplay-turn-off.html

Follow these directions to make sure that AutoPlay is enabled for "nonvolumedevices" (like your camera).  Use Option One here:  https://www.sevenforums.com/tutorials/183209-autoplay-enable-disable-non-volume-devices.html

The file you downloaded -- which should be named Enable_AutoPlay_for_Non-volume_Devices.reg -- will make modifications to your Windows Registry.  This way, you don't have to manually edit the Registry and take the chance of making a mistake. 

As described in steps 5 and 6, you double-click the downloaded file to run it (you can also right-click it and select "Merge").  You may have to respond to several prompts in order to complete the operation.  This is because editing the Windows Registry is potentially dangerous, and Windows wants to make sure that you really want to do it. 

If you want to know exactly what the file will do, you can right-click and select Edit, which will open the file in Notepad to reveal this:

  contents of Enable_AutoPlay_for_Non-volume_Devices.reg

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Policies\Microsoft\Windows\Explorer]
"NoAutoplayfornonVolume"=-

[HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\Explorer]
"NoAutoplayfornonVolume"=-

The hyphen (-) after the equals sign (=) means that the value of the named variable (if it exists in the key) will be deleted.  If you look at the corresponding file for disabling this sort of AutoPlay behavior, you would see that setting the variable to 1 is how "NoAutoplay" is turned on (this is a double negative:  turning on "NoAutoplay" turns off AutoPlay).

  contents of Disable_AutoPlay_for_Non-volume_Devices.reg

Windows Registry Editor Version 5.00

[HKEY_CURRENT_USER\Software\Policies\Microsoft\Windows\Explorer]
"NoAutoplayfornonVolume"=dword:00000001

[HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\Explorer]
"NoAutoplayfornonVolume"=-

As a general rule, when a registry key appears both in HKEY_CURRENT_USER and HKEY_LOCAL_MACHINE, the setting in HKEY_LOCAL_MACHINE will take precedence.  That's why the "disable" file only turns on NoAutoPlay in HKEY_CURRENT_USER so that different behavior for another user on the machine can be set if desired.
