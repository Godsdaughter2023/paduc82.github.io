¡firedog say:

I have entered a test CSV contacts file using Notepad.

There are several caveats to be observed when constructing a CSV file for import into Windows Live Mail - and most probably in general: 

  1. There should be no space either side of the separating comma, neither in the header line nor in the data lines. The space following the comma in your examples will be prefixed to the email address, rendering it invalid. 
     
  2. Windows Live Contacts have no data field called Full Name. This is in Outlook Express and (IIRC) in Office Outlook a concatenation of the First Name, (Middle Name?) and Last Name fields. The equivalent field in Windows Live Contacts is called simply Name, but it's not a good idea to use it in import/export, because it has special significance*. Import the fields separately, e.g. as First Name,Last Name
     
  3. While it's not necessary in most fields, enclosing every field in quotation marks will avoid many errors caused by trying to import data containing a comma or quotation marks (e.g. a street address like "The Droppings", Shitterton). Any double quotation marks should be changed to single ones ('The Droppings') before attempting the import. Your first lines would thus read
    "First Name","Last Name","Email Address"
    "John","Doe","johndoe.com" 

* Name is the value shown and transmitted in the To: line of a sent message. It is constructed by Windows Live Mail from contact data fields and consists of the first match found when scanning data fields in the following order:

  1. Nickname
  2. First name + Last name (in the order specified at Sort by in the Contacts window)
  3. First name or Last name
  4. Company
  5. E-mail address
