linney say:

By way of the Accessed Denied message and explanation, I received this comment from Microsoft back in the days of Vista when I was asking a similar type of question.,  

"The junctions are there to only provide appcompat for legacy apps and aren’t meant for a user to traverse through. The junctions have been explicitly set to block read through them by setting Everyone Deny Read. The main reason here is because these are just links to the actual location, so you dont want backup tools and other tools operating on your data twice, once from the original path and once via the junctions. There are scenarios where some of these junctions actually form a loop to support the appcompat for the old namespace in comparison to the new and in those cases allowing read through them is disastrous, for e.g. setup was broken for a week when the file system wasnt honorign the deny read.

Also as far as a user goes, you will never see these as they are system hidden, and you will need to take explicit action to see them by default."

a) Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

net stop wuauserv

net stop cryptSvc

net stop bits

net stop msiserver

 

c) Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

ren C:\Windows\SoftwareDistribution SoftwareDistribution.old

ren C:\Windows\System32\catroot2 Catroot2.old

net start wuauserv

net start cryptSvc

net start bits

net start msiserver

 

e) Type Exit in the Command Prompt to close it.

 

Now you may try running the Windows Updates and check if the above steps resolve the issue.

Surface bluetooth incorrectly pairing with speaker - Microsoft Community

reginald e geurts say:

Surface bluetooth incorrectly pairing with speaker

I have a Satechi Bluetooth speaker but it does not disconnect and re-connect properly. It worked the very first time I paired it but after turning it off and turning it back on it would no longer play sound. Removing the device and adding it back did not fix the problem.

 

If you go into the Control Panel -> Hardware and Sound -> Sound -> Manage audio devices to bring up the "Sound" window after turning the device off and back on or changing the mode to audio cable then back to Bluetooth the Headphones BT SPEAKER Stereo is not set as default as it should be in order for it to start emitting sound and you can not manually "Set Default" either.

 

The Surface only puts back the Headset BT SPEAKER Hands-Free which then gets set to the default sound device but this does not work to emit sound from the speaker. I tried disabling the "headset" option which hides it but that does not work to fix the problem.

 

The best work around I have found is in the Control Panel -> Hardware and Sound -> Devices and Printers -> Device Manager window. Under Sound, video and game controllers you will find "BT SPEAKER Stereo". Right click it and choose "disable". Choose "Yes" when prompted to confirm "Do you really want to disable it?". The screen will refresh and it should now display as disabled. Right click the "BT SPEAKER Stereo" device again and this time choose "enable".

 

I have tried this with both the Surface RT and the Surface Pro and both have this issue and the work around works on both.

 

In addition I noticed that in the Device Manager under Audio input and outputs the "Headphones (BT SPEAKER Stereo)" is removed when I turn off the speaker. When I turn the speaker back on it is not added back. When I disable/enable the "BT SPEAKER Stereo" under the Sound, video and game controllers the needed entry of "Headphones (BT SPEAKER Stereo)" is added back under the Audio input and outputs.

 

The two inputs for "Headset (BT SPEAKER Hands-Free)" in the Audio inputs and outputs are because there is one for playback and one for recording. If you've disabled them in the "Sound" window as described above they will not be added back in the Device Manager under Audio input and outputs. You can right click any device in the "Sound" window and choose to "Show Disabled Devices" to unhide them and then Enable them if you need/want to.

 

If anyone has found a better work around or fix please let me know.
