A few(2-3) months back, even before installation of Windows 7, my Vista system keeps getting in this freeze mode.  And whenever I try to perform a file backup (instead of system backup), sometime during the backup the system prints the blue screen and crashes.  So I figured it might be a disk issue and bought a new hard-drive to replace C & D.

1.  How can I re-attach partition F & G from disk 2 without losing any information?
2.  Or can I install the new drive as Disk 2 and move the boot parition from Disk 0 to Disk 2 ?

mouneshwar r say: Hi BThoughts,

Thank you for visiting Microsoft Answers!

How can I re-attach partition F & G from disk 2 without losing any information?

Since you’ve Windows 7 installed on drive F, and you’ve data on partition G, it’s not possible to extend the partition to make it one complete partition without losing the data.

If you’ve enough space on the drive F, I would suggest you to copy the data from drive G to F. Once you’ve copied the data, you can format the drive G and then you’ll be able to extend the partition and make it one.

Now, perform the following steps after you copy the data from drive G to drive F.

   1. Click on Start; in the start search type diskmgmt.msc and hit Enter.
   2. Right click on drive G and format it.
   3. Once the formatting is done, right click on drive F and click on Extend.

That’s all.

Can I install the new drive as Disk 2 and move the boot partition from Disk 0 to Disk 2?

It is not possible to move the boot partition from Disk 0 to Disk 2. But yes, you can make the Disk 2 as a primary master and Disk 0 as a primary slave and then you can try to boot.

Hope this information is helpful.

Regards,
Mouneshwar R – Microsoft Support

If Windows 10 fails to boot normally three times over, the fourth time it enters by default in an Automatic Repair mode. Using this mode, you can boot into Safe Mode. To trigger the Automatic Repair mode, you must interrupt the normal boot process three consecutive times: use the Reset or the Power button on your Windows 10 PC to stop it during boot, before it finishes loading Windows 10. If you use the Power button, you might have to keep it pressed for at least 4 seconds to force the power off. When Windows 10 enters the Automatic Repair mode, the first thing you see is a screen that tells you that the operating system is "Preparing Automatic Repair."

Try repairing boot records, using these steps:

Hi JohnSchwab,

1. What type of backup are you trying to perform?

2. Did you make any changes to the computer before this issue occurred?

 

I would suggest you to try the steps mentioned below.

 

This issue occurs if the backup catalog files are corrupted.

 

Method 1: Try to run the backup by putting the computer in a clean boot state

A clean boot helps in eliminating software conflicts. Visit the link below to know how to perform a clean boot.

http://support.microsoft.com/kb/929135

Note: Once the troubleshooting is over, reset the computer to start as usual.

 

Method 2: Delete the backup catalog files and then try to re-run the backup

Let us stop the Windows Backup service which would allow us to delete backup catalog files.

a.    Click on Start; in the start search type cmd.

b.    Right click on cmd.exe and choose to “Run as administrator”.

c.    Type the following command and press Enter.

           net stop sdrsvc

d.    Minimize the command prompt.

e.    Now enable hidden files and folders. Click: how to enable hidden files and folders in Windows.
http://windows.microsoft.com/en-US/windows7/Show-hidden-files
f.     Click on Computer. Go to the following location.

           C:\System Volume Information\Windows Backup\

g.    Delete all the catalog files and folders.

h.    If you receive access denied error, take the ownership of System Volume Information folder. Click “Take ownership of a file or folder” http://windows.microsoft.com/en-US/windows-vista/Troubleshoot-access-denied-when-opening-files-or-folders
i.     Once you delete the files and folders successfully, restore the command prompt and restart the Windows Backup service by typing the following command.

Net start sdrsvc.

 

Now try to re-run the backup and then check if you’re able to run the backup successfully.
