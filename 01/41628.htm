a. user say:

Hi Abby,

Thank you for posting in Community.

We understand your concern and we will provide the information required to help you resolve the issue.

As per the information that you require, I would suggest that you follow the steps below and check if it helps

1. Tab on Win+R and in the dialog box type regedit . It'll open Registry Editor. Now expand "HKEY_CLASSES_ROOT" key.

2. Now look for a sub-key having same name as the file type which you want to add in "New" menu. For example, if you want to add XML file type to "New" menu, look for .xml key under HKEY_CLASSES_ROOT key.

3. Now right-click on it and select "New -> Key" option. It'll create a new key, give it name ShellNew

4. In right-side pane, right-click and select "New -> String Value". Give it name NullFile and press Enter.

NOTE: Sometimes this method might not work for a few file types in newer Windows versions. In such condition, you'll also need to create a dummy template file after following the above steps as mentioned in following steps:

5. Create a new file of same file type which you want to add in "New" menu (in our example we wanted to add XML file type to "New" menu, so we'll create a new .xml file), give it any name and copy it to C:\Windows\ShellNew folder.

6. Now in Registry Editor, create another String with the name FileName in same key where you created NullFile string in Step 4 and set FileName string value to the full path of the file pasted in "C:\Windows\ShellNew" folder.

Since in our example, the full path of Template.xml file is C:\Windows\ShellNew\Template.xml, We have set the same as value of FileName string in Registry. Now the new file type will appear in New.

Hope this helps

Hi Carlos

There have been many name changes to the folder heirarchy since Windows XP.

An example is  C:\Documents and Settings is now C:\Users.
Application Data is now C:\Users\user name\AppData\Roaming
Local Settings is now C:\Users\user name\AppData\Local

If you have selected the Show Hidden Files option, you will still see folders with the older names. These are called Junctions and the reason they are there is to redirect programs to the new locations in Windows 7.

For instance if you use a program to move a folder to Windows 7 that had the path C:\Documents and Settings\user name, the Junction will automatically redirect the folder to C:\Users\user name.

These Junctions can be identified by the Shortcut Arrow overlay on the folder icon. Junctions cannot be used like normal folders and they have heavily restricted access. You should not attempt to change these because this can have devastating effects on the file system.

If you can still find the parent folders for these files, such as financial, musical, political, legal, and personal, I would recommend adding these folders to the Windows Exlorer Favorites list which is located at the top of the Navigation pane.

Locate one of the folders, drag and drop it onto the Favorites in the Navigation pane. This will create a shortcut to that that you can use to go to the location, and open that folder.

Let us know if you have more questions.
