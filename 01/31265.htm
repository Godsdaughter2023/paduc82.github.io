john john __ say: As indicated by the Disk Management tool the Windows files are on drive D (the Boot partition) and the system files required to boot the Windows installation are on drive C (the System partition).  The results of the Set System and Dir commands confirms the status of the drives.  One sure-fire way to find out which installation is in use is to simply try to delete the Windows folder on both drives, you will be able to delete the folder on drive C, Windows will treat it as a data folder, but you will not be able to delete the Windows folder on drive D, deleting this folder would halt the Windows session and Windows will not allow this to happen.  (Of course it goes without saying that you wouldn't try this on a properly working multi-boot system!)

You can delete everything on the C: drive except the following:
c:\boot.ini
C:\NTDETECT.COM
C:\ntldr

The following files may be removed (with the noted proviso):
C:\hiberfil.sys (only needed if you hibernate the Windows installation)
C:\pagefile.sys (may be in use by the operating system)

Windows will just recreate these folders if you delete them:
RECYCLER
System Volume Information

Your hard disk has a primary partition [C:] and an extended partition [D:].  Extended partitions are non-bootable, the computer can only boot a primary active partition.  Having a basic understanding of how hard disks are arranged and how a computer boots helps when setting up multi-boot systems.

The first sector on the disk (sector0) holds the Master Boot Record (MBR).  The MBR contains an Initial Program Loader (IPL) and the partition table which holds information about the size and location of the partitions.  A hard disk can have up to 4 partitions but there can only be one active partition at any given time.  Only primary partitions can be active, extended partitions cannot be marked active.  The active flag can be moved from one primary partition to another so any primary partition on the disk can be marked as active.  The partition boot sector is created when the partition is formatted.

Sector0 is the single most important sector on the disk, if this sector becomes physically damaged the disk will be unusable.  If the sector becomes logically corrupted it will not be able to boot, to gain use of the disk again the MBR will have to be restored from a backup or it will have to be rewritten.  If the partition table becomes corrupted all the data on the disk is at risk of loss, the partition table will have to be restored from a backup or disk recovery utilities may be able to recover the partitions.

In a nutshell this is what happens when you boot your computer:

1-  The Power supply starts and does a self test, if the power is good it sends a power good signal to the processor, if the processor doesn't receive a power good signal it halts and shuts down the computer.

2-  The processor loads the BIOS, the BIOS performs hardware/POST tests.

3-  The BIOS looks for a bootable device, the startup sequence (boot order) is stored in the BIOS, the BIOS goes down the list until it finds a bootable device.

4-  When instructed to boot a hard disk the BIOS loads sector0 of the disk and passes the boot process to the Master Boot Record.  The Initial Program Loader is executed and it looks in the partition table for the active partition.  

5-  The IPL loads the boot sector of the active partition and passes the boot process to the boot sector.

6-  The boot sector contains a small bit of code which locates, loads and then passes the boot process to the bootstrap loader.  The bootstrap loader loads the operating system, NTLDR is the bootstrap loader for Windows XP.

You need to understand this before you attempt to install Linux on the same disk as your Windows installation:

The Windows boot loader (ntldr) cannot boot Linux, you will have to use a Linux boot loader or another third party boot loader to load your different operating systems.  Linux cannot be installed on an NTFS file system, when you install Linux on your primary partition it will be reformatted to a format that Linux can understand.  Ntldr has a built-in mini file system driver that allows it to read FAT12, FAT16, FAT32 & NTFS, it cannot read Linux file systems so you will have to move the ntldr, boot.ini & NTDETECT.COM files to another partition, this is usually not a problem when Windows is on a primary partition but yours is on a logical drive.  Depending on the Linux boot loader you select the MBR & IPL may be rewritten, if you later have problems or decide that you do not wish to keep Linux you may have to rewrite or restore the MBR and you will have to reformat the primary partition to get rid of the Linux file system and return it to NTFS.  You will then have to place the boot.ini, ntldr & NTDETECT.COM files back in the root of the partition.  It would be safer if your disk was made up of 2 primary partitions rather than having an extended partition as it is now, if things get botched you can then just rewrite the MBR and move the active flag to the Windows partition.

While it is possible to install Linux on a disk holding a Windows installation it is not recommended, it is best to have the Linux installation on a hard disk of it's own.  So, before you decide to do this just make sure that you understand the risks, if you use this computer for important work just make sure that you have a complete backup ot the whole disk that can be restored if need be, I recommend Terabyte's Image for Windows, there are many other such disk imaging utilities out there.  You should also have a bootable USB stick with tools on board which will permit you to restore or rewrite the MBR and to reformat the primary partition to NTFS.  

I've seen this before, I have seen people doing it successfully and I have seen others coming back crying that Linux redid the whole disk at installation time and that they lost their Windows installation and all their data.  Really, it isn't Linux that redid the whole disk, they just didn't understand what was going on and they unknowingly allowed it to happen, not to mention that they neglected the most important step of backing up their precious files before attempting to install Linux on the same disk as their Windows or data disk.

I strongly encourage people to experiment but at the same time I want to make sure that you understand the risks which includes the possibility of loosing everything on the disk if things get botched!  Installing an operating system is a major disk operation, mistakes and glitches can and do at times happen, just be aware of the risks and be prepared if things go wrong!

BCCode: 0A     0x0000000A

Cause

This bug check is issued if paged memory (or invalid memory) is accessed when the IRQL is too high.

The error that generates this bug check usually occurs after the installation of a faulty device driver, system service, or BIOS.

If you encounter bug check 0xA while upgrading to a later version of Windows, this error might be caused by a device driver, a system service, a virus scanner, or a backup tool that is incompatible with the new version.

Resolving the Problem

If a kernel debugger is available, obtain a stack trace.

To resolve an error caused by a faulty device driver, system service, or BIOS

  1. Restart your computer.
  2. Press F8 at the character-based menu that displays the operating system choices.
  3. Select the Last Known Good Configuration option from the Windows Advanced Options menu. This option is most effective when only one driver or service is added at a time.

 

To resolve an error caused by an incompatible device driver, system service, virus scanner, or backup tool

  1. Check the System Log in Event Viewer for error messages that might identify the device or driver that caused the error.
  2. Try disabling memory caching of the BIOS.
  3. Run the hardware diagnostics supplied by the system manufacturer, especially the memory scanner. For details on these procedures, see the owner's manual for your computer.
  4. Make sure the latest Service Pack is installed.
  5. If your system has small computer system interface (SCSI) adapters, contact the adapter manufacturer to obtain updated Windows drivers. Try disabling sync negotiation in the SCSI BIOS, checking the cabling and the SCSI IDs of each device, and confirming proper termination.
  6. For integrated device electronics (IDE) devices, define the onboard IDE port as Primary only. Also, check each IDE device for the proper master/subordinate/stand-alone setting. Try removing all IDE devices except for hard disks.

 If the message appears during an installation of Windows, make sure that the computer and all installed peripherals are listed in the Microsoft Windows Marketplace Tested Products List.

Comments

Before upgrading to a new version of Windows, remove all third-party device drivers and system services, and disable any virus scanners. Contact the software manufacturers to obtain updates of these third-party tools.
