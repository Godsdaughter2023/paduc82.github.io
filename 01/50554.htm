Windows 10 doesn't shut down properly..

vanessa yar say:

Hi AndreiF7,

It seems that your computer is in hybrid shutdown mode that's why it appears not fully turned off. To sort out this behavior, I suggest disabling the hybrid shutdown in the Power Options. Please follow these steps:

  1. In the Cortana search, type in powercfg.cpl, and hit Enter.
  2. Once the Power Options window is up, click Choose what the power buttons do on the left pane.
  3. To activate Shutdown settings, click Change settings that are currently unavailable.
  4. Uncheck Turn on fast startup, then click Save changes.
  5. Shutdown your computer, then turn it back on.
  6. Once up, re-access the Power options and re-enable the Turn on fast startup, then save the changes.

Please observe the lights on your computer and see if the lights remain on when you shut down.

Looking forward to your reply.

Regards.

vanessa yar say:

Thanks for the update, Andrei. It seems that your computer just enters into hibernation when the fast startup is enabled. In that case, please turn off the fast startup again in order for your computer to fully shut down.

Feel free to let us know if you need further help.

Is this a new observation/information:

The search options still only shows an isolated checkbox with no text where the search KB choice should be.


In your GPE, set the option from Not Configured to Enabled, exit GPE, reboot, then change it back to Not Configured, then take a look.

Usually with H&S problems I also ask this question:

Have you now or have you ever used any third party applications on your system where the product description includes such descriptive phrases as:

cleaner, mechanic, detective, doctor, cure, optimize, inspector, clean up, tune up, speed up, fix up, compact, compress, faster, power, boost, booster, eraser, muncher. one click fix, new, intuitive, mistakes, stops crashes, stops errors, improves, saves you money, risk free, guaranteed or any third party program that has an icon resembling a gear, cog, tool belt, wrench, spanner, screwdriver or swooping arrows?

If you want to reinstall your H&S you can do that too:

Using Windows Explorer, browse to and locate c:\windows\inf\pchealth.inf, right click the file and choose to Install.

If your XP is installed in the WINDOWS folder on your C drive, you should be able to find the file in the following folder:

c:\windows\inf

If you do not see any .inf files in the c:\windows\inf folder, click Tools, Folder Options, View and tick the option:

Show hidden files and folders

Then UNtick the option:

Hide extensions for known file types

(You can undo those adjustments later if it is your desire)

Now you should be able to see all the .inf files in the folder (you can undo this Explorer adjustment later if desired).

When you reinstall your Help and Support Center, you may be prompted to enter your XP installation CD as it looks for the installation files.  Most people do not have an XP installation CD, so you need to tell the installation where to find the files it needs by looking in other places on your hard disk.

If Help and Support was working before, the files are on your system somewhere and you just need to point the installation process to the files and their locations.

You may want to look below at the list of files that the reinstallation is going to need and locate the files on your system before you start the installation so you will know where to tell the installation to look when it needs a file.

When the installation starts, if it asks for a file and you don't have an XP installation CD, change the "Copy file from" dialogue (by clicking the Browse and Retry buttons) to point to the appropriate folder for each requested file so the installation can find it on your hard disk.

Usually, the files are in these folders.

HelpCtr.exe  (c:\windows\system32\dllcache) or c:\windows\pchealth\helpctr\binaries
racpldlg.dll (c:\windows\system32) or c:\windows\system32\dllcache)
UploadM.exe  (c:\windows\system32\dllcache) or c:\windows\pchealth\UploadLB\Binaries
pchdt_w3.cab (c:\windows\pchealth\helpctr\binaries)
HCAppRes.dll (c:\windows\pchealth\helpctr\binaries)
atrace.dll   (c:\windows\system32)

If those files are not in the same place on your system, you need to just stop the installation part for a minute and use the built in Windows Search function to find the file, then when you find where it is, you can resume the installation and point it to the folder that contains the file it needs.

Perform a Windows Search on your system for the file you need and then point the installation to the correct folder on your system so the installation can find the file.  If you just can't find the file, let us know what file you need and you can download it from some nice person's SkyDrive so you will have it (just leave the installation hanging/waiting until you get the file(s) you need).

After reinstalling the Help and Support Center, reboot and make sure the Help and Support Service is running and set to Manual or Automatic Start.

To do that, click Start, Run and in the box enter:

services.msc

Click OK to open the Services applet.

Locate the Help and Support Service, highlight it, right click it and choose 'Start' (or click the right pointing arrow on the top toolbar).

When the Help and Support Service is running, you will see the word 'Started' in the Services Status column.

BCCode: E1   0x000000E1

Bug Check 0xE1: WORKER_THREAD_RETURNED_AT_BAD_IRQL

The WORKER_THREAD_RETURNED_AT_BAD_IRQL bug check has a value of 0x000000E1. This indicates that a worker thread completed and returned with IRQL >= DISPATCH_LEVEL.

Important Info If You Have Received a STOP Code

If you received a blue screen error, or stop code, the computer has shut down abruptly to protect itself from data loss. A hardware device, its driver, or related software might have caused this error. If your copy of Windows came with your computer, call the manufacturer of your computer. If you purchased Windows separately from your computer, Microsoft provides support. To find contact info for Microsoft or your computer manufacturer, Contact Support.

If you have experience with computers and want to try to recover from this error, follow the steps provided in the Microsoft article Resolving STOP (Blue Screen) Errors in Windows.

These actions might prevent an error like this from happening again:

  1. Download and install updates and device drivers for your computer from Windows Update.
  2. Scan your computer for computer viruses.
  3. Check your hard disk for errors.
WORKER_THREAD_RETURNED_AT_BAD_IRQL Parameters

The following parameters are displayed on the blue screen.

Parameter Description

1

Address of the worker routine

2

IRQL that the worker thread returned at

3

Work item parameter

4

Work item address
Cause

A worker thread completed and returned with IRQL >= DISPATCH_LEVEL.

Resolution

To find the driver that caused the error, use the ln (List Nearest Symbols) debugger command:
