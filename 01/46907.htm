mdoc7 say:

I have solved this problem, albeit only a workaround.  Reduce your planned backups and run them in batches.

The reason it fails is because shadow copies are being created and, if your backup is a very long list, it times out before it's finished with the shadow copying.  Another factor is whether you're using an external drive connected to USB 2.0 instead of USB 3.0.  USB 3.0 is 10 times faster than USB 2.0.  This can contribute to the error, but it depends on how large your backup is. 

So, do the backups in batches.  My best estimate of the maximum size for a single batch is 85GB (under USB 2.0).  Disclaimer: I have not searched for the biggest size before the timeout limit.

This assumes, of course, that you've applied all the other solutions outlined above.  But I believe the main problem is that timeout limit.

Some malware will recognize efforts made to find and remove it and prevent them from running and they may also change your system policies or file associations so things you might use to find and remove it will not work (like System Restore, Task Manager or regedit).  The malware wants to fool you into thinking that you need to do drastic things like a System Recovery, Repair Install or totally reinstall your Windows XP from scratch when none of those things are likely to really be necessary.

rkill.exe (from BleepingComputer) can be used to stop malicious software applications that it knows about from running so that you can follow up with malware scanners that might remove the potential infection.  It also looks for common policy changes and attempts to undo them, tries to undo common file association adjustments that malware might have changed or undo any changes that might prevent you from getting out to WWW sites to download other removal tools. 

There are also lots of legitimate programs that are not digitally signed.  If you want to see a bunch of them, download and run Autoruns...  Everything you see in pink is an unsigned file but that doesn't mean you need to get rid of it or worry about it.  Autoruns is safe, runs on demand and installs nothing extra on your system that might slow it down.
