¡firedog say:

I'm afraid you've got it a~~e about face. If a message you send can't be delivered, the server responsible for delivering it will often return a bounce message (properly known as a Delivery Status Notification or DSN) to the server that originally transmitted the message. This will then appear in your mailbox in the same way as any other message sent to you. Account settings may dictate that it's consigned to junk, so you should also check there for DSNs. Any DSN about a message that couldn't be delivered will usually contain a Non-Delivery Report (NDR) explaining why.

Windows Live Mail doesn't discriminate between different types of message, so if you're not receiving DSNs about particular messages, it's because none was ever sent (which often happens when messages are trapped in spam filters) or because there's some setting somewhere that's preventing their delivery to your mailbox. Windows Live Mail has no part in the process: it will happily receive and display any message sent to your account.

¡firedog say:

I wonder what a non technophobe is ;)

If you receive bounce messages into your online Gmail account, then you've established that the remote server (the one that the recipient would have downloaded the email from) is indeed sending bounce messages - not all of them do, although they're supposed to. This means that the bounce messages that you're not receiving in Windows Live Mail are being stopped somewhere, or else they're hiding from you. These are some of the possibilities:

  1. Windows Live Mail is putting them in the account's junk email folder. I take it you've looked there.
  2. Your own mail server (the one you download messages from) is putting bounce messages in the junk folder. If this is a POP3 account, you would usually have to visit the account in webmail to see the contents of the junk folder, because POP3 only downloads messages from the inbox. Have you looked there?
  3. Your own mail server's spam filter is blocking messages from the remote server, because that server is on a blacklist. This isn't easy to troubleshoot.

It would be helpful to know who the Email Service Provider (the owner of your mail server) is. This is usually apparent from the part of the email address after the '@' symbol. Don't post your email address - the forum software will hide it. Just the bit after '@'.

My suggestion would be: 

     
  1. Add your Gmail account to Windows Live Mail. Press Ctrl-Shift-T to start the Add your email accounts wizard, then complete the wizard accepting the defaults. Windows Live Mail should apply the right settings to set up the account to use IMAP.
     
  2. Once the account is set up, the folders have been downloaded and then filled with messages, send a test message from this new account to one of your duff accounts that you know will bounce it back. The bounce message should eventually arrive in your Gmail account in Windows Live Mail; again, be sure to check the junk folder.
     
  3. The Delivery Status Notification will contain a Non-Delivery Report something like this:

Delivery to the following recipients failed.

       <email address>

Reporting-MTA: dns;mailserver1.domain.com

Received-From-MTA: dns;mailserver2

Arrival-Date: Tue, 19 Sep 2016 14:21:03 -0700

Final-Recipient: rfc822;  <email address>

Action: failed

Status: 5.5.0

Diagnostic-Code: smtp;550  <email address> unknown user account

     

    It may not be straightforward to find the NDR; please say if you need help.
     

    When you've found it, please copy it and paste it into your reply. Before clicking Submit, please replace any '@' symbols in the NDR with something else, like '£'. Otherwise, again, we won't be able to see them.
