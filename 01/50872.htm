Hey AlexRules6969, sorry to hear about the issue. From my experience it looks like 1 of 2 things, either some corrupted files have built up on the console, which can happen from a number of things, or the console is trying to install part of a system update, but can't download it.

To try an isolate the problem I would recommend clearing the persistent storage on the console as well as clearing the system cache.

* To clear the persistent storage, go to Settings, select Disc & Blu-ray, select persistent storage and click the option for 'Clear persistent storage'.

* To clear the system cache, press and hold the power-button on the console for 6-10 seconds, or until the console turns off. - Do this immediately after clearing the persistent storage.

After you do both of these, disconnect the power-supply unit from the console and the socket. Wait for the light on the power-supply to go from orange to off, leave it for 11 seconds, then re-connect the power-supply into the socket, and then into the console, then power on the console.

If the above steps don't work, I would recommend doing a full factory reset on the console. To do this go to Settings, select System and click the option for Restore factory defaults. The screen will go black after a few seconds, and will display a green loading bar, and when completed will bring you through the original set-up. Go through this and your issue will hopefully resolve your issue.

Apparently, at least one reason why the Authentication tab is missing is that neither the Wireless Zero Configuration (WZC) service nor the Wired Autoconfig Service (DOT3SVC) is running.  See http://support.microsoft.com/kb/950725

The default for WZC is to start automatically and the default for DOT3SVC is manual.  You shouldn't need either of these services to connect by Ethernet cable to a home router, but why not visit http://www.blackviper.com/service-configurations/black-vipers-windows-xp-x86-32-bit-service-pack-3-service-configurations/ and make sure that all of the services have the startup type (disabled, manual, automatic) listed in the "Default" column for your version of XP.

Limited Connectivity or Wi-Fi dropping issues can be affected by drivers or if the connection is poor. In Windows 10, Wi-Fi drops the connection if the internet is in Public connections for safety reasons.

Method 1: Change Wi-Fi network to private.

  1. On the right side of the taskbar, select the Wi-Fi network icon.

  2. Under the name of the Wi-Fi network that you’re connected to, select Properties.

  3. Under Network profile, select Public or Private.

For more information about settings, refer the Microsoft article on Make a Wi-Fi network public or private in Windows 10.

Method 2: Change the Power Management.

  1. Right click on the Wi-Fi network icon on the right side of the taskbar.
  2. Click Open Network and Sharing Center. Click on Change adapter settings. 
  3. Right-click your Wi-Fi adapter and click Properties. 
  4. Click Configure. 
  5. Go to the Power Management tab. Uncheck the box ‘Allow the computer to turn off this device to save power’. Click OK. 

Method 3: Change "Scan Valid Interval”.

  1. Tap on the Windows & R key at the same time on the keyboard.

  2. Type: devmgmt.msc

  3. Expand Network adapters. Right click on Wi-Fi card.

  4. Click on advance tab and change the Scan Valid Interval value from 60 to 120.

Once you have done that, simply do a clean install of Windows 7 64 bit disk. In Windows 7 64 bit disk, launch Windows Easy Transfer, follow the on screen wizard, point to where you stored the the WET file (external hard disk), and it will restore your personal files and settings.
