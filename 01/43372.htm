net user administrator /active:yes

mellison say:

Clarification of File History options

I've been using file history in Windows 10 to keep my files backed up to an external hard drive, and I'd like to know precisely what the options for "keep my backups until..." will do.

shenan stanley say:

Ah, it seems you are reading too much into it.

The way a backup would work is that if there is a file located on your active system that is selected because of your backup settings, there will always be a copy of that file in the backups.  Only if you delete that file from your active system or move it to an unprotected by backups area is there a possibility of it falling out of the backup scheme thanks to the feature you mention (how long backups are kept/if you change the default of forever.)

Now, if you do not move the file (or remove the file) out of the protected area, the worse that will happen due to your setting of "keep my backups" would be the loss of history (you might have gone in and edited the file once and the backups caught the changes, thus you could roll back to the pre-edited version as well as the current.  If you edited daily and your file history/backups were frequent enough to catch those changes say... Daily ... Then it is possible that at the limit of your "keep my backups" you would see history reversion possibilities drop off, but you would always have a copy of the file that was being actively protected - you just may allow all the historical changes to leave your backups and have only the current exact copy left.

BTW - you should, really, also (as in addition to this backup method) backup your files to media you disconnect other than when making said backups.  While the backups being discussed here help prevent you from making a typo and ruining your day, if something more serious occurs - the connected backups could be just as destroyed as the original data.  Physical separation of the backup data and the original is wise.
