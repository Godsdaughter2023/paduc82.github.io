... a sign in box appears ...

Is this the 'sign in box' you mean?

If so, this has nothing to do with email. It simply determines which calendar and contacts list to load.

  • If you don't sign in, the contacts list and calendar you see are the default   ones.
  • If you do sign in,  the contacts list and calendar you see are the ones that belong to the Microsoft Account you're signed in with.
  •  It doesn't matter what the email address is. If it was registered as a Microsoft Account, it will still be a Microsoft Account even though you no longer use the address for email.

If you're happy only using the default contacts list and calendar, you can Stop signing in. To do this, sign in when you're asked to, then press Ctrl-Shift-O for Options. Click the button on the Connection tab - you won't see the sign-in box the next time you launch the program.

This problem generally occurs on computers with an exceptional number of attached drives, whether they are networked, or hard-wired. Computers with multi-format card readers can have this issue. USB drives don't always find an available drive name that is compatible with their naming conventions. For example, if the USB drive is named "G:" and your computer has a networked drive "G:" or additional hard or optical drives named "G:" your USB drive will not show up.

 

This problem can be resolved by remapping a drive to a different name; this can either be a temporary name change, or permanent.


You need to navigate to the path mentioned below to change the name –


Computer> Right Click (USB drive)> Properties> (Enter the name that you desire).

 

Also try to access the USB drives on a different computer. Check if that helps.

 

 

Method –

 


You need to follow the steps mentioned below only if you are using Remote Desktop Connection–


a) Plug the thumb drive into your computer's USB slot.
b) Close any applications using the thumb drive, including Windows Explorer windows displaying its contents. Redirection requires exclusive access to the device; it won't be available on the local computer while it's being redirected.
c) Click "Start," "All Programs," "Accessories" and "Remote Desktop Connection."
d) Click the arrow to the left of "Options" at the bottom of the Remote Desktop Connection window.
e) Click the "Local Resources" tab. Click the "More" button under "Local Devices and Resources."
f) Expand the "Drives" section by clicking the plus sign to its left.
g) Click the checkbox to the left of your thumb drive's drive. If you want additional thumb drives you plug in to also be available in the remote desktop session, click the "Drives that I Plug in Later" checkbox too.
h) Click "OK" and then the “General” tab.
i) Type the address of the computer you want to connect to in the "Computer" box and the user name on the remote computer you want to log in within the "User Name" box.
j) Click "Connect."
k) Type the password for the user account when prompted and press "Enter."
l) Double-Click the thumb drive icon in the Computer window and check.

aaron_sum say:

Hi,

Thank you for posting to Microsoft community.

The Microsoft Reserved Partition (MSR) is only used on the UEFI systems using GPT partition structure. It is created to reserve a portion of the disk space for possible subsequent usage by the Windows Operating System. This partition does not contain any meaningful data or any user data. However, it is assigned and ideally used for remapping damaged sectors. 

Beginning in Windows 10, the ‘Minimum’ size of the MSR is 16 MB. However, the recommended size for an MSR partition is 128 MB. You could check this link for more information on legacy systems and see if it is helpful.
