armin zingler say: I know this is not a new thread. However, I had the same problem as the OP, but I found the solution on my machine, and this might help others, too.

I've spent about three days to find a solution. I won't list all the unsuccessful attempts now. The only thing left would have been a Win7 reinstall which really should be avoided. Being an experienced software developer myself, I then started to do native debugging to find the cause of the problem. After some days (and nights) this turned out to be the solution:

I uninstalled the driver of my TV card. Why TV card? It's a DVB-S card ("Technisat SkyStar", DVB via satellite) that is supplied with two alternative drivers:
  • Using one driver, it is listed as a multimedia device (BDA).
  • Using the other driver, it is listed as a network card because there is the option for internet via satellite. This is the reason why a TV card can be involved in this networking issue at all. Even though currently installed as a BDA device, not as a network card, something must have gone wrong in the past weeks, probably when switching forth and back between the two drivers.

So, after uninstalling the driver, including checking the option "[X] completely remove driver files", and after removing some remaining registry entries and resetting IP settings using "netsh", I am now able to enter the dialogs again, and finally they keep their settings after confirming and reopening them. Everything still works even after resinstalling the (required) BDA driver for the TV card.

1) Update your BIOS and drivers from the PC or motherboard maker's Support Downloads web page, using the full model number, Serial Number or Dell Service Tag on sticker. Compare the latest drivers available for download with the ones presently installed in Device Manager reached by right clicking the Start Menu. Updating drivers from the Device Manager or Windows Update is rarely enough.

2) Turn off Fast Startup at Settings > System > Power & Sleep > Additional Power Settings > Choose what Power buttons do > Change Settings Currently Unavailable > uncheck the box for Fast Startup, Save Settings.

3) Test the RAM overnight to stress it with the best bootable test memtest86, following these steps which test both the sticks and the slots: http://www.sevenforums.com/tutorials/105647-ram...
