maroselle cag say:

When your GPU is processing intensive graphics operations but encounters an error during the process, the operating system tells the driver not to access the hardware or memory and gives it a time for the currently running threads to complete. This process is known as Timeout Detection and Recovery (TDR) which has a default value of 2 seconds. If those threads are not completed within the timeout (which is default to 2 seconds), that is when the system will give you a bug check of 0x116 VIDEO_TDR_FAILURE which you usually encounter.

Based on the details that you have provided, it seems that you have already attempted all necessary troubleshooting steps to eliminate the possibility of a failing software. You can try debugging the TDR by changing some of its registry key values such as increasing its default timeout threshold (TdrDelay) and check if it makes the difference. If you feel like doing so, you can check this article regarding TDR Registry Keys. (Important:  This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, refer to the following Microsoft Knowledge Base article: How to back up and restore the registry in Windows)

It's a good thing that you have attempted to test out different driver versions for your GPU. This will lead us to conclude that it is more likely an issue with the hardware component.

When using the Microsoft Basic Display Adapter driver, it is unable to call or utilize the full functionality of the graphics component of your APU. It is possible that one of those hardware components of the GPU, which only the AMD driver can call, is faulty and causes the BSOD. We would suggest that you check some hardware issues that impact the ability of the video card to operate properly, including:

  • Over-clocked components, such as the motherboard
  • Incorrect component compatibility and settings (especially memory configuration and timings)
  • Insufficient system cooling
  • Insufficient system power

If you have confirmed that all of the above-given possibilities were already eliminated, then it would be safe to conclude that the issue is caused by some defective parts (motherboard, APU, etc.)

Let us know if you have other questions.
