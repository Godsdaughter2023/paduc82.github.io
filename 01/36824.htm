spencer simon say:



I have had the same problem for weeks, causing me to go insane enough to go into my registry and tinker with anything Windows Update-related. (NOT RECOMMENDED UNLESS YOU UNDERSTAND WHAT YOU ARE DOING)! I google'd for hours and realized how many people had the same Error code. 

Link to the post that helped me: 

"Windows Diagnostic Policy Fix" I apologize, as I do not know the rules pertaining linking to other sites.

To cut it short, the reason that Windows Update was bringing up the Error code was due to the fact that "...Got a message that Network Diagnostics cannot run because the Diagnostics Policy Service is not running. Opened Service Control Manager to try and start that service, and got a message that Windows could not start the service on Local Computer, followed by Error 5: Access is denied."  

I traced the problem to "Diagnostic Policy Service" not "Running" (open "run" using WIN+R, type in "services.msc", and check if the "Diagnostic Policy Service" is "Running".) Right click DIAGNOSTIC POLICY SERVICE, and in the properties, make sure that its start-up type is Automatic, Automatic (Delayed), or Manual. ANYTHING EXCEPT "Disabled".

To fix this problem:

  1. Right click the Windows Icon in the lower left-hand corner, select "Command Prompt (Admin)." 
  2. At the prompt, type: "net localgroup Administrators /add networkservice" then press enter. It should follow saying that the command had been successful, or something being a success. 
  3. Then type:"net localgroup Administrators /add localservice" Press enter, then type: "exit", and press enter. The same line about the command being a success. 
  4. Restart your computer (even though idk why I chose not to).
  5. Open the Services task (using the WIN+R or "run" command, and enter "services.msc".
  6. Make sure that the task states that it is "Running", and if it does not say so, go into the properties of the Diagnostic Policy Service and click on "Start". If you do not receive any error message(s), and the Service says "Running" now, then you are one step closer.
  7. Go to Windows Update and click "Retry". Hopefully, you will no longer receive Error 0x80070015, but instead it will start downloading all of those updates, and soon enough, they will all go from Status: Pending install, to Installing:, to prompting you to restart your computer.

Hopefully this will helped out everyone with the "Error 0x80070015" problem. If it did not, I will try my best to back track the previous steps incase they did anything useful haha. Good luck, and let me know what happens!

robert aldwinckle on forums say:

got a message saying that these updates are NOT RELEVANT to my computer.

Microsoft do a horrible job of keeping us informed about superseded updates.  They just depend on all the automated processes working correctly and assuming that then nobody needs to know how it should work.   So my guess is that you probably already have something, such as the latest rollup, that supersedes the ones that you are trying to install.

Let's take a look at that possibility using the first update you asked about since it essentially only involves one module.

http://support.microsoft.com/kb/3002885

In the File Information section we are told that this module is being updated:  Win32k.sys

Let's use Powershell  to find out the state of that module on your system.

PS C:\windows> ls -Re -Fi Win32k.sys | sort LastWriteTime | ft Length, FullName -AutoSize

I'm giving the full command line so you know where to start.  Use an elevated Powershell (aka Administrator: Windows Powershell) to minimize warnings and error messages.   Note that even though I am specifying AutoSize to try to show the full path name (as a representation of the module's version), you need to make the window big enough to make it all capturable.

FWIW I am seeing that the currently installed version is the same as the version that supposedly came with that update.  However, since then (e.g. assuming that the Sort stage is doing something useful) it looks as if some other earlier versions have been compressed.  So maybe the only thing that you have wrong is that you can't get done whatever is supposed to happen to get your obsolete versions compressed?  Etc.

frederik long say:

I also hear that you cant defrag & Optimize a SSD.

It's not so much that you can't defrag an SSD but there is no point in defragging it. With a magnetic disk you can reduce access times a little bit by ensuring that data blocks are contiguous, thus allowing the read head to read the several blocks in one sweep instead of hopping from one fragment to the next and perhaps waiting for a full disk revolution.

With SSDs this is pointless. Why? Because it takes exactly the same amount of time to read two adjacent blocks of data or else to read two blocks that reside at different addresses on the "disk".

frederik long say:

Wouldn't it have to read the First Block of Information and then Jump to the last block to find the rest of the information leaving the two middle blocks empty, and wouldn't that take extra time???

No, it would not.

Semiconductor data is read like so:

  1. The CPU places an address into its address register, e.g. FF 20 38 75 12.
  2. It fetches the data block stored at that address.
  3. It places the next address into its address register, e.g. FF 20 38 AB 00.
  4. It fetches the data block stored at that address.

Placing different addresses into the address register does not take extra time.

Reading data from different areas of the memory does not take extra time either.

This is truly "random access": The CPU can access any area on the disk in a totally random fashion, same as you can telephone anyone on the globe instantly, if you have his number handy. The distance is largely irrelevant.

A video cassette is truly sequential access. Information can be read in a sequential manner only.

A magnetic disk is partly random, partly sequential access. The CPU might have to wait up to one disk revolution before it can read a data block. To reduce this wait time it pays to keep the disk defragged.

linney say:

Were there any problems with the Windows 7 install?

You could try uninstalling Nvidia, then running System Restore to get back to before you even installed Nvidia, then test the machine.

Alternatively you could scrub everything and have another go at installing Windows 7.

Perhaps a look in the Event Viewer (and if it is full of errors) might convince you that a reinstall is a good decision?

Working with Vista’s new Event Viewer
http://www.petri.co.il/vista-event-viewer.htm

http://windows.microsoft.com/en-us/windows7/What-information-appears-in-event-logs-Event-Viewer

 

How to repair the operating system and how to restore the operating system configuration to an earlier point in time in Windows Vista (or 7)
http://support.microsoft.com/kb/936212/#appliesto

 

Have a look at the hardware health of your machine.

Try running ChkDsk to check your drive for errors. Right-click your Drive icon/ Properties/ Tools/ Error Checking.  Try it first by not checking either box (this will run it in a Read-only mode) to see if it flags any hard drive or file problems.  If it does, restart  it by ticking both boxes, and rerun it to allow it to attempt to fix any found problems.


The hard drive manufacturer will have free diagnostic software that is bootable, or that  may even run from within Windows, that will thoroughly check the condition of your hard drive.

How to Use the Memory Diagnostics Tool in Vista or Windows 7

If you are unable to sign in with Microsoft account at the login screen due to internet connectivity, I suggest you to login with local user account and update the network adaptor drivers.

Follow the steps provided below to update network adaptor drivers.

Method 1: Update Network adaptor.

I suggest you to update your network adaptor drivers and check.

  1. Press Windows key + R key.
  2. Type devmgmt.msc and press Enter.
  3. In device manager go to Network adaptor.
  4. Right click and click on Update drive software.

You may also download the latest drivers from the manufacturer’s website and install them on your computer.

After updating the network adaptors drivers, I suggest you to login with Microsoft account and check if you are able to login.
