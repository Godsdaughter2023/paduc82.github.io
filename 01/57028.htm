wcornelson say:

System Restore checkpoints.

Using XP Pro SP3.  System Restore will not create restore points automatically or manually.  After a manual attempt fails, I am advised to0 restart the computer and try again.  I've confirmed that the required services are running and no errors are recorded in the Event Log.  Lots of room on my HDD.

a. user say:

XP wants to create a new checkpoint automatically every 86400 seconds (every 24 hours).

XP will not create an checkpoint automatically if some other Restore Point has been created in the last 24 hours.

XP will also not create a checkpoint automatically if your system is not in an idle state for a certain amount of time.  For me, that "certain amount of time" seems to be about 15-20 minutes.

If you are running some process that consumes more than 10% of the CPU even for a fraction of a second, the "timer" gets reset and the countdown to being idle long enough to create a checkpoint will start over again.

If you startup your system and do some things and then shut down before the automatic checkpoint is made, you may have missed the window for that 24 hour cycle and end up with no RP for that day.  This is very commmon but does not mean something is haywire.

I have seen folks (and me) that startup and work all day emailing, typing, browsing, etc. then shutdown until the next day.  XP is never idle for them long enough for XP to create a RP and they can go many days or even a week or more without a checkpoint and wonder what is going on, but XP is working as designed.  If they start up one day and don't do anything for 30 minutes - viola! a checkpoint will be created provided no checkpoint has been made in the last 86400 seconds (this is all configurable).

Idle means no mouse, no keyboard, no surfing, no automatic or manual antivirus/antimaleare scanning, no defrag, no chkdsk, no instant messenger, no Skype, no movies, no Magicjack, etc.

If you are using Microsoft Security Essentials (MSE), whenever it gets an automatic update, it will also create a new RP before the update is installed (it downloads the update, makes a Restore Point and then installs the update).  Those MSE updates will show up in your System Restore calendar as "Software Distribution Service 3.0".

A MSE update that creates a new RP "counts" as a Restore Point, so if MSE gets updated daily or more frequently than every 24 hours you might go days without seeing a System Checkpoint and only see several days in a row of the SDS 3.0 RPs in your calendar and no checkpoints.

If you are using MSE, you may never see a System Checkpoint in your System Restore calendar again.  You will probably see almost daily Software Distribution Service 3.0 Restore Points instead.  It is all explainable though.

Some antimalware and antivirus applications can also interfere with the System Restore Process as they try to protect your system.

When your manually attempt at creating a Restore Point fails, what does the informational message say exactly?

Have you successfully manually created a Restore Point in the past?

Do you see old Restore Points available in the System Restore calendar (you don't need to try any).

What are the required services that need to be running for System Restore to work?

If your system currently has or has had certain malicious software afflictions, your System Restore process could be so afflicted, you may just need to reinstall System Restore to get it working again (not reinstall XP - you just reinstall the System Restore part).  Reinstalling System Restore will delete all the current Restore Points.

What is your system make and model?

Describe your current antivirus and anti malware situation:  McAfee, Symantec, Norton, Spybot, AVG, Avira!, MSE, Panda, Trend Micro, CA, Defender, ZoneAlarm, PC Tools, Comodo, etc.

Does the afflicted system have a working CD/DVD drive?

Do you have a genuine bootable XP installation CD (this is not the same as any Recovery CDs that came with your system)?

Do, or do not. There is no try.

The SDS 3.0 Restore Points mean that they were created from some Windows update that came in.  They are not quite the same in their creation as other Restore Points bit are still valid Restore Points.  The updates come in and are finished downloading, a new SDS 3.0 RP is created, then the updates are installed.

A SDS Restore Point will fulfill the desire of XP to create a RP every 24 hours, so you may never see a System Checkpoint in the calendar again.  That does not mean SR is broken of course, but you should be seeing SDS 3.0 Restore Points every time MSE updates.

When you reinstall System Restore, all of your current Restore Points will be removed and you must restart your computer when 
the installation is complete.  After the restart, that will automatically create one new System Checkpoint in your clean SR calendar.
