This window flickers away after a few seconds as the connection crashes, then up again, then down again, etc., for a certain number of times. If Samsung Kies is open, it briefly shows a "connecting" screen followed by a "connection lost" message, etc. At other times, Windows figures out that the driver is buggy and pretends the phone isn't there (i.e. does not show up in Devices and Printers, Kies does not see it altogether, etc.). When the latter case occurs, I can see my Android phone with a yellow badge in Device Manager ("This device cannot start. (Code 10) The process hosting the driver for this device has terminated.") Whether Windows takes one approach or the other changes with time and especially across laptop sleep cycles.

NOTE: I have already tried different USB cables of course. The phone is a Galaxy S4 with the latest firmware.

* As for the Ericsson device, when this issue pops up, I lose all geolocation features. Probably its 4G connectivity feature fails too but I never used it so I can't check.

I cannot pinpoint exactly when the problems started. The phone connection was definitely working fine a couple of months ago. Then I started noticing the "Explorer filckers in and out multiple times when phone is connected" behaviour, yet for example I could backup my phone with Kies, browse its flash, and do what I needed. Later I upgraded the phone's firmware, Kies and (bundled with Kies) the USB drivers, and now the UMDF framework crashes quickly and frequently enough that it is impossible to do anything.

============== Event Viewer

There are lots of noisy messages about this. Notably "DriverFrameworks-UserMode" events 10110, 10116, 10115. The errors all seem to start from the Samsung USB driver first, and then the hosting framework is killed taking down all connected devices, which really as far as I can see is the Samsung and Ericsson drivers only. On boot I also get 10114, but this is only an information message which has always been there (even when the devices were working) and might appear harmless (?).

============== Software versions

As far as I can see, I have the latest version of all drivers. The Ericsson drivers come from the Lenovo website, version 8.1.3.0. The Samsung drivers are what is bundled with the latest Kies version - Kies 3.2.14055_3, USB driver 1.5.40.0. The phone is updated to the latest Android 4.4.2 firmware (it was never rooted, by the way) and Windows is version 8.1 with Update 1 and all patches installed.

In other words, it might seem that Windows is not actually using any Samsung driver file. Maybe this is only part of the story though. In any case if I try to manually "Update driver..." by pointing to the Samsung files, Windows is adamant that the installed driver is already the right one.

How do we proceed from here? Is there something I should send you?
