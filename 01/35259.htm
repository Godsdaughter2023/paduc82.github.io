lemp say:
First of all, Windows XP Fax Services is notoriously fussy.  It is "guaranteed" (right!) to work only if the modem is on the Windows XP Hardware Compatibility list.  Unfortunately, Microsoft has disappeared the Windows XP Hardware Compatibility List, so it's easiest to respond to you by saying that it's a hardware compatibility problem.  See, e.g., this ancient response (Russ Valentine was one of the best XP fax experts around).

Did your system ever work properly to send faxes?  If so, what happened between the time it worked and the time it didn't?

First see if your modem is properly responding to simple commands: Directions here.

Then turn the modem's internal speaker on so that you can listen as it attempts to send the fax and perhaps hear whether there is a dial tone or busy signal. 

  • Open Device Manager (Start > Run > devmgmt.msc > OK)
  • Double click "Modems" then double click the entry for your particular modem.
  • Click the Advanced tab
  • In the box under "Extra initialization commands" enter the following

         M2L3

  • OK your way out
The "M2" command specifies that the modem's internal speaker should be on at all times and L3 sets the speaker's volume as high as it will go (which isn't much). FYI

          M0 : The speaker is always off.
          M1 : Enables the speaker during call establishment, but off after the carrier is detected.
          M2 : The speaker is always on.
          M3 : Disables the speaker during the dialing period and receiving carrier, but on during answering.

You also can use HyperTerminal (which comes with Windows XP) to test your modem.  These are the standard directions for doing so.  For some reason (that I'm unwilling to spend time investigating) those directions don't work for me, so before you try them, try the following, which works for me:

  • Open HyperTerminal.
  • Click Cancel on the "Connection Description" window
  • In the open HyperTerminal window, type the two letters AT and press Enter.  If HyperTerminal responds "OK" then you can proceed to testing (see below).  If you don't see anything at all, type the four characters ATE1 and press Enter (that's a number 1 after the E).  Again, if you get an "OK" response, you're good to go.  If you don't get an "OK" then follow the standard directions linked above.

The simplest way to test your modem is to type the four letters ATDT in the HyperTerminal window followed by your cell phone number (or any other telephone number in your house that is NOT on the line to which your modem is connected) and then press Enter.  Type the number exactly the way you would dial it from your home phone.  For example, if your cell phone is (212) 555-1212 and your phone company requires you to dial all ten digits -- but not a preceding 1 -- then you would enter the following in the HyperTerminal window:

     ATDT 2125551212 {Enter}

If everything is working, your cell phone will ring.  If you answer, you'll hear your modem attempting to initiate a data call. 

You can also enter the speaker volume commands in the HyperTerminal window:

    ATM2L3 {Enter}

Hi,

Resolving a corrupt profile can be problematic and time consuming at best. Many times the easiest and fastest solution is to (as you already did) create another profile and then move your files from the messed up profile into the working one. Once everything you need is moved over, just delete the corrupted profile.

What you need to do is to copy the files to the Public user. Once done, sign in to the second account and move the files from the Public user to the second account.

Note: You can't copy all files from one user to another - only data files (Documents, Music, Pictures, Videos, Downloads, Favorites.)

You can also follow the steps below on how to copy files to the new user profile.

  1. Log on as a user other than the user whose profile you are copying files to or from.
  2. In Windows Explorer, click  on View and select Option.
  3. Click the Change Folder and search option, click on View tab and Show hidden files and folders.
  4. Uncheck the Hide protected operating system files check box, and then click OK.
  5. Locate the C:\Documents and Settings\Old_Username folder, where C is the drive on which Windows is installed, and Old_Username is the name of the profile you want to copy user data from.
  6. Press and hold down the CTRL key while you click each file and subfolder in this folder, except the following files:
    • Ntuser.dat
    • Ntuser.dat.log
    • Ntuser.ini
  7. On the Edit menu, click Copy.
  8. Locate the C:\Documents and Settings\New_Username folder, where C is the drive on which Windows is installed, and New_Username is the name of the user profile that you created in the Create a New User Profile section.
  9. On the Edit menu, click Paste.
  10. Log off the computer, and then log on as the new user.

Hope the above information helps.
