Windows Adds Numbers to Network Names - Microsoft Community

banjo999 say:

Windows Adds Numbers to Network Names

I have been trying to resolve this issue for months and have finally found a definitive solution. Despite all my Google searching, I have not found an actual answer anywhere.

The problem here is that Windows will add a number to a network name whenever a change is made to said network. For instance, your computer is connected to "Network," then you change the encryption for the wireless connection from WPA to WPA2. You connect using the new encryption key, and suddenly Windows says you're connected to "Network 2."

This has absolutely no affect on anything--you can still connect to other computers on the network, the SSID remains the same, you can still use the Internet--except it's kind of annoying to look at. Maybe it's a tad bit of obsessive compulsiveness, but I personally don't like it.

I had the same issue on Windows Vista, only more extreme. I needed to make a series of changes to my router when I was running Vista to adjust to the new house and devices, and each time I'd get a new network name. Instead of replacing the old network name like Windows 7 does, however, it adds a new network to the list. I'd have the choice to connect to "Network," "Network 2," "Network 2 3," "Network 2 3 4," etc., and they would all connect to the same access point. Odd.

Point is, I've come across the issue once again while running Windows 7 and have at last found a solution.

SOLUTION: Control Panel>>Network and Internet>>Manage Wireless Networks

Simply right click on and delete all versions of the network that are unneeded (i.e. "Network" and "Network 2"), then right click on and rename whatever network it is you're connected to (i.e. "Network 3" changed to "Network").

cmdrkeene say:
No duplicates are created, you're just looking at the same file from two different angles.  I know this answer is long, but it will explain...

Libraries are not different folders or locations.  They're actually just combined views.  Put literally, they operate like this: when you open your Documents library, it runs a search of two locations: your documents, and the public documents folders, then shows you every file or folder it found in those locations.  The point is to give you one combined view of all the documents you care about, regardless of where they are.  If you had documents on a network drive or external hard drive, you could tell the library to search those folders too and include them in the combined view.

When you save something to a library, the library just puts it in the default location (in this case, the default would be something like C:\Users\John\Documents) and it begins showing up in the library's search result.  If you delete something, you're deleting the original:  there never were two copies or anything, the library is just there to combine multiple places to make it easier to see all at once.

You might think you have no use for this -- and you could very well be right.  If that's the case, just use it normally and don't bother including extra folders in the library.  That is to say, just pretend that your Documents library is your Documents folder.

I, on the other hand, use it like this:  I have a lot of movies, TV shows, video clips and other video content.  So much that it takes up several hard drives.  I told my Video library to show all these locations so that I can browse all my movies from one Library, rather than trying to remember which drive a particular show was on.
