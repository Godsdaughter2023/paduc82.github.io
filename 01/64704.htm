hairyfool say:

Although ostensibly the operating system is Windows it is an ARM processor in the Surface so runs Windows RT. Any software able to run on the Surface has to be written for that platform and none written for previous versions of Windows will work on it. This is exactly the same as the fact that Windows software will not run on an iPad, Andriod tablet or even natively on a Mac.

 

To a certain extent it is a separate issue but in addition Windows RT will in the main only allows software to be installed via Windows Store in a similar way that other tablets are also limited. There is an exception where companies create their own line of business application which can be "sideloaded". This is being done to ensure software publicly available is written and tested to meet a standard that is fit to be published.

 

The Surface is a "real" computer in that it can run any program written for it, if you need the functionality of legacy software then you will have to wait for Surface Pro next year or look at alternative offerings from 3rd party manufacturers.

wunders say:

What happened was that you ended up with a corrupted profile and when Windows noticed that at startup it generated another profile for you so that you can log into the machine.  IMHO the biggest drawback to the Encrypting File System (EFS) is that Windows manages the keys for you while maintaining transparent access to your encrypted data.  That's fine, but when Windows fails, so does the access to your data.  Chances are that you never did back up your encryption keys.    Your encryption keys are themselves stored encrypted in your old user profile -- that would be the ntuser.dat file in your old folder.  Your best chance to access your files would be to log into your old profile...  and you're right, the best bet you have at this point in time is to do a  System Restore to a date before this problem happened.  It should restore your old user profile and with any luck you would be able to log into your original profile after the restore.  I recommend you perform the System Restore in Safe Mode to prevent interaction with anti-virus programs that can cause the restore to fail.

"How to restore Windows XP to a previous state"
  < http://support.microsoft.com/kb/306084 >

Once you regain access to your files, be sure that you manually back up your encryption certificates.  The following article will show you how and I consider it mandatory reading for anyone using EFS.  Note the paragraph "Why you must back up your certificates":

"Best practices for the Encrypting File System"
  < http://support.microsoft.com/kb/223316 >

As a fall-back position, there is a 3rd party commercial software package calledAEFSDR that some people in this forum have reported success with.

Myself, I avoid EFS in preference to the freeware Truecrypt in which I maintain control of the key via a passphrase (other options are available).  I have had several Windows systems shot out from under me over the years and have never lost any encrypted data.  It will also work on XP Home.

Hope this helps,
  JW

unkn0wn1 say:

Generally, UPNP will make no difference if you have the Xbox in DMZ (Demilitarized Zone). Since DMZ opens every single port, or Exposing to the internet.

UPNP's function is to automatically open ports for a device requesting a certain action. Since DMZ opens every port, there is no room for UPNP to function to benefit the Xbox.

I have a Motorola SBG6580 gateway. Last time I checked, I wasn't able to make a static IP address from the Xbox's Networking settings, as I got connection errors. If you also get a connection error, you can try Method 2: A alternative way to create a static IP address, use DHCP Reservation. (DHCP Reservation can be found on the Basic tab, then click DHCP) It will ask for the Xbox's MAC address (this can ensure that the modem will connect the Xbox to the Static IP address you create) You can create your own IP address, as long as it is in this format 192.168.0.x.    x = any single digit number or a (2 digit number if you have to) However, just as what littleweasel74 said, you can use "192.168.0.2" for the Xbox, Just make sure no other device is using that IP address, otherwise you will get a IP Address conflict error on the Xbox. Now as for the Host name, just type Xbox 360.

unkn0wn1 say:

Note: If you do the method above you will have to set IP settings and DNS settings on your Xbox back to Automatic.
