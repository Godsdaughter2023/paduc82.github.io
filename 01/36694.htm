Both PCs are set for automatic IP address allocation as you recommend. I have not changed the settings and so assume they have been set that way from new.

However they were initially, and to use better terminology, on different subnets - only the first 2 elements of the IP addreses were the same. While this was the situation they could not join the same Homegroup.

ipconfig /renew

at a command prompt (you may need to run cmd as Administrator to do that).

On some network devices, you can assign different subnets to different ports, but if you had a router capable of that and you had done it I guess you would have remembered that.

Hi

Welcome to Skype Community!

Are you using the microphone from a headset or a built-in microphone of your computer? If you're using a headset, try using a different cable or plugging it into a different port. 

If you're using the built-in microphone of your device, Microsoft has a tool to automatically diagnose and fix audio problems. 

If the problem is still persisting, please perform the steps below:

  1. Make sure that Skype is no longer running.
  2. Press the Windows  and R keys on your keyboard at the same time.
  3. Type %appdata%\skype into the Run window and click OK.
  4. Find and delete the shared.xml file. If file extensions are not displayed on your computer, the file will just be called shared.
  5. Open the folder that has your Skype name.
  6. Locate and delete the configor config.xml file.
  7. Press the Windows  and R keys on your keyboard at the same time again.
  8. Type %appdata% into the Run window and click OK.
  9. Locate and rename the Skype folder to Skype_Old.
  10. Uninstall Skype.
  11. Restart your Windows computer.
  12. Download the most up-to-date version. 

I hope this helps.
