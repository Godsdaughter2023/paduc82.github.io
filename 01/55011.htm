d gittins say:

How to use Task Manager and Resource Monitor to find out what is slowing down your computer

Summary
What's my computer doing? Why is it running so slowly?
Details

Task Manager and Resource Monitor are both great tools for finding which programs are slowing down your computer.

Press Ctrl+Alt+Del and then click Task Manager to open the program.

What's my CPU doing?

Click the Performance tab. Here you can see graphs for the CPU (the computer's processor) and Memory usage.

Look closely at the CPU graph. Most of the time, it should be very low with an occasional spike. If it's pegged near 100%, or even running very high most of the time, it could be that the CPU is being overloaded.

To track down the culprit:

1.       Click the Processes tab in Windows 7 or the Details tab in Windows 8.

2.       Click the CPU heading to sort the results by that column in descending order.

3.       Look for a process that consistently shows a big number. This is the process that is currently keeping the CPU busy. Make a note of the name of that process.
Note You can ignore the "System Idle Process".

For example, the MsMpEng.exe process is using a high number, this is OK. MsMpEng.exe is the Microsoft Malware Protection Engine file, so this indicates that your anti-malware program is busy performing a scan.

On the other hand, the process could belong to a failing program or even malware. Search the internet for clues about what the process is and which program is responsible for it. Then see if an update is available for the program that might fix the issue.

Tip: If necessary, right-click the process and then click End Process. In extreme cases, this could help to get the computer back to a usable state so that you can continue to find a solution.

What's my hard disk drive doing?

Modern computers have very fast CPUs and lots of fast memory. However, sometimes it might be the hard disk that is slowing things down.

The computer uses a large "cache" file on the hard disk to assist with memory, so if the disk is overloaded, your computer's performance could be affected – everything might feel slow.

1.       In Task Manager, click the Performance tab and then click Open Resource Manager.

2.       Click the Disk tab, and then click to expand the Disk Activity bar.

3.       In the Disk Activity list, click the Write (B/sec) heading to sort by that column in descending order.

4.       Look for a process that consistently shows a big number. This is the process that is currently keeping hard disk busy. Make a note of the name of that process.

For example, you might find that the Windows.EDB process is using a high number. This is also OK. Windows.EDB is the database file that the Windows Search Index uses.

On the other hand, the process could belong to a failing program or even malware. Again, search the internet for clues about what the process is and which program owns it. See if an update is available for the program.

Hi Ted,

This issue occurs when the hibernation file for your virtual machine gets corrupted. For Windows Virtual PC (the version that comes with Windows 7), the default action for close is Hibernate.Hiberfil.sys is a snap-shot of the state of your computer that is taken by windows when it becomes inactive. However, virtual machines don't store their hibernation files in hiberfil.sys like a physical Windows system does. The hibernation file for a virtual machine is referred asthe storage of the current state of the virtual machine. So the hibernation file for Windows Virtual Machine is stored in the following location which has a file extension*.vsv

C:\Users\ (Username)\ AppData\Local\Microsoft\Windows Virtual PC\Virtual Machines directory

I: To rename the file *.vsv, follow the steps mentioned below.

1. Click on Start and click on computer.

2. Go to the following location:
    C:\Users\ (Username)\ AppData\Local\Microsoft\Windows Virtual PC\Virtual Machines directory

3. You’ll find a .VSV file in the Virtual Machines Directory.

4. Rename the file.
    For example: If the file name is VM.vsv, rename it toVM.old

5. Close all the windows and restart Virtual Machine. You should be able to run it without having any problem.

In order to prevent this problem from happening in the future, you can do a shutdown of the Virtual PC rather than the default hibernation.

II: Steps to change the option from "hibernate" to "shutdown" on your Windows Virtual PC

1. Open Virtual PC.

2. Click on Tools and then click on Settings.

3. Under the Close tab, change the option from Hibernateto shutdown.

4. Restart your computer and start your Virtual PC for the changes to take effect.

I would suggest you to put the computer into a clean boot and then try installing updates to see if you’re able to install the updates successfully. To put the computer into a clean boot, follow the steps in the link below.

a. user say:

I was wondering if you might be able to give me instructions on removing and re-creating the same account without losing anything.

Thanks.

OK. So, you did not set up the account yourself.

It will take some doing...
Open Windows Mail > click Tools at the top > click Accounts > click the account in question > click Properties >
click General : copy down everything that were entered, including checkmark.
click Servers : do the same
click Connection : do the same
click Security : same. click Advanced : do the same.
click OK.

Now you are back on Internet Accounts window > click the account in question again > click Remove.

The old account is now removed.

Now recreate the account ...
Open Windows Mail > look up at the top, click HELP > click view Help > click Windows Mail : Setting up an account from start to finish.
Follow the instructions and use the information you have copied down.
