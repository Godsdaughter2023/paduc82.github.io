¡firedog say:

This sounds a bit unlikely. POP3 has survived so long (it's about 35 years old, almost pre-dating the first PCs) because it's so robust and fail-safe. If there is a new message on the server, POP3 will do its **** to download it. If it can't, there will almost always be an error message saying so. Do you not see any error messages when a message you know to be present on the server isn't downloaded by a POP3 session? It will be indicated by a warning sign  on the status bar (the status bar must be enabled by clicking its button on the View tab). A single click on the yellow triangle will open the Errors tab of the send/receive progress window. You can click on an error message to select it, then use right-click to copy it and paste it into your reply here.

It might be worthwhile checking the program log. Here are the steps to generate a log of client/server transactions:

A.  Enable logging

  1. When viewing your message list, press Ctrl-Shift-O for Options.
  2. On the General tab, deselect Send and receive messages at start-up.
  3. On the Advanced tab, click Maintenance. Under Troubleshooting at the bottom of the dialogue, select Email only.
  4. Click Store folder and copy or make a note of the path to the store folder. Click OK.
  5. Click Close,

    B.  Find the log

    1. Open File Explorer (Windows key + E)
    2. On the View tab, ensure that both File extensions and Hidden items are selected in the Show/hide section of the ribbon.
    3. Navigate to the store folder. If you copied the path at A.4, paste into the address bar to do so.
    4. The log is a plain text file called WindowsLiveMail.log. If it exists, rename it to Windows Live Mail.old

    C.  Generate a new log

    1. Launch Windows Live Mail.
    2. On the Home tab of the ribbon, click the bottom half of the Send/Receive button and select the account whose transactions are to be logged.
    3. When the session ends (and the error message is shown, if any), dismiss the progress window and close down Windows Live Mail again.
    4. Wait a few moments for the program to finish its housekeeping and write the new log.

    D.  Post the log

    1. In File Explorer, right-click on WindowsLiveMail.log and select Open with > Notepad.
    2. Click inside the Notepad window and press Ctrl-A C to select All and Copy it.
    3. Paste into your reply.

    The POP3 transactions recorded in your log are all pretty straightforward: the STAT request produces the response +OK 27 493103, meaning there are 27 messages in the mailbox totalling 493,103 bytes. The LIST request then produces a list of these 27 messages by serial number and size in bytes. Next comes the UIDL, which lists the unique identifier for each message. The program will then compare each UID against those of messages already received. In this case, there were apparently no new ones, because the next request sent is QUIT. Had there been any new ones, the program would have sent a RETR request to retrieve the new messages.

Since some new messages are indeed downloaded, the fault (if it is a fault) must be at the server end.     1. Are OEM licenses distributed to any country with only for branded machine (factory fitted os)???

    Yes, no matter where it is sold, it is only to be distributed with a new computer.

    2. Can I purchase separately OEM licenses for 50 Nos of computer instead of factory fitted os of 50 Nos of computers from branded computer manufacturer ????

    You can purchase OEM System Builder licenses for existing and new computers, but the terms of the license agreement insist that it be only distributed with new hardware.

    OEM versions of Windows 7 are identical to Full License Retail versions except for the following:

    - OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

    - OEM licenses are tied to the very first computer you install and activate it on

    - OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

    - OEM versions cannot be used to directly upgrade from an older Windows operating system

    3. Are there recovery cds for factory fitted os with branded machine?

    New computers that come preinstalled with Windows 7 often have what is called a recovery partition. This is used to reinstall the operating system in the event of a system crash. To access it, you will need to boot into when you start your computer by pressing a function key. This can be either F1, F2, F9, F10, F11, F12 or even DEL or Tab key. Consult the owners manual that came with your PC for instructions about how to reinstall Windows.

    This is how the recovery partition is accessed for the most popular makes

    For Dell, press CTRL + F11 directly after powering up the unit

    For HP, press F11 directly after powering up the unit

    For Toshiba, press and hold '0' BEFORE and during the power up

    For Acer, press and hold ALT + F10 as soon as you see the logo

    For Asus, press F9 as soon as you see the Asus logo.

    For Advent, restart your computer. Then Press F10 repeatedly until the message "Starting System Recovery" is displayed
