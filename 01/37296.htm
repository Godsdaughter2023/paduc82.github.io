Windows 10 Pro 64-bit

 

There are numerous threads regarding networking problems which occur after upgrading to Windows 10, or from updates to Windows 10 systems. The upgrades and updates corrupt networking on Windows 10. Unfortunately, the consequences of the corruptions are a wide variety of networking problems - many of which appear unrelated. As a result, many threads speak of computers not appearing in the network, "0x80070035 network path not found" errors, Windows Sockets errors, protocol error messages, and other strange errors. All these errors are due to the network settings being corrupted by the Windows 10 upgrades and updates.

 

The following is not a perfect solution, but often corrects many issues:

 

Reset your modem/router to factory settings.

Start | right mouse | Command Prompt (Admin)

   Run the following two commands, then reboot.

      netsh winsock reset

      netsh int ip reset resettcpip.txt

 

Resetting the modem/router to factory settings makes sure the router is not the issue.

Running the two netsh commands resets network component settings.

 

If you have a home network,

in "Open Network and Sharing Center",

make sure it is "private".

 

In order to see your local home networked computers using advanced sharing to their drives,

in "Change advanced sharing settings",

in Private (current profile), enable the following:

   Turn on network discovery

   Turn on file and printer sharing

   Use user accounts and passwords to connect to other computers

 

In order to see your local home networked computers using advanced sharing to their drives, both your PC and the other local network PC require an account on both computers with the same name.

In addition, for each computer, you must go into their

"Credential Manager" | "Windows Credentials" tab

and give the Password for the common account to the other computer.
