Are the BIOS settings at default/auto, or have you over-clocked the cpu or changed the RAM settings,
particularly the RAM speed ?
Setting the RAM speed over the motherboard default without over-clocking the whole system can
lead to lock-ups or BSOD (more commonly BSOD, and generally more random). While not so common with current cpu's and the improved memory controllers, it's worth reverting to default/auto for troubleshooting if you are  using custom settings.

Running games with too high settings can also cause lock-ups, but as with the RAM settings this is
generally more random than what you seem to be experiencing.
This can happen even when frame rates are adequate if the gpu is pushing 99% of it's capacity.
You may want to try my quick fix for this anyway.
Leave the settings all high and lower the res one notch  eg - 1920x1080 to 1680x1050. This will drop the gpu use several points and help with stability.
This worked for me with Skyrim, which locked up a couple of times, and Crysis 2 which had a couple
of TDR crashes before lowering the res.

considerate_guy say:

Sure, I'll try and detail what I did.  First I made a copy of the file so I wasn't working with the original  Then I just opened the file up in Notepad, no fancy XML editor or validator needed.  In fact, the XML is actually still valid so a validator would not help.

I scrolled to the bottom of the file; at the end there is a <BoundPlaceholders> section.
It says that the "Text" section has an extentID of 4.

Looking just above that you should see a section that starts with <ExtentSelector extentID="4"
That lists all the added titles and captions by number. Make a note of all these because you'll need them later.

Now search further up for each section starting with <TitleClip. Ignore any sections that have a complete section between the <Effects> and </Effects> tags.  Eventually you'll hit a few that just have <Effects /> (an empty Effects section).

First, check if they are in the list of ExtentIDs in the Text section from before (the ones you made a note of).  If they aren't you can ignore them, but if you find that it has an empty Effects section *and* it is listed in that block of text ExtentIDs, that's a problem.

That's the one that needs to be replaced with <Effects> ... </Effects> from a working title. I usually fill in the <BoundPropertyStringSet Name="string"> section with a recognizable string like "FIX ME!" so I can find it later.   Usually there is only one caption that is corrupted, but in the case of yours I think there were 3 or 4.

Now after replacing the empty Effects sections in those captions with good data, you can save the file.  After saving the file, it should once again open in WLMM. You'll find the corrupted title(s) and will need to fix the position, text and styling.

If you have a difference tool, you can just do a comparison between your original project and what I corrected to see specifically how it changed.  That's how I first started figuring out what the Microsoft team was doing behind the scenes.  It's really not all that magical and can be done easily, if you are careful and only work in a copy.

a. user say:

If a system overheats and gets a thermal event it will shutdown (power off) to prevent permanent damage.  There will usually be no warning message, no Blue Screen of Death (BSOD), nothing logged and sometimes you will have to wait for the system to cool off a bit before it will restart.  This is not the same as a BSOD when some part of Windows OS crashes.

Wouldn't it make sense to see what your system temperatures are now before you start making adjustments?

Then you will be able to know if the problem is heat related and after that you will be able to know if your adjustments are having any effect or not - but first you need to see what things looks like now.

If you download the free version of Speccy you can see what your temperatures are and lots of other things:

https://www.piriform.com/speccy

You can uninstall Speccy later if you don't like it.

You can provide all the information requested by Canadian Tech (and lots more) in one reply if you do this:

Launch Speccy and after it has analyzed, click File, Publish Snapshot, Yes, Copy to Clipboard:

That will copy the URL of your system information to the Windows clipboard.


In your next reply, press Ctrl-V (the Windows paste operation) to post the URL to your Speccy snapshot.

When Speccy is running temperatures in green are good and temperatures in red would be bad (in this example everything is green=good and red=bad):

Remember:  Green=Good  Orange=A Concern  Red=Bad

If you minimize Speccy to the system tray the average temperature will still be displayed next to the system clock so you can keep an eye on it and if it gets into the red, you have a thermal problem:

See if you can reproduce the shutdown problem while Speccy is running and showing the temperature of things.

Keep an eye on the temperature while you are doing whatever you do that causes the system to shutdown.

canadian tech say:
One common cause of overheating in a laptop is using it while it is on a soft surface.  Laptops generate a lot of heat and it is critical that that heat be removed quickly to prevent over-heating which can ruin the components.  The underside of your laptop has tiny vents through which it pulls the air to cool itself.  If your laptop is on a soft surface, it can block those vents and cause over-heating.
