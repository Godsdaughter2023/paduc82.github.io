jim reid --- tcg computers say:

So this is actually a fairly complicated topic.

First, Gustovier is correct that the Xbox One's EDID only tells the computer that it accepts 2 channel audio. This can be verified by opening the Sound applet in Control Panel on the computer while it is connected to the Xbox One via HDMI. Then right click the Xbox One device on the Playback tab and choose Properties. Finally, on the Supported Formats tab, you will see that "Max Number of Channels" is only 2. In addition, if you attempt to "Configure speakers" on the Xbox One device in that same applet, you will only see "Stereo" as a speaker option.

However, this actually doesn't cause a problem for Windows Media Center. The reason is that WMC has a known bug that if it is using HDMI for audio out and has Windows set to higher than stereo speakers, the audio will skip a few times and then a "Video Error" will be displayed. The workaround for that bug is to keep Windows set to only 2 speakers, but set "Exclusive mode" in the Sound applet in Control Panel. It is a series of two check boxes on the Advanced tab in the Playback device's properties. You then configure your surround sound speaker options, 5.1 or 7.1 channel depending on your setup, directly within WMC rather than in Windows itself. This does have the drawback of some third party programs that don't support Exclusive Mode not being able to output surround sound correctly, but it isn't the Xbox One's fault. As I mentioned earlier, it is a known bug and you can find tons of information about it if you do a search for something like "wmc hdmi speakers video error."

Because the workaround for that bug in WMC is to keep Windows set to 2 channel audio and WMC itself set to the correct surround sound configuration directly, it doesn't really matter that the Xbox One's EDID only represents itself as a max of 2 speakers. I have it set this way in my configuration and it works fine.

Speaking of EDID, though, I've been using a Gefen HDMI Detective for quite a while in my WMC setup to store my TV and AVR EDID info and keep provided constantly to my Media Center computer. This prevented the problem of HDMI handshake issues with the computer, which can cause resolution issues, sound dropping, and long black screen wait periods when turning on the TV and AVR while the computer is running. When I first bought my Xbox One, I was hoping that I could remove the HDMI Detective and the Xbox One would provide a constant EDID to the computer, since the Xbox One is "always on." I was disappointed that it didn't and it had the same HDMI handshake issues I had before, so I reconnected the HDMI Detective back inline between the computer and the Xbox One. However, I recently decided to try it again and to my surprise it works now. I assume that one of the updates that has come in the last year has improved the EDID support to provide it constantly to the connected device. Between that and the fact that the Xbox One can now boot directly to the TV app, I've finally removed the HDMI splitter from my setup and my family is now watching all our TV on our Media Center computer through the HDMI pass-through on the Xbox One.

To troubleshoot most computer problems it helps to look at available clues in the System Information file and the Event Viewer logs. An experienced eye can pick out the needles in the haystack and see more clearly the precise nature of the problem or problems.

 

Please provide a copy of your System Information file. Type System Information in the Search Box above the start Button and press the ENTER key (alternative is Select Start, All Programs, Accessories, System Tools, System Information). Select File, Export and give the file a name noting where it is located. Do not place the cursor within the body of the report before exporting the file. The system creates a new System Information file each time system information is accessed. You need to allow a minute or two for the file to be fully populated before exporting a copy. Please upload to your OneDrive (formerly Sky Drive), share with everyone and post a link here. Reports in normal mode preferred. Please say if the report has been obtained in safe mode.

 

Please upload and share with everyone copies of your System and Application logs from your Event Viewer to your OneDrive and post a link here.

 

To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window expand Windows Logs and select System. Place the cursor on System, select Action from the Menu and Save All Events as (the default evtx file type) and give the file a name. Do the same for the Applications log. Do not provide filtered files. Do not place the cursor in the list of reports before selecting Action from the menu. Do not clear logs whilst you have a continuing problem.
