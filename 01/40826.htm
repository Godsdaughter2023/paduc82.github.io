Recovery methods:


If you are running XP/SP3, then you should have a backup of your dbx files in the Recycle Bin, (or possibly the message store), copied as bak files.


To restore a bak folder to the message store folder, first find the location of the Message Store.


Tools | Options | Maintenance | Store Folder will reveal the location of your Outlook Express files. Write the location down and navigate to it in Windows Explorer or, copy and paste it into Start | Run.

 
In WinXP, the .dbx files are by default marked as hidden. To view these files in Explorer, you must enable Show Hidden Files and Folders under Start | Control Panel | Folder Options | View.


Note: If you have new messages in the folder you are going to restore, move them to a user created folder first or they will be lost. They can be moved back once the old messages have been restored.


Close OE and then in Windows Explorer, click on the dbx file for the missing, or empty, folder and drag it to the Desktop. It can be deleted later once you have successfully restored the bak file. Minimize the Message Store.

 
Open OE and, if the folder is missing, create a folder with the *exact* same name as the bak file you want to restore but without the .bak. Eg: If the file is Saved.bak, the new folder should be named Saved. Open the new folder and then close OE. If the folder is there, but just empty, continue on to the next step.

 
First, check if there is a bak file already in the message store. If there is, and you removed the dbx file, go ahead and rename it to dbx.


If it isn't already in the message store, open the Recycle bin and right click on the bak file for the folder in question and click Restore. Open the message store back up and change the file extension from .bak to .dbx. Close the message store and open OE. The messages should now be back in the folder.

 
If the messages are successfully restored, you can go ahead and delete the old dbx file that you moved to the Desktop. 

If you do not have bak copies of your dbx files in the Recycle Bin, then:


DBXpress run in Extract From Disk Mode is the best chance to recover messages:

The problem: Every time the monitor (not the PC) goes to sleep and wakes up again all icons and windows on the desktop are repositioned.  This is extremely annoying when you use a well-organized set of your most frequent directories and programs as icons on the desktop.

This problem has been discussed in the past, going back to Windows 7, and the problem was tracked down to a set of configuration keys in HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\Configuration.  This key contains a series of different monitor configurations, including one for the current monitor when awake and a few fallback configurations for when no monitor is detected.

Previously, I have been able to fix the problem, as suggested in the past threads, by adjusting various parameters in the fallback configurations to indicate a monitor with the same size and other parameters as the actual monitor.  However, after I finally did a clean Windows 10 reinstall, this solution does not appear to work any more.  I made all the adjustments, but the repositioning still happens.

sngbrdb say:

This is not a driver issue.  The problem is one of two things:

(1) Your DisplayPort connector doesn't conform to the DisplayPort specs.  This is highly likely, as 98% of cables for sale don't  : | 

References: 

  http://www.displayport.org/cables/how-to-choose-a-displayport-cable-and-not-get-a-bad-one/

  http://www.displayport.org/products-database/?products_category%5b%5d=cables-connectors&products_manufacturer=

(2) When your computer goes to sleep, Windows switches through a "default" monitor configuration, one that is ridiculously undersized to something like 1024x768.  When it does this, all windows are forced to resize accordingly - maximized windows are now "maximized" at 1024x768, the desktop is resized and rearranges icons accordingly, and so forth.  This is an absurd PITA.

Some links below, but the registry keys you're after are located under HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\GraphicsDrivers\Configuration\.  You're looking for keys with NOEDID and SIMULATED in the name... in Windows 7 there were two keys that started with those values, but in Windows 10 there seems to be three keys, beginning with MSBDD_NOEDID, MSNILNOEDID, and SIMULATED.  I'm still setting this up on my new laptop, so I'm not sure if all three are necessary.

Under those keys are a 00 sub-folder, which in turn has another 00 sub-folder.  Look at the other monitor configuration keys for examples of what's in there, but you're primarily interested in the values for PrimSurfSize, Position, and Stride.

The actual monitor configuration keys are concatenations of the monitor IDs, beginning with the primary display, followed by a GUID.  In my case, I have an ASUS 4K monitor and my Lenovo 1920x1080 display, with the ASUS monitor as the primary; my key is ACI32A4F5LMQS077816_15_07DF_FC+LEN40BA0_00_07DF_96^A2C52BC6ABD9890174D16A0275434102.  Under that are two folders, 00 and 01 for the ASUS and Lenovo displays, respectively.  Backup your registry, then copy PrimSurfSize.cx, PrimSurfSize.cy, and Stride from the root 00 folder to the 00 folder of the SIMULATED key.  Then copy the ActiveSize, DwmClipBox, PrimSurfSize, and Stride values from the 00\00 folder to the 00\00 folder of the SIMULATED key.  Start with just the SIMULATED key; reboot, and if that fixes the issue, don't mess with the other two.  If not, you can try the NOEDID keys, as well.

Also look for a small app called DesktopRestore:  http://www.midiox.com/desktoprestore.htm .  This will allow you to right-click on the desktop and save or restore icon positions for a particular monitor configuration.  There are other apps out there, but this is the one I know and have used since Windows XP.

Some links with partial instructions - I added values from my personal experience, as well.  Also, some are for Windows 7; adapt for Windows 10.  YMMV.

http://superuser.com/questions/267086/how-to-stop-windows-resizing-when-the-monitor-display-channel-is-turned-off-sw - look for Nomura's answer

http://superuser.com/questions/503298/windows-7-resizes-desktop-when-i-switch-of-a-monitor - Robert's answer
