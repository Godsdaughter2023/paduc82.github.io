woward say:

Here are more details on what is causing this error:

 

1. A normal install of Win8 creates a hidden Recovery Partition of 300MB with about 80MB of unused space.  The files in the partition include Winre.wim at about 190MB.  After the install the hidden Recovery partition has about 80MB of free space.  If the user attempts a System Image Backup, the process runs and completes correctly.

 

2. When that same Win8 PC is "updated" to Win8.1 using the Microsoft store update routine the contents of the hidden Recovery partition are changed.  The partition remains the original size with a total space of 300MB but the Winre.wim file is now larger, approximately 235MB.  Because of this larger Winre.wim file, the total free space in the hidden Recovery partition is reduced to less than 50MB.  As a result, when the user attempts a System Image Backup the backup routine errors out giving the 0x80780119 error - this apparently happens because the system image backup process requires that the hidden Recovery partition have AT LEAST 50MB of free space.  Since the Win8.1 update process increased the size of the Winre.wim file without increasing the size of the Recovery partition, the new W8.1 system does not have enough free space to run an image backup. 

 

3. This situation also causes Backups using Win Server 2012 Essentials to fail for the same reason.  So not only are many installations of Win8.1 affected but so are ALL installations of Microsoft's latest server products.

 

One way to "fix" the problem for future upgrades would be to modify the "update" process so that either the Winre.wim file is kept to a size that does not cause the total remaining free space in the hidden Recovery partition to shrink to less than 50MB OR to increase the size of the hidden Recovery partition to a size that allows for an adequate amount of free space no matter what size the update process makes the Winre.wim file.

 

However, that doesn't help any of us who have already updated from Win8 to Win8.1 and can no longer properly backup our system files.  For all of us, Microsoft needs to find another solution and do it promptly.
