  1. Delete only those %Windir%/$NtUninstallKB number$ folders that were created more than a month ago as backup files for Windows updates. Do not delete those that were created within the last 30 days.

    • setup*.log
    • setup*.old
    • setuplog.txt
    • winnt32.log
    • set*.tmp

I asked the clarification question for very specific reasons (the architecture of your OS question.)  It matters.

You have 32-bit Windows.  It only supports up to 4GB of memory (and only makes up to 4GB of memory available to anything running under said 32-bit Windows OS.)   The "up to" is explained by the way your hardware (video card is usually a large chunk) also needs that memory for some processes.  Usually a 32-bit Windows installation in a system with 4GB or more physical RAM will show between 2.89GB and 3.75GB available to the system to use.  So if you have 16GB of physical memory - you still only have access to between 2.89GB and 3.75GB of it through the 32-bit OS you have chosen to run.  The variance, again, depends on factors including your hardware.
 
If you want to use the memory between 4GB and 8GB, even in a virtual machine, you will need to install 64-bit Windows.

You could go deeper into this, figure out exactly why your video device (and other components) use up some of the memory up to 4GB in your 32-bit OS - but since you are not (as far as I can tell) using a Macintosh and your system is most likely 64-bit capable... your best bet is to get a 64-bit architecture version of Windows installed and actually take advantage of the hardware you have purchased.
