Another example of using the Win32 device namespace is using the CreateFile function with "\\.\PhysicalDiskX" (where X is a valid integer value) or "\\.\CdRomX". You do not describe symptoms consistent with a need to implement KB307545.

Suggesting KB307545 as a first troubleshooting step is akin to saying "I have no idea what the problem is or how to fix it", especially and even worse when you don't have any of the symptoms.

My system does not use dmboot.sys, but obviously yours does (or would like to).  If yours is afflicted, you can usually replace it using backup copies already on your system, so I look into that first.

It is too bad the MS Answers forums does not prompt for any system information when a new question is asked so we know absolutely nothing about your system so here is what I would do...

Since you can get into Safe Mode, do this:

cmd

Click OK to open a command prompt window.

The dmboot.sys file should be in this folder (assuming XP is installed on your C drive):

c:\windows\system32\drivers

Depending on your installation, enter this command:

copy c:\windows\system32\dllcache\dmboot.sys   c:\windows\system32\drivers

Respond in the affirmative if asked to overwrite the existing file and then you should see a message that 1 file(s) was copied.

If the file was not found in the dllcache folder (it really should be), enter this command:

copy c:\windows\ServicePackFiles\i386\dmboot.sys   c:\windows\system32\drivers

If it still is not found, then I can upload one to my SkyDrive and you can download it and replace the one on your system.

After replacing the file, reboot and see how things look.
