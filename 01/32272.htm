WHEA_UNCORRECTABLE_ERROR (124) bug check.

A fatal hardware error has occurred. This fatal error displays data from the Windows Hardware Error Architecture (WHEA).

If we run an !errrec on the 2nd parameter of the bugcheck (address of the WER structure) we get the following:

----------------------

There is only so much you can do with a bugcheck like this until it comes down to a faulty processor that will need to be replaced. Start from 1 and work downward:

1. Ensure your temperatures are within standard and nothing's overheating. You can use a program such as Speccy if you'd like to monitor temps - http://www.piriform.com/speccy

2. Clear your CMOS (or load optimized BIOS defaults) to ensure there's no improper BIOS setting - http://pcsupport.about.com/od/fixtheproblem/tp/clearcmos.htm

3. Ensure your BIOS is up to date.

4. The only software conflict that can usually cause *124 bugchecks are OS to BIOS utilities from manufacturer's like Asus' AI Suite. If you have something like this software-wise, remove it ASAP.

5. If all of the above fail, the only left to do is replace your processor as it is faulty.

a. user say:

Not sure what feature you're referring to, but here's something to keep in mind:

Once a software developer stops supporting their application, it's just a matter of time before it will stop working properly on Windows 10. That 'matter of time' may be over quickly or it may last for years.

When an application loses support it's no longer being updated to keep up with the changes to Windows 10. As long as changes to Windows don't impact the application, you're good. But eventually, we're all going to have to replace our existing software with Windows 10-compatible versions.

To make matters more complicated - because, why not? - since Microsoft claims that '10' is the last edition of Windows, it won't be enough to say that an application is supported on Windows 10. Developers will have to say which version of Windows 10 they'll support. They might say, for example, that they'll support up to version 1607 but later versions of Windows 10 won't be supported.

Compatibility mode can extend the life of older software, but it won't keep it alive forever. And software installed in compatibility mode may not be able to take advantage of future Windows technologies.

And the above also applies to hardware, which requires driver support.

What is "Digital Rights Management"?

Digital Rights Management (DRM) technology is used by content providers such as Xbox LIVE Marketplace to control how game and video files are used and distributed. Xbox LIVE Marketplace sells and rents DRM-protected videos, games, and add-in content that are covered by media usage rights that allow specific uses of that content.

When you purchase content, you receive a license to use that content. The license has two parts: the system license and the local license. The system license applies to the gamertag that purchases the content, and the local license applies to the console where you first downloaded the content

Can I transfer a content license from one Xbox 360 console to another?

Yes, you can transfer content licenses from one console to another once every four months.

If your console was recently repaired or if you’ve purchased a new console, use the license transfer option on your console to transfer content licenses. Here’s how:

1.On your console, sign in to Xbox LIVE using the gamertag that you used to purchase the content.

•Need to find your Microsoft account? See Find your Microsoft account.

•Can't remember the password for your Microsoft account? See Recover your Microsoft account password.

2.Go to Settings and select Account.

3.Go to Your Billing Options and select License Transfer.

4.Follow the instructions to transfer the content licenses.

After the licenses are transferred, you need to download the content again to update the local licenses. Then the content can be used by other profiles on your console and when you’re not connected to Xbox LIVE.

For information about how to download previously purchased content, see Buy and redownload content from Xbox LIVE

What you are describing seems to be an issue with the display driver.  It is likely that the installation process could not find a suitable display adapter driver for you display adapter hardware.  You may be using the "Microsoft Basic Display Adapter Driver" which is an "if all else fails" driver.  Remember that Microsoft is not responsible for providing updated display drivers for the thousands of possible display adapters out there in the ecosystem. 

I suggest that you determine which display adapter you have in your machine, check with the manufacturer's web site for the latest driver, download it and unpack it on your computer, then use Device Manager to Update the display adapter driver, pointing the process at the folder to which you downloaded the driver.

You may not be able to find an explicit Windows 8 driver yet.  If so, download a Windows 7 driver. Most likely that will work OK.  If not, set the driver files and/or installer to run in compatibility mode for Windows 7 (or earlier if need be), then do the driver upgrade.

Note that since your machine is 2009 vintage, presumably your display adapter is of similar vintage.  That makes it "not modern" in computing terms and it is quite possible your display adapter manufacturer will not bother building a specific Windows 8 driver for your adapter.  Unfortunately its fairly common practice to not provide new OS drivers for anything but the latest hardware.

This process worked well for me on a laptop much older than yours, and it has done for many others. If you need any assistance with any of these steps, let us know and we'll help out there.
