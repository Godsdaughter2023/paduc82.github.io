Hi,

To add to Swathi's excellent advice.

BCC a is :

Cause

This bug check is issued if paged memory (or invalid memory) is accessed when the IRQL is too high.

The error that generates this bug check usually occurs after the installation of a faulty device driver,
system service, or BIOS.

If you encounter bug check 0xA while upgrading to a later version of Windows, this error might be caused
by a device driver, a system service, a virus scanner, or a backup tool that is incompatible with the new
version.

Resolving the Problem

If a kernel debugger is available, obtain a stack trace.

To resolve an error caused by a faulty device driver, system service, or BIOS

1.Restart your computer.

2.Press F8 at the character-based menu that displays the operating system choices.

3.Select the Last Known Good Configuration option from the Windows Advanced Options menu. This option
is most effective when only one driver or service is added at a time.

To resolve an error caused by an incompatible device driver, system service, virus scanner, or backup tool

1.Check the System Log in Event Viewer for error messages that might identify the device or driver that
caused the error.

2.Try disabling memory caching of the BIOS.

3.Run the hardware diagnostics supplied by the system manufacturer, especially the memory scanner. For
details on these procedures, see the owner's manual for your computer.

4.Make sure the latest Service Pack is installed.

5.If your system has small computer system interface (SCSI) adapters, contact the adapter manufacturer to
obtain updated Windows drivers. Try disabling sync negotiation in the SCSI BIOS, checking the cabling and
the SCSI IDs of each device, and confirming proper termination.

6.For integrated device electronics (IDE) devices, define the onboard IDE port as Primary only. Also, check
each IDE device for the proper master/subordinate/stand-alone setting. Try removing all IDE devices except
for hard disks.

By now, you already know Windows’ Device Manager (by default) doesn’t display ghosted devices. To show hidden devices in Device Manager, follow the simple steps below.

rob koch say:

That is a fundamental issue for non-technical users and also why it's now common for the ISP, such as cable or phone companies, to provide a wireless router as an optional service.  This leaves the operation and maintenance of the router to the technical support of the ISP rather than the individual consumer, who often isn't capable of or interested in such technical details.  However, this often includes additional cost to the consumer, in some cases even an additional monthly charge they wouldn't need to pay if they owned the router themselves.

I can't answer precisely which direction you should choose, since that's up to your own willingness to attempt the various steps required or to pay for the technical support or a replacement device.

However, since the options that don't require you to perform the firmware upgrade yourself all have a cost, the answer is a relatively simple trade-off of cost versus other actions.  If you'd rather not take the time or risk that your current router fails, then replacing the router might be the quickest method, since then you could simply install and configure the new router with a changed password, which is also more commonly an assumed part of the initial setup nowadays.

If instead you'd like to save the money if possible, then you could attempt the firmware upgrade, but also be prepared to replace the router if the process fails.  As long as you realize that you may potentially lose Internet access if the firmware upgrade fails and are able to travel to a nearby consumer electronics store that carries these in order to replace the device, the loss of the device is no worse than simply replacing the router in the first place.  Since buying a new router locally may cost more and require travel, that's the trade-off for taking the risk that you'll need to replace it quickly if the firmware update fails.

The fact that your specific router is even still supported with a current firmware upgrade to fix the problem is a good thing.  In my case a router a few years old was no longer supported and required replacement as the only option, but that meant the new device was also much more capable and stable than the old device.  I'd been having occasional issues with unstable connections to the Internet for the previous year anyway, so the roughly $35 I paid for a new router online was worth the money.

You can pay anywhere from $20 or $30 online to hundreds either online or in a store, but the typical consumer doesn't need anything costing more than $50.  The features of the more expensive devices will be the norm in a few years, so the routers you'd buy then for less than $50 will be better than a $250 device today.

This is just the nature of electronic devices, not some big conspiracy, since technology is always changing and getting better for lower cost.  This means that you may find yourself in a similar situation in another several years, so getting riled up about it or spending too much now will do no good at all.

If you end up replacing the device, get the model number off the old router to take with you to the store.  That will allow them to determine exactly what you need to replace it and not overbuy capabilities you really don't need.  The cheapest router isn't necessarily the best, but I'd be suspicious if the salesperson attempts to get you to pay more than $50 for a basic consumer model.

Rob

marc cai say:

Hello,

The situation is a bit interesting. But we need clarification, do you have multiple Windows 10 system in a single rig? If you're only using a single Windows 10 in a single computer and after an update, you're being shown with 3 operating system options, it is possible that the update you received is the major update 1803, and the older version of Windows was kept automatically by the BIOS/UEFI. You will not be able to boot with the older Windows 10 option(the first Windows 10 option on your end). You're correct that this is not common. And yes, it is an issue. I'm afraid I will not be able to tell which triggered the issue. But on my own theory, this can be due to corrupted update files caused by a program conflict; Example: a third party antivirus that was running in the background when the system was updating. 

The solutions you performed were helpful, and should have addressed the issue already. We have a few additional ways that you can try in resolving it:

A. Clean up system files by perform Disk Cleanup.

  1. In Disk cleanup, select Clean up system files.
  2. Select the file types to get rid of. To get a description of the file type, select it.
  3. Select OK.

B. Delete the not working OS and set the current as default.

Given that the Windows 10 (second option, third line) is working correctly, this OS shall be used to delete the failing OS: Go to msconfig.msc > Boot tab. In this section, you will see all the available OS on your computer. The one with the label Current OS, is the one currently being used. The other one(s) must be for your multiple boot setup, and in our case, one of it is the non-functional Windows 10. High light it and click Delete.

If the methods did not work, please  let us know, with details that you think can help and we will continue. 

Kind regards.

davem121 say:
Hi NT3285, if that app is stalling your PC's Shutdown

Click your Start Button, type regedit and hit Enter - this will open the Registry Editor

In the left pand, navigate to:
HKEY_LOCAL_MACHINE - SYSTEM - CurrentControlSet - Control

Then in the Right Pane double click WaitToKillServiceTimeout and change the value to 1000

Click OK

Then back in the left Pane, close up all those folders (keys) again (click in the left pane and press left arrow on your keyboard a few times)

In the left pand, navigate to:
HKEY_CURRENT_USER - Control Panel - Desktop

Then in the right pane, right click in the blank space and choose New - String Value

Name the String Value WaitToKillServiceTimeout

Then double click WaitToKillServiceTimeout

Set its value to 1000

Save and close the Registry Editor

That's It!
