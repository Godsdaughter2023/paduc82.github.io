Ok , I got the problem FIXED, yes FIXED

I dont know what caused the animation to not run in the first place, but I got the two computers I have here to run the animation now.

I read some place that the video  buffer... if it does not have enough memory assigned to it, it  wont play the animation.

I also read in that same post some place,  that the person didn't know what they were talking about. but not so.

I have three computers here all the same, they are Older Dell GX 270s, the bigger desk top ones.

One computer was displaying the animation but two were not. All had the same updates and up dated the same day.

after two days of solid trouble shooting and trying everything, I checked the video buffer memory in the BIOS of the computer that did run the animation, that  computer had the video buffer set at  8 MB,

On my computers, I have two choices in the BIOS, 1 MB or 8 MB for the video buffer. I went to check the other two computers that refused to run the animation and found those were set at 1 MB. I changed one computer so it had 8 MB available in the Video buffer and rebooted. The animation worked perfectly. So I tried setting it back to 1 MB and it would not run the W7 animation but would run the Vista type.  I then re set the BIOS for 8 MB for the video bufferand the W7 animation with the colored lights came back on perfectly.

So then I went to change the second computer so it had 8 MB in the video buffer too, and did that and I rebooted it but sadly noting was changed... I still got the vista style screen.. ok now what..? Well even though I was changing the buffer MB value on the BIOS screen, I wasn't sure if the change was actually taking place.

I wanted to try to re flash the BIOS in this second computer to make sure it was working properly and no bad files,  but the program I found from Dell won't let me re flash with the same BIOS (A03) so I looked up the last updated version for the computer which is A07.

I was a bit nervous about doing this but I thought I would try it. I re flashed the BIOS and now the animation works perfectly again ! So I fixed both computers.

I don't know why the failure in the first place but I found a fix.

You cannot.  The native email client is for tablets and is just like smart phone apps (you can't import a PST file into your iPhone or Android either).

forum foks,

on a windows 8.1 machine, what is the difference between these two commands?

netsh int ip reset

&

netsh winsock reset

after reading a couple short articles on the web i get the impression the commands do basically the same thing.  is that correct?

thanks!

troy

a. user say:

Hi Troy,

Thank you for posting your query to us.


To answer your question,

netsh int ip reset is a command in windows to reset the TCP/IP. TCP/IP stands for transmission control protocol/Internet Protocol, it is a type of protocol used to communicate with other computers. If TCP/IP is not functioning then you would not be able to browse, even though if your computer is able to ping other Ip addresses and still if you are not able to pull up any webpages then reset the TCP/IP by using the above command.

netsh winsock reset is a command in windows to recover the computer from any socket errors which arise when you download some unknown file, or due to some malicious script on the computer.

Winsock settings contain your computer's configuration for Internet connectivity. Winsock errors can occur when the settings get corrupted, usually because of spyware or Web browser hijacking. If you are having trouble opening websites, it could be due to corrupted Winsock settings.
