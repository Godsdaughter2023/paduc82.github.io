ccurts say:

Look in Control Panel, Administrative Tools, Component Services, look at the very bottom of the tree where it says Services.  Display the long list of services.  Sort services alphabetically.  Look for one service, double click on it, go to the Dependencies Tab (far right), and look in the bottom box.  It will show you what other service in the list is required to be working or 'started' before the audio one you want will work.   

 

IF you need to start another service, then go to that needed service and look on the General Tab (far left) in the Startup field.  If it is disabled, use the field's drop down menu to select manual or automatic.  (always try manual first.)  If it is already set on manual, then change it to automatic.  Click Apply, then Ok.

 

Now you are ready to do the same verification for the second audio service.

 

CAUTION:  This is like doing surgery, so take great care when working in Component Services.  One mistake can cause more problems, even serious problems.

 

If  you continue having problems, consider going to the BlackIce website to see the default values for all the services for all the Windows operating systems (including Windows 8 now). 

It might be best to start there to see what services you have that are not set to the correct startup values, and you may fix your problem automatically by fixing all the service settings.

 

Hope this helps.

cverrier say:

I've been running Windows 8 (RTM) on a TouchSmart 300-1025 for some time (I wanted a wall-mounted touchscreen 'kitchen computer' with no mouse or keyboard to clutter the place up).

 

First the NextWindow touchscreen hardware used on these older machines is infrared not capacitive, and only supports two simultaneous touch points as a time.  This means it doesn't meet the minimum standards demanded for full Windows 8 functionality.   It'll work, but you won't get the optimum experience.

 

The Win7 TouchSmarts have a raised plastic bezel around the screen (for the optical detectors), which makes it hard for Windows to detect the 'swipe' motion needed to open the Charms bar - you really have to get a fingernail right into the edge of the screen to 'grab' the charms bar.   On a proper Win8 touchscreen, the glass (and the touch sensitivity) extends beyond the edge of the display area (like an iPad) - so it's much less hassle.

 

There's a range of Windows 7 drivers on the HP support site for things like the volume control, but the standard installer package does a test of your Windows version and refuses to run when it detects Windows 8.   I usually managed to overcome this by using a utility to unpack HP's self-extracting ZIPs into a folder, and then manually browsing to the drivers files from inside Device manager.

 

I managed to get the volume controls working, but it was intermittent.  Some apps detected them, and some didn't (Windows volume ignores the buttons, as do browser apps like iPlayer).    Windows Media Center works with them, and a few others, but it's very twitchy.  Device Manager on my machine is showing everything operational.

 

There's a registry key that defines the application that runs when you press the 'House' button (default is the HP 'SmartCenter.EXE' application)

 

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\ControlSet

\Control\MobilePC\HotStartButtons\1\ApplicationPath

 

You can change the regkey to make it run something else but I think it only does EXE files, which might be a limitation for some situations.

 

I still have an outstanding issue with the TouchSmart bluescreening when it resumes from sleep.  Cold boot is fine, but resume always fails.
