ferdil say:

So I found a fix for this - The issue here is a Windows 10 machine that I log onto using a PIN, and trying to connect to shares on an Ubuntu/Linux SAMBA server

Go to Control Panel / Settings and search the Control Panel (top right) for "Credentials", click on the "Manage Windows Credentials" link

Then Add a Windows credential for the machine you are trying to connect to as follows:

Internet or network address: MACHINENAME

Username: WORKGROUP\USERNAME

Password: PASSWORD

replacing MACHINENAME, WORKGROUP etc. with your own values.

NB - For Username you must use the WORKGROUP or DOMAIN name of the machine you are trying to connect to (an Ubuntu Linux Samba server in my case) followed by the backslash followed by the username

Once you have done this, you can connect to \\MACHINENAME\SHARENAME using a mapped network drive or adding a "network location" to the share you are trying to connect to. I'm not sure if one is able to browse the server, but at least I can connect to my shared data now.

Hope this saves someone some frustrating time when trying to connect from Windows 10 to a Linux Samba server.

alison hamilton say:

I can better the answer now, the cause of the authentication issues is actually Windows 10 fast startup feature (hibernate boot or what ever you want to call it).

Windows 10 when you select shutdown doesn't actually shutdown, it hibernates the workstation by default. There are guides online suggesting that holding the shift key when selecting shutdown will perform a full shutdown or editing some registry keys will change the behavior of shutdown also, see this discussion for some more details: 

https://serverfault.com/questions/793295/how-to-disable-fast-startup-using-a-group-policy

I realized this was the issue as thinking about it, if I came into work in the morning, turn on my device I am prompted for credentials, if I make no changes and just restart the device suddenly outlook is working fine no more prompts for credentials. This is because "restart" actually performs a complete shutdown, killing all services, running tasks, network connections (the critical part to all this I'm assuming).

Hope the above helps someone else who may be facing this infuriating problem. Thanks Microsoft for implementing OS changes that actually break all your own stuff. Dumb.
