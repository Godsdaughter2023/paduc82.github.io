a. user say:

Why don't you take a look in Task Scheduler at the Windows Backup Tasks.

Those tasks are set up to run on the schedule you define and they are also configured to run "as soon as possible" if the task is missed.  What than means is if you have a task set to run a backup at a certain day/time and the task is missed for some reason (like the system is turned off) the next time Windows starts it will realize the task was missed and then Windows will run the task "as soon as possible" which will probably be the next time your system is running again:

From Backup and Restore if you choose to "Turn off schedule" that should disable the three Scheduled Tasks associated with Windows Backup but maybe yours isn't for some reason or maybe the installation of Norton has somehow mixed things up.  So check it our yourself.

Click the start button/orb and in the box enter Task Scheduler and choose to Run as Administrator:

Then navigate to Task Scheduler Library, Microsoft, Windows, and in the bottom of the list choose WindowsBackup and see if the tasks are Disabled and if they are not, right click and Disable all three Tasks:

Like this:

You might also check the Last Run Time column to see if that looks like the date/time when you know the Windows backup was running.

Restart your system and check the tasks again to be sure the changes "stick".

If sometime later you see or think the Windows backup is running again go back and look at the tasks again to see if somehow they got enabled again.
