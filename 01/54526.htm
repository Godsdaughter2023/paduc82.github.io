The atl100.dll is part of the Microsoft Visual Studio (ATL=Active Template Library) and program developers that code in C++ need to have a matching version of the redistributable files (DLL=Dynamic Linked Libraries) installed on the system where their programs are going to run.

The files are not part of a standard Windows installation so no amount of Safe Mode, Clean Boot, System Repair or sfc /scannow is going to help (sorry Microsoft engaged Support Engineer "experts").

There are many versions of the C++ Redistributable package so depending on what those programs are you may need to have more than one version of the package installed.

Since you can't tell what version of Visual C++ different companies might be using, it is possible (and likely) that you are going to need different versions of the C++ Redistribuable library files installed on your system at the same time so if you look in your Add/Remove programs you might see several versions installed:

You can check your Add/Remove programs to see what you have installed now.

People often ask if they can uninstall the old versions and the answer is "maybe" since nobody but you knows what other programs you have installed that might be using the libraries from those old versions and if you uninstall them those program will not work anymore.

It is probably a good idea to have at least the latest version installed so visit this page and get at least the 2015 version installed (that seems to be the latest right now):

https://www.microsoft.com/en-us/download/details.aspx?id=48145

I have had this exact problem, but with launching programs from external storage on my Surface Pro 5 (SD, USB, etc.) After a lot of trial and error, I found a permanent fix (for me, at least).

Launch msconfig and select the "Services" tab. Uncheck "Hide all Microsoft services" if it was selected, and scroll down to "MultiPoint Service."

Disable "MultiPoint Service," select "Apply" then "OK," and reboot. As soon as my Surface rebooted, the error went away and everything worked perfectly.
