pozop say:

Solved this problem using Mark Russinovich's excellent Process Monitor.

It is an access denied issue and I traced it to the temp folder at C:\Users\username\AppData\Local\Temp.

To fix I granted the ComputerName\Users group change permission to the entire sub-directory starting at C:\Users\username\AppData\Local. This took a while to write the permissions and to some sub-directories I didn't have authority to change but I just skipped those.

Note that you can probably just change the specific permissions to the folder path but it was easier just to do the entire sub-directory. Note also that this will allow any user of the system access to these directories but if the system is single user anyway IMHO it doesn't much matter.

Once the changes were made, clicking "try again" in the error dialog box successfully wrote the file to the destination - which incidentally was a USB drive and had nothing to do with the temp directory.

If this doesn't work for you try adding the EVERYONE group to the same permissions.
