¡firedog say:
So I guess there are 2 very different but very annoying problems ...

 

 1 ) Not being able to reply to web sites that use it as their email service


It's not the website that decides what email service to use - this is an operating system (Windows)function. When you click on a Contact us link on a web page, your browser calls the Windows mailto : protocol. Windows then has to find an email client program on your computer to compose and send an email message.

You may have more than one client: every Windows XP system has Outlook Express built in, for example, while Vista has Windows Mail. There is no built-in client on Windows 7, which may be why you have Windows Live Mail. Many editions of Microsoft Office include Outlook, a more complicated mail client program particularly designed for business users.

One of the installed mail clients has to be designated as the default so that Windows known which one to invoke when it's asked to, e.g. by the mailto : protocol. If there is only one client, then this will of course be the default.

To find out why the message won't send, we'll have to see the error message (that's what it's for). You can click on it to select (highlight) it, then use right-click to copy it and paste it into your reply.

 2 ) Preventing Windows Live Mail from opening and copying all the emails in my Yahoo inbox to Windows Live Mail


When you set up an account in WLMail to access a Yahoo mailbox using POP3, the program will try to download all new messages in the Yahoo inbox. New messages are those that it hasn't downloaded before, so for a fresh WLMail account, this will mean all messages in the inbox. 

If you don't want this to happen, open your mailbox in Yahoo webmail and move the old messages to user-defined folders. While you're at it, clear out the junk and trash folders. Then WLMail will only download newly-arriving messages, because it only connects to the inbox.

geethu b say:

Hi Lokesh,

 

 

To increase the memory space for a disk partition, there should be an unallocated space available next to it. If you have two partitions for example C drive and D drive and if both the partitions have data in it, you will not be able to extend the C drive. As you do not have any data on D drive, to extend the C drive you may delete the D drive volume. So that you get a free space next to C drive, after that you will be able to extend the volume of C drive.

 

 

Note: Deleting the volume will erase all data on it. Backup any data you want to keep before deleting.

 

 

 

To increase the space of the C drive, you may follow these steps:

a. Press Windows key + R on the keyboard.

b. Type “diskmgmt.msc” without quotes and press Enter.

c. Right-click on the C drive and click on Extend Volume.

d. Click on Next and select the amount of space you would like to add to the C drive.

e. Click on Finish.

 

 

If you have any free space left after extending the volume, follow these steps to assign a drive letter: (For example D drive)

a. Right-click on the Free space and select New simple volume.

b. Click on Next.

c. Specify the volume size.

d. Assign a drive letter or path.

e. Click on Next and click Finish.

 

 

Hope this information helps. Reply to the post with updated status of the issue to assist you further.

"It's not a true crash, in the sense that the Blue Screen was initiated only because the combination of video driver and video hardware was being unresponsive, and not because of any synchronous processing exception".

Since Vista, the "Timeout Detection and Recovery" (TDR) components of the OS video subsystem have been capable of doing some truly impressive things to try to recover from issues which would have caused earlier OS's like XP to crash.

As a last resort, the TDR subsystem sends the video driver a "please restart yourself now!" command and waits a few seconds.
