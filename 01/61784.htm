cmdrkeene say:
In reply to deleted message

I'm sorry, that was for an older version of the defrag UI.  Windows 8 has the new "maintenance window" timeframe where it does all maintenance tasks (updates, cleanup, defrag, problem reports, and about 8 other things).  That schedule can be set in the action center, but it only lets you set the time of day (default is 3:00 AM local time). If the computer is off or in-use, it will do it the next time the system is on but idle.

You can open Task Scheduler to find and see the defrag and maintenance tasks, but you'll note that the trigger (where time/day would be) is blank.  This is because the tasks were scheduled with new abilities in the AT scheduling engine that aren't in the Scheduled Task UI interface yet, so it cannot show the actual schedule being used.

I think perhaps the easiest workaround, instead of trying to get those AT commands at a command prompt, would be to just disable the schedule in the defrag UI, then schedule your own defrag using task scheduler directly.   Just set up a scheduled task that executes this command:

defrag.exe /c

Assign it the schedule days/times you want in the Task Scheduler.
