Please be informed that when you use Microsoft Edge to download a file from the Internet, it is saved to the default downloads folder. The default downloads folder in Microsoft Edge is your %UserProfile%/Downloads folder.

Currently, Microsoft Edge does not include a setting to change the default downloads folder.

However, a work around would be to change the default download location from within the File Explorer.

1. Launch File Explorer and locate the Downloads folder on the left.

2. Right-click on the Download folder and select Properties.

3. Click the Location tab, and enter a new path. Click Apply.

This creates a new download location. If you want to move the existing one (taking any completed downloads with it), click on Move, choose a new folder location and click Yes.

wunders say:

It is the job of Windows Explorer to navigate the disks, make sense of what's there, and make sure everything goes where it belongs.  Should a bad spot in the disk appear where Explorer needs to read and Explorer reads erroneous data, it may try to make sense of it and act on it or it may just throw up its arms and give up.  This is probably what's happening when Windows Explorer crashes with the error.  Chkdsk also does its best to recover when it discovers a bad spot on the disk (that's why it takes so long), but often it makes things worse than before if it doesn't interpret the charred remains correctly.  My guess is that your system runs just fine as long as you stay away from that bad area on the disk.

So the next step is to ask why the bad spot in the disk appeared.  If your system shuts itself down instantly and without warning while it is trying to write data to the disk, that could easily cause the problem.  On the other hand, loading bad data from a disk and executing it can crash it too. The two most common causes of instant and random shutdowns are overtemperature and bad RAM.  To address overtemp, there are some programs that you can download that will monitor your CPU temperature just like the thermal circuit does.  I found the freeware "Core Temp" helpful for this. <http://www.alcpu.com/CoreTemp/ > .  If you look around, you can also find some RAM test programs.  The problem is that RAM tests also tend to put a lot of stress on the CPU which again can cause overheating -- but probably worth trying.  If your machine can run a RAM test without shutting down, you probably don't have a thermal problem.  Microsoft has a RAM diagnostic procedure here
  "Hardware Malfunction Results in System Error"  <http://support.microsoft.com/kb/315223 >

The other cause of a bad spot in a disk is a bad disk itself.  I once dealt with a disk failure mechanism on a laptop where I would re-install Windows on the drive and it would work fine for the rest of the day.  Leave it sit overnight and it wouldn't boot the next morning.  I reinstalled Windows over a dozen times before I realized the problem was the disk drive (sometimes I'm a bit thick-headed).  Disk tests do a good job of reading and writing in various patterns on a disk drive but are almost always done over a short period of time and what I had was a problem where it would "forget" when sitting overnight.  Replacing the disk drive ended up solving that problem for good.

Pick your path and good luck, let us know if you solve it.

HTH,
  JW
