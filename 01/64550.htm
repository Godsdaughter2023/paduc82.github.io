malkeleah say: As you have discovered, you cannot change a user account by simply changing the name. That will produce a cosmetic change only and none of the actual profile files/folders will be changed. Follow the instructions below to fix things to your liking and to set up user accounts properly in Vista.

1. Create the new user account (named as you wish) as a *Standard* account (not administrator) and at least one extra account with administrative privileges. Call the extra account "CompAdmin", "Tech", or the like.

2. Log into the new user account once so the user profile folders are created and then log out and then log into CompAdmin.

3. Go to Control Panel>System>Advanced>User Profiles>Settings

4. You will see where you can copy user accounts. Copy the old misnamed account to the new correctly-named one from there. Use the Browse to select your new account (C:\Users\newaccount). If you don't want to copy all the settings, then instead just navigate using Windows Explorer to the old user's Documents, Music, etc. and copy the data to the new user's folders.

5. Log into your new correctly-named user account. When you are sure you have everything you need, you can delete the misnamed user account from Control Panel>User Accounts if you like.

General information about setting up users in Vista:

You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista is a multi-user operating system with built-in system accounts such as Administrator, Default, All Users, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it will require third-party tools and working outside the operating system.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. If your regular user account is an administrator, after you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

Steps to perform SFC scan:

 

1. Click the Start button

2. From the Start Menu, Click All Programs followed by Accessories

3. In the Accessories menu, Right Click on the Command Prompt option

4. From the drop down menu that appears, Click on the 'Run as administrator' option

5. If you have the User Account Control (UAC) enabled, you will be asked for authorization prior to the command prompt opening. You may simply need to press the Continue button if you are the administrator or insert the administrator password etc.

6. In the Command Prompt window, type: sfc /scannow and then press Enter

7. A message will appear stating that 'The system scan will begin'

8. Be patient because the scan may take some time

9. If any files require replacing, SFC will replace them. You may be asked to insert your Windows 7 for this process to continue

10. If everything is okay you should, after the scan, see the following message "Windows resource protection did not find any integrity violations"

11. After the scan has completed, close the command prompt window, restart the computer and check.

How to use the System File Checker tool to troubleshoot missing or corrupted system files on Windows Vista or on Windows 7 http://support.microsoft.com/kb/929833

  6. In the left pane expand the sub-folder (Key) marked Windows NT
  7. In the left pane expand the sub-folder (Key) marked CurrentVersion
  8. In the left pane expand the sub-folder (Key) marked WinSAT
  9. Then, once you are in the folder (Key) Winsat, on the right hand side look for the v

    jimwynne say:

    You have done two risky things:  You've apparently used the "Live" file system (aka packet-writing, aka UDF file system) to use an optical disk like a floppy or flash drive and you've used a double-layer disc, the performance of which may be unpredictable in Windows.   Double whammy.   Using an optical disc as a flash drive is risky because it makes an optical disc do things it wasn't designed to do.  If you're going to use optical disks for backups you should use the standard burning method and finalize the disks each time.  This means that if you do regular backups, you will use a lot of discs because once finalized they're not reusable.  It's much better to use a flash drive or external hard drive to do backups. 

    Having said that, you can download the free version of ISOBuster which can help you to (a) confirm that the files are present on the disc(s) and (b) recover them.
