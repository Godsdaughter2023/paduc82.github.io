eric ber say:

Hi Matt,

Conflicts with the power plan settings is one of the possible reasons why your laptop is shutting down once the charger is removed. A power plan is a collection of hardware and system settings that manages how your computer uses power. Resetting the power plan to default sometimes resolve the issue. To reset power plan to defaults, kindly follow the steps below:

  1. Press the Windows key, and type command prompt, and press Enter.
  2. Copy and paste the command below into the command prompt, and press Enter.

    powercfg -restoredefaultschemes.

  3. When finished, you can close the command prompt.

Hello benz0989,

The error code 0x80070005 refers to access denied. Let's try the steps below to take ownership of the folder that's being denied.

  1. Start your PC in safe mode. Please refer to this link.
  2. Click the Start button > type cmd > right-click on it then select Run as administrator from the context menu.
  3. Type " takeown /f "C:\Program Files\WindowsApps" /r /d Y" (the Y represents the first letter for 'yes', replace it with the first letter of the word yes in your language) and press Enter.
  4. Type " icacls "C:\Program Files\WindowsApps" /grant "%USERDOMAIN%\%USERNAME%":(F) /t " and press Enter.
  5. Type " rd /s "C:\Program Files\Windows Apps " if you want to delete the folder or either "rename "C:\Program Files\Windows Apps" "C:\Program Files\Windows Apps(old)" " if you prefer to rename it.
  6. Reboot your PC, then try to perform System Restore.

particularly do this) so install a driver - reboot - check to be sure it is installed and repeat as needed.

Repeat at Device Makers - BTW at Device Makers DO NOT RUN THEIR SCANNER - check

I suspect that you haven't told us the whole story, because the error you're seeing indicates that the hard drive on which your store folder lives is a different one from the one it was originally located on, or that you have recently changed the storage manager that controls the drive. Is this the case?

If so, then the problem you're facing has nothing to do with calendars specifically. It may affect any application on your computer that uses ESENT databases stored on the same drive. This includes Windows Search, Windows Update, Event Viewer and several others. Windows Live Mail reports it as a calendar error only because the calendar database is the first one that the program tries to load. The message database and possibly the contacts databases will be equally affected, but the program never gets as far as trying to load them.

If you have indeed recently installed a new hard drive, most probably a bigger one than before that uses large sector sizes, please see this article for a description of the problem: Microsoft support policy for 4K sector hard drives in Windows

Your best bet is to visit the hardware manufacturer's website to search for possible solutions.

"How to Refresh Windows 8 and Windows 8.1"

  • The PC boots into Windows RE (Recovery Environment).
  • Windows RE scans the hard drive for your data, settings, and apps, and puts them aside (on the same drive).
  • Windows RE installs a fresh copy of Windows.
  • Windows RE restores the data, settings, and apps it has set aside into the newly installed copy of Windows.
  • The PC restarts into the newly installed copy of Windows.

"When you reset your PC, the drive will be completely reset, and all your apps and files will be removed from it.
