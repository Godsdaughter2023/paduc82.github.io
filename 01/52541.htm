PrtScr - aka Print Screen - copies what you see on your monitor to the clipboard. Now you can paste it anywhere you like. Suppose you need to show someone what you're seeing on your monitor. Just press PrtScr and paste the image where you need it to be.

Hello Dr. Abdul Rasheed,

Thank you for posting your query on Microsoft Community.

As per the issue description you are unable to see the files saved in different hard drive partition. The issue could be if there is some settings issue or it could be if there is some disk management naming issue.

I would suggest you to try the steps provided below and see if it helps.

Method 1:

Make sure that the drive has a drive letter assigned in Disk Management: (This method applies to Disk Drives, USB flash drives or CD and DVD drives)



If your hard drive, thumbnail drive, memory stick, or flash memory is recognized in Disk Management but not displayed in Windows Explorer, the drive may not have been assigned a drive letter by Windows.

  1. Open Computer Management by pressing Windows key + X, selecting Control Panel from the list, and click System and Security viewed by Category View.
  2. Click Administrative Tools, and then double-clicking Computer Management.

    If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
  3. Look for a drive that does not have an assigned drive letter and is not labeled as System Reserved.

    Note:     The System Reserved drive is reserved by Windows.
  4. Right-click the unlabeled drive and click Change drive letter and paths…
  5. Click Add and then click OK to select the first available drive letter.
  6. If the Change drive letter and paths… option is not available, make sure that the drive is Online.
  7. If the drive is Offline, right-click the disk number and click Online.

For more information on Adding, changing or removing drive letters for Windows 7, click the link below to see the article on the Microsoft website:

Change, add, or remove a drive letter

For more information on Adding, changing or removing drive letters for Windows Vista, click the link below to see the article on the Microsoft website:

Change, add, or remove a drive letter

Method 2:

Make sure that the disk drive is Enabled (This method applies to Disk Drives, and USB flash drives)



If the drive has been Disabled the disk will not be visible in Windows Explorer. To see if the disk is Disabled, follow the steps below to Enable it:


  1. Press Windows key + X and select Device Manager from the list
  2. Locate the Disk drives item in the list and click the small arrow to the right to expand the list of drives
    • If a disk drive is disabled you will see an icon with an arrow pointing down over the icon of the disk drive
    • If you see an icon with an arrow pointing down over the icon of the disk drive, double-click the icon and click the Enable Device button
  3. Click Next on the Enabling a device dialog box, click Next, and then click Finish
  4. Click Close and then close the Device Manager window

    Note: If you don't see your disk drive in the listed drives, the drive may not have been detected by Windows.

    Note: If you receive an error message when Enabling the disk drive, you may have a defective disk drive.

Method 3:

Make sure that the drive is Online If the drive is currently Offline, the disk will not be visible in Windows Explorer. To check the current status of your disks, follow the steps below:

  1. Open Computer Management by pressing Windows key + X, selecting Control Panel from the list. and click System and Security.
  2. Click Administrative Tools, and then double-clicking Computer Management. If you're prompted for an administrator password or confirmation, type the password or provide confirmation.
  3. Look for a drive that does not have an assigned drive letter and that is not labeled as System Reserved.

    Note The System Reserved drive is reserved by Windows.
  4. Right-click the drive and click Change drive letter and paths…Click Add and then click OK to select the first available drive letter.
  5. If the Change drive letter and paths… option is not available, make sure that the drive is Online.
  6. If the drive is Offline, right-click the disk number and click Online.

a. user say:

Hello,

Thank you for your interest in Windows 10.

I suggest you to run System File Checker tool and check.

System File Checker is a utility in Windows that allows users to scan for corruptions in Windows system files and restore corrupted files.

Press Windows key + X and click Command Prompt (Admin)

Type the following command and press Enter

sfc /scannow

Note: Do not close this Command Prompt window until the verification is 100% complete. The scan results will be shown after this process is finished.

If the issue still persists, create a new user account and check.

Press Windows key + R to open RUN box

Type control userpasswords2 and press OK

Under User Account window and under Users tab, click Add tab

Under How will this person sign-in windows, click the option at the bottom Sign in without a Microsoft account (not recommended)

On the next window click Local Account at the bottom

Choose a user name (use password if you want to or you can leave the password section blank as it is just to test)

Click Next and then click Finish

As you click Finish, the previous User Accounts window remains on the screen

Select newly created user and click Properties

Click Group Membership tab

Click to select Administrator and click Other and select Administrator from drop-down menu

Click Apply and then click OK

Log-in to the new user account and check if all the icons including Sound Recorder are showing fine.

When you know they are fine, you can then switch to Microsoft Account.

Hope this information is helpful. Please feel free to reply in case you face any other issues with Windows in future.

Any idea how I can track all currently open files (or windows) - Microsoft Community

ophirkendler say:

Any idea how I can track all currently open files (or windows) in a text file or other sort of log? Something similar to the Applications Tab in TaskManager, but with a path

I am interested in creating a batch-file or executable that will keep track of all currently-open windows (files and applications)
, so that when Windows restart I can more easily get back to all I was doing before.  

This can be done either by logging every file-open operation and removing items from the list when an item (window or file) is closed (only logging at that time)
, or by checking every 5-10 seconds what windows and files are currently open.    

I have tried using TaskList or OpenFiles for this purpose but cannot get it right, plus those also do not provide a path for open files - I would want to keep paths so that items can be recovered (re-opened) in a single click.   Any ideas how to do this?   I am using Windows 7 Professional.

My idea is to have the ability to go back to all I was working on, without having to remember the list...  It is very useful for people who keep many files and windows open much of the time, especially if for any reason Windows crashes or otherwise shuts down (e.g. restarting following an update...).    MS Office products can do it (by saving temp files and letting you recover them if ever the program was not closed correctly), an internet browser can do it (by keeping a history and using session-managers) - why can't Windows do it?  

Furthermore, if such list of open windows and files is maintained then it might be possible to also keep some history and "recently closed files/windows", similar to how Google Chrome can display recently closed tab.  

The application I am trying to develop (for my own personal use, though I would offer it to anyone else) will:
- keep a history and a list of current open files/windows, including a path for each open file.  
- Closed files/windows are removed from the "currently open" list and moved to the "recently closed" list, which has a length-limit.
 - - file/window close operations should be associated with the user's command. Otherwise open files that are closed 
     by Windows during shutdown will no longer be identified as open.   
 - - one option would be to ensure, somehow, that this application is always closed first during a regular shutdown process.
 - - another option is to simply assume that there is no problem if Windows has crashed and that the user is reminded to close the app before shutting down.
 - - another option is to have the list trackable, such that a user can go a few steps back to a point before Windows began shutting down. 
 - - another option is to have the "last session" identified as the one, say 2 minutes before shutdown, while keeping a list of items opened after that time.
- items in "recently closed" list that are too old (have been pushed to the bottom of the list and beyond its length-limit are moved to history. 
- history can be set to limited length or can be cleaned manually or automatically (periodically), or one can define a length limit for the history list. 

* the application/batch/executable is located in the Windows Startup folder and it:
- is started when Windows starts
- upon being started (automatically or manually) will display a dialog with the list of items that were open before windows shut down - see session creation above.
- user can uncheck items from the list if he/she does not wish to open them. Unchecking items from list moves them to the 'recently closed" list or to history.
- upon identifying which items will not be kept, or simply keeping everything, the user clicks a single button and the application opens all items that are still checked.  To avoid slowing down the system, an option is available to open files gradually, in which case a rate can be set.  Setting a priority list is also an idea.
