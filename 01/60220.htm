ravinath p say:

Hi LS_1_890,

 

No, do not turn System Protection on for the System Recovery drive. That is locked and only should be used when you need to do a factory restoration (which reinstalls the operating system). Just keep System Restore turned on for the System partition on which your Windows installation exist.

System protection is a feature that regularly creates and saves information about your computer's system files and settings. System protection also saves previous versions of files that you've modified. It saves these files in restore points, which are created just before significant system events, such as the installation of a program or device driver.

System protection is automatically on for the drive thatWindows is installed on. System protection can only be turned on for drives that are formatted using the NTFS file system.

The other drive that is present in the list will be another partition on the system. You can turn on the system protection for the drive so that the restore points will be created for that drive also.

What is system protection?

keith a. miller say:

Example: the "Documents" folder on my OneDrive

- In Windows File Explorer, when I SHIFT_RIGHT_CLICK directly on this folder, the path is: D:\OneDrive\Documents

- When I pin this very folder to Quick Access, and I SHIFT_RIGHT_CLICK on it in the Quick Access list, the path is indeed the same.


This is quite confusing, UX-wise.

Yes, it can be confusing at first. But if you take a minute to understand the Windows Shell Namespace. Some key point:

"The ultimate root of the namespace hierarchy is the desktop. Immediately below the root are several virtual folders such as My Computer and the Recycle Bin."

Both OneDrive and This PC are children of the Desktop. OneDrive is a link/shortcut to the file system folder D:\OneDrive. This PC is a virtual folder with links/shortcuts to the User's folders, drives, devices, and network locations.

When OneDrive is set to backup Documents, the user's This PC\Documents folder is the same file system folder as the OneDrive\Documents folder.

The behavior of Quick Access is to only display a file sysem path if there is no corresponding namespace path ('Intel' below). But if a namespace path exists, it takes precedence (Music below). Because my Music folder is not synced to OneDrive, they have seperate file system folders, thus I can have seperate Quick Access pins.

In your case, you have two "competing" namespace paths for the same file system folder. And it seems Quick Access gives priority to This PC over OneDrive. That's hard-coded, not much we can do about that.
