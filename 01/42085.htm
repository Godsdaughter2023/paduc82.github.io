Some of the commands that you say work - like dir, set...  are "built in" to the command prompt and don't use your PATH variable so that is why they "work".  There is no dir.exe or set.exe...

Enter 'help' at the command prompt to see the built in commands.

Commands you type in like ping, sfc...  will use the PATH variable.  There are ping.exe and sfc.exe executable file in the system32 folder that will use the PATH variable to locate them.

If from your command prompt, you change directory to your system32 folder, do the commands ping and sfc work?  For example, enter this at your command prompt, then try your commands:

cd \windows\system32   (then the prompt should say c:\windows\system32)

Do ping and sfc work now?  Of course they do.

If you click Start, Run and in the box enter:

msinfo32

Click OK, do you get a message like this:

Can't Collect Information
Cannot access the Windows Management Instrumentation software.  Windows Management files may be moved or missing.

Your PATH statement looks like it should work, but in my system, executing PATH from a command prompt does not contain the variables %SystemRoot%, but instead look like this:

PATH=C:\WINDOWS\system32;C:\WINDOWS;C;\windows\system32\wbem;......

There is some reason that the variable %SystemRoot% doesn't resolve/work but I can't remember why, so assuming XP is installed on your C drive, why not just fix your PATH statement and replace every %SystemRoot% with C:\WINDOWS (or c:\windows) so the first three things in your path have the exact name of the path (like your WindowsPowerShell)

Right click My Computer, Properties, Advanced and then at the bottom, Environment Variables,

In the System variable section, double click 'Path' and change every %SystemRoot% to C:\WINDOWS (or c:\windows).  It looks like you need to change it in three places. 

Be careful with the backslashes and semicolons (they look okay and do not need changing).

Reboot and test.

When you open a command prompt (Start, Run, cmd) , what folder are you in?  I always like to adjust things so I end up in c:\windows\system32 (so lazy am I).
