¡firedog say:
@Troubleat't'mill

Which program to use to open a particular file type is a Windows function, not one belonging to Windows Live Mail. Whenever you try to open a file, the program you're using at the time looks in the registry to find out which program to invoke to open the file. The registry entries are made in various ways: by the user, for example by using Open with in Explorer to specify the default program, or by programs themselves on installation. You may have several programs on your computer capable of opening a particular file type, and which one of those is the one to be used in a particular situation may well vary. 

If .eml files are at some stage being opened in Windows Live Mail and then at some later stage stop being able to be opened in that program, then something is messing up your registry. That could be security software, it could be an update to the operating system (although I've never known this to happen) and it could be another program that's trying to usurp the file association. MS Outlook is well known for aggressively changing registry settings that it ought to leave alone. 

If you right-click on an .eml file and select Open with, you should then be able to browse to the program you want to use as the default for .eml files. Use the browse function to find wlmail.exe in your program files directory and select it as the default.
