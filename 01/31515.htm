    ralf_g say:

I only used EasyBCD to get rid of the opening menu which gave me a chose between loading Win 7 or Vista. This menu was created when I installed Win 7 and kept Vista. Is there something on C that starts Windows even if it is on another drive?

    Thanks again for you patience and support


    There are, I think, 3 considerations here. Firstly, because you had a dual boot situation you now have a boot manager file that tells the computer where to find either OS that you might want to boot into. If you were running Vista on C: drive when you created the dual boot situation then that boot manager file will reside on the C: drive. The second OS can be anywhere else but the computer will look for the boot manager file on C: to tell it where the boot OS is located. Windows 7 keeps its own version of the boot manager in a different location. If you remove or wipe the C:\ drive, the computer won't be able to find either OS. To get a smooth transition the boot manager would have to be moved to the new drive before swapping out the C: hardrive. Apparently the EasyBCD software you were using has the ability to do this for you. https://neosmart.net/wiki/easybcd/basics/changing-the-boot-partition/


    The second consideration is whether your computer BIOS supports booting from a physical harddrive other than C:  and the third consideration is whether your computer BIOS is able to read the data on a GPT partitioned drive before Windows 7/8.x is actually running. 

     

    If the method at the first link doesn't work for you this link has some suggestions for repairing the boot problem. I'm not recommending you buy any software from them, just try the do-it-yourself methods they outline.
