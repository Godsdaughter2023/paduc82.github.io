Non-latin characters in username should be avoided at the creation of - Microsoft Community

w.lin say:

Non-latin characters in username should be avoided at the creation of Microsoft account

Hi, as a traditional Chinese user of Windows 8 and 8.1, I have a little suggestion regarding user naming convention of creation of Microsoft account.

Currently when sign-in with Microsoft account is created, the firstname is used as the account's username, with the user folder named so too. However this causes problem in CJK users, if their Microsoft account has non-latin characters (which should be reasonable).

First, many traditional applications don't handle paths containing non-latin characters well.  Since in this situation %userprofile% contains non-latin characters, erratic behaviors may occur.

Second, even modern apps have problems handling such paths.  In my own experience, at least 2 apps malfunction. For example, PPS, a popular video streaming app in China, fail to start streaming at all.  Where is My Water, on the other hand, can't remember game progress, making the game totally unplayable.  These apps got bad and extreme ratings in regions where non-latin names are prevalent.  Moreover, some bugs are rather inconspicuous;  for example, the persistent login feature of Facebook doesn't work.  There might be more unattributed issues which are caused by improper handling of user path containing non-latin characters.  IMHO requesting each app writer to target and fix the bug would be impractical.

Third, it might cause problem if the user has to authenticate remotely.  Remote access is not always equipped with IME, and encoding is prone to error.

Fourth, a cultural issue:  afaik, many C(JK) users name their accounts with nicknames, which are subjective to change.  If someday they changed their Microsoft account with another nickname, but found out that their user folder still had the old name, it would be a little disappointing.

So, my suggestion is to name the account as the user's registered email address of Microsoft account (minus the @outlook.com part) by default.  Display of user folder can be assisted by string localization (via desktop.ini), just like how the Public folder is handled.  Those had Microsoft account with romanized or western names would not be affected at all, but the user experience (regarding the apps) of those with CJK names would improve greatly.

Thank you.

Weber Lin

The major differences between an m3 and an i5 sp4 are the processor and the cooling systems.
