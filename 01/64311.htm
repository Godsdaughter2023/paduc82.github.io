tryx3 say:

Your Admin account is always required to authorise installations. 

 

If you already had a password on it then that password would still have been required when booting into Safe Mode. 

 

If your other users have standard user-level accounts then they cannot abuse Safe Mode unless they have discovered what your password is or if they know about booting into another environment using a USB/DVD connected to the system.

 

- - - - - - - - - -

 

It sounds like you have enabled the Built-In Admin account [often called Default Admin].  This is not necessary and it is best to keep that account pristine [any user profile can get corrupted in use and you might need to rely on using the Built-In Admin to rescue you one day].  Create a conventional 2nd admin account for normal emergencies - if just one admin account exists then it might be that one that suffers user profile corruption; recovery [though possible & not too difficult to do] can be a bit nerve-racking the first time.  Having a conventional spare Admin account ready means that you can get in using that to sort out whatever has gone wrong.

 

- - - - - - - - - - - -

 

The only extra step you should consider is setting the BIOS to be password-protected.  This needs careful consideration as losing that password would cripple you.  Setting the BIOS password then setting the system to boot from the HDD-only prevents your little darlings from booting from an external DVD/USB and gaining complete control over the system despite whatever you have set up in user accounts. 

 

This precaution will force you to go back to the BIOS & enable booting from a DVD/USB on those occasions when you need to do so [during troubleshooting / repairs]. 

 

The BIOS passwords that are possible & the significance of each one varies between PC makers and they all use different names for them [my Dell has 3 possible passwords in the BIOS & each one does a slightly different job]. 

 

Look through your BIOS [often this is done by pressing F2 during bootup but your manual will tell you] to see what BIOS passwords can be set up, what each one does & what the relationships between them are - one of them might be a master password, one might control what you can boot from [HDD/USB/DVD] & one might be a password for HDD access. 

 

Personally, I have the bootup password set so it can only be booted from the HDD & I have the master password set so nobody can get in the BIOS & change the first password.  This does mean that someone could remove the hard drive from my PC & read its contents in another PC but I accept that risk.  This combination means that there are no extra passwords to remember during normal use [only when I want to boot from a USB].

 

I witter on about this in more detail in

Precautionary measure 6 - Secure your PC          [PDF version]

frederik long say: When you attach a password to any admin account then that password will stick in any mode. However, there are means and ways of promoting an ordinary account to admin level. The technique is based on booting the machine with some boot CD. You can prevent this like so:
  1. Check the children's account. Demote it if it is at admin level.
  2. Get into the BIOS setup.
  3. Find out which key is used to invoke the boot selection menu (it is often the Del or the F12 key), then disable this selection while still in the BIOS.
  4. Check if this measure is effective.
  5. Get back into the BIOS setup and set an admin password.
  6. Carefully note this password.
On most BIOSs this measure will prevent users from changing the boot device, thus preventing the use of boot CDs or boot flash disks.

If this is a laptop and if you forget the password then you have to return the machine to a service center to have the password reset. You were right to put a password on the Built-In Admin account.  As you had already done so I thought my reference to making sure you had admin account passwords would not contradict you.  For the avoidance of doubt, I agree that having a password on that and all other admin accounts is worthwhile.

 

I was just trying to explain that it was worth having that account disabled so it has little chance of being affected by any user profile corruption - such corruption can occur in use.  The Built-In Admin's password remains in force even though it has been disabled & that password would be required if you ever decided that you needed to re-enable it [or they tried to do so behind your back].  You can test this yourself.

 

If your own Admin account ever suffered from user profile corruption then you would have a problem.  That's why I suggested creating a second one as a fallback.

 

If you boot from a DVD/USB then you are not in Windows 7 but are in whatever OS is on that DVD/USB.  Your accounts are only of significance to Windows 7 so if you have booted into something else then you have no protection.  You might boot into a Linux [OS] CD, for example.  [You are also booting into a different OS if you ever boot up using your Windows 7 Installation DVD/USB, your Windows 7 System repair disk or the recovery CD of a backup-restore/antivirus utility].  Personally, I would not know how to use Linux or anything else to alter the Windows 7 settings or installed applications but, at the very least, all your HDD's content would be exposed for unregulated manipulation limited only by the nature of whatever OS has been booted into.  But I do know, again just for example, how to boot into the Windows 7 Installation DVD/USB or the Windows 7 System repair disk in order to get in & reset the Built-In Admin account [you'll read about this in the link I gave you before - you will probably be as shocked as I was when I first found out about this which is why I then investigated how to avoid the risk - the article starts at A guide to recovering from lost passwords or other faults preventing... and the earlier link was to the section of it dealing with a specific topic within the article].

 

- - - - - - - - -

So, in short, having a password on all the admin accounts should include the Built-In Admin account even though it has been disabled.  That prevents people abusing Safe Mode.  Leaving the Built-In Admin disabled just reduces the risk of its user profile becoming corrupted through usage.  To prevent other, completely unregulated access, you would need to prevent booting into other OS and therefore would need to prevent booting into other drives such as DVD/USBs.

When I upgrade a preinstalled (OEM) or retail version of Windows 7 or Windows 8/8.1 license to Windows 10, does that license remain OEM or become a retail license?

If you upgrade from a OEM or retail version of Windows 7 or Windows 8/8.1 to the free Windows 10 upgrade this summer, the license is consumed into it. Because the free upgrade is derived from the base qualifying license, Windows 10 will carry that licensing too.

If you upgrade from a retail version, it carries the rights of a retail version.

If you upgrade from a OEM version, it carries the rights of a OEM version.

Full version (Retail):

- Includes transfer rights to another computer.

- Doesn't require a previous qualifying version of Windows.

- Expensive

Upgrade version (Retail):

- Includes transfer rights to another computer.

- require a previous qualifying version of Windows.

- Expensive, but cheaper than full version

OEM :

OEM versions of Windows are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

What happens if I change my motherboard?

As it pertains to the OEM licenses this will invalidate the Windows 10 upgrade license because it will no longer have a previous base qualifying license which is required for the free upgrade. You will then have to purchase a full retail Windows 10 license. If the base qualifying license (Windows 7 or Windows 8.1) was a full retail version, then yes, you can transfer it.

From the end user license agreement:

15. UPGRADES. To use upgrade software, you must first be licensed for the software that is eligible for the upgrade. Upon upgrade, this agreement takes the place of the agreement for the software you upgraded from. After you upgrade, you may no longer use the software you upgraded from.

17. TRANSFER TO ANOTHER COMPUTER. a. Software Other than Windows Anytime Upgrade. You may transfer the software and install it on another computer for your use. That computer becomes the licensed computer. You may not do so to share this license between computers.

Should I upgrade my existing version of Windows to Windows 10 Preview?

Since Microsoft made available the first pre-release build of Windows 10 in early October 2014, there has been some misunderstanding that this is not a product you should use to replace a production installation of Windows such as Windows 8 or 7 or Vista. Instead, if you must test it on a single machine, the best recommendation is to create a partition or have an a spare hard disk ready where you can install it. Windows 10 in its current pre-release state will expire and at the same time, it is unfinished work. So if you must evaluate it, the best way is by installing it beside your commercial installation of Windows.

If you do choose to upgrade your existing version of Windows to Windows 10, please be warned, it will make your recovery partition in operable, requiring that you have separate recovery media for reinstallation if necessary.
