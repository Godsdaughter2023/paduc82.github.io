lemp say:
To start at the end, what you should be looking for in Autoruns is not the error message per se, but the appearance, most likely in the "Image Path" column, of the string
     E:\windows\system32\sti.dll

Please explain "F(C:)  goes with a USB on the front of the tower" in a little more detail.  Particularly, are you saying that both drive F: and drive C: are assigned to USB external drives?

It's unusual to have Windows installed on drive E.  I'm not saying that it's impossible for a system to work this way, but it can lead to problems, especially if a sloppily-written installer assumes that Windows is in C:\Windows rather than relying on system variables (like %windir%).

sti.dll is a Windows file that should be protected by Windows File Protection.  That is, normally, it should be difficult to delete or corrupt this file without resulting in its automatic replacement by Windows.  I suspect something of this nature may be at the root of your problems.

Click Start > Search
Select a search for "All files and folders"
In the "all or part of the file name" box, type sti.dl*
Under "Look in" select Local Hard Drives
Click the button next to "More advanced options"
Check the box to "Search system folders"
Check the box to "Search hidden files and folders"
Check the box to "Search subfolders"
Click the Search button

How many files are found by the search?  In what folders?

For each copy of sti.dll (assuming that you have at least one copy), right click, select Properties, and report the version number. 

One of the files found by the search may be named sti.dl_ in E:\i386.  You won't be able to get a version for this file (because it is compressed).
