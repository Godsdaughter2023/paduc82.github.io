john rubdy say:

Hi,

 

 

When you backup files on a Windows 7 computer you may receive error 0x81000019 with an error message similar to the following:

 

"A shadow copy could not be created. Please check the 'VSS' and 'SPP' application event logs for more information. Details: Insufficient storage available to create either the shadow copy storage file or other shadow copy data."

 

This error can happen because of having too many Windows language packs installed on your computer.

The cause is that the Backup process involves the system writer that reports files in the important applications installed on the machine. In the case where a large number of language packs is installed, the metadata file reported by this writer becomes huge in size and does not fit the 100 MB volume. This causes backup to fail since there is not enough space for the shadow copy creation. This is not something that you can check on to validate since this metadata file is created during backup and deleted at the end.

 

Method 1:

 

To resolve the issue, uninstall all of the unused language packs that are currently installed on your computer. You can do this by

 

a)      clicking Start, and then typing Lpksetup.exe in the Search box

b)      And then clicking Lpksetup.exe in the list of programs.

c)       Once the Language Pack installer starts, click Uninstall display languages.

d)      Uninstall the languages that you do not use, and then restart your computer and restart the backup.

 

Method 2:

 

Check whether Volume shadow copy is running:

 

a)     Click Start.

b)    In the search box type Servcies.msc

c)     Right click Volume shadow copy and Go to Properties.

d)    Select Automatic (Delayed) from the Startup type drop down menu.

 

Method 3

 

Check if there is enough disk space to perform backup on your computer. You can also use the disk clean up to free up some disk space and try to perform the backup.

http://windows.microsoft.com/en-us/windows7/Delete-files-using-Disk-Cleanup

 

Method 4:

 

Set active partition.

 

a.     Click on start button, type diskmgmt.msc and press enter.

b.      Now right-click on C: partition and select Mark partition as active (By default C: drive has windows installed on it. If you have installed windows 7 on any other driver then, select that partition).

c.       Close Disk management.

 

 

Method 5:

 

Also check the drive for errors

 

Check a drive for errors

http://windows.microsoft.com/en-US/windows7/Check-a-drive-for-errors 

 

Important: While performing chkdsk on the hard drive if any bad sectors are found on the hard drive when chkdsk tries to repair that sector if any data available on that might be lost

 

 

If you are creating a volume shadow copy, deselect the create drive image checkbox when you choose the contents of the backup.

Well,after much searching I found a solution, hopefully it can help some other people with the same issue:

Apparently windows 8 and windows 10 both have an issue with the bios and will lock up when restarting (it happens even once I've got windows 10 installed), but a cold boot will work. So basically, run the upgrade, and every time it tries to reboot your PC you shut it off and wait a few seconds then turn it on. Worked like a charm.
