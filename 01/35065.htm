BCC Code 116 is usually a video driver problem.

You may find what to do by reading this wiki article:

https://answers.microsoft.com/en-us/windows/wiki/windows_10-update/bcc116-bcc117-your-video-driver-has-stopped/fba02c53-7796-4166-83a2-9c44865c13af

If you can't figure it out from there you need to provide some more information (some you did) and make a few adjustments.

Do this:

Click the Start button/orb in the lower-left corner of your screen. Or, press the Windows logo key on your keyboard.

In the search box enter:

msinfo32

Select/click msinfo32.exe and the System Information dialog box will open.

For video driver information, expand the Components, click Display, click Edit, Select All (Ctrl-A), Copy (Ctrl-C) and then paste (Ctrl-V) the information back here.

There will be some personal information (like System Name and User Name), and whatever appears to be private information to you, just delete it from the pasted information.

Configure your system so when there is a crash it generates a crash dump file instead of restarting:

Here's how:

On your Desktop right click Computer, Properties, Advanced system settings and make it look like this:

There is no need to create the Minidump folder as Window will create one if it needs to.

Then click OK enough times to close all the dialog boxes and restart your system for the changes to take effect.  After rebooting you can go back and check the settings again to be sure the adjustments "stick".

When you accumulate a few (about 3) crash dumps this wiki article has a section that tells you how to upload them so we can take a look at them with the Windows Debugging tools:

https://answers.microsoft.com/en-us/windows/wiki/windows_10-update/bsod-finding-and-fixing-them/1939df35-283f-4830-a4dd-e95ee5d8669d

Another cause of BCC Code 116 crashes is overheating.

This usually happens during video intensive things like playing games or watching videos.

Sometimes there is no warning - the system just shuts down and it might not restart right away until it has a chance to cool off a bit.

Any of that sound familiar?

Would you say there is anything in particular you do to make the system crash and if so what would that be?

There is a free program you can use to monitor your system temps.

Do this:

If you download the free version of Speccy you can see what your temperatures are and also monitor them while you are performing video intensive things that might cause a BSOD:

https://www.piriform.com/speccy

You can uninstall Speccy later if you don't like it.

When Speccy is running temperatures in green are good and temperatures in red would be bad (in this example everything is green=good - red=bad):

If you minimize Speccy to the system tray the average temperature will still be displayed next to the system clock so you can keep an eye on it and if it gets into the red, you have a thermal problem:

See if you can reproduce the BSOD while Speccy is running and showing the temperature of things.

Is any temperature getting into the red?

Keep an eye on the temperature while you are doing whatever you do that causes the BSOD.

Three options.
 
1: Grab the corners of the window and drag it to full screen size. Do not use Maximize. Close all other windows first via the Taskbar and this one last. Windows will remember the size used of the last window closed the next time you open the program.

a. user say:

Hi Marjie,

Thank you for posting your query on Microsoft Community.

I understand that you are getting test mode watermark on Windows 10. I will certainly help you in resolving the issue.

The test mode watermark may appear in the bottom right part of your Windows desktop, if you install an application whose drivers are not digitally signed by Microsoft and if they are still in the testing phase.

I recommend you to follow below steps to disable the test mode watermark and check if it helps.
1. Right click on Start, click on Command Prompt (Admin).
2. Type bcdedit -set TESTSIGNING OFF and press Enter.
(The TESTSIGNING boot configuration option is disabled through the above BCD Edit command)
3. Check to see if the issue is resolved. If not, restart the system and check.

Reference:

How to disable the "Test mode" message that is displayed in Windows

Note: References to Windows 8 also apply to Windows 10.

Hope this helps. If the issue remains unresolved, please get back to us and we would be happy to help.
