chris - support engineer say:

Hi Catroast,

 My apologies for asking redundant questions.

The operating system uses the information stored in firmware on the mouse to identify itself.

This process can be corrupted by an anti-virus that is mistaking the isntallation as an attack and preventing the correct information from being transferred.

You can try to install the mouse with the anti-virus temporarily disabled to see if it affects the issue:

 

http://windows.microsoft.com/en-US/windows7/Disable-antivirus-software

Changing the classification or the type of device a piece of hardware is being identified as is a matter of updating the driver.

Of course the driver needed is selected based on the information the firmware of the mouse.

I think the issue in this case is the version of Windows being 64bits.

There is the possibility that the chipset drivers that are installed may need to be updated.

They are the other component in the hardware identification system.

The chipset drivers are for the on-board devices of the motherboard (USB, PCI, sound card if on-board) and can sometime be outdated or corrupted.

This can cause hardware to be misidentified as well. I suggest checking the PC manufacturers website to see if those drivers have an update available.

One last suggest would be to see if a BIOS firmware is also available from the PC manufacturer.

It helps in the hardware installation process, working in tandem with the chipset drivers and BIOS settings.

 

I see that the website is recommending Windows 98 up to XP for operating system compatibility.

It's possible that they have not updated the device drivers for the 64bit platform.

Honestly I'm surprised it worked on the Win7 32bit PC, but it's using the generic driver in that case.

A 64bit driver is likely much more specific to the device, and therefore a generic driver may not be available.
