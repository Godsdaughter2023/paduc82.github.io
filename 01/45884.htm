a. user say:

The conventional wisdom has always been to not jump on a new version of Windows as soon as it has been released. And make no mistake - a feature update like v.1803 is a 'new version of Windows.'

In the past, corporate users would wait several months before installing a new version of Windows. Often they would wait until the first service pack. This practice was recommended for individual users as well.

The idea was to let early adopters and corporate testers discover any early problems, and to give Microsoft an opportunity to fix them (or at least acknowledge them.)

This is even more true for Windows 10. In past there were several years between successive versions of Windows. Today there is a new version of Windows 10 - excuse me, a 'feature update' - twice a year.

Also, Microsoft does a much better job of managing the distribution of updates in Windows 10. Updates are sent out in waves to selected groups of users. After every wave, Microsoft receives telemetry ('feedback') from updated computers which provides information on how the update is succeeding. This helps Microsoft identify problems with the update.

Sometimes Microsoft can identify specific hardware that is adversely affected by an update, and it will stop sending the update to computers with that hardware. (Unfortunately, some people disable the telemetry in a misguided desire to stop Microsoft from 'spying' on them.)

For the best chance of success, the smartest thing to do is to wait until the update comes to you. If you want to make sure that your computer will not be included in the early waves - as I do - and you have the Professional edition, you can delay the update by a length of time that you specify. I chose 90 days, so I'm not even going to be a candidate for v.1803 for at least that length of time.

Those 90 days will also give me ample time to learn about this new version of Windows - the new and improved features, the fixes, the security improvements, and even the known bugs (which Microsoft publishes) - and to learn from the experiences of the early adopters and corporate testers. By the time I finally get the update, I'll be ready.

Of course, I'm not telling you what to do. You do what makes sense to you. In general, I take a conservative, careful approach to operating my computer. It may not be exciting, but I don't have any of the problems that people yell and scream about in this forum.

Windows 10 - Hard disk 100% usage and missing start menu problems - Solution 03.2016

Hi all,

I want to share my findings, after three long, long days trying to recover my laptop.

My problems began with an automatic update, that applied the following:
KB3140743
KB2880510
KB3140768
KB3114426
KB2956110
KB3114901
KB3114900
KB890830
KB3114894

These updates made me lose the start menu, Cortana and the store, and produced the annoying Critical Error window.

My first reaction was to uninstall the most recent updates, but the problem remained.

Then I read lot of pages with lots of suggestions, some of them worked momentarily (like the sfc / scannow command - that by the way takes a lot of time to execute and after some reboots loses effect), others were more complex and the effects not clear (like using the command Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"} ).

After spending more than 14 hours trying to fix it with the commands, I took the decision to take the system to a recovery point inmediately before the updates.

 

Windows could do it, but the price was that I began to have problems with Visual Studio, since the XAML editor presented an error (0x080070002) with little information in Internet about it.

I tried one suggested solution, that is to create a local user account. I did it, but when I used it I lost access to my original account. Horrible!

Then I decided to use a recovery point dated one month ago, far away from these updates, as I recalled that at that time my system was working well.

I did it, and FINALLY, I got my start menu, Cortana and the store. BUT, the hard disk began to work like crazy, making every response VERY slow. When I finally could access Task Manager I saw 100% disk usage.

I searched many solutions, and the ones that I applied are:

1. Mark my WiFi connection as metered (Control Panel -> Networks and Internet -> Advanced Options -> Metered Connection), to make Windows not apply automatically the updates. Otherwise, in a couple of days you
will have the updates applied, and the problem will be there again (Use the second switch to enable it)

2. Stop and disable Superfetch, BITS and Windows Search services.

3. Resize virtual memory: Go to Control Panel -> System and Security -> System -> System Properties -> Performance -> Configuration -> Advanced Options -> Virtual Memory
Change -> Custom size (I put there initial 4,096 and max 8,192, since my system has 4 GB of RAM). You can use you RAM amount times 1 for the initial and times 2 for the max size.

4. The most important action for me was to stop OneDrive. I used this command: C:/windows/syswow64/onedrivesetup.exe /uninstall. That took the hard disk usage to almost 2%. Amazing.

I rebooted the system, and after that reboot it was working fine, but after some minutes the hard disk began to work a lot again. I checked Task Manager and found that a process was causing this (svchost). I found comments that this process is somehow responsible for the updates. And you can not terminate it without causing a shutdown. So I left it work for some time, and looks after it got the updates (that I am not going to apply yet) it stopped using the disk.

I lost two complete days of work because of this, and the hours spent trying to fix it were around 40, so I hope my findings could help others to fix their systems in much less time.

And yes, the forums and communities are great ways to find info for troubleshooting, I invite you to share your findings.

Good luck!

David Diaz

lemp say:
You can use cleanspl but you don't have to.

The program that is using those files is the Print Spooler service.  Those files appear any time you print, but normally they are deleted after printing has been successfully completed.

In order to delete the spool files, you first have to stop the spooler service.   

You don't specify what prompted you to go looking in the spooler's folder, but the following are my usual instructions for dealing with the fairly common "Print Spooler is not running" problem.  The first step (after the antimalware scans) performs the deletion of the files you mentioned.  The cleanspl option is discussed in the "alternative" method.

Often, but not always, the symptoms you described are caused by a corrupt print job stuck in the queue or a corrupt printer driver.  However before cleaning things up, on general principles you might want to download, install, update, and run full scans with each of these two free programs:

MalwareBytes AntiMailware
SUPERAntiSpyware

Don't run both scans simultaneously.  Each will take some time, so start it and then go do something else for a while.

Clean up printers

NOTE:  If after completing step A the print spooler still does not stay running after you have issued the "net start spooler" command, you will not be able to follow all of the steps in "Article One."  Use the Special Procedure for Lexmark printers described below and then follow Article One.  If still no joy, use the alternative method linked below.

A.  Clean out the pending print jobs

  • Open a Command Prompt window (Start > Run > cmd > OK)
  • Type the following in the black Command Prompt window and press Enter after each line

                           net stop spooler
                           del /Q "%windir%\system32\spool\PRINTERS\*.*"
                           net start spooler
                           exit

B.  Clean out the old printer drivers and install the latest drivers using the directions in Article One. 

Special Procedure for computers that have or had a Lexmark or Dell-badged Lexmark printer.

Open a Command Prompt window (Start > Run > cmd > OK)

Type the following in the black Command Prompt window and press Enter

      sc config spooler depend= RPCSS
      exit

note that there is no space before the =

If you want to use a Lexmark or Dell-badged Lexmark printer after doing the above, you'll have to reinstall it.

Alternative to Article One's method: http://members.shaw.ca/bsanders/CleanPrinterDrivers.htm

1)      Do you get this error when performing a specific action or when working on a specific program?

WerFault.exe process is a part of Windows error reporting system and helps sending errors logs from your computer. The error you are getting may be caused if the WerFault process is overloaded because of errors caused by another program or the Werfault process itself is corrupted due to a security threat. Let’s try these steps and check what is causing this issue.

 

Method 1: SFC scan

 

System File checker (SFC) scan is done to check if there are any corrupted system files that could be causing this issue. Refer to these steps to perform a SFC scan:

 

a)      Press Windows key + X, select Command prompt (Admin) to bring up elevated command prompt.

b)      In the command prompt type the following command: “sfc/scannow” and press enter.

c)       Now, restart the computer.

 

Method 2: Memory Diagnostic tool

 

If the above steps didn’t help, I would suggest you to run the memory diagnostic tool and check. Refer to these steps:

 

a)      Press Windows key +X, select Control panel.

b)      Change the view by option on the top right to Large icons.

c)       Click on “Administrative tools” and select “Windows memory diagnostic”.

d)      Click “Restart now and check for problems” .

 

Method 3:

 

If none of the above steps helps, you can temporarily disable the WerFault service to stop getting these errors. Refer to these steps:

 

a)      Press Windows key +R, type “services.msc”.

b)      In the services window, locate the “Windows Error Reporting” system.

c)       Right click on the service and select properties.

d)      In startup type, change from “Manual” to “Disabled” and click OK.

 

Hope this helps. Please get back to us with the results if you need any further assistance.
