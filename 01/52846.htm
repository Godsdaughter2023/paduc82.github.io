aileen alf say:

Hi,

Windows uses a naming convention called Long Filenames (LFN). The LFN system supports file names up to 255 characters. You might get an error when copying or deleting a file with a very long file name if the location where you'll be moving the file has a longer path than the file's original location, for an instance, the Recycle Bin.

In order to delete a filename that is too long, follow these steps:

  1. Open File Explorer and navigate to the directory where your files are located.
  2. Press and hold Shift, then right-click on an empty area. Choose Open command window here. A Command Prompt window will open, focused on the directory you’re in.
  3. Enter the following command in the Command Prompt window: DIR /X
  4. If the directory contains more files than can be displayed on a single screen, use the command DIR /X /P instead, so that it will pause at every screen length so you can examine the file list. This command will list all the directories and files in the current directory, and it will also list the old 8.3 filename for all the files and directories. (An 8.3 filename is a filename convention used as an alternate filename to the long filename for compatibility with legacy programs.)
  5. Issue a DEL command for the file: DEL FILENAME.TXT
    (Replace FILENAME.TXT  with the name of the file you want to delete.)
  6. Windows will delete the file without complaint. You can run DIR /X again to check the directory in Window Explorer.
