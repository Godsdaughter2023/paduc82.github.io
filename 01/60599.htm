Hi roliverlam,

AutoPlay is a Windows feature that lets you choose which program to use to start different kinds of media, such as music CDs, or CDs and DVDs containing photos. There are several reasons which might be causing this problem.

a) AutoPlay might be restricted by Group Policy.

b) It could be canceled by a program that is running.

c) The service that AutoPlay needs to start might not be running.

Adding to Daniel’s suggestion, you may follow these methods & check if it helps.

Method 1

a. Type "services.msc" in the start search box, click on OK.

b. When the Services window loads, navigate to the "Shell Hardware Detection" service, the double click on it to open it.

c. Make sure the "Startup Type" says "Automatic". If it doesn't, use the drop down menu to change it.

Method 2

Let’s follow the steps given in the section  AutoPlay doesn't start when I insert a disc or plug in a device’  from this link.

Troubleshoot AutoPlay problems

Method 3

You may also hold down the Shift key when you insert devices or media into your computer to open AutoPlay, regardless of the default setting.

Additionally, as you have mentioned autoplay is enabled in control panel make sure that in the list next to the device or media, ‘Ask me every time’ option is selected from the dropdown list for all  device & media.

Refer:

AutoPlay: frequently asked questions

Hi Montecarlo,

 

  • When you encrypted the files did you backup the Encrypting File System (EFS) certificate?

 

You can lose access to encrypted files if you install a new operating system or upgrade your current one, or if your current operating system fails. The following steps can help you regain access to encrypted files.

 

 

You must have a backup copy of your encryption key and related certificate on a disc or other removable media (such as a USB flash drive) to follow the steps below.

 

If you encrypt data on your computer, you need a way to recover that data in case something happens to the encryption key. If your encryption key is lost or damaged and you don't have a way to recover your data, the data is lost. You will also lose data if you store your encryption key on a smart card and the smart card is damaged or lost. To make sure you can always access your encrypted data, you should back up your encryption certificate and key.

mryanmarkryan say:

I only have time to answer one question..

What you are not understanding is that encryption works by initially generating a PRIVATE key that is "known" by your login into Windows under your OS. SO given that key.. Each time the file is accessed, if the key is available to you, then it will unencrypt the file "under the hood" and give you the results. It does not work like a zip file where you view the file as a zip and have to extract it manually. In your case, you had a key that was assigned to you per your login under windows. You encrypted the file. You then moved to a different operating system where your key was a different key. If you want the file back you have to go back to the same operating system that actually encrypted the file and under your original NT login. When you changed to a different operating system, the key used to encrypt the file is no longer available and that is why you lost the file. No key no access.

Your best bet is to look into other solutions for securing your files "on the road" EPS seems to have lots of user issues.
