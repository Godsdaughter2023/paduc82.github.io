    Solution: Xbox 360 console stuck in restart loop after starting a - Microsoft Community

    spud of doom say:

    Solution: Xbox 360 console stuck in restart loop after starting a download

    I was having this problem earlier today, and during my search for a fix I saw a number of people on the forum here have had the problem in the past but did not always get a clear fix. After about an hour of searching and messing around on my Xbox, I think I figured it out. Since those topics are closed I figured maybe if I made a topic somebody could find it later on through the search. Hopefully this helps someone eventually.

    Problem description: After initiating a download from the Xbox Live marketplace, the active download begins and then the console restarts itself. After restarting, the console gets to the dashboard before restarting again 10-30 seconds later. This loops forever.

    Disconnecting the target storage device from the system fixes it, but only until you next boot with the HDD back in. Deleting the download from the queue fixes it, but if you try to download it again the same problem will start again. I was able to repeat this on my Xbox 360 S console about 3 or 4 times trying to download The Witcher 2.

    Solution:

    1) Turn off your xbox 360 console

    2) Remove all active downloads from the queue. Do this via the website at https://live.xbox.com/Download/Queue

    3) Turn on your xbox 360. It should no longer be resetting itself

    4) Go into Settings > System Settings > Storage and select the device you were downloading to. Scroll down to find content on the HDD for the item you were downloading (e.g. in my case, The Witcher 2). This will be either in alphabetical order, or down the very bottom of the list marked as "incomplete" with a yellow ! for its icon.

    5) Delete all content associated with the problematic download


    You should now be safe to begin the download from the store again without your console getting stuck in a loop.

    wunders say:

    The best way I know to attack this problem is to download and run "Process Explorer" from the Microsoft website.  Process Explorer is like Task Manager on steroids.

      Process Explorer:  < http://technet.microsoft.com/en-us/sysinternals/bb896653.aspx >

    After starting it, you will notice the same instance of "svchost" that is consuming your CPU.  You then hover your cursor over the "svchost" name and a popup window will appear that will list the Services that are controlled by this instance of svchost.  Write down this list, then start the "Services" window:

       Start -> Run -> "services.msc"

    When the Services window starts, locate each of the services that you had written down in the list and either Stop or Pause that service by double-clicking on it and selecting the appropriate action.  If the CPU suddenly falls back to a reasonable setting when you do this, you've found your problem service.  In my experience, "Windows Management Instrumentation" is a common culprit, so if that is in your list, try that one first.

     

    HTH,

       JW
