Tenure is based on the total years of Xbox LIVE Gold time completed. The clock stops when you go Silver/Free and Xbox LIVE Silver time does not add (or subtract) tenure. lemp say:

Note, if you don't have a manual for your KX-P1123, get one here: http://safemanuals.com/314159.php?k=80df3754c2c20dcbba291cb15f7a88ef&ID=202796&q=PANASONIC%20KX-P1123

You didn't mention how the printer is actually connected to the computer.  These ancient printers were intended to connect to a computer's parallel port, a device that is missing from more and more modern computers.  If your computer uses a USB-to-parallel adapter (or USB-to-parallel cable), then I would suspect that device, or its driver, first.  The following discussion assumes that the printer is connected to the parallel port of the computer.  You can do the following steps (A-F) in any order. 

A.  Assuming that all of your computers are using Windows XP , the simplest issue would be that the print driver on the "print server" (the Windows XP computer to which the printer is physically connected) has become corrupt.  Clean out the current driver and reinstall the printer on the print server as follows (if some of your computers are not using XP, they will use a different driver, and it seems unlikely that multiple drivers would become corrupt in the same way at the same time):

  1. Go to Printers and Faxes
  2. Right click on the icon for the KX-P1123 and select Delete
  3. Click File > Server Properties > Drivers tab
  4. Select each driver related to the KX-P1123 and click Remove
  5. Reboot the computer
  6. Go to Printers and Faxes
  7. Click "Add a printer"
  8. Click the radio button for "Local printer attached to this computer" and UNcheck the box to "Automatically detect and install my Plug and Play printer"
  9. In the drop-down box for "Use the following port" select LPT1 (again assuming no USB to parallel adapter)
  10. Select Panasonic KX-P1123 from the list
  11. Give the printer a name
  12. Click the radio button for "No" (assuming that you don't want this as the default printer)
  13. Give it a share name (use the same share name it had before or you'll have to reconfigure all of the other computers)
  14. Print a test page.
  15. Finish the Wizard

B.  As an initial troubleshooting step, check that the printer prints properly apart from Windows.  The printer's Self Test mode is described at page 2-16 of the manual:  turn off the printer, then turn it on while pressing the LF (line feed) button.  This should print all of the ASCII characters in draft mode followed by the 4 LQ (letter quality) fonts.  The test will continue in draft mode for 20 minutes or until you turn the power off, whichever is sooner.

Assuming that the printer self test prints normal characters, I can think of four possible causes of your problem (other than the USB issue above):

  1. there is something wrong with the parallel port on the computer to which the printer is connected (either the hardware or its configuration);
  2. the parallel port hardware inside the printer has failed;
  3. there is something wrong with the cable; or
  4. some configuration change has been made to the printer (I used to have a KX-P1092, which is only slightly older than your 1123; the 1092 was configured using tiny "DIP" switches under the printer's cover -- but the 1123 made things easier to change -- and thus easier to mess up -- by using the front panel buttons).

C.  If you have another computer with a parallel port, connect the printer to that computer and install it (steps 6-15 above).  If it works, then the problem is with the first computer, and most likely the first computer's parallel port.  If it doesn't work, then the problem is either the cable, the printer's internal hardware; or the printer's configuration.

D.  Logically, the next step would be to substitute a known-good cable, but you probably don't have one and don't want to go buy one at this point.

E.  Checking the printer's configuration is a bit complicated.  I suggest that you carefully read Sections 3.1 and 3.2 (pages 3-1 to 3-11) and especially Section 3.4 (pp 3-18 to 3-24) in the User Guide.  You probably should print out the current settings as described on page 3-9 and the You may have to interpret the output using the matrix on page 3-3.  The printer should be in "program" mode in order to allow your Windows applications to control the font setting. 

Notice that on page 3-20, one possible setting for the "Initial Setup" is to use one of 3 different graphic character sets.  If your printer somehow got into this mode, that could be your problem.

F.  If at this point you still haven't found the problem, do the following (as a substitute for Section 4.4 of the manual -- I highly doubt you have Basic on your Win XP computer):

  1. Create a short test file in Notepad and save it in the root directory, e.g., C:\test.txt
  2. Open a command prompt window (Start > Run > cmd > OK), type the following, and then press Enter
  3. copy C:\test.txt > LPT1

If you still get your all-symbol output, the problem probably is hardware, any you should take the printer to be serviced -- and ask them to check it out on their computer, with their parallel cable, first.

ken blake say:

See your other replies, and let me add the following:

Why do you want to do a clean reinstallation In my view, it's usually a mistake. With a modicum of care, it should never be necessary to reinstall Windows (any version).  I've run Windows 2.0, 3.0, 3.1, 3.11, WFWG 3.11, Windows 95, Windows 98, Windows 2000, Windows XP, Windows Vista, Windows 7, Windows 8, Windows 8.1, and now Windows 10, each for the period of time before the next version came out, and each on two or more machines here. I never reinstalled any of them (with a single exception--a problem that I worked on for weeks unsuccessfully), and I have never had anything more than an occasional minor problem.

It's my belief that this mistaken notion stems from the technical support people at many of the larger OEMs. Their solution to almost any problem they don't quickly know the answer to is "reformat and reinstall." That's the perfect solution for them. It gets you off the phone quickly, it almost always works, and it doesn't require them to do any real troubleshooting (a skill that most of them obviously don't possess in any great degree).

But it leaves you with all the work and all the problems. You have to restore all your data backups, you have to reinstall all your programs, you have to reinstall all the Windows and application updates, you have to locate and install all the needed drivers for your system, you have to recustomize Windows and all your apps to work the way you're comfortable with.

Besides all those things being time-consuming and troublesome, you may have trouble with some of them: do you have or can you find all the drivers you need? can you find all your application CDs? Can you find all the needed installation codes? Do you have data backups to restore? Do you even remember all the customizations and tweaks you may have installed to make everything work the way you like? Occasionally there are problems that are so difficult to solve that Windows should be reinstalled cleanly. But they are few and far between; reinstallation should not be a substitute for troubleshooting; it should be a last resort, to be done only after all other attempts at troubleshooting by a qualified person have failed.

And perhaps most important: if you reformat and reinstall without finding out what caused your problem, you will very likely repeat the behavior that caused it, and quickly find yourself back in exactly the same situation.

If you have problems, post the details of them here; it's likely that someone can help you and a reinstallation won't be required.
