a. user say:

The powrprof.dll is one of the 3498 files that Windows File Protection looks after.

If you rename it,  Windows File Protection will quickly and silently replace the now "missing" file with the backup copy from the dllcache folder and log an event like this in the Event Viewer System log:

Event Type:    Information
Event Source:    Windows File Protection
Event Category:    None
Event ID:    64002
Description:
File replacement was attempted on the protected system file c:\windows\system32\powrprof.dll. This file was restored to the original version to maintain system stability. The file version of the system file is 6.0.2900.5512.

You won't have time to get your CD out of the case before the file is replaced.

So really, if you want to get a fresh copy, simply rename the one in system32 (so it is now "missing") and WFP will copy in a fresh one for you automatically!  Try it and see for yourself.

Why don't you click Start, click Run, type sfc.exe in the Open box, and then click OK and tell us what happens.

SFC will not replace a missing or corrupted powrprof.dll file in the system32 folder.  That is not how SFC works, that is not what it does, and that is not what it is for.

Good try though.
