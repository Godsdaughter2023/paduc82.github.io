If you are accessing these drives as a true NAS then you should not have a recycle bin connection to those drives.  A true NAS should be plugged into a network hub or router and not a computer directly. You access the drive through a Mapped share. The NAS has it's own server software on it that provides all communication to and from the drive through the network.  If this is how it is attached to your network then you need to advise the exact make and model of NAS.  Look on the manufacturer's web site for information on this issue.  I have three different NAS drives (320, 320 & 500 GB) on my home network, as well as a Microsoft Home Server with 3 TB of drive space. If the drive is attached to a computer through a USB, eSATA or Firewire connection then it is NOT a NAS drive.  Right click Recycle Bin and ensure that each drive showing in the recycle bin is set properly and you can turn off the recycle bin on individual drives.

This error occurs when a user-mode subsystem, such as WinLogon or the Client Server Run-Time Subsystem (CSRSS), has been fatally compromised and security can no longer be guaranteed. In response, the operating system switches to kernel mode. Microsoft Windows cannot run without WinLogon or CSRSS. Therefore, this is one of the few cases where the failure of a user-mode service can shut down the system.

 

Resolution

Running the kernel debugger is not useful in this situation because the actual error occurred in a user-mode process.

 

Resolving an error in a user-mode device driver, system service, or third-party application:

Because bug check 0xC000021A occurs in a user-mode process, the most common culprits are third-party applications. If the error occurred after the installation of a new or updated device driver, system service, or third-party application, the new software should be removed or disabled. Contact the manufacturer of the software about a possible update.
