considerate_guy say:

The .msdvd file is NOT a video format. It's a saved project file from Windows DVD Maker. That program was retired back in 2011 as part of Windows 7. It is not included in Windows 8, 8.1 or 10, nor can it be installed there.

If you have a .msdvd file, it requires the original source video/photo files anyway. So the best you can do is to go back to the PC where the .msdvd files were created, reopen Windows DVD Maker and actually create the DVD images. At least you'll then have playable movies in VLC or similar.

As for movies created with Windows DVD Maker, the resulting video DVDs are encoded in the MPEG-2 format which is a legacy format for SD video (from the era of analog TVs) and doesn't support HD video. The MPEG-2 video format was also not carried forward into Windows 10 so you can't edit DVD (MPEG-2) footage in Movie Maker.

For more information, read:

What kind of files can I use with Movie Maker?

Be sure to read the Notes sections because some formats were only available in Windows Vista and Windows 7 and then not brought forward.

Properly speaking, I should have called it the Windows Recovery Console.

There are at least two knowledge base articles about it.  They are:

  KB314058 , a Description of the Windows XP Recovery Console for advanced users, and

  KB307654 , How to install and use the Recovery Console in Windows XP.

Unfortunately, one usually gets the error message: Setup cannot continue because the version of Windows on your computer is newer than the version on the CD (as described inKB898594 ).

This happens if your current incarnation of WindowsXP has had a Service Pack installed after installation (which is almost certainly the case).

KB900871 , How to integrate Windows XP Service Pack 2 files into the Windows XP installation folder, tell you how to remedy this problem.

Note that KB900871 talks about Service Pack 2 (which is no longer supported).  The exact same procedures can be used with Service Pack 3 to integrate SP3 into either an installation folder or an installation CD.  The installation folder can be shared and then used on a remote computer to run the "winnt32.exe /cmdcons" command.  I integrated SP3 into a "SlipStream" CD using Paul Thurott's procedure at:http://www.winsupersite.com/showcase/windowsxp_sp2_slipstream.asp .  I find having SP3 integrated with my installation media very convenient.

You can also run the recovery console from your installation CD.  Boot it and type "R" where it suggests to boot to the Windows Recovery Console.

Once the Recovery Console boots, it will prompt you to select the system partition from which you boot WindowsXP.  This is usually C:\Windows.

You will be prompted for the Administrator's password.  Type <enter> if you haven't set an Administrator password.  Otherwise, type the correct password.  Once you've done that, WRC will continue until you get a prompt.  The command to run chkdsk is slightly different from WindowsXP.  You will want to type: chkdsk /p <drive_you_want_to_check>:

WRC is also good for dealing with viruses, Trojans and RootKits, too.
