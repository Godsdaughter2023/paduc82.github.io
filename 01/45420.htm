I eventually found the answer myself after reading the articles kindly identified by Sumit in that answer. Although the articles did not take me directly to the solution, they suggested searches that I could try, and Google managed to find the answer. Unfortunately, the same searches through Bing did not give me any clues.

The problem was caused by the value in the following registry entry:

HKEY_CURRENT_USER\Software\Policies\Microsoft\Windows\Explorer\DisableNotificationCenter

This value is a DWORD(32 bit) setting that had the value 1 (disable). Either deleting the value completely, or changing it to zero removed the grey from the Action Center switch and allowed me to switch on the icon.

There is a related value:

HKEY_LOCAL_MACHINE\Software\Policies\Microsoft\Windows\Explorer\DisableNotificationCenter

and this affects all users. My PC did not have this entry.

cmdrkeene say:

Hey Jim,

Robocopy is built-in to Windows, but it's a command-line utility.  The good news is that basically you only have to come up with the command once, then you can schedule it to run for you in the background (overnight, or several times through the day, etc.).

But actually as I compare this and one other option, another built-in command called "xcopy", I think the latter would suit you more. It has an option with the /D command that will make it only copy files that have changed on or after the version that was already backed up.  Combined with the /Y switch (to automatically overwrite the existing file that has the same name), this would effectively just update your backup with the newest set of files, without storing any file more than once.

Also just a side note, a colleague of mine mentioned that if you turn on NTFS compression, two identical files in the same location would effectively only take up the space of 1 file.  I was informed that a similar technology is in use on Exchange server, so that if one employee sends an attachment file email to the whole company, they don't have to have server space to store all those identical copies of the same file for every person's inbox.   I haven't tested this, it's a bit out of my norm, but it sounds interesting to me.

I do personally use File History myself (set to just keep as many backup copies as possible until it runs out of space, which my 2 TB-size external USB drive hasn't filled up yet).  I've got roughly 350 GB of personal files.
