¡firedog say:

It sounds as if you have set up the account to use POP3. This only downloads messages from the inbox; no other folders are involved, and deleting messages from the inbox in webmail will have no effect on already-downloaded messages.

Set up the account again, this time to use IMAP, which will synchronize the contents of all folders between client (Windows Live Mail) and server (Sky Yahoo):

  1. Right-click on the existing account and select Properties. On the General tab, deselect Include this account when receiving mail or synchronizing. Click Apply and OK.
  2. Press Ctrl-Shift-T to start the Add your email accounts wizard. Enter your email address, password and the sender name to be shown on outgoing messages, select Manually configure server settings and click Next.
  3. Select the account type as IMAP.
    Enter the incoming server imap.tools.sky.com
    Set the incoming server port to 993 for IMAP.
    Tick the box for Requires a secure connection (SSL).
    Make sure the Authenticate using drop down selection is set to Clear text.
    Enter the outgoing server smtp.tools.sky.com.
    Set the outgoing server port to 465 for SMTP.
    Tick both boxes for     Requires a secure connection (SSL) and Requires Authentication
  4. Click Next and then Finish.
  5. Right-click on the newly created account name in the folder pane and select Properties.

    Change the account name if you want.

    On the Servers tab, make sure that the Email username is your full Sky email address and that the other settings on the page are as specified under (3).
    On the Advanced tab, check that the server ports are as specified under (3) and that This server requires a secure connection (SSL) is selected for both.

    Click Apply and OK.

The account should connect and attempt to download the account folders. Leave the program alone while this is happening and wait until it's quite finished before checking the folders. Test the account by composing a new test message and sending it to yourself. If you see an error message, click on it to select it, then use right-click to copy it and paste it into your reply here.

When you're sure that the account is working properly, you can copy any messages from the old account's folders - e.g. Sent items and any drafts you need to keep - to the equivalent folder in the new account. When you have copied all of the messages in the old account that you need to keep, you can right-click on the old account and select Remove account.

Bootrec /fixmbr

justice 3x say: Nothing is preventing you from using it, but there are certain restrictions that apply to it in regards to licensing:

OEM versions of Windows 7 are identical to Full License Retail versions except for the following:

- OEM versions do not offer any free Microsoft direct support from Microsoft support personnel

- OEM licenses are tied to the very first computer you install and activate it on

- OEM versions allow all hardware upgrades except for an upgrade to a different model motherboard

- OEM versions cannot be used to directly upgrade from an older Windows operating system

OEM licenses are to be installed by professional system manufacturers only. Under Microsoft's OEM License Agreement, they are not to be sold to end-users under any circumstance, and are to be preinstalled on a computer using the OEM Preinstallation Kit (OPK) before shipment to the customer, and must include at the very least the manufacturer's support contact information. They are, therefore, designed for installation only on a single computer and are not transferable, even if the original computer is no longer in use. This is not usually an issue for users who purchase new computer systems, because most pre-assembled systems ship with a preinstalled operating system. There are few circumstances where Microsoft will allow the transfer of an OEM license from one non-functioning system to another, but the OEM System Builder License Agreement (SBLA), as well as the OEM End User License Agreement (EULA) do not contain any allowance for this, so it is entirely up to Microsoft's discretion, depending on the situation

http://www.microsoft.com/oem/en/licensing/sblicensing/pages/licensing_for_hobbyists.aspx

DSP OEI means “Delivery Service Partner, Original Equipment Installer.” It is an OEM version of Windows distributed to smaller computer manufacturers and system builders. According to the license, it is only supposed to be pre-installed on a new PC that will be resold.
