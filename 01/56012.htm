Hello,

As per the description, I understand that the computer goes to Blue Screen of Death (BSOD) while trying to perform remote desktop sharing. This issue is intermittent.

I would appreciate if you can provide us the following information to help us understand the issue better.

  1. What is the error code that you get after BSOD?
  2.  Have you installed or updated the display driver recently?

The event viewer logs clearly shows that the issue is caused by the driver: RDPUDD.dll. This issue occurs because the Remote Desktop Protocol (RDP) display driver (Rdpdd.sys) does not check whether an object is NULL before Rdpdd.sys accesses the object. Therefore, this issue occurs when Rdpdd.sys accesses an object that has a NULL value. This clearly indicates that either the current display driver is not installed properly or the driver has gone corrupt.

First, I would suggest you to update your display driver either through Windows update or manually from Computer or Video card manufacturer website.

Update the driver for hardware that isn't working properly.

http://windows.microsoft.com/en-in/windows/update-driver-hardware-isnt-working-properly#update-driver-hardware-isnt-working-properly=windows-7

If updating the driver doesn’t help, you may try to remove all the traces of the current display driver completely from the computer and install the latest version of the driver.

Note:Before uninstalling Video driver, I would suggest you to refer the Microsoft article below: Uninstalling the video driver on a computer running Windows 7 may result in a black screen. http://support.microsoft.com/kb/2425157/en-us
