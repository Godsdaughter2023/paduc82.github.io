a. user say:

Yeow.

Another reason it might take an extra long time (and everything takes a long time) is if it is an IDE drive and the Transfer Mode has slipped from DMA mode to PIO mode.  This is a good thing to check when there are performance problems and often overlooked.  Everything else could check out fine, but before you are done, it never hurts to check this when doing a tune up.

See?  If the MS Answers would ask these questions up front when a new topic is created, things would go alot smoother.

Here is how to check for that:


If you are using IDE drives, use Device Manager to verify the Transfer Mode of the IDE channels is set to something like DMA if available (depends on your hardware) and not the slower PIO mode.
  
Remember:  PIO is the slowest, DMA is the fastest.

This is easy to check and generally easy to fix and the mode would not have changed by itself,so if it has changed to PIO, change it to DMA and then figure out why it changed and fix it.

To launch the Device Manager console, click Start, Run and in the box enter:

devmgmt.msc

Click OK.

Expand the IDE/ATA controller section to see your IDE channels. Right click each, choose Properties, and for each channel that has an Advanced Settings tab, determine the Transfer Mode.  There are usually 4 channels to check in a desktop, maybe fewer for laptops.  Check them all.

The fastest selection will always be some kind of DMA selection (usually: DMA, DMA5, DMA if available, UDMA, etc.).  If it is PIO, change it to DMA.
  
If you are not sure about what you see post back for help and advice.
  
If you make any adjustments, reboot and check the settings again to make sure any changes "stick".
  
What may have caused the change?
  
http://support.microsoft.com/kb/817472
