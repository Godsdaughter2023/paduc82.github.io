mk1090 say:
This was a problem that occured on systems with an AMD APU (e.g. A8 7600, A10 7850k, Athlon 5350 ...). Now, the same problem seems to occur also with discrete amd graphic cards. The last working AMD driver for APUs (in my knowledge) was the 14.11.2 Beta (14.301.1013-141117a-177876E, apparently a driver from november 2014). I'm using this driver at the moment and everything works fine (except that I can't use new amd drivers, that means I'm missing new driver features!).

Since the 14.11.2 Beta, every newer driver (called "omega" drivers) made problems with JPEGs. First there were broken (e.g. black) thumbnails in Windows. The photo gallery crashed upon creating any thumbnails. You could use a workaround: deleting or renaming the JPEG-decoder files of AMD. Then Windows used automatically its built-in JPEG-decoder (a part of the so called "windows imaging component"). The last remaining problem was (and still is) the issue with the white or chess-board-like blocks in the image viewer.

Recently, the JPEG-decoder-bug (for APUs) was fixed by the last driver-update (15.7 WHQL driver). But the issue with the white or chess-board-like corner remains. For discrete graphic cards, this seems to be a new problem. Apparently, you have to go back to the last stable catalyst driver (for APUs even before the omega-drivers!).

Kind regards

Please follow the below steps and check if you had given full control to the guest account that you had created.

1.      In Windows Explorer, right-click the file or folder you want to work with.

2.      From the pop-up menu, select Properties, and then in the Properties dialog box click the Security tab.

3.      In the Name list box, select the user, contact, computer, or group whose permissions you want to view. If the permissions are dimmed, it means the permissions are inherited from a parent object.
