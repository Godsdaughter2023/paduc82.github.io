  Answer: Hi this is a response I found on VNUNET.Com from a Robert Clayton which worked for my situation which is similar to yours. I tried to access a printer on my XP Home machine in another room via a wLAN, from my Windows Vista desktop. Constantly had a "cannot access printer" message until I followed the instructions from Bob Clayton. Seems Windows Vista will only access a 'local printer' not a network one. From memory I needed to download the latest Windows Vista compatible driver for the printer attached to the XP machine, onto my Windows Vista machine before I took the steps below.

"Posted: 05-Aug-2007 19:09 Reply

Been trying to connect my wife’s new Windows Vista laptop (yuk) to print out to my networked XP desktop's printers. I could see the files and printers on the XP machine from the laptop. Tried all the usual, firewalls, file and printer sharing turned on. Would not connect!

Apparently this is a problem that hasn't been solved.

This "workaround" solved it for me.

1.) Verify that File and print sharing is turned on and properly configured on the XP machine.

2.) Locate and notate the Win XP computer name (i.e. "Desktop") (Found in System Properties in the Computer Name tab)

3.) Locate and notate the printer share name on the XP computer (i.e. "Printer") (Found under the Sharing tab on the Printer Properties page)

4.) On the Windows Vista machine launch the "Add a Printer" wizard

5.) Choose "Add a local printer"

6.) Select "Create a new port" radio button and choose "Local Port" from the drop down menu

7.) Press Next

8.) In the "Enter Port Name:" box enter the following: \\Desktop\Printer where "Desktop" = the computer name from step 2 and "Printer" = the printer share name from step 3.

9.) Select the Printer driver from the list provided or use the "Have Disk" button to install the correct Windows Vista driver if it is not in the built-in list.

It is critical that the driver you use is Windows Vista compatible.

10.) Press next

11.) Give the printer a distinct name and choose whether to set as the default.

12.) Press Next

13.) Print a test page to verify functionality.

14.) Press Finish
Hope this helps someone else.  Bob

adamaaa say:

Nvidia have dropped the ball on the GTX 560M (Vendor/Device 10DE:1251). This model is in may notebooks installed as a secondary GPU behind and another (often Intel GPU) which is used for 2D/desktop rendering.

john john __ say:

The best thing to do would be to consult the documentation for your machine and see if there is information for the service partition.

You can look at the partition with disk editing tools like Acronis Disk Director or you can use a simple partition editing tool like Symantec's PtEdit32 to change the partition type and you will then be able to look at the partition with Windows Explorer.

Take note of the partition type before you change it so that you may then set it back as it was.  The partition is most likely a hidden type 1C, if it is you want to change it to a type 0C to unhide it.  A reboot is necessary for Windows to see and mount the partition so that it becomes available to Windows Explorer or the Disk Management tool.

WARNING!!!

Slippy fingers or user errors when using disk or partition editing tools can lead to the loss of all the data on the disk!  Using PtEdit32 is fairly easy and straightforward but it does require a bit of knowledge and understanding of (or a willingness to learn) how partitions are arranged and identified.  You can download PtEdit32 from the Symantec FTP server, easy to find with a search on the net: http://search.yahoo.com/search;_ylt=A0oG7mbsbOxMsWcBUlZXNyoA?p=ftp.symantec.com+ptedit32&fr2=sb-top&fr=yfp-t-501

You can look but don't delete or change any files inside the service partition, if you do the restore process may fail when you need it most!

davem121 say:
Hi Andrei,

Start your PC, just as Windows attempts to load (spinning dots), press and hold Power Button for 5 - 10 seconds to perform a Hard Shut Down

Do this twice

On the third start Windows will boot into the Recovery Environment and from there you can access System Repair, Safe Mode, Command Prompt . . . etc.

Please try each option one at a time, then see of Windows will boot . . .

Go to Troubleshoot - Advanced Option - Revert to previous build of Windows
If available, this would be the best option . . .

Go to Troubleshoot - Advanced Option - Startup Repair
See if Windows startup can repair itself

Go to Troubleshoot - Advanced Option - System Restore
Check to see if you have a restore point you can go back to

If you do not have a Restore Point, go to Troubleshoot - Advanced Option - Startup Settings and click Restart
Upon restart, press 4 to enter Safe Mode
Once in Safe Mode, backup your data if you do not already have a backup
Open Device Manager and check to make sure all your Drivers are up to date
Reboot to see if Windows will start Normally

If the above fails, go to Troubleshoot - Advanced Options - Reset this PC
Choose to keep your files and apps, just your files or to keep nothing
Then start the Reset
