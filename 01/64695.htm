This is due to the fact that our display configuration has to be set to "extended desktop". Ultimately, Windows says "These applications are running on the primary desktop" because our secondary display is being considered as an extension to our primary desktop.

So, what happens is that when we are in a full screen mode, and click on our secondary display, Windows views this as clicking on an application running on the same desktop as our full screen application and reacts by minimizing it. I chalk this up to the coding of how Windows responds to applications running in "Full Screen" mode and how Windows responds to applications running in "Windowed/Maximized" mode. zexalex say:

After reading all the posts and googling, I found out that the only way to get rid of the point64.sys file is to move it on top of the c:\windows\system32\drivers\point64.sys (they are in effect the same file).

I used the "advanced" button under file properties / security tab and took ownership of the file; then removed all permissions and granted "full control" to "Everyone". Had to use the "remove inheritance" button, etc.

Do the same for all folders, up to the "windows.old" folder.

This will enable you to cut/paste the file to "c:\windows\system32\drivers\point64.sys" (replacing it).

Side note: you will not be able to remove it because "it is in use by another program" (but you can rename it). Whatever.

After moving the file you will be able to remove the windows.old folder (if you changed the folder permissions the same way).
