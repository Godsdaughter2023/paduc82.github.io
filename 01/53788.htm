Error opening JPG with photodraw - Microsoft Community

darryl_b say:

Error opening JPG with photodraw

This question relates to an issue regarding opening JPG files using PhotoDraw 2000 V2.

In late November 2010 Photodraw began to error when opening NEW (e.g. just downloaded from camera).  The error stated that the file could not be opened as it was corrupted.

The machine configuration is:

Hardware: Dell XPS 420

Windows Vista Home Premium, Service Pack 2 - current with all update patches to the date of this message.  (Windows live essentials had been installed and subsequently removed in troubleshooting this issue)

Office 2003 Professional

Photoshop CS and Photoshop elements 7

Photodraw 2000 V2

BACKGROUND:

This computer has been running Photodraw and the Adobe products flawlessly since the machine was purchased in 2008.  There was absolutely no issue with opening any of the supported file types in Photodraw.  My wife uses Photodraw almost exclusively while I use the adobe products.

In late November she noticed that she could no longer open any of the newer JPG files in her photo library.  Older (before November 2010) still opened fine.  Each time she attempted to open one of the newer files Photodraw presented an error stating that the file was corrrupt and could not be opened.  These same JPGs could be opened in the Adobe products as well as in Picture Manager, and several other photo editing software.  Examination of the files content using XnView and a hex editor revealed no corruption in the files.  This led me to believe there was a software issue that was the result of an update.  Note that software additions to this computer are fairly tightly controlled with the exeption of MS Update which runs automatically.  Windows Live Essentials was installed around that same time frame.

RESOLUTION

After several hours of very tedious file comparisons I determined that the files JPEGIM32.FLT and GIFIM32.FLT both had time stamps of 07/26/2010 and file sizes of 228 and 173 KB respectively.  I suspect that the new files are a component of Windows live.  I renamed the two newer files (located in C:\Program Files\Common Files\microsoft shared\Grphflt) to .FLT_OLD and copied the original files from the Photodraw distribution CD to that directory.

Photodraw is now back to its full functionality . . . and more importantly my wife now has her favorite application back.

Editorial comment:  It is a shame that Microsoft discontinued this product as there is little else available that can compare to its simplicity and intuitive functionality.

QUESTIONS:

1.  Can you please confirm the source of the newer filter files so that I can ensure functionality of some other application has not been disrupted ( I haven't found anything yet )? 

2.  What has changed in the newer file?  The file sizes of the new filters are smaller and there has clearly been some functionality changes.

 

Thank you,

Darryl

Method 1: Start the Windows Installer service and check if that works.

  1. Click "Start", type "Services.msc" (without the quotes) and press {ENTER}
  2. Double-click Windows Installer
  3. Set the Startup type of Windows Installer to "Manual"
  4. Click "Start" to start the service.
  5. Click "OK".

  1. Click "Start", click "Run", type "MSIEXEC /UNREGISTER",(without the quotes) and then click "OK". Even if you do this correctly, it may look like nothing occurs.

  3. Even if you do this correctly, it may look like nothing occurs, or you may briefly see an hourglass. After you run this command, the operation is complete.
  4. Try your Windows Installer-based application again.
