arthur.dent say:

1. After Windows has downloaded an update, it WILL be installed, and if that installation involves a reboot, then it WILL be rebooted.  Installs which don't require a reboot may happen at just about any time. Installs which do require a reboot will be scheduled automatically, choosing at any time outside your currently configured "Active Hours" daily block of time. (If your desktop PC is powered off but plugged in at the time the system selects, then the computer's BIOS/EFI will be automatically configured to wake itself up at the designated time to perform the installation. Ditto, if your laptop is powered off but has any charge left in its battery at the time the system selects.)

If you happen to be using your computer after it downloads a rebooting update, but before it reaches its automatically chosen install time (as described above, potentially anywhere outside your "Active Hours"), then you will have the option of selecting when to perform the install/reboot - but all you can do is specifically state your preferred time. You do not have the option to prevent installing the update at all.

[EDIT] In the release-to-manufacturing version of Windows 10, and in version 1511, you used to have the option of requiring Windows to actively notify you of any pending updates which would require a reboot. If you selected this option, then you would be given a notification informing you that an update was waiting to install, and asking you to select the time when the install should happen. It automatically proposed a time, and you had a chance to explicitly give it a different time. You could only dismiss the notification by agreeing to the proposed time or by selecting a time. However, if you didn't acknowledge the notification at all (for example if you were away from the computer), the install would just go ahead and happen at the proposed time anyway.

Starting with the Anniversary Update (version 1607), this "Notify to schedule restart" option has been removed.[/EDIT]

2. See above.

3. This feature was deliberately removed in Windows 10.
