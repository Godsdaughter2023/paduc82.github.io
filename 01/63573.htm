Back in the days of DOS, the PrtScn key used to print the screen. But
in all versions of Windows, this works differently, and the name of
the key is now an anachronism. The key does not print the screen.
PrtScrn captures the whole screen, and Alt-PrtScrn captures the active
window.

Either one captures the image to the Windows clipboard. Once it's in
the clipboard you can paste (Ctrl-V) it into any application that
supports graphics (Windows Paint, other graphics programs, even your
favorite word processor). You can edit or add to the image as you
wish, then print it.

This ability to manipulate the image in a program before printing it
is an improvement over the original DOS method of just printing it.
But if you'd like that old facility back, there are several
third-party freeware/shareware programs that can do this, such as
PrintKey2000 at
http://www.sharewarejunkies.com/00zwd2/printkey2000.htm

nicklam1 say:

Date & Time Change in Windows 10

I use a date sensitive program where I have to change the date and time back every time I use it.  

In the past I had the setting "set time automatically" turned off.

Since the last Windows 10 upgrade, this option is now greyed out.  I've checked our company's central policy with IT and this hasn't been changed.  Can someone please tell me how to stop the date and time automatically changing itself.  

devlin sco say:

Hello Nick,

If you're unable to modify the option to set the time automatically from Settings, you can configure the Windows Time service instead. This service maintains the date and time of your system.

  1. Click the Start button.
  2. Type Services.msc and then hit Enter.
  3. Double-click Windows Time.
  4. Change the Startup type to Disabled.
  5. Click Apply and then OK.
  6. Restart your PC to complete.

Please take note that disabling this service will halt automatic date and time synchronization. That being said, this procedure might affect other programs or apps that may require the date and time of your system. Should this happen, you can change the Startup type of the service back to Automatic.

Let me know if you need further assistance.
