rami-- say:

From:  Steve Riley [MSFT]

Do you have any error messages from the preparation tool?

 

Without a TPM, the only option for key storage is a USB drive. You don't need any special USB "pre-boot" support, because the computer is actually booted from the small boot volume that the preparation tool creates. During this process, Windows prompts you to insert the USB drive. Windows reads the storage root key from there, uses that to decrypt another key, and continues loading the operating system from the encrypted system drive.

 

While we do support using a PIN _with_ a TPM, we don't support using one _without_ a TPM. This is because it's nearly impossible to achieve the kind of entropy needed with a simple password to protect something as important as the BitLocker volume master key. The recovery password does, but as you note, remembering a such a long number is next to impossible.

 

The car key metaphor is actually pretty good. Get yourself a small USB drive with a little loop so that you can hang it from your key ring. Then you'll use this key to "start" your computer. You can remove it once the computer finishes booting. You'll need it each time you boot and each time you bring the computer out of hibernation (but not sleep).
