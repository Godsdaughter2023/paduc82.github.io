Enable System Icons via Local Group Policy Editor

  1. Press Windows + R keys on the keyboard to open run command.

         Volume: Remove the volume control icon
         Network: Remove the networking icon
         Power (battery) Icon: Remove the battery meter

    5. Close the Local Group Policy Editor.

    6. Restart Explorer, Log off and log on again, or restart computer to make the change effective.

Hey Obul,

 

I haven't ever seen this happen by itself.  Normally there are latent snapshots that are attached to the Production Vm.

 

Someone likely rolled one of them back.

 

To prevent something like this from happening or to have an audit trail I would suggest that you use SCVMM 2012 as it will log all of these respective changes.

 

The the best of my knowledge I don't believe this is a known issue or bug.  On a side note if you cannot get SCVMM 2012 as a management console.  You should at minimum be backing up the VM's using Windows Backup.  It will take a VSS based backup of the workloads.  Store it on an external drive or somewhere on the network.  This will allow for rapid recovery of the physical VM if this happens again.

 

You should also backup the internal contents of the Virtual Machine itself.  If you already have the System Center Suite I would suggest looking at DPM "Data Protection Manager 2012".
