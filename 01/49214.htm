We’ve already identified the specific cause of this error message – it’s a scheduled task named “BackgroundContainer” that attempts to run the BackgroundContainer.dll malware file, but fails because that file has already been removed from your PC. The fact that the “startup item” for this malware file is a scheduled task is actually good news, because it means that you don’t have to edit your registry, or use the Autoruns application, or uninstall/reinstall any program in order to remove the error message – all you have to do is delete the BackgroundContainer scheduled task with the procedure provided here:

http://answers.microsoft.com/en-us/protect/forum/protect_other-protect_scanning/error-message-on-start-up-windows-8/7f29c7ad-2d55-4f1c-8110-ddca3902f596
