pcr just pcr say:

I did a dual boot briefly once, but into unallocated space on the same internal HDD. It shared the same EFI (Extensible Firmware Interface) partition. But the recovery tools partition still belonged only to the original Win 10. The new instance of Win 10 put its tools into a folder on its C:partition. Not sure what happens in your case of installing to a 2nd HDD.

A new boot entry was put into the BCD (Boot Configuration Data) store on EFI. Every boot afterward had a new screen from which I could select the operating system to boot. It isn't a BIOS screen. Forget BIOS. The Windows boot loader puts it up. You have control over it with...

https://docs.microsoft.com/en-us/previous-versions/windows/it-pro/windows-vista/cc709667(v%3dws.10)

I think your plan is fine. Your BIOS is set correctly in UEFI mode, & the Windows Boot Manager is on top. When selecting the Windows Installation DVD at the BIOS boot menu - if two DVD choices are offered - select the one with "UEFI:" to its left - even if you must stand on your head to do it.

When done, each boot to Windows will have a new screen asking that you select an operating system to boot. If they are the same name, you may use BCDEdit to change them.

Things may be somewhat different for you, seeing as you are installing to a separate HDD. See what might go wrong here (but BCDEdit can fix it)...

pcr just pcr say:

The Windows Boot Manager will not be involved in a boot to DVD. Later - after the dual boot has been established - it will put up a new screen from which you may choose which OS to boot. It never will have anything to do with the DVD, that I know of.

You pressed F10 at boot to get to the BIOS Setup screens, right? Instead...

(a) Press F9 repeatedly at boot to get to the BIOS boot menu options.

     (Just like you did with F10.)

(b) At that screen, select the option that mentions "DVD".

      If two are offered, choose the one that also mentions "UEFI".

(c) Soon will appear a "Press any key to boot the DVD" message.

    If you fail to press one in time, I believe the boot will go to Windows.

However, I've also seen an article that says the message will reappear. But the 2nd time, if you press a key in time, it will boot the DVD in Legacy Mode. You don't want that - so press any key quickly the first time. Here is the article. Don't be scared off by it. I'm just posting it to illustrate the point...

https://support.hp.com/us-en/document/c03729992

Advisory: HP Z1 / Z210 / Z220 / Z420 / Z620 / Z820 Workstation - Windows 7 x64 May Fail to Install on MBR Partition When DVD is Booted to UEFI Boot Source

"How to Refresh Windows 8 and Windows 8.1"

  • The PC boots into Windows RE (Recovery Environment).
  • Windows RE scans the hard drive for your data, settings, and apps, and puts them aside (on the same drive).
  • Windows RE installs a fresh copy of Windows.
  • Windows RE restores the data, settings, and apps it has set aside into the newly installed copy of Windows.
  • The PC restarts into the newly installed copy of Windows.

"How to Reset Windows 8 and Windows 8.1"

"When you reset your PC, the drive will be completely reset, and all your apps and files will be removed from it. Windows 8 will be reinstalled, and you'll need to change any settings and reinstall any apps that you want to get back after resetting. If you know if any files that you don't want to lose, then you should back those files up before starting the reset"

Hi Ashley,

 

What is the make and model of the network adapter/computer?

 

The error code KERNEL_DATA_INPAGE_ERROR states that the requested page of kernel data from the paging file could not be read into memory. It appears that there are some issues with the hard disk. This error may also be caused by network adapter driver.

 

Try running check disk on the hard drive to see if there are any corruptions.

 

You may run Check Disk using Command Prompt. To run “ChkDsk” utility using Command Prompt, you can follow these steps:

a) Press “Windows Logo” + “X” keys on the keyboard and select “Command Prompt (Admin)” from the list.

b) In the command prompt window, type chkdsk /r and press “Enter” key on the keyboard.

c) Answer “Y” when “chkdsk” asks you if you want to check the drive next time computer boots (restarts).

 

Important: While performing Error-checking or ChkDsk on the hard drive, if any bad sectors are found and when attempting to repair that sector, the data available on that sector might be lost.

Volume KMS Client and Volume MAK product keys, are volume license keys that are not-for-resale. They are issued by organizations for use on client computers associated in some way with the organization. Volume license keys may not be transferred with the computer if the computer changes ownership. This form of licensing typically applies for business, government and educational institutions, with prices for volume licensing varying depending on the type, quantity and applicable subscription-term. A volume license key (VLK) denotes the product key used when installing software licensed in bulk, which allows a single product key to be used for multiple installations. For example, the Windows Enterprise edition is activated with a volume license key.
