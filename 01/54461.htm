First of all, if you're using Windows 7, you stand a better chance at preventing these things from happening in the first place. When Windows 7 was built, it was finally the breath of fresh air everyone was waiting for from Microsoft-------they had finally built a robust OS that was no longer riddled with all the little hiccups which used to plague the many versions before it, especially the unforgiving Windows Media Player-------Every version from Vista and back had untold issues with that sorry excuse of a media application, but performance was obviously improved  in Windows 7----virtually no more crashing when you want to create or move different libraries, etc...

In reality each person's situation and machine (including both hardware and software configuration) can be very different, not to mention the fact that, generally speaking, if users do not know what they are doing-------then they should not be messing with stuff.

No one is saying you can't adjust your system the way you like, but indiscriminately downloading and installing third-party applications ( which might not play nice with Windows internals ) is definitely a good way to start causing some damage.

As far as Windows Updates are concerned, I completely understand your point and it is echoed by many frustrated users, many of which (myself included) have been burned quite a few times by Microsoft's faulty patches, but that was also before I started to realize that the best way to deal with bad updates is not to set WU to accept updates automatically-------but instead, it would be safer to set it to check for updates but then let you choose whether to install them or not. That way you get to decide them. Search multiple sites and links and when you see a common denominator-------this could very well help you decide whether to install or not.

If you've been having corruption issues (Though you didn't specify what those were), and as you mentioned, you were ignoring them, then you really only have one person to blame for that---sorry to say. Depending upon what type of corruption we're talking about, there are all types of troubleshooting techniques diagnostics that can be implemented to determine and fix the underlying issues-------you just have to know where to look-------and again, I can't stress this enough-------a user must know what they're doing when they implement fixes or edit the registry or replace Windows components that you don't like or anything else. One can't just modify the system all willy-nil and think everything is going to be honky-dory. Program parameters exist for a reason. If you choose not to follow them, then you assume the risk that comes with that.

System File Checker is a great tool to use to check for and fix Windows corruptions, however there is a right way and a wrong way to implement this tool. The very first thing you should do is power down your computer completely. Shut it down, then turn off the power switch to your power supply, wait about one minute, then unplug the power supply itself. Next hit the power on button to your computer. This will drain all the residual power from the system and will cause it reboot into a clean power state once rebooted. After the clean power cycle you want to reboot into Safe Mode and then open the Command Prompt to run SFC. If System File Checker doesn't find anything after one scan, it might after subsequent scans, so if you're sure something is awry but can't put your finger on it yet, running SFC a couple of times, is sometimes needed in order for it to find something.

Your choice of security, sorry to say, is quite laughable to say the least. MSE is fundamentally weaker than many other security options available from other vendors and there is a good reason why-------because this really isn't Microsoft's strong point to say the least. Why else do you think so many security patches from them fail all the time. Not enough time and resources is spent on testing and tweaking these patches before they are released.

If you want solid system protection that can offer far greater "System-Hardening" than MSE, then you're are better off with security from vendors like ESET, Kaspersky and Bitdefender, just to name of few. Also, implementing EMET or HitmanPro Alert, is a good idea. Both of these can actually work to help layer your system security with things like ASLR and Heap Spray protection. If you can understand concepts such as these in particular, then you're already way ahead of the game.

Additionally, Microsoft has a tool called Microsoft Baseline Security Analyzer, which when implemented, runs a special diagnostic test to determine the health of your Windows machine and whether you're missing some critical updates and/or components. It even allows you to customize which update server your pc will use to check for Windows Updates-------something which might actually work in your case, if nothing else has up to this point. Another thing to remember too, if you've re-installed Windows, you will also have to re-activate it and until you do, Windows Update will not fully function the way it is supposed to.
