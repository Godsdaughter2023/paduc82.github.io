hasanzahidi say: Hello,
What you are specifying is called MFU list (Most Frequently Used programs list). MFU list appears below the Pinned list on the Startmenu. This list keeps track of how often programs are used and displays them in order of most used (top) to least used (bottom).
So what exactly is your problem? Is this list not showing any program? or it shows some programs that were used but is not adding new recent programs to the list?
Possible solutions:

1.Right-click on the Start Menu and select Properties. Under the Start Menu tab click on the Customize button. under the General tab, in the program section make sure that the "Number of programs on Start" is not set to 0. If there is your required number already set, then clear the list by clicking on the Clear List button. Now open some of your programs and check if the MFU list is now displaying the programs or not.

2. If it didn't work for you, then now you need to check your registry settings
Open regedit (Start > Run > regedit > OK)
Navigate to the either of following keys
  • HKEY_LOCAL_MACHINE \ Software \ Microsoft \ Windows \ CurrentVersion \ Policies \ Explorer
  • HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Policies \ Explorer
Delete the NoStartMenuMFUprogramsList & NoInstrumentation values or set the value to zero.
For more information and details visit http://support.microsoft.com/default.aspx?scid=kb;en-us;292504

Important This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Took all day from 8am to 4pm but did it :) gives you a buzz when you solved something like this, that had me  :scratch: all day.

Right.  First problem (that ended up not needed in end) was to fix the GPEDIT issue, why wasn't Administrators in there.  I think the GP was buggered, so asking over at Technet forums I got told about the secedit command (which I'm sure I've heard of before but long forgot).  Got given this link

http://support.microsoft.com/kb/313222

Being on XP I ran this on the machine at a CMD

secedit /configure /cfg %windir%\repair\secsetup.inf /db secsetup.sdb /verbose

Did what it had to do.  Checked gpedit again and sure enough the Administrators group was back in the place it should be in the image in first post.  But it was all still greyed out, but never mind as admin account was now able to remove devices.  Also got told the icon means it's locked by Group Policy and can't be edited.  The blue 1's and 0's means it's not locked and can be edited.

So, now, why wasn't Process Monitor still working?  It was still saying Admin wasn't in the Admin group but it was.  On Technet they later said the Admin has to be in the debug group as well for Process Monitor to run, but I'd fixed it before then, so not sure if that was the issue.

Anyway.  To fix the admin account permissions I used Trinity Rescue Kit

http://trinityhome.org/Home/index.php?wpid=1&front_id=12

Booted up from that, ran

winpass

To allow you to reset a local account, unlock or up it's privileges.  I choose to up the admin accounts privileges as it was saying the admin account only had Normal rights.

Booted into Windows and result, Process Monitor was now able to run :)

However, the hanging issue (not sure if I mentioned that) was still a problem.  I then used msconfig (which ain't great) but it's easy with that to hide all the Microsoft processes so that you can then disable all other, 3rd party ones.  Did that and rebooted.  No more hanging.  Then had to re-enable them one by one till the hanging started again.

What was it?

Our Helpdesk software.  It had two processes running as services at start-up and they were causing the hang.  I'm about to look through the Process Monitor logs of it working and then not working.  To see if what I suspect to be happening is happening.  That is, now the PC is on a new domain, these two services are trying to talk or doing something with the old domain and can't.  Causing explorer to then hang as they maybe just get stuck in a loop.

Was a good feeling when it was all fixed and I had drilled it down to the exact two things causing the issue.

Now it geekness I'm off to look through the process monitor logs.

    Hi Hazeema,
     
    We understand how you feel about the situation with network connectivity on your computer. We're sorry for the trouble this has caused.
     

    We will assist you to resolve the issue.

    Which browser are you using?

    It means that the browser is unable to fix the name you are trying to reach. It could be an incorrect proxy setting in your browser, or a problem where antivirus or a firewall is blocking the address or possibly a network problem with your connection or a setting in your router.

    To prevent all possible causes except browser, try going to the same address using a different browser, if that works, then it is a browser setting problem.
    If that does not work, then try to turn off firewall and antivirus briefly and see if it works. If it does work, then it is a firewall or antivirus problem, turning off one at a time will determine which the cause is.

    If that does not work, then it may be a problem with your NIC card not getting an address. To check that on Windows use start, run and type in cmd, then in the black window type in ipconfig /all and see what address is displayed. If it is all zeros or a 169.254.***.*** address then it is a connection problem. If it displays a proper address, then the next possible place could be in a router setting or an incorrect DNS setting.

    Router status should display all connection information if it is working properly, the WAN address must have an IP for it to work. If it is all zeros then refresh or turning it off and then on may reset the connection.
