If you are using Skype for Windows 10, hiding a conversation is not currently supported. If in case you are using Skype for Desktop, you have the capability to hide your conversation but the functionality to unhide your conversation is not available.

 

What you can actually do to somehow locate the conversation is to try and find something that has been said in a conversation, and to find a particular message within your conversation history, with a contact or group, you can search for a specific word or phrase within the conversation.

1.   In your Contacts or Recent list, select the conversation you want to search.

2.   Press Ctrl + F on your keyboard, or select Conversation > Find in the menu bar.

3.   In the search box that appears, type the word or phrase you're looking for and press Enter. You can use the Next and Previous buttons to move through the search results.

For more information about how to manage your conversation you can check these links: https://support.skype.com/en/faq/FA392/ and https://support.skype.com/en/faq/FA34636/

trekdozer say:

Can You Move Windows 7 To New Computer

If it's a retail Full or Upgrade license - yes. You can move it to a different computer as long as it's only installed on one computer at a time (and if it's a Windows 7 Upgrade version the new computer must have it's own qualifying XP/Vista license). The previous Windows 7 installation on the old computer must be formatted/deleted. You might have to call Microsoft and explain what happened to complete the activation. Activating it on the second computer will automatically in effect "deactivate" the license for the first computer. The key will work with both 32 and 64 bit, but only one can be installed at a time.

If it's an OEM license - no. OEM licenses, including Windows preinstalled on a computer before purchase and Windows bought separately, are tied to the first computer they are installed on and can not be transferred to a different computer. To install Windows on a different computer you will need to buy another copy.

If you have trouble activating Windows 7 you can call Microsoft to activate by doing the following:
1. With Windows running click Start, then in the Search box type:  slui.exe 4
2. Press Enter on your Keyboard
3. Select your Country.
4. Select the Phone Activation option, then call the number given and hold for a real person.

canadian tech say:

I found this to work once:

Just this minute fixed a client's laptop with same error code & symptoms (BITS not starting etc), this section from kb article kb910337 seemed to do the trick:

 

Windows could not start the Background Intelligent Transfer service on the local computer. For more information review the system event log. If this is a non-Microsoft service contact the service vendor and refer to service-specific error code -2147024894. (0x80070002)

If you receive this error message, follow these steps:

  1. Click Start, click Run, type regedit, and then click OK.
  2. Locate the following registry subkey:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\BackupRestore\FilesNotToBackup

  1. If the registry subkey exists, go to step 4. If the registry subkey does not exist, create the registry subkey. To do this, follow these steps:
    1. Right-click Backup Restore, point to New, and then click Key.
    2. Type FilesNotToBackup, and then press ENTER.
  2. Quit Registry Editor.
  3. Click Start, click Run, type services.msc, and then click OK.
  4. Double-click Background Intelligent Transfer Service.
  5. In the General tab, click Start.

On a fresh installation BITS is installed as Manual.

If Windows Update or MSE finds something to download it will change BITS to Automatic (Delayed Start).

Prior to Windows Vista after the downloading was finished BITS would return back to Manual:

https://msdn.microsoft.com/en-us/library/windows/desktop/aa362826%28v=vs.85%29.aspx?f=255&MSPPError=-2147217396

But that doesn't see to happen in Windows 7 - once it goes to Automatic (Delayed Start) that's where it seems to stay.

If there is nothing for BITS to do it should not be consuming any/much CPU or network bandwidth (zero):

Notice the BITS Service is running under a svchost Process with PID 1100 but PID 1100 doesn't even show up as using any network resources or even using a TCP/IP connection because it is not using any network resources:

If you think (or anybody else) your BITS is consuming network bandwidth and it shouldn't be tell us how you figured that out.

Windows Update has been afflicted for many months and since Windows Update uses BITS if you have not fixed the known problems with Windows Update yet you should probably do that first and then see how your BITS looks.
