dax1792 say:

There is a difference between the message you quote in your post title and that in the body of your post.


The 'Test Mode' message is shown when you have enabled test signing with bcdedit. This allows you to run with unsigned drivers. This is cleared by the bcdedit command you quote.


The 'For testing purposes only' message occurs when you install a private hotfix supplied by Microsoft support. The only way to remove it is to uninstall the hotfix. If the message is still displayed after the hotfix has been uninstalled, it may be that the  system has the Microsoft Test Root Authority certificate installed into its Trusted Root Certification Authorities store.  In this case, remove the certificate as described in this blog post.


http://blogs.msdn.com/b/ntdebugging/archive/2011/08/05/for-testing-purposes-only-and-other-desktop-watermarks.aspx



  


timpentz say:

How do I know which hotfix to remove?

dax1792 say:

This would be a hotfix you received directly from Microsoft support which hasn't been released to the general public.


As explained in the blog I linked to, the name of the fix should contain the words FOR TESTING PURPOSES ONLY. The KB number will probably not correspond to an actual KB article.

My recommendation would be to install the updates in batches, such 5 at a time, exclude hardware/driver updates. gokul t say: Hey CTerrian,

Thanks for posting in Microsoft answers!

The screenshot you’ve provided is a notification for program outside of Windows if it tries to make changes to a Windows setting. It is a security setting in Windows to make sure unauthorized programs do not change any Windows settings.

It's usually safe to allow changes to be made to Windows settings without you being notified. However, certain programs that come with Windows can have commands or data passed to them, and malicious software can take advantage of this by using these programs to install files or change settings on your computer. You should always be careful about which programs you allow to run on your computer.

User Account Control (UAC) is a feature in Windows that can help you stay in control of your computer by informing you when a program makes a change that requires administrator-level permission. UAC works by adjusting the permission level of your user account.

Refer the link below for more details on UAC:

What is User Account Control?
http://windows.microsoft.com/en-us/Windows7/What-is-User-Account-Control

User Account Control (UAC) notifies you before changes are made to your computer that require administrator-level permission. The default UAC setting notifies you when programs try to make changes to your computer, but you can change how often UAC notifies you.

I would suggest you to access the link below and carefully read through the table that provides a description of the UAC settings and the potential impact of each setting on the security of your computer.

What are User Account Settings?
http://windows.microsoft.com/en-us/windows7/What-are-User-Account-Control-settings

The only way to avoid receiving the notification is to set the UAC settings to “Never notify”.

To change UAC settings,

Click the Start button, and then click Control Panel. In the search box, type uac, and then click Change User Account Control settings.

Note: Microsoft does not recommend its users turning off UAC settings for security reasons.
