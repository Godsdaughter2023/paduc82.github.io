--------------------------------------------------------------------------------------------------------
I think I got to the cause, which is Gmail and Yahoo only support client access via POP3 and IMAP. And MS accounts only support clients access via DeltaSync or POP3 protocols.

Thus::  only IMAP/DeltaSync protocol actually two-way synchronizes with the mail server.
However, POP3 protocol only collects email from the inbox on the server.However, Any emails on user-created folders will not be downloaded(collected).
  • MS accounts added with using POP3 protocol, POP3 protocol only collects emails from the default folders (the inbox and other default folders) on the server to mail client, then will synchronize them( any messages deleted from default folders on mail client will be deleted from remote server)
  • However, any Gmail accounts added with using POP3, POP3 protocol  only collects emails from default folders( the Inbox folder and other folder) on the remote server into mail client, but will not synchronize them with mail client. (Any messages deleted from the mail client will not be deleted from remote server)
In sum, MS Accounts configured with using POP3 in mail client, pop3 protocol only collects emails from the default folders, then will synchronize them. However,  Gmail accounts configured with using POP3 protocol in mail client,  POP3 protocol only collects emails from the default folders on server, then will not synchronize them

 On the other hand,  non-Microsoft accounts configured with using IMAP and Microsoft accounts(Outlook accounts) configured with using Deltasync in mail clients, IMAP or Deltasync will collect emails on all the folders(default and user-created folder) on server, then will two-way synchronize them with mail server (deletion in mail client or remote server will be reverberated on either).

m allard say:

Hi,

This might help others having the same problem and searching for days like I did. The update would perform and then fail (undo) on restart ending up with a failed update in my windows update history list.

Turns out I have a SSD drive and I had changed the default Program Files Directory to D:\Program Files in replacement of the C:\Program Files default in the registry.

Well, by changing this back to the default C:\Program Files , the update performed flawlessly and now everything is up to date. This also prevented me to upgrade to the Creators update and now I can update without a problem !

- Simply run Regedit from the command prompt(admin).

- Go to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion

- Change the value of "ProgramFilesDir", "ProgramFilesDir (x86)" and "ProgramW6432Dir" to C:\Program Files (whatever drive your Windows install is on...)

- Reboot

-Try the update

Hope this helps !
