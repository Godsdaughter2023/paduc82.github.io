Thats the purpose of the Digital Entitlement every times you make significant hardware changes or reinstall Windows 10.

When you upgraded from a previous version of Windows or receive a new computer preinstalled with Windows 10, what happened is the hardware (your PC) will get a digital entitlement, where a unique signature of the computer will be stored on Microsoft Activation Servers. The Windows 7 or Windows 8 genuine license you were previously running will be exchanged for a diagnostics key.

Anytime you need to reinstall Windows 10 on that machine, just proceed to reinstall Windows 10. It will automatically reactivate. You will prompted to enter a product key a couple times through the installation, click I don't have a key and Do this later.

Is there some way to tell what messages would be examples of the above quote? 

No - there could be any number of reasons why a particular message can't be downloaded.

To try and clear the blockage, log in to the account in webmail and start by emptying the folders for spam and deleted items. Then create a new folder or folders and move all of the messages out of the inbox and into the new folder. When the inbox is empty, select the account from the dropdown list on the bottom half of the send/receive button in Windows Live Mail. No new email messages should eventually appear in the status bar.

Back in webmail, move half of the messages that haven't yet been downloaded by Windows Live Mail back into the inbox and again try send/receive for this account in Windows Live Mail. If the fault recurs, we now know which half of the messages the bad one is in. Repeat the process, taking half of the messages each time until the dodgy message can be identified and dealt with in webmail.

Also - when WLMail encounters an error - is there a way for WLMail to delete the messages from the server that were sucessfully downloaded? 

I don't think so. POP3 is a fail-safe protocol; if a download session is interrupted for some reason, it will start again from the beginning to ensure that all messages are downloaded, unless the server puts a stop to it. I think the command to delete downloaded messages isn't sent until the session is completed.
