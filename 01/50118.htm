Why does Remote Desktop Connection disable tasks running on the - Microsoft Community

celliott22 say:

Why does Remote Desktop Connection disable tasks running on the GPU?

I run Boinc/SETI@Home on several computers.  Occasionally, it is necessary to perform maintenance on this computers, such as installing a new video driver.  With VNC I just click and I am there; I perform the maintenance task, and click, VNC is gone and I am done.  However, VNC has recently become expensive.

 

The computer doing the remote connection has Windows 8, and the computer being connected to run Windows 7.

 

With Remote Desktop Connection it takes at least an hour to connect to a remote computer.  First, I had to set up an account on the remote machine, then set up System/Remote settings to allow that new account to connect.  No matter what I tried, setting the connection to use Network Level Security meant the connection would not work.  So much for Remote Desktop Connection's vaunted security.  Finally, using the less secure connection type, I was able to connect. 

 

 

When I get to the remote computer none of the 4 Boinc/SETI@Home tasks are running on the two GPUs.  I do whatever I have to do and then log off.  From a monitoring program on the local computer I can see that still none of the GPU tasks on the remote computer are running.  So I walk over to the local computer, turn on its screen and find that I now have to log on.  Why is that?  Why would the user be logged off to enable the remote session?  In any case, the only way to restart the GPU tasks is to stop Boinc/SETI@Home and restart it.  Why is that?  The Boinc/SETI@Home CPU tasks are not affected.

 

 

I am sure the reader is thinking, but Boinc/SETI@Home is not a "real" job; it is just a trivial plaything.   Maybe so, but I would remind you that there are well over 200,000 people who are buying multiple expensive computers, software, and video cards to process tens of millions of trivial, non-real work units per day.  We are helping to pay your salaries.  Even if Boinc/SETI@Home were trivial and non-real, what about the organizations that are building huge server farms with large numbers of GPUs attached to each of their computers to solve real problems.  Is Microsoft going to write them off too?

 

 

The bottom line is that Remote Desktop Connection is useless as an entrypoint to a remote machine because it stops whatever GPU processing is being done on that machine, and the only way to make the GPU jobs restart is by physically going to the remote machine, loggin on again, and restart Boinc/SETI@Home.  The user might as well just forget Remote Desktop Connection and journey to the remote computer in the first place.  Then at least he/she would not have to attach a keyboard and mouse just to accomplish a small maintenance task.  All this has been true since at least Windows XP.
