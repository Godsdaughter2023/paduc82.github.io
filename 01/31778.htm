petegould say:

I have an HP LaserJet 2300d printer connected to a Windows 7 64 bit system.  I need to be able to share this printer with systems running 32 bit versions of XP and Win2K.  When I download 32 bit drivers from HP, unzip them into a folder, and tell Windows 7 to "add additional drivers" it tells me that it cannot find a compatible driver at the location I have selected - but I'm pointing it to the folder containing the 32 bit drivers for Win2K/XP.

alan morris papercut software say:

Win2k does not support 64bit print drivers.  I have printed from Win2k to a 64bit print server but when you attempt to open the printer properties you get an error.

I assume the XP machine works fine.  It almost sounds like the devmode (device mode) for the printer is not handled by the Win2k machine and the Win2k machine interperates the error as access denied.

Rather than attempting to even read the devmode from the 64bit machine, I would create a local printer on the Win2k machine that just writes the print formated data to the share on the other machine.

Local Printer, create Local Port in the format \\remotemachine\printershare , select the driver you want and complete the Wizard.  If you have a share name with spaces you must create the port\\remotemachine\"printer share name why is the share name the driver name?".  My suggestion, make the share name one word, less than 8 characters.  There are several Knowledge Base articles for creating this type of printer to get around cross OS driver issues.
