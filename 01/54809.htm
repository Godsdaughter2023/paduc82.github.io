2 OS's on one Hard Drive - Microsoft Community

amped energy say:

2 OS's on one Hard Drive

I curently have Win XP  and I was wondering how I could install the new windows Vista and KEEP Windows XP. I have 40 GB of space left so that won't be a problem.
rcwhite say: Hi, Amped Energy.

As Mick said, 40 GB is not nearly as much as it used to be, and you may well
run out of space. But, keeping that in mind...

As Mick also hinted, the Golden Rule of dual-booting is to always install
the newest OS last. Setup.exe in both Vista and Win7 know how to deal with
the pre-existing WinXP, but WinXP's Setup has no idea about the later
versions, which did not exist when that Setup.exe was written.

Are you familiar with Disk Management (diskmgmt.msc)? That utility has been
built into every version since Windows 2000, but many users still haven't
found it and learned how to use it. One of its features is "Shrink Volume".
You can use this to shrink your existing volume (Drive C:?) to leave 40 GB
of "unallocated" or "free" space. Then create a new volume in this space
and assign it a drive letter; let's say "V:" for example.

Then, from the WinXP Desktop, insert the Vista (or Win7) DVD and run its
Setup. Tell it to clean install (not Upgrade) into your Drive V:. Answer
just a few questions and then wait. The installation should take around a
half-hour and involve several reboots. Just keep hands-off and let Setup
handle the job.

In addition to installing Vista (or Win7), Setup will preserve the WinXP
startup files in your System Partition (probably Drive C: and probably the
first partition on your HD), then add its own startup files to that same
partition - while putting almost all of Vista (or Win7) into Drive V:, as
you instructed. Drive V: will now be the "boot volume" for Vista/Win7,
while Drive C: will remain the boot volume for WinXP. (See KB 314470 for
the counterintuitive definitions of "system volume" and "boot volume".)

You also could choose to boot from the Vista or Win7 DVD and run Setup from
there. But then the new OS would have no idea what drive letters had
already been assigned by WinXP. The new system would assign the letter C:
to its own boot volume, making it necessary to assign D: to what WinXP calls
C:. This will not confuse the computer at all, but it DOES bother many
users who are caught unawares by this change from WinXP's behavior. (I
always use Disk Management to write a label to each volume, then I know
which one is "Vista x64", even if WinXP calls it V: and Vista might call it
C:.)

Win7 RC has a smaller footprint on the hard disk than Vista, mostly because
it does not include many of the accessories built into WinXP (such as
Outlook Express), but it will quickly grow as you add applications and data.
Forty GB will be plenty to get you started.

patrick barker say: Hi,

Most if not of the attached DMP files are of the DRIVER_IRQL_NOT_LESS_OR_EQUAL (d1) bugcheck and are consistently faulting NETIO.sys which is the Network I/O Subsystem system driver (and also not the true cause). Usually with NETIO blue screens, they are most commonly caused by a 3rd party antivirus or firewall conflicting with the NETBIOS ports, etc. Also, they can be caused by 3rd party device drivers in general as well. They can also be caused by hardware failure.

However, there are a few *D1's faulting dxgmms1.sys (DirectX MMS) and mentions of memory_corruption which could imply a hardware memory related issue.

We also have PFN_LIST_CORRUPT (4e) for a bugcheck.

The page frame number (PFN) list is corrupted, typically caused by passing a bad memory descriptor list (MDL).

Usual causes are a bug in a device driver, BIOS or hardware issue (in most cases memory).

Looking through your drivers list, you seem fairly clean software-wise and I couldn't see any antivirus or firewall software to recommend for temporary removal, so I am going to recommend starting off with hardware diagnostics. Let's start with a Memtest for NO LESS than ~8 passes (several hours):

Download Memtest86+ here:

http://www.memtest.org/

Which should I download?

You can either download the pre-compiled ISO that you would burn to a CD and then boot from the CD, or you can download the auto-installer for the USB key. What this will do is format your USB drive, make it a bootable device, and then install the necessary files. Both do the same job, it's just up to you which you choose, or which you have available (whether it's CD or USB).

How Memtest works:

Memtest86 writes a series of test patterns to most memory addresses, reads back the data written, and compares it for errors.

The default pass does 9 different tests, varying in access patterns and test data. A tenth test, bit fade, is selectable from the menu. It writes all memory with zeroes, then sleeps for 90 minutes before checking to see if bits have changed (perhaps because of refresh problems). This is repeated with all ones for a total time of 3 hours per pass.

Many chipsets can report RAM speeds and timings via SPD (Serial Presence Detect) or EPP (Enhanced Performance Profiles), and some even support changing the expected memory speed. If the expected memory speed is overclocked, Memtest86 can test that memory performance is error-free with these faster settings.

Some hardware is able to report the "PAT status" (PAT: enabled or PAT: disabled). This is a reference to Intel Performance acceleration technology; there may be BIOS settings which affect this aspect of memory timing.

This information, if available to the program, can be displayed via a menu option.

Any other questions, they can most likely be answered by reading this great guide here:

http://forum.canardpc.com/threads/28864-FAQ-please-read-before-posting

Chkdsk:

There are various ways to run Chkdsk~

Method 1:

Start > Search bar > Type cmd (right click run as admin to execute Elevated CMD)

Elevated CMD should now be opened, type the following:

chkdsk x: /r

x implies your drive letter, so if your hard drive in question is letter c, it would be:

chkdsk c: /r

Restart system and let chkdsk run.

Method 2:


    Open the "Computer" window
    Right-click on the drive in question
    Select the "Tools" tab
    In the Error-checking area, click <Check Now>.

If you'd like to get a log file that contains the chkdsk results, do the following:

Press Windows Key + R and type powershell.exe in the run box

Paste the following command and press enter afterwards:

get-winevent -FilterHashTable @{logname="Application"; id="1001"}| ?{$_.providername –match "wininit"} | fl timecreated, message | out-file Desktop\CHKDSKResults.txt

This will output a .txt file on your Desktop containing the results of the chkdsk.

Run all tests EXCEPT: Fix All, Long Generic, and anything Advanced.
