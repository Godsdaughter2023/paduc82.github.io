bobmcfly247 say:

Usually the IPs are assigned by the router which act as local DHCP server and not through the ISP. The ISP only provides an IP to the public site of the router/modem, not the internal (private) network.

So if youo have tried to configure the IP of your One manually and the router tells you it is in the wrong subnet when trying to configure portforwarding, you have assigned a wrong IP to your One. So if trying to assign an IP to your xbox manuallay, be sure it is within the same IP range your router is working in. You can chech this IP range in the settings of your router.

danaloew say: I had the same issue. I then realized that my mistake was that a fixed disk is required. The reason a fixed disk is required, if I understand it correctly, is because when you use the recimg /createimage command not only does this command create an image but it also sets a new path for the refresh function to find the new custom mage file. Thus when you try to refresh your computer in the future it will use this new file, and not the original image file. If it were to put the image file on a removable drive (such as a USB drive) the drive path may change, thus making your new image file useless. I used the diskmgmt.msc command to shrink my C: drive by 16 GB, gave it a drive letter and then formatted the 16 GB unallocated drive to NTFS using the wizard. I then used the recimg /createimage as in the article in the link posted by Vijay Korupolu M above.
