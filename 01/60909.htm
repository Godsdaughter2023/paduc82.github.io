a. user say:

Is that naive thinking?

Quite naive, I'm afraid. That's not at all how it works.

A driver is the software that makes it possible for a hardware device to work with an operating system. When the operating system changes, the hardware manufacturer has to test their existing driver to make sure it still works with the operating system. Frequently, the driver has to be updated to keep up with the changes in the operating system.

The owner of the operating system - in this case, Microsoft - can't update the hardware manufacturer's driver. In the first place, Microsoft doesn't have access to the source code; that's the hardware manufacturer's proprietary knowledge. Furthermore, even if Microsoft were to somehow gain access to the manufacturer's source code, perhaps through hacking, that's the manufacturer's intellectual property that Microsoft has no right to change.

The best that Microsoft can do is to publicize upcoming changes to Windows enough in advance to give hardware manufacturers an opportunity to update their drivers by the time Microsoft releases the next update. Then it's up to the hardware manufacturer to decide if they're going to test and update their drivers. It's totally in the manufacturer's hands.
