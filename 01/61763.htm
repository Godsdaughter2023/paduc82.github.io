1. Start your computer by inserting the Windows startup disc or the Windows installation disc if your computer can start from the CD drive.
2. When the Welcome to Setup screen appears, press R to select the repair option.
3. If you have a dual-boot or multiple-boot computer, select the Windows installation that you want to access from the Recovery Console.
4. Type the administrator password when you are prompted to do this. If no administrator password exists, press ENTER.
5. At the command prompt, on the drive where Windows is installed, type chkdsk /r, and then press ENTER.
6. At the command prompt, type exit, and then press ENTER to restart your computer.
7. After you repair the volume, check your hardware to isolate the cause of the file system damage.

If this procedure does not work, repeat it, but type fixboot instead of chkdsk /r in step 5.

IMPORTANT: Running Chkdsk /r can cause you to lose data. Back up your data first. If the above methods don't work, then your hard drive or motherboard may have failed and hardware troubleshooting is in order.

http://www.elephantboycomputers.com/page2.html#Hardware_Tshoot

If you have not got a recent backup of your data, I would be proactive and retrieve the data FIRST before doing any of the above troubleshooting. You can use one of the following methods to do this:

1. Pull the drive and put it in a USB drive enclosure or use a USB drive adapter. Attach this to a computer running a working install of XP/Vista/Windows7. Use the working Windows Explorer to copy the data to the rescue system's hard drive and then burn the data to cd or dvd or copy it to an external hard drive. I prefer not to do this if I know the drive is infected because there is a chance of infecting the host system. In these cases, I use #2 below.

2. You can boot the target computer with a Bart's PE (XP only) or a Linux Live CD such as Knoppix or Ubuntu and retrieve the data that way. This has the additional advantage of telling you whether the problems you're having are hardware or software-related because if the computer behaves perfectly under Linux you know that Windows (software) is the culprit. Conversely, if the computer misbehaves under Linux you will know that there is a hardware problem. Here is general information on using Knoppix for data retrieval when you can't get into Windows:

You will need a computer with two cd drives, one of which is a CD/DVD burner OR a USB thumb drive with enough capacity to hold your data OR an external USB hard drive. ¡firedog say:

The attachment will not be included in the reply to a message with an attachment - there is no point in sending a file back to where it came from. So you would not find either the original message or the attachment in your Sent items folder. The message with attachment will normally be in the inbox, unless you moved it or a rule is in place to move it. When the Conversations feature is enabled, messages with the same subject will be grouped together, so in one-line view only the oldest of them is visible unless the conversation is expanded. In two-line view, it's the newest that is visible.

I don't quite understand the sentence None of the incoming messages that contribute to the conversation now appear in my inbox, only showing as part of the text of the final outgoing message. If messages that were in the inbox aren't there any longer, they have either been deleted or moved. Another possibility is that they're hidden - on the View tab, check that Hide read messages isn't selected. This is a per-folder setting. Any outgoing message will normally be in the Sent items folder, not the inbox. 

You say this is an outlook.com account. Please check that you don't have a rule at outlook.com like Keep the latest message (and delete the rest) if sender's address is <email address>.
