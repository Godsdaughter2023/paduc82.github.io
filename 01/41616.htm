r. subudhi say:

I have subsequently found the cause and the right resolution of the 0x8078011D error.

The error is caused due to the fact that the system partition files are in a hidden partition of 100 MB size and this partition along with the partition where windows 10 has been installed (C:\ drive) are marked by the system to be imaged by default.

Solution: Eliminate the dependency on the hidden partition by moving the contents of that partition to the Windows partition, i.e., drive C:\; in other words, by making the system to boot from drive C:\ itself. This can be done making the Windows partition active.

How to do that?

1. Before proceeding, ensure that you have a System Repair disc or bootable USB flash drive or a windows installation disc ready with you, from which you can boot into the system to repair system start-up, because after the steps below are implemented, the system will not boot normally.

2. Right-click "This PC" on your computer, or from file explorer. Click on Manage.

3. Click on Disk Management.

4. From the partitions displayed, select C:\ drive (healthy, system, pagefile, crash dump etc.). Right-click it and select "Mark partition as active". Click OK to any message that pops up.

5. Restart the system.

6. Enter into the bios and set first boot location to CD drive/USB. Insert the CD/DVD/USB, save the changes in bios and exit.

7. After the system boots from the boot device, choose appropriate key board (in most cases, it should be US), choose start-up repair from the subsequent advanced options menu (sorry, I do not remember the exact sequence).

8. When the system finishes the start-up repair procedure, it will restart automatically. When the system is restarting, remove the CD/USB. The system will then start normally.

9. Now you can create a system image on a hard disk or on an external disk.

That's all.
