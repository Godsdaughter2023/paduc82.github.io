todd_s_88 say:

It sounds like this is all just trying to log into the new local profile for the first time? With the "User profile service failed the sign in" usually it's one of two scenarios:

  1. The profile has been logged into before and has become corrupt. It sounds like this one you're familiar with since you were referencing the registry so I'm assuming you're referring to the HKLM\Software\Microsoft\Windows NT\CurrentVersion\ProfileList key
  2. The profile hasn't been logged into before, and the permissions on the default profile are incorrect which causes an access denied message, and the profile is never actually created (which would explain it not being in the registry)

A good reference for the second issue is: https://helgeklein.com/blog/2013/09/error-message-explained-user-profile-service-failed-logon/

Essentially what you do in that situation is:

  • Make sure you can view hidden folders, and go to C:\users. Right click on the Default folder and go to Properties, then go to the Security tab->Advanced->Change Permissions
  • Click on each one of the security principal's listed, check the box "Replace all child object permissions" and hit apply. Do this for each one in the list.
  • Try logging into the local profile again (or recreate it)
