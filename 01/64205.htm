greg carmack - mvp 2010-2020 say:

Unless you know for sure XP will install on that model, there are likely several things working against it on modern hardware. 

First if you have a UEFI BIOS as most modern PC's do, XP cannot be installed except in CSM or Legacy BIOS mode to an MBR formatted hard drive.  If you have UEFI install now to GPT format hard drive that means you'd need to change the UEFI Firmware settings and reinstall both OS's to wiped hard drive that converts to MBR format, with XP going first so Windows 10 would know how to configure a Dual Boot menu (which XP doesn't). The only way to get around this would be to enable CSM or Legacy in BIOS/UEFI firmware and install XP to a separate hard drive with the Windows 10 drive unplugged or disabled. You can tell if you have a UEFI install on GPT drive by the presence in Disk Mgmt of an EFI System partition instead of System Reserved partition on Legacy MBR drives. Afterwards plug back in other HD; boot XP via the Toshiba F12 BIOS Boot menu key, or install EasyBCD to 10 to add XP. 

Next, most modern SATA drivers are not in the XP CD.  Since this is the one driver that must be present for the installer to even see the hard drive, if it doesn't when booted  what's required is to Slipstream SATA Drivers  into Windows XP CD  since the only way to directly load them is via ancient defunct floppy disk.

Finally, whenever XP is installed (unless to a separate hard drive with the other one unplugged or disabled) after another OS it isn't smart enough so steals the boot from the newer OS to itself.  What this requires after install in a Legacy install to MBR is  Marking Active  the newer OS' present Active partition in Disk Mgmt,   run  up to 3 separate Startup Repairs until it starts, then adding XP to a Dual Boot menu from EasyBCD (click Download - no Name or Email required) installed to the newer OS.  EasyBCD can also be used to move the Boot Mgr back from XP following Bootmgr - Move to C with EasyBCD - Windows 7 Forums .

So it's not impossible unless you only have one available UEFI  hard drive to use, or don't want to reinstall Windows 10 in Legacy Mode to an MBR disk which can host XP, in which case you should install XP first anyway since any newer OS installed afterward should configure a Dual Boot with it, and if not you can use EasyBCD to add XP. 

You might have better luck virtualizing XP from within Windows 10 using freeware like Virtual Box or VMWare. 

I hope this answers your questions.

cbarnhorst say:

Use virtualization for XP.  Run it in a VMWare virtual machine or one of the other virtualizaion products like VirtualBox.  Dual booting XP with Windows 10 is a bad idea.  A really bad idea.  

I use VMWare Player (free) to create XP virtual machines on Windows 10.  The problem you will have is that XP does not include anything later than IE6 and IE6 does not access web sites anymore.  You will need IE8.  You can't download IE8 using the bundled IE6 in XP so you have to download the IE8 installer from Microsoft using IE11 in Windows 10, put it on a usb drive and then install it in the XP vm.  Then you will be in business.

You do NOT want to install XP in a dual boot scenario with Windows 10 because of the incompatibility of the Volume Shadow Copy drivers in XP with later versions of Windows.  Every time you boot XP the VSS driver will delete all VSS based backups and previous versions of files in Windows 10.  And of course there are driver issues with trying to install XP on modern computers (there usually are no drivers available).  

If gaming is your concern, run a separate computer for XP.  Don't try to use a modern machine.
