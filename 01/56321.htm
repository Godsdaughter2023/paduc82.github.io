lemp say:
will I be able to download the drivers on to another (Lenovo) computer and create transfer disc?

Yes, that's exactly what you should do.  You can put them on a USB flash drive (your USB ports should work even if you haven't installed the chipset driver yet ... but check to see that you can, in fact, read files from a USB flash drive; if you can't, you'll have to burn the downloaded drivers onto a CD).

You won't, of course, be able to use Dell's "Detect My Product" scanner, but entering your Service Tag should get you to a Dell support page that's customized for your individual computer. 

For example, if you go to the Dell support page for a generic Inspiron 1300 (http://www.dell.com/support/home/us/en/04/product-support/product/inspiron-1300x/drivers), you'll see 4 listed network adapter drivers.  If you use the Service Tag, probably at least one of the wireless adapter drivers will not be shown.

If you are faced with a choice of drivers even after you enter your Service Tag and can't figure out which one(s) to use, download and install only the above driver.  That should enable you to go online with the Dell, get to the Dell support page and use the "System Analysis" or "Detect My Product" buttons.  See "Drivers by scan" and "Drivers by Tag" at the bottom of this page --> http://www.dell.com/support/contents/us/en/19/article/Product-Support/Self-support-Knowledgebase/hardware-support/drivers-help

As you know, XP is no longer supported, and thus using XP on a computer connected to the Internet exposes you to increasing risk.


Although Windows Update will continue to provide all of the necessary updates for Windows XP that were released prior to April 8, 2014, it will no longer do so unless the Windows XP installation is already at service pack 3.  Thus, if you insist on continuing to use XP, you'll have to install service pack 3 manually, as follows:

    dism /online /cleanup-image /restorehealth

  • Norton Antivirus
  • Other AV solutions
  • iCloud
  • IDT Audio (faulting module IDTNC64.cpl)

Hi Afsharipour,

• Have you tried to perform a system recovery of the computer?

Step 1: Perform System Restore.

I would recommend that you may perform a System Restore on the computer to restore back the previous settings.
To restore the operating system to an earlier point in time, follow these steps:

1. Click Start, type system restore in the Start Search box, and then click System Restore in the Programs list. 
If you are prompted for an administrator password or confirmation, type your password or click Continue.
2. In the System Restore dialog box, click Choose a different restore point, and then click Next.
3. In the list of restore points, click a restore point that was created before you began to experience the issue, and then click Next.
4. Click Finish.

Note: When you perform System Restore to restore the computer to previous state, programs and updates you installed are removed.

How to repair the operating system and how to restore the operating system configuration to an earlier point in time in Windows Vista
http://support.microsoft.com/kb/936212

If the issue still persists, go to Step 2.

Step 2: Run System file Checker [SFC] scan.

You may run SFC scan on the computer, the scan replaces the corrupted system files with the original version of Windows Files on the computer; follow the steps below to run the scan:
1.      Click Start, and then type cmd in the Start Search box.
2.      Right-click cmd in the Programs list, and then click Run as administrator.
If you are prompted for an administrator password or confirmation, type your password or click Continue
3.      At the command prompt, the following line, and then press ENTER: sfc /scannow
When the scan is complete, test to see whether the issue that you are experiencing is resolved.
Additional information: How to use the System File Checker tool to troubleshoot missing or corrupted system files on Windows Vista or on Windows 7: http://support.microsoft.com/kb/929833

If the issue still persists, go to Step 3.

Step 3: Enable User Access Control and rebuild Icon cache.

The Folder icons customization will not work if User Access Control [UAC] is turned off on the computer.
I suggest that you check the UAC settings from Control Panel.
Try right clicking on the desktop>>View>>make sure Show Desktop Icons is selected.

This can be related to instable IconCache or corrupted IconCache. Have you tried to rebuild the IconCache?

Rebuild the IconCache:

1. Click Start, and open Computer.
2. On the Tool bar, click Organize and choose Folder and Search Options.
3. Click the View tab, and check to select the Show hidden files and folders option, and click OK.
4. Then, enter the following directory: %USERPROFILE%\AppData\Local 
5. Find the IconCache.db file, and rename it as IconCache.db.old.
6. Close the window, and restart the computer.
