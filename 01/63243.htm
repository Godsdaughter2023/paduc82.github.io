[quote user="A Angry Sloth"]since he turned off my router they disappeared.[/quote]

I recently replaced my HP laptop with a Dell Vostro 3450 and I suddenly found that 'Play To' feature wasn't working with my Samsung LED TV UA55C8000 .To confirm that it wasn't a firewall or format related issue I even used Samsung PC Share software on my PC to act as the DLNA server...With PC share as the server WMP12 didnt find any problems 'Contacting the Media Server' ...but with its own library it just remained stuck there, with an eventual failure message. 1. Closed WMP12.

2. Go to Services [Control Panel->System and Security->Administrative Tools->Services]

3. Locate the service with name 'Windows Media Player Network Sharing Service'

4. Stop the Service manually.

5. Locate the service 'UpnP Device Host'. Stop the service and start it again.

6. Start 'Windows Media Player Network Sharing Service' now.

7. Close control panel and all associated windows. Open Windows Media Player...right click on a playlist and see if PlayTo option lists the TV. Try playing now...it started playing for me at this stage!

Obviously there is a small chance of a malware disguising or hiding in the file in question, but

false positives are fairly common with Anti-virus programs. Norton is fairly notorious.

If you downloaded the drivers from the AMD site there should be no need to worry about malware.

- Try running a cross check with Malwarbytes .

Sometimes you just have to disable both the Firewall and AV temporarily when dealing with stuff

that gets blocked by the AV.

Or... go through the hoops of allowing an exception for whatever gets blocked that you know is trustworthy.

Malwarebytes : Free anti-malware download

There are several generic files for both AMD and Nvidia graphics that live in System32 and SysWOW64. Some won't delete because they are Windows System files, but there is a small

possibility a malware is blocking them from being deleted (or lives here because the file can't be

deleted).

(In the past (XP) some of the Windows graphics  files for AMD and Nvidia could be deleted, but

were reinstalled when rebooting into Windows. Win7 may do similar).

- The safer option to deleting, is to move, or rename the files before deleting. If a new file gets installed by Windows when you re-boot, or by the drivers you are installing,

you  can safely delete the old file.

- Generally the only time you need to get down and dirty with graphics driver removal is when

the current driver install is corrupt, or when switching between AMD and Nvidia.

One of the better programs for this is Driver Fusion.

Treexy external page

- First uninstall the drivers  via Programs and Features - reboot.

- Run Driver Fusion, and being careful to select only the AMD graphics driver, allow Driver Fusion

to delete any remaining files and registry entries - reboot.

- Install only the graphics driver components you use ( eg - If you don't use HDMI don't install the

AMD HD audio drivers),  and... reboot.

- If you have not already done so - disallow the 'Optional' graphics driver update in

Windows Updates from installing graphics drivers.

This update is commonly out of date, and if you let Windows Updates install automatically

it can corrupt the drivers.

- I've never had an OpenGL problem and don't know what leads to problems, but Windows

errors  can be misleading. 'OpenGL is missing' could result from a number of  issues,

even when the file is not missing.  A corrupt file or program install are two examples.

- The problem could also be related to the project you are working on with this software.

I've never used this software, but when modding Oblivion similar problems (missing textures, etc.), were due to user error.

- Also, depending on the age of the graphics card, new drivers may not be the best for your

system with that software. If an earlier driver was working well, reinstall that.
