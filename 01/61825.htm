lemp say:
Desktop.ini is a system-created file that determines the appearance and behavior of "special" folders.  For example, it can specify that the icon for the "My Music" folder shows a musical note rather than just the simple folder icon.

You don't see desktop.ini because by default Windows hides certain files and folders.  If you want to see these files and folders, click the Tools menu in Window Explorer, select Folder Options and then the View tab.  There are two options that you can change:  "Show hidden files and folders" and "Hide protected operating system files (recommended)."

Be careful.  If you do show hidden files and folders and/or protected operating system files, make sure that you know what you're doing before you move or delete them.

In the case of desktop.ini, if you move it to a new location (for example, by using "Select All"), the behavior and/or appearance of the destination folder may change.

  6. Repeat step 5 for all of your user-created folders

lemp say:
The "System" attribute of a file is separate and distinct from the "hidden" attribute.

Windows search will find files with the "System" attribute set if and only if the option to "Hide protected operating system files" is DEselected.

It's not necessary to do a search. 
  • Pick a folder where you experience the warning message.
  • Click Tools > Folder Options > View and confirm that there is no check in the box to "Hide protected operating system files" AND that the radio button to "Show hidden files and folders" is selected.  Both of these settings must be made in order to view a file like Desktop.ini that has BOTH the "hidden" and "system" attributes set.
  • Click View and select List and then select Arrange items by > Name.  If the item at the top of the list starts with a letter towards the end of the alphabet, click View > Arrange items by > Name again. 
  • Note that any sub-folders will be at the top of the list.
  • Scroll down to the "D"s and look for Desktop.ini.  Note that unless you have UNselected the option in Tools > Folder Options > View to "Hide extensions for known file types," the file will show simply as Desktop.

Group policy should have nothing to do with this -- unless you've managed to find and change some obscure setting.

Do you really get the error message about Desktop.ini when attempting to copy ONE file?  If not, perhaps an explanation of what you're doing might help get to the bottom of your issue.
