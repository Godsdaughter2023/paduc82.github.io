WoW64 (Windows-on-Windows 64-bit) is a subsystem of the Windows operating system that is capable of running 32-bit applications and is included on all 64-bit versions of Windows — including Windows 2000 Limited Edition, Windows XP Professional x64 Edition, IA-64 and x64 versions of Windows Server 2003, 64-bit versions of Windows Vista and Windows Server 2008, as well as 64 bit editions of Windows 7 and Windows Server 2008 R2. WoW64 is designed to take care of all the differences between 32-bit Windows and 64-bit Windows, particularly involving structural changes to Windows itself.

 
The WoW64 subsystem is a lightweight translation layer that has similar interfaces on all 64-bit versions of Windows. Its primary purpose is to create a 32-bit environment that provides the interfaces required to allow 32-bit Windows applications to run unmodified in the 64-bit system. Technically, WOW64 is implemented using three dynamic-link libraries (DLLs): Wow64.dll, which is the core interface to the Windows NT kernel that translates between 32-bit and 64-bit calls, including pointer and call stack manipulations; Wow64win.dll, which provides the appropriate entry points for 32-bit applications; and Wow64cpu.dll, which takes care of switching the processor from 32-bit to 64-bit mode.

turn off apps running in the background - Microsoft Community

rajeswara raob v v say:

turn off apps running in the background

how do I turn off unnecessary  apps running in the background ?  

a. user say:

That's an excellent question. There are several ways to turn off apps running in the background.

The best way is to open the app's settings (sometimes called Options or Preferences) and look for the setting that prevents the app from starting automatically with Windows.

The next best way is to contact the app's developer and ask how to prevent the app from running in the background.

These two methods are best because you have (or will have) a solution that is supported by the developer.

The next two solutions are effective, but they interfere with the normal functioning of the app, which is something you want to avoid if possible.

You can use Autoruns or a similar program to prevent the app from starting with Windows by editing the Windows registry. Let Autoruns edit the registry - don't try to do it yourself.

Finally, you can use Task Manager [CTRL+ESC+DEL] to End task the app, which will stop it from running in the background, but it will start again the next time you restart your computer. Moreover, killing an app that's running can sometimes have unpredictable results, and not always good unpredictable results.

End tasking is best applied when you come up against one of the many scams on the internet claiming that your computer is infected or your license has expired. Just End task your web browser.

BlueScreenView tries to locate the right driver or module that caused the blue screen by looking inside the crash stack.  However, be aware that the driver detection mechanism is not 100% accurate, and you should also look in the lower pane, that display all drivers/modules found in the stack.
