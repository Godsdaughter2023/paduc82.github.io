Boot.ini tab is missing in MSCONFIG? - Microsoft Community

ivan arno say:

Boot.ini tab is missing in MSCONFIG?

I found this answer in a different website and it worked!

When you start MSCONFIG.EXE (System Configuration utility), the BOOT.INI tab which usually appears, may be missing.

This may simply be a file permissions issue. You can verify that by typing C:\Boot.ini in Start, Run dialog. If you get Access Denied or similar error, that explains the problem.

Here is another obscure possibility:

  • Open Regedit and navigate to:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Shared Tools\MSConfig

  • Backup the key by exporting it to a REG file

  • In the right-pane, delete the value named Boot.ini  (if found)

  • Close Registry Editor and open MSCONFIG.

Hope it works for anyone else that needs it.

SOLVED: Synaptics touchpad driver will not install on brand-new Dell with Windows 10 pre-installed

The Synaptics driver was not installed for my brand-new Dell laptop computer (Inspiron 3152) with Windows 10 pre-installed. As a result, I am unable to set touchpad features such as scrolling or automatically disabling the touchpad when typing.

I have contacted the laptop manufacturer, but they have been unable to fix the problem. It appears to be a compatibility issue between Windows 10 and the Synaptics driver.

According to the laptop manufacturer the correct driver is Synaptics driver version: 19.0.13.0, which can be downloaded from: http://www.dell.com/support/home/us/en/19/Drivers/DriversDetails?driverId=8DC47&fileId=3485595818&osCode=WT64A&productCode=inspiron-11-3152-laptop&languageCode=EN&categoryId=IN

Alternatively, a higher version (Windows 10 PS/2 and SMBus Devices ver, 19.0.19.1) can be downloaded from the Synaptics website: http://www.synaptics.com/en/drivers.php

When Dell technical support attempted to install the correct driver, the installation says the Synaptics driver was installed correctly, but only the generic Windows PS/2 compatible mouse driver remains. Neither the driver from the Dell website nor the Synaptics website works.

When attempting to access Windows touchpad additional mouse options, the following error appears: "Unable to connect to the Synaptics Pointing Device"

If I manually install the driver from the Device Manager, the touchpad does not function at all until I rollback to the generic Windows PS/2 compatible driver.

SOLVED: 

I was finally able to get my Synaptics touchpad working by running the Synaptics 19.0.19.1 64-bit Windows 10 driver setup and then manually updating the driver following this guide:
http://dellwindowsreinstallationguide.com/dell-touchpad-drivers/ 

(Beforehand, I uninstalled the Synaptics program, stopped the Synaptics processes and services, and deleted the Synaptics files in the Program Files and ProgramData folders. I don't know if these steps are necessary though.)
