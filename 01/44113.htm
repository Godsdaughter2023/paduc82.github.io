Hi acc_55,

The error “server did not register with DCOM within the required timeout” is signaling that one or more DCOM (Distributed Component Object Model) software is failing to register itself in a reasonable period of time. This normally occurs on devices connected on a server.

In the meantime, I suggest you to change startup type of Function Discovery Resource Publication as an initial troubleshooting. Simply follow these steps:

  1. Press Windows key + R to open a Run window.
  2. Type “services.msc” and hit Enter.
  3. Scroll down through the services name and find Function Discovery Resource Publication. Right-click on it, then choose Properties.
  4. In Properties under the General tab, expand the drop-down menu near Startup type and set it to Automatic (Delayed Start).
  5. Click Apply and Restart PC.

For additional troubleshooting, temporarily disabling 3rd party firewall if you have, and see if the issue will persist.

Who needs a Skype Number for one-way calls only? The caller or the - Microsoft Community

sabine d'haeseleer say:

Who needs a Skype Number for one-way calls only? The caller or the person who receives the call?

Hello - I want to make one-way only phone calls from an Australia mobile/landline to a Belgium landline (the person in Belgium will not make calls to Australia, only receive calls from Australia, on their landline). I will opt for a Skype subscription (instead of Credit or SkypeToGo). My question is: Do I create a Skype number for myself in Australia or for the person who receives the calls in Belgium (I don't want to create a Skypenumber for the person in Belgium)?  Thanks heaps in advance.

techfreak_ say:

In your example there would be no real benefit to purchasing a Skype Number. A Skype Number is only used for people to reach you using a standard phone, not the other way around. The only other reason you might want to purchase an Australia Skype Number is to possibly appear like you are calling from Australia. However, Caller ID is not available for Australia Skype Numbers, so that cannot be accomplished by purchasing a Skype Number. Verifying your Australia mobile number would allow you to use its Caller ID when placing calls through Skype or Skype To Go.

If you only want to contact the person one-way, it would be best to simply purchase an unlimited subscription to Belgium landlines. You would use the Skype client to place those calls and appear to be calling from your Australia mobile, if you verified and enabled Caller ID in your Skype settings. If using the Skype client is not possible, the only alternative is using Skype To Go to place the phone calls to your Belgium contact's number. If a Skype To Go local access number or local Skype Number for that contact is available, you would either dial the access number, then dial the speed dial fro the contact's number when prompted. or dial the direct Skype number available to reach that Belgium contact. Your unlimited subscription would cover the cost of the phone calls. If the closest Skype To Go Access number or Skype To Go number for the contact is not a local call, you would be charged by your mobile provider whatever the rates are to reach that non-local number for the duration of the call. You can set up your Skype To Go prior to purchasing anything to see if there is a local access number or direct phone number offered that will allow you to reach your Belgium contact.

Skype Client ----> Belgium contact using Belgium landline subscription.

-- or --

(Australia Mobile) phone call ---->  Local Australia Skype To Go Access Number  ---> speed dial prompt for contact's number (using Belgium landline subscription)

-- or --

(Australia Mobile) phone call ---->  Local Australia Skype To Go Number for contact (using Belgium landline subscription)
