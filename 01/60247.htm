lemp say:

The way utilities like KillDisk and DBAN work is a several step process.

  1. Create a bootable CD with the downloaded file.  KillDisk includes such a utility (Boot Disk Creator) bundled with the downloaded files.  You obviously have to do this on a working computer.
  2. If necessary, configure the computer's BIOS to make it "boot from CD."  On some computers, a small menu will pop up just after power-on that will let you select the boot device.
  3. If the hardware is operational -- i.e., the reason why the computer "no longer recognizes the C drive" is that your Windows installation is corrupted -- the utility will start using its own operating system and attempt to recognize the drive(s) installed in the machine.

The KillDisk Users Guide is here:  http://www.killdisk.com/guide.htm

You can start with section 3.

I note that the KD user guide mentions that you need to register the program before making the bootable CD.  I don't know if that applies to the free version -- if it does, you'll have to decide whether to pay for the registerable version or use DBAN instead.  DBAN just provides an ISO file for download.  You then create a bootable CD from the ISO using either a native Windows application (for Windows 7 or newer) or free ImgBurn or CDBurnerXP.

¡firedog say:
I setup POP3 in Windows Live Mail ... How do I setup the email so the sent emails are also stored online as well?


POP3 is a method of retrieving email from a mailbox. All it does is look for messages newly arrived in the mailbox and deliver them to the inbox on your computer. It has nothing to do with sending email.

If you set up an account in the mail client (in this case Windows Live Mail) to use POP3, it will use a different method - SMTP - to send mail. Some types of account, notably Gmail, will store sent messages in the Sent mail folder you see when you access the account in webmail. Most don't do this.

If your mail service supports IMAP - yet another method of receiving and sending mail - then you can select which of the account folders to synchronize between client and server, including Sent items. Outlook accounts (@hotmail, @msn, @live, @outlook etc.) configured to use HTTP automatically (and non-configurably) synchronize all folders between client and server.

So, you can set up your account to save sent email on the server if the mail service supports it, otherwise not. Who is your mail service provider (usually obvious from the part of your email address after the '@' symbol)?

keith a. miller say:

First & foremost, that gray square in the 'Read-only' checkbox is not a status indicator --- it always appears in this 'neutral' state when the Properties dialog is first displayed for any folder. Furthermore, a folder having it's Read-only attribute set does not prevent saving or editing files in that folder (it is used to tell the OS to process the desktop.ini file). If you have problems accessing/sharing files, focus on Permissions.

As for losing folder view settings, if you are opening the same folder & its view settings aren't remembered, you've possibly reached the limit of saved views (5000) and Windows is starting to delete one view to make room for another. To check this, copy & paste the following command into a PowerShell Window:

((gp "HKCU:\SOFTWARE\Classes\Local Settings\Software\Microsoft\Windows\Shell\BagMRU").Nodeslots).count

It will return the number of saved views in the Registry. If you've maxed out, we can clear those.

Also, you can set the default view settings for each folder type(General, Documents, Music, Pictures, Videos) by using the 'Apply to folders' button found on the 'View' tab of the Folder Options dialog:

You have to do this once for each folder type you want to customize.

Also, the 'Apply this template to all subfolders' setting only controls folder type & icon mode, not Permissions. Permissions are under the 'Security' tab in the folder's Properties dialog.
