Auto zip files based on name - Microsoft Community

weebleroxanne say:

Auto zip files based on name

I have a huge karaoke archive that I'm wanting to compress, not only to clear up some space but to also make things a little neater. Right now most of the songs come as two files, an mp3 and a cdg file and are named the name (songA.mp3 and songA.cdg, songB.mp3 and songB.cdg, etc).

Is there any way to automate zipping all of these files so songA.mp3 and songA.cdg end up in songA.zip?

robertjwin say:

Assuming all MP3s and CDGs are in the same folder, it's simple. Just follow the instructions:

1) Download this file and extract zip.exe from inside it to your MP3/CDG folder.

2. In File Explorer with your MP3/CDG folder open, go to File and click on Open command prompt.

3. At the command prompt copy and paste the following command and press Enter:

for %f in (*.mp3) do zip "%~nf.zip" "%~nf.mp3" "%~nf.cdg"

4. Wait for all the MP3+CDG pairs to be zipped up, test them and then you can get rid of the MP3s and CDGs if you want.

Note: You must have enough free space to create all the ZIP archives.

1. the status bar of file explorer: If I select some files, the status bar won't show how many files/folders in the folder I have selected.  Also, the number of how many files are in the folder is wrong. It may work sometime, but most of the time it looks like "being frozen".

These are examples of performance problems after an update which could be changed with as little modification to your OS as doing an extra full Restart.  E.g. if the only Restart is whatever happened after your updates you should definitely do another one.

Another cause of performance problems is having an interferer present when the update is installed and applied.  Examples are third-party security packages and one that has often affected me, Hyper-V.  I don't know why it has such a significant effect on performance but I always at least stop the Hyper-V service before allowing a cumulative update.  However, I have also found it necessary to uninstall the update, uninstall Hyper-V, and then allow the update to proceed without that service even being installed.  Ditto for MalwareBytes in my mother's machine.  Typically that restores performance after the update.
