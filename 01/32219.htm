  • Reboot the machine with a Windows Repair CD, then use System Restore to set Windows back by a few days (works only if he changed the password recently).
  • Log on under a spare admin account, then reset the password to your usual account (works only if he set up a spare admin account).

Hi Aussiemight,

 

This problem occurs because the backup process detects that system drive is dirty.

 

To resolve this problem, run the disk check (Chkdsk.exe) to repair the errors on the system drive. To do this, follow the steps mentioned below:

 

 

Step 1: Run the disk check at the command prompt

 

1.    Click Start, click All Program, click Accessories, right-click Command Prompt, and then click Run as administrator.

 

2.    At the command prompt, type chkdsk system_volume:/f, and then press ENTER.

 

For example, type chkdsk c:/f.

 

3.    If you receive the following message, type Y, and then press ENTER:

 

Chkdsk cannot run because the volume is in use by another process. Would you like to schedule this volume to be checked the next time the system restarts? (Y/N)

 

4.    Restart the computer to start the disk check

 

 

Error message when you try to perform a system backup operation on a Windows Vista-based computer: "Catastrophic failure (0x8000FFFF)"

http://support.microsoft.com/kb/973558

 

If the issue persists,

 

Step 2: Make sure “Microsoft Software Shadow Copy Provider” is set to automatic and started. To do this, follow these steps:

1.    Click on Start-Run, type services.msc, the click on OK.

2.    When the Services window loads, navigate to the " Microsoft Software Shadow Copy Provider ", the double click on it to open it,

3.    Make sure the "Startup Type" says "Automatic" and started. If it doesn't, use the drop down menu to change it.

4.    If it already says: "Automatic", then click on the "Recovery" tab at the top, and change the "First Failure" line to "Restart the Service", then click on "apply", then OK.

Follow the same steps to start “Volume Shadow Copy service”

frederik long say: Windows ignores the read-only attribute of folders. It has no effect.

keith a. miller say:

Are you having any issues creating or moving files? If so, that's a permissions issue.

If you've just noticed the Read-Only section of the Properties dialog, then you should understand that the square bullet within the checkbox is not a status indicator. If you want to know which files & folders have the read-only attribute set, add the Attributes column to the Details view in Explorer.

When the read-only attribute is set for a folder, it doesn't affect write permissions, it tells Windows to apply the settings found in the desktop.ini file (custom icon, display name, etc.)
