I'm afraid it turned out that this is a fundamental limitation in the Skype infrastructure.

The chat history/call log is only updated by a running client, thus if Skype is not running, no missed calls will be logged into the chat/call history.

There is a second system that does log missed calls when logged out.

This system shows up only in the windows notification when I start the classical windows Skype, including version 8.

The problem is that the second system doesn't talk to the call log and thus we only get the windows notification. When the notification disappear after a few seconds it's gone. The second system although already very limited has further limitations.

These notifications only show up on the windows Skype version 8.

They don't show up on any other clients such as the windows 10 store version or the android version.

Worse is that if you start one of these version before the version 8, the missed call notification is consumed without being shown and when you start Skype version 8 no notification is left to show.
