tryx3 say:

That MS download site only provides Windows 7 ISOs for those holding a full Windows 7 "Retail" licence.

You'll have to buy a Windows 7 licence. 

  • The licence will almost certainly be of the type "OEM" aka "System builder" and will be tied to the first computer it is installed on.  It will come with a DVD that will be either 32 bit or 64 bit [but not both] so make sure you buy the right type for your needs.  Nobody is allowed to sell Windows 7 licences without providing the installation media so if somebody offers you a Windows 7 Product key without a DVD, reject it - it will be fake.  There are lots of fakes around - only buy from a reputable seller who is going to be around for another three years for you to complain to if Windows starts off looking Genuine but then gets detected as a fake licence later on {this happens a lot}.
  • Windows 7 "Retail" licences can be transferred to another computer in the future but are very expensive and hard to find.  Retail licences come with both a 64 bit DVD and a 32 bit DVD.

Before even considering this, think about the hardware you have. 

  • Review Windows 10 Specifications and System Requirements [earlier versions of the reqs also said that the CPU version of the page].
  • You will need to find Windows 7 drivers for your hardware.  Go to the Dell support site using your Dell Service Tag then change the OS selection box to Windows 7 to see what they provide. 
  • Then see Where to find hardware drivers for further guidance. 
  • You might well find that there are apparently suitable drivers for all your hardware but the only way you can confirm this is by installing Windows 7 and trying it all out.  That could prove to be a costly & time-consuming experiment so search this forum and elsewhere for reports by other users of your computer model to see how they got on.

If it is any encouragement, I upgraded my 11 year old Dell Inspiron 9300 from XP to Windows 10 and it has only a couple of minor shortcomings [Task scheduler cannot wake it from sleep, the PowerCfg -BatteryReport utility does not work].

Hello Bob,

 If you use the Recovery drive to store files, or if a backup or restore program uses the Recovery drive to store files, the drive quickly becomes full and can cause serious problems when you need to use system recovery. It is best that you don't store files on the Recovery drive.

To stop Low Disk Space errors, ensure that System Protection is turned off for the Recovery drive, and remove files that are not related to system recovery from the drive. Perform the following steps:

  1. Move your mouse pointer to the lower left corner of the screen (or Start menu button), right-click, and select System from the menu.
  2. Click System Protection from the left side of the window.
  3. In the "Protection Settings" section, look for the Recovery drive in the list of Available Drives. If the drive is listed, check whether protection is turned off or on for the drive.

  • If protection is turned Off, close this window.
  • If protection is turned On, click the Recovery drive name to highlight it, and then click Configure. Select Disable system protection, and then click OK. If a confirmation message appears, click Yes or OK.

     4. Move your mouse pointer to the lower left corner of the screen, right-click, and select File Explorer from the menu.

     5. Before you can remove any backup, restore, or personal files and folders that might be stored on the Recovery drive, you need to be able to view hidden system files, as follows:

  • In the menu bar of the File Explorer window, click the View tab, and then click Options.
  • In the Folder Options window, select the View tab.
  • Select Show hidden files, folders, and drives, and remove the selection from Hide protected operating system files. If a Warning message appears, click Yes or OK to confirm. Click OK to close the Folder Options window.

    6. From the Explorer window, double-click the Recovery drive.   

    7. If a Window opens stating "You don't currently have permission to access this folder," click Continue.

    8. If you previously created or copied any files into the Recovery drive and you want to keep them, copy the files to another drive - such as the C: drive.

    9. Find and delete any files that you previously saved in the Recovery drive. Select the files or folders, and press Shift + Delete to permanently remove the files.

Look for any folders that might be associated with a backup program on your computer. If a backup program is saving files to your Recovery drive, move the folder to a different location, and change your backup settings so that the files save to a different drive.

Do not delete any files that are associated with system recovery! Do not delete the items: $RECYCLE.BIN, boot, hp, EFI, Factory Update, preload, Recovery, RM_Reserve, system.sav, bootmgr, RMCStatus.bin, BT_HP.FLG, CSP.DAT, DeployRp, HP_WSD.dat, HPSF_Rep, language, or RPCONFIG! Doing so can prevent a future system recovery from the hard drive. If you are not sure, do not delete the file.

    10. Several confirmation windows open. Click Continue, and Yes until the files and folders are deleted.

    11. Set Windows Folder Options back to their original setting as follows:

  • In the menu bar of the File Explorer window, click the View tab, and then click Options.
  • In the Folder Options window, select the View tab.
  • Select Don't show hidden files, folders, or drives and select Hide protected operating system files.

    12. Close Explorer. The Out of Disk Space errors should no longer appear.
