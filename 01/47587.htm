That's weird. It shouldn't say "limited connectivity" if it's just with the DNS. But anyways, you can try changing the DNS setting of your network.

DNS = Domain Name Server. It's basically just the "Yellow Pages" of the Internet which transforms a "domain name" to an IP address (e.g. if you type "answers.microsoft.com" on your address bar, it translates it to an IP address like 23.50.18.59) That way, people won't have to remember IP addresses when connecting to the web servers. But that's not really your concern. Nuff said. Here's what you really need to do:

  • Clear your DNS Resolver Cache. Press Windows key + X > click Command Prompt (admin) > type ipconfig /flushdns > hit Enter. Check if that resolves the issue with the DNS. If not...
  • Use other common, but more reliable, DNS. Well, it's a lengthy one to write so I just did a quick search and found this Wiki article on How to change IPv4 DNS server address to public DNS in Windows. It wouldn't state that it's applicable for Windows 10, but it actually is. Just follow the same steps provided for Windows 7, 8, and 8.1 and you should be good to go.

If changing the DNS won't do the trick, then maybe we need to look into something else.

spiritx say:
Hi,

Unfortunately Windows Updates often finds older certified drivers. So the best method is to stop
the automatic updating of drivers while leaving the other updates on automatic. Then since updated
drivers can be very important to a maturing OS the answer there is to update them manually. New
computers ship with early stage drivers so they really need to be updated fairly often especially
when there are major changes in the driver platform which is true of both Vista and Windows 7.

Part of the problem is since the driver platform changed and there is a maturation process Microsoft
is over-whelmed with driver updates to put through the certification process. So by the time a driver
passes certification a maker probably has released several major updates to that one. There now
being 32 and 64 bit platforms just adds to the certification load.

    Hi Chicocuban,

    Here are the places where you can look for the Bitlocker recovery key:

    • On a printout you saved. Look in places you keep important papers.

    • Saved on a USB flash drive. Plug the USB flash drive in to your locked PC and follow the instructions. If you saved the key as a text file on the flash drive, use a different computer to read the text file.

    • In your Microsoft account. To get your recovery key, go to BitLocker Recovery Keys.

    • In your Azure Active Directory account. For work devices where you sign in with an Azure Active Directory account, to get your recovery key, see the device info for your Microsoft Azure account.

    Also, if your device is connected to a domain (usually a work or school computer), ask a system administrator for your recovery key.

    If you still can't get in, you'll need to reset your device. Please refer to this article on how to reset your Surface to factory conditions using a downloaded recovery image.
