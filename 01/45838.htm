cmdrkeene say:

I think File History would do what you want -- it's making a daily backup of your Documents, and only makes an extra copy if the original has changed in some way.  If you didn't open/modify any of those files for a week, it wouldn't make 7 identical copies because it already has a backup of the newest one.

It can't be browsed in the file system, it can only be browsed using the File History interface, which reads the manifest file to determine which versions to display as you navigate forward and back in the timeline interface:

I didn't actually modify all these files on Sunday at 8:52 PM.  In fact, most haven't been modified in many months.  So although I see it on all the days in the timeline, there's really not this many extra copies of it.  It's intelligently using the hard drive by only storing extras when there's something new.  So if I went back in time to October 12th (the real time I last modified this PDF file), I could restore it to what it looked like on that day.  Or I can even open this file here and then navigate among the versions directly, rather than just browsing the timeline.

---

If you just want to make an extra copy of your Documents on a thumb drive every day, you have a few options. I like File History, but maybe it's overkill for your needs.  But it has other benefits -- such as caching the backup until you plugin your drive next time so it can write all the backups.   You could use a batch file, set with Windows Task Scheduler to fire every day, but it would fail if the flash drive wasn't connected at that time (something File History would take care of for you).

You could write the batch file to make a folder with the current date, then copy the whole document folder into it, every day.  When the flash drive ran out of space you'd have to manually take care of it.  This would also cause a lot of duplication as I said above, so another option is to use ROBOCOPY command and only copy the files that have changed, overwriting the previous backup.

Another tool that does this job is one that has a strong following of enthusiasts, called Microsoft SyncToy.  Read up on that you may like it better than any of the above.

cmdrkeene say:

It's because it uses versions to keep multiple copies of the same files so you can explore back in time and pick the one you like the most.  It's just like Apple Time Machine.

You should never try to browse these files using the normal file browser.  Use the File History interface only or you will corrupt your backups.  Even just clicking the "administrator continue" option to access the files can change the ownership and break your backups.
