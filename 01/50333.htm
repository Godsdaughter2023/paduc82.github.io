a. user say:

Ah yes ... whether they're called 'Microsoft Support Engineers' or 'Microsoft Forum Moderators' they're still just clueless script readers sending unsuspecting Windows users on time-wasting wild goose chases.

The built-in Administrator is not a user account - it's a system account. You need to run Windows from a user account.

The built-in Administrator is your lifeline in the event that you can't sign in to Windows any other way. You should never use it for day-to-day computing and never store files there. It should always be at default configuration settings.

If you don't have a user account with Administrator permissions, use the built-in Administrator to create one. Then sign out of the built-in Administrator and disable it. Sign in to the Administrator user account and use it to create a Standard user account. That's the account you should be using for day-to-day computing. The Administrator user account is only needed for tasks that require Administrator permissions.

If you continue to use the built-in Administrator you will find that it cannot do many of the things you want it to do, because it was not intended to work like a user account.
