That file msvcp90.dll or files like it are usually not something you just replace or recover (a System Restore might work maybe).

These files usually fall into the area of influence by the Windows XP implementation of Side by Side (SxS) technology.

Most of the time, DLL files would be found in the C:\windows\system32 folder, but sometimes there is a need (depending on the applications you have installed) to have two different DLLs with the same name on one computer side by side at the same time.  Ouch!   How are you going to do that?

Since the names of the files are the same they can't both be in C:\windows\system32 at the same time (this affliction is called DLL ____) and you might not be able to just overwrite one and replace it, so XP is able to keep multiple copies of the file with the same name in some other folders in the Windows Side By Side directory, something like: 

C:\windows\WinSxS....

XP will keep track of the different versions and swap them in and out pulling the files on demand as needed.

If you search your system for the missing file, you may find a file of that name in many other folders, but you need to put the right version of the missing file in the right folder.

For example, the Microsoft .NET Framework might use one version, Adobe products might use another version, Visual Source Safe might use still another
version, etc.

If you look in the C:\windows\WinSxS folder, you may see a lot of subfolders that look similar with the first part of the folder name but then have some cryptic characters for the rest of the folder name.

If the case of the MSVCP90.dll file, you might see some subfolders that look like this:

C:\windows\Winsxs\x86_Microsoft.VC90.CRT_1fc8b3b9a1e18e3b_9.0.21022.8_x-ww_d08d0375
C:\windows\Winsxs\x86_Microsoft.VC90.CRT_1fc8b3b9a1e18e3b_9.0.30729.4137_x-ww_d494ac0e

Under each of those folders would be the correct msvcrt90 files (or several files) needed for a particular application.  When the application needs one of the files (or several files), Windows XP will pull the files it needs on demand from the correct folder that matches the needs of the application that needs them.

If some application is complaining about one of the Windows Side by Side files is missing, you need to determine what the application is and you may have to reinstall it to restore the contents of the Windows Side by Side folders.  More information is needed.

In the case of these Side By Side kinds of files, you just can't download or copy the missing DLL file into your C:\windows\system32 folder with the rest of the DLL files, since the application will be looking for it in the one of the Windows Side by Side folders.

Each event is sorted by Date and Time.  Errors will have red Xs, Warnings will have yellow !s.

How to Find SSID (Network Name)

  1. Plug the computer into a LAN port via an Ethernet cable.
  2. Open a web browser.
  3. Log into the router by entering the default gateway into the address bar of the internet browser. Example 192.168.1.1
  4. The router will then prompt you to enter the username and password. The default user name and password are: "admin" and "password"/"admin"/"password1".
  5. Select Configuration from the tabs across the top.
  6. Select Wireless from the menu on the left.
  7. Select Basic from the submenu.
  8. The Wireless Network Name (SSID) will be listed next to Network Name (SSID). The network name can be changed by entering the new name here.
  9. If the network name was changed click the Save button.

 

 

Wireless Passphrase

  1. Plug the computer into a LAN port via an Ethernet cable.
  2. Open a web browser.
  3. Log into the router by entering the default gateway into the address bar of the internet browser. Example 192.168.1.1
  4. The router will then prompt you to enter the username and password. The default user name and password are: "admin" and "password"/"admin"/"password1".
  5. Select Configuration from the tabs across the top.
  6. Select Wireless from the menu on the left.
  7. From the submenu select Security.
  8. The wireless passphrase will be listed as "Key 1" or "WPA Shared Key". If a new passphrase is desired a new one can be entered here.
  9. If a new passphrase was entered make sure to click the Save button at the bottom of the page.

 

 

Changing Wireless Security Type/Disabling

  1. Plug the computer into a LAN port via an Ethernet cable.
  2. Open a web browser.
  3. Log into the router by entering the default gateway into the address bar of the internet browser. Example 192.168.1.1

MAC Filtering

  7. Select MAC Filter from the submenu.
