adrianne buc say:

Bootmgfw.efi is a firmware boot manager the manages the boot process. This file could be either missing or corrupted which causes Windows not to boot. This can be resolved by peforming startup repair which requires a system image backup or a Windows 10 installation media. If you don't have one, you can create an installation media on a different computer using the Windows 10 media creation tool from this link. You can put the installation media on either USB flash drive or DVD.

Once the installation media is ready, you may have to access the BIOS/UEFI firmware setting of your computer to set the USB flash drive as first in boot priority settings. That way, your device will boot from the installation media instead. Once it's up, follow these steps:

  1. On Windows Setup screen, select your language preferences, and click Next.



  2. Click Repair your computer at the bottom.



  3. On the Advanced Startup screen, select Troubleshoot > Advanced options > Startup Repair.

If the above steps did not resolve the issue, try repairing the MBR (Master Boot Record) by following the steps below:

  1. On the Advanced Startup screen, Select Troubleshoot > Advanced options > Command Prompt.
  2. On the Command Prompt window, type the following lines and press Enter after each line.

    bootrec.exe /rebuildbcd
    bootrec.exe /fixmbr
    bootrec.exe /fixboot

A Custom Image is more up to date than a normal System Image.  A System Image is virtually your day one settings, it is the default Image used when you do a Refresh of Windows.  The Custom one allows you to create a System Image that is current and up to date so that you can restore the later Image of Windows via a Refresh of Windows.
