    1) Check and compair the BIOS settings between the one that works and the motherboard on the 2nd that does not recognize the hard drive.

    2) The Maxtor is an older drive and may only be a SATA-I drive where the Hitachi Deskstar 30500IDK is a SATA-II drive. If the Maxtor is SATA-I, some motherboards can not "Auto Negotiate" and automatically drop back to SATA-I mode. Check the BIOS on the outside chance there is a BIOS setting that lets you drop back to SATA-I speed or try Legacy, IDE, modes.

    3) If you have more than one drive, disconnect it.

    4) Check Seagate/Maxtor's web site and download their drive diagnotic utility, look to see if there is a option to write zerros to the entire drive or better still the first and last part of the drive (its quicker and the zerro write effectively sets the drive back to as closes to an "New out of the box" condition).

    First, check in a Windows Explorer window under the View menu that "Thumbnails" is selected.  You might try selecting something else (e.g., List) and then re-selecting Thumbnails. 

    Windows XP has a media and video thumbnail handler (shmedia.dll), but AFAIK, it only handles files of the following types:  .asf, .asx, .avi, .mpe, .mpeg, .mpg, .ogg, .ogm, .wmv

    You can see if the commands described here help, but because .flv and .mp4 aren't in the list above, I suspect that they will not --> http://www.mydigitallife.info/how-to-enable-or-disable-media-and-video-thumbnail-preview-in-windows-xp/

    In the event that the thumbnail handler is still in the system and is either corrupted or needs to be re-registered (as described in the link above), you may be able to determine its file name by using ShellExView  If you run ShellExView, sort the output by "type" (if it isn't already) and look for the "thumbnail" entries.  If you right-click on an entry and select "Properties" you'll get a list of the file extensions that it handles.
