Preparing Your Desktop Loop: What's the Root Issue?

Hi, folks. I started having the "Preparing your desktop..." problem with Windows 7 Professional in the summer. I've been fighting it since and have tried numerous supposed solutions, even one that I kind of discovered on my own, but none of them has kept the problem from resurfacing within a week or two. "Hi, all. This same problem started happening to my Windows 7 Professional system in July. To be clear, out of the blue I started to get the "Preparing your desktop..." message every time I logged into my account. Once logged in I would get the default Windows 7 theme, a bubble message letting me know I had been logged on with a temporary profile, and all of my Windows and program settings entirely gone. Any settings changes I made would be lost on restart. I think it's safe for me to say I've tried all of the advice I could find on the internet. After nearly three months of failed troubleshooting I think I've finally found a simple solution to the problem. The issue appears to be a corrupt user profile registry key. Deleting the key will cause Windows to recreate it, thus purifying your user account without having to reinstall Windows or migrate to a new account.

Here's what I did.

1. Log in to an administrator account.
2. From the Start Menu type 'regedit' (without quotes) and open the registry editor. If your profile is really screwed up this might not be possible. If you can't open the registry editor, try to create a new user account from the control panel, then log into that account and continue.

*** NOTE: At this point it may be a good idea to back up your registry in case you do something terribly wrong. To do this, select 'Computer' from the tree on the left, then hit File and Export. ***

3. Browse to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\NT\CurrentVersion\ProfileList. You should see a few subkeys with short numeric names and then one (or more if you have multiple user accounts) subkey(s) with a longer name filled with numbers. It should look like the image below.



4. Select the longer-named registry key with the ProfileImagePath variable name of the account giving you trouble. Mine is "JM" as you can see in the image.
5. Right-click that key (the entire folder selected from the tree on the left, not the variable) and delete it.
6. Reboot the computer and log in to the account that was giving you trouble. You will see the "Preparing your desktop" message, but don't worry. Windows is simply recreating the registry key you deleted.
7. Open the registry editor and browse back to the same location given in step 3. You will notice that Windows has recreated the registry key.
8. Select the new profile subkey. It will now have a different value for the ProfileImagePath variable and it should look like this: "[Drive Letter]\Users\[User Name].[Computer Name]". Right-click the ProfileImagePath variable and click Modify...
9. In the Value Data field, delete the ".[Computer Name]" part from the end so that only the user name path is in the field. Then press Ok.
10. Reboot the computer once more. Then log back in to your account.
11. Rejoice in the fact that your computer has been healed!!!


One thing I noticed after applying this fix was that I got a few hiccups where it looked like the problem happened again, but I hadn't been logged into a temporary profile; somehow some of the environment variables failed to get set properly. If this happens you won't see the awful bubble message (see top of this post). This can be corrected by rebooting or running a batch file with these commands:

set USERNAME=[User Name]
set USERPROFILE=[Drive Letter]:\Users\[User Name]

where the terms in brackets are replaced with appropriate values. Then log off and log back on (or close explorer.exe from the task manager and open it again).

I would be glad to hear if this works for you. Please leave comments so others can know whether this is a reliable solution! Cheers and good luck!"

Do you get BSOD(Blue Screen Of Death) when you access a particular application?
 
The system goes to a Blue Screen Of Death because there are some exceptions happened in the kernel (a device drivers, backup software or anti-virus services, etc.), and Windows implements this mechanism: When it detects some errors occur in the kernel, it will kill the box in case some more severe damage happens. Then we get a blue screen or the system reboots (it depends on what the system settings are).
 
The IRQL_NOT_LESS_OR_EQUAL bug check has a value of 0x0000000A. This indicates that Microsoft Windows or a kernel-mode driver accessed paged memory at DISPATCH_LEVEL or above.
 
Bug Check 0xA: IRQL_NOT_LESS_OR_EQUAL dd {margin-bottom:0em; margin-left:1.9em; }.divclass {behavior:url(#default#savehistory);}
 
Cause
 
This bug check is issued if paged memory (or invalid memory) is accessed when the IRQL is too high.
 
The error that generates this bug check usually occurs after the installation of a faulty device driver, system service, or BIOS.
 
If you encounter bug check 0xA while upgrading to a later version of Windows, this error might be caused by a device driver, a system service, a virus scanner, or a backup tool that is incompatible with the new version.

Hello Loana,

You upgraded your old laptop to 10.  It received a digital license with that upgrade.  The new laptop also has a digital license.  So you are all set to go, if you want to change your HDD on the old laptop.

If you encounter problems with activation after replacing your HDD on the older laptop, you will simply run the Activation Troubleshooter.
