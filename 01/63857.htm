lemp say:
Win XP (Home) will not let me configure another PC as an ad hoc ethernet connection (so the other PC can use the wireless internet on this one).

Either you're using the wrong terminology or you're going about things the wrong way.  Windows can only create an ad-hoc wireless network.  Windows doesn't do an "ad-hoc Ethernet network."

As I understand your setup, you have:

Computer ONE, running XP Home (service pack ?), that is connected to a wireless network and thence to the Internet

Computer TWO, running (what version of Windows and what service pack), that you would like to connect by Ethernet cable to Computer ONE in order to share Computer ONE's Internet connection.

My first observation is that if by "wireless connection" you mean that Computer ONE is connected to a wireless router (or wireless access point), the simplest solution is for you to get a wireless adapter for Computer TWO and have it connect directly to the same wireless router.

Assuming that you can't or don't want to add a wireless network adapter to Computer TWO, what you want to do is called "Internet Connection Sharing." Unfortunately, the website that I used to rely on to explain this is no longer completely functional, so here are some directions:

CAVEAT:  Windows Internet Connection Sharing forces the LAN adapter (the Ethernet network adapter on Computer ONE that will be used to connect to Computer TWO) to have an IP address of 192.168.0.1.  The ICS software will assign an IP address of 192.168.0.x to Computer TWO.  If your wireless router is configured to use addresses of the form 192.168.0.x (not common, but happens) ICS will (may) not work.  If this is how your wireless router is configured, you'll have to (should) manually re-configure it to use another subnet (e.g., 192.168.1.x).  If you need directions, provide the make and model of your wireless router.

1.  As the page from Practically Networked linked above says, the easiest way to proceed is to run the Network Setup Wizard (you must be logged on as a user with "computer administrator" privileges). 
  • Start > Control Panel > Network and Internet Connections > Network Setup Wizard. 
  • Select the top radio button to permit other computers to "connect to the Internet through this computer."
  • Select your Internet connection (presumably, your wireless network adapter).
  • Leave the Computer Name and description as is.
  • Leave the Workgroup name as is.
  • Select your desired file and printer sharing option
  • "Just finish the wizard."
  • On your second computer, ensure that you configure the Ethernet adapter to "Obtain an IP address automatically."

2.  If you want to do things manually, see http://www.petri.co.il/configuring_ics_on_windows_xp.htm

Run the Audio Troubleshooter

This troubleshooter is an automated Windows 8 inbuilt tool which finds and fixes the common issues with the audio settings of the computer. Perform these steps to run this tool and check if that helps.

  • Press Windows Logo + W keys from the keyboard.

  • Type Troubleshooting in the search bar and press Enter.

Method 2: You may update the Audio or sound driver from Device Manager manually and check if this helps.

1. Press (Windows key + X) and click on “Device Manager”.

2. Expand “Sound, video and game”.

3. Right click on the sound driver and select “Update Driver Software”. Once this is done, restart the computer and check if the issue persists.

Method 3: Further, if there is no updated drivers available on the computer manufacturer’s website, then trying installing it in compatibility mode. Compatibility mode runs the program using settings from a previous version of Windows. 

Follow these steps to install the drivers in compatibility mode and check if that helps.

a. Download the driver from the manufacturer’s website.

b. Right-click on the driver setup file and click on ‘properties’.

c. Click on the ‘compatibility’ tab and check the box ‘Run this program in compatibility mode for’ and select Windows 8 or Windows 8.1 operating system from the drop down menu and proceed with the installation. Once this is done, restart the computer and check if the issue persists.

Running Windows XP Mode requires:
 
1. Windows 7 Professional, Enterprise, or Ultimate editions.
2. A computer capable of hardware virtualization. This means your computer has a central processing unit (CPU) with either Intel-VT or AMD-V virtualization features.
3. Virtualization features turned on in your computer’s basic input/output system (BIOS).
 
To determine whether your computer’s CPU is capable of hardware virtualization, go to the Windows XP Mode and Windows Virtual PC support website and download a utility that can check whether your Intel or AMD CPU is compatible.
 
http://www.microsoft.com/windows/virtual-pc/support/configure-bios.aspx
 
To find your brand of processor so you can download the appropriate utility, click the Start button , right-click Computer, and then click Properties. Under System, next to Processor, you can see what type of CPU you have.
 
Even if your CPU can run Windows XP Mode, you might also need to turn on virtualization settings in your computer’s BIOS. Procedures vary depending on the BIOS manufacturer. For more information, see BIOS: frequently asked questions.

http://windows.microsoft.com/en-us/windows7/BIOS-frequently-asked-questions
