a. Boot from the Windows 8 DVD.

b. Once you get the welcome screen of installation with the option “Install now”

c. Click on Repair your computer

d. On the next page click on Advanced Options

e. Select System Restore and check you can start the computer.

 

 

Method 2:
If that does not help, you may try to boot into Safe Mode from Windows Recovery Environment.
You need to enter this command in Command Prompt from Windows Recovery Environment.

Bcdedit /set {bootmgr} displaybootmenu yes
http://support.microsoft.com/kb/2809468/en-us
If you are able to boot into Safe Mode, then try to disable graphics card driver from Device Manager. Restart the computer and check if you are able to boot normally.

 

 

Method 3:
If that does not help, then try rebuilding BCD and check the issue.

a. Boot from the Windows 8 DVD.
b. Once you get the welcome screen of installation with the option “Install now”

c. Click on Repair your computer.

d. Click on Troubleshoot and then select Advanced Options.

e. Click on Command prompt and type the following commands and press Enter after each command:

Bootrec /fixmbr

Bootrec /fixboot

Bootrec /rebuildbcd

Now, check if you are able to boot.

 


Let us know if it helps.

kevmatic say:

I'm replying to this ancient thread because it is quite a puzzler and turns up high in search queries about this issue.

 

The correct answer is becuase of the "VirtualStore." Programs are permitted to write to their "Program Files" folder for legacy reasons, but they do not have previlages to do so, so they get transparently redirected to "C:\Users\"User"\AppData\Local\VirtualStore\Program Files (x86)" This causes the files a program saves to its Program Files folder to appear there WHEN IN THAT PROGRAM(or other programs that have saved to the Program Files before), but not in Explorer, making you question your sanity. 

 

For Example, to the guy above, the files he is looking for are in C:\Users\"User"\AppData\Local\VirtualStore\Program Files (x86)\TeamViewer.
