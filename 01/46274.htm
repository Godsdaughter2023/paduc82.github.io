scottce say:

Unfortunately, that may not be the same thing.

When your computer is joined to a domain that has Group Policy settings that come from the server, those settings will sometimes tell your computer to look at a local Windows/Microsoft Update server (Windows Server Update Services, or WSUS) instead of going to Microsoft Update over the Internet directly. This can be done either with a dedicated WSUS server or using WSUS via SCCM, as it in in my organization's environment.

When this is the case, clicking the Check for updates button redirects your computer to check the WSUS/SCCM server by default, which may only provide Windows updates and not updates for other MS products. Windows detects that this is happening and will often give you the alternative Check online for updates from Microsoft Update link, but clicking this link only tells the computer to use the Microsoft Update servers instead of your organization's WSUS/SCCM server.

Thus, if the Give me updates for other Microsoft products when I update Windows option is not selected, the computer is probably only looking for updates to Windows itself and not Office, Silverlight, Endpoint Protection, etc. The only way to know for sure if you are getting updates to those other MS products is to watch the update download and install process as it occurs.

I happen to have three Windows 10 computers running as I write this, with all three on the same domain with the same Group Policy settings for Windows Update. One is 10 Enterprise Insider Preview 1607 build 14955.1000, which has the setting greyed out. The second is running 10 Enterprise 1607 build 14393.447, with the option enabled and not greyed out. The third is 10 Pro 1511 build 10586.633 with the option enabled and not greyed out, and is in the process of being updated to 1607 because it somehow has automatic updates turned off (!), so I'll update this post after the upgrade with the status of this setting.

a. user say:

For Event ID 100 Windows has three kinds of entries or Levels:

Warning

Error

Critical

If you have a custom event to log boot times you can scroll through those results and every entry should be one of those three.

The Level that get assigned to the event depends on the calculated boot time:

PostBootMinorThreshold_Sec(30) // The time in seconds for postboot must exceed to consider minor issue.

PostBootMajorThreshold_Sec(60) // The time in seconds for postboot must exceed to consider Serious issue.

BootMinorThreshold_Sec(60) // The time in seconds excluding postboot must exceed to consider minor issue.

BootMajorThreshold_Sec(120) // The time in seconds excluding postboot must exceed to consider Serious issue.

There are several factors that come into play to establish the final time and what level get assigned:

60 seconds and less is a Warning

60-120 is an Error

120 and greater is Critical

Look through your Event Viewer log - you probably have some of all three, right?

None of them necessarily mean something is wrong or there is something to fix - many factors come into play.
