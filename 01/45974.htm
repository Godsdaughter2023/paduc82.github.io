I know of 4 files in the c:\windows\system32 folder that could contribute to this error (and sometimes SESSION5)

Windows\System32\smss.exe  <--most likely, so replace this one first
Windows\System32\ntdll.dll
Windows\System32\ntoskrnl.exe
Windows\System32\drivers\ftdisk.sys

There should be backup copies of these files on your system in one of these folders:

c:\windows\sytem32\dllcache
c:\windows\ServicePackFiles\i386
c:\i386

The dllcache folder is where XP keep backup copies of 3498 important files so that when XP is running, if one of these files comes up missing for some reason, Windows File Protection will quickly and silently replace the missing file from the backup copy in the dllcache folder.

If you installed SP3 from an Internet download, you will have a c:\windows\ServicePackFiles\i386 folder on your system (unless it has been deleted).  That contains the files from the SP3 update installation.

If your system came with XP preinstalled, you may or may not have a c:\i386 folder, but the files in that folder could be out of date unless your installation already included SP3.

If I were you, I would boot into the XP Recovery Console and methodically, one at a time, rename those current files (one at a time) to something you can remember (like smss.old), then I would copy in one of the backup files from either c:\windows\system32\dllcache or c:\windows\ServicePackFiles\i386, reboot and test.

If things are the same, repeat for the next file, reboot and test, etc. until you have done all four of them one at a time.

You need to do this from the XP Recovery Console and not from any kind of Safe Mode.

Do you have a genuine Windows XP installation CD from which you can boot into the XP Recovery Console or do you need instructions to create a bootable XP Recovery Console CD?

Note that any kind of system recovery CDs that came with your system are not suitable.  If you are not sure what you have for bootable media, make a bootable XP Recovery Console CD (no XP media required) and be sure.

gerry c j cornell say:

I just tried to bring up the "blue screen of death" to get the entire error message

In safe mode look in Event Viewer.

Event Viewer Reports

1. Normally when an error occurs on your computer looking in Event Viewer should be your starting point for finding a solution. Most system related errors are logged and getting an exact copy of the relevant report is important. Unfortunately understanding the reports is not easy and most computer users need help with their interpretation. I have more to say later on interpretation.

2. Event Viewer comprises three main Windows logs. These are Application, Security and System. For troubleshooting purposes System is by far the most important.

3. To access the System log select Start, Control Panel, Administrative Tools, Event Viewer, from the list in the left side of the window select Windows Logs and System. Place the cursor on System, right click and select Filter Current Log. Check the box before Error and click on OK and you see only Error reports. Click on the Date and Time Column Header to sort. You may need to click a second time to see the latest Report at the top.

4. A tip for posting copies of Error Reports! Run Event Viewer and double click on the error you want to copy. Select the Details tab and check the box before XML View. Click on the Copy button to place a copy on your Clipboard and close Event Viewer. Now start your message and paste into the body of the message. Make sure this is the first paste after exiting from Event Viewer.

5. There are three types of Report, being Information, Warning and Error reports. In most situations it is Error Reports that offer the best information but occasionally Warning Reports provide useful clues.

6. All reports have date and time stamps and when troubleshooting it is important to concentrate on more recent reports. Study reports since the point when the computer was last booted and then check whether a similar report appeared in the previous session. If errors do not repeat investigation as to why they occurred is wasted effort.

7. Within individual reports the more important information is Event ID and Source as these help when looking for help on the internet. The description is equally important and copying the exact text for use as the search criteria greatly helps getting better results when using Google. Do not paraphrase descriptions when asking others for help.

    a. user say:

    Hi,

     

    Thank you for posting your query on Microsoft Community.

     

    I understand that you are facing issues with Windows 10. Sorry for the inconvenience caused to you.

    The issue could occur if the headphone or headset driver is not installed properly. 

    In order to help you better, please provide the information:

    Have you tried to plug in to other USB ports?

    Refer to the below methods and check if it helps.

    Method 1: I suggest you to perform the Hardware and Devices troubleshooter and check if it helps.

    1. Click on search bar next to Start Menu.

    2. Type “Troubleshooting” without the quotes and click on Troubleshooting.

    3. Click on View All on the left panel of the screen.

    4. Click on Hardware and Devices.

    5. Click on Next and follow the prompts to run the troubleshooter.

    If the issue still persist, refer to the below method.

       

    Method 2: Try to update the headphone driver and check if it helps.

    1. Press Windows Key + R Key.
    2. Then type “devmgmt.msc” without the quotes and hit on Enter Key.
    3. Expand Audio inputs and outputs.
    4. Right click on the     headphone driver and click on Update driver software.

    If the issue still persist, refer to the below method.

    Method 3: Try to uninstall the headphone driver and reinstall it. Check if it helps.

    1. Press Windows Key + R Key.
    2. Then type “devmgmt.msc” without the quotes and hit on Enter Key.
    3. Expand Audio inputs and outputs.
    4. Right click on the     headphone driver and click on Uninstall.

    5. Restart your Computer and it will reinstall the default driver. 

    Reply to us with the information, so that we can assist you better.

    Get back to us if you need any further assistance on Windows related issues and we will be glad to help.
