¡firedog say:

Faulting module name: ESENT.dll ...

This kind of error is often caused by faulty hardware, like a bad bit of memory. This causes a minor corruption of an ESENT database, which then causes ESENT.dll to crash when the program tries to load the corrupted database.

Windows Live Mail uses a number of ESENT databases, one of which is the Mail.MSMessageStore that indexes your messages and which you have already replaced. Others contain your calendars and contacts lists. Any one of these could be the faulty one. To test whether this is in fact the cause of the crash, then, we'd have to stop the program trying to load the database concerned. This may be quite a simple procedure.

1.   Copy the text in the box: 

2.   Right-click on the start button and select Command Prompt (Admin). Give UAC permission to open an administrator command window.

3.   Right-click inside the window and select Paste, then press Enter.

4.   Close the command window.

5.   Find the file called default-user.txt on your desktop and open it.

6.   Click inside the Notepad window and press Ctrl-A C to select All and Copy its contents, then paste into your reply here.

We can then identify the databases that are loading (or rather, not loading) on program start and do something about it.
