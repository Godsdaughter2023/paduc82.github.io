    lemp say:

    What happens is that the "status" for at least one, and possibly more, of the Services listed in services.msc gets changed from "Automatic" to "Manual."

    You seem to be confusing "Startup Type" with "Status."

    The "Status" of a service describes its state "now" (that is, when you open the Services applet).

    The "Startup Type" of a service describes how and when the service starts.  "Automatic" means that the service is configured to start when Windows starts.  "Manual" means that the service will start when some other program needs it to start (at least in theory, see Black Viper comment below).  Manual does not mean that "the program associated with it will not start as scheduled."  On the other hand, a Startup Type of "Disabled" does mean that "the program associated with it will not start as scheduled."

    Scheduled tasks are not the same as services, although a task may well start a service.  In addition, in order for scheduled tasks to work properly, the Task Scheduler service must be configured with a Startup Type of Automatic and must continue to run.

    If the problem you're trying to solve is why your backup doesn't run when it's supposed to, you need to look at the Task Scheduler rather than the Services applet. 

    If agent.exe is the application that performs the backup job and it is configured to be run as a service, I would expect that its Startup Type would indeed be "Manual" because you don't want it to run automatically when Windows starts.  I would further expect there to be some scheduled task that starts the agent.exe service.  If you opened the Services applet at that point, you should see its Startup Type still as Manual but its Status would be "Started."

    Normally, Startup Type does not change.  If you are in fact seeing Startup Type change for various services, something that's running at boot time is doing that.  If this only happens intermittently, you're going to have a difficult time. 

    The usual method used to detect odd behavior that happens on startup is to run the System Configuration applet (msconfig.exe), prevent everything from starting and see if the problem goes away.  Assuming that nothing gets "reset" when in the "Diagnostic startup" mode, you would then enable half of the items on the "Startup" tab and reboot again.  This process repeats until you narrow it down to the one startup item that is causing the problem.  This process can be even more tedious than usual if your problem isn't repeatable on every startup.

    One of the best online sources for information about Windows services is http://www.blackviper.com/

    Here's Black Viper's explanation of the various Startup Types:

    Configuration Information

    • Automatic ~ With a service in this state, it will start at boot time. Some services, when no longer required, will also automatically stop when not needed. If you find you do not need a service, place it into Manual or Disabled.
    • Automatic (Delayed Start) ~ With a service in this state, it will start just after boot time. Some services, when no longer required, will also automatically stop when not needed. If you find you do not need a service, place it into Manual or Disabled.
    • Manual ~ Manual mode allows Windows to start a service when needed. However, very few services will start up when required in Manual mode. If you find you need a service, place it into Automatic.
    • Disabled ~ This setting will stop a service from starting, even if needed. Errors in the Event Viewer will show up complaining of that fact. Some services, while Disabled, will constantly complain. However, this situation is taken care of if placed in Manual. The service descriptions identifies those that should be in Manual vice Disabled.
    • “Started” ~ A service that either set to Manual or Automatic that is running by default at or shortly after boot time.
    • “Not Started” ~ A service set into Automatic that is not running by default at or shortly after boot time.
    • “Not Available” ~ A service that does not apply to the particular version.
    • “Uninstalled” ~ The ability to uninstall a service that I recommend to be removed by using Add/Remove Windows Features.
