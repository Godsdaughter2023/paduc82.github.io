Since my original post, I've gotten my hands on some more USB drives so I got a chance to do a little testing based in part on some of the things you brought up.

1) I checked the original 16GB USB recovery drive. Looking at it in the Windows 10 Disk Management tool, I can see there is no special formatting/partitioning going on. It is merely a single FAT32 partition that takes up the entirety of the storage space. There is no hidden boot partition or anything like that. So, the drive is bootable merely because of the files you can see in File Explorer. I don't believe there are any hidden files since I usually set File Explorer to show them by default, and I would have noticed the difference in the icons displayed. I believe I checked when I first ran into this issue.

2) I tried the original USB recovery drive and the backup Blu-ray disc to see if anything would have changed. I doubted it since I hadn't done any firmware updates. Using F12 at startup to get into the boot options, the USB recovery drive boots to the recovery environment as it did before. That was in UEFI Mode with Secure Boot on. Then to test the Blu-ray disc again, I had to boot in Legacy mode (with Secure Boot Off, as that is the only option in Legacy mode), and it wouldn't boot into the recovery environment. So, nothing special here. It just took a few minutes to confirm the behavior is consistent compared to when I set up the system.

3) Back to UEFI Mode with Secure Boot on, I took a newer 64GB USB flash drive formatted to FAT32, and wiped it clean. I copied and pasted the original USB recovery drive files to it directly from the 16GB USB recovery drive in File Manager, and then successfully booted into the recovery environment using the 64GB USB drive. I wiped the 64GB USB drive clean again and did the same thing, copying from the Blu-ray disc. The 64GB USB drive worked again in getting me to the recovery environment. No surprise with the Blu-ray disc since it was just a backup of the files on the 16GB USB recovery drive.

So, it appears you can back up your USB recovery drives to optical media, external hard drives, NAS, or any other media you want, as long as you copy it back to a clean USB flash drive (FAT32 needed?) for the actual recovery. There's nothing special to the backup process. It's just a simple backup of the files/folders on the recovery drive. No imaging needed or displaying hidden files.

Of course, I didn't go through with performing an actual recovery, as I wasn't going to play around with a computer that didn't need it, but I assume someone can test that easily enough when they're using recovery media and pound the final nail in this issue.

    1. Open your Task Manager

    2. Find Windows Explorer and right click on it.

    3. Select End Task (you may select Restart, but it didn't worked in my case)

    4. After this all running Windows Explorer instances will be closed. Start menu, Taskbar or Run command wont work.

    5. Alt+Tab navigate to Task Manager

    6. Click on File > Run New Task, the Run window will appear

    7. Type explorer.exe in the text box and press enter

    After new Windows Explorer instance will start. Now (hopefully) the problem wont persist.
