The skins worked just fine under Win7 32-bit.  I know how to load and change skins.

Why are these skins not working anymore?  Are they not compatible with 64-bit?  If they're not compatible with the latest version of WMP, why does WMP direct me to the MS site from the skin chooser?

geeked at birth say: Can no one from MS answer this question?

geeked at birth say:

Found an answer on another forum that solved my problem:

WMP downloaded the skins to the wrong place. The issue is seems to be a confusion between the 32 and 64 bit handling Microsoft has put in place.  WMP is a 32 bit application and it wants to install it's skins is to:

C:\Program Files\Windows Media Player\Skins

Whereas Windows 64-bit redirects those skins is to:

C:\Program Files (x86)\Windows Media Player\Skins

Rather bizarrely though, the first location actually exists even though it isn't used.

To solve the problem, find your skin files. They all have a ".wmz" extension, so you can search for them using Windows Explore or other file management tool.  Then move them to the C:\Program Files (x86)\Windows Media Player\Skins folder.
