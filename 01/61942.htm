Changing User Account Names - Microsoft Community

timothy l say:

Changing User Account Names

Whenever I change an account name in Windows Vista Home Premium of Windows Vista Ultimate, it does not change the user profile folder name and apparently not in the registry. The support person for the medical software suggested that we reinstall the operating system. Is there any way to avoid this step?

malkeleah say: You cannot change a user account by just changing the name. Just changing the name will make a cosmetic change only and not affect the underlying files/folders in the profile. Your computer is behaving exactly as it should. The Fry's Electronics user was a generic user created by Fry's (where you bought the computer, a floor model perhaps?) because naturally they didn't know who would buy the machine.

1. From an administrative account, rename the account to what it originally was. Then create the new Standard* user account named the way you want and at least one extra Administrator account. Call the latter "CompAdmin" or "Tech" or the like.
2. Log into the new Standard user account once so the profile files/folders are created and then log into CompAdmin.
3. Now go to Control Panel>System>Advanced>User Profiles>Settings**
4. You will see where you can copy user accounts. Copy the old account to the new one from there. Use the Browse to select your new account (C:\Users\newaccount).
5. Log into the new correctly named account and make sure all your stuff is there and that it works the way you want. When you are satisfied, you can log out and into CompAdmin and delete the old user account from Control Panel>User Accounts.

*Running as a Standard user is best practice for security purposes and will help protect your computer from infection. You don't want to run as an administrator for your daily work.
** If you don't want to bother copying the old misnamed account settings, you can skip this bit and just copy your data to the new user account instead.

User accounts - Recommended Setup (Vista and
You absolutely do not want to have only one user account. Like XP and all other modern operating systems, Vista and Windows 7 are multi-user operating systems with built-in system accounts such as Administrator, Default, and Guest. These accounts should be left alone as they are part of the operating system structure.

You particularly don't want only one user account with administrative privileges on Vista and Windows 7 because the built-in Administrator account (normally only used in emergencies) is disabled by default. If you're running as Administrator for your daily work and that account gets corrupted, things will be Difficult. It isn't impossible to activate the built-in Administrator to rescue things, but it may require more work than you want to do. Best not to get into a bad situation to begin with.

The user account that is for your daily work should be a Standard user, with the extra administrative user (call it something like "CompAdmin" or "Tech" or the like) only there for elevation purposes. Running as a Standard user is best practice for security purposes and will help protect your computer from infection. After you create "CompAdmin", log into it and change your regular user account to Standard. Then log back into your regular account.

If you want to go directly to the Desktop and skip the Welcome Screen with the icons of user accounts, you can do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

As per the description of the issue you get error “USB device not recognized” from the bottom icon bar. You get another message related with Bluetooth Device but you do not have any Bluetooth device connected to your computer.

 

We require some more information form you to assist further.

 


1) Did you try to connect any device to your computer through Universal Serial Port of your computer?
2) Have you used any Bluetooth Device earlier on your computer?

3) Are you getting this error once after restarting the computer?

4) Did you connect any external devices before getting this error message?

 

You could face this issue when the drivers for USB controller are corrupted or when some files are missing. If you have older version of Chipset drivers installed in your computer. Follow the methods given here to resolve the issue.

 

Method 1: Run Hardware and Device troubleshooter.

 

If your computer is having problems with a recently installed device or other hardware, try using the Hardware and Devices troubleshooter to fix the problem. It checks for common issues and makes sure that any new device or hardware attached to your computer was installed correctly.

 

Follow the steps given below:


a) Press Windows + W together.
b) Write Troubleshooting in the search box right up corner and hit enter.
c) Click on Troubleshooting tab.
d) Click on view all option on the upper left corner.
e) Select the Hardware and Device options from the list.
f) Click Next to run the Hardware and Device Troubleshooter.

 

Check for the issue and move to next method if the issue persists.


Method 2: Uninstall and Reinstall the Universal Serial Port (USB) Drivers.

 

Follow the steps given here to uninstall the USB Drivers.

 

a) Press Windows key + X key when you are at desktop.
b) Select Device Manager.
c) Find and double-click Universal Serial Port driver from the left panel to uninstall
d) Right-click the device, and click Uninstall.
e) Windows will prompt you to confirm the device’s removal. Click OK to remove the driver.
f) Restart the computer.

 

 

Method 3: Install the latest Chipset Driver from the manufacturer’s website.

 


Go to the manufacturer’s website, download and install the latest chipset drivers.

Feel free to write to us if the issue still persists. We will be happy to assist you.
