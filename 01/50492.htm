bobbymi say:

My O2Micro Smartcard reader failed after resuming from sleep

Hello,

 

I have an 02Micro Smartcard built into my DELL Precision M65 and when resume from Hibernation the card no longer works until I reboot the computer. I’ve tried disabling and then re-enabling the reader device in device manager but then I get a code 10 problem on the card reader.

   

Thanks

 

Christi

bobby mikkelson say:

Hello Christi,

 

This issue can happen if the embedded hub controller is having a problem on resume from Selective Suspend. If this happens, PnP removal of the device is initiated.

 

This is really a hardware issue with the O2Micro USB hub (USB\VID_0B97&PID_7761) not supporting Selective Suspend correctly. The workaround is to disable Selective Suspend of this particular USB hub controller. To do this, follow these steps:

1.       Click Start, right click Computer, click Properties  and then click Device Manager

2.       Click View  from the menu and click Devices by connection

3.       Expand the device tree down to the USB Host Controllers by clicking the following: (ACPI PC->ACPI-Compliant System->PCI Bus->USB Host controller)

4.       Expand all the USB Host Controller sub-trees to locate the Smartcard reader device

Note: The parent of the Smartcard reader device is the USB hub that caused the problem.

5.       Right click on the USB hub device and select Properties

6.       Click the Details tab and click the Hardware Ids Property to make sure the hardware ID of the hub is listed as (USB\VID_0B97&PID_7761)

7.       Click the Power Management tab and uncheck the box  “Allow the computer to turn off this device to save power’

8.       Click OK, and exit device manager, reboot the computer and test the scenario again. It should not reproduce the issue any more

 

Thanks
