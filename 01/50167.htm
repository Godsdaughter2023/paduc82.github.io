azeez n say:

Hi,

Follow the steps mentioned below.

a) Plug one end of the audio cable into your cassette player's headphone or "Line out" jack, and the other end into your computer soundcard's "Microphone" or "Line in" jack. To prevent possible soundcard damage, use the "Line in" jack if possible, and not the "Microphone".

b) Start Audacity, and change the input type to "Line in" or "Microphone", depending on what soundcard jack you're using. This tells Audacity to record whatever it "hears" on the soundcard input. To do this, open the Edit menu, then Preferences. You'll then see a screen that should do what you need: Use the "Device" selector to pick your input (soundcard, microphone, audio jack, etc), and change the "Channels" selector to pick from mono or stereo recording.

c) Press the record button (red circle) on Audacity, then press play on your cassette tape.

d) Press the stop button (yellow square) on Audacity when the tape is finished playing.

e) Use the "File -> Export as WAV" menu item to save your recording. The resulting WAV file can be saved on your computer, converted to MP3, or burned to a CD. Get fancy with Audacity's audio editor features: You can cut, paste and add effects easily.

You should record a ten-second clip at first, so you can play it back immediately in Audacity (green triangle button) and make sure your volume levels are right. If the recorded audio is too loud or has too much static, decrease the volume on your cassette player.

greg carmack - windows mvp 2010-2020 say:
Dave, Macrium has nothing to do with Windows built-in imaging which is being phased out and unreliable. You needed to save the System Image with Macrium which is always the best and most reliable option. Cloning is a different and separate operation that doesn't save the image before applying it directly to the drive.

In the UEFI firmware screenshots you show the setting is correct for Windows Boot manager which must always be set first in a UEFI system. BUT there is nothing there disabling the SSD which if it appears would be elsewhere under storage settings where the hard drives are listed.

So what likely happened is that the boot files remained on the SSD so that when you wiped it they went away leaving the HDD clone unable to boot.

You can try Startup Repair on it from the booted WIndows media as I wrote before to see if it will jump start the EFI System partition as long as it was included in the clone, or what it reports.

If that fails then you can reinstall on the HDD or do a rescue install to the SSD which should create a dual Boot menu to start the clone. Then post back a screenshot of Disk Mgmt and I'll help you remove the SSD: http://answers.microsoft.com/en-us/windows/foru...

If it doesn't create the Dual Boot then install EasyBCD to add it by drive letter shown in Disk Mgmt on the Add OS Entry tab:

If the new HDD is formatted using 4096 bytes per sector that is the problem.

Microsoft did not start supporting system images on HDDs with 4096 bytes per sector until Windows 8.

To see what you have, do this:

Click the Start orb and in the box enter:

command

Right click Command Prompt and choose to Run as administrator to open a Command Prompt window and enter the following command substituting the drive letter of your new drive (I'm using drive D so you put in your drive letter):

fsutil  fsinfo  ntfsinfo  d:

Look at the Bytes per sector value and if it is 4096 that is the problem.  Windows 7 only supports 512 Bytes per sector like in this example from my D drive that indicates 512 Bytes per sector (this will work):

In that example the Bytes per sector is 512 so it will work fine for System Image.

What is the Bytes per sector on your new HDD?

If the Bytes per sector on your new HDD is 4096 you will need to reformat it to 512 BPS, use a different HDD or a third party program that supports 4096 BPS.

https://support.microsoft.com/en-us/kb/2510009

the problem seems to be fixed. The following happend since the last post:
