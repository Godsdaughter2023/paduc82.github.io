How to clear all Event Logs using command prompt

  1. Open Notepad and copy-paste the following text into it: 
    @echo off
FOR /F "tokens=1,2*" %%V IN ('bcdedit') DO SET adminTest=%%V
IF (%adminTest%)==(Access) goto noAdmin
for /F "tokens=*" %%G in ('wevtutil.exe el') DO (call :do_clear "%%G")
echo.
echo Event Logs have been cleared!
goto theEnd
:do_clear
echo clearing %1
wevtutil.exe cl %1
goto :eof
:noAdmin
echo You must run this script as an Administrator!
echo.
:theEnd
  2. Save it as a batch file and give it any name you want for example: ClEvtLog.bat or ClEvtLog.cmd.

  5. Run the batch file from the command prompt: ClEvtLog.cmd. You can also directly run it without opening the command prompt or using cmd /c so the command prompt closes after running it.

How to clear all Event Logs using PowerShell

  1. Open PowerShell as administrator (see how).
  2. Type or copy-paste the following command into PowerShell: 
    wevtutil el | Foreach-Object {wevtutil cl "$_"}

a. user say:

It is normal to see many svchost Processes running in Task Manager.

Under each one are important Windows Services - each svchost Process might be running one Windows Service or it might be running several.  Those processes are "hosting Windows Services" hence the name.

In Task Manager with the Process ID (PID) column enabled you can see what is going on for each one.

If you are curious about them, right click one and choose Go to Service(s) and on the next screen you will see highlighted the Services being hosted by that Process.

For example checking the Services for the svchost Process with PID 720 I see that Process is hosting 3 Windows Services:

You say you "have the svchost issue" but do not describe that the issue is.

In Task Manager most of the time the svchost Process should be showing very little CPU activity so if there is one that is running amok using large amounts of CPU/RAM that is a problem but the problem is not with the svchost process and there is no such thing as a "svchost memory leak" like you read about on the Internet.

If the svchost had a memory leak millions of systems all over the world would be afflicted and they're not.

If one svchost Process seems to be running amok the problem is not with the svchost Process - the problem is what is running underneath that afflicted svchost Process.

Allegedly there is some malicious software that can disguise itself as a svchost Process or the malicious software could be running underneath the svchost Process and you would not see or notice that in Task Manager unless you look at the processes running underneath the suspicious svchost Process.

So describe your issue a little better and include a screenshot of what you are seeing using the instructions in this link:

How to include a screenshot in your post

You could right click any suspicious svchost Process to see the Services running under it and include that screenshot too.
