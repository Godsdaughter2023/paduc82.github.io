Bug report: GPO has same effect whether disabled or enabled - Microsoft Community

jeremy2008 say:

Bug report: GPO has same effect whether disabled or enabled

GPO:
Computer Configuration\Administrative Templates\Windows Components\Windows update\Remove access to use all Windows update features

When Enabled, it creates a registry value:

HKLM\Software\Policies\Microsoft\Windows\WindowsUpdate\DisableWindowsUpdateAccess = 1

When Disabled, it creates a registry value:

HKLM\Software\Policies\Microsoft\Windows\WindowsUpdate\DisableWindowsUpdateAccess = 0

When Not Configured, it deletes the registry value:

HKLM\Software\Policies\Microsoft\Windows\WindowsUpdate\DisableWindowsUpdateAccess = (deleted)

EXPECTED BEHAVIOR:

When this GPO is "Disabled" and the registry value is "0", we expect the GPO's behavior to NOT be applied.  So when Disabled, users should NOT have their Windows Update access removed.  This is useful for ensuring that other GPOs up the tree that might be set to Enabled will be overridden by this Disabled GPO.

ACTUAL BEHAVIOR:

When this GPO is set to "Disabled" and the registry value is "0", the behavior still applies and removes access from the users.  The "Check For Updates" and "Install Now" buttons are greyed out.  I'm using fully updated Server 2016, the latest ADMX files, and fully updated Windows 10 1803.

WORKAROUND:

To fix the issue, I have to change the GPO to "Not Configured", which is an ugly workaround because "Not Configured" is supposed to mean that we are not enforcing any preference, so some computers may have this enabled and other computers may not.

REQUESTED SOLUTION FROM MICROSOFT:

Change Windows to interpret a "0" value in that registry value to be the same as if the value did not exist at all.  The only time the registry value should have an effect is if the value is set to "1".  (This should be the same behavior for ALL registry values and GPOs, imo)

Any help would be appreciated.

rohit siddegowda say:

Hi Lawrence,

 

Welcome to the Microsoft community.

 

It seems that you are not requested for a WEP (Wired Equivalent Privacy) password while connecting to a wireless network.

 

WEP is an older network security method that's still available to support older devices, but it's no longer recommended. When you enable WEP, you set up a network security key. This key encrypts the information that one computer sends to another computer across your network. However, WEP security is relatively easy to crack.

 

To manually create a network profile using WEP shared key authentication, follow these steps:

a.     Open Control panel.

b.    In the search box, type “network” without quotes, and then click Network and Sharing Center.

c.     Click Set up a new connection or network.

d.    Click Manually connect to a wireless network, and then click Next.

e.    On the Enter information for the wireless network you want to add page, under Security type, select WEP.

f.      Complete the rest of the page, and then click Next.

g.    Click Change connection settings.

h.    Click the Security tab, and then, under Security type, click Shared.

i.      Click OK, and then click Close.

 

Hope this helps. If you have any further issues on the computer, please post your question regarding Windows and we will be happy to help you.
