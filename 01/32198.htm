I have been researching this issue and have seen a lot of discussion of using powercfg.exe to change values of different GUIDs, but even after enabling dimming and setting the timer it doesn't seem to do anything but dim right before powering off.

  • "powercfg -list" --> to get a list of current power plan GUIDs. Note/Copy the GUID for the plan you want to query.
  • "powercfg -aliases" --> to get a list GUIDs for plan sub-sections. Note/Copy the GUIDs for "SUB_SLEEP" and "SUB_VIDEO".
  • "powercfg -query <GUID of power plan> <GUID of sub-section>" --> to get all sections and values.

  • powercfg /setacvalueindex SCHEME_CURRENT SUB_VIDEO VIDEOIDLE 120
  • Should set the idle value to 120 seconds.

I found a reference to a change done for Windows 8.1 where over on the Surface forums found a documented "design limitation" around the dimming feature no longer being configurable.

http://www.surfaceforums.net/threads/wheres-the-dim-display-after-power-setting-in-windows-8-1.6577/

PS.  If you search your registry for "17aaa29b-8b43-4b94-aafe-35f64daaf1ee", you will see the "dim display after" root, with each of the power plan GUIDs as children with each having values for "AcSettingIndex" and "DcSettingIndex". (MSDN Documentation)
