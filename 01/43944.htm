If it is not working on a different computer also, then the problem will be with the modem and I would suggest you to contact the modem manufacturer for getting any further assistance.


You may also try the following methods and check if it helps.

Method 1:
To resolve this issue, remove the modem, and then restart the computer. When the computer restarts, Windows detects the modem and attempts to install the correct driver for it. To do this, follow these steps:
a. Log on to the computer as administrator or as a member of the Administrators group.
b. Click Start and then click Control Panel.
c. Double-click Phone and Modem Options.
d. Click the Modems tab.
e. In the following modems are installed list, click the modem that you want to remove, and then click Remove.
f. Click Yes in response to the "Are you sure you want to remove the selected modem(s) from your system?" message.
g. Restart the computer.
h. The Found New Hardware Wizard starts. Follow the instructions in the wizard to reinstall the modem.


Method 2:
You may re-install the modem driver from the manufacturer website and check.

5. Restart the computer.

NTBackup tool creates catalog files for every BKF file and of every backup operation which is completed. These files are stored in the following folder on your Windows computer:

%SystemRoot%\Documents and Settings\Account\Local Settings\Application Data\Microsoft\Windows NT\NTBackup\catalogs

The Catalog files are shown under the Restore tab of NTBackup utility, so that you can select the part of backup file that you want to restore.

When you create more than one backup at a time, the catalog files share identical timestamp attribute. In this case, the backup set that was attributed first, emerges under the Restore tab. Other backups started concurrently do not appear under the Restore tab and you can’t add them manually.

This process corrupts BKF file and bkf repair becomes necessary to achieve precise recovery.

randyau93 say:

Hi! I've seen this happen (I've had this happen) and it's easily fixed. Hopefully you have fixed it since this was posted, but just in case ...

If your console will not see any Wi-Fi networks after a reboot, power-cycle it. To do this, press and hold the power button on the face of your console for several seconds, holding it down until the console shuts off. After about 15-20 seconds, restart the console (it may take a bit longer than normal, but this is not unusual). When it starts back up, it should see the same Wi-Fi networks as you had before.

If it doesn't, it's time for a step that's a bit more drastic. Go to Settings -> System -> Console Info -> Reset console. This is a factory reset, but there is a VERY important step you need to take when this starts. It will ask you if you want to wipe everything or keep your games & apps. Be sure to select "keep my games & apps", so you don't have to reinstall everything.
