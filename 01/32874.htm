Right click on Windows Start > Right click on Windows Powershell (Admin) > Copy the following from below and right click in Powershell to paste > Enter

Get-AppxPackage -allusers *WindowsStore* | Remove-AppxPackage

Get-AppxPackage -allusers *WindowsStore* | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register “$($_.InstallLocation)\AppXManifest.xml”}

You can use this code to check which user is logged in to the computer, and use that information to allow or deny them access to your database. This method has the added security bonus of not requiring you to store user passwords in your database.

Public Function GetUser() As String

    Dim WNet As Object

    Set WNet = CreateObject("WScript.Network")

    GetUser = WNet.UserName

    Set WNet = Nothing

End Function

Public Sub CheckUser()

    Dim RcSet As Recordset

    Set RcSet = CurrentDb.OpenRecordset _

    If RcSet.RecordCount = 0 Then

        MsgBox "You are not authorized to use this application", vbInformation

        DoCmd.Quit

    Else

        MsgBox GetUser

    End If

End Sub

You can add a filter in the Form_Load Sub

Private Sub Form_Load()

Prior to Windows Vista, applications were typically run by administrators. As a result, applications could freely access system files and registry keys. If these applications were run by a standard user, they would fail due to insufficient access rights. Windows Vista and later versions of Windows improve application compatibility for these applications by automatically redirecting these operations. For example, registry operations to the global store (HKEY_LOCAL_MACHINE\Software) are redirected to a per-user location within the user's profile known as the virtual store (HKEY_USERS\<User SID>_Classes\VirtualStore\Machine\Software).
