Outlook Express, loss of input emails - Microsoft Community

john m mills say:

Outlook Express, loss of input emails

Outlook Express, 3 new emails were being viewed, and then all previous read and non-read emails disappeared by just clicking on the next email. The 3 latest emails remained and everything else was lost.

On 'searching' for '.dbx', found 'inbox (1). dbx', containing the 3 current input emails, and 'inbox.dbx' which seemed to contain all the lost emails.

How can one retreive all 'input' email messages? No files lost from any other element, and no files added to 'delete'. Would an 'Outlook Repair Kit' software solve the problem?  If so which one? System uses - Windows XP Professional SP3,  and Microsoft Outlook Express 6.

bruce hagen ~ mvp say:

Close OE and rename the Inbox.dbx with the missing messages to Inboxold.dbx and then drag it to the Desktop.

Open OE and create a folder with the *exact* same name as the one on the Desktop, Inboxold.. Open the new folder and then close OE. (You must do this). Go to Windows Explorer and locate the Message Store folder for your OE identity, but don't open it. Click on the Desktop and drag the file from the Desktop in the right hand pane to the OE store folder in the left pane. Prompt - "Do you want to overwrite......."? Click Yes.

*Note* If you have moved the Outlook Express store folder from it's default location on drive C to another drive, drag the dbx file to a location on that drive and not the Desktop.   


If this method fails, you will need a tool to extract the messages.

DBXpress
http://www.oehelp.com/DBXpress/Default.aspx

You may need to run it in the Extract From Disk Mode if its normal recovery mode fails to find the lost messages.

To help prevent this in the future:

Do not archive mail in the Inbox or Sent Items. Create your own user defined folders and move the messages you wish to save to them. Empty Deleted Items folder daily. Although dbx files have a theoretical capacity of 2GB, I recommend about a 300MB max for less chance of corruption.

Information about the maximum file size of the .dbx files that are used by Outlook Express:
http://support.microsoft.com/?kbid=903095

After you are done, follow up by compacting your folders manually while working *offline* and do it often.

Click on Outlook Express at the top of the folder tree so no folders are open. Then: File | Work Offline (or double click Working Online in the Status Bar). File | Folder | Compact all folders. Don't touch anything until the compacting is completed.

Turn off e-mail scanning in your anti-virus program. It is a redundant layer of protection that eats up CPUs and causes a multitude of problems such as time-outs and account setting changes. Your up-to-date A/V program will continue to protect you sufficiently. For more, see:
http://www.oehelp.com/OETips.aspx#3

And backup often.

Outlook Express Quick Backup (OEQB Freeware)
http://www.oehelp.com/OEBackup/Default.aspx 

steve cochran say:

You can right click on the inbox in the folder list and go to Properties to see what actual file OE is using.  It sounds as if the original inbox.dbx got corrupted, so OE made a new one called inbox (1).dbx which it is now using.   Before you do anything to the inbox.dbx file, make a copy of it in case the messages are still in it.  Then you can try importing it as Bruce suggested (see also www.oehelp.com/backup.aspx#imp1).  But if you try and import it that way and its corrupted and has messages in it, OE may just wipe out the contents, so make sure you make a backup first (you should backup the entire message store at this point).

Also check your Recycle bin and the message store for an inbox.bak file that is a backup if it exists and might contain the lost messages.  See www.oehelp.com/OETips.aspx#2 

Otherwise see Bruce's response.
