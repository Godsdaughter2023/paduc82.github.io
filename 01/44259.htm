ralf_g say:

A wireless keyboard cannot be detected during the early part of the boot sequence (POST) because the drivers that the keyboards' wireless dongles require only work in Windows, which is not yet running at that point.

You can only use hard-wired keyboards and/or mice to access BIOS/CMOS settings. Depending on how the computer BIOS handles powering up the USB ports some computers may only respond to PS2 connected keyboards/mice during the earliest stages of booting-up.

jimwynne say:

You can only use hard-wired keyboards and/or mice to access BIOS/CMOS settings.

While this is true of older computers, more modern ones (with UEFI) allow use of wireless mice and keyboards when accessing the settings during POST (before Windows loads).   It appears that the OP's keyboard's basic functions work; it's just the Fn key (controlled by the keyboard driver) isn't working.  See my earlier explanation.

Well you have a few issues.

  1. Microsoft does not pre-install any MAPI email program into Windows 8 (or Windows 7 for that matter).  They did back in Windows Vista, but the European Union forced them to stop because they said consumers should pick their own email program and not have a default (the EU also required Microsoft to provide a version of Windows without Internet Explorer and Media Player for the same reasons).

Verifier picked on Lycosa.sys, as Andre mentioned.

The other dump showed a  0x124 - that's WHEA_UNCORRECTABLE_ERROR - "This bug check indicates that a fatal hardware error has occurred."  This can indicate that you have  some faulty hardware, unfortunately, probably the processor or perhaps the motherboard. Additional things you can try, though, just to see... Ensure that the system has adequate cooling and ventilation. Test RAM, reseat RAM and the processor if you can.  If the BIOS is the latest, consider reverting to a previous version and see if that helps.  If not, upgrade to the newest BIOS available.  Also double-check BIOS settings; perhaps, note current settings, reset the BIOS defaults, and then adjust those settings you know are necessary to adjust.  Double-check voltage, etc. and ensure/verify CPU compatibility with the motherboard.  If you're overclocking, stop.

The specific details about the hardware error you encountered indicate a fatal machine check exception due to Micro-Architectural Error (X64_0x124_GenuineIntel_PROCESSOR_MAE).  The specific reason is difficult to assess - it is characterized as "Internal unclassified".  

You do have a few stale drivers that you could remove or update to check the issue, but if that doesn't have an effect, focus on the referenced hardware.
