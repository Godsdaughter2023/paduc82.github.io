The Watchdog violation plagued us during the Insider Preview versions.

Assuming the VAIO uses a Synaptics Touchpad:

The touchpad drivers by Synaptics have two versions:

USB

PS/2

By default, Windows 10 kept installing the USB drivers on my netbook. Touch the touchpad and lockup with Watchdog Violation. A lot of us suffered this problem.... a lot of frustration and seeking driver fixes.

Microsoft Support never, ever understood the problem... uninstall drivers and the next time Windows Update ran, Watchdog would return.

My netbook is set dual boot back to Windows XP...

Win XP used a P2/2 Synaptics driver... mm...

Win 10 kept installing a USB Synaptics driver... that's the base problem.

Use Device Manager to see which type you have installed... if USB, then try to get PS/2 to install.

Downloading "Synaptics_v17_0_19_C_XP32_Vista32_Win7-32_XP64_Vista64_Win7-64_Acme_Inc" from the Synaptics web site cured my problems. It installed the PS/2 drivers... from then on Windows 10 Update started to install PS/2 drivers and not the USB drivers.

Current Windows 10 driver that works is: Synaptics PS/2 Port TouchPad, version 19.0.12.95 dated 15 July 2015.

Watchdog cured... but edge scrolling does not... that's another known problem.

I found this info on another message board
Here's how to fix this issue:

1. CTRL-Alt-Del to bring up Task Manager.
2. Click File | New Task > type run > ok.
3. Type regedit in the Run box and click OK.
4. Browse to the following registry key:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution.options

5. Under this key there will be subkeys named explorer.exe and iexplorer.exe.
These keys are pointing to files that have been removed (virus/spyware); explorer32dbg.exe and iexplore_dbg.exe. Delete the explorer and iexplorer keys entirely. They should not be listed under the Image File Execution.Options key.
6. Close the Registry Editor.
7. Restart the computer.

tryx3 say:

Task manager in Windows 10 is not the same thing as Task manager for Windows 7 & earlier. 

  • You can, for example, alter Startup items that you used to use MSConfig for in earlier Windows versions.

  • If you are logged in to an Admin user account, Task manager in Windows 10 always challenges for Admin permission when it starts. 
  • If you are logged in to a Standard user account, Task manager in Windows 10 will open without any Admin challenge.

  • If you are logged in to an Admin user account, Task manager in Windows 10 will allow you to alter entries in the Start-up tab such as disabling an item. 
  • If you are logged in to a Standard user account, Task manager in Windows 10 will only allow you to alter entries in the Start-up tab that affect that user account only.  If you try to alter an item that affects other users, the disable command is greyed out. 

Denis
