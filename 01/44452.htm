SFC /scannow is a super-useful command you can use in any Windows version. When the SFC (System File Checker) command is used with the /scannow switch, the tool will scan all of the important Windows files on your computer and replace them as necessary.
 
Missing and corrupt operating system files (like many DLL files) are arguably the biggest cause of major Windows issues. Considering that, plus the fact that SFC /scannnow is completely automatic and very easy to use, the tool should usually be one of your first troubleshooting steps.
 
a. Click Start, click All Programs, click Accessories, right-click Command Prompt, and then click Run as    administrator. If you are prompted for an administrator password or for a confirmation, type the password, or    click Allow

   [The sfc /scannow command scans all protected system files and replaces incorrect versions with correct    Microsoft versions.]

c. If any files require a replace SFC will replace them. You may be asked to insert your Win7 DVD for this process    to continue
d. If everything is okay you should, after the scan, see the following message "Windows resource protection did not    find any integrity violations"
e. After the scan has completed, close the command prompt window, restart the computer and check.

Das könnte beispielsweise daran liegen, dass zwei oder mehr Leute in dem Match selber ein moderates oder Striktes NAT haben. Das führt dann eben zu Problemen, bei denen selbst ein Nutzer mit offenem NAT nicht einsteigen kann. Hin und wieder ist auch eine gleichzeitig laufende Party mit unterschiedlichen NAT Zuständen Schuld daran.

Here is the BootRec help screen...

=============

X:\windows\system32>bootrec /?

Repairs critical disk structures. The following commands are supported:

/FixMbr -- writes the master boot record of the system partition using the master boot record compatible with Windows. This operation does not overwrite the existing partition table.

/FixBoot -- write a new boot sector onto the system partition using the boot sector compatible with Windows.

/ScanOs -- scan all disks for installations compatible with Windows and display the entries that are currently not in the boot configuration store.

/RebuildBcd -- scan all disks for installations compatible with Windows and allow the user to choose which to add to the boot configuration store.

=============

Is the computer a UEFI system? "/FixMBR" & "/FixBoot" are not for UEFI systems, but I much doubt they hurt anything. "/RebuildBCD" is the right one for UEFI - but it won't repair an existing BCD (Boot Configuration Data) entry. It will only add a new BCD entry to the current BCD Store. So - if the BCD entry you've got is bad - it must be deleted or renamed first. Edit: The following actually renames the entire current BCD Store. (It can be retrieved by renaming it back, if necessary.) Then, it generates a new current BCD store & offers to include each bootable partition that is found.

But where is it? It could be in a folder on C:\, but more likely is on a separate partition - the ESP (EFI System Partition). You will have to give it a letter...

That put a letter on it, which was D: in my case. Now, "DIR" shows the current BCD Store is in "D:\EFI\Microsoft\Boot"...

You will want to rename it, using the letter you got, if different from D:\. After that, try the rebuild again. This time it will find a Windows bootable installation to add...

===========

X:\windows\system32>REN D:\EFI\Microsoft\Boot\BCD  BCD_OLD

X:\windows\system32>BootRec /RebuildBCD

Scanning all disks for Windows installations.

Please wait, since this may take a while...

Successfully scanned Windows installations.

Total identified Windows installations: 1
[1]  C:\Windows
Add installation to boot list? Yes(Y)/No(N)/All(A):y
The operation completed successfully.

===========
