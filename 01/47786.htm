Hello,

Welcome to Microsoft Community Forum.

Svchost.exe is a process on your computer that hosts, or contains, other individual services that Windows uses to perform various functions. For example, Windows Defender uses a service that is hosted by a svchost.exe process.

There can be multiple instances of svchost.exe running on your computer, with each instance containing different services. One instance of svchost.exe might host a single service for a program, and another instance might host several services related to Windows.

You can use Task Manager to view which services are running under each instance of svchost.exe.

Refer the link:

To view which services are currently running under svchost.exe

However I would require some more information to assist you better.

1. Do you remember making any changes to the system prior to this issue?

I would suggest you to try the following methods:

Method 1:

Step 1: Boot the computer in Safe mode and check if the issue persists.

how do i get out of safemode with msconfig.

a. user say:

When you ran msconfig to get set up for Safe Mode, did you just enable/check the /SAFEBOOT option on the BOOT.INI tab or did you do something else?

There is some malicious software that if you use the /SAFEBOOT option, you will not be able to use your system again until you remove the /SAFEBOOT switch from the boot.ini file so you can boot normally.

This may not be your exact problem, but I will never suggest to anyone to use the /SAFEBOOT option ever again - too risky when troubleshooting since you may not ever boot again or will never login again until you remove the /SAFEBOOT switch.

Anywho, if that is all you did, you can boot into the XP Recovery Console and either make a new boot.ini file that doesn't contain the /SAFEBOOT switch or just rename the boot.ini you have to something like boot.ini.old so then you have no boot.ini file (I know it seems like an odd idea).

In a single partition configuration, XP does not even need a boot.ini file to boot.  XP will complain if there is no boot.ini file, but will still boot just fine without one (non believers - try it!).

After you get booted up and logged in, then you can rename boot.ini.old back to boot.ini and run msconfig again to remove the /SAFEBOOT option and never use it again.
