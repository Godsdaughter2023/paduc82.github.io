cmdrkeene say:
So just a few things:

1 - If your video card can't power CAD's demands while also displaying Aero, you can set it to automatically disable Aero while CAD is running.  To do this, just right-click your CAD icon and go to Properties, then set the Compatibility tab to disable desktop composition.

2 - It's meant to leverage the video card.  Otherwise, your video card would go unused. That's just wasted money at that point.  Why?  Well...

With the previous GDI graphics, in use from the 1980's until Vista, your video card sat idle.  All the graphics work went to your CPU, which was very ineffecient because it could not cache textures or graphics in a buffer. When you moved, minimized, or resized any window, the actual main process had to literally go back to the programs that were now-visible, and ask "please re-draw the part of you that is now visible".  This is why you would see black areas or the weird cascading duplicating effect when your system was slow and you moved a Window: the CPU was too busy to get responses and re-draw the screen.*  For the same reason, GDI graphics have been the #1 cause of blue screen fatal errors for over a decade.

With Aero, all the display work goes to the much more powerful and efficent video card.  When you move a window, your CPU and programs don't even know about it. The graphic data is already buffered and ready.  You know how your mouse pointer can still move even when the system is totally frozen?  That's because the mouse pointer is controlled at the hardware video card level and drawn on to the screen without the OS knowing about it.  With Aero, all your display graphics get this same benefit.  And your main CPU gets to focus on actual processing.  Using Aero, with even a mediocre video card, effectively doubles your performance and increases your reliability almost 8 times.



*Nerdy fact: the old Solitaire game would block display queue writes on purpose to achieve this cascading card effect.

Hi Alec,

Rather than using a 3rd party product, Windows 10 has a backup feature that's different from Windows 7 and I haven't heard anything about it being deprecated.

lemp say:

It's very dangerous for casual users to use encryption software, either Bitlocker (which is native to some editions of Windows), the Windows Encrypting File System,  or third-party software such as Folder Lock. 

The whole point of encryption software is to keep data secure.  In order to achieve that purpose, careful developers of good encryption software will make strenuous efforts to avoid having any "back door" that would enable encrypted data to be accessed without knowledge of the user's passphrase.

The analogy of a "spare key" or "locksmith code" for car keys is somewhat flawed, but in fact Folder Lock does have something similar to a "locksmith code"  -- the serial number of the copy purchased by the user.

Unfortunately, the Folder Lock serial number can't be used as a "locksmith code" unless the user has taken some explicit actions.  See FAQ excerpts below.

According to the Folder Lock FAQ (http://www.newsoftwares.net/doc/folder-lock-7-guide.pdf), one can only use the serial number of the program if (1) you registered your copy with the publisher, (2) you configured a Master Password, and (3) you have "kept the Master Key option enabled in Password Security settings."

The bottom line is that encrypting computer data is dangerous unless you are particularly careful and follow best practices, which typically include keeping either a copy of the encryption key or a "recovery agent" in a secure but unencrypted location.  See, for example, https://support.microsoft.com/en-us/kb/223316, in which the very first point is:

Teach users to export their certificates and private keys to removable media and store the media securely when it is not in use. For the greatest possible security, the private key must be removed from the computer whenever the computer is not in use. This protects against attackers who physically obtain the computer and try to access the private key. When the encrypted files must be accessed, the private key can easily be imported from the removable media.
