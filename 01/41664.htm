Yeah - if you are going to donate (I do too), you might just want to do a clean install, but you might be able to fix your SESSION5...

Usually, a SESSION type message (of which there are several) comes after some antivirus/antimalware tool or some overzealous cleaning has deleted critical XP system files.

There are at least five possibilities (that I know of) for these missing files that result in the SESSION3 or SESSION5 type message and there is no harm in just replacing all 5 files while you are there...  one or some may be missing, so you will just overwrite with the backup copies that should already be on your system, assuming XP is installed on your C drive:

C:\Windows\System32\smss.exe  <-------- most likely, so replace this one first
C:\Windows\System32\ntdll.dll
C:\Windows\System32\ntoskrnl.exe
C:\Windows\System32\winlogon.exe
C:\Windows\System32\drivers\ftdisk.sys

After booting into the XP Recovery Console, you should be in the C:\WINDOWS folder.  If you are not in the C:\WINDOWS folder you have some other problem that you need to fix first.

We know that XP keeps backup copies of critical system files in this folder:

c:\windows\system32\dllcache

Depending on how you installed Service Pack 3, there may also be backup copies in this folder:

c:\windows\ServicePackFiles\i386

You need to copy the first 4 files one at a time from one of those two folders into this folder where they belong:

c:\windows\system32

When copying files, you may be replacing existing files with the copies, so respond in the affirmative if asked to overwrite existing files.  If the copy succeeds, you should see a message that the file was copied successfully. 

In the Recovery Console, from the C:\WINDOWS prompt, enter the following commands:

copy c:\windows\system32\dllcache\smss.exe    c:\windows\system32

Reply in the affirmative to any "are you sure" messages so 1 file(s) is copied

If the file is not found to copy enter this command:

copy c:\windows\ServicePackFiles\i386\smss.exe    c:\windows\system32

Just repeat this process for the first 4 files even if they are not missing.

The ftdisk.sys file needs to be copied into this folder:

c:\windows\system32\drivers

So you would enter:

copy c:\windows\system32\dllcache\ftdisk.sys   c:\windows\system32\drivers

Remove the CD and enter 'exit' in the Recovery Console, say three hail Mary's and your system will reboot.

a. user say:

Hi Jesty,

The primary difference between the 2.4 GHz and 5GHz wireless frequencies is range as the 2.4GHz frequency is able to reach farther than the 5GHz frequency. This is a result of the basic characteristics that waves attenuate much faster at higher frequencies. So if you are more concerned with the coverage, you should select 2.4GHz rather than 5GHz. The second difference is the number of devices on the frequencies. 2.4GHz suffers more interference than 5GHz.

In general, any Wi-Fi adapter that works with Windows' built-in Wi-Fi manager will work with in Service Set Identifier (SSID).

Adapters are listed by vendor, then model with what they support.

For example:

A means the adapter supports 802.11a 5GHz
B means the adapter supports 802.11b 2.4GHz
G means the adapter supports 802.11g 2.4GHz
N means the adapter supports 802.11n

 

If the adapter supports A and N, then it generally supports 5GHz N. If it supports B or G and N, its 2.4GHz only.

For example: N600 Wireless Dual Band USB Adapter (WNDA3100) supports [A/B/G/N], i.e., it can work on both 2.4Ghz as well as 5Ghz.

So, if your router enables both the bands does not mean that the Wi-Fi Adapters installed in either of your computers support both the bandwidths. Thus, it depends on the specifications of the Wi-Fi Adapters installed in your computers.

To check the Wi-Fi Adapters installed in your computers, you may follow these steps:
