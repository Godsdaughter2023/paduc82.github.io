a. user say:

Just as an FYI: A restore point is not a backup. They are two different things.

The reason that restore points are deleted after every feature update (= new version of Windows 10) is that you wouldn't want to restore an old restore point after a feature update. That would give you a combination of old and new system files and registry hives, which is like asking for trouble.

If you're going to do a clean install, do yourself a big favor: Start making image backups. It takes some learning and you'll need third-party image backup software but holy cow is it worth it.  You can restore a backup in just minutes and it's like season seven of Dallas (if you're old enough to remember that.)

When you backup every day ... who needs System Restore? If your computer goes on a 'bad trip' (again, you have to be old enough) just send it back in time.

obsessive power say:

Using the calculation that defines the difference between the GB's that the manufacturer of a storage device stated available and the space that system can see, a 320GB hard drive will have about 297GB, of which several GB's will be taken by the console as a hard drive cache.  That is where your missing GB's have gone.

Without getting complicated, the simple rule is that there is roughly 930 MB's per stated GB. The hard drive in my PC is sold as a 1TB drive, but in actual fact it has 930GB free.  The other 70 odd GB's are lost in translation.

for derrick

  1. Launch Registry Editor (Start >> Run >> regedit.exe)
  2. Expand HKEY_CURRENT_USER
  3. Navigate to HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ Shell
  4. If it doesn’t exist, create the AttachmentExecute key
  5. Create the {0002DF01-0000-0000-C000-000000000046} key
  6. Create a new Binary Value with the Name InfoPath.Solution.3
  7. Re-launch IE and test – should be now working ok

There is a very simple solution - at least it worked for me.

1. Go to the folder that contains files of your album. Delete all AlbumArt*.jpg files from the folder and Folder.jpg file; you may need to confirm your administrative rights, if you are not the owner of the files.

2. Close WMP and open it again. Go to album you want to change art for, and try again pasting or dragging your picture.

The art will change even though WMP shows the error message again.

Apparently, it is a WMP bug - it cannot delete jpg file owned by someone else, and shows misleading error message that does not explain anything.
