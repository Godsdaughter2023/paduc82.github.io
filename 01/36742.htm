a. user say:

It sounds like the boot order on your system of unspecified make and model is out of order.

In the system BIOS you can specify the priority of the devices from which you might want to boot - maybe something like:

Maybe you have the portable USB drive set as the first boot priority and you might really want the hard drive to be the first boot priority.


With the external driver disconnected, the system looks for it, can't find it and moves to the next priority which is probably your hard drive.

If you ever need to boot from your CD/DVD you might need to adjust the boot order to put that as the first priority.

Sometimes when you restart there will be a message to press some key that lets you choose the boot order but that will not be a permanent change.

You make these adjustments by accessing the system BIOS before Windows starts to load you might press the Del key, or some F key to get into the BIOS to make the adjustment - look for a brief message when the system restarts to tell you what key to press.

If you don't know how to change that and tell us the make and model of your system maybe we can help you figure out how to adjust things.

Here is a link that tells you how to include a screenshot in your post

How to include a screenshot in your post

Depending on the unspecified third party backup software you are using what you might have to do is keep your images/backups on your external drive and then using their software create either a bootable CD/DVD or a bootable USB thumb drive and stash that away so you have it when you need it.  The image you have on the external drive is unlikely to be bootable by itself.

Then when you need to use your image, you retrieve the CD/DVD or USB thumb drive you made before, boot the system on that which will load their software and from there you can restore your image.

How all that works depends on the software you are using and all that information will be on their WWW site.
