Is creating folder shortcuts via SHIFT-CTRL/Drag-And-Drop no - Microsoft Community

- rick - say:

Is creating folder shortcuts via SHIFT-CTRL/Drag-And-Drop no longer supported in Windows 7 (64-bit)?

In Windows Vista, there exist two posibilities to create a shortcut to a folder via drag-and-drop by pressing different key combinations while dragging-and-dropping:

'ALT'-key: create a normal shortcut (double-clicking on that shortcut navigates to the linked folder)
'SHIFT-CTRL'-keys: creates a folder shortcut (junction; the linked folder shows up as a subfolder of it's container folder)

In Windows 7, it is only possible to create normal shortcuts. 'SHIFT-CTRL'/drag-and-drop does not seem to work in Windows 7. It just behaves in the same way as 'ALT'/drag-and-drop does.

Is my observation correct?
Is there any possibility to enable the Vista-like behavior in Windows 7 (e.g., by modifying the registry)?

Some remarks:

Folder shortcuts that have been previously created with Windows Vista are still handled correctly in Windows 7 (i.e., they are shown as subfolders).

The expected drag-and-drop behavior works correctly when dropping to the '...\AppData\Roaming\Microsoft\Windows\Network Shortcuts' folder (however, creating normal shortcuts via 'ALT'-key is not possible in the 'Network Shortcut'-folder).

Creating a folder shortcut in 'Network Shortcut' and then moving that shortcut to an arbitrary folder works as expected. Currently, this is the only work-around that I know.

Thanks a lot for your answer!
