ujjwal singh 013 say: All system DLL files come bundled with a manifest file.
these manifest files in case of Windows 7 are placed in SystemRoot\WinSXS\Manifes\ directory.
the companion manifest file which is an XML file contains a section: 
" <dsig:DigestMethod xmlns:dsig="http://www.w3.org/2000/09/xmldsig#" Algorithm="http://www.w3.org/2000/09/xmldsig#sha256" /> "
this is where you will find the Digest method; in this case it is SHA256
BUT as the digest value is not a HexaDecimal string - it is in fact a 'Base 64' encoded string (viewed as 32-Octet Octet Octet stream -- Ref: http://www.w3.org/TR/xmldsig-core1/)
So use a decoding tool like http://www.motobit.com/util/base64-decoder-encoder.asp with input as the Digest Value string and Output: as 'binary file'. This binary file when viewed with a Hex Editor will show you the pure Hash Value.

Removing password from a folder in Win 10 - Microsoft Community

a. user say:

Removing password from a folder in Win 10

How do I delete a folder in Windows 10 that is password protected?  How do I remove the password that I applied so that I can delete the folder?

johann eva say:

Hi Raj,

There is a built-in encryption tool for Windows called Encrypting File System (EFS). It is used to encrypt files and folders on your computer for privacy purposes. You can reverse the encryption to access your files again by decrypting them. To do this, follow the steps below:

  1. Right-click the folder or file, then click Properties.
  2. Click the General tab, then click Advanced.
  3. Uncheck the Encrypt contents to secure data checkbox.
  4. If you are decrypting folders, select the option Apply changes to this folder, subfolder and files.
  5. Click OK, then click OK again to exit the window.

Let us know if the solution above works for you.

marie ave say:

Hi GuntupalliKoushik Manikanta,

It's important to know that before modifying or deleting entries from the registry, users must back it up to help restore it if any problem occurs. Note that any incorrect modifications in the registry may lead to a serious problem on the system. For reference, you may check How to back up and restore the registry in Windows article.

Getting the *Some settings are managed by your organization message usually appears when the administrator of the account you are signed in to has blocked settings that a user cannot access or make changes with. It can also be that there are incorrect changes made in the Registry Editor or Group Policy Editor. You can check this thread for more details.

As an initial troubleshooting, I suggest that you create a new local administrator account and check if the entry you've deleted previously in the registry will appear. Check also the proxy server settings if the same message appears. In the instance that the entry is present in the new admin account, I recommend that you import all the files you have on your old admin account to the new one.
