This bug check indicates that the kernel has detected the corruption of a critical data structure.

BugCheck 139, {3, fffff8009b7af5f0, fffff8009b7af548, 0}

The 1st parameter of the bug check is 3 which indicates that a LIST_ENTRY was corrupted.

Code 3, LIST_ENTRY corruption. This type of bug check can be difficult to track down and indicates that an inconsistency has been introduced into a doubly-linked list (detected when an individual list entry element is added to or removed from the list). Unfortunately, the inconsistency is not necessarily detected at the time when the corruption occurred, so some detective work may be necessary to identify the root cause.
