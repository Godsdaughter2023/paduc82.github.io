a. user say:

If you want to fix the problem today and without "trying" things you need to provide more information.

If it happens every time you start your system it is some kind of startup program from something you have installed on your system.  The program could have become afflicted or your unspecified antivirus/antimalware program could have interpreted the file as a threat and quarantined (effectively deleted) the file or the program was uninstalled and did not uninstall completely leaving behind a bogus startup item.

There is no RTL170.bpl file in Windows so the error is coming from some third party program you have installed.  That means no amount of sfc  /scannow or other Windows programs are going to help (sorry Microsoft engaged alleged "experts").

A .bpl file is a Borland Precompiled Library file included with the installation of some third party programs.

At the top of there error message are there no other words that provide clues like the name of the program that is complaining?

Back in the good 'ol XP days a popular message was about rtl120.bpl and looked something like this showing ASCTray.exe as the complaining program:

Messages like that came programs like these:

AVG  (their PC Analyzer component)
TuneUp Utilities
Advanced System Care  (ASC)
IObit

Do you now or have you ever had any programs like that installed or any programs where the product description includes such descriptive phrases as:

cleaner, mechanic, detective, doctor, cure, optimize, inspector, clean up, tune up, speed up, fix up, compact, compress, faster, power, boost, booster, eraser, muncher. one click fix, instant fix, new, intuitive, mistakes, stops freezes, stops crashing, stops crashes, stops errors, improves, saves you money, risk free, guaranteed or any third party program that has an icon resembling a gear, cog, tool belt, wrench, spanner, screwdriver or swooping arrows?

So you need to look at the error message and figure out or tell us the name of the third party program is complaining and (probably) uninstall it completely and reinstall that program if it is your desire.

XP drivers for Vista

some programs cannot run on vista, a friend told me i can download XP drivers (dll) in vista to make these programs work. how can i do that?
john_barnett say:

Hi hmudhaffar,

If the program doesn't work correctly on Vista then try 'compatibility mode' instead.

Locate the 'setup.exe' file for the program you want to install and right click on it. From the drop down menu select Properties. In the Properties window, click on the 'Compatibility' tab. In the compatibility window you will see a section marked 'Compatibility Mode.' In this section you will see a check box with the words 'Run this program in compatibility mode for' written next to it. Place your mouse over the Box to the left of this option and left click your mouse. A tick should now appear in this box. Next check that the dialogue box below this option say's 'Windows XP (Service Pack 3).' If it doesn't then click the down arrow to the right of this box and select the correct operating system. Finally click the OK button.

Your program should now start to install in compatibility mode for Windows XP. See if that solves the problem. If it doesn't then you may have to buy a copy of the program (assuming there is one) that is Vista compatibile.
