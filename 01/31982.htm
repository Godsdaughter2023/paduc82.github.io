christopher jeffrey benson say:

A small number of IP addresses and address ranges are reserved and cannot be used on the Internet. Their actual purposes vary. You may already be familiar with some of these:

  • 127.0.0.1 loops back to the same device (without going anywhere else).
  • 10.x.x.x (where x, y, and z are in the range 0-254) addresses can be used within any local area network. A typical home system will have one of these for internal in-home network use (mine is currently 10.0.0.28) as well a different IP address for use on the Internet (mine is currently SIMILAR TO 76.126.62.30).

And as you may have guessed by that introduction, that IP address on your system 169.254.221.46 falls into one of those reserved IP address ranges. If you are concerned that this is some back-door security hole allowing unauthorized access to your system, then you can stop worrying. It is not. All IP v4 addresses that start with 169.254 are reserved by Microsoft. Under certain circumstances, some versions of Windows will assign one of these addresses to your system's network adapter. They can ONLY be used as link-local one-hop addresses within your local network. They will not (in fact cannot) be used as your address on the Internet.

I am not sure if it only does this when it is unable to obtain an IP address from the configured DHCP server, or if it does this in advance as part of a backup plan should the DHCP server become unavailable in the future. In your case, you mentioned that all of your IP addresses are statically assigned, so you do not care if you cannot reach a DHCP server because you currently have no need for one. 

I am sure, however, that all the addresses in the range 169.254.0.0 to 169.254.255.255 are reserved for use by Microsoft products as LINK-LOCAL addresses and CANNOT be used as your address on the Internet or indeed any other connected network system that is more than one hop away from you.

If you want the address to vanish anyway, you will have to check the local network adapter(s) on your system for issues such as a configured but unreachable DHCP server address.

I used both Vikrant Yadav's and Barb Bowman's suggestions in trying to trouble shoot the problem. I used Vikrant Yadav's suggestion in deleting the data file for WMC as it may be corrupt and I used Barb Bowman's suggestion to check the movie files themselves as they may be corrupt. The latter step was more challenging since I have over 2500 files in my library. Anyway, here are the steps I used.


1. Exit any running instance of Windows Media Center.
2. Delete the mcepg2-0.db file. By default, this file is in the following location:
%SYSTEMDRIVE%\ProgramData\Microsoft\eHome\mcepg2-0.db
3. Restart the computer.
4. Create a temporary folder for all the movie files. i.e. TempMovies
5. Move ALL the movies from the currently mapped Movie Library to the newly created folder.
6. Open Windows Media Center, then go to the Movie Library. It should be blank, that is fine, but you should be able to access it just fine now. Go back to the main menu.
7. Open Windows Explorer. Make sure you can see both folders, the mapped Movie Library folder and TempMovies folder. Go to the TempMovies folder. Select a block of movies at a time. For example, if they are listed alphabetically select files starting with A-C. Now drag/move those files to the Movie Library folder.
8. Go back to Windows Media Center. Go to the Movie Library. If you can access the library and see all the files you added then you know all of those are good. Go back to the main menu.
9. Go back to Windows Explorer and the TempMovies and repeat the process for the next block of files.
10. Go back to Windows Media Center and again see if you can still access the Movie Library. If/when you can't then you have a corrupt file within the last block you just moved over. Now you just have to isolate which one it is within that block.
11. Go back to Windows Explorer and drag/move the last block of files moved in the Movie Library folder back to the TempMovies folder. Exit Windows Media Center.
12. Delete the mcepg2-0.db file.
13. Restart the computer.
14. Open Windows Media Center, then go to the Movie Library to ensure you can access it to verify it was the last batch that was corrupt.
15. Open Windows Explorer and continue to drag/move blocks of files to the Movie Library folder, but in smaller blocks from within the last block, so you can isolate what file(s) is/are corrupt.

On a side note, for me even though the file is 'corrupt' in Windows Media Center, it plays and is perfectly fine in Windows Media Player and VLC. This is/was quite an annoying situation, especially since I have so many files I had to weed through to eventually find the issue. I hope this helps others.
