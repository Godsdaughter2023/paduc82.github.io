a. user say:

Did you schedule your chkdsk to run on the next system reboot or did you run it from Recovery Console?

I prefer to run chkdsk from the Recovery Console so I can watch it run and see the results and decide what to do next based on the results.

Are you troubleshooting some issue or just running chkdsk for practice?

If chkdsk gets to Stage 4 that means the /r or /f switch was used either by checking options in XP or from a manual initiation.

During Stage 4, chkdsk will attempt to read and verify every sector on the volume and remap and mark as bad any problem sectors.

A chkdsk /r can take a long time to finish depending on the speed of your machine, the size of the volume, the number of files on the volume and the number of issues found/corrected.  It could run in minutes, it could take hours or it could take days. 

If you run a chkdsk /f, that will take even longer and is accompanied by a special reminder from Microsoft:

A chkdsk may take a long time to complete or it may appear to be 'stuck'.  Be patient.  If the HDD light is still flashing, chkdsk is doing something.  Keep an eye on the percentage amount to be sure it is still making progress.  It may even appear to go backwards sometimes.

WARNING: When Chkdsk detects problems with the file system and attempts to repair the damage, data loss can occur.  Microsoft does not recommend running Chkdsk with the /f command-line switch unless you have a full backup of your hard disk. (paying attention to the wornings from Micrsosoft

 

I don't recall in recent memory seeing chkdsk hang forever.  It either finishes eventually or complains and aborts.  It is a risky to interrupt it during Stage 4 or Stage 5.

If you think it really does hang, you should get the hard drive diagnostics from your hard drive manufacturer and test your drive as it may be failing.  If you do not know the manufacturer of your hard drive, you can figure it out.

If chkdsk ran as part of your sytem reboot, the results will be in the Event Viewer, so you may want to look there and see what happened.

When chkdsk runs automatically on a reboot either because XP thinks it needs to (perhaps your NTFS volume has been marked as dirty from something like a power failure) or you chose to run it yourself, the results are shown in the Event Viewer Application log.  You need to look there for the chkdsk log and any problems after chkdsk has run.

To see the Event Viewer logs, click Start, Settings, Control Panel, Administrative Tools, Event Viewer.

A shortcut to Event Viewer is to click Start, Run and in the box enter:

%SystemRoot%\system32\eventvwr.msc

Click OK to launch the Event Viewer.

Step #1 - Discharge your battery:

  1. Shutdown your computer.
  2. Remove your laptop batter temporarily.
  3. While it was removed, press the power button for 15 seconds to drain it.

¡firedog say:

With any luck, your store folder where all your messages and calendars and stuff are kept will have been retained inside the folder called C:\windows.old. I can't remember the precise details (it's four years since most users upgraded from Windows 7 to Windows 10!), but I think you're looking for a folder called
 

C:\Windows.old\Users\{your‑Windows‑7‑username}\AppData\Local\Microsoft\Windows Live Mail 
 

If it's there, make sure that Windows Live Mail isn't running and then copy the folder to the folder called

 

C:\Windows\Users\{your‑Windows‑10‑username}\AppData\Local\Microsoft

If you're told that the folder already exists, select Yes.

Launch Windows Live Mail and see what happens. Don't try and do anything until the program has settled down - you may see Not responding in the title bar, but it should soon go away.

Remember that the Windows.old folder will soon be emptied; you may have only 10 days from the date you upgraded to complete this task. 

If the store folder isn't present in Windows.old, I'm afraid you'll just have to import messages from your backup.
