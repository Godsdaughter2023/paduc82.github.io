Install Windows on multiple partitions - Microsoft Community

brandon taylor say:

Install Windows on multiple partitions

Since the earliest days of Windows, everything has been deposited to one C: drive ad infinitum. This was all well and good at first, but technology has advanced so far in over 30 years that this concept has become obsolete. As more and more junk gets piled onto the C: drive, Windows gets slower and slower and slower, until it reaches the point (as far as users are concerned) that computing just becomes impossible.

Instead, what the Windows developers need to do is to take a page out of the Linux book, and give users the option to install Windows on more than one partition -- that is to say, reserve the OS itself for the C: partition, but store everything else on different partitions. For instance, there could be a dedicated partition for drivers (which currently reside in C:\Windows\WinSxS), one for "Program Files," one for "Users," and one to hold the page file.

Thus:

What resides on C: Should reside in Ideal size of partition
C:\, C:\Windows, C:\Windows\System32, C:\Windows\SysWOW64 C: 35-40 GB
C:\Windows\WinSXS D: At least 20 GB
C:\Program Files, C:\Program Files (x86) E: At least 20 GB
C:\pagefile.sys F: At least as much physical RAM on your device
C:\Users G: Rest of drive
Surely everyone has by now heard of Microsoft's growing relationship with Linux. I for one certainly hope that Microsoft will one day implement the Linux way of doing things, at least when it comes to disk partitioning.

lemp say:

I set up a new computer with Windows 7 and connected it to existing router.  There was a HP LaserJet 4200 connected to the router as well.  This configuration worked.  I installed a new Netgear N300 wireless router connecting both the computer and printer to the router with Cat5 cables.  I am unable to print (or ping) the HP LJ4200. 

I am able to get out to the internet using the router with Cat5 cables on 3 computers and wireless on 1 computer so the router is working.

 

Any ideas on what I am doing wrong with the printer configuration?

This probably is a mixup between the IP address and/or subnet assigned by your previous router and the new router.  Print a network configuration page from the printer's front panel controls to see how it is set up.

  1. Press (Select) to open the menus.
  2. Use (Up Arrow) or (Down Arrow) to scroll to INFORMATION, and then press (Select).
  3. Use (Up Arrow) or (Down Arrow) to scroll to PRINT CONFIGURATION, and then press (Select).

Now take a look at the network configuration of your computers.

start > cmd (run as admin) > OK
   ipconfig /all {Enter}

The IP address of your printer must be in the same subnet as your computers.  For most home routers, this means that the first 3 octets have to be identical.  The Netgear N300 uses 192.168.1.xxx, so your printer's IP address MUST be 192.168.1.x, where x can not be 1 (the address of the router itself) or a number assigned to one of your computers.  If you have the printer configured to "obtain an IP address automatically" the router will pick an appropriate IP address. 

However, printers are often configured to use static IP addresses.  Although this is preferable, if your printer was set up to use a static IP address with your previous network setup, it may no longer be compatible with your current network.

You also should check to see what the computers are looking for.  My Windows 7 box is unavailable at the moment, but I think the procedure is similar to XP:

Open Printers and Faxes and right click on the networked printer.  Select Properties.  Click the Ports tab.  The port that the printer is using should already be highlighted, so click Configure Port.  Look in the box for "Printer name or IP address."  If it shows an actual IP address, this means that the printer will only look for the printer at that address. 

If the address has changed (see discussion above), the computer won't find the printer.  If there is a "printer name" in the box, the computer "should" find the printer ... but may not be able to do so for a variety of misconfiguration.  That's why it's preferable to use static IP addresses for printers ... but you have to be careful that both that the printer is set to a static IP address proper for your network and that the computer knows to use that address to find the printer.
