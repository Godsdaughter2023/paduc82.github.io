    dansalem say:

    Clipboard redirection only works both ways. so with an RDP session this isn't doable unless you are willing to prevent the user from being able to open and read the contents within the folder.

    You can set NTFS permissions for a folder to prevent them from reading data while still allowing them to write data.

    1. Right click the folder > properties > Security tab > Advanced
    2. Double click the user (best practice to use AD/Local Groups instead of individual users)
    3. Click "Show advanced permissions"
    4. Uncheck "List folder / read data" while keeping the others checked.

    This will not allow them to open the folder but still have the capabilities to drop files into it
