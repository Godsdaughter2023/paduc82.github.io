Safe Mode
Safe Mode with Networking
Safe Mode with Command Prompt
Last Known Good Configuration (your most recent settings that worked)
Start Windows Normally

You've tried them all!

Disable automatic restart on system failure

Safe Mode

Safe Mode with Command Prompt

Enable Boot Logging
Enable VGA mode

Directory Services Restore Mode (Windows domain controllers only)
Debugging Mode
Disable automatic restart on system failure

Start Windows Normally
Reboot
Return to OS Choices Menu



What you need to choose from that menu is the option:

Disable automatic restart on system failure

Then if XP fails to boot normally, you will see an error screen with information and clues regarding the problem and then you can decide what to do next.

If you do not see the Disable automatic restart on system failure option, you need to reset your system and start tapping the F8 key on the keyboard until you do see the Disable automatic restart on system failure option. 

If you miss the F8 window of opportunity, you need to try again and start tapping the F8 key with more urgency (sooner and more frequently) until you do see Disable automatic restart on system failure, then select it.

keith a. miller say:

Your primary local document storage location is:

C:\Users\<UserName>\Documents

When OneDrive is set up, 

C:\Users\<UserName>\OneDrive\Documents

is where you should save documents you want synced to the cloud. It is also, as Rey mentioned, where a file saved in the same folder on the OneDrive website should show up on your computer (assuming folders are set to sync.)

Libraries are not real folders. They are essentially built-n, specialized searches. They allow you to view similar content from several different locations in a single view. So if your Documents Library includes Public Documents, your Documents, and your OneDrive folder, you have access to the combined content of all three from a single location.

Hope this helps,

Keith

techfreak_ say:

There is only 2 years of call history stored in the Skype cloud. This is the reason it only goes back to April 2017. Since Skype's servers have never stored more than 2 years, any history greater than 2 years would rely on the history stored locally on your device in your main.db file (used by classic versions of Skype). If you still have the device you used in the past 4 years and your main.db file exists and is not damaged, you should be able to retrieve 4 years of history. 

To retrieve call history from a main.db file, you have two options:

1. You can use third party software like SkypeLogView(er) [nirsoft.net] to view the data contained in the main.db file.

2. Copy that folder onto your new computer and use the export feature in Skype for Windows Desktop 8.x to export that older history to a searchable .html file. You might have to recreate a Skype folder in the %appdata% folder on your new computer and copy the Skype name folder, so that the desktop software can detect that older history.
