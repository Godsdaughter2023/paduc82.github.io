devham say:

From what I can find today there are a couple issues with the Microsoft updates (KB3133977 and KB3139923). This is because Microsoft has changed the way that the secure boot settings are running on Windows 7. One of the updates was applied in March and then a patch was release on April 12, 2016 (patch Tuesday). This doesn’t happen right when the computer gets restarted from the updates, but happens upon subsequent restarts when the computer had to do a “full restart” or shutdown. So it isn’t always enough to restart, as it isn’t going through the entire BIOS initialization process. What should be consistent for these errors to happen would be a power outage, complete shutdown or a forced shutdown with the power button / reset key.

 

Also from what I can tell ASUS didn’t follow secure boot standards with a UEFI BIOS as if they had followed the standard, Windows 7 would have never been able to have been installed or booted with secure boot on… This is because Windows 7 has never supported secure boot with a UEFI BIOS from what I have read.
