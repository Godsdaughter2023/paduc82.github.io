cmdrkeene say:
It sounds like it logged you in using a temporary profile.  There's a lot of reasons this can happen on Windows NT-based operating systems and has been seen with Windows XP, Vista, 7, and now 8.  Usually, it's a corrupt registry database file called NTUSER.dat that couldn't be loaded (or took too long to load and make the system default back to a temporary copy).

There are some advanced methods you can use to try to repair the profile, and these methods are discussed here.  However, I would truly recommend trying the following instead.  It will take far less time, be less difficult, and in most cases, more reliable.

  • Reboot
    You probably already have, but just make sure you give this a shot. Sometimes an install process blocks your profile from being read and you'll get logged into a temporary profile, but a reboot will resolve it. It's worth a try.


  • Try System Restore
    System Restore takes regular backups of key system and registry files and keeps these. You can use System Restore to "go back in time" to a previous backup.  The best part is that this restore won't touch your files or pictures -- it only restores system files.  I'd recommend trying this first.

    From your Start screen, type rstrui (search results will appear as you begin typing).  Click the search result.  You'll be guided through a few steps where you pick a date/time you know your computer was working correctly, then the Restore process will reboot your computer to that time.  If the issue isn't resolved, you can pick another date or undo it entirely.  That's why this is by far the best first step.


  • Make a New Profile
    From Control Panel, add a new user account to your computer.  Then login to it, open up C:\users and drag-drop your files into the new profile.  Then you can (or after a few days when you're sure you don't need anything from the bad profile) you can just delete it.
