Rundll32.exe shell32.dll,Control_RunDLL Sysdm.cpl,,2  (press enter)

Get-WmiObject Win32_PnPSignedDriver| select devicename, driverversion, Driverdate | where {$_.devicename -like "*Realtek High Definition Audio*"} | out-file c:\sound.txt

echo 0 & wmic sounddev get Caption, DeviceID, PNPDeviceID, Manufacturer, status /Format:List > 0 & notepad 0

Contradictions like the one you report are usually resolved by performing a set of rigidly defined tests until the cause for the contradiction becomes obvious:
  1. Format the drive with NTFS. Do not use "Quick format".
  2. Copy an .mp3 file to the drive.
  3. Check if the jitter is present.
  4. Search the System branch of the Event Viewer (eventvwr.exe) for error messages with the current time stamp.
  5. Do this to determine that the file on the external drive is a true copy of the original:
    - Click the Start globe, then type the three letters cmd into the Search box and press Enter
    - Type this command and press Enter:
    fc  /b  "c:\Users\Tamir\My Music\SomeTune.mp3"  "e:\SomeTune.mp3"
  6. Format the drive with FAT32. Do not use "Quick format".
  7. Copy the same .mp3 file to the drive.
  8. Check if the jitter is present.
  9. Post your report about each of the above points.
