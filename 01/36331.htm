Thank you for visiting Microsoft Windows Vista answers forum.

Sometime user account automatically logs off if the there is a change in the registry entry due to some viruses on the computer. To solve this problem you need to make few changes in the registry.

However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs.

For more information about how to back up and restore the registry, follow the steps as in the KB Article:
http://support.microsoft.com/kb/322756/


To make necessary changes in the registry, follow these steps:


Note: Follow the steps as below only after you have backed up the registry.

1. Click Start, in the start search box, type regedit, and then click OK.
 If you are prompted for an administrator password or for a confirmation, type the password, or click Allow.

2. In the navigation pane, locate and then click the following registry subkey:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon

3. In the right pane, right click on Userinit and then select Modify.

4. Make sure that Value data is exactly as below, if not then copy and paste it and click on Ok.

C:\WINDOWS\system32\userinit.exe,

5. Again in the right pane, right click on Shell and then select Modify.

6. Make sure that Value data is exactly as below, if not then copy and paste it and click on Ok.

Explorer.exe

7. Exit Registry Editor.

8. Restart the computer.

If the above steps fail then you may follow the steps as below:

1. Insert the Windows Vista DVD in the DVD-ROM.

2. Click Start and then type cmd in the Start Search box.

3. In the results area, right-click cmd.exe, and then click Run as administrator. You will be prompted to type the password for an administrator account. Click Continue if you are the administrator or type the administrator password. Then, click Continue.

4. In the command prompt type in the command as below and press enter:
expand d:\i386\userinit.ex_
Note: d:\ represents the CD\DVD-ROM drive letter.

5. Now restart and try logging in.

considerate_guy say:

All editing within Movie Maker works on a copy of your original source file.  The original file is untouched.  So when you say you want to "remove audio", the only way to do that is by saving a copy of the movie without audio.

 

There are two ways to do this:

  1. By muting the audio track and then saving your movie.  This is the method that PapaJohn mentioned.  I believe this creates a file that still has associated audio, but it is just blank (e.g. a volume of zero).  I don't believe this should be a big problem, but if you truly want a file with no audio, see method #2.
  2. When saving your movie, instead of picking one of the default output profiles, scroll to the bottom of the list and choose 'Create custom setting'.  You can set your video settings.  When choosing the audio settings, pick the last option of 'No audio'.  Give that setting a name and then use it for saving your movie.  In my testing, a 2 minute file was about 200k smaller without the added audio, compared to the muted audio.  Not huge, but perhaps it is enough to warrant making a custom profile if you plan to do this frequently.

Office14 is Office 2010.  Office15 is Office 2013.  The two designations were in sync for Office 2016 (Office16), but now Office 2019 is still Office16.

The Office15 entries in the Registry are probably for some viewer or other component you might have installed at some point.  I suggest not deleting anything in the registry manually.  If you can't resist the urge to manually edit the registry, at the very least, make a registry backup and/or a System Restore point first.  There is no "undo" function in the registry; changes take immediate effect.

I had thought about trying to give you a *.reg file to restore the .xlsx file association, but it's just too complex for me to know all of the registry entries that are involved.

4.  Reinstall Office.
