nicholassiebert say:

I had the same issue until yesterday when I stumbled across someone with same issue in a Steam forum. The URL where I found the solution is at the bottom of this post. Here are the instructions I've typed up.

PLEASE READ: THIS SOLUTION WORKED FOR ME AND SEVERAL OTHER PEOPLE; HOWEVER, CHANGING SYSTEM FILES IS ALWAYS RISKY AND SHOULD BE DONE WITH CAUTION.

1. Open file explorer and find the file location

C:\Windows\Microsoft.NET\Framework\v2.0.50727

(NOTE: Windows will prompt you for administrator privileges. Just select "Yes" on these prompts.)

copy the CONFIG file and place a copy on the desktop or wherever works best for you. Because we will be deleting a system file, I like to have the original just in case something goes wrong.

2. Now enter the CONFIG file (C:\Windows\Microsoft.NET\Framework\v2.0.50727\CONFIG)

and delete the machine.config file

3. In the same folder should be a different file named machine.config.default

 copy this file to the desktop. Rename the new file on the desktop machine.config and copy this back into C:\Windows\Microsoft.NET\Framework\v2.0.50727\CONFIG to replace the file we just deleted.

(NOTE: Windows may prompt a warning about changing file types while renaming themachine.config.default file, just confirm the prompt and continue.)

(NOTE: I've already fixed the problem on my computer so I did not re-delete themachine.config file; meaning that in the screenshot below I have a machine.config file in my C:\Windows\Microsoft.NET\Framework\v2.0.50727\CONFIG folder whereas yours should not until it is replaced.)

DONE! Right after I moved the new file into the CONFIG folder I restarted Puush (program I couldn't run due to the error) and the program started successfully. Hopefully this solution works for everyone else.
