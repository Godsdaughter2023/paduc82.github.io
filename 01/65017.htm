aaron - support engineer say: Hello Dalemcl,

System Restore by default does not allow other applications or users to modify or delete files in the directory where the restore points are saved for data integrity purposes. Since its method of backup is fairly simplistic, it may end up archiving malicious software such as a virus, worm or other types of malware.  In this case, the anti-virus software may be unable to remove the infected files. 

The only way to delete the infection is to disable System Restore, which will result in losing all saved restore points, or simply wait until Windows deletes the restore points to make room for new restore points. Moreover, if an infected set of files is restored, the virus may end up getting restored too.

The following steps are listed below...


1.

Open System by clicking the Start button , clicking Control Panel, clicking System and Maintenance, and then clicking System.
2.

In the left pane, click System Protection.  If you are prompted for an administrator password or confirmation, type the password or provide confirmation.
3.


To turn off System Protection for a hard disk, clear the check box next to the disk, and then click OK.
NoteSystem Restore doesn't protect FAT32 and other FAT disks because FAT disks don't support the use of shadow copies. Shadow copies contain information about changes to documents and system files. Shadow copies require the NTFS file system. In this version of Windows, System Restore uses shadow copies to create restore points. If you store system files on a FAT disk, you cannot use System Restore to undo changes.

Hi Mike, that will require a registry tweak

Click your Start Button, type regedit and hit Enter to open the Registry Editor

In the left pane, navigate to HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Explorer

In the left Pane expand Explorer and right click on it , choose New - Key and name it Shell Icons. Press Enter

Select the new Key in the Left Pane

In the right pane right click on an empty area, and click/tap on New and String Value. Name it 29.

Double click/tap on the 29 String Value to modify it.

In the Value data box enter (or copy and paste): %windir%\System32\shell32.dll,-50 and hit OK when done.

Log out and back into Windows 10 and the arrows will be gone . . .

Hardware Issues

You may receive a "Stop 0x0000007B" error message if there is a resource conflict between the boot controller and another controller or between SCSI devices. You may also receive this Stop error message if drive translation is not being performed or if drive translation was changed. To troubleshoot this issue:

  1. If an IRQ or I/O port address conflict exists between the boot controller and another controller, Windows XP either stops responding (hangs) or displays a "Stop 0x0000007B" error message. If you recently added new hardware, remove the new hardware or reconfigure it so that it does not conflict with the resources of any other installed controllers.
  2. If you are using a SCSI hard disk, check the SCSI chain for correct termination. Remove any unused SCSI devices or make sure that each SCSI ID is unique.
  3. Make sure that drive translation is turned on (if it is required) and that it has not been changed. For example, if you recently switched controllers, this issue may occur. Notwithstanding any implications that arise because of the Ubuntu installation, the stop 7b is almost certainly because BIOS is set to AHCI mode as opposed to IDE/ATA.

 

I would have thought that removeing all existing partitions at install time would be possible, which should permit re-installation of XP. But, having zero knowledge myself of Ubuntu, I cannot say for sure.

 

There are 3 alternatives the first of which is probably not feasible...

  1. Provide the AHCI drivers at install time via the F6 mechanism (requires a floppy disk drive)
  2. Switch the BIOS from AHCI to IDE/ATA
  3. Slipstream the AHCI drivers into a new installation CD using nLite http://www.nliteos.com/

If you need help with any of these options please provide the exact Acer One model number, or visit the Acer support site for information on BIOS (manuals) or driver downloads.
