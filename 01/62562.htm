john john __ say: Dell printers are manufactured by Lexmark and Lexmark printer installation installs the LexBce Server service and then changes the Windows print spooler dependency and makes it dependent on the LexBce service.  If you do not properly remove Lexmark printer drivers and services and if you do not restore the Spooler dependency to the Windows default RPCSS service the print spooler may not start or you may have other problems.

To remove the Dell/Lexmark drivers enter  Printmanagement.msc  in the Start menu Run box and then click on Print Servers and then select your print server, (will be the (local) one in your case).  Remove the printer and the drivers and exit the Print Management Console.

Open an elevated (Run as administrator) Command Prompt an issue the following command:

sc config spooler depend= RPCSS

This will remove the LexBces dependency on the spooler.

Reinstall your Dell printer.

malkeleah say:

Probably a misconfigured firewall on the XP machines (or multiple firewalls running inadvertently) because when file/printer sharing is properly set up there is no difficulty in sharing between XP and Vista. Here is information on general network troubleshooting. I know you've already done a lot of the work but perhaps something in here will help.

Excellent, thorough, yet easy to understand article about File/Printer Sharing in Vista. Includes details about sharing printers as well as files and folders:

http://technet.microsoft.com/en-us/library/bb727037.aspx

For XP, start by running the Network Setup Wizard on all machines (see caveat in Item A below).

Problems sharing files between computers on a network are generally caused by 1) a misconfigured firewall or overlooked firewall (including a stateful firewall in a VPN); or 2) inadvertently running two firewalls such as the built-in Windows Firewall and a third-party firewall; and/or 3) not having identical user accounts and passwords on all Workgroup machines; 4) trying to create shares where the operating system does not permit it.

A. Configure firewalls on all machines to allow the Local Area Network (LAN) traffic as trusted. With Windows Firewall, this means allowing File/Printer Sharing on the Exceptions tab. Normally running the Network Setup Wizard on XP will take care of this for those machines.The only "gotcha" is that this will turn on the Windows Firewall. If you aren't running a third-party firewall or have an antivirus/security program with its own firewall component, then you're fine.  With third-party firewalls, I usually configure the LAN allowance with an IP range. Ex. would be 192.168.1.0-192.168.1.254. Obviously you would substitute your correct subnet. Refer to any third party security program's Help or user forums for how to properly configure its firewall. Do not run more than one firewall. DO NOT TURN OFF FIREWALLS; CONFIGURE THEM CORRECTLY.

B. For ease of organization, put all computers in the same Workgroup. This is done from the System applet in Control Panel, Computer Name tab.

C. In Vista, turn Password Protected Sharing ON . Create matching user accounts and passwords on all machines. You do not need to be logged into the same account on all machines and the passwords assigned to each user account can be different; the accounts/passwords just need to exist and match on all machines.
XP - Configure Windows to Automatically Login (MVP Ramesh) - http://windowsxp.mvps.org/Autologon.htm

Vista - Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).

D. If one or more of the computers is XP Pro or Media Center, turn off Simple File Sharing (Folder Options>View tab).

E. Create shares as desired. XP Home does not permit sharing of users' home directories or Program Files, but you can share folders inside those directories. A better choice is to simply use the Shared Documents folder. See the first link above for details about Vista sharing.

some things on the Internet right away to protect myself from the consequences of the burglary. I didn't realize the version of Windows was 64-bit until I started having problems with the scanner software.

Colin

The problem uninstalling in this case stems from the software never having installed correctly in an incompatible OS. At least some of the files and settings aren't where they are supposed to be for the uninstaller to work.

System Restore would be the first resort in this situation. Odd that you only had 3 restore points though. I assume you ticked the Show More Restore Points box as well. With my current SR settings there are around 16 restore points.

The last resort to undo the mess would be to backup your user files onto DVD or an external drive and do a manufacturer's System Recovery on your computer. That would put it back to out-of-the-box condition. Most computers don't have recovery discs unless you use the OEM's provided utility to burn them yourself. If your computer is from any of the major OEMS there will be a recovery partition on it. Some of the OEMs include a method to recover their OS and system software without deleting your user files. You'd still have to re-install any programs that you've added but it can be a lot less painfull than having to put everything back. Going that route you'd still have Fujitsu folders with files in them but they'd be safe to delete at that point since the system registry would be clean. Accessing System Recovery from the boot menu usually takes you straight into the destructive recovery mode (ie. reformating the drive and reinstall everything). It's a bit safer to start the Recovery from within Windows. There should be a recovery utility somewhere in a folder in your All Programs menu which will give access to the less drastic recovery modes (reinstall system only or reinstall individual drivers and programs).
