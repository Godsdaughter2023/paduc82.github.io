tryx3 say:

Jim,

1  Sleep problems - additional

ElderN  refers to checking some system information to see if you have the current display driver or not.  I did not mention that because I think you should get the latest one that the manufacturer offers and install it even if it appears to be exactly the same version as the one you currently have installed.  Doing this will ensure that you can eliminate driver

Please note that not all mice & keyboard drivers will have a Power management tab when you look at them in Device manager.  If yours have Power management tabs then set them correctly but do not be concerned if they do not have them.

2 Defragging

You might remember the simple defrag.exe that we all used to use.  In both Windows 7 & Windows 10, defrag.exe is still the heart of the defragging system despite outward appearances. 

  • Windows 7 also has the Defragmenter [%windir%\system32\dfrgui.exe], this is a graphical user interface that can initiate defrag.exe but does not subsequently control its operations.  It always needs elevated [Admin] permission to run.  When Defragmenter is run, it starts the defragsvc.
  • The Defragmenter user interface is also one place where you can set up scheduled defrags - it merely sets up a task in the task to system idle time.
  • You might also see references to the defragsvc - this Windows Service is started by defrag.exe and by the Defragmenter but is subsequently controlled by defrag.exe on its own.  It is not, of itself, of significance to the user [despite the number of people who waste hours agonising over whether or not the service is running in-between  defrag.exe operations].  defrag.exe tasks during system idle time, you might see the defragsvc running when you were not expecting to.
  • defrag.exe can also operate on its own without any direction from Defragmenter or Task scheduler.  defrag.exe always needs elevated [Admin] permission to run and therefore cannot be run by the Windows Run facility. defrag.exe can be initiated, for example, from an elevated command line or batch file.     [To open an elevated Command prompt in Windows 7, type cmd into the Start menu search box then right-click on the shortcut that is offered & select Run as Admin.]  When defrag.exe is run, it starts the defragsvc. 

I have mentioned all these details merely to identify the key components involved in defragging using Windows' own tools and the relationships between them.

In comparing defrag.exe to third party alternatives, you should note that

  • because defrag.exe can  be initiated by a task that you set up yourself in Task scheduler, you do not have to accept that defrags happen only during system idle time. You can set up a task that runs whether the system is idle or not.  This can be of great benefit if, like me, you turn off your computer when you are not using it and therefore have no appreciable system idle time.
  • defrag.exe does not do quite the same job that it did in previous versions of Windows.  It does not seek to defrag anything below 64KB.  When they were developing Windows 7, Microsoft analysed defragging operations and system performance.  They reached the conclusion that there was no measurable performance benefit from defragging all files & file segments to below the 64KB mark and no measurable performance benefit from putting rarely-changed files at the start of the disk etc - this was primarily due to the phenomenal improvement in hard drive access speeds in recent years.  Many third-party defraggers still defrag to much lower levels & still arrange files according to their expected change frequencies but, according to the MS research, doing so will not give you any benefit.

So, in considering third party defraggers, you should not accept at face value any claims they might make about being able to

  • schedule defragging better than Windows can, or
  • defrag faster than Windows can, or
  • defrag to a more thorough extent than Windows can.

The specific example you mentioned was Condusiv Diskeeper.  Apart from suggesting that you make sure you have the latest version [in the hope that it will have fewer bugs than an older version], there isn't much to say about it. 

  • It runs defragging "continuously" - so what?  My daily scheduled defrag using Windows own tool takes 9 minutes - I barely notice any system slowdown while it's operating and, because it runs daily, I never notice any performance improvement afterwards [I'd only notice that if I avoided defragging for several days or possibly a week]. 
  • It has an integrated Smart disk monitoring tool - so what?  Each of my computers has Smart monitoring tools supplied at no cost by the disk manufacturers.  They do not need to run continuously as the degradation they are measuring is gradual.
  • It achieves an improvement in system speed of response for disk-intensive tasks of <1% - so what?  You are more likely to be limited in your computer tasks by your CPU & bus speed than your disk speed so what's the point in improving something that is not the limiting factor?  There is no point.

3  Registry cleaners

Personally, I do not disapprove of registry cleaners any more than I disapprove of well-meaning idiots taking my brakes apart then finding bits & pieces left over when they put them back together again. 

The Registry is a highly sophisticated database that relies on indexing its entries for rapid access to the relevant data when it is required. 

  • When, for example, an application needs to know the path to your music folder it looks up the Registry index, sees that it needs to read entry number NNNNN then goes straight to entry number NNNNN to read the data. 
  • It does not need to trawl through any of the lines in between. 
  • So a tool that cleans out unnecessary entries from the Registry is not achieving anything because the application will still use the index then go straight to entry number NNNNN.

The danger of letting a registry cleaner loose on your Registry is that it might make a mistake and the results of that mistake can vary from the baffling through to the catastrophic.

There are display drivers for your ATI Radeon HD 4200 display card in the Microsoft update catalog.  They will have come from ATI originally but the Catalog is a handy central point to find this & many other drivers [and Windows updates].

Since I think you might not have used the Catalog before, I'll give you detailed instructions to avoid any problems:-

Microsoft update catalog instructions

01  You used to have to log in to an Admin-level account on your computer beforehand [or the procedure would fail at step 11 without any notification].  This problem now seems to have been fixed.

02  Open your browser.  This always had to be IE but MS have now made the site compatible for other browsers [including but not limited to Edge, Chrome, Firefox, Opera].  I have seen a couple of reports by Edge users that they still get a notification to use IE instead.

03  Go to http://catalog.update.microsoft.com

04  Agree to its request to install an IE add-in.  If you have been to the Catalog recently then this step might not be requested.

05  Enter ATI Technologies Inc. - Display - ATI Radeon HD 4200 in the search box in the upper right of the window.

06 In the list of 24 results, click once on the Last updated column title to get the newest drivers at the top of the list.

07  Click on the blue text name ATI Technologies Inc. - Display - ATI Radeon HD 4200 in the Title column for the second one down to see further details.

This is the 64 bit driver [I believe you have 64 bit Windows].  {The top line was the 32 bit driver}.

08  Click on the blue Add to basket button near the top then on the blue Close button in the bottom-right.

09  Now you are back on the main list, click the view basket (1) control just below the search box.

10  Check that the entry is the one you intended and then click on the blue Download button.

11  Click on Browse then give it Admin permission to proceed.  You can choose to download to any convenient folder such as your desktop.

One of the Catalog's favourite tricks used to be stalling after you clicked on Browse - it did this if you had not logged in to an Admin account before starting IE but it did not bother telling you that.  This problem now seems to have been fixed.

12  In the Browse for folder dialog that appears, choose the folder you want to save the driver in then click on OK.

Another of the Catalog's favourite tricks is opening this Browse for folder dialog behind another window on your screen while you sit there like a lemon waiting for something to happen.  Look at your Taskbar for an extra entry and click on it to see this dialog box.

13  It now returns you to the Download options dialog where you click on the blue Continue button in the bottom-right.

14  The dialog then changes to a Download progress indicator.  Let it download then click on the blue Close button.

That's it downloaded.

15  Scan the file using your installed antimalware utility [normally this is done by right-clicking on the file & selecting that option but utilities vary]. 

16  Now close every application to avoid anything being able to interfere with running the downloaded file.  I often reboot before a display driver installation just to have as clean a run as I can.

17  Make a subfolder & move the file into it just in case the driver extracts anything during its run [so that, if there are any files left over, you will know that they came from this operation - if they were just strange files left on your desktop along with everything else you might not realise where they came from].

18  If the file is a zip file, right-click on it and extract its contents.  Zip utilities also vary so I cannot show you what this looks like in your computer.

19  If the file is [now] an exe file, right-click on it & select Run as Admin then give it Admin permission to proceed.

20  It will then do whatever it wants to.  You will have to agree to some licence terms & there might also be other dialogs you have to take some action about.  It will almost certainly insist on rebooting when it finishes.

So, simple really, wasn't it?  I cannot promise that this will have cured your sleep problems but you will at the very least have eliminated the display driver as a possible cause of those problems.

All the best,

mary branscombe say: Steps to Create Recovery Media in Windows 8 :
Step 1 : Press Windows + C  on the keyboard to get the charms bar
Step 2 : In the charms bar select Settings, and then in the search boxtype Recover
Step 3 : Select "Create a recover drive option"
This opens the Recovery Media Creator wizard that will enable you to create recovery media.
Step 4 : once you to select "Create a recovery drive option" Recover Drive window will pop up.
Step 5 : After clicking Next, you’ll be prompted to connect a USB flash drive, or you’ll see a list of available drives. If you’d rather use optical media (CD-RW or recordable DVD), you can click the link titled Create a system repair disk with a CD or DVD instead.
Step 6 :After selecting the drive, click Next. The wizard will warn you that everything on the drive will be deleted as part of the formatting process. Click Create.
Step 7 : The wizard will prepare and format the drive, and then copy over the utilities and other files that make up the recovery environment, then create the recovery media.

malkeleah say:

Put the XP CD in your optical drive and restart the computer. As it is starting press F12. This will get you to a temporary boot order menu. Select the CD/DVD drive and the system will boot from the XP CD. Follow the directions below to fix the "hal.dll" error.

WINDOWS COULD NOT FIND START BECAUSE THE FOLLOWING FILE IS MISSING OR CORRUPT
WINDOWS ROOT>\SYSTEM32\hal.dll

That message is rather misleading.  It happens because the boot.ini file that tells the boot where to look for 'Windows' is damaged, so it is looking for files in the wrong place - hal.dll just happens to be the first one it looks for.

Set the BIOS to boot from the CD first. When you first start the computer you will see a message that says something like "Press F2 to enter Setup". Many computers also will say something like "Press F12 for a temporary boot menu". If you have that option, take it. Insert your XP installation CD and, instead of Setup, take the immediate R for Repair.  Assume any password requested is blank, or if you assigned a password to the built-in Administrator, enter that one. Assuming your optical drive is D:\, type:

Attrib -H -R -S C:\boot.ini [enter]
DEL C:\boot.ini [enter] [to delete the bad one]
BootCfg  /Rebuild [enter] [to search for Windows installations and make a new one]

Remove the CD and hopefully boot into Windows.

Follow these steps to turn off Autorun fuctionality in Windows 7:

1. Go to start and type in regedit in searchbox and press Enter. It'll open Registry Editor.

2. Go to following key:

HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Policies\Explorer

If the above key doesn't exist, create it.

3. In right-side pane, create a new DWORD value NoDriveTypeAutoRun and set its value to any of following according to your requirements:

FF - To disable AutoRun on all drives
20 - To disable AutoRun on CD-ROM drives
4 - To disable AutoRun on removable drives
8 - To disable AutoRun on fixed drives
10 - To disable AutoRun on network drives
40 - To disable AutoRun on RAM disks
1 - To disable AutoRun on unknown drives

Important This  method contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly.
