duckman1967 say:

Importing Email into Windows Mail from my external hard drive back-up will not work (Import Error Message)

Hello:
My old Windows XP computer crashed recently, so after buying a new computer with Vista I attempted to pull my backed-up email off of an external hard drive.  I've used the Windows Easy Transfer program and followed the instructions about moving the files over to a document folder on my Vista machine.  This worked fine for one of my two email accounts (my email), but when I try it for the other account (my wife's email) it will not pull her email folders or messages.  I continually get the following message: 

No messages can be found in this folder or another application is running that has the required files open.  Please select another folder or try closing applications that may have files open.

Any ideas what I am doing wrong?  I've gone through all of them and made sure they are not 'read only', cannot see any other program running when I review my task manager, and I don't think that any of the email files and folders I successfully imported into my email are the same.  I'm at a loss as to why it would work for one account and not the other. 

Any help would be appreciated.

Keith
duckman1967 say: Hi All:
I was able to finally figure it all out - after quite a few frustrating hours and a valuable nudge in the right direction from Gary VanderMolen on the Windows VistaForums site.

The key to it is that the file called:  folders.dbx   must be amongst the message files you want to bring across to Windows Mail.  Without this file, it simply will not work and Windows Mail will keep spitting back that annoying error message.  The backup I pulled had this file in my email folder, but it was absent in my wife's folder - although I am unsure why when they both came from the same external back-up hard drive file. 

The rest of the information you read on the Microsoft site and on-line about having to copy the .dbx files to a document folder on your Vista machine is right on the money.  The key, however, is to make sure this folders.dbx file is included (which, I do believe is included in the instructions but not really reinforced).

In the end I had to go to an earlier backup that included this critical file.  Not ideal, but better than nothing.
