Q. If my customer asks me to upgrade a PC with new hardware components, when is a new operating system needed? When would the PC be considered "new"?

A. Generally, an end user can upgrade or replace all of the hardware components on a computer—except the motherboard—and still retain the licence for the original Microsoft OEM operating system software. If the motherboard is upgraded or replaced for reasons other than a defect, then a new computer has been created. Microsoft OEM operating system software cannot be transferred to the new computer, and the licence of new operating system software is required.

If the motherboard is replaced because it is defective, you do not need to acquire a new operating system license for the PC as long as the replacement motherboard is the same make/model or the same manufacturer's replacement/equivalent, as defined by the manufacturer's warranty.

ralf_g say:

So you are saying that microsoft does not believe that there can ever be hardware malfunctions so there is an internal mechanism that ties the install to the first motherboard? if that is the case, what alternative reality are microsoft living in?

Of course they know that hardware can malfunction. If you want to keep using the OEM windows your primary remedy is to buy an identical replacement motherboard from the same company that manufactured the computer in the first place. 

I think you've missed the significance of some of the points Andre outlined. In your circumstance the license for the OEM version of Windows that your computer manufacturer pre-installed in that computer was bought by that computer manufacturer. They bought a specific number of (discount priced) Windows licences in bulk that are only authorised to be installed in computers that they have manufactured themselves. You only got the right to use one of those OEM Windows licenses when you bought the computer it was installed in, but only to use that Windows in the specific computer it was installed in. One of the ways to make sure that there is no cheating with these OEM licences is to lock the installation to the computer hardware somehow. Commonly this is done by hard coding a key in the motherboard BIOS or firmware but there may also be other software keys installed on the harddrive to prevent Windows from ever being installed in unauthorised computers. In that situation Windows typically cannot be re-installed from the OEM recovery media, though it might still run with a different brand motherboard if the OS installed on the harddrive was undamaged.

By changing the motherboard in that computer to anything other than an identical replacement motherboard from the original computer manufacturer you've essentially done the same thing as changing to a different computer altogether. That is enough to invalidate the OEM Windows license.

Copy Account Settings and Files from one MS Account to another MS Acount (Same PC)

I upgraded from W7 to W10.  In doing so all my personal files and settings were saved to an older Microsoft account used by other family members.  I have just created a new primary Microsoft account for myself.  I now want to transfer all the files and settings from that older account to the new account. 

Both accounts are on the same PC.

I know there are ways to do this in previous versions of windows, and would like a how-to for windows 10.

Please let me know if I need to clarify anything.

raju s das say:

Hello,

Thank you for posting your query in Microsoft Community.

Follow the steps to copy the files from the old Microsoft account to the new Microsoft account:
a) Press/tap Windows and x together and select Windows Explorer.
b) Click/tap on View tab and click/tap on Options at the top right corner.
c) Click/tap the View tab, and then click/tap Show hidden files, folders, and drives.
d) Uncheck the box which states Hide protected operating system files.
e) Click Yes to confirm, and then click OK.
f) Locate the C:\Users\Old_Username folder, where C is the drive that Windows is installed on, and Old Username is the name of the profile you want to copy files from.
g) Select all of the files and folders in this folder, except the following files:
•Ntuser.dat
•Ntuser.dat.log
•Ntuser.ini
h) Right click/tap and Select Copy.
 i) Locate the C:\Users\New_Username folder, where C is the drive that Windows is installed on, and New Username is the name of the new user profile you created.
j) Right click/double tap inside the folder and select Paste.
k) Restart your PC and check if you are able to log in to the computer using the new Microsoft account with all the files of the old account.

I hope the above information helps. Let us know if you need further assistance.
Thank you.

Are you using Microsoft Account credentials to login into the system?

The NTUSER.DAT file is a registry file. Each user’s NTUSER.DAT file contains the registry settings for their individual account. The Windows registry is a central hierarchical database that contains information about the software, hardware and user profiles contained on a computer. Windows is constantly referencing its registry files throughout the course of its operation. The configuration of the HKEY_CURRENT_USER branch of the registry is supported by the current user’s NTUSER.DAT file.

The user's NTUSER.DAT file picks up EVERY little settings that comes along, whether it actually does anything useful or not.

You are to copy all the files and folders located in the old user profile except for the following files or folder,

  • Ntuser.dat

  • Ntuser.dat.log

  • Ntuser.ini

If the issue persist, run Microsoft Account troubleshooter from the link below, and check if the issue is resolved.
