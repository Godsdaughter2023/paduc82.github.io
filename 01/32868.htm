For instance, for file extensions that work (open their associated program by double clicking), the default context menu item "Open" is normally bold. For me, it isn't any more.

Also, I normally don't use Windows Media Player. Therefore, I usually don't bother with going through the initial set up wizard that wants to enable automatic fetching of DRM keys and stuff like that. In trying to troubleshoot this problem, I did go through that wizard, and now all files WMP wants to associate are permanently associated to it, and double clicking for instance a WAV file directly opens WMP without going through the "Open with" dialogue. However, when I go to the "Default programs" control panel applet, I find myself unable to uncheck any of WMP's associations, or associating another application with them.

I can go to the association listing for foobar2000 (my audio player of choice), and "Select all" file associations, then "Save". Lo and behold, when I go back to the listing after saving, it is empty again (This program has 0 out of 124 defaults).

Now as mucking with the registry seems to be the way to go about these things, as evidenced by most advice on the internet as well as on this very site, I decided to test a theory. In HKEY_CLASSES_ROOT, for all the file types WMP has been stealing, the (Default) value has been WMP11Assoc.Something. I decided to change these values to their foobar2000 type equivalents (like foobar2000.MP3), and hello! Suddenly, foobar2000 has "29 out of 124 defaults". Surely that has to count for something.

Nope. Opening one of the 29 file types where I've applied that change, simply opens the "Open with" dialogue again. At least it doesn't open WMP anymore, thank Bob for that. But still, we're back to square one. The initial problem.

Now in an interesting case study, I've invented a new file type on the spot, that wasn't used for anything else. In this case, I used ".lam". I made an empty file with that extension, and configured it to open in Notepad. That works. But where it gets interesting, is when I try to then change the association. I opened the context menu, and selected "Open with", and (arbitrarily) decided to open it with 7-zip, checking "Always use the selected program to open this kind of file". But the next time I opened it, it still opened in Notepad! Getting closer to the smoking gun, I searched for the novel file extension using Regedit. The three results that came up were in the following keys:
HKEY_CURRENT_USER\Software\Classes\.lam
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\FileExts\.lam
HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs\.lam

The first of which links to lam_auto_file. I deleted all three keys, and the lam_auto_file key. After refreshing explorer, I could select a different association, and it stuck! So that's a solution for file types that don't normally associate with anything else.

However, when I tried that same technique with .mp3 (noting that the association for that is system wide, and therefore in HKEY_CLASSES_ROOT instead of HKEY_CURRENT_USER\Software\Classes), deleting all related keys and associations, when I try to associate it with something, it still won't stick. Subsequent attempts to open the file still pop up the "Open with" dialogue.

Normally I can find a solution to pretty much any problem, either through extensive Googling, or by throwing some logic mixed in with trial and error at it. This one has me stumped. Does anyone have any ideas, aside from the usual "Have you tried turning it on and off again", "have you tried super registry cleaner ultimate Xtreme ($500)", "delete system32", "reinstall", "you have a virus", etc. ?
