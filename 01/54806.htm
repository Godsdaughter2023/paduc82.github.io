Since you were able to run Safe Mode using the Advanced startup options, you have already performed a clean boot on your PC.

If your issue still exists after running the Security File Checker tool, you can try to perform a System Restore while on Safe Mode.

Follow the steps below on how to do a System Restore on Safe Mode:

  1. Turn on the computer, then press and hold the F8 key before the Windows logo appear.
  2. On the Advanced startup options screen, select Safe Mode.
  3. Upon booting on Safe Mode, Click Start and type "Restore" in the Search program and files dialog box.
  4. Click System Restore from the list of results.
  5. In the System Restore windows, click Next.
  6. Select the date and time (must be the time before you encounter the issue) you wish to restore from.

a. user say:

Hi,

This error may occur if the user profile folder was deleted manually. A profile folder that is manually deleted does not remove the Security Identifier (SID) from the profile list in the registry. Refer to these and check if these help you in fixing the issue.

Important Follow the steps in this section carefully. Serious problems might occur if you modify the registry incorrectly. Before you modify it, http://support2.microsoft.com/kb/322756 back up the registry for restoration (http://support.microsoft.com/kb/322756/ ) in case problems occur.

1. Press the Windows + R keys to open the Run dialog, type regedit, and click/tap on OK.

2. If prompted by UAC, then click/tap on Yes.

3. In Registry Editor, navigate to the location below:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList

4. In the left pane under ProfileList, click on a SID key (S-1-5-21....long number)

5. In the right pane of the SID key, look at the ProfileImagePath value to see if it is for the same user account name.

6. If you improperly renamed the user profile folder (C:\Users\XXX) of the affected user account by right clicking on it and click/tap on Rename, then manually rename the user profile folder in File Explorer back to what it shows as the correct name in the ProfileImagePath value in the registry

7. If you have either one or two SID keys with the same S-1-5-21....long number with one without .bak (ex: ...-1014), and one with .bak (ex: ...-1014.bak) at the end.

NOTE: The SID key without .bak is the current bad user profile, and the SID key with .bak is the hopefully good backup of the user profile.

8. If you have Only One S-1-5 (SID) key with .bak at the end.

9. In the left pane, right click on the SID key (ex: ...-1014.bak) with .bak at the end of the numbers, and click/tap Rename.

10. Remove only .bak at the end of the numbers, press Enter.

11. If you have Two S-1-5 (SID) keys with the Same Long Number.

12. In the left pane, right click on the top SID key (ex: ...-1014) without .bak at the end of the numbers, click/tap Rename, add .bk to the end of the numbers, and press Enter. (

13. In the left pane, right click on the bottom SID key (ex: ...-1014.bak) with .bak at the end of the numbers, click/tap Rename, remove only .bak at the end of the numbers, and press Enter.

14. Now go back and Rename the first SID key with .bk to .bak now at the end of the numbers, press.

15. In the right pane of the SID key (ex: ...-1014) without .bak now, double click/tap on the RefCount DWORD to modify it.

16. If you do not have RefCount, then right click or press and hold on an empty space in the right pane, click/tap on New and DWORD (32 bit) Value, type RefCount, and press Enter.

17. The value for this entry will reset and return back to the original value after you have restarted the computer and logged on to the account.

18. Type 0 (number), and click/tap on OK.

19. In the right pane of the SID key (ex: ...-1014) without .bak now, double click/tap on the State DWORD to modify it.

20. If you do not have State, then right click or press and hold on an empty space in the right pane, click/tap on New and DWORD (32 bit) Value, type State, and press Enter.

21. The value for this entry will reset and return back to the original value after you have restarted the computer and logged on to the account.

22. Type 0 (number), and click/tap on OK.

23. When finished, close Registry Editor, restart the computer and check the issue status.
