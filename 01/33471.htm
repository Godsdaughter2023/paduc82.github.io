¡firedog say:
Two points:
(A)  I'm fairly certain that the error message is not coming from the server but from your anti-virus software. Avast likes to manage security itself and although you have specified port 110, Avast will try to use port 995 - the standard one for POP3/SSL. Mail scanning is not necessary, so I should turn off Avast's Mail Shield if that's what you're using.

(B)  There has been an error in the data used by Windows Live Mail to configure BT accounts.  This should fix it:
  1. Right-click on the account name in the folder pane and select Properties.
  2. On the Servers tab, tick My server requires authentication.
  3. On the Advanced tab, untick This server requires a secure connection (SSL).
  4. Close WLMail and wait five minutes.
  5. Restart WLMail and try sending again.
Did you install WLMail, or was it pre-installed when you bought the computer? I ask because this particular error was fixed years ago, but some computer vendors were still pre-installing old versions of the program.

The Custom 1 and Custom 2 fields are only saved in the library database, and don't get written to the actual music files. This explains why you don't see their contents on other computers.
