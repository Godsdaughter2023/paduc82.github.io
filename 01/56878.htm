fitzcarraldo1 say:

I suspect that they are two distinct problems. The problem reported by NCC-1701-A seems to me to be a protocol problem as he stated, or possibly some form of deadly embrace (a.k.a. deadlock). On the other hand the problem reported in the thread:

https://answers.microsoft.com/en-us/skype/forum/skype_linux-skype_startms/online-status-automatically-on-online-not/fe34cf0f-b571-4821-8328-9d710f09496b

seems to me to be because Skype for Linux Beta 5.3.0.1 does not save (locally or remotely) the Skype availability status when you sign out of Skype. In the old Skype for Linux 4.3.0.37, the availability status was stored in the file ~/.Skype/<your Skype account name>/main.db and you could even change it using an SQLite database application for Linux such as sqlitebrowser (SQLite Database Browser). There was a Skype database table with a field called set_availability containing a number corresponding to the user's status:

6     Invisible
5     Do not disturb
3     Away
2     Online
1     Offline

However, the Electron-based Skype for Linux Beta 5.3.0.1 no longer uses ~/.Skype/<your Skype account name>/main.db and so the old way of altering/fixing one's status before signing in to Skype no longer works. Incidentally, if you look in the file main.db of the earlier incarnation of Skype for Linux, it contained all the user's personal information in non-encrypted form. That was also the case in the Windows version of Skype, although I expect Microsoft has changed that too by now.

Skype for Linux Beta 5.3.0.1 and the earlier Alpha releases are essentially Skype for Web (try Skype for Web and you'll see it looks and works the same as Skype for Linux Beta 5.3.0.1). This is because Skype for Linux Beta 5.3.0.1 has been produced using Electron, an open-source software framework that enables desktop GUI applications to be developed from the components of Web applications.
