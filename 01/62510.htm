Text escape bug with Microsoft Fax Server TSID/CSID - Microsoft Community

a. user say:

Text escape bug with Microsoft Fax Server TSID/CSID

I found a bug where an ampersand is escaped in one place but not in another place, so it is displayed inconsistently.

To reproduce:

1) In the Microsoft Fax Service Manager, go to Devices and Providers > Devices > [your device] > Properties > General tab

2) On that tab, find "Transmission Settings > Transmitting Subscriber ID (TSID)"

3) Type a value with an ampersand, like "testA & testB"

4) On that tab, find "Transmission Settings > Called Subscriber ID (CSID)"

5) Type a value with an ampersand, like "testC & testD"

6) Make sure that the header is enabled and also email receipts is enabled with attachment enabled
7) Send a fax and receive a fax
8) View the two emails you received.  The email attachment that says the fax was sent shows the TSID as "testA _testB" (instead of the correct "testA & testB"). The email that says a fax was received shows the CSID as "testC & testD".

To work around this problem so that both strings show up correctly, I have to type "testA && testB" as the TSID, and "testC & testD" as the CSID.

I am using Fax Service Manager 6.2.9200 on Windows Server 2012.
