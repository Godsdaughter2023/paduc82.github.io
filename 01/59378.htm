Svchost.exe is a generic host process name for services that run from dynamic-link libraries. Services need to be executed as a .exe file which is helped by this process. Those services are organized into logical groups, and then a single svchost.exe instance is created for each group. For instance, one svchost.exe instance runs the 3 services related to the firewall. togo_ say:

Could your WLAN network be using FreeRadius2 to handle WLAN authentication? There is a known issue in FreeRadius 2 version 2.2.6+ preventing it from authenticating a client using TLS 1.2.

A possible solution is to try disabling use of TLS 1.2 of the Windows 10 client:

  • Create DWORD HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\RasMan\PPP\EAP\13\TlsVersion and set the associate DWORD value to C0.
  • Restart service EapHost service.

Simplified procedure:

  1. Download the registry script Disable_TLS_1.2.reg (click to download)
  2. Double-click on the downloaded file
  3. Ignore the security warning and click on "Run", then "Yes"
  4. Reboot

To re-enable TLS 1.2 support, use Re-Enable_TLS_1.2.reg, or delete the DWORD value "TlsVersion".
