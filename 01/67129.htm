lemp say:

If all of the wav files are in one folder and all of the mp3 files are in another folder (i.e., no subfolders), and assuming that the wav and mp3 files have the same names but for the file extension, this is a crude but effective method:

Open a Command Prompt window and type the following commands, press Enter after each

dir /b path to wav folder > wavnames.txt

dir /b path to mp3 folder > mp3names.txt


This will create two text files located in whatever directory you're in when you open the Command Prompt, unless you specify a different directory for the output.  Once you have the two files ...

  1. Open Microsoft Excel and drag and drop the first text file into Excel
  2. Drag and drop the second text file into Excel
  3. Select all of the contents of the second Excel file and change the font color to something noticeable, like red
  4. With all of the contents of the second Excel file still selected, cut and paste into the cell at the bottom of the list in the first Excel file
  5. Select all in the first Excel file (which now includes both lists, with one in a different color font), and sort the files
  6. You should now be able to manually detect which wav files don't have corresponding mp3 files
