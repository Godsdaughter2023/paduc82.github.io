As PJ says, it's from a VERY high view.

It's like Obi-Wan Kenobi telling Luke Skywalker, the truth, from a perspective.

Windows Movie Maker is compatible with those formats with the caveat that you have pre-existing software or codecs installed that support those formats. I think it's a little bit disingenuous by Microsoft to not indicate that when people are looking to work with Windows Movie Maker. I think one of the largest differences between Windows Movie Maker and other video editing packages is that when those other video editing applications say they support those formats, when the application is installed, those codecs are installed to support those formats; Windows Movie Maker makes no attempt in this area. 

Windows Movie Maker is a free application for Microsoft and I believe the reason that Microsoft does not include the required codecs for the reporting formats that supports, I believe, is an issue of licensing and additional fees that Microsoft would have to pay in order to offer Windows Movie Maker fully functional. I think most users would appreciate a little bit of full disclosure.

Being able to import media directly in to Windows Movie Maker is the most efficient process [using the proper codecs], but if you are running in to issues with being able to work with various types of media, you might want to think about converting the media to something ubiquitous to Windows Movie Maker.

The native formats of Windows Movie Maker are those of Microsoft, WMV and WMA/WAV. These formats should be the best bet for working with Windows Movie Maker. In my experience, converting video files to WMV, or converting audio files to WMA or WAV has never failed me.

 

One of the biggest problems with Windows Movie Maker is the fragility of the application. The application is susceptible to system updates and changes made by other applications. As I mentioned before, Windows Movie Maker is highly dependent upon other applications and codecs to be able to function correctly. Once those change, or if they become corrupted, the whole premise of the application working properly changes completely.

You cannot change a user account by simply renaming it. As you have found out, this makes a cosmetic change only. Create a new user account named the way you want. Copy Files to the New User Profile

   1. Log on as a user other than the user whose profile you are copying files to or from.
   2. In Windows Explorer, click Tools, click Folder Options, click the View tab, click Show hidden files and folders, click to clear the Hide protected operating system files check box, and then click OK.
   3. Locate the C:\Documents and Settings\Old_Username folder, where C is the drive on which Windows XP is installed, and Old_Username is the name of the profile you want to copy user data from.
   4. Press and hold down the CTRL key while you click each file and subfolder in this folder, except the following files:
          * Ntuser.dat
          * Ntuser.dat.log
          * Ntuser.ini

   6. Locate the C:\Documents and Settings\New_Username folder, where C is the drive on which Windows XP is installed, and New_Username is the name of the user
profile that you created in the "Create a New User Profile" section.
   7. On the Edit menu, click Paste.
   8. Log off the computer, and then log on as the new user.

If you are trying defragment the system reserved partition, then do not worry about it. It is not an issue that you can't defragment it. This will not degrade your system performance. It is better to leave it as it is to avoid any further complications.

The system partition is not used to run the system. It only has the files while are used for booting the computer and also for the recovery environment. They are not active while the system is running,  so the fragmentation of this partition has no influence on system performance. Do not try to defragment this partition.
