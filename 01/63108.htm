giorgosvam say:

Good news: the above fix by Microsoft seems to have solved the problem (at least in my case).

Before installing it, I deactivated the High Definition Audio Controller (listed under system devices) and tested whether drain occurs while in hibernation. Drain did not occur which indicated that the device is indeed responsible for the drain. Hence I re-activated the device, installed the fix and tested again. Although this time the hibernation period was quite short (2 hours), the update seems to have fixed the problem.

George

Edit: The Fix can be found at:

http://support.microsoft.com/kb/970030/en-us

giorgosvam say: Hello to all,

battery drain may be caused by a number of reasons. Most of them are meant to be like that (Wake-on-lan capability, scheduled wake up tasks, hybrid sleep, etc.). Others are "bugs", in that they are not meant to use power when hibernating. When trying to eliminate battery drain, ALL of the possible factors have to be adressed, otherwise drain may still be the case.

If there is no drain when hibernating on battery or hibernating and briefly removing (and re-inserting) the battery afterwards, the High Definition Audio Controller is most likely the culprit. Applying the fix mentioned above should solve the problem.

If there is drain, then you must look at all other possible causes first. The High Definition Audio Controller may still be drawing power when hibernating while jacked-in and not removing the battery afterwards, but you will only find out once all other causes have been eliminated (and at least one will be the case). In the rarest of cases, will a faulty battery be the reason.

Please refer to the following thread and those linked from it in order to check most other causes of power usage during hibernation:
http://forum.thinkpads.com/viewtopic.php?t=52879

George

Hello,

This normally happens when you mistyped the name, Skype Name, email or phone number of the contact you're searching on Skype.

If you have confirmed that you're typing the correct information, please consider the following steps to troubleshoot the application:

  1. Make sure that Skype is no longer running.
  2. Press the Windows  and R keys on your keyboard at the same time.
  3. Type %appdata%\skype into the Run window and click OK.
  4. Find and delete the shared.xml file. If file extensions are not displayed on your computer, the file will just be called shared.
  5. Open the folder that has your Skype name.
  6. Locate and delete the configor config.xml file.
  7. Press the Windows  and R keys on your keyboard at the same time again.
  8. Type %appdata% into the Run window and click OK.
  9. Locate and rename the Skype folder to Skype_Old.
  10. Restart Skype.

I'm not greatly optimistic, but try deleting it via a command prompt:
1. Open an elevated Command Prompt by clicking the Windows Orb (Start) > All Programs > Accessories and right-click Command Prompt, then ‘Run as Administrator’. From the prompt type CD %userprofile%\Desktop and press Enter, now type del name (where name is the file to be removed, e.g. abc.def) and press Enter. This should remove the file. If it does not, then keep the Command Prompt open and
2. Hit Ctrl+Shift+Esc to open Task Manager. On the Processes Tab, select the explorer.exe process and end it to halt the desktop. Switch to the Command Prompt window and retry the del command.
3. Now restart explorer by going to the Application Tab of Task Manager > File > New Task. Enter explorer.exe and click OK to restart the desktop.

If you still can’t delete it, try booting to Safe Mode (tap F8 when you switch on) and deleting it.
