swapnil rustagi say:
 I don't know the right syntax. How do you use reg query?

Type reg query /? at the Command Prompt to know how to use reg query and it's syntax along with parameters.

Also, the ROOTKEY is abbreviated when using reg query as follows-

HKEY_CLASSES_ROOT is abbreviated to HKCR

HKEY_CURRENT_USER is abbreviated to HKCU

HKEY_LOCAL_MACHINE is abbreviated to HKLM

HKEY_USERS is abbreviated to HKU

HKEY_CURRENT_CONFIG is abbreviated to HKCC


So when using reg query for HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run the command would be-

reg query HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run

followed by any parameters.

The following information was gleaned from a Windows 8.1 Enterprise installation.

Location of Family Safety log:

Event Viewer
-> Applications and Services Logs
--> Microsoft
---> Windows
----> ParentalControls
-----> Operational

To export log entries to a file that can be more easily browsed/searched, right-click "Operational" and choose "Save All Events As..."  There will be 4 different options for file type.  Listed below are the results of saving the 1,125 events as each of the different file types:

*  Event Files (*.evtx) - 1,092 KB; opens in Event Viewer; binary file; contains all details
*  Xml (Xml File) (*.xml) - 1,270 KB; opens in IE; text file; contains all details
*  Text (Tab delimited) (*.txt) - 142 KB; opens in Notepad; text file; contains summary info
*  CSV (Comma Separated) (*.csv) - 143 KB; opens in Excel; text file; contains summary info

The "Text" and "CSV" file types only contained the following information and were NOT at all helpful:

*  Level
*  Date and Time
*  Source
*  Event ID
*  Task Category

The "Xml" file type would probably be the most useful format to browse/search through.  Unfortunately, it appears the "Xml" file does not contain any line-breaks--all of the content appears in one, single, continuous line--so depending on the size of the file and your chosen method for browsing/searching it (e.g. IE, Notepad, find.exe), things might look "funny"... just something to consider.

For instance, to "force" each discrete event to be on its own line, you might perform the following command from a PowerShell prompt:

PS> Get-Content C:\temp\familysafetylog.xml | ForEach-Object { $_ -replace "</Event>", "</Event>`r`n" } | Set-Content C:\temp\fslog.xml

To open "C:\temp\fslog.xml" in IE and have it display properly, you might need to open it in Notepad and save it with UTF-8 character encoding.

Hope this helps.
