a. user say:

Hello Sheik,

 

Thank you for posting on Microsoft Community. We regret the inconvenience caused and will assist you in resolving the issue.

 

You may be facing the above issue due to the system components conflicting with the network device or changed HomeGroup network settings after the upgrade on Windows 10.

 

Method 1: Running the troubleshooter will help in detecting and identifying the issue and in certain cases will fix to enhance the performance of the system. I suggest you to run the HomeGroup troubleshooter by following the steps below and check if any of the HomeGroup settings or components is missing.

 

  1. Press Windows logo key on the keyboard, type Troubleshooting and select the top most search result.
  2. Click on View all from left side panel of the screen.
  3. Select HomeGroup, click on Advanced option and check Apply repairs automatically option.
  4. Follow the on screen instructions and wait for the troubleshooting to complete.
  5. After the troubleshooting is complete, check if any of the changed or HomeGroup components missing from the network.

 

Method 2: If the issue persists, check/change the HomeGroup settings by following the steps below:

 

  1. Press Windows logo key on the keyboard, type Control Panel and select the top most search result.
  2. Click on View by located to the top right of the window and select Large option.
  3. Click on Network & Sharing center and select the Change advanced sharing settings from the left side panel of the window.
  4. Under Private network and Public network, check the boxes Turn on network discovery, Turn on file and printer sharing and Allow Windows to manage homegroup connections (recommended).
  5. Click on Save changes, close the window and check if the changes are effective.

 

Refer to the links below on HomeGroup from start to finish and check if it helps:

http://windows.microsoft.com/en-US/windows/homegroup-help#homegroup-start-to-finish=windows-81&v1h=win81tab1&v2h=win7tab1

 

Note: Steps given in the above articles for Windows 8.1 applies to Windows 10 also.

 

Write to us with the status of the issue for further assistance. Your reply is important for us to ensure we assist you accordingly.

 

Thank you.

marcel kuri say:
Hello Chuck, my name is Marcel and I am an Independent Advisor.
I am here to help.

Unfortunately it is set by default that the original file creation date to be changed when copying or moving files. I recommend that you use Robocopy to keep your source date/time without any changes with the creation date.
Go to the command prompt and enter the command:
ROBOCOPY <Source> <Destination> [File…] /copyall
<Source> - Provide the location/path from where you are copying the files
<Destination> - Provide the location/path to where you are copying the files
[File…] - If you are copying a particular file, provide the name or if you’re copying multiple files you can use *.* or *.<file extension>
/copyall - Copies all file information including time stamp

Get more information on using the commands. It is also applicable to Windows 10
https://docs.microsoft.com/en-us/windows-server...
