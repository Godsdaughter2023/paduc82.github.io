wunders say: HC,

OK. Here's what's going on.  When the files were created on computer #1, they were likely owned by the user that created them.  Note that by "user", I mean the SID of that user or the S-1-5-21-11[....] number that you quote  -- this is a unique identification for that user on that computer.  When the disk was moved to Computer #2, this user did not exist on that computer and thus the only access to the files is whatever permissions were granted to general users on these files.  Whenever anyone attempts to access these files by connecting via network to Computer #2, they will always authenticate as the "Guest" user (if Simple File Sharing is enabled) or as whatever Computer #2 user they authenticate as (if Simple File Sharing is disabled and not a Domain computer).

So it appears that you would like to give "Everyone" network access to these files on this external disk that is mounted to Computer #2.  Dropping to the bottom line...  I am a fan of command-line control.  The "cacls" command will do what you want in one command.  So assuming that this external disk mounts on computer #2 as Drive E: , perform the following steps:
  1. Log in to computer #2 as an Administrator user
  2. Bring up a command prompt window  (Start -> Run -> "cmd")
  3. Enter the following command in the command prompt window:
        CACLS  E:\  /T  /E  /C  /G  Everyone:F
This will give "Everyone" full permissions (:F) to everything in the E: drive (the /T option will include all subdirectories).  If you would only like to give Everyone Read-only permission, simply replace the  :F  with  :R
For more information on the CACLS command:
     < http://technet.microsoft.com/en-us/library/bb490872.aspx >

Alternatively, you can take ownership of these files and folders on Computer #2 and then assign permissions as you require:
    "How to take ownership of a file or folder in Windows XP"
    < http://support.microsoft.com/kb/308421 >
    "How to set, view, change, or remove special permissions for files and folders in Windows XP"
    < http://support.microsoft.com/kb/308419 >
