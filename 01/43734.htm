Method 1:

Method 2:

===============

1. Put the Windows Windows 7 installation disc in the disc drive, and then start the computer.

3. Select a language, time, currency, and a keyboard or another input method. Then click Next.

4. Click Repair your computer.

6. In the System Recovery Options dialog box, click Command Prompt.

7. Type Bootrec /RebuildBcd, and then press ENTER.

 

Method 3:

===============

1. Put the Windows Windows 7 installation disc into the disc drive, and then start the computer.

2. Press a key when the message indicating "Press any key to boot from CD or DVD …". appears.

9. Type Drive:\boot\Bootsect.exe /NT60 All, and then press ENTER.

volume z say:

The General Windows 7 Update Search issue lasted from Patch Day August 2015 until mid-January 2017. It was about Update Clients responding ill-behaved to a small set of Updates in case they were applicable and available. One triggering Update being part of the list of Updates to be offered was enough to spark the entire syndrome, afflicting all versions of the Windows Update Client released prior to June 2016 including KB3138612 and the Convenience Rollup.

The first Update to trigger the issue was KB3078601, released 8-11-2015. This Update, due to never being superseded in Windows 7, triggered the issue if applicable (not installed) from the beginning to the end.


The second Update to trigger the issue was KB3087039, released Patch Day September 2015. From September 2015 on, the total number of triggering Updates never went below 2.

The third Update to trigger the issue was KB3097877, released Patch Day November 2015, raising the total number of triggering Updates to 3.

The fourth Update to trigger the issue was KB3109094, released Patch Day December 2015, superseding KB3097877.

The fifth Update to trigger the issue was KB3124000, released Patch Day January 2016, raising the total number of triggering Updates to 4.

The sixth Update to trigger the issue was KB3134214, released Patch Day February 2016, superseding KB3124000.

The seventh Update to trigger the issue was KB3139852, released Patch Day March 2016, superseding KB3134214.

The eighth Update to trigger the issue was KB3145739, released Patch Day April 2016, superseding KB3139852.

The ninth Update to trigger the issue was KB3153199, released Patch Day May 2016, raising the total number of triggering Updates to 5.

The tenth Update to trigger the issue was KB3161664, released Patch Day June 2016, superseding KB3153199.

At the end of June 2016 a non-susceptible to triggering 100% CPU consumption of svchost.exe version of the Windows Update Client was released: KB3161647.

KB3161647 was made available only as a part of the optional June 2016 Update Rollup, which in turn had the prerequisite of a Servicing Stack Update (then KB3020369 only). The June 2016 Update Rollup was replaced by the optional July 2017 Update Rollup (KB3172605) a month later, still requiring a Servicing Stack Rollup.

The eleventh Update to trigger the issue was KB3168965, released Patch Day July 2016, superseding KB3161664.

The twelfth Update to trigger the issue was KB3177725, released Patch Day August 2016, raising the total number of triggering Updates to 6.

The thirteenth Update to trigger the issue was KB3185911, released Patch Day September 2016, superseding KB3177725.

The fourteenth Update to trigger the issue was the October 2016 Security Monthly Quality Rollup.

The fifteenth Update to trigger the issue was the November 2016 Security Monthly Quality Rollup.

The sixteenth Update to trigger the issue was the December 2016 Security Monthly Quality Rollup.

From an unknown point in time on applicability of the Excel and Word specific Updates for Compatibility Pack SP3, Office 2007 and Office 2010 and potentially the Cumulative Security Update for IE11 contributed to triggering the issue.

MS Office Updates not counting, the total number of triggering Updates was 2 when the issue was fixed by Microsoft in mid-January 2017. No Updates are known to trigger the issue in Windows 7 at this time.

If the issue were still going on, alternatives to KB3172605 (and its by now two potential prerequisites KB3020369 and KB3147677) would be available. The Security Only Quality Updates of April and June 2017 (KB4015546 and KB4022722) include even newer versions of the Windows Update Client. So does every Security Quality Monthly Rollup at least as recent than April 2017.

  • Files in all users' C:\Users\(user-name) folders. (ex: Desktop, My Documents, My Music, etc...)
  • Wireless network connections
  • Apps from the Windows Store will be kept.
  • Mobile broadband connections
  • BitLocker and BitLocker To Go settings
  • Drive letter assignments
  • Personalization settings such as lock screen background and desktop wallpaper
  • Choices you made during Windows Welcome, such as computer name and user accounts, will not change.
  • Network, Libraries, and Windows Update settings will not change.
  • Customer Experience Improvement Program settings will not change.
  • Windows Error Reporting settings will not change.
  • Some PC settings will be changed back to their defaults.
  • Installed Windows Updates will be removed.
  • Some drivers may or may not be removed.

Cheers.
