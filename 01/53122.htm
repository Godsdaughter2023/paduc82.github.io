CHKDSK problem - Microsoft Community

paul k h say:

CHKDSK problem

When trying to defragment my C drive I get a message saying tht CHKDSK/f is scheduled,when I try to run it I get a message saying that I will have to schedule CHKDSK for the next time I turn on the laptop,however this still does not occur although it did with the D drive

a. user say:

When trying to defragment a volume and Disk Defragmenter reports:

Disk Defragmenter has detected that Chkdsk is scheduled to run on this volume: (C:)
Please run Chkdsk /f.

This means that the volume you are trying to defragment has been marked "dirty" by XP.  It doesn't necessarily mean any data has been lost on the volume, it just means that XP thinks the volume needs to be checked for errors because of something that happened before. 

You will not be able to defragment the volume until the dirty bit has been cleared.

It does not mean that a check disk is scheduled to run and you cannot unschedule what needs to happen in this case because that is not the problem. 

As long as the dirty bit is not set on a volume, you can still run a disk defragment with a scheduled check disk pending (try it). 

Therefore, the Microsoft engaged Support Engineer "expert" advice regarding a Scheduled Task has NOTHING to do with your issue and entering any commands to remove a Scheduled Task or any commands to reset or clear a pending check disk will not help you.  These ideas are just more of their wild goose chases and a waste of time - that is not the problem.

A volume can be marked dirty if the system suffered a power interruption (power failure, plug pulling, battery removal, power button), aborted restart or ungraceful shutdown .  Have any of those events occurred?

When XP starts, it automatically checks the dirty bit on every volume.  You can also tell XP to ignore the dirty bit on any volume, but that is probably not a good idea in case there is a real problem that needs to be fixed.  If you have a hard drive where the dirty bit will just not clear, you can tell XP to not check it. 

If the XP startup finds the volume is dirty, it will automatically run check disk with error correction (chkdsk /r) and try to correct any problems it finds.

If the chkdsk succeeds, it will clear the dirty bit and then you can defragment the volume. 

You can check the status of your dirty bit from a Command Prompt window.

To do that, click Start, Run and in the box enter:

cmd

Click OK to open a Command Prompt window.

For example, to query the dirty bit on drive C, type:

fsutil  dirty  query  C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Is your volume dirty or is it not dirty?

You cannot use fsutil or chkntfs from a Command Prompt to clear the dirty bit on a volume.

The only thing that can clear the dirty bit on a volume is chkdsk /r (or chkdsk /p).  You could tell XP not to check the dirty bit on a volume where the dirty bit just will not clear, but that is probably not a good idea since there is some kind of problem on the volume.

You will also not fix this problem by editing your registry (pay attention Support Engineer and Forum Moderator "experts") - that is not where the problem is.

Once a volume has been marked dirty, the only way to clear the dirty bit is to run "chkdsk /r" or "chkdsk /p" when the volume is not in use.  The only way a volume can be not in use is while XP is just starting to load or from the XP Recovery Console.

If XP will not run chkdsk properly when your system reboots or the dirty bit will not clear as it should, you need to look in the Event Viewer for the chkdsk log and see what is going on and fix it.

If XP still won't clear the dirty bit automatically, you should run chkdsk /r from the XP Recovery Console so you can watch it and see what does and run it again until it does work.  If your hard drive is failing, you may never be able to clear the dirty bit, in which case you can also tell XP to never check the dirty bit on the afflicted volume.

Telling XP not to check the dirty bit on a volume would be risky behavior since there would still be some kind of problem with your hard disk.
