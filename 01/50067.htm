hairyfool say:

No it's not contraindicated, just not really relevant to the problem.

All setting up a HomeGroup does effectively is create a user that has the same credentials on each member and sets up permissions for specific folders on each computer for that user. However HomeGroup requires the IPv6 protocol and one of the diagnostic steps may be to disable IPv6, in this case you would need to remove HomeGroup 1st

Presumably the software has created a MySQL or MSDE SQL database on the host PC and created connectors to that database on each PC. The settings must be correct or else it would not work at all.

You could prove network connectivity to make sure the PCs are talking to each. Open Command prompt  on one of the client PCs type in the command ping -t ip-address-of-the-server-PC. This sends a continuous pulse to the server from the client. Watch it for a few minutes then minimise the window and run your program. If it stops switch back to the command windows and check if the ping is running. If not then the problem is a networking one rather than the database connection.

As this is primarily an SQL issue the vendor should be supporting you.

    WHQL stands for Windows Hardware Quality Labs. Hardware Manufacturers that whose hardware passes this test gets to display the 'certified for windows' logo. No it is not BETA  WHQL has nothing to do with whether software is in beta or not.

    Normally new driver updates for NVidea and ATI are made available through Windows Updates.
