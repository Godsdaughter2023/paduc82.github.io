Hi,

Thank you for the information. We can use the default product key to activate Windows, use a command to force the activation, or use SLUI 4 command and activate through a phone. Please follow the steps below:

Activate using Default product key

  1. Press Windows key + R to open Run dialog box.
  2. Type in SLUI 3 (please don't forget to put space between SLUI and 3) then hit Enter.
  3. It will prompt you to activation page where you can put the product key.
  4. Copy and paste this default key for Windows 10 Home YTMG3-N6DKC-DKB77-7M9GH-8HVX7.
  5. Click Activate.

If the same issue persists, use command line through Command Prompt.

Using slmgr command

  1. Type Command Prompt on Search bar, on the result, right-click on Command Prompt and choose Run as administrator.(Click Yes if the UAC popup.)
  2. On Command Prompt, type command slmgr /ato to activate Windows if the system already has the product key installed. Hit Enter and see if it resolve the issue. If not, we need to remove the install product key and reinstall it.
  3. Type command slmgr /upk and hit Enter to uninstall product key.
  4. Then, type command slmgr /cpky and hit Enter to totally uninstall key from the registry.
  5. After that, type command slmgr /ipk followed by your original product key (e.g. slmgr /ipk XXXXX XXXXX XXXXX XXXXX XXXXX) and hit Enter to install product key. (Please be mindful of the spacing when running command)
  6. Finally, type command slmgr /ato to activate Windows.

Use Phone activation

  1. Press Windows key + R to open run dialog box.
  2. Type in SLUI 4 and hit Enter. (Please make sure there is a space between SLUI and 4.)
  3. Select your language and hit Next.
  4. Call the toll free number to submit your installation ID.
  5. After that, you will get a confirmation ID. 
  6. Click on Enter confirmation ID and type it in.
  7. Finally, click on Activate.

You can also see the steps provided on this link on how to reactivate Windows after a hardware change.

All the best.

I have tried so far: (1) re-install the driver 13.1 (with and without compatibility mode), (2) use the later 13.9 version for Windows Vista (with and without compatibility mode), (3) get a newer driver through Windows Update, and (4) use the Microsoft provided driver in KB4057291. Somewhere I found that I should try installing the driver without internet connection. That also failed. Any suggestions that would prevent me from making a clean install?

Thanks

andremun say:

While I appreciate the responses, getting a new card or downgrade the software were not the "solutions" that I was looking for. After all, there is a 2015 Windows 10 driver provided by Microsoft that should solve the issue. Nevertheless, after some fiddling and more web research, I came to this procedure which solved the problem:

  1. I updated Windows to the latest version 1903 (this by itself was not the solution, but it gave me an important clue later on. Besides, if I had no way to get to an early version, why not?).
  2. Download from AMD the Legacy 13.4 beta driver and the .net framework version 3.5
  3. I uninstalled all AMD/ATI software and removed all the drivers for the adapter through the "Device Manager".
  4. At this point, Windows attempted to re-install the driver automatically, but was unable to do so. A notification appeared saying that a dll component from the driver had triggered the "Memory Integrity" module. I disabled this module from "Windows Security".
  5. Disconnect internet, removed the driver one more time and restarted Windows.
  6. Once back, I installed the .net framework v3.5.
  7. I set the compatibility mode for the Legacy 13.4 driver installer to Windows 8.1, and run it as administrator. I installed only the driver and not the Catalyst Control Centre. This got the driver working. No

    Unfortunately, I don't remember the pages that provided most of these hints. However, I would not have discovered the "Memory Integrity" issue without getting the newer Windows 10 version.

    I hope this may come handy to somebody else with the same issue.
