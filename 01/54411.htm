Hi Everyone

 

After working with support for a considerable time we have determined the following:

 

It seems that if the first network adapter driver fails to properly install (in this case the onboard Ethernet adapter that we knew to be incompatible), Windows 8 does not start the WLAN Service or it starts it and it hangs (gets stuck in a wait state) and then even though the subsequent wireless network adapters install fine and look good in Device Manager, they are not available to the system. 
Even if the failed network adapter is disabled (via the Devcon command or via the GUI) it still causes the WLAN Service failure and subsequent failures to occur.  The user must get the failed network adapter out of the Network Adapters category in Device Manager by requesting that the driver be updated and when the update fails, the system then puts the device in the Unknown category.  If the user then disables the Unknown device, all is well from then on.
The sequence is:
1. Start the system
2. When complete there is no network connectivity and the incompatible adapter is in a Not Working state in Device Manager.
3. In Device Manager request the driver be updated for the incompatible device.  The update fails and the device is put in the Unknown category and removed from the Network Adapters category in Device Manager.
4. In Device Manager disable the Unknown device.
5. Restart the system and the bad network adapter remains a disabled, unknown device and the first network adapter encountered now becomes the wireless adapter which installs and initializes just fine.

Everything works fine across restarts and shutdown / startup cycles.

The System Update Readiness Tool creates a log file that captures any issues that the tool found or fixed. The log file is located at the following location:
•%SYSTEMROOT%\Logs\CBS\CheckSUR.log
•%SYSTEMROOT%\Logs\CBS\CheckSUR.persist.log

The system root is normally C. If the files are not there, checkC:\windows\ logs\CBS.

You may need to change your settings in Windows to be able to see the file. You may need to scroll down to see the second item. You should also make certain that the box before "Hide extensions for known file types" is not checked.

Do not use Search to find the files. Open Windows Explorer (keyboard shortcut Win+E), locate and expand the file path (item 1 in the image below) Computer>Local Disk (C:)>Windows>Logs>CBS (item 2 below).
