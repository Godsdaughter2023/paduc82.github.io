michael..b say: Further investigation:  I had wondered if it was because all the cells of the originating tables are set to "text" (because of Excel's predilection for enthusiastically interpreting cell contents as anything but text if it can manage it).  However, even after formatting them as "date" and "number" appropriately, the error still occurs.

a. user say:

Hi, Michael..B,

For assistance with this issue I recommend you repost to the Microsoft Excel Community.
https://techcommunity.microsoft.com/t5/Excel/ct...

michael..b say:

Further further investigation:  Turns out I was on the right track after all.  Excel is overwriting the Data Type set in the Data Model based on the formatting of the original cells.  What threw me off was the fact that changing the format of a cell is taken by Excel not as an instruction, but more as a guideline for possible future consideration, should you ever decide to re-enter all the values into the thousands of cells you just "reformatted".

After re-entering the values into the cells, so that Excel would deign to take into consideration my formatting "suggestions", the data types in the Data Model appear to have stabilized, and are no longer reverting to "Text".

I still think it is a bug, though - a Data Type set in the Data Model should override any formatting set in the originating table - either that, or you should not be allowed to think you can override it when actually you cannot.
