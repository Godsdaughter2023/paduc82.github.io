ken blake say:

First, note that you *do* have Paint. It's part of Windows.

Second, note that PrtScn takes a screenshot of the entire screen, and Alt-PrtScn takes a screenshot of just the current window.

Third, note that if you take a screenshot, there's no need to put it into the clipboard; it's already there. That's what PrtScn or Alt-PrtScn does; it puts the screenshot into the clipboard.

enamul say:

Hi Zodiac220,

Please make sure you have a full backup of the HDD in question.

There are many reasons why a good HDD with NTFS partition should prompt for initialization. Some of them are bad partition table, initialized in too many controllers, error in RAID config, etc.)

Disk initialize is low-level formatting. It will recreate the HDD's partition table and new partition. Formatting is re/creating file system

But no matter what the reason is,you will lose data if initialized into the new machine.

The best way to deal with your situation, take the HDD to the original controller and backup all data in it. Then initialize into the new machine and move the data back.

You if have already initialized the HDD and lost data, before you do anything, you should try recovering data using tools like :http://www.piriform.com/recuva

I think you're seeing some really old XP boot code coming back to haunt the new install.  This is more of what we've seen since release with Upgrade version being corrupted by previous OS's, some of which date back so far the disk is effectively a junkyard.  NTLDR is a file only used to boot XP for example.  There may be an old XP System partition on the disk that has had the Active flag switched to it and is trying to boot, or the code might have merely survived formatting since it's only wiped by Diskpart Clean Command.

The system memory that is reported in the System Information dialog box in Windows Vista is less than you expect if 4 GB of RAM is installed:
