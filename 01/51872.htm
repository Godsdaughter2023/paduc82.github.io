a. user say:

Okay.  Here is the deal...

If you look here:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon

I think you will find an entry for GinaDLL that is set to OSDGina.dll that was put there by the software you installed and that is what XP is trying to use, and that file is probably not on your system or it is broken (you can look in c:\windows\system32 and see).

In a default configuration, there is no GinaDLL entry in the Winlogon registry hive and XP will load the default MSGina.dll file, so if you see one there, delete it - or change the registry value to MSGina.dll so XP will load the Microsoft GINA and I think you will then be able to boot.

If the GinaDLL value is NOT in the Winlogon section (there is not on my system), then XP will default and use the MSGina.dll.

To get things back to "normal", you should be able to just delete the GinaDLL entry from the registry in:

HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon

If you can't get it to work for some reason, copy the msgina.dll and call the copy OSDGina.dll then see if you can boot, then remove the entry from the registry in Winlogon...
