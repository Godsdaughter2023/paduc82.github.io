pvandck say: I had something similar to this problem, twice, with this update on my Win 7 64-bit Pro. It hung on 32% system configuration, then it hung again on the update uninstall. I thought I was going to have to re-install the operating system.
Eventually I resolved the situation and everything is now working again perfectly.

Restart your computer and immediately press F8, and keep pressing it until you are presented with the start-up option screen. The top option (above Safe Mode) should allow you to restart in Repair Mode. If you have this option, click on it.
The system then continues to load a language and keyboard language screen, then a login screen. Choose your language and keyboard, and log in to your profile from the drop-down menu.
Then you are presented with various options including Restore and Memory Test. These are the options you want (not Start-up Repair)
Select the earliest reasonable restore point and follow the on screen instructions. It may well fail, but it shouldn't. When it's done that restart the system, again pressing F8, to get back to the Repair Mode. It will tell you whether the System restore has worked and you can check anyway by going to the System Restore (in Repair Mode) option and repeating the same steps. If it worked then there will be no later restore points after the date you selected. If the restore failed, do it again with an earlier restore point if there is one. Then, instead of restarting the system, select the Memory Test option and run that. The system should restart itself automatically.
In my case, everything has worked perfectly since.
In fact when I tried again to install this particular update (Security Update for Windows 7 for x64-based Systems (KB2862330)), by itself, exactly the same thing happened. This time I went straight to the System Repair mode and the System Restore option worked first time. Everything is working as it should.

The problems I believe were simply as a result of driver incompatibility. Certainly there were no viruses or malware involved, and no conspiracy on the part of MS to mess up my computer. In fact this is the first time my Win 7 PC has ever had an update problem, or any problem at all. The system has been solid as a rock for the last few years, and it is used for several hours every day. I expect it will continue that way.

I should say that anti-virus and malware scanners aren't helpful during Windows updates. And when something goes wrong because of an update (which in the scheme of things is very rare, considering the number of Windows machines in use), false malware/virus detections and consequent file deletions can be catastrophic. If there is one lesson everyone should learn it is - back up your data, and frequently. If you have to reinstall the OS, you'll be glad you did.
