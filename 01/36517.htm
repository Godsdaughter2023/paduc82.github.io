parker_9909 say:

I have a file insystem32 called cmimageworker.exe. It looks like a virus but I cannot find anything about it anywhere except in Japanese.

When it runs it takes around 70% of available resources. Deleting it is very difficult. Has anyone seen this?

I had the exact same problem on my Surface Pro 6 running build 1809 of Windows 10 Pro.

It took a while to isolate this. However, after a clean install of Windows on a new Surface Pro 6 (courtesy of Microsoft), I believe that I have narrowed this problem down to being an issue with an executable/process related the newer Windows Defender Application Guard (WDAG).

I am not sure that this is the case for previous builds but in 1809 whenever WDAG is enabled, cmimageworker.exe shows up under: C:\Windows\System32\cmimageworker.exe, and when WDAG is disabled, the process is no longer running or listed in the \System32\ files. 

The only other place that cmimageworker exists when WDAG is disabled is under "C:\Windows\WinSxS\amd64_microsoft-windows-container-manager_31bf3856ad364e35_10.0.17763.1_none_83a5e7eb35d0cc3b\cmimageworker.exe"

My best guess is that this process is used to maintain the container that WDAG utilizes as Windows 10 makes use of a container and a separate kernel for virtualized browsing in Edge. This is likely the case given the name and location of the folder and application.

The WinSxS folder mentioned above is used to maintain/store the components needed for updating windows and turning features on and off. So, it makes sense that cmimageworker.exe is located in \WinSxS\ when WDAG is turned off and located in \System 32\ when it is enabled. Additionally, in the WinSxS folder, cmimageworker.exe is located along with two .dll's: CmService.dll and cmclient.dll. Both of these are supposed to be there and are more clearly documented by Microsoft.

While all this does not necessarily mean that cmimageworker is an offical windows program or feature, it certainly looks more like a poorly documented windows program/process for a minimally utilized windows feature than a virus. Thus, I feel confident in the theory that this process is not a virus but indeed a bug related to WDAG and potentially other services that need "container management" in Windows 10. **I would wait for Microsoft to confirm that this is the case before you take my word for it.**

Regardless, the best course of action that resolved this bug for me was to turn WDAG off. 

You can turn this feature off by going to Start > Turn windows features on or off > (Uncheck the option for Windows Defender Application Guard) > Select OK > Restart Now

Then I would check to make sure this fix works by checking for the application under C:\Windows\System32\cmimageworker.exe. If it says file not found or a similar error, cmimageworker.exe should not create anymore problems for you.

If this fix does not do the trick, I would contact Microsoft Support and point them to this page. Additionally, I would be careful with using some of the "Virus Removal", free antivirus, or similar programs suggested in this forum to "remove" cmimageworker as many of will likley do more harm than good. 

If other people have this issue and end up attempting this fix, if you could reply with the outcome of disabling WDAG, that would be great in order to help Microsoft and other users resolve or diagnose this issue. 

-Parker L.

NOTES:

  • If this theory is true (If some one from Microsoft could confirm this that would be great...) affected users will be running Windows 10 Pro or above.
  • This "bug" should most likley affect users running build 1809 but it seems possible that it may affect 1803 and some more previous builds in the case of some enterprise users.
  • It is likely that some users may not be able to turn off WDAG due to policies set by their company/enterprise.
  • This process may be enabled by other Windows 10 virtualization features.
