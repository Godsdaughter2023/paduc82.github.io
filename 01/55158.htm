a. user say:

Hello,

Thank you for posting your query in Microsoft Community.

SYS file errors are typically caused by faulty hardware or corrupt device driver files. Because of the importance of Cng.sys in the functionality and other Windows functions, any corruption or damage to this file can create critical system errors in the form of a "blue screen of death" (BSOD).

In order to fix KMODE_EXCEPTION_NOT_HANDLED error, the computer must be restarted in safe mode, and the drivers of problematic hardware device must be installed or updated. Step-by-step instructions given below should be followed to update or install the latest drivers:

  1. As the computer restarts, press F8 key continuously to display the Advanced Boot Options screen.
  2. On Advanced Boot Options screen, select Safe Mode with Networking by pressing down arrow key, and press Enter to start the computer in safe mode with networking.
  3. Once started in safe mode, log-in to computer with the administrator account.
  4. While you are on desktop screen, press Windows key + X and click Device Manager.
  5. On Device Manager window, expand the Other devices category.
  6. Once expanded, right-click Unknown device whose drivers are to be installed or updated.
  7. From the context menu that appears, click Update Driver software to install the driver of the selected device.

  8. On Update Driver Software – Unknown Device box, click Search automatically for updated driver software to install the latest driver of the device from the Internet. Alternatively Browse my computer for device software to install the drivers manually located in the hard disk.
  9. Wait till the selected driver gets installed from the Internet and restart the computer normally when done.

  • You may also visit the computer manufacturer's support website and install the latest drivers available for the latest operating system (it may be Windows 8.1)

    Hope this information is helpful. Please feel free to reply in case you face any other issues with Windows in future.
