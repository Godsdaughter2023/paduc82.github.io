You will have to do a clean installation, there is no in place upgrade path from Windows 7 32 Bit to 64 bit.

Using the Windows 7 64 bit disk, install the Windows 7 Windows Easy Transfer Utility from the DVD (To find it, just navigate to x:\support\migwiz, 'x:' represents the name of your disk drive. In this folder, you will see a collection of files, the important one you need to look for and double click is 'migsetup'.) , run it and create a WET file (Windows Easy Transfer file), if you have an external hard disk, you can have it stored there so you can access it later when you do a clean install of Windows 7 64 bit disk.

Once you have done that, simply do a clean install of Windows 7 64 bit disk. In Windows 7 64 bit disk, launch Windows Easy Transfer, follow the on screen wizard, point to where you stored the the WET file (external hard disk), and it will restore your personal files and settings. Reinstall any necessary applications.

2econd2on say:

I found a procedure that solved this problem (for me).  I've learned that there are several problems that can cause a computer to not shut down. 

Try this.  It may work for you.

 

[Start] / “My Computer” / “Local Drive (C:)” / “Properties” / “Tools” tab / “Error Checking” / “Check now ...” 

 

Under “Check disk options,” select “Automatically fix file system errors.”  Right click “Start.”

 

In a brief period, your computer will ask you if you wish to schedule a disk scan at the next startup.  Indicate yes. 

 

Then force a shut down: [Ctrl]+[Alt]+[Del].  In Task Master, while pressing [Ctrl] key, right click “Turn Off.”  In a few seconds, your computer will turn off.

 

Restart the computer at the Power Button.  The disk scan will then start and it will take awhile.  It’s progress percentage will be indicated.

 

When the procedure is complete, Windows will start up normally.  Then “Restart” to see if it worked.

hairyfool say:

The Network name will be set to Ginger which is completely separate to the SSID. The SSID is only used by the Wi-Fi.

The network name can be somewhat obscure as to how to change with domestic routers, sometimes the only way is to completely reset the router. It may adopt the new SSID as the network name eventuallybut because it already has one it may not. The Network Name is of little value in a home network.

Incidentally changing the SSID does absolutely nothing to provide additional security. The SSID is normally broadcast and can be seen by anyone in range of the Wireless network so if anyone was using your Wi-Fi then they would at least be able to see the change. You can turn off SSID broadcast which would then make changing it of some use but it tends to make creating a connection somewhat unreliable.

It is a relatively basic process for any Wi-Fi hacker to find "hidden" SSIDs so turning off its broadcast is fairly ineffective. In fact it tends to flag up the Wi-Fi network as potentially "interesting" to them. Changing the password is the primary solution, preferably with a complex password.

hairyfool say:

I am going to give what is mostly a useless answer Yes and No.

No, because your router will have been configured with its own Wi-Fi password which will be different to yours so you are locked out of each others. No network is completely immune to hacking and breaking a hidden SSID is easy. Breaking the password is not but hidden SSID is sufficiently unusual to make someone think it could be worth trying.

So why yes? I do not know if others do this but BT in the UK have a particular feature in that their router hosts 2 Wi-Fi nets. Your own private one and a separate public one. The two are isolated, one cannot see the others traffic and the private has full bandwidth priority over the public. All it requires is when someone connects to the public nets they have to enter their BT account username and password.

As a BT customer I therefore can have access to thousands of Wi-Fi hotspots out and about and by way of a trade off other BT customers can use mine. When that happens I should not seen any bandwidth impact and guest traffic is not counted as mine.

I change WiFi password from default not because anyone would know it but so I can use a passphrase I can remember.
