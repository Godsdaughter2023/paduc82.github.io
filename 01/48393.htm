greg carmack - mvp 2010-2020 say:

If that isn't sufficient then these steps may be necessary:

What's happened here typically is that a backup program has written into the System partition and taken up the extra space needed by the Windows 10   to edit the boot files for Upgrade. 

With the older Legacy BIOS which uses MBR disk with a System Reserved partition, it is easy to give SysReserved a letter to browse in to delete the intruding file (which should have never gotten in there since programs cannot write to a partition without a drive letter) or extend the partition to a larger size using a Partition Manager like the outstanding Partition Wizard  which can even Extend a partition into space from a non-adjoining partition. 

But EFI System partition cannot extend because the adjacent MSR (Microsoft Reserved Partition) partition is a fixed type which cannot move.  We have had some success shrinking C from the left with Partition Wizard Resize Partition - Video Help  function,  then copy MSR to the right, to make room to Extend EFI System.  This often requires Startup Repair from the Windows disk or Recovery drive and as such risks failure.  You should as always have your files backed up along with a System Image Backup.

What's worked best is to right click the EFI System  to assign a letter in Partition Wizard, so you can right click to Explore to find the intruding backup folder.  Once you confirm it's there, to extract it requires a boot disk or stick.  The one which works best is Paragon Rescue Kit Free Edition 11.0 Free  flash stick, choosing File Recovery Wizard, then highlighting the intruder to Delete.  An example is here of what you'll see, although you wouldn't want to delete the file highlighted in this picture.  Highlight the unwanted backup folder(s) then click the red X next to the Source box to delete it.  

In the worst case you can always delete and recreate EFI System as a FAT32 partition, then run Startup REpair to rebuild it, or Use Bootrec.exe to Rebuild BCD.  Again have your files and a System Image backed up.

Let us know how it goes.

Hi Jolley Roger,

This issue can occur if you have an outdated version of Movie Maker, corrupted codecs, or outdated video card driver. To troubleshoot this, please follow these steps:

 

1. Ensure that you have the latest version for Windows Movie Maker. You can download the latest version here.

 

2. Ensure that you have the proper codecs to play your videos and audio files.  Audio and Video require codecs to play the file and it’s important to have the right codecs installed or updated in your PC. 

 

·         Try running the Microsoft Fix It tool for codecs.

·         Media info will be able to identify the codec required for your audio and video files, and will provide a link to the website for the required codec. You can get it here. 

 

Note: If you can't find the codec you need you may convert the video to a supported format (eg. wmv) and then add it to your Movie Maker project.

 

3. Ensure that your video card drivers are up to date. Movie Maker makes extensive use of the video card GPU (graphics processing unit). The display driver affects the reliability, performance, and compatibility of the video card with various software.

To determine the version and video card driver date, run the DirectX Diagnostic Tool (dxdiag):  

    a. Click on Start button and type dxdiag in the Search programs and files box and press Enter key on the keyboard.  
    b. Click on the Display tab and you will see the driver details including the driver version.  
    c. If your video card driver seems outdated please go to the video card driver manufacturer and see if a newer version is available.
