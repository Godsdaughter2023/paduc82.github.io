gts-nj say:     Your check disk output shows 5212KB in bad sectors!!  That is a very large number and indicates a severely failing drive. You need to back up your data and replace the drive immediately.

   There are probably many errors in the file system accounting for the conflicting numbers.  Running chkdsk in windows with no parameters doesn't fix errors.  AFTER backing up data, run chkdsk /r in elevated command prompt.  This may clarify the numbers, and possibly fix some damaged files. Then do another backup and replace the drive.

gts-nj say: chkdsk /r doesn't "fill up" the drive with bad sectors. It detects the bad sectors and marks them so they will not be used.  What you're doing guarantees that you're going to end up with damaged files and data loss.  The fact that the number of bad sectors found doubled just confirms that the drive is severely failing and needs to be replaced.

markinkent say:

Your computer believes that the network address (IP address) it's using is the same as another device on the network.  Now in the normal scheme of things this shouldn't happen if DHCP is working on the router / PCs etc.  You say there are no other devices in the house, what about smartphones, sky on demand box etc etc?  If these use your internet they will get an IP address - its how devices connect and communicate on a network.

Quick way to try to resolve the conflict - try to get a new IP for the computer.

Press the Windows key and R

The run box will pop up, in here type cmd

The cmd (command) box will pop up, in here type ipconfig /release

Once this has run, type ipconfig /renew

Once this has finished it should have assigned a new IP address

Good luck!
