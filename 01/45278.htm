a. user say:

Hello,

Thank you for posting your query in Microsoft Community.

Several factors are responsible to get the dump and temporary internet files get created:

  • Page file must be on OS drive
  • Page file base allocation size must be > than installed physical RAM
  • Windows Error Reporting (WER) system service should be set to MANUAL
  • Set page file to system managed on the OS drive
  • Set system crash/recovery options to "kernel Memory Dump"
  • User account control must be running.
  • Sometimes SSD drives with older firmware do not create DMPS (update firmware)
  • Cleaner applications like Ccleaner delete DMP files so dont run them until you are fixed.

Ideally, they can never be zeroed as any erroneous activity, big or small, gets reported and logged. However, if you delete the dump files on regular basis, this keeps your system performance optimized.

If, however, the size of the dump is too large, then you can try the following steps:

a) Go to the location: C:\windows\memory.dmp.

b) Try deleting it (or moving it to another location).

c) Now, run the Disk cleanup and check what it reports.

Hello TylerH111,

In particular case, I would recommend making use of a utility in Windows called DiskPart. For the following instructions, any commands listed in quotation marks should be typed without the quotation marks.

1. Press the Windows and R key at the same time. A Run prompt will appear, type in cmd and click OK. Command Prompt window will appear.
2. Type "diskpart" into the Command Prompt window to run DiskPart.
3. Once DiskPart has loaded, type "list disk" to list all of the drives on the computer.
4. Type "select disk X" (replace X with the number that represents your SD card, the easiest way to tell is by the size of the disk, which should closely match the size of your SD card)
5. Type "clean" to run the clean utility, which will remove any data on the SD card.
6. Type "create partition primary" to create a new partition on the SD card.
7. Type "select partition 1" (it should be the newly created partition from Step 6)
Step 8. Type "format fs=ntfs quick" to format the partition to the desired file system.
9. Type "active" to mark the partition as active. Windows should now be able to read the SD card and assign a drive letter so you can access it.
10. Type "exit" or click on the X in the top right corner to close DiskPart.

At this point, you should now be able to open and access your SD card as well as transfer files to/from the SD card. I hope this helps. Let us know if you need further assistance. Thank you.
