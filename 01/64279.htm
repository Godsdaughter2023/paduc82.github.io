gts-nj say:

The inability to boot on different hardware, even in safe mode, is not due to drivers but to the Hardware Abstraction Layer (HAL).  In all NT derived versions of Windows the HAL is built during the initial installation.   This is a software layer that works with the target CPU and chip set.
Here are a few links explaining the HAL -
http://en.wikipedia.org/wiki/Hardware_abstraction_layer
http://searchcio-midmarket.techtarget.com/sDefinition/0,,sid183_gci214424,00.html

It is indeed unfortunate that it is no longer possible to perform an in-place repair reinstallation in Vista and Windows 7 as it was in XP  (not only for hardware migration, but sometimes this was also a valuable last desperate repair possibility for a damaged system).   As you noted, if Win7 will load an upgrade installation can be done and is the closest equivalent. If it will not load, there is no such option.  Microsoft changed the installation process from a file based to an image based mechanism and is, apparently, indifferent to this loss of functionality.

The only option I know of to migrate a system to different hardware is Acronis® Backup & Recovery™ 10 Advanced Server software with a feature called Universal Restore.  This is an expensive product ($365) that can perform a backup of a functional system that can then be migrated to different hardware.
http://www.acronis.com/backup-recovery/advanced-server/universal-restore.html

Sorry. I don't know of any solution for you, but this may help explain the issue a bit.
