Hi

Some shortcuts to websites will initially have their own custom, personalized icon when you create a shortcut. This icon is called a 'favicon.ico' and is stored in the Temporary Internet Files (TIF) cache, which is where all internet website files are stored, when you visit them.

When the user performs some maintenance and deletes the TIF cache, all of these personalized icons are removed with everything else. This will cause the default Internet Explorer icon to be displayed on those shortcuts. This will also happen when the alloted maximum disk space for the TIF cache is exceeded and files are automatically deleted to maintain that disk space usage.

If you want to maintain the custom icons for a longer period of time, you can increase the disk space alloted for the cache. In Internet Explorer go to Tools/Internet Options. In the Browsing History section, select the Settings button. You can change the disk space there. 

If you want to maintain custom icons permanently for some shortcuts, you can open Windows Explorer and navigate to C:\Users\yourname\AppData\Local\Microsoft\Windows\Temporary Internet Files (AppData is a hidden folder. You can go to Organize/Folder and Search Options/View Tab and select to show Hidden Folders and Files).

In the TIF folder, locate the custom icon you want to save. Right click the file and select Copy. Create a new folder somewhere in your user folders and paste the file in that folder. You can then rename the custom icon with an intuitive name.

Go to the shortcut on the Desktop, right click the shortcut and select Properties/Change Icon button. Click Browse and navigate to the folder where you saved the custom icon and select it. Click Open and OK your way out of that dialog.

The shortcut will display that icon permanently. 

Hope this helps.

Thank You for using Windows 7

shopper1 say:

'Compress Old Files' Not Resetting in Windows XP

After running Disk Cleanup with 'Compress Old Files' checked, it still shows 79K. I rebooted and ran Disk Cleanup again but the 79K remains.

Why wasn't the amount reset to 0K?

a. user say:

What would you choose to Compress Old Files?

Do you know what it does, how it does it, why it does it and what the consequences are?  Most people don't - they just choose it because it "sounds" like the right thing to do or it "might be" a good idea.

A popular complaint from users is when they choose that option, their system appears to hang, perhaps for hours while XP tries to figure out what "old files" are and what to do about them.  How do you know what is an "old file" and what if XP thinks files are old but they are not old to you?

The more files you have on your system the longer it will take and it may just get stuck forever and never finish (or you might get tired of waiting).  If you have a million files on your system you could be waiting a very long time for Disk Cleanup to finish or it may never finish.

And - if Windows finds files to compress and if that part works and if something you do on your system later needs those compressed files, they are going to have to be uncompressed before they can be used.  This will make your system run slower (then you will wonder why that is and what to do about it).

Here are some thoughts to consider that involve just removing that option from Disk Cleanup - then your Disk Cleanup will run in a flash:

The most likely reason Disk Cleanup runs slow, appears to hang or uses excessive CPU is usually because of the Compress old files option.

Compress old files is from the good old days when hard disks were small and slow and you needed every byte you could get on you hard disk for programs and things.  Nowadays, with modern technology, it is hardly necessary.

Compress old files is supposed to compress files that "haven't been accessed in a while" I have never found anybody that can explain what "haven't been accessed in a while" means.  What is "a while" to XP?  Is it a a week, a month, a year, 5 years and how do you change it if you wanted to?

If you want your Disk Cleanup to run faster, I would just remove the Compress old files option in XP Disk Cleanup since it is known to hang or give the illusion of being hung.  If you have a million files on your system (or even a few hundred thousand), you are going to be waiting a very long time when you run Disk Cleanup and your system may appear to be hung (and maybe it really is).

I really don't think that with the technology available today that compress old files is necessary.  It will also allegedly slow you down if XP needs to uncompress a file.  Besides, nobody seems to be able to tell me how old an old file is in XP terminology...   just when does a file become old?

To get the best performance, I just remove that "feature" and never see it again and then Disk Cleanup runs in a flash since it doesn't have to look for old files to compress (whatever old means).

That adjustment will greatly reduce that amount of time it take your Disk Cleanup to run (I mean greatly).

It is such a "problem" for some systems that Microsoft made a KB article about how to disable it (mine is disabled).

http://support.microsoft.com/kb/812248
