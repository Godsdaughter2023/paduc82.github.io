caesarsr say: This is because of the high resolution and relatively small screen of the Surface.  To mitigate this affect, your Surface applies display scaling on it's local screen.  However, when you're making a RDP connection to a remote machine, the remote machine doesn't know to apply this scaling.  It further appears that you can't apply this setting in a remote session, so that kinda sucks.

musical9 say:

The surface pro 3 default text size slider setting is somewhat larger (the 3/4 setting on the slider). As a result, unless the remote machine has the same setting, a sign out on the remote is needed.

Simply sign out on the remote machine, and the next time you sign in remotely the icons/taskbar should be sized correctly. At least that was my experience.

The downside: you need to log out of the remote before logging in with any machine with a text size setting that is different from the last login.

It would be nice if this sign out process was not necessary.
