a. user say:

A System Restore will restore your registry to the state it was in at the time the restore point was made - if changes were made to the registry in the meantime they will be reverted to the former version (and any installations including updates made to the system between now and the restore point will be removed).  The trick is the restore point has to be BEFORE the change you're discussing to the registry occurred.

Any file on the DVD/CD can be deleted with no troubles for your system.  I'm not sure about that file on your system - it could very well be a valid and important system file (it sounds important though I'm not sure about it - I looked it up and it seems to exist as a valid Windows Vista file).  I would not delete it from C:\Windows\System32 without knowing much more about it.  I would instead try to correct it as follows:

Let's check some of your system files:

Go to Start / All Programs / Accessories / Command prompt and right click on command prompt and click run as Administrator.

Type sfc /scannow and enter and let it run.  It will scan and try to fix some of your system files.  Hopefully it will complete with no corruption it could not repair (if there is such corruption post back here or try to analyze it to find the problem file(s) using http://support.microsoft.com/kb/928228.

While in Command Prompt, type chkdsk /f /r and enter and let it run.  It will scan and try to fix any corruption or bad sectors on your hard drive and mostly remove that as a cause.



If the SFC scan found corruption it could not repair or if the problem persists, I suggest a system repair/upgrade using the genuine Windows Vista Installation Disk (one you own or one you can borrow from ANYONE).  Here's the procedure: http://www.vistax64.com/tutorials/88236-repair-install-vista.html.  Although this will not affect your data, settings, or programs, you should still backup your data before starting just to be on the safe side. This will reset the system file to that which is on the disk and correct any related problems. You may have a lot of updates to re-install (including any service packs you had to remove).  If the version on the system came with SP1 or SP2 pre-installed and the disk is an earlier version, then you'll need to make a slipstream disk as follows: http://www.vistax64.com/tutorials/151606-vista-sp1-slipstream-installation-dvd.html.

RE: Disk error 32, AX 429F, Drive 9F, are you running Linux on your system (perhaps in a dual boot scenario or from a disk)?  This error seems to be related to linux - I could find nothing relating to Vista systems.  I don't know why you are seeing it unless you are using a Linux OS.

I hope this helps.

Good luck!

Hi TJNEfam , 
My name is Lucia. I am an Independent Advisor. Thanks for contacting our support.

Change a new user account created from your brother to an administrator account:
In Settings> Accounts> Family and other users, select the name of the owner of the account and then Change the account type.
In Account Type, select Administrator and OK.
Log in with the new administrator account.

When you access your brother's account with Administrator privileges then check if you can then delete your child's account.
