tryx3 say:

Linda,

Ignore the above replies because they have both misunderstood your question.

When you are logged on to an Admin account and do something like install or uninstall a program, you will get a dialog that looks like this

"Consent UI" for an Admin user

This indicates normal Windows behaviour. 

  • There is no fault to be investigated or rectified and you have not lost any rights at all. 
  • This is just how Windows works - an Admin user is actually logged in with only Standard user privileges** but this dialog is used to allow you to invoke your Admin status for the application identified in the dialog [only].

Just for info, if you were logged in to a Standard user account and did something like install or uninstall a program, you would get a dialog that looked slightly different

"Credential UI" for a Standard [a non-Admin] user

Please note that if you were to lower your UAC setting as advised above, you would also be lowering your defences against online hackers.  With UAC at its lowest setting, for instance, neither you nor an online hacker would generate any warning dialogs when installing applications.  So if you ever reduce the UAC setting, do it offline for a particular task and then raise it to its default level or its highest level before going back online.

[** One exception to this has been found by Windows 10 users.  A network connection can have its "metered" property turned on or off by an Admin user without invoking that dialog whereas a Standard user cannot alter the setting at all, it is greyed out.  This exception flies in the face of all published Microsoft documentation about user accounts.]
