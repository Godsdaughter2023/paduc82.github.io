﻿Hi, Curt39.

In a Command Prompt window, use the Dir command (from the days before Directories became Folders) with the /x switch.  (As with any command in a CP window, type the command followed by /? to see a mini-Help file explaining the switches and parameters available for that command:  Dir /?)

Dir /x should produce a directory listing that looks normal on first glance. But look for a column before (to the left of) the usual filename column. This should be in ALL CAPS and will show the SFN (Short File Name), also known as the 8.3 filename, for any file whose LFN (Long File Name) does not comply with the 8.3 filename rules.  Use this SFN to move or otherwise deal with that file.  (Upper/lower case does not matter except to us humans.)

CORAL~89.MP3 looks like an 8.3 filename.  But look closely at the listing from Dir /x to be sure that you are reading it correctly.  Also, be sure that you are using the Ren command properly.  If the LFN has any spaces or other "illegal" characters in it, you will need to enclose the entire pathname in "quotes".  A made-up simple example:
Copy C:\Downloads\Coral~89.mp3 "C:\Music\Cora Lee.MP3"

Depending on the methods used by the source, the filename may have been converted to SFN before or during your download.  The full name of the song may not have been downloaded.
