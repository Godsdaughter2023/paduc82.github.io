Windows 8.1 Group Policy ADMX Template Overview - Microsoft Community

sainath irp_mj_create say:

Windows 8.1 Group Policy ADMX Template Overview

Summary
There are many situations where Administrators has to understand the functionality of ADMX templates , this article provides good understanding of ADMX templates and its functionality.
Details

Local Group Policy:

Administrators can configure Policy settings for the computers which are not joined to Active Directory domain, this is achieved with the help of Local Group Policy. Some of the scenarios includes KIOSKS machines which are placed in public places, Medical Systems, Workstations / Server Operating Systems placed in Demilitarized zone etc.

 

 

 

 

ADMX Overview:

Windows 8.1 Operating system displays registry based policy settings using ADMX format ( XML Format ). The ADMX replaces ADM files, but Administrators can still load ADM templates as Local Group Policy Editor reads ADM and ADMX templates from the store. Unlike Domain Joined computers where the ADMX templates are stored under central location, for Workgroup Windows 8.1 Client systems, Administrators have to load the ADMX templates / ADM templates on to the Windows 8.1 client local store which then be read by Local Group Policy Editor.

 

Policy Templates Location in Windows 8.1 Workgroup

File Type

File Location

ADM

%systemroot%\inf

ADMX Language Neutral

%systemroot%\PolicyDefinitions

ADMX Language Sepcific ( ADML files )

%systemroot%\policydefinitions\Language folder (ex: En-US)

                         ADMX Template Location in Windows 8.1 Client.

 

 

 

Policy Templates Location for Domain Joined Windows 8.1 Operating System

Active Directory Administrators configures the Central Store to populate the ADMX files and this Central Store will be accessed by Windows clients for any Group Policy activity. In Windows 8.1 , Group Policy Object Editor reads the ADMX files from Sysvol folder located on Domain Controller or reads the ADMX files from local store when Domain Controller is unavailable.

 

File Type

File Location

ADMX Language Neutral

%systemroot%\PolicyDefinitions

ADMX Language Sepcific ( ADML files )

%systemroot%\policydefinitions\Language folder (ex: En-US)

 

 

 

For domain joined Windows 8.1 , Administrators creates Group Policies at the Domain level and tie the policies to specific containers called Organizational Units, when the domain based policies are created, they gets stored in Active Directory and in SYSVOL folder, each Group policy created will have dedicated folder with Group Policy files. This behavior is different when a group policy is created locally or on Windows 8.1 configured in Workgroup mode, when local group policy gets created in Windows 8.1 client, a corresponding folder gets created under C:\Windows\SysWOW64\Group Policy
