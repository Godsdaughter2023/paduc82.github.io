Windows 2003 is so far out of support that you are putting your business at risk. Win2003 is not getting any security updates.  You really need to upgrade. 

Has this bat file been running successfully for some time or is this something new?

On Dest1 run "net session" to see if Source is keeping sessions open to the other Windows server. If it is then you know that the problem is Source is not closing the session. 

On the NAS box, do you have a way to display and close sessions?  Can you close all of the orphan sessions. Then retest the bat file interactively and see if the problem reoccurs. If it does, the problem would appear to be with the NAS box. Reboot it and see if the problem goes away.

If Source is not closing the session, then the first thing I would look for is hung cmd.exe processes that were launched by the task scheduler. Download Process Explorer. https://docs.microsoft.com/en-us/sysinternals/downloads/process-explorer 

The process tree is indented by parent/child. Find the task scheduler and see if there are cmd.exe's under it. The lower pane can show handles. Your bat file should be there.

Also look for processes that are not indented. That indicates that its parent process has terminated. Likely it is hung. Reboot Source and see if your problem goes away.  

Your script should also run faster on Source if you use the local drive letter and not the Source UNC path. IE:  copy "G:\DB Daily Dump\DayEndDump.dmp" "\\192.168.10.Y\d$\Dump\10.X\"
