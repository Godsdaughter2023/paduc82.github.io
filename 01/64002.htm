    a. user say:

    If your system will enter Stand By or Hibernation manually from the Turn Off Computer menu, that is a good sign that the mechanism itself is not afflicted.

    The system has to be "idle" to enter Stand By or Hibernation.

    If there is anything on your system that consumes 10% or more of the CPU for even a fraction of a second, that is enough to keep your system from entering Stand By or Hibernation.  If you sit there staring at Task Manager waiting for that to happen, you might not even notice it.

    Idle means no mouse, no keyboard, no surfing, no scanning, etc.

    There are lots of things you might have installed/enabled/running that can cause your system to not be in an idle state.

    Does your system have installed Windows Desktop Search (uninstall it), is the Indexing Service enabled, (disable it), Skype, iTunes, MagicJack, SnugTV, RSS feeds, any kind of Instant Messenger, any kind of malware scanners doing scans or doing automatic updates like MSE, AVG, avast!, Norton, McAfee, Bit Defender?  

    One user reported that the Java Quick Start Service was their problem and you don't need JQS starting and running all day with nothing to do anyway (like so many Services like it), so if you see Java Quick Starter in your Services, disable it there or from the Java Control Panel and see how things look.

    You may have to use a process of elimination to disable things one at a time until the system starts to behave.

    What I like to do is set the timer to 1 minute, double click the system clock in the bottom right hand corner or your screen so you can see it, then watch (in agony) for the second hand go around for one minute (or two or three) and see how it looks.  If it works, you will know that you have something running that is keeping your system from doing what you want.
