"System Restore" is for "System" files.  Your stuff is not considered critical to the functionality of the "System".  If you did a true "System Restore" in the Microsoft sense of the words - then you lost nothing.

If, when you use the words "system restore", you are referring to some third party OEM procedure that restores the system to factory default status OR you manually restore it to 'as shipped' status with a format/install - your stuff is gone.

Hey awesome - the updates actually roll out on a staggered schedule, typically over a 2 to 4 day range. This is largely to reduce the servers being overloaded by rolling out all updates at once. I'm not sure how the "schedule" works, but for example, a set number of consoles would receive the update beginning on Day 1, another group would receive the update on Day 2, and then any consoles which have not received the update while on Standby are prompted to update the next time they connect to Live.

a. user say:

Hi Rick,

Thank you for participating in the Microsoft Community. Let me assist you.

As you have lost the Administrator privileges on the main account, you may log on to a different Admin account (in case you have one) and change the status of the account to Administrator.

Method 1:

Login to an Admin account on the system and follow the below steps to change the status of the main account.

Steps:-

1. Right click on the Start menu and select Control Panel.

2. Click on User Accounts and select Manage another account.

3. Double click on your user account.

4. Now select Administrator and click save and ok.

Method 2:

In case you do not have another Admin on the computer, enable the built in admin account from the Windows Recovery Environment, login to that account and then change the status of the present account from Standard to Administrator. You may get an error when you try to enable the hidden admin account as the command only runs on an Elevated command prompt, i.e., an admin Command prompt. As you can’t access that on your system, I suggest you to go to Windows Recovery Environment and use the Command Prompt there, which is elevated.

Follow the below steps:-

1)    Press Shift and Restart from the power icon (together)

2)    Select Troubleshoot.

3)    Go to Advanced Options.

4)    Select Command Prompt.

5)    Type "net user Administrator /active:yes"

6)    Hit Enter.

A user account with the name Admin will be created. Log in with that account and you will be able to access Admin privileges. You can also create and change the status of other accounts with that account.

Hope this information is helpful. Please do let us know if you need further assistance, we’ll be glad to assist you.
