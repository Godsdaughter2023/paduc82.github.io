hairyfool say:

As I said this is not a Windows problem it has to be the motherboard doing it. The problem is where you look for the setting  will depend on the BIOS developer for your board. Whereas classic BIOS were mostly the same it is not the case for the UEFI BIOS, they can be different

From PCGuide.com

Halt On

Some PCs give you the ability to tell the BIOS specifically which types of errors will halt the computer during the power-on self test section of the boot process. Using this, you can tell the PC to ignore certain types of errors; common settings for this parameter are:

  • All Errors: The boot process will halt on all errors. You will be prompted for action in the event of recoverable errors. This is the normally the default setting, and is also the recommended one.
  • No Errors: The POST will not stop of any type of error. Not recommended except for very special cases.
  • All But Keyboard: The boot process will stop for any error except a keyboard error. This can be useful for setting up a machine without a keyboard, for example for a file or print server.
  • All But Diskette/Floppy: All errors will halt the system except diskette errors. In my opinion, if your floppy drive has recurring and known problems, it is most likely best just to replace (or disconnect) the drive rather than using this.

Warning: Telling the system not to halt for any error types is generally not wise. You may end up missing a problem with your system that you will want to know about.

http://www.pcguide.com/ref/mbsys/bios/set/stdHalt-c.html

System BIOS can handle devices connected to the USB ports and so can find a USB Keyboard/Mouse/Drive during startup. This is because it has a minimal driver built in. This is not the case when connecting via Bluetooth either using an integrated interface or one itself connected through USB. Until the Bluetooth interface has initialised, found and paired with the keyboard the system has no keyboard attached and it generates this error.

This Menu option has been available for decades to get over devices such as servers not having a permanent keyboard attached. That problem is still valid so I would be a little surprised if it is not in the BIOS screens somewhere.

Hello Astralvs,


OEM (Original Equipment Manufacturer) copies of Windows are supplied to system builders. They are also supplied at a reduced price to that of retail versions of the operating system. The OEM copies are less expensive because the 'system builder' is responsible for any after sales support.


For support you need to contact the supplier of your PC. If they are unable, or unwilling to help, you would need to contact Microsoft directly and take out a support contract with them.


You state in your original post that you have 'medical data for my patients' on your computer. Medical facilities, either General Practitioners or Hospitals, usually employ their own administrators for computer systems and, in turn, have suitable support staff. Perhaps you need to enquire whether this is the case in your own practice.

I'm not trying to be rude or flippant but there is only so much that moderators, MVPs and the general helpers on this forum can do - we are, after all, only human!
