If you have more than one account on your PC, you can switch to a different account without signing out or closing apps. To switch to a different account, follow these steps:

1. Open Start by swiping in from the right edge of the screen and then tapping Start. (Or, if you're using a mouse, point to the lower-left corner of the screen, move your mouse all the way into the corner, and then click Start.)

2. Tap or click your account picture, and then choose another account from the menu.

And there is another option to switch between the user.

  1. Press CRTL+ALT+DEL
  2. Select SWITCH USER
  3. Select the user account which you want to sign in.

 

Make sure to save any open files before switching users because Windows doesn't automatically save files that are open. If you switch to a different user account and that user shuts down the computer, any unsaved changes you made to files that are open on your account will be lost.

james.burgess say: Generally speaking virtualized environments aren't great for gaming.. mainly because the graphics accelerators aren't present (Most virtual machines install a very basic software graphics driver like the old Intel Integrated Graphics), or if they are they have nowhere near the performance of the same adapter on the Host OS.

With this in mind for legacy gaming, I've always just either partitioned my HDD and done a dual boot on the same drive.(which can be a bit painful if you have problems with either OS) or depending on your machine, install a 2nd Hard disk, and Install XP to the second disk and use the Bios Boot Select to switch between OS's when the machine boots.

The later method means if your windows 7 or XP installs go pear shaped then you can simply remove the drive or Change boot order and boot your machine normally using the other OS, without having to worry about MBR's and all that.

I know VMware has been working very hard to improve graphics performance in Virtual machines, but not 100% sure if Virtual PC or VirtualBox has similar DirectX acceleration.

Also since the XP install would be for a few games only, it may be worth checking out a tool like nLite (This removes components from XP, you can ultimately end up with an install that uses as little as 40mb ram, and a small amount of HDD space.)

apretceille say:

create the boot system for windows vista ultimate 64 bits

I want create a dvd boot system windows vista ultimate 64 bits ?

naman r say:

Hi Apretceille,

Welcome to Microsoft Vista answers Forum!

I have few steps here that might fix the issue.

  1. From your technician computer, open a command-line window and run Copype.cmd to create a local Windows PE build directory:

Cd Program Files\Windows AIK\Tools\PETools\

Copype.cmd arch destination

Where arch can be x86, amd64,or ia64 and destination is a path to a local directory, for example, type:

Copype.cmd x86 C:\WinPE_x86

  1. Copy additional tools such as ImageX into your Windows PE build directory. For example, type:

Copy “C:\Program files\Windows AIK\Tools\x86\imagex.exe” C:\WinPE_x86\iso\subfolder

Wheresubfolderis any folder structure required to support your tools.

  1. Using Notepad or any text editor, create the following configuration file, and then save it with the name wimscript.ini:

[ExclusionList]

ntfs.log

hiberfil.sys

pagefile.sys

"System Volume Information"

RECYCLER

Windows\CSC

[CompressionExclusionList]

*.mp3

*.zip

*.cab

\WINDOWS\inf\*.pnf

Note

This configuration file instructs ImageX to exclude certain files during the capture operation you will do in the next step.
  1. Save the configuration file to the same location as Imagex as specified in the previous step. For example:

C:\WinPE_x86\iso\subfolder

ImageX automatically detects wimscript.ini if it is located in the same location.

  1. Create an image file (.iso) by using the Oscdimg tool. To do this, type:

Cd Program Files\Windows AIK\Tools\PETools\

Oscdimg -n –bc:\winpe_x86\etfsboot.com c:\winpe_x86\ISO c:\winpe_x86\winpe_x86.iso

  1. Burn the image (Winpe.iso) onto the blank CD-ROM. You now have a Windows PE CD with ImageX.

Additional information: http://technet.microsoft.com/en-us/library/cc721929%28WS.10%29.aspx#BKMK_CREATE_IMAGE

Hope this information is helpful.
