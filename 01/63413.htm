Are you using a laptop with a touchpad?  Many touchpads these days come with "multi-touch" on them, so that you can pinch two fingers to zoom in and out (similar to a smart phone).  Unfortunately many touchpads are also so sensitive that they think they detect a second finger even if you don't mean to do it.

dave eberly say:

My install of the update "got stuck" at 32%.  But it was not really "stuck."  I launched Task Manager and it showed high disk usage (nearly 100%). I selected the Performance tab, and then clicked on Open Resource Monitor (at bottom of window).  In the Overview page, click on the Disk bar to see the current disk activity.  I selected the "Total" column for sorting.  Turns out that SetupHost.exe was reading a large number of files on my hard disk.  In fact it might have read every file.  I have symbols installed for Windows 10 (for debugging during development) and SetupHost.exe was reading all the symbol files.  Later it was reading a lot of the files in the "C:\Program Files (x86)" directory tree.

If you see similar disk activity, just be patient and let the setup continue (don't reboot the machine).  Eventually, the installer restarted the machine and got me to the standard plain Windows Update screen ("copying files", "installing drivers", "finishing" progress controls).  Understand that this is effectively a new install of an OS; it is not a simple 1-minute Windows update...

I had a couple of problems when the OS finished installing.  Firstly, after opening Microsoft Outlook 2016, after about 20 seconds my monitors went black and were receiving no signals from the computer.  It might be that my graphics card driver was updated during the install.  A reboot fixed that problem.  Secondly, opening Microsoft Outlook 2016 again, I got the blue Outlook window with "Starting..." or "Processing..." but Outlook never launched.  Rebooting did not fix this.  I created a new Outlook Profile (via Control Panel -> User Accounts -> Mail), after which Outlook 2016 was successful in launching.

ajay k say:

Hi JoelBresler,

Since when are you facing this issue?

Let’s first understand what Check disk is, CHKDSK (short for Checkdisk) is a command on computers running windows operating system that displays file integrity status of hard drive and can fix logical file system errors.

CHKDSK scans the computer automatically if integrity violations are found.

To answer to your questions:

1.    Yes, those are system files.

2.    CHKDSK will automatically fix them; you don’t have to download them.

3.    Corrupted files are repaired by CHKDSK or SFC (system file checker) scan.

 

I would suggest you to try the following steps to fix the errors:

 

Step 1: Run CHKDSK again and make sure automatically fix file system errors is selected.

 

1. Open Computer by clicking the Start button, and then clicking Computer.

 

2. Right-click the hard disk drive that you want to check, and then click Properties.

 

3. Click the Tools tab, and then, under Error-checking, click Check Now.  If you are prompted for an administrator password or confirmation, type the password or provide confirmation.

 

To automatically repair problems with files and folders that the scan detects, select automatically fix file system errors. Otherwise, the disk check will simply report problems but not fix them.

 

To perform a thorough disk check, select Scan for and attempt recovery of bad sectors. This scan attempts to find and repair physical errors on the hard disk itself, and it can take much longer to complete.

 

To check for both file errors and physical errors, select both Automatically fix file system errors and Scan for and attempt recovery of bad sectors.

 

4. Click Start.

 

 

Step 2: The system file checker (SFC) utility allows administrators to perform a scan of all protected resources to verify that they are the correct version. If SFC should find any incorrect versions in any of these protected resources, SFC will replace them with the correct versions.

You can always run a SFC scan on the windows vista computer. To run a SFC scan follow the steps below:

a. Click on start and in the start search bar type cmd, right click on the command prompt icon in the programs area and then click on run as administrator.

b. at the command prompt type sfc /scannow and then press enter.

 

For more information, see the link below:

http://windows.microsoft.com/en-us/windows-vista/Check-your-hard-disk-for-errors


http://technet.microsoft.com/en-us/library/bb457122.aspx
