Re-enabling a hard drive after disabling it in Device Manager

How can I re-enable the storage drive to behave just as it had before I disabled it in the Device Manager if I cannot log into my User account? I tried logging into a Guest account but apparently that's not possible either.

The storage drive still appears in BIOS, so I know it is recognized. If only I could enable it again just as it was before I disabled it. Then the Users folder would be intact and in the correct space for the OS to recognize it.

I can configure the BIOS to boot from the CD-DVD drive (with Windows CD in the drive) to attempt a OS repair, but I'm not sure this is the most efficient way to preserve the data in the Users folder as well as keep the settings I had set up and been happy with before. Any suggestions?
bjnavillus say: Thank you for the help so far, Palcouk. However, this thread is getting a little off topic and I have found a solution to the problem that I initially posted about so let me explain what I did.

In my case, because I had two hard drives and only disabled the hard drive that is used for storage (as opposed to booting the OS and running various software) and because the disabled hard drive was the one that contained the actual \Users\ folder, I ran into this particular issue. Under these circumstances, I was able to boot Windows but not able to sign into my main User Profile because its preferences were inaccessible on the disabled drive. To resolve this, I had to go through the following:

1 - Restart in Safe Mode with Command Prompt
2 - Activate the Administrator User Profile by entering the following code into the Command Prompt: "net user administrator /active:yes"  - after entering this, I received confirmation in the Command Prompt window that it had executed properly
3 - Restart normally
4 - When selecting a User Profile with which I would log in, I chose "Other User" (or something similar, I can't remember exactly what appeared) and entered "administrator" as the user name, with no password. Press Enter
5 - New User Profile preferences were created and saved, including Desktop information
6 - Navigate to the Device Manager in Control Panel, find the Disabled hard drive. Right-click and hit Enable
7 - I followed the instructions in the link given to edit the Windows Registry files that pertained to user profiles (Method 1): http://support.microsoft.com/kb/947215

After restarting normally, I was able to log into my main User Profile and all of the preferences I had prior to Disabling my hard drive were still there.

One more question for Palcouk, if you're still with me: If I had 2 HDDs in the PC (in addition to the primary boot SSD) and 1 HDD functioned as the active storage drive with \Users\ folders, and the other HDD functioned as a mirror image of the first HDD (think raid 0 1), would that work as I wanted in the previous post I made? The only difference is using the second HDD as an exact copy of the first HDD rather than using it as a place to put the Windows backup system image.

The official explanation of Svchost is:

svchost.exe (Service Host, or SvcHost) is a system process that hosts multiple Windows services in the Windows NT family of operating systems. Svchost is essential in the implementation of so-called shared service processes, where a number of services can share a process in order to reduce resource consumption.

I find the last part humerus, it is meant to reduce resource consumption . . .

OK, basically a lot of WIndows processes are mounted using svchost, that is why you will see multiple instances of svchost running in Task Manager.

The issue as i see it is, which process running on svchost is hogging your system resources.

So, go to task manager, click the arrow beside the instance of svchost that is hogging your resources to see which actual Windows Service is associated with it
