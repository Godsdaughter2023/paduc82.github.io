I figured out what was causing my Skype to crash.

In my case, the new Skype App crashed every time someone started screen sharing. I have an old Dell Vostro 3550 with Switchable Dynamic Graphics, and this appears to be the cause of my particular issue.

I was able to resolve the issue by turning Switchable Dynamic Graphics to "Force power-saving graphics" under Power Options:

  1. Right-click on the battery in the task bar and select "Power Options"
  2. Click "Change advanced power settings".
  3. Scroll down to Switchable Dynamic Graphics > Global Settings:
  4. Set "On battery: Force power-saving graphics"
  5. Set "Plugged in: Force power-saving graphics"
  6. Click "OK"
  7. Restart Skype

On a side note, this also resolved the issue I was having of other people seeing a black screen when I shared my screen.

I think I has this error before, once when I didn't know much about computers and I had automatic restart enabled and then again a few years later, when skipping a CHK/DSK.

c000021a is like "A process or thread crucial to the system operation had unexpectedly exited or been terminated." in so many ways. It means the termination of Windows Logon (Winlogon.exe) or Client-Server Runtime SubSystem (csrss.exe). It is a user-mode related error so best not try anything meant for kernel-mode debugging, it probably won't work.

You might want to launch Startup Repair as I did and if this doesn't work, Windows might launch AUTOCHK (CHKDSK.)

If you have easy enough access to Startup Repair and CHKDSK, the problem should be a real piece of cake to fix (providing it is merely a software error (even better if it's a disk error) and not a hardware fault). If you don't have access to any of those, you might need to use a Recovery Disk, Last Known Good Configuration, Recovery Console, MS-DOS and sometimes, as a last resort, Factory Reset.

I do hope the problem gets solved before it gets to the point you have to do a factory reset.

I would like to inform you that ASP.NET Machine Account is created when the Microsoft .Net Framework 1.1 is installed on the computer. In order for you to remove the account I would suggest that you perform the methods mentioned below:

If you are working with asp.net development work then you must keep that account, otherwise ASP.net projects will no longer function correctly.

If you are a developer and want to disable the password prompt, please follow these steps:

  1. Tap or press Windows Key + R.
  2. Type control userpasswords2 (there is a space after control) and hit enter.
  3. Uncheck the option “Users must enter a username and password to use this computer”.
  4. Click on Apply and Ok.

If you are not a developer, you may delete the ASP.NET Machine Account and check if it helps.

  1. Press Windows key + X key.
  2. Click on Control Panel.
  3. Under view, select large icons.
  4. Go to User Account.
  5. Click on Manage another account.
  6. Click on “Manage another account” and then select “ASP.NET Machine Account”.
  7. Click on “delete the account”.
  8. Depending on whether you want to “Keep the files” or “delete the files” choose the appropriate option.
  9. You can delete the other accounts similarly.

    Note: Once deleted, the user accounts cannot be recovered. Make sure you back up any data if required.
