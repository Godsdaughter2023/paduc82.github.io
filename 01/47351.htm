Error 1079: The account specified for this service is different from the account specified for other services running in the same process.

 

The issue may occur if the service is started by the Local System account instead of by the Local Service account (NT AUTHORITY\LocalService). The Local System account may have insufficient permissions to start the service.

 

Set the service and any dependent services to run under the NT AUTHORITY\LocalService account:

a.     Press Windows key + R on the key to bring up Run.

b.    Type “Services.msc” without quotes and press Enter.

Note: If you are prompted for an administrator password or confirmation, type your password, or click Continue.

c.     Right click the "Peer Name Resolution Protocol" service and choose Properties.

d.    Click on the "Log On" tab.

e.    In the This account text box, type “NT AUTHORITY\LocalService”.

f.      Set both Password fields blank.

g.    Click Apply and then OK.

h.    Repeat these steps for the ”Peer Networking Grouping” and “PNRP MAchine name Publication Service”.

i.      Restart all the services.
