I face exactly the same problem and it is frustrating. You should have tested this very simple thing before rolling out Windows 10 upgrade.

The issue is that the service "Windows Installer" is set to "Manual" Startup in "services.msc", and also it is grayed out, so you can not set it to Automatic start up.

This is causing the following behavior, when you try to install any software, such as Microsoft Visual Studio 2013, in the middle of the install, you would see that the installation process is hanging .. If you run the Task Manager, you would see many stalled "Windows Installer (32)" that has 0% CPU utilization and a 1.1MB memory, and you can wait for the lifetime and these processes would not move, if you Kill this processes, then the Installation process would continue.

If you run "services.msc" and then click on "Start" manually, then you start the Installation process, it would hang at "Creating System Restore Point" till you kill another stalled "C# Command Line Compiler" but it will continue the installation properly.

So, it looks that you have a clear issue with the Setting of the Service "Windows Installer". You can not change the Startup to Automatic, as the entire data for the service is Greyed Out.
