gts-nj say:

For the purpose of protecting the computer from lightning and power surges does putting a computer into sleep or hibernate mode protect the computer or is the computer still as vulnerable as if it were on. My basic understanding is there is not much difference between the computer being on and sleep mode other than boot up time and that the only difference would be the cost of electricity of the computer being on.

It adds no physical protection as far as possible damages to components.  If it is hibernating or in hybrid sleep there is protection against data loss of corruption from a crash due to loss of power (assuming no ups is in use).

When the computer is in sleep mode is it vulnerable to viruses that may be in a network system? 

No.

Is it better to use sleep mode, leave the computer on or shut it off?          

Better for what?  The purpose of sleep or hibernate is to give you faster startup.  If you're concerned about storm damage unplugging it is best.

patrick barker say:

Hi,

When the system goes into sleep mode, it's still turned on, but goes into a 'standby' mode and places the system-state into RAM to be quickly retrieved/restored upon wake cues. Given the system is still consuming energy, and technically still on during this process, a power surge would still have the potential to damage the system.

kavita m123 say: Hi Alan Treen,

Thanks for visiting answers.microsoft.com forums.    I was doing research and found some useful advice from another one of our Support Engineer’s. Try the steps below to see if it fixes your problem.  

 Verify that the Windows Installer Service is not already in use.

 

The Windows Installer service can only run (1) setup instance at a time. If you receive this error it is because another setup or configuration is already in progress.

 

If you have not started an install or uninstall action this may indicate a problem with another program that is continually trying to configure itself on startup. If you see a notification, every time that you boot, that a program is being configured this particular program is the culprit.

 

How to temporarily resolve this issue.

 

You will need to open the Windows Task Manager and identify whether or not the installer service is presently running. If it is running in we can manually end that particular task, and this will allow you to proceed with your installation.

 

Caution: Please use caution when ending a process or thread in within the Task Manager. Ending a process can cause a program to close and cause any unsaved data to be lost. If a system process is closed, a system component may no longer function correctly.

 

1.     To end the Windows Installer task do the following:

2.     Right-click the Taskbar and select Task Manager.

   - or -

3.     Press Ctrl+Alt+Delete and then select the Task Manager.

4.     Click on the Processes tab, and then click on Show processes for all users.

5.     Click on Continue or provide administrator credentials if prompted.

6.     Under the Image Name column search for a file called msiexec.exe 

7.     If you locate this field, and have not started an install/uninstall we will want to end this process.

8.     Right-click the msiexec.exe process, and select End Process.

9.     You will receive a confirmation, and you will need to click on End Process.

10.  Once you have done this go pack to Programs and Features to see if you can successfully install/uninstall.

 

How to permanently resolve this issue.

 

If the previous troubleshooting resolved your inability to install/uninstall programs you will want to confirm that this behavior does not happen every time you turn on your computer. If this issue is persistent then you will want to identify what program is constantly using the Windows Installer (msiexec.exe) service.

 

Often times when you manually end the process as described above the program will automatically try and restart its installation. You may see a window saying “<Program name> is being configured…” which would readily indicate which application is at fault. You would simply need to uninstall, and then reinstall the application at fault.

 

If you aren’t so lucky you may need to manually locate the program. One method you can try is using Selective Startup (Clean Boot). If you clicked on the previous link you can skip the section titled “Identify the system service that is causing the problem” section. Once you have narrowed down which application is the problem you can remove and reinstall it.

 

Please let us know if you still have problems.

naman r say:

Hi,

Method 1:

If you get this error message at the end of the installation, it simply means that the installer has failed to detect DirectX 9c. There are two steps to solve this:

Install the latest DirectX End-User Runtimes :

http://www.microsoft.com/download/en/details.aspx?displaylang=en&id=35


This won't fix the installer. However, it'll insure that you've got the right dll files so that the actual game can run without crashing.

Method 2:
Remove the DirectX folder from the installer's directory :

If you've got the digital download version, that's a fairly straightforward process. Unzip the installer somewhere using the executable that was given to you. The installation should start automatically. Cancel it without installing. Go to wherever you've unzipped the installer and delete the subfolder named DirectX.

Start the installation by running autorun.exe within the installer's directory. Install the game normally. At the end of the installation process, the program will ask that you reinsert the disc. Just press OK without changing anything.

The installer should skip the part where it verifies your installation of DirectX (which means you'll avoid the error message).


If you've got the DVD version, you'll need to copy it on your hard drive first (so that you can remove the subfolder). I haven't tested this.

Invalid CD-KEY during the installation

For some obscure reason, uninstalling the game and reinstalling it in the same folder results in this error message.

The uninstall leaves a file behind called nwncdkey.ini. It appears that this file is actually the culprit here.

Just delete it before trying to install again.

As an example, if you're trying to install the game in
C:\Program Files\Atari\Neverwinter Nights 2\

Make sure that there's no nwncdkey.ini there first!

 

Method 3:

You may also try to install the game in Clean Boot state.

Putting your system in clean boot state helps in identifying if any third party applications or startup items are causing the issue. If yes, you may have to contact the program manufacturer for any updates or uninstall and re-install the program.

Refer the following Microsoft KB article for more information on how to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7:

How to troubleshoot a problem by performing a clean boot in Windows Vista or in Windows 7
http://support.microsoft.com/kb/929135 

Note: After troubleshooting, make sure to put the computer to start as usual as mentioned in Step 7 in the above KB article.

azeez n say:
Hi gary weiner,

There is no gpedit feature in Windows 7 Home premium

In this case, it is true that terminating the backup and restarting would make the overall backup faster. The issue manifests when the data getting backed up as part of same backup is huge. Breaking the data getting backed up into parts would help alleviate the performance issue considerably. Since file backup is inherently incremental, the backup after the previously canceled one would only backup data that was not backed up as part of previous backup.

It is also true that choosing "Better Performance" over "Quick Removal (default)" also improves the backup throughput. But accidentally removing the external disk (or power shutdown) could cause some of the data backed up to not get persisted to the disk. So need to deal with this cautiously.

You might want to temporarily disable your security software and try again.

The issue has been identified to occur on systems with large data set size.  Please refer to the following post for further details and workaround.

http://social.technet.microsoft.com/Forums/en/windowsbackup/thread/1ba794f6-0bac-443b-9e22-5b8f175c69ba

Windows Backup always does compressed backup. As a result backup time will be more when compared to simple file copying. However since the subsequent backups would be incremental, where in only the files that change would get backed up, the subsequent backups should be much faster.

Also, are you including system image backup as part of backup configuration? If so, the data getting backed up would also include the used space in the boot/system volumes

You may cancel the process by getting into Task manager(Crtl+Alt+Del) >> Processes >> locate and "End Task" and then put your computer in a Clean Boot state and try back up again.

a. user say:

Hi,

 

There isn't a way to determine the codec used to compress a file with absolute certainty, but the following are your best options:

  • To determine what codec was used with a specific file, play the file in the Player, if possible. While the file is playing, right-click the file in the library, and then click Properties. On the File tab, look at the Audio codec and Video codec sections.

  • Use a non-Microsoft codec identification tool. Several are available on the Internet.

You might be able to tell the format of a file by looking at the file name extension (such as .wma, .wmv, .mp3, or .avi). However, there are limits to this approach. Many programs create files with custom file extensions. And it's possible for anyone to rename a file without changing the file's format. A file with the extension .mpg or .dvr-ms, for instance, is usually just an AVI file that has been compressed by using some version of an MPEG video codec.
