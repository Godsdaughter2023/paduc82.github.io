Sorry the suggestions did not help.

My recommendation to users going forward is not to pounce on every new Windows 10 Feature Update that is released. If your system has been working just fine with the current version, wait it out a while or skip it all together. If there is some specific feature in the update you want, then thats an exception, but these feature updates are becoming more difficult to install on older devices; and it sometimes makes you wonder if the inconvenience is worth the hassle just to keep doing the same thing.

If you look at whats considered the flagship features of Windows 10 1709, they are aimed at newer generation devices like the new Surface Book 2. Otherwise welcome features like Story Remix, which you have to upgrade to 1709 to get, could be supplanted with third party alternatives.

There are some compatibility issues with Windows 10 version 1709 and certain hardware or software. There is no explanation why a lot of these problems are happening in Windows 10 1709, but the release is significantly flawed. Software and Hardware that once worked in previous versions, just suddenly stops working in Windows 10 1709 after an upgrade and sometimes loses functionality or stability.

Just to touch back on why I personally think it might be broken. When Microsoft starts preparing for a new release, it goes through a process called refactoring. It seems this is introducing some significant code changes that breaks certain protocols and API's required by older apps and drivers. Considering the Windows Driver Model has not changed much in its near 30 year history not mention the application programming interfaces such as Win32, this might be culprit.

Microsoft might be making plans to leave behind WIN32 soon, as evidenced by the edition called Windows 10 S which only uses Store apps.

jamesharwood3 say:

Microsoft Surface Laptop function keys for brightness and volume won't work - FIXED

Hi all,

New to the forums but wanted to share this fix as it has frustrated me for many many months that there was no a solution available.

I've previously searched the forums and spoken with Microsoft Support but have been unable to find a solution, some being running command prompts, and uninstalling the keyboard drivers. It was at this point I tried something new.

Noticing the Human Interface Drivers above they Keyboard Drivers I found the solution;

1. in the search bar type 'Device Manager' and select it

2. Expand 'Human Interface Devices'

3. Right Click on GPIO Buttons Driver

4. Click 'uninstall device'

5. Restart your device to re-install the driver

The problem should now be solved. (you may have to try using the 'fn' key at first but my device is now fixed using the above method.)

I hope this helps people out, I'd be interested to hear if anyone else has success using this method!

James
