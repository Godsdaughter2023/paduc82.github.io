Cannot update Windows 8 - no updates will install - error 0x800F081F or 0x8024A000

I upgraded to Windows 8 a couple of weeks ago and have so far had no problems, but with the release of the updates a few days ago I started to suffer the same problem as it seems a lot of others are, only mine seem more extreme in that it's not just KB2769165 and KB2770917 that I cannot install, no update will install no matter how I try.

 

Background information and configuration:

  • Upgraded from Windows 7 Home Premium to Windows 8 Pro
  • Dell XPS 8200 Desktop
  • Core i7 2.8GHz
  • 256GB Samsung 830 Series SSD
  • 2 x 250GB Seagate HDD in R1 mirror using Intel ICH10 controller and Intel RST driver
  • 16GB RAM

Symptoms began with the batch of updates released mid November, some of these installed correctly - KB2727528, KB2729462, KB2737084, KB2761226 & KB2769034 - but two refused to install - KB2769165 and KB2770917.

No matter how often I try to install these updates, installation fails and I see one of two error messages, either 0x800F081F or 0x8024A000.

Error 0x8024A000 is seen when I run Windows Update after running the troubleshooter and the troubleshooter claims to have fixed the problem, if I click the 'try again' button, Windows Update will repeat the attempt to find updates and will succeed in finding them the second time, however, the download starts but then fails at 5% with the 0x800F081F error.

Whenever I run the Windows update troubleshooter it will claim to have found and fixed:

  • "Potential Windows Update Database error detected 0x800F081F"
  • "Windows Update components must be repaired"

It will repeat this no matter how many times I run the troubleshooter, even if I run it twice immediately in succession, which suggests this is either a standard fix or that the problem persists and the troubleshooter is not fixing it.

What I have tried to date (each one individually and in combination):

  • Stop Nalpeiron license service
  • Stop all non-Microsoft services and disable all startup items (using msconfig)
  • Manual install of updates downloaded from MS download rather than using Windows Update
  • Disable Norton 360 security and AV
  • Uninstall Norton 360
  • Run the System Update readiness tool as described in http://support.microsoft.com/kb/947821?wa=wsignin1.0

Whatever I try, I end up back in the same place - unable to install any updates and with the same error(s) reported by the Windows Update troubleshooter.

stuartpeacock say:

Well, I'm able to answer my own question now, and not so long after posting it either.

After a lot of faffing around all morning with this issue I have finally resolved it, with the help of a combination of solutions proposed by others and some other ideas contributed by me.

I believe that a significant contributor to the solution has been the System Update Readiness Tool - DISM - and in particular the use of it proposed by DocSlax in this thread on 19th November 2012: http://answers.microsoft.com/en-us/windows/forum/windows_8-windows_update/cant-do-any-windows-8-update-error-code-800f081f/5cf38679-a6bd-4ca6-ab20-da6cc39058c6?page=5

For anyone else experiencing this issue, I will summarise what I found to be the fix for me, this may or may not work for others, from what I've seen there are a variety of similar issues and fixes and what works for one person does not seem to work for others. However, as I seem to have had what appears to be one of the worst cases of this problem with all the symptoms reported by others and none of the proposed fixes working, then I hope that the fix described below will reliably fix the issue for others too.

I suggest you print this out and read it through once before starting as you'll need to refer to other posts and KB articles.

  1. Run the Windows Update troubleshooter (in admin mode) and let it do its thing, I'm not sure this is strictly necessary as I suspect that DISM will do much the same thing.
  2. Uninstall all Windows updates installed to date. I believe this to be significant as I suspect that it is an earlier update that is preventing the installation of these most recent ones. You only need to uninstall the Windows ones, not the Office or others.
  3. Restart only after uninstalling all these, you don't need to restart each time you are prompted to.
  4. Run DISM as described here: http://support.microsoft.com/kb/947821?wa=wsignin1.0
  5. Clean out the failed updates using the technique described by DocSlax in the post mentioned earlier. Below are three lines of batch file that should do the job for you. These are provided as-is with no warranty and you are responsible for any problems you experience as a result of using them, not me!
    for %%F in (c:\windows\servicing\packages\package_?_for*.cat) do DISM /Online /NoRestart /remove-package /packagename:%%~nF
    for %%F in (c:\windows\servicing\packages\package_??_for*.cat) do DISM /Online /NoRestart /remove-package /packagename:%%~nF
    for %%F in (c:\windows\servicing\packages\package_???_for*.cat) do DISM /Online /NoRestart /remove-package /packagename:%%~nF
  6. Restart the PC again (probably not necessary but just to be sure)
  7. Stop the Nalpeiron licensing service if you have it running (if you have any Adobe product other than Acrobat Reader then you almost certainly do)
  8. You may also wish to stop all other non-core services and startup items using MSCONFIG as described elsewhere, though I found this not to be necessary. If you do then you'll need to restart your PC again and stop Nalpeiron if it's running.
  9. Run Windows update (from control panel) and tell it to check for updates
  10. You may experience an error 0x8024A000 (I did), if so just click 'try again' and it should proceed correctly the second time.
  11. You should see a message that says something like 'nn Important updates are available', click on this to see which ones they are.
  12. Any updates you have previously hidden will reappear so

     

    With any luck your PC will restart and all the updates will be applied cleanly.
