This has to be done in the registry, in the ProgId key for that file type. If you've used the Registry Editor before, and comfortable with editing the registry, read below:

For example, for .txt (which is mapped to ProgId "txtfile"), you change the (default) value here:

HKEY_CLASSES_ROOT\txtfile

from "Text Document" to "Text File"

And rename "FriendlyTypeName" to "FriendlyTypeName-"

It now shows as "Text File" in Explorer.

But if "Open with" was used to associate the file type with other program, say Notepad++ you need to edit the appropriate ProgId.

rick rogers say: Hi,

You don't "turn off" safe mode once you're in it, you have to reboot into normal mode. Safe mode loads a limited part of the operating system allowing it to start when normal mode fails. It sounds like your actions corrupted the user profile. When this occurs, the only real solution is to create another one for yourself. This you can do in safe mode, then restart and log into it. From there, you can recover any data stored in the old one prior to removing it (there's no point in keeping it as it is no longer useful).
