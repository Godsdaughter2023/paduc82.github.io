When you 'move' a message from one folder to another in WLMail, all that happens initially is that the message database records a new location for the message. The message file itself may not be moved immediately, depending on how busy the program is. When you select a folder to view, the program displays in the message list all the entries from its database that have that folder recorded as their location. If you have the reading pane visible, or you open one of those messages, the program will dig into the message store to retrieve the contents of the message from its .eml file and display them.

The problem you see occurs when the indexing process is incomplete. The program can't find the message content and shows either a blank message body or, in some circumstances, the body of some other message. It's not clear why this sometimes happens, but the most likely causes are
(a) the program is too busy doing other things like polling for new messages on the mail server, or
(b) security software (e.g. anti-virus) is slowing up the process by insisting on intercepting the files involved and scanning them before it allows them to be written to disk.

To prevent it from happening, or at least make the problem less frequent, click Work offline before moving messages around. This will minimize problems caused by (a).
