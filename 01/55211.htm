a. user say:

It is extremely unlikely that the svchost process is the cause of a problem on your system - but sometimes folks think that becasue they don't understand how it works - or why do they see so many running in Task Manager.

If you were to look in Task Manager, Processes tab you would see several to many svchost Processes running and that is normal:

Behind/Under each of those svchost Processes important Windows Services are running.  There might be just one Service running or there might be several Services running under each svchost Process so it is usually not a good idea to terminate them.  Each svchost Process is "hosting" some Windows Services.

If you were to right click one of the svchost Processes and choose Go to Service(s) you would see what Windows Services are being hosted by that particular process.

It WOULD be a problem is one of the svchost Processes seemed to be running amok consuming large amounts of CPU or memory but as you can see in the example most of the time the svchost Processes are consuming zero CPU so that is a good thing.

What Marque Lor the Microsoft engaged "expert" should do next is demonstrate some expertise by being sure the system is configured to generate crash dump files, prompt for some fundamental system information, and then you upload some recent crash dump files where they can be analyzed.

A hint from the screenshot you posted shows a STOP error of 0x000000F4 and those are usually hardware problems.

7)      A general reboot of the XB1 seems to fix most issues, but why? WHY should I ever have to reboot if I can’t see others online, can’t join a party, can’t join a game, etc.? “Hold while I reboot m console” is a common theme with us.

malkeleah say: Your dad apparently just renamed his user account with your name. As you have discovered, you cannot change a user account by just changing the name. Just changing the name will make a cosmetic change only and not affect the underlying files/folders in the profile. If this is important to you:

1. From an administrative account, rename the account to what it originally was. Then create the new Standard* user account named the way you want and at least one extra Administrator account. Call the latter "CompAdmin" or "Tech" or the like.
2. Log into the new Standard user account once so the profile files/folders are created and then log into CompAdmin.
3. Now go to Control Panel>System>Advanced>User Profiles>Settings**
4. You will see where you can copy user accounts. Copy the old account to the new one from there. Use the Browse to select your new account (C:\Users\newaccount).
5. Log into the new correctly named account and make sure all your stuff is there and that it works the way you want. When you are satisfied, you can log out and into CompAdmin and delete the old user account from Control Panel>User Accounts.

*Running as a Standard user is best practice for security purposes and will help protect your computer from infection. You don't want to run as an administrator for your daily work. Keep CompAdmin for your administrative account and use it only for elevation and emergencies.

** If you don't want to bother copying the old misnamed account settings, you can skip this bit and just copy your data to the new user account instead.

If you wish to log into your new, correctly-named account automatically, do this:

Start Orb>Search box>type: netplwiz [enter]
Click on Continue (or supply an administrator's password) when prompted by UAC

Uncheck the option "Users must enter a user name and password to use this computer". Select a user account to automatically log on by clicking on the desired account to highlight it and then hit OK. Enter the correct password for that user account (if there is one) when prompted. Leave it blank if there is no password (null).
