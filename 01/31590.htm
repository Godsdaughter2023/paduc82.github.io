why does it take so long for disk cleaner to calculate - Microsoft Community

gracecreel say:

why does it take so long for disk cleaner to calculate

am running windows xp;  the calculating window sits there forever;  i don't know how to fix it and don't even know if it is broke.

a. user say:

If you mean the XP "Disk Cleanup" seems to hang and take forever, I offer you this advice:


When you run Disk Cleanup, it calculates how much space you could save on your HDD if you choose to compress "old" files. Nobody seems to have a good answer for what XP thinks is an old file though... and with modern technology there is not going to be much of an improvement in your disk space and in fact, your system may run slower since it may need to uncompress these files when they are needed later.

It is just not a good idea to let XP compress your "old" files (whatever old means seems to be unspecified).

If you have a million files on the volume that XP thinks might need compressing and it is going to check them all, it could take a long time to figure that out and your system could appear to stop responding.

Sometimes users interpret this as a problem with their system (and power off or reboot), but Windows is just doing what you told it to do.

It may also may really be hung since MS also tells you how to stop this check for file compression in a knowledgebase article (maybe they don't like it either).

If you have no interest in compressing old files and/or it just takes too long, you can have Disk Cleanup skip that calculation entirely (a good idea).

You'll be cleaning up your system in no time when that check for files that might need compressing is removed.

http://support.microsoft.com/kb/812248

jeff kreider say:

Accessing Another User's Files Windows 8.1 Inappropriately

I have found a problem with the security of Windows 8.1.  When I open Windows Explorer and attempt to access another user's personal folder (call it Ted), I'm getting the expected 'User Denied' error message.  However, If I do a search say *.pdf, the search will find files in Ted's personal folder and allow me (Jeff) to open those files.  I have checked the security settings on the folders and only administrators, the logged in user, and System have access to the personal folder and folders within the personal folder.  I am logged in as a Standard User.  I do not understand how I, as a Standard User, can open another user's files in their personal folder.  How do I fix this?

Jeff

a. user say:

Hi Jeff,

I see that this is your first post in this forum, welcome to the Microsoft Community.

I tried reproducing the issue you have mentioned by searching for a pdf file in the administrator account from a standard account. I typed ‘.pdf’ in the search box in File Explorer and the search results didn’t show the pdf file in the administrator account. So, this isn’t a security issue with Windows 8.1.

The issue could occur in your computer due to some incorrect security settings or due to some corrupt system files. I would suggest you to try these steps and check if it helps:

Method 1:

First, I would suggest you to refresh the security settings for that user profile and check again. Refer to these steps:

  1. Log in to the administrator user account and open File Explorer.
  2. Navigate to ‘C:\Users\’ (where C:\ is the driver where Windows is installed) and right click on the ‘ted’ folder and select ‘properties’.
  3. Click on the security tab. In the ‘Group or username’ field, highlight the ‘Jeff’ user profile to check its permissions.
  4. Make sure that all the boxes in the “allow’ column is unchecked for that user profile.
  5. If there is any check boxes selected already, click on the ‘Edit’ button and unselect them, and save changes.
  6. If none of the boxes were selected already, I would suggest you to click on the ‘Edit’ button and check the box against ‘Full control’ to give full permission. Save the changes. Restart the computer and then visit the same settings.
  7. Now, uncheck the box against “full control’ and save the changes. This way the security settings would be refreshed.

Method 2:

If the issue persists, it is possible that some of the system files are corrupt. System File checker (SFC) scan is done to check if there are any corrupted system files that could be causing this issue. Refer to this article:

http://support.microsoft.com/kb/929833/en-us

Try these steps and let us know the results. We’d be happy to help if you need any further assistance.
