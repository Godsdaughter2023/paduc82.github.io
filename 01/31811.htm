The error code 0x80073712 signifies 'ERROR_SXS_COMPONENT_STORE_CORRUPT'. The component store has been corrupted, the update problem is only a symptom of the underlying system issue.

Using the error code or description as a search term in your favorite search engine will provide you with a wealth of information on causes and fixes for this problem.

All of the attached DMP files are of the CRITICAL_PROCESS_DIED (ef) bug check.

This indicates that a critical system process died.

3: kd> !process ffffe0000427e240 3
GetPointerFromAddress: unable to read from fffff80047bc3000
PROCESS ffffe0000427e240
    SessionId: none  Cid: 0248    Peb: 7ff630f59000  ParentCid: 0240
    DirBase: 105841000  ObjectTable: ffffc00001a4dd80  HandleCount: <Data Not Accessible>
    Image: csrss.exe

^^ The process is csrss.exe which is the user-mode portion of the Win32 subsystem; Win32.sys is the kernel-mode portion. Csrss stands for Client/Server Run-Time Subsystem, and is an essential subsystem that must be running at all times. Csrss is responsible for console windows, creating and/or deleting threads, and implementing some portions of the 16-bit virtual MS-DOS environment.

In other words, something is causing corruption and is driving this process to unexpectedly be killed. Detective work will be necessary, as with most if not all 0xEF bug checks.

This issue may occur if the Enable IEEE 802.1X authentication for this network check box is selected. If you reestablish the wireless network connection, it disconnects again after a short time.

To disable IEEE 802.1X authentication, follow these steps:

Press "Windows + R" key on the keyboard to open Run window.

Type ncpa.cpl and then hit Enter.

Right-click your wireless network connection, and then clicks Properties.

Click the Wireless Networks tab.

Under Preferred networks, click your wireless network, and then click Properties.

Click the Authentication tab, click to clear the Enable IEEE 802.1X authentication for this network check box, and then click OK two times.
