STOP errors of 0x000000F4 are usually hardware problems.

If the first parameter is ...0003 it is even more likely to be a hardware problem.

If the SSD doesn't show up in the BIOS or boot menu unless you cycle power that has nothing to do with Windows so it still points to hardware problems.


That is 3 strikes for hardware problems.

Is this a new system or a new SSD in a system that used to work fine?

Setting up paging files and crash dumps on an SSD need special attention so you need to pay attention to the suggestion from ZigZag about how to set things up (firmware, etc.).

It is possible to generate BSOD crashes on demand so after you make your adjustments you can test by initiating a crash manually and seeing if a dump gets generated.  If no dump is generated go back to the drawing board and keep adjusting until you do get a crash dump file - that way you don't have to wait all day to find out if dumps are being created or not.

It is tricky to pinpoint hardware problems from a crash dump.

You can use NotMyFault to generate a crash dump on demand and keep using it until you succeed in generating a crash dump file - then you know you have things set up properly.:

https://technet.microsoft.com/en-us/sysinternals/notmyfault.aspx?f=255&MSPPError=-2147217396

This is good for practicing/learning about debugging crash dumps and to test to be sure your system is able to generate a crash dump.
