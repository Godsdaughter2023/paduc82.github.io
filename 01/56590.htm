ruwim.b say:

This issue is caused by a bug in Skype for Windows 10 App (Application version 14.38.138.0). In this Skype application, on the Open command a copy of the image file is created in the \Downloads\All\Skype folder. However, this copy does not have any file extension, so Windows doesn't know what application to use to open this file.

 

This bug is not present in Skype for Desktop application, where a copy of the image file with the correct extension is created in the %temp% folder, and this file opens with the default image application.

 

The currently latest 8.38.0.138 version of Skype for Desktop application can be installed using this direct download link:

 

https://download.skype.com/s4l/download/win/Skype-8.38.0.138.exe
