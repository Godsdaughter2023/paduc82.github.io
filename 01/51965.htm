This usually means that the program simply can't find the message to display its contents. This might be because (a) it hasn't yet been downloaded from the server, or (b) it has already been deleted but the message database didn't record the deletion.

(a): check the envelope icon in the message list. Here are the three commonest ones:


If it hasn't yet been downloaded, there may be something about it that's interfering with the download. Locate in in webmail, remove it from the inbox and deal with it.

(b): This indicates a minor corruption in the database index. You can usually correct it by selecting the message in the message list and pressing Shift-Delete on your keyboard. This bypasses the Deleted items folder and permanently gets rid of both the message and its entry in the index.

A bit of background

The entries in the message list are taken from the message database index. When you select one of them for viewing, the program retrieves the body of the message from the location recorded in the database. If it doesn't find the message there, it may display the body of some other message or even none at all. So what you're seeing is a symptom of a corrupt database index.

There are many causes of corruption like this. A common one is not allowing the program to complete its housekeeping after closing it down, e.g. by shutting off the computer while that housekeeping is underway. Power cuts can also have this kind of effect, and uncharted causes include interference from security software (e.g. anti-virus) and - particularly - performance-enhancing software.

Give the program plenty of time to index messages, e.g. after moving a lot of them around or copying or deleting messages in bulk. Wait five minutes after closing down the program before logging off Windows.

You may find that it all comes right if you close down the program and wait a few minutes before re-launching, repeating the process if necessary.
