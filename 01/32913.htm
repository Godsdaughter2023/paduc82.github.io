    Windows Update Reboot Infinite Loop - Microsoft Community

    johnbottiger say:

    Windows Update Reboot Infinite Loop

    Recently, I keep getting these Windows Update pop-ups indicating that I need to reboot my computer.  I checked to see what updates are being installed, but Windows Update won't disclose any.  I talked with my system administrator, and we both discovered that an update KB3033929 was the culprit.  From what we researched, there was supposed to be update KB3035131 installed PRIOR to KB3033929...KB3035131 was never installed while KB3033929 was.

    I went in and removed Windows Update KB3033929 and rebooted.  When I opened Windows Update, my intention was to manually install update KB3035131 first, then reinstall KB3033929 as per the Security Bulletin (which I believe you published).  However, I performed the following tasks in Windows Update with the respective results:

    • Clicked "Check for updates":  Windows Update displays the window "Windows Update cannot currently check for updates, because you must first restart the computer so that a previous installation can be completed."
    • Clicked the "View update history" link:  No update history displayed.
      • Clicked the "Installed Updates" link, and searched for KB3033929...confirmed it was removed.
    • Most recent check for updates:  Yesterday (12/22/2015) at 2:19 PM
    • Updates were installed:  12/17/2015 at 8:50 AM (Failed).  View update history (which I already did in previous step).

    Is there something else that is causing this infinite loop?  I thought that getting rid of the KB3033929 update would solve the issue...it didn't.  Please advise as to how I can install updates again without having to reboot again and again and again.  This is disruptive to my ability to get my tasks completed at work.  I need to be able to specifically load update KB3035131 successfully prior to reinstalling KB3033929 as per your Security Bulletin.

    Thank you very much for your attention to this issue.

    V/R,

    John J. Bottiger

    johnbottiger say:

    UPDATE:  I had an online chat with a Microsoft representative who was very helpful.  She told me that the next step was to do the following (from a link she sent me...the solution was provided by "kieferschild"):

    • Click/press the Start button, and type "regedit" in the search field to launch the Registry Editor tool.
    • In the left pane with the directory tree, go to HKEY_LOCAL_MACHINE -> Software -> Microsoft -> Windows -> CurrentVersion -> WindowsUpdate -> AutoUpdate -> RebootRequired
      • If this key is displayed (i.e., not deleted, data displayed when selecting this key), then that indicates that it is corrupted as Windows Update is supposed to automatically remove the key upon completion of Windows Updates.  Delete this key.
    • Close regedit, and reboot the system.

    After performing these steps, I was now able to successfully pull available updates from the "Check for Updates" link.  Given that KB3033929 was on the list, but not KB3035131, I checked every one of the updates except for KB3033929 for downloading.  Restarted PC when Windows Update completed the downloads, and now I no longer have that infinite loop with the Windows Update restart process.

    Thanks to everyone involved...my system administrator at work, the technical rep at Microsoft, and all who have read my question.  Have a safe and happy holiday season!

    V/R,

    John J. Bottiger
