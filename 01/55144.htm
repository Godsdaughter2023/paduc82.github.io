This issue may occur if the user profile folder is manually deleted. A profile that is manually deleted does not remove the security identifier (SID) from the profile list in the registry. If the SID is present, Windows will try to load the profile by using the ProfileImagePath that points to a nonexistent path. Therefore, the profile cannot be loaded.
To resolve this issue, see here: http://support.microsoft.com/kb/947215 or http://www.vistax64.com/tutorials/130095-user-profile-service-failed-logon-user-profile-cannot-loaded.html
