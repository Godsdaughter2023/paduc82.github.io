I don't quite understand your answer to No. 5, particularly "The hardware was not recognized by Windows, it requires manual booting without Windows handling the device connection to the router."

In Windows, there are basically two ways that a wireless adapter can be controlled.  Either Windows can control the device using a feature called "Wireless Zero Configuration" or a utility provided by the adapter manufacturer.  Generally, you want one or the other -- not both active at the same time.

If the icon that you reference is a "Z" in a box (http://tinyurl.com/cjn3mje) then the Zyxel utility is in charge.  That's perfectly OK (and probably provides more features than Windows WZC), but I'm not familiar with the details of using the Zyxel utility.  If you identify the particular model of your adapter, I can see if I can find it's User Manual online and explain the steps you have to take
Whether you use the Zyxel utility or Windows WZC, the basic steps you need to take are the same:

1.  Search for available wireless networks.
2.  Identify your network in the list displayed (this is your SSID or "Network Name")
3.  Click "Connect"
4.  Enter the encryption password when prompted.

Have you done the above?  What happened?


Windows WZC automatically saves the connection information (SSID and password) once you successfully connect.  I don't know if the Zyxel utility does this automatically or if you need to explicitly save a "profile" with the data.

Whichever utility you use, you need to know both your router's SSID and the encryption password.  The Verizon tech who installed your router should have provided you with this information.  It may also be written on a sticker on the bottom of the router.

If you don't have -- or have lost -- that information, you can find it out by connecting to the router.  Preferably you should do this using an Ethernet cable -- assuming that you can get your computer close enough to the router.  You will need to know the userid/password to log into the router.  Again, the Verizon tech should have provided this information to you.

Can You Upgrade From Windows 7 Release Candidate To The Final Version

The RC is a test version and is much different under the hood then the final RTM of Windows 7, it was not designed for continued use or for upgrading. For info about the Windows 7 Release Candidate expiring and what to do:
http://windows.microsoft.com/en-us/windows7/release-candidate-Windows-7

Sorry, Microsoft does not support an upgrade from Windows 7 Release Candidate (RC) to the final version. You can backup your files and settings with Windows Easy Transfer before reinstalling XP or Vista, and then move to the final release version of Windows 7 if you wish. You will need to reinstall your programs.

The Windows 7 Upgrade software requires XP or Vista to be installed first, if you have a qualifying version of Windows (XP or Vista) you can install it on your computer and buy an Upgrade version.  The Upgrade software doesn't allow an upgrade version of Windows 7 on a blank hard drive to be activated.  The installation procedure does not ask you to insert a Windows disc in the drive for verification, the Upgrade disk software requires the actual qualifying operating system to be installed.

If you do not have a qualifying Windows operating system with a genuine license, then you cannot use the Upgrade version of Windows 7 - you would need a "Full version" Windows 7 license.

In summary:

1. The Upgrade software needs a qualifying Windows operating system (XP/Vista) to be installed.
2. The qualifying Windows operating system must have a genuine license (product key) and it must be activated. If you don't have a qualifying license you will need a Full version.
3. To move from XP/Vista to 7, boot to the qualifying Windows desktop, insert the Windows 7 Upgrade disc in the DVD drive.
4. When the setup menu appears, select Custom (advanced) to initiate a clean install procedure, or Upgrade if you are upgrading.

tryx3 say:

There are plenty - such as making a Ubuntu boot disk.


There are also Windows tools for this - by booting from the System repair disk or installation disk you can get to a Command prompt  and you can then hunt around for your files & xcopy them to an external drive.


I do not know if an upgrade disk can be used for this task but I think it can.  So try booting from it - that way you will be able to rescue your files without making any new disks.  If you can reach the Command prompt [step 5 below] then it can handle everything else.


To rescue your existing data,

1  boot from the System repair disk or installation disk then

2  select keyboard & language,

3  [on installation disk you then also select - Repair your computer],

4  select Use recovery tools... [and note the drive letter being used for the Windows drive before clicking on Next as it might not be the normal drive letter] then

5  select Command prompt. 


As soon as you reach the Command prompt, enter Notepad.  Then use the normal WinKey left/right arrows to show both windows at the same time.  In the Notepad, File open dialog, select *.* for the files and then click on Computer then the appropriate drive.  You can then use Notepad as a rudimentary form of Windows explorer so you can look around to figure out what to copy & where to copy from & where to copy to then put in the relevant xcopy / robocopy / etc commands in the Command prompt window[s] to copy your files to an external drive.  You can have multiple instances of Notepad & the Command prompt open at the same time - you might find this handy if you want to copy from many different places without getting yourself lost.

The responses you are getting for the C:\ drive indicate that there is nothing on it but you also referred to having been setting permissions on it & being able to run SFC so I do not understand the combination of symptoms.  There is nothing you can do within Windows 7 permissions that can account for not being able to see it in the "WinRE" environment that you have just booted to.


Assuming, of course, that you did not set any kind of drive encryption such as BitLocker.  If you did this then there is nothing that can be done without the password set up for it.


What follows is therefore no better than what I would try.  To be frank, I do not know enough about para 1 but think that it might help you - I simply do not know enough about that subject.  If Startup repair did not fix it then, with your combination of symptoms, I would skip the rest of para 1 and go straight to para 2 personally.


1  In the menu from which you selected the Command Prompt, try Startup repair to see if the problem is that the file system has been so corrupted that it cannot be read.  If this task seems to be trying to fix it but cannot do it all successfully [you'll see all this in its completion statement] then search the forum for bcdedit bcdboot bootrec to find one of the posts that seems to be similar to your situation.  I have never had to use these commands myself.  If the system becoming unstable message referred to the Master Boot Record being corrupted then one of the threads you'll find here should help you.  I have always thought that these tools only became necessary if Startup repair could not fix everything on its own. 


I do not suggest that you run these powerful commands without better guidance than I can offer.  You can find out more about them in the Command prompt.  Enter bootrec /? and you will get this:-


BootRec is a command on the installation disk - this is its help file

Repairs critical disk structures. The following commands are supported:

/FixMbr -- writes the master boot record of the system partition using the master boot record compatible with Windows 7.  This operation does not overwrite the existing partition table.

/FixBoot -- write a new boot sector onto the system partition using the boot sector compatible with Windows 7.

/ScanOs -- scan all disks for installations compatible with Windows 7 and display the entries that are currently not in the boot configuration store.

/RebuildBcd -- scan all disks for installations compatible with Windows 7 and allow the user to choose which to add to the boot configuration store.


You can get the text for the others by entering bcdedit /? and then bcdboot /?  See also TechNet Bcdboot  TechNet Bcdedit [I could not find a BootRec page in TechNet].
