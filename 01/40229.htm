That is nonsense. POP3 is a protocol for downloadiing messages from a mailbox to a mail client. SMTP is a protocol for sending messages to a mailbox.

¡firedog say:

Gmail only allows the use of port 465, the standard port for authenticated, secured SMTP. Hotmail allows the use of port 25, the standard SMTP port, and 587, the standard port for authenticated SMTP. Hotmail does insist, though, on SSL (meaning secured - encrypted - transmission of credentials) on both ports.

Your French ISP is undoubtedly blocking access to SMTP servers other than its own on port 25. This is to prevent anonymous users from using their service to send spam. To get round the block, use port 587: right-click on the account name in the folder pane and select Properties. Make the port change on the Advanced tab.

rohn007 say:
The problem is that there is a file that cannot be easily moved placed at that 467GB location on the partition. If Windows Defrag does not / cannot move the file that would be for a couple of possible reasons.

Some programs place license files in specific locations. So they cannot be moved by the defrag tool.

The Windows installation process can scatter files all over the drive. If the file is being actively used by Windows, the defrag tool may not be able to close it to move it. In that case you would have to use a defrag tool that boots from a DVD/USB drive so that Windows is not being run from the C: drive.

In Disk Management, to the the help and look for the subject they suggested: "Shrink a basic volume".

The other is to find the "Event Viewer" in start menu, start it.
click on the "Windows Logs" category
click on the "Application" sub category
Click on Action menu > Find
Search for "Defrag".
scroll through the logged entries to see if you can identify the specific file causing the problem.

Take action based on the specific file causing the problem.

arminatorx say:

[quote user="Reno Vega"]Cannot connect to your DHCP server[/quote]

How do you connect your Xbox One to the modem? A cable connection or wirelessly?

The purpose of DHCP is "only" to provide the Xbox an internal IP address. Are you able to manually give your console an internal IP address from the IP range of your own network?

You can try the following: On your PC open up a command prompt (on Windows use Windows Key + R and type "cmd" in the small window and press OK).

In the black window type the following command:

ipconfig

You should get a list of IP adresses. One of these lines should show you the address for your "default gateway". In most cases it's something like 192.168.XXX.YYY. Lets say for example the default gateway is 192.168.1.1

Take the first three numbers and think of a number between 1 and 254. Let's say you randomly thought of 123. On your PC enter the following command:

ping 192.168.1.123

If there is no response, manually set the IP address of your console to 192.168.1.123. Subnet mask to 255.255.255.0, Gateway and DNS to 192.168.1.1

Should your router provide a different IP range (for example 192.168.178.YYY) change the third octet accordingly (i.e. 192.168.178.123 for the console IP and 192.168.178.1 for the Gateway. Subnet will always be 255.255.255.0)

Does that work?
