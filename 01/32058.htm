I know this question has been asked numerous times, but I have done these steps and still can't get Defragmenter to work.  My Disk Doctor says I have no problems and my Defragler program works -- but I do have a "dirty" C: drive.  I have Windows XP Home Edition and do not know how to run a XP Recovery Console.  Is there an easy way just to get the Defragmenter to work?...when I run Chkdsk.....it goes through its steps and then closes and the problem is still there.  Defragmenter just stopped working, my computer is just running fine.  Please help.

a. user say:

Hmmm...  It should only take a few minutes for ImgBurn to create the bootable XP Recovery Console CD, so maybe one of the pieces you downloaded (either the xp_rec_con.iso or IgmBurn install) did not go quite right.

You want ImgBurn to play a happy tune when it is successful and an "Oh no!" message when it fails, but it should not take hours to burn that little ISO file.  You might try another system.  You can always uninstall ImgBurn when you are done.

I say "be patient" when running the chkdsk /r since sometimes it appears to be stuck (or even go backwards), but it should finish eventually.  Depending on the size of the volume, the amount of data on the volume, the speed of your system and what the chkdsk finds to do, you could be looking at several hours. 

Look at these words right from the mouth of Microsoft:

If you use chkdsk /f on a very large disk (for example, 70 gigabytes) or a disk with a very large number of files (for example, millions of files), chkdsk might take a long time (for example, over several days) to complete. The computer is not available during this time because chkdsk does not relinquish control until it is finished.

At the time of that writing, I guess a 70GB drive was "very large" but that is quite modest by today's standards.

Now with today's technology, it is unlikely to take days, but it could still take a long time!  If your HDD is afflicted with a hardware issue and dying, it may never finish or be unable to fix things.

When XP starts it will automatically check every volume to see if it is dirty and will try to run chkdsk of the afflicted volume(s).  Sometimes that is good enough to clear the dirty bit, sometimes it is not.  You will see a message at startup if the volume needs to be checked for consistency and XP will do the best it can.

That's exactly what the message is telling you:

Disk Defragmenter has detected that Chkdsk is scheduled to run on this volume: (C:)

You can query the dirty bit on a volume from the command prompt window.

Click Start, Run and in the box enter:

cmd

Click OK to open the command prompt window.

To query the dirty bit on drive C, type:

fsutil dirty query C:

Sample output:
Volume C: is dirty
Volume C: is not dirty

Type 'exit' to close the command prompt window.

If you are suspicious of your HDD and think it might be failing, now would be a good time to backup your personal data (documents, pictures, music, etc.) to some external location - either a separate USB attached drive or some cloud on the Internet.
