goreski say:

.I found this somewhere and it worked for me...  It really works... !!

For Windows 10.  Gets rid of the old deleted program in the Notifications and Actions list.... which I think stays there because of the old Win 7 or Win 8 data from the Win 10 upgrade.... which for me, was the problem.

Others have just gone partway... just deleting only  the IconStreams and PastIconsStream (works if you never upgraded from Win 7 or 8 - or if you installed the now deleted program AFTER upgrading to Win 10), but for me the dang deleted program 'icon' stays in the Notifications and Actions... because it was in the Win 7 or 8 data area... 

You need to delete ALL keys in TrayNotify, not just some of them to get rid of program icons from Win 7 or 8 !

Note: Using Regedit is always a bit dangerous.... be VERY CAREFUL or you can screw up your

Operating System easily !!!   Do NOT do this is you are not sure what you are doing.

1. Open the Registry Editor  (Run: Regedit)

2.  Browse to:
    HKEY_CURRENT_USER\Software\Classes\Local Settings\Software\Microsoft\Windows\CurrentVersion\TrayNotify

3. Delete the following KEYS in this folder:  (Windows 10)
    IconStreams
    LastAdvertisement
    PastIconsStream
    PromotedIconCache   (this I think is old Win 7 and 8 Icon data and likely the culprit)
    UserStartTime

Exit Regedit.

4. Open the Task Manager.   'Control/Alt/Delete' on keyboard
 
   RESTART every Windows Explorer or Explorer process - you may have more than one running !
   (this is necessary because Windows Explorer has a backup copy of the keys in the RAM and writes it back to the registry on shutdown - but NOT restart)

The LAST Windows Explorer running when Restarted will black out the screen background for a second, then restore it.... that is when you know you've gotten all Explorer processes restarted.

 

5.  Go check Notifications and Actions to see if the Deleted Icon is there... you should have Nothing listed now - but as you use your current programs the Notifications area will be restored...

zaki j say:

Hi mbnoimi,

Thank you for using Microsoft Answers Forums!

To make a relative path shortcut here is a simple workaround. Make a shortcut to cmd.exe which executes the start command to open the current directory. Then set the cwd in the
shortcut properties.

Follow these steps:

Make a new shortcut to: cmd.exe /c start.

Now right-click on it and get properties of the shortcut (shortcut tab).

The full shortcut path should have been set to this:
%windir%\system32\cmd.exe /c start .

Set the "Start in" directory (i.e. cwd) to be a relative path:
(Example for a shortcut on the Desktop

)
...\SendTo

Click on "Change Icon..." and set "Look for icons in this
file:" to:
%SystemRoot%\system32\SHELL32.dll

and press enter. The standard folder icon will probably be in the
lower right-hand corner.

Now you can copy this file and just change the "Start in:" directory
to make new relative shortcuts.

NOTE: The above steps are just an example hence try this at your own risk. And check if it works.
