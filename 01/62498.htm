a. user say:

It would help to know the exact error message.

Putting your computer in a Clean Boot state will not solve this problem - that is just another wild goose chase.

Here's the deal...  Microsoft has several flavors of the Visual C++ Runtime Distribution files.   These installations only contain runtime files and are not the entire C++ development package - just the runtime files that some third party applications need.   I have 4 different versions installed (and their associate updates) and each gets put into it's own folder on your system somewhere under here:

C:\windows\WinSxS

Depending on the applications you have installed, you could need several versions of the libraries installed and many folders under your c:\windows\WinSxS folder.  Windows SideBySide is a technology that allows different versions of files and libraries with the same names installed on your system at the same time. 

Since you can't have two files of the same name in the same folder at the same time, applications you install will dynamically pull the files they need from separately installed folders.

Different applications need different sets of files, so XP puts them in separate folders for you. 

For example, .NET Framework products might require one version of a file, Visual Studio another and Adobe products might require even another version of the file(s) with the same name, so they need to be installed into separate folders and XP will take care of that for you.

If the Pinnacle people can not tell you what C++ distribution you need (that would be pretty poor if they don't know), and if you already have the C++ 2008 installed and if Pinnacle used to work, then there is no harm in uninstalling and reinstalling the C++ 2008 files and seeing if that fixes the issue.  Look in your Add/Remove Programs to see what versions of the C++ installations you have installed.

If you upgraded Pinnacle lately they may have added some things that need newer versions of the C++ files or maybe they need some files from the 2008 and some files from the 2010 installation that you can download from here:

http://www.microsoft.com/download/en/details.aspx?id=5555

I think I would see what Pinnacle tells you they need to have installed and/or uninstall your C++ 2008 and if that doesn't work, install the C++ 2010 (the latest version).  You may or may not need both installed depending on what third party applications you have installed.
