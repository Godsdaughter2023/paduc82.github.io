stan-logue say: "The system image restore failed."    Windows 7

Users are reporting problems when using System Image to restore hard drive partitions.
These two examples might help clarify how System Image works.  And it really does a great job.
 
The C:\ partition (70 GB, 25 GB used),  one of 8 partitions in a 320 GB hard drive,
was backed up into a 500 GB USB hard drive, using Windows 7 "system image".
The destination for the restored partition C:\ was a 120 GB hard drive.
The image restore failed to start.   Error: "The system image restore failed." Why?
Even though that C:\ partition was small, its backup had a 320 GB "allocation" assigned.
That 320 GB size could not fit into the 120 GB drive's available space.

If all but the C:\ partition were deleted, reducing the “allocated” space to about 50 GB,
and that C:\ partition was backed up, then that backup can be restored into a 60 GB hard drive.

The overall space allocated within the source hard drive that was backed up must
be less than the available space in the hard drive that is to receive the backup.  
It is not the hard drive's "label" size, but its allocated size, that matters.

Obviously such special backups can't be created after the hard drive has "crashed".
So, do all the recovery preparations in advance.

Resized the 8 partitions in the 320 GB drive, reducing unused space, making the
total space allocated for all less than 120 GB. Then backed up all 8 partitions.
Now the allocated space for that 320 GB drive backup is less than 120 GB.

That backup can restore all 8 reduced-size partitions, originally in a 320 GB hard drive, into a 120 GB drive.
The hard drive receiving the restore  can be new:  Unallocated, no partitions, un-formatted.
Because for "Choose additional restore options", you check "Format and repartition disks".
Then the restore will configure as needed.   The restore of all 8 reduced-size partitions succeeds.

Now you have a special full 8 partition backup of the 320 GB hard drive that can restore into smaller drives.

You could expand the reduced-size source drive's partitions back to their original sizes.

To resize / create partitions, change allocated sizes, use this excellent free program:

     MiniTool Partition Wizard Bootable CD 6.0
           http://partitionwizard.com/partition-wizard-bootable-cd.html

Get the boot-from-CD version. It can boot start the computer even if the hard drive is totally empty.

For anybody else that might happen across this thread with Google, here's what eventually fixed this:

1. Made a Windows ISO on a USB flash drive with the Microsoft tool

2. Ran an in-place upgrade of Windows 10

Get-AppxPackage -allusers Microsoft.WindowsStore | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"}

4. Reboot
