tokenfin say:

I had this on my computers too, and have found a solution.

The Anniversary Update version of Windows 10 has removed the previous option in Power Options. There is now a setting in Settings > Accounts > Sign-in Options. This setting should appear right at the top of that screen, above the Windows Hello option, and is called Require sign-in (choices: on wake, or never).

There is a bug. If the current local account has no password the above option is not displayed, this is annoying because users without passwords are the ones most likely to want the sign-in prompt removed. Microsoft might call it a feature, but the following simple workaround shows it really is just a UI bug.

Workaround:

Add a password to the local account, reboot.

Change the Require Sign-in option to never.

Remove the password by choosing Change Password and entering nothing.

Enjoy never having to click Sign-in after wake again.

I've done this on several computers now and it works every time.

a. user say:

What is your XP version and Service Pack?

Has System Restore ever worked properly in the past?

What is the issue you are having that you think System Restore will resolve (or are you just practicing)?  Remember - System Restore is not a time machine and also does not know or care about any of your personal files (documents, images, pictures, music, etc.).  However, System Restore certainly should work and if it doesn't you should see if you can fix it.

System Restore is a frequent target of malicious software since the malicious software wants to try to fool you into thinking that you need to perform a Repair Install or reinstall your XP completely when none of those things are likely to be necessary. 

If you have run the free versions of Malwarebytes and SUPERAntiSpyware to try to rule out a malicious software infection and also ruled out that it is not your antivirus/antimalware application interfering (it may be 'protecting' your system too much) and System Restore does not even work in Safe Mode, you may need to reinstall System Restore (not reinstall XP - just the System Restore part).  Sometimes this is necessary when SR is just so afflicted it will not work from malware, tinkering or other corruptions. 

Reinstalling SR will whack all your old Restore Points and then leave you with a fresh System Checkpoint when you are done.

Here's how:

Sometimes System Restore is so afflicted and none of your Restore Points will work or maybe you don't trust any of them (if your AV software reports they contain infections) it is sometimes best to just start all over and reinstall System Restore.  That will get you a fresh start.

When you reinstall System Restore, all of your current Restore Points will be removed and you must restart your computer when the installation is complete.  After the restart, one new Restore Point will automatically be created as a new System Checkpoint in your SR calendar.

If your system had a malicious software infection, it could be that some of the files in your old Restore Points are also infected so wiping out the old ones and starting fresh is often a good idea so you don't wonder if the old Restore Points are infected or not.

As you go through the reinstallation of SR, the reinstallation will ask you for some files as you go along and ask for your XP installation CD.

If you do not have a genuine bootable XP installation CD that is the same Service Pack as your installed Service Pack (most people don't) , you can change the path of where the installer looks for files one at a time as they are needed.  More on that later.

Assuming your XP is installed in the Windows folder on the C drive of your machine:

Browse to c:\windows\inf and right click the following file:

sr.inf

Choose Install

If you cannot see c:\windows\inf folder or the sr.inf file therein, configure your system to show hidden files and to show file extensions.

Here's how:

In Windows Explorer, click Tools, Folder Options, View and enable "Show hidden files and folders" and uncheck "Hide file extensions for known file types" then you should be able to see the c:\windows\inf\sr.inf file.  You can undo these adjustments later if it is your desire.

The installation may ask for files from the c:\windows\inf\386 folder.  That would be your genuine bootable XP installation CD if you have one and the Service Pack installed on your system needs to be the same as the CD you are going to use.  If you don't have that (most people don't), not a problem.

You do not absolutely need a genuine bootable XP installation CD to reinstall SR since the files must already be on your system some place if SR was ever working before.

If you are asked for a file and you can't find it where I suggest in my example below (it works on my system), you will have to search your hard disk and find where the files are on your system.

If asked for a file and you don't know where it is, leave the installation waiting and click Start, Search and search for the requested file and find it on your system.  If SR ever worked before, the file is already on your system somewhere.

When Search finds the requested file, switch back to the installation and enter the path to where you found the requested file and click OK.  The installation may ask for several files along the way, but you can find them all.

If you still can't find the files, let us know what the file is you need and some nice person can upload it to their SkyDrive and you can download it.

Here is how my system works when I reinstall SR when I pretend I do not have a genuine bootable XP installation CD:

When asked for:

sr.sys

Point the install to  c:\windows\system32\drivers

When asked for:

srclient.dll

Point the install to  c:\windows\system32

When asked for:

filelist.xml

Point the install to c:\windows\system32\restore (or c:\windows\ServicePackFiles\i386)

If asked for:

rstrui.exe

Point the install to  c:\windows\system32\restore

When asked for:

srframe.mmf

Point the install to  c:\windows\system32\dllcache (or c:\windows\system32\restore)

When the installation completes, you will get a message that you need to restart your computer.

Restart your computer and look at your SR calendar.  You should have one new System Checkpoint in your calendar.

Now is a good time to test your System Restore process from end to end.

The day you think you need System Restore is not the day to find out it doesn't work.

Reboot your system and with nothing running (no browser, no Outlook, etc. - just the basics), manually create a new Restore Point, reboot your system and see if you are able to successfully restore your system to the Restore Point you just created.
