Hi Jim,

The kernel power event ID 41 error occurs when the computer restarts or shut down unexpectedly. An underpowered power supply might also cause this behavior. This can also occur if your audio driver needs to be updated. To troubleshoot your kernel power event ID 41 error, you can perform the following steps:

  1. Update your audio driver. To do so, please refer to this link. If Windows doesn't find a new driver, you can try checking for one on your device manufacturer's website.

  2. Change your power settings since there might be some settings on your computer that are affecting the power usage. Here's how:

    A. Go to Control Panel > Hardware and Sound > click "Change when the computer sleeps" under Power Options > Change advanced power settings.  
    B. A Power Options window will appear. Look for Hard disk, then expand it. Set the "Turn off hard disk after Setting (Minutes)" option to Never.
    C. Look for Sleep, then expand it. Expand the "Sleep after" option, then set it to Never.
    D. Click OK.

  3. Turn off fast startup. It is a feature in Windows that helps your computer to start faster which is enabled by default.
    Here's how: Go to Control Panel > Hardware and Sound > Under Power Options, click "Change what the power buttons do" > untick the box beside "Turn on fast startup (recommended)" > Save changes.

Extra file created when copying files from Mac to PC

When I copy files from a Mac to my PC a duplicate file ofd the same name is created with the "_" in front of the file name. The file is not usuable though. I was told there is a utulity to locate and trash these files. Has anyone heard of it. Thank you.
malkeleah say: Yes, this is one of those Mac-to-PC annoyances. See if this helps:

http://support.apple.com/kb/HT1629

To configure a Mac OS X user account so that .DS_Store files are not created when interacting with a remote file server (this also means when you are sharing files with your PC) using the Finder, follow the steps below:

Open Terminal and at the command prompt type:

defaults write com.apple.desktopservices DSDontWriteNetworkStores true [enter]

 Either restart the computer or log out and back in to the user account.

If you want to prevent .DS_Store file creation for other users on the same computer, log in to each user account and perform the steps above—or distribute a copy of the newly modified com.apple.desktopservices.plist file to the ~/Library/Preferences folder of other user accounts.

These steps do not prevent the Finder from creating .DS_Store files on the local volume, and these steps do not prevent previously existing .DS_Store files from being copied to the remote file server.
