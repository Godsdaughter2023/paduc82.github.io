How to suppress flashing taskbar icons. - Microsoft Community

roger.harris say:

How to suppress flashing taskbar icons.

I want to completely suppress the "feature" of flashing taskbar icons.
cmdrkeene say:

If a program tells the taskbar to flash, the taskbar is obligated to do so.  There used to be a registry setting to control this, but companys and the government complained that they wanted their programs to be able to force the flash when needed (so as to force users to notice a pending message, chat, etc).  Microsoft engineers obliged.

 

 

If this solves your issue, click the 'Propose as Answer' link below, and vote as helpful by clicking the green triangle to the lelft. Thanks!

divya r say:

Hi roger.harris,

 

-      Does the icons of all the applications flash? Or is it any particular application icon that keeps flashing?

-      Was this issue occurring earlier too? If not, what’s the recent change that you made to your computer after which this issue has started to occur?

The icon in the taskbar flashes when a program needs attention. The problem is that it does this when one program wants attention while you are working with another application.

Follow this step and check if it fixes the issue.

 

Important: This section, method, or task contains steps that tell you how to modify the registry. However, serious problems might occur if you modify the registry incorrectly. Therefore, make sure that you follow these steps carefully. For added protection, back up the registry before you modify it. Then, you can restore the registry if a problem occurs. For more information about how to back up and restore the registry, click the windows help article below.

Back up the registry
http://windows.microsoft.com/en-us/windows7/Back-up-the-registry


 

a.    Go to Start and type regedit.exe.

b.    Navigate to the following location: HKEY_CURRENT_USER\Control Panel\Desktop.

c.    Double click the 'ForegroundFlashCount' key. Change the notation to decimal.

Note: If you enter 0, the taskbar icon will blink indefinitely until clicked.

d.    Hence set an value other than 0. Click OK.

e.    Restart your computer and check if the issue persists.

 

For more information, refer:

ForegroundFlashCount

    increaseuserva Megabytes Specifies the amount of memory, in megabytes, for user-mode virtual address space.

    On 32-bit editions of Windows, applications have 4 gigabyte (GB) of virtual address space available. The virtual address space is divided so that 2 GB is available to the application and the other 2 GB is available only to the system.

    The 4-gigabyte tuning feature, enabled with the increaseuserva option, allows you to increase the virtual address space that is available to the application up to 3 GB, which reduces the amount available to the system to between 1 and 2 GB. The BCDEdit /set increaseuserva Megabytes command can specify any value between 2048 (2 GB) and 3072 (3 GB) megabytes in decimal notation. Windows uses the remaining address space (4 GB minus the specified amount) as its kernel-mode address space.

    Please follow the method mention below to further narrow down the root cause of the issue.

     

    Method 1: Delete increaseuserva.

     

    1. Open command prompt as administrator.

    2. Type in bcdedit /deletevalue increaseuserva and hit enter.

     

    Method 2: System restore.

     

    If you have a restore point on the system, you can revert the system back to previous working condition.

     

    I suggest you follow this article to perform System Restore, look for the topic to restore your pc to an earlier point in time from this article and follow the steps.

     

    The System Restore will restore your Windows back to an earlier point in time, called a restore point. System Restore doesn't change your personal files, but it might remove recently installed apps and drivers.

     

    Method 3: Create a new user account.

     

    This issue could also occur when there is a user account corruption or an account is converted to temp account, in this case you can try to create a new user account and check if the issue still persists on the new user account.
