KB2687623 solves our problem with slow Outlook calendar . (pardon my english, trying to get this post recognized by search engines so other people with slow Outlook calendar problems can find this post)

We've found out the following:

When we're installing Office 2010 (without SP1) and update it by installing all updates simultaniously, after installing all updates, Outlook 2010 is running version 14.0.6117.5001 (which does not include KB2687623).

When we than try to install KB2687623, we get an error telling us there is something wrong with the "detection". So no dice to install the update.

 

However, if we first install SP1 (and reboot of course) and than install the remaining updates, Outlook 2010 is running version 14.0.6126.5000 (which includes KB2687623)

On a side note: after installing SP1, there are more updates found than before installing SP1 (which is strange, because you'd think SP1 makes several updates obsolete)

 

So, the work around at the moment is to de-install SP1 and all consecutive updates, reboot, reinstall only SP1, reboot, and than install all remaining updates (KB2687623 should be shown in your update list)

 

Which is quite a hassle, so I hope Microsoft can come up with a solution for this.

 

(For the die hards: It's of course also possible that it's not KB2687623 per sé which solves our problem, but it does show that there is something wrong with the Microsoft update mechanism when you reach a certain software level on Office 2010)
