I'm not familiar with EraseData by Tech Ember, but taking the company's web site at face value, it says that the program has 4 options:

  1. Full Data Erase:  Completely erase all data on the hard drive including the operating system
  2. Erase User Data:  Erase personal data but keep the operating system intact
  3. Erase Free Space:  Permanently erase previously deleted files
  4. Erase Mobile Device:  Erase data from an iPhone or iOS device

You apparently chose option 2.  If you didn't also choose option 3, you should at least do this. 

For the reasons I discuss in detail below, I recommend that you choose option 1 (which is the same thing that Darik's Boot and Nuke does -- for free, instead of having to pay for EraseData).  Option 1, of course, will remove Windows along with everything else.

It's not at all clear what option 2 really does.  I suspect this option is rather incomplete. 

In Windows 7, each user account includes by default (that is, aside from anything you may manually create) at least 10 folders.  These are stored (by default) under C:\Users\<username>\  (where <username> is the account name, e.g., your first name).

  1. AppData
  2. Desktop
  3. Documents
  4. Downloads
  5. Favorites
  6. Links
  7. Music
  8. Pictures
  9. Saved Games
  10. Videos

The least that option 2 should do is to delete the contents of folders 2 through 10. 

Folder 1 presents an interesting case.  This normally-hidden folder includes many subfolders, probably at least one for every company whose software is or has been installed on your computer. 

That raises the question of installed software.  Strictly speaking, these are neither "the operating system" nor "personal data."  So does option 2 of EraseData delete installed programs?  My guess is that it does not, although you can easily confirm this one way or the other.  If installed software is not deleted, it is likely that the AppData folder is also not deleted.  There will likely be personal information buried in AppData and its subfolders.

In addition to the AppData folder, some software creates additional folders under C:\Users\<username>\.  Particularly troubling are "cloud storage" applications such as iCloud and Dropbox.  There may well be copies of your personal files in these folders.  I've also seen a "Contacts" folder here, which will include information about people you know.

And in addition to folders in C:\Users\<username>\ , some software creates yet more folders in the "root directory" e.g., C:\.

It's seems pretty clear to me that option 2 of EraseData at least leaves the user file structure intact:  you still see your name when you log on.

Let's assume that EraseData considers "User Data" to include installed software in addition to the contents of the named folders in each user account.  Let's also assume that EraseData is smart enough to delete all folders under C:\Users\<username>\ except the 10 folders listed above.  That still leaves the possibility that someone or some software has created a folder or folder structure in the root directory.  So let's make one more assumption:  EraseData is smart enough to delete all folders (and subfolders) in the root directory (C:\) that aren't there in a default installation of Windows.

The plus side of leaving the operating system intact is that you can give the computer away to someone and they won't have to buy a copy of Windows. 

The negative side of leaving the operating system intact is that even with all of the assumptions above, there is still personal information buried in the Windows operating system.  Both your name and email address(es) are almost certainly somewhere in the Windows Registry.  In addition, passwords and program license keys are embedded in the Windows Registry -- and much of this information is easily retrievable by using utility programs readily available on the Internet. 

To give you just one example that doesn't run afoul of Microsoft's policy of not providing advice concerning how to bypass passwords, you can install "keyfinder" software that will dig the product keys for many (not all) installed application programs out of the registry.  See https://www.lifewire.com/free-product-key-finder-programs-2625119

For all of these reasons, if you want to be really sure that there's none of your personal data left, you need to "nuke" the entire disk and make whoever gets the computer install their own operating system.

The "Read-Only" attribute on the properties page does not apply to folders.  Instead, it means that special properties have been set up for the folders.  For further information, read this article:

"You cannot view or change the read-only or the System attributes of folders in [...] Windows 7"

  < http://support.microsoft.com/kb/326549/en-us >

Your problems seem to center around your USB Pen drive. Flash drive technology has a limited number of Write cycles.  After a while, it simply "wears out".  Most pen drives have extra memory blocks built-in and as memory blocks fail, they are replaced by these spare memory blocks.   When all the spare memory blocks are used up, then the pen drive becomes "Read-Only" by design.  It sounds like your pen drive has reached its end-of-life.
