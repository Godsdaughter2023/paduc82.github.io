lemp say:
The only thing I can think of that might cause a situation such as you describe requires some rather unlikely factors:  a static IP address assigned improperly plus an abnormally short DHCP lease time or a too-small IP allocation range also coupled with  an abnormally short DHCP lease time.

Other than the router power outage example that Tricky described earlier, the most common reason for duplicate IP addresses is that some device on the network is configured to use a static (fixed) IP address that is in the range of IP addresses used by the router to assign addresses automatically.

Network admins often configure devices that all users will want access to (such as network printers) to have static IP addresses, but the correct way to do this is to pick an address that's outside the allocation range.  For example, if the router automatically assigns IP addresses between 192.168.1.100 and 192.168.1.150, you would configure a printer with a static IP address such as 192.168.1.25.

An IP address conflict caused by the use of a static IP in the wrong range will arise when the router automatically assigns an IP address that's the same as the pre-assigned static IP address -- and that shouldn't happen at a precise 30 minute time delay.  The only thing I can think of is that the router has the DHCP lease time set to something slightly more than 30 minutes.  This would be very unusual; the default DHCP lease time on home routers is 24 hours.

On your computer, go to Network Connections, right click on your wireless adapter and select Properties.  On the "General" tab, select "Internet Protocol (TCP/IP)" and click the Properties button. The normal situation is to have both "Obtain an IP address automatically" and "Obtain a DNS server address automatically" selected.  What does your setup show?

Do you have any printers or other non-computer devices connected to your network?  This does NOT include printers or other devices physically connected to a computer and then shared over the network.  If you do have a network printer, use its front panel controls to print a "network status page" and see if it is configured to obtain its IP address automatically or if it uses a static IP address.

There are also a few things to check in the router's configuration, if you have access to that.  I would look at
  • the range of IP addresses that the router uses (maybe it was originally set up when there were fewer users and only has a range of 10 addresses, so if all 12 of you try to connect at once there will be a problem)
  • the client lease time
  • the DHCP client table (which will show all of the devices connected to the router)

If you identify the make and model of the router, I should be able to tell you where to find that information.
