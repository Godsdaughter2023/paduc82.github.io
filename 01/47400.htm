cmdrkeene say:

Remember that gmail is a special type of service that doesn't work like any other email services ever in history.

Gmail doesn't have folders, it has labels.  Sure it "looks" like folders, but a message can be tagged with unlimited labels.  When viewed through a regular email program, the same message will appear to be duplicated in all the folders for which it is tagged.

Gmail.com also has options for "what to do when another email program marks a message as archived or deleted".  You can pick if you want Gmail to just remove the label (thus causing the message in question to fall into the "All Mail" folder), or in the cases of deletes, whether or not you want it to go to the same "All Mail" or truly delete it (Gmail really tries to convince you to never delete, that's the goal of Gmail is to archive it all and find it again later).

I have mine setup so that archiving in Windows Mail puts it in the Archive folder (in the mail app) but I have Google's IMAP settings linked that "folder" to the All Mail folder on the gmail web interface.

frederik long say:

I assume this is a wired connection. Here are the first few steps I would take:

  • Check in the Control Panel / Network and Sharing Center / Change Adapter Settings if the adapter is enabled and if the cable is plugged in. You need to set the View to Details to see these items.
  • Take the CAT5 cable from one of the working devices and use it for the problem device. Make sure to use a router port that you know to work.
  • Mark the following command with your mouse, then press Ctrl+C to copy it.
    cmd.exe  /c  ipconfig  /all  >  "%UserProfile%\Desktop\ip.txt"
  • Click the Start globe and press Ctrl+V to paste the command, then Enter to run it.
  • Open the file IP.txt on your desktop and report the following items:
    - DHCP enabled
    - IPV4 address
    - Subnet mask
    - Default gateway
    - DNS server
