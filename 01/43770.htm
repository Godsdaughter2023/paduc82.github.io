It depends on how they deploy Windows in the 1st place and it varies depending on the version of Windows.

See Bill's links on Volume licensing.

They can distribute images which contain the key in an answer file included with the image, manage licenses through a key management server or just enter the key during a single installation. It depends on the individual circumstances.

On the Side question, not necessarily. An image is the snapshot of the entire computer drive at a fixed point in time so could be termed as being a backup but with limited flexibility.

Properly managed backups can be selective, only storing the data files you need without including stuff you don't and sequentially managed so you can restore a specific version of file or files from particular points in time. Quite an old computer but I have a same generation HP desktop with a similar motherboard and chipset that is currently running Windows 7 without issues. The biggest difference is that mine has a discrete PCI-E graphics card. Your integrated graphics might remain a sticking point but you will need proper chipset drivers (or at least working ones) regardless in order to get the bulk of the system working as well as it can.

HP typically only provides drivers for the OS that came on the computer (XP) and possibly for OS upgrades that HP officially supports for that computer (Vista maybe?).  If HP does not have drivers for the OS you want to install and if Windows does not happen to already include the drivers you need, you will have to find all of the missing drivers yourself from the individual component manufacturers .. ie. graphics(-Intel), modem(?), audio(-Realtek), network card(-Realtek), etc..

First however, this link from Intel should update the chipset drivers for Windows 7. After you download the .exe file, right click on it and select Properties. Set the compatibility mode to Windows 7 and "run as administrator". Double click to start the installer and reboot when prompted.

https://downloadcenter.intel.com/Detail_Desc.aspx?agr=Y&DwnldID=20019&lang=eng&wapkw=infinst_autol.exe

During the reboot Windows should hopefully start to detect the chipset components by name (rather than as generic hardware). As it detects the chipset components it will also begin detecting other installed peripherals by name. It will need a working internet connection before it can automatically download and install new drivers.

You might still have to locate a compatible graphics driver, etc. yourself. If there are none available for your integrated graphics in Windows 8 you can try using Windows 7/Vista or even Windows XP drivers, in that order of preference. Set the compatibility mode of the driver installer to whatever OS version the driver was intended for. The driver will either work in Windows 8 or it won't. Remember to save all of the working drivers that you've downloaded onto a CD or DVD in case you ever need to reinstall in future. These things do tend to disappear off of the web over time. In the worst case scenario you can still buy an add-in video card.

Your motherboard apparently does not have any AGP or PCI-Express slots so if Intel does not have drivers that can be used for the on-board graphics chips then the only option for an add-in video card would be to find one that is PCI. Those have been rare for years already but there are still a few available. They tend to have lower-end and older GPUs, like the Radeon 3450 Graphic Card - 512 MB - PCI card so be careful about Win8 driver compatibility for those before buying one.
