dand76 say:

This is what worked for me.  (Windows 7 Pro, Lenovo R61 with Intel 82566mc adapter)

I was lucky enough to be in the process of rebuilding 2 R61s with Windows 7.  One laptop was working fine, the other the network adapter would fail with the code 10 error.

First - temporary fix - shutdown, unplug power, unplug battery.  Reconnect power, battery and restart.  Network adapter will work for awhile.  Usually until the next restart.

 

Permanent fix that has been working for me so far: 

Look at the Properties of the 82566mc adapter, Driver tab (don't go to Driver Details yet)

The laptop that was working fine had the driver:  Driver Provider: Microsoft, version 9.13.4.10.

The laptop that was failing had the driver:  Driver Provider:  Intel, version 9.13.41.0  (didn't write it down but I think that was the version#).  Intel being the provider is what to look for.

 

Then under the Driver Details:

On the working laptop:  Driver files:  c:\windows\system32\drivers\e1e6032.sys   (note: Provider here says Intel)

On the failing laptop:  Driver files:

c:\windows\system32\drivers\e1e6232.sys

c:\windows\system32\e1000msg.dll

c:\windows\system32\e1e6232.din

c:\windows\system32\NicCo36.dll

c:\windows\system32\NiclnE6.dll

 

On the failing laptop, I deleted the 5 files listed above, shut down the computer, unplugged power and battery, reconnected power and battery, restarted the computer. (unplugging power/battery here is probably not necessary.  Had become routine while troubleshooting this for hours:-).

When the computer restarted it picked up the Microsoft driver and has been working fine thru several shutdowns/restarts.

This worked for me.  Hopefully others also.

APC_INDEX_MISMATCH (1)

This is a kernel internal error. The most common reason to see this
bugcheck is when a filesystem or a driver has a mismatched number of
calls to disable and re-enable APCs. The key data item is the
Thread->KernelApcDisable field. A negative value indicates that a driver
has disabled APC calls without re-enabling them.  A positive value indicates
that the reverse is true. This check is made on exit from a system call.
Arguments:
Arg1: 82cb6cd5, address of system function (system call)
Arg2: 000000ff, Thread->ApcStateIndex << 8 | Previous ApcStateIndex
Arg3: 00000000, Thread->KernelApcDisable
Arg4: 00000000, Previous KernelApcDisable

Those are strange BSOD errors. Make sure you have the latest drivers installed and plase check for a BIOS update.
