imran chand say:

Hi shanu-s,
1. What kind of backup are you doing?
2. How are you trying to back up?
3. Is it on an external drive or on an internet drive or to a CD?
4. Is the backup getting initialized?
On my research, I found that many users have come across similar kind of error and they managed to fix it by making few changes in the registry.
a. Click on Start, type regedit in the search box.
b. On the Registry Editor window, navigate to the below key:
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsBackup\ScheduleParams backup flags value 1
c. Make sure backup flags value is set to 1 under ‘ScheduleParams’.
d. Exit Registry editor.

    samc1 say: Did the camera work with Windows 7 before you had this problem?
    If it never did, this is most likely a driver problem and you must obtain a Windows 7 driver from the manufacturer (presumably from Acer).
    If the camera worked before and having a problem now, error 43 is just a generic device failure indication. The recommended resolution is running Troubleshooting Wizard and checking the necessary connections/wiring if this is applicable. It is also worth going into the Device Manager, delete the existing driver, and force it to reinstall by scanning for new hardware (which you've most likely tried). If the problem is established to be Windows 7 driver, the device can be disabled in the Device Manager until the driver is made available.
    I had a similar problem with a USB webcam and the problem was soon resolved by downloading and installing a compatible driver.
