The inability to delete a PDF file was also a problem in Windows 8 and continues in Windows 10. The problem is associated with use of the Preview Pane in Windows Explorer. There are 2 workarounds:

This bug only affects PDF files. Other file formats delete without issues in Windows Explorer.

You then need to analyse the list of errors and decide where you are going to get the missing files from. If you have access to another Windows 7 computer similar to the one you are about to repair then that can be the easiest way forward. If the computer being repaired is 64 bit then you need another 64 bit computer. Also match version e.g. Ultimate to Ultimate if you can.

 

The first 30 errors all are to be found in the winsxs\manifests folder. The full file path is c:\windows\ winsxs\manifests. Open Windows Explorer in the computer you are obtaining the files from, using the keyboard shortcut Win+E and place the cursor on C. Click the arrowhead pointer to the left of C to expand and keep repeating the procedure to expand windows\ winsxs\manifests to find the first file. Copy each file in turn to a flash drive.

 

You now need to follow the procedure described in KB2700601. Copy the files to the %WinDir%\Temp\CheckSUR directory of the corrupted computer in the following subdirectory format and then re-run the System Update Readiness tool:

•All files of type *.mum and *.cat should be placed in the %WinDir%\Temp\CheckSUR\Packages directory

•All files of type *.manifest should be placed in the %windir%\temp\CheckSUR\manifests directory.

 

The remaining errors all relate to Security Update for Windows 7 for x64-based Systems (KB3003743):
http://www.microsoft.com/en-gb/download/details.aspx?id=44776

 

A. Download and save the package that contains the missing files -Windows 7 for x64-based Systems (KB3003743)

B. .Copy the Windows 7 for x64-based Systems (KB3003743) to the Packages folder under %WinDir%\Temp\CheckSUR. You may need to create the directory.

C. Re-run the System Update Readiness tool.

 

Good luck.
