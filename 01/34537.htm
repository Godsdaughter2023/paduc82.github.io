tryx3 say:

Then I can only suggest that malware has installed a utility that suppresses the UAC setting when Windows loads [in the hope that you do not notice the change and that the malware can therefore go further by installing other malware without bringing up an Admin challenge dialog box].  This symptom was reported some time ago and eventually diagnosed as a malware symptom - see Windows 7 UAC (user access control) is being disabled on every reboot

In addition to a full scan with your installed anti-malware utility, download and run a scan with Malwarebytes Anti-Malware Free [MBAM].  It is much harder for malware to hide its presence from two different antimalware scanners.  MBAM can exist alongside other antimalware utilities so it will not cause any problems. 

Of course, it might be a system fault instead but it is not one that I have seen reported in this forum before.  Windows has no built-in procedure that alters the UAC setting so it is difficult to envisage any procedure going wrong in such a way as to result in that symptom.
