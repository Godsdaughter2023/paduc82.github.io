The most significant difference between the ATA port driver and other system-supplied storage port drivers is the protocol that the ATA port driver uses to communicate with other drivers. All other system-supplied storage port drivers use SCSI request blocks (SRBs) to communicate both with higher-level drivers, such as storage class drivers, and with miniport drivers. Driver Verifier:

What is Driver Verifier?

Driver Verifier is included in Windows 8, 7, Windows Server 2008 R2, Windows Vista, Windows Server 2008, Windows 2000, Windows XP, and Windows Server 2003 to promote stability and reliability; you can use this tool to troubleshoot driver issues. Windows kernel-mode components can cause system corruption or system failures as a result of an improperly written driver, such as an earlier version of a Windows Driver Model (WDM) driver.

Essentially, if there's a 3rd party driver believed to be at issue, enabling Driver Verifier will help flush out the rogue driver if it detects a violation.

Before enabling Driver Verifier, it is recommended to create a System Restore Point:

Vista - START | type rstrui - create a restore point
Windows 7 - START | type create | select "Create a Restore Point"
Windows 8 - http://www.eightforums.com/tutorials/4690-restore-point-create-windows-8-a.html

How to enable Driver Verifier:

Start > type "verifier" without the quotes > Select the following options -

1. Select - "Create custom settings (for code developers)"
2. Select - "Select individual settings from a full list"
3. Check the following boxes -
- Special Pool
- Pool Tracking
- Force IRQL Checking
- Deadlock Detection
- Security Checks (Windows 7 & 8)
- DDI compliance checking (Windows 8)
- Miscellaneous Checks
4. Select  - "Select driver names from a list"
5. Click on the "Provider" tab. This will sort all of the drivers by the provider.
6. Check EVERY box that is [B]NOT[/B] provided by Microsoft / Microsoft Corporation.
7. Click on Finish.
8. Restart.

Important information regarding Driver Verifier:

- If Driver Verifier finds a violation, the system will BSOD.

- After enabling Driver Verifier and restarting the system, depending on the culprit, if for example the driver is on start-up, you may not be able to get back into normal Windows because Driver Verifier will flag it, and as stated above, that will cause / force a BSOD.

If this happens, do not panic, do the following:

- Boot into Safe Mode by repeatedly tapping the F8 key during boot-up.

- Once in Safe Mode - Start > type "system restore" without the quotes.

- Choose the restore point you created earlier.
If you did not set up a restore point, do not worry, you can still disable Driver Verifier to get back into normal Windows:

- Start > Search > type "cmd" without the quotes.

- To turn off Driver Verifier, type in cmd "verifier /reset" without the quotes.
・    Restart and boot into normal Windows.

How long should I keep Driver Verifier enabled for?

It varies, many experts and analysts have different recommendations. Personally, I recommend keeping it enabled for at least 24 hours. If you don't BSOD by then, disable Driver Verifier.

No Power

One or two of the speakers do not work

2. Check the connections from the back of the computer to the speakers. The speakers should connect to the line-out jack on the back of the computer.

Crackling Sound
1. Check the cables and make sure they are firmly connected between line-out plug on the back of the computer and the speakers. If you have powered speakers and connect them to the "speaker out" connector you can get Crackling.
2. Also, check the volume control in Windows and make sure that it is not set more than half-way

Radio Feedback
1. Move the speakers in a different location or rotate them to see if it helps. 
2. Buy cables that have better shielding.

Static Noise
1. Make sure you are using the Line-out plug on the back of the computer.

Popping Sounds
1. Check All speaker cables for proper connections.
2. Move the speaker wires away from other cables.
3. Plug the speaker AC adapter into a different outlet
4. Buy a pack of ferrite beads that attach to your speaker wires which filter out noise

awgie dawgie say:

Don't feel too bad.  I've searched for a lot more than 3 hours and didn't come up with any more than you did.  The fact that you searched for 3 hours is a great credit to your character.   

The  Disk Cleanup tool in Windows Vista has the same effect as deleting the thumbnail cache files manually, which believe it or not, I knew how to dobefore I ever ran the Cleanup tool.  It takes you right back to square one, and you have to rebuild the entire cache from scratch.  Your suggestion to "open each folder once and let it run overnight to rebuild the cache while you are sleepting" is quasi-humorous, but not feasible.  The thumbnails are only generated once the files become visible, so you'd have to not only open each folder (I have probably a few hundred), but you would also have to scroll all the way through each of them.  If that were all there was to it, it would still be a daunting task.  However, as soon as you browse away from the folder, the thumbnails stop loading for that folder, so you would have to have an open Explorer window for each folder, and you'd have to have scrolled through every one of them.  Even if I thought my system could handle a hundred active windows at once, I'm not entirely certain my brain could, so I think that option is out.  My thumbnail cache is 1.76GB.  Imagine how small a thumbnail's footprint is, and then divide that tiny little number into 1.9 TRILLION bytes, and you see why I don't want to delete my cache.  I am presently in the process of reorganizing my photo library, that extends back nearly 20 years.  Perhaps when I finish that, I will rebuild my cache, as I'm sure there are dead entries in it due to my moving and deleting files.  Of course, had I realized 20 years ago how big my library could get (I'm a bit of a shutterbug), I would have organized it better from the start.

I had thought of a rationale, albeit a faulty one, for Vista's not having the Refresh Thumbnail option.  In XP, every folder's thumbnail cache was contained in a separate system file called thumbs.db, and stored in that folder.  Vista stores all the thumbnails in one central location.  Now anyone who didn't think about it too much could accept that because the cache is stored in one location, Windows can't simply go in and delete the old entry and generate a new one.  That is theonly excuse I can imagine them having for their removing a feature, when progress typically dictates adding features.  The trouble with that argument is that Windowscould do exactly that.  If each thumbnail were alotted a set number of bytes, the old thumbnail could simply be overwritten by the new one, so Windows would not have to re-save 1.76GB, but could simply rewrite a few KB.  We were doing direct access reads and writes to disks 30 years ago - you know they can do them now.  Obviously, Windows can access the thumbnails in a very short amount of time when it loads a folder, so it doesn't have to be a problem.  And yet somehow it is.

Your third suggestion is a free plug-in from a line of software called Real World - that's what the "rw" in the URL stands for - which I find ironic, since for anyone running even Windows XP, most of the image types the plug-in can extract thumbnails for are already handled as standard thumbnails.  There are only a few specialized types for which I could see it being useful.  Of course, it's the whole range of software programs that is named "Real World", so it's not really a misnomer.  It's mention of there being no Refresh option in Vista, and of the Cleanup tool, is just letting you know that deleting the cache is the only provision Windows Vista makes for rebuilding thumbnails.

You may have noticed I skipped over the second option.  I did that to save the best for last.  I marked your post helpful (you're welcome) for the second option.  No, my question was not answered by any means, but it did help me solve another problem that hadn't irritated me enough to ask about.  All of my video files show thumbnailsexcept MP4's, which I found strangely annoying.  More than just strangely annoying in fact, if I browsed to an MP4 on an external drive, and then tried to safely remove that drive two hours later, Windows would tell me it could not stop my volume, or some such bilge-water.  Upon examining Process Explorer, I discovered that Explorer still had a handle on every single MP4 I had browsed to, even if I never opened them or even clicked on them.  To eject the drive, I had to kill Explorer and then remove the drive, and then restart Explorer.  Turned out that somehow an improper value had been assigned to the thumbnail handler in the Registry.  I put the correct value in, and everything was beautiful again with all my MP4's -something for which I am quite happy, since my camera stores video in MP4 format.  So again, thank you.

And if that weren't enough, I also marked it helpful for a second reason.  The article you referenced is a wealth of useful information.  It shows you how to turn thumbnails on and off manually for images or video (you have to do it manually if you want thumbnails for one but not the other).  It shows you how to enable thumbnail views on a per-filetype basis.  Lots more if you are still running a machine on XP, but would like your thumbnails to look more like Vista.  He even includes a link to the K-lite MegaCodec Pack, which is essential to seeing thumbnails for many video formats, such as Flash video, DivX, etc., not to mention being equally important to being able to watch all those videos in Windows Media Player, instead of needing three or four different players. 

But besides thumbnail issues, he includes links to other useful software, like the best free image viewing/manipulating program I have ever used, and have been using for many years - IrfanView.  There is a file property and attribute editor called Properties Plus.  According to the readme file, the program is 12 years old, but it still works beautifully for modifying file date/time.  He also includes both of the only two free utilities I have found so far that allow the user to change the time of day a photo was taken.  Windows Vista has a built-in EXIF editor that allows you to change the date taken, but will not let you change the time.  Oh, I could keep going - I'm only halfway through his post - but if I keep going on his post, I won't get to my surprising discovery.  Which is...

Back to the original issue...

I did come up with two different work-arounds that would help with the problem, even though I have not found the solution I am really looking for yet.  And I thought of them both in the middle of typing this message.  Which method to use depends on whether you've viewed the folder containing the file(s)

  • If you have already viewed the folder and suddenly realized too late that the extensions are wrong: First, fix the extensions.  If you have a lot of files to fix, like I did, you can use a utility to speed things up.  I use Bulk Renaming Utility.  Once the extensions are corrected, select the file(s) and using theright(secondary) mouse button, drag the files back onto themselves, and select Copy Here from the pop-up menu.  What you'll get are exact copies of the files in the same folder, except the copies will generate a new thumbnail because they are new files.  The problem is, once Vista has assigned a thumbnail to a file, it will not assign a new one.  That's where the Refresh Thumbnail option in XP was so useful.  So from my own example, if the extension is .html, then Vista says "there is no thumbnail, so just use the default icon for that file type."  If I simply change the extension, Vista notes that change in the thumbnail cache, so it still gets just the default icon.  By copying the files, I create new files, which Vista will then correctly create thumbnails for.  Then, I can delete the original files, and use Bulk Renaming Utility to remove the " - Copy" suffix from the corrected files.  The downside to this, apart from the fact that it's cumbersome, is that the new files will all have the date and time you created the copies, not the original file date and time.  The copies should retain all of the EXIF data from the originals.
  • If you have not viewed the folder yet in Explorer or in any file dialog such as a Save or Open dialog, it is much easier.  Open a command prompt (for those joining the broadcast already in progress, click on the Start button, then in the search box, type "cmd" (without the quotes) and press enter).  If you started it in Administrator mode, the prompt will say "C:\Windows\System32>"  If you started it in standard mode, it will read "C:\Users\your_name>"  Where your_name actually would show your user name for Windows.  Navigate in the command prompt window to the folder containing the files with the incorrect extensions.  For example, if the files are in a subfolder called "Blizzard of 2010" under your Pictures folder, and you are in your user folder, type "CD Pictures\Blizzard of 2010" and hit enter.  If you are in the System32 folder, type "CD\Users\your_name\Pictures\Blizzard of 2010" instead.  From my own example again, the incorrect extension of all the files is .html, and it should be .gif, so at the command prompt, type "RENAME *.HTML *.GIF" and press enter.  If there was only one folder to fix, you're done with the command window and can type "exit" and press enter to close it.  Because you had not yet viewed the folder in Explorer, Vista had not yet handled the thumbnails for the files, and so it can generate them correctly the first time.  No files to erase, and no files that need to be renamed a second time.  And the files retain their original created and modified dates.
  • Alternate to option 1 - don't change the file extensions first.  Instead do option 2, but in place of "RENAME *.HTML *.GIF" type "COPY *.HTML *.GIF" - by copying from one extension to the other, and without specifying a different destination, you will create copies already having the correct extension.  Then you can use Explorer and delete the files with the wrong extension, and you won't have to rename them a second time to remove " - Copy" from the end, because it's not there. 

No, it's not a perfect substitute for the Refresh Thumbnail shell extension that was there in XP, but it still beats a poke in the eye with a sharp stick.  And for my own purposes of having downloaded ten years' worth of daily comics - all 3,699 files being perfectly good GIF files with the wrong extension - and considering I am equally comfortable operating in DOS as in Windows, the second option I just proposed would have been just as easy as highliting all the files in Explorer and hitting Refresh Thumbnail.  But that's just me.  Now ask me why I didn't think of it three months ago when I really needed it.

You ever have one of those days when you're driving across town, and you suddenly remember where in the garage you put the flashlight that you searched for all night when the power was out last Christmas?  Well, that's how I feel right now.  I've been racking my brain for three months and coming up empty, and now poor Lorien has gotten in on the act, and suddenly in less than an hour I came up with the first two options, and as soon as I posted it, I opened it right back up and added the alternate.  I could just kick myself - but my foot won't reach my head, so why bother?
