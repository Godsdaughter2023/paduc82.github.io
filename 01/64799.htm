In order for a DVD to play in a standard DVD player, it must be structured in a certain way.  The video must be encoded in MPEG2 format stored in .vob files.  And if you are adding menus, you must have various .ifo and .bup files.

 

So you have a couple problems:

  1. You are using a CD disc, not a DVD disc.
  2. You are not creating a video DVD in the appropriate format.  You cannot use the 'Save to CD' option from Movie Maker 2.1 to make a DVD.

Unfortunately, Windows Movie Maker 2.1 does not have any ability to create a video DVD.  Nor is there any other software in Windows XP designed to create a DVD.

 

Very often, if you have a DVD burner in your PC, it will come with software to make a video DVD.  In that case, you can save your movie to your computer (from Movie Maker) and then use that software to make a video DVD.  Alternatively, you might look at freeware options like DVD Styler or DVD Flick.  Since this is outside the realm of Windows Movie Maker features, however, you are on your own to watch tutorials or figure these steps out.

First, you can't just burn the movie file onto a DVD disc.  It has to follow the DVD standard layout in a VIDEO_TS folder with the contents I mentioned.

Here's what you can do when folders get screwy...  (and no apologies necessary, as no offense was taken... but thanks for the courtesy anyway!) 

First, refresh the folders on ALL clients that have an IMAP connection.  Make sure they ALL have the same folders downloaded to them.  Doing this will vary based on the mail client making the connection.  There are too many client programs to potentially explain them all effectively here.

And if you're using a perl, python script, service or other program to 'manage' your e-mail (fetchmail, etc) then make sure those are all using the same folder naming structure.  

If it's one of the 'special folders' then make sure that ALL client programs using an IMAP connection are all using the SAME names for the folders.  Sent Items, Junk, Drafts, whatever.  Don't have one using Sent while another uses Sent Items, while others use Sent-Items, Sent-Mail, sent_mail, sent.mail, etc...  Case matters... sometimes, so be consistent.  Make sure they're all pointing to the same folder names.  

Once they're all configured the same, quit them entirely.  Don't have anything leaving an IMAP connection open.

Then Use the Outlook.com web interface to fix the folder names.  If there's stuff in the wrong folders then move them using the web interface.  Once you're sure the correct folders are in place and the wrong ones deleted (remember to empty the Deleted Items folder, then reconnect the various IMAP clients.  Do a folder refresh on them again and confirm they're using the right folders.

IMAP has had a long history of 'uneven' implementations, both on the client and on the server side.  There's plenty of blame to go around.  But don't waste time on that, just 'cut to the chase' and get the folders renamed/deleted as your needs require.

If you genuinely see two folders with exactly the same names then you're going to have to move the contents out of both of them into a new folder, delete both the old ones, recreate the new "one" and move any items back into it. 

Notice I mentioned case being important?  That's potentially one place where an IMAP client could get it wrong.  Something creates "My Folder" but then another creates "my folder" and the server thinks they're two distinctly different ones.  They are, as their names are composed of different letters; upper and lower case.  Now, some clients will 'gloss over' this, while others won't.  Best to name them consistently, even better to avoid using spaces in their names (but only the oldest and most defective of IMAP clients or scripted programs would have a problem here).

IMAP is a remarkably durable software interface and I'm grateful Microsoft and others have continued to make use of it.  I'd be happier if they all used it consistently, but I'm willing to work-around when necessary.
