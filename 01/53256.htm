    Hi, Edward71:

    Files in System32 folder are protected by Windows Resource Protection (WRP). Basically, this means that only Microsoft via Windows Update, a Service Pack, etc. will be able to update those files. The protection is based on a "user", TrustedInstaller, that owns all of those files so that any other user, including an administrator, cannot replace or modify them.

    Are you sure that your Photoscreensaver.scr file is corrupted? From an elevated command prompt (Cmd.exe), type this command and hit ENTER:

    sfc /VERIFYFILE=%systemroot%\system32\photoscreensaver.scr
