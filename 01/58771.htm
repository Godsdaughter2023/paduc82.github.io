It rarely ever means the file is really corrupted or missing (unless it happened via human hands).  

Running chkdsk /r from the XP Recovery Console will straighten out the file system if nothing else is wrong and I would always start with chkdsk /r when these kinds of messages are encountered.

These can cause corruption in the file system which must be fixed before you do anything else.

If any of those events have occurred (or even if they have not occurred), you should verify the integrity of your file system before doing anything else (especially "trying" things).

It makes zero sense to start trying to start trying to edit or copy files around on a hard disk that has a corrupted file system so this must be fixed first using the XP chkdsk program, and running chkdsk /r on your drives may resolve your issue entirely.  

Boot into the Windows Recovery Console using a bootable XP installation CD.
