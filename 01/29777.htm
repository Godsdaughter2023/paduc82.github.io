Hi Georgie,

Thank you for getting in touch with Microsoft Community.

This issue occurs if you back up a library that contains custom folders that are located under your user profile. This issue may also occur when you are using USB devices other than storage devices.

In order to prevent backup from failing, you can use one of the following instructions to eliminate this error message during future backup operations.

Let’s exclude the folders from the files to back up in the Backup settings, and then include those folders in their original location. To exclude the folders in the Additional Locations and to include those folders in the original location, follow these steps:

  1. Press Windows + X key.
  2. Select Control panel.
  3. Double click on Backup and Restore.
  4. Click on set up backup button.
  5. Double click on the drive you want to Backup.
  6. Select let me choose option and click on Next button.
  7. Expand and navigate to the location,

Data Files > account user name Libraries > Additional Locations

  2. Expand Computer, expand the item for your system drive (for example, expand Local Disk (C:)), expand Users, expand account user name, and then click to select the AppData, Contacts, and Searches check boxes if they exist.
  3. Click Next, and then click Save settings and exit.
  4. In Backup and Restore, click Back up now to perform a new backup. The backup operation should complete without errors.

Hello Keshav,

This issue can occur when Send To OneNote 2010 printer driver is missing.

I would suggest you to install the generic printer driver in Windows and check if it helps. To do this, follow these steps:

a. Click Start, type printers in the search box, and then click "Printers" in the search results.

b. Click "Add a printer" and in the "Add Printer" dialog box, click "Add a local printer".

c. Click to select "Use an existing port" option, select the appropriate port and then click "Next". For example, you

can select "LPT1".

d. Under "Manufacturer", select "Generic", select "Generic/Text Only" under "Printers" and then click "Next".

e. When you see "Which version of the driver do you want to use?" message, click to select "Use the driver that is currently installed(recommended)" option and then click "Next". Click "Next" and then click "Finish."

Now, configure the generic printer for OneNote. To do this, follow these steps:

a. In the printer list, right-click "Generic/Text only printer" and then click "Properties".

b. Click "Ports" tab and then click to select "Send To Microsoft OneNote Port" check box.

c. Click "Advanced" tab, select "Send to Microsoft OneNote Driver" in driver list and then click "Print Processor".

d. Under "Print processor", select "OneNotePrint2010", select "RAW" under "Default data type", and then click ok.

e. Click "General" tab, change the name of the printer to "Send To OneNote 2010", and then click ok.
