You can remotely install games online so that the game is ready to play on the Xbox One console. There are a couple of things that you’ll need to do before you get started:

  1. Rename your console: this will help you to identify it easily.
    1. Press the Xbox button to open the guide, go to System > Settings > System > Console info.
    2. Select Name, then enter your new console name.
      Note Your console name may contain only letters, numbers, and hyphens.
    3. Select Restart now.
  2. Set your console to Instant-on: your console will need to be in standby mode for game installations to be triggered remotely.
    1. Press the Xbox button to open the guide, go to System > Settings > Power & startup > Power mode & startup.
    2. Select Power mode to change the setting to Instant-on.
  3. Enable Remote Install: this may be disabled on your console by default.
    1. Press the Xbox button to open the guide, go to System > Settings > System > Updates.
    2. Select Allow remote installs.

Now when you redeem a game, or even if you already own that game, you can remotely install it on your console:

  • From a web browser: On the store page for the game (click the link to go to the store page for Earth Defense Force), click the ellipsis next to the See Price button and select Install on my devices. Select your console from the list and then select Install now.

Since this is a backwards compatible game, you may not see this option from the Microsoft Store. If this is the case, then select See Price which will redirect you to the game's page on the Xbox 360 Marketplace (click here to get to the page). From here you can claim the game so that it is on your account, and then you can start the download from the Xbox's Ready to install section in your game library.

Chrome crashes for HomeDepot, Delta on HP Spectre x360 or Lenovo Yoga Convertibles

When accessing some commerce sites (HomeDepot.com, Delta.com among others) from Chrome, Chrome immediately crashes entirely - the whole browser crashes, not just the affected tab.  The problem affects mainly the HP Spectre x360 and Lenovo Yoga lines, although there are a few mentions of the MicroSoft Surface and other laptop/tablet convertibles.  Chrome has no problem reaching these sites from other laptop hardware platforms.

The problem has been isolated to the HID Collection V2 sensors.  If these are disabled (rotation and ambient light) there is some minor loss of functionality, but the sites do not crash.

The crash has been tracked to a double free in Windows sensorsAPI from JavaScript.  Chrome was able to ameliorate the problem with Chrome 71, but it still persists for many users.  Chrome is unable to fix the problem because the root cause is internal to the Windows API.  Edge can open the sites OK, but there's something related to these specific hardware sensors (Spectre x360, Yoga) that is causing Chrome to crash in the sensorAPI.  

The same problem was fixed by the Windows update to 1803 for the HP Elite x2 1012 G1, but persists with 1803 and 1809 for the Spectre x360 and Lenovo Yoga.

Details can be seen in Chrome bug CR789344.  The problem was first reported 11/24/17 for Windows 1511 and persists through 1809.  I'm a Chrome community volunteer, with over a year's notes on this problem and can provide additional pointers.  There are workarounds to avoid the problem, but these are tedious and cumbersome.  The root problem needs to be resolved by Microsoft.

Let me know if you've seen other reports of this problem, or have any tips on how to escalate it at Microsoft.  If you are aware of efforts or problem reports by HP or Lenovo to address this Chrome crash at commerce sites, also please let me know.
