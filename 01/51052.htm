To determine if there is a task set up for chkdsk, do the following.

chkntfs C:

chkntfs /x C:

Ensure you run this command on the devices that start chkdsk on reboot.

  1. Press Windows Key + X on the keyboard and then select “Command Prompt (Admin)” from the menu.

    • net stop cryptSvc
    • net stop bits
    • net stop msiserver
  3. Now rename the SoftwareDistribution and Catroot2 folder. You can do this by typing the following commands in the Command Prompt. Press the “ENTER” key after you type each command.

  4. Now, let’s restart the BITS, Cryptographic, MSI Installer and the Windows Update Services. Type the following commands in the Command Prompt for this. Press the ENTER key after you type each command.

    • net start cryptSvc
    • net start bits
    • net start msiserver

Now you may try running the Windows Updates and check if the above steps resolve the issue.

so.. what I basically do is have the person do a local install with USB like normal and then switch the printer to wireless or wired network, and on the computer add the standard tcp/ip port by ip to the printer ip address, if you can open the printer on the network through your browser by IP address.... that's the trick....

some things to consider...

1. make sure no other HP printers / software are installed... home users... typically one printer, one computer... if multiple hp old printers still installed can cause problems...

2. make sure the devices & printers only show the basic built-in printers....

3. download the hp printer package.. and #1 is verified, and #2 makes sense.... make sure printer USB unplugged... install package, do the custom and uncheck the hp overhead in the customization tab.... then select USB connected.... wait for the HP dialog to ask for the usb to be plugged in with printer on.... then......

4. plug the printer into their computer with a usb cable, and have the install finish.. do the test page... reboot... print another test page... then.......if all good...

5. unplug the usb cable.... with the printer go wireless or wired to the network...... get the network IP from the printer..... and bring up that IP address in your browser... and if the printer appears.... in your browser.. just go to the printer properties.... Ports..... add port... Standard TCP/IP.... type in the IP address// twice in each box.... let it finish.... print test page.... voila!!
