Definition of the Word "Rootkit" - Microsoft Community

microcode man say:

Definition of the Word "Rootkit"

(Sorry if this is the incorrect Topic, but at least it includes the word Definition ;-)
There are two competing definitions for the word; Rootkit

The first is:
- ROOTKIT = 1st: "conceal itself", 2nd: "malicious", performs actions detrimental to network, system, or user(s). It must to the do the 1st, and the 2nd is assumed.
     (In this definition the first part, root means somthing which lies below the surface.)
The second is:
- ROOTKIT = Software which includes Privileged Code, utilizing protected system resources and functions. The term originated with Unix System Utilities which supplied administrative functions not in the operating system. The current use is for both legitimate privileged software, and malicious software which gains and utilizes privilege through unauthorized means.
     (In this definition the first part, root means the privileged or administrative account on a Unix System.)

People who work with or for one the AV Utility organizations, use the second. They will commonly refer to a rootkit detection of a system modification made by a valid software package as a Legitimate Rootkit, and the detection as a False Positive. So they specifically differentiate between Legitimate Rootkits and Malicious Rootkits. The Virus Definitions for all utilities include a list of these Legitimate Rootkits to eliminate the False Positive detections.

It would appear that the dividing line might be IS Professionals and "lay people". But the Microsoft document, Understanding Anti-Malware Technologies, seems to be using the first definition. The intended audience is business people, but the expectation is it was written by an IS Professional.

Is there a Senior Security Analyst at Microsoft who might have actual experience back to the origin of this word. (No offense kiddies, but looking for some who actually fought in the "Little Indian-Big Indian war".)
