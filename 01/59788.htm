    That works fine and dandy if all you want to do is make a shortcut to an app.  However, if you need to open an app by command-line or in a script, it's a little trickier.  You have to get the activatable ClassID of the app you wish to launch.  While viewing "shell:appsfolder" in File Explorer, switch to the details view.  Then, press [Alt] to make the menubar appear.  Click View -> Choose details...  In the dialog, tick AppUserModelID and click [OK] (due to some bug, you can't enable this column by simply right-clicking the column-header).  In this column is the full app identification code.  You can launch an app by using its code like this:

Settings app:

"C:\Windows\explorer.exe" "shell:AppsFolder\{app code here}"

xbf amber m say:

Hi Star,

Thanks for replying. When it comes to any factory reset (soft = keep games & apps, hard = full removal), the process would remove any profiles on the console and they would need to be logged/added back onto the console. You would need to be sure to have the log in credentials (email address + password) for the profiles prior to performing any reset. And no worries about losing any games. Any games (or other items) associated with/owned by the accounts do not need to be repurchased and any game progress would be saved to the profile as well, but if you perform a full reset, you would need to of course install them again.

As previously stated by our Ambassador, you may want to locate a diagnostic tool for your external HD prior to doing any reset as the reset is solely for the console. It would be best to detach any external HD from the console prior to performing a factory reset. Additionally, you may want to hook that external HD up to a Window 10 device (like a laptop) to see if it is recognized. If not, then the external HD may have stopped working altogether.  The reasoning behind performing a factory reset to the console is to remove any potentially damaging files on the console's OS that may be causing the issue to begin with.
